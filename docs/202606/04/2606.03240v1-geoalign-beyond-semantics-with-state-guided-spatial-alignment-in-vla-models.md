## 摘要
当前的视觉—语言—动作（Vision–Language–Action, VLA）模型通常侧重于优化语义基础，而可执行的操作任务则需要具备几何感知的空间对齐与动态可供性选择。我们提出 GeoAlign，一种用于 VLA 策略学习的状态引导空间对齐架构。GeoAlign 使用机器人领域的 RGB-D 监督对 RGB 几何分支进行后训练，从而为策略展开生成由 RGB 推导的几何增强后训练（Geometry-Enhanced Post-Trained, GEP）特征。机器人的本体感知状态查询 GEP 特征网格，生成紧凑且依赖操作阶段的几何 token，用于动作预测。GeoAlign 在 LIBERO 上达到 99.0%，在三个 SimplerEnv-Fractal 任务上达到 85.3%，并在八个对几何要求较高的真实世界 ALOHA 任务上达到 78.8%；消融实验验证了几何后训练与本体感知状态引导查询的价值。

## Abstract
Current Vision--Language--Action (VLA) models often optimize for semantic grounding, whereas executable manipulation requires geometry-aware spatial alignment and dynamic affordance selection. We introduce GeoAlign, a state-guided spatial alignment architecture for VLA policy learning. GeoAlign post-trains an RGB geometry branch with robot-domain RGB-D supervision, yielding RGB-derived Geometry-Enhanced Post-Trained (GEP) features for policy rollout. The robot's proprioceptive state queries the GEP feature grid, producing compact, phase-dependent geometry tokens for action prediction. GeoAlign achieves 99.0% on LIBERO, 85.3% across three SimplerEnv-Fractal tasks, and 78.8% on eight geometry-critical real-world ALOHA tasks, with ablations confirming the value of geometry post-training and proprioceptive-state-guided querying.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 方法整体框架

GeoAlign 是一个两阶段框架：

1. **离线几何后训练阶段**
   - 使用机器人领域 RGB-D 数据对一个单目深度模型进行后训练；
   - 目的是让 RGB 编码器学习机器人操作场景中的几何结构；
   - 后训练完成后，丢弃深度预测头，只保留编码器侧特征。

2. **VLA 策略训练与推理阶段**
   - 输入为多视角 RGB 图像、语言指令和机器人本体状态；
   - 不使用真实深度图、点云或预测深度图作为策略输入；
   - 冻结几何编码器，从 RGB 图像中提取 Geometry-Enhanced Post-Trained features，即 **GEP 特征**；
   - 使用机器人本体状态生成查询 token，去 cross-attend 图像空间几何特征网格；
   - 得到少量紧凑几何 token，与 RGB-语言语义 token 拼接后送入动作解码器。

---

## 2.2 RGB-derived GEP 几何特征

- GeoAlign 初始化几何分支为 **Depth Anything V2-Small**。
- 使用机器人领域 RGB-D 数据进行后训练，使其能够从 RGB 中学习机器人工作空间的几何结构。
- 后训练损失使用 **Scale-Invariant Logarithmic loss, SiLog loss**，用于监督预测深度与真实 metric depth 之间的差异。
- 后训练完成后：
  - 丢弃 depth prediction head；
  - 不把预测深度图送入策略；
  - 只使用编码器侧的中间几何特征作为 GEP features。

这样设计的意义是：

- 避免真实深度传感器在透明物体、薄结构、环状物体上容易出现的缺失和噪声；
- 同时利用 RGB-D 监督让 RGB 表征具备几何敏感性；
- 在策略 rollout 时只需要 RGB 输入，部署更方便。

## 2.3 图像空间几何特征网格

- 输入图像在几何分支中被处理为特征网格。
- 文中设置：
  - 输入分辨率为 `518 × 518`；
  - patch size 为 14；
  - 得到 `37 × 37` 基础 patch grid；
  - 经过 reassemble factor 2 后得到 `74 × 74` 特征图；
  - 每个视角产生 `5476` 个空间 token；
  - 多视角情况下，总几何 token 数为 `N × 5476`。
- 每个 token 通过线性投影到 `dg = 256` 维。
- 同时加入：
  - 2D 位置编码；
  - 视角 embedding。

这使得模型保留了**图像空间中的局部几何结构**，而不是简单压缩成全局特征。

---

## 2.4 状态引导的空间对齐机制

GeoAlign 的关键创新是：  
**不是使用固定 learned queries，也不是全局池化几何特征，而是让机器人本体状态生成查询 token，动态选择当前操作阶段相关的局部几何信息。**

具体流程如下：

- 机器人本体状态 `st` 首先经过状态编码器，得到状态 embedding：

  - `ht = LN(Es(st))`

- 然后通过 MLP 生成 `K = 8` 个查询 token：

  - `Qt = reshape(Wq ht, K, dg) + Pq`

- 这些查询 token 对完整的 GEP 几何特征网格进行 cross-attention：

  - Query 来自机器人当前状态；
  - Key / Value 来自图像空间几何特征网格。

- cross-attention 后再经过 FFN，得到几何对齐后的 token：

  - `Gt`

- 最后投影到 VLA 动作解码器所需维度：

  - `Zgeo_t = Wz LN(Gt)`

- 输出为 **8 个紧凑几何 token**。

这一机制的直观含义是：

- 当机器人处于接近目标阶段时，关注目标和末端执行器附近区域；
- 当机器人处于插入阶段时，关注开口、边界、孔洞、容器口等局部几何；
- 当机器人释放物体时，关注放置区域、支撑面和相对位置；
- 几何注意力会随机器人状态变化，而不是静态地看全图或平均全图。

## 2.5 动作生成器

- GeoAlign 继承 **Isaac-GR00T N1.6-3B** 的 VLA 架构。
- RGB-语言 backbone 编码得到语义 token：

  - `Zvlm_t = Evlm(It, l)`

- 几何 token 与语义 token 拼接：

  - `Ct = [Zvlm_t; Zgeo_t]`

- 动作解码器使用 **flow-matching DiT action head**。
- 动作输出是一个 action chunk：

  - `At ∈ R^{H × da}`

- 训练时：
  - 从高斯噪声采样；
  - 使用 flow time `τ` 构造带噪动作；
  - 解码器预测从噪声到真实动作的 velocity；
  - 使用 GR00T 的 action mask normalization 计算损失。

- 推理时：
  - 从高斯噪声初始化；
  - 使用 Euler steps 积分生成动作；
  - 执行动作 chunk 后重新规划。

## 2.6 算法流程概括

GeoAlign 的策略训练流程可以概括为：

1. 输入 demonstration batch：RGB 图像、语言指令、机器人状态、真实动作 chunk 和 action mask。
2. 使用 VLA encoder 编码 RGB 与语言，得到语义 token。
3. 使用冻结的 GEP 几何编码器提取 RGB-derived geometry features。
4. 加入位置编码和视角 embedding，形成几何特征网格。
5. 使用机器人状态生成 8 个 query token。
6. query token cross-attend 几何网格，得到紧凑几何 token。
7. 将几何 token 与语义 token 拼接。
8. 使用 flow-matching DiT 动作头预测动作 velocity。
9. 根据动作监督信号更新下游 projector、alignment module 和 action decoder。

## 6. 主要结论与发现

## 6.1 总体结果

GeoAlign 在多个 benchmark 上取得了较强结果：

- **LIBERO**
  - GeoAlign 平均成功率：`99.0%`
  - RGB-only baseline：`97.0%`
  - 绝对提升约 `+2.0` 个百分点。

- **SimplerEnv-Fractal**
  - Pick Coke Can：`100.0%`
  - Move Near：`85.5%`
  - Open/Close Drawer：`70.3%`
  - 平均：`85.3%`
  - 相比 RGB-only 平均提升 `+5.7` 个百分点。

- **真实 ALOHA**
  - GeoAlign：`78.8%`
  - RGB-only：`65.0%`
  - π0.5：`67.5%`
  - 相比 RGB-only 提升 `+13.8` 个百分点。

## 6.2 消融发现

### 几何后训练很重要

- w/o post-training：
  - LIBERO 平均 `95.9%`
- GeoAlign：
  - `99.0%`

说明仅使用未经机器人领域适配的通用 DA-V2 特征不够，机器人领域 RGB-D 后训练能显著增强几何表征。

### 空间查询机制很重要

- w/o spatial querying：
  - `91.6%`
- GeoAlign：
  - `99.0%`

说明将几何特征简单做全局平均会丢失关键局部信息。

### 本体状态引导查询很重要

- w/o state queries：
  - `96.2%`
- GeoAlign：
  - `99.0%`

说明使用机器人状态动态生成 query，比固定 learned query 更适合选择当前操作阶段相关的空间几何。

### 冻结几何 encoder 更优

- w/ unfrozen encoder：
  - `95.93%`
- GeoAlign：
  - `99.0%`

说明策略训练阶段直接更新几何 encoder 可能破坏后训练得到的几何表征，冻结反而更稳定。

## 7. 优点与亮点

## 7.1 方法设计亮点

- **不直接依赖深度输入**
  - 策略 rollout 时不使用真实深度、点云或预测深度图；
  - 减少了对深度传感器质量的依赖；
  - 对透明物体、薄结构物体更有实际意义。

- **利用 RGB-D 监督但只保留 RGB-derived features**
  - 后训练阶段吸收深度监督；
  - 推理阶段只使用 RGB；
  - 在部署便利性和几何感知能力之间取得平衡。

- **状态引导的动态空间查询**
  - 不是静态几何 token，也不是全局平均；
  - 由机器人当前本体状态决定关注哪些图像区域；
  - 更符合机器人操作的阶段性需求。

- **紧凑 token 设计**
  - 从每视角 5476 个几何 token 中提取 8 个紧凑 geometry tokens；
  - 控制了动作解码器的上下文长度；
  - 计算开销相对可控。

- **与现有 VLA 架构兼容**
  - 基于 GR00T N1.6-3B 和 DiT action head；
  - 作为额外 geometry-conditioning 模块接入；
  - 方法具有一定通用性。

---

## 7.2 实验设计亮点

- 覆盖了：
  - 大规模仿真 benchmark；
  - 共享 SimplerEnv-Fractal 任务；
  - 真实 ALOHA 几何关键任务。

- 消融实验针对性强，能够分别验证：
  - 几何后训练；
  - 空间查询；
  - 状态 query；
  - encoder 冻结策略。

- 真实任务设计合理：
  - 透明容器；
  - 环状胶带；
  - 被遮挡透明胶带；
  - 盘子相对位置放置；
  - 都是语义容易、几何困难的典型操作场景。

- 报告了推理速度和训练配置，复现信息较完整。

## 8. 不足与局限

## 8.1 方法本身的局限

- **没有显式碰撞、可达性或接触建模**
  - GeoAlign 提供几何 token，但并不直接建模碰撞约束、接触力、稳定性或运动学可达性。
  - 对插入、装配、接触敏感操作仍可能不足。

- **依赖视觉覆盖**
  - GEP 特征来自 RGB 图像；
  - 如果关键几何区域被遮挡、超出视野或视角不足，模型难以恢复完整几何。

- **依赖机器人领域 RGB-D 后训练数据**
  - 几何特征质量与后训练数据分布密切相关；
  - 如果换到新机器人、新相机、新工作空间或新物体类别，可能需要重新适配。

- **没有持久场景记忆**
  - 当前系统每个 action chunk 主要基于当前观测和状态；
  - 没有显式维护长期 scene-level memory；
  - 对长时程遮挡、物体暂时不可见、多阶段复杂任务可能不够。

- **没有使用力觉或触觉**
  - 对视觉几何不足以判断的任务，例如紧配合插入、柔性物体、接触后微调，可能需要 force / tactile feedback。

---

## 8.2 实验覆盖的局限

- **真实机器人试验数量有限**
  - 每个真实任务只有 20 次 trial；
  - 虽然总计 160 次，但对真实机器人泛化能力的统计置信度仍有限。

- **真实任务集中在 tabletop manipulation**
  - 没有覆盖移动操作、动态环境、大范围场景变化、多机器人或户外场景。

- **相机和光照条件固定**
  - ALOHA 实验中相机、机器人 base 和 lighting 固定；
  - 对相机外参变化、照明变化、背景变化的鲁棒性尚未充分验证。

- **公开方法横向比较不完全公平**
  - LIBERO 和 SimplerEnv 中部分方法结果来自公开表格；
  - 不同方法的训练数据、模型大小、评估基础设施可能不同；
  - 论文也承认这些只作为 public-context。

- **Open/Close Drawer 仍未完全解决**
  - GeoAlign 在 Fractal Open/Close Drawer 上为 `70.3%`；
  - 虽有提升，但说明状态引导局部几何对齐并不能完全解决受约束抽屉操作。

---

## 8.3 潜在偏差风险

- 几何后训练数据来自 RobbyVLA / LingBot-Depth 中的 Franka 和 UR7e 机器人数据，而真实评估使用 ALOHA；
  - 虽然评估任务未包含在后训练数据中，但机器人域数据仍可能偏向特定工作空间、相机视角和操作分布。

- 真实任务选择强调几何关键场景，这有利于展示 GeoAlign 优势；
  - 对纯语义泛化、长时程规划、复杂语言推理等任务的收益可能没有那么明显。

- 方法基于较强的 GR00T N1.6-3B backbone；
  - 在较小 VLA 模型或不同动作头上是否同样有效，需要进一步验证。

---

# 总体评价

GeoAlign 的核心贡献在于把 VLA 模型中的“语义理解”和“可执行几何对齐”明确区分开，并提出一种实用的桥接机制：通过机器人领域 RGB-D 后训练得到 RGB-derived geometry features，再用机器人本体状态动态查询局部几何区域，生成少量紧凑 token 供动作解码器使用。

实验结果表明，该方法在 LIBERO、SimplerEnv-Fractal 和真实 ALOHA 几何关键任务上均带来提升，尤其在透明物体、环状结构、相对空间放置等场景中表现突出。其方法设计较简洁，和现有 VLA backbone 兼容，并且不需要在推理时依赖深度传感器。

不过，GeoAlign 仍主要是视觉几何增强方法，尚未解决碰撞、接触、力控、长期记忆和复杂物理约束等问题。真实实验规模也仍有限，未来需要在更开放、更动态、更长时程的机器人任务中进一步验证。
