## 摘要
视觉-语言-动作（VLA）模型越来越多地将机器人策略建立在历史信息、深度信息或 4D 特征的条件之上，以解决长时程操作中的歧义。然而，更多的时空证据并不一定更好：当注入的证据不具备运动一致性时，可能会引入几何漂移、碎片化的时间线索以及不稳定的动作生成。这引出了一个简单的问题：VLA 应该记住过去的帧，还是记住连接这些帧的运动？我们提出 MotionVLA，这是一种运动历史接口，可将一个仅包含过去信息的短视频窗口转换为紧凑且时间连续的轨迹场 token。MotionVLA 并非将历史视为一组稀疏的、独立提升的帧，而是将近期观测表示为物理上连贯的运动证据。当前视觉 token 查询该历史以检索与任务相关的运动信息，随后在基于轨迹的监督下将其重新耦合到 VLA 流中。在仿真基准和初步真实机器人执行中的实验表明，MotionVLA 提升了长时程操作性能，同时产生更平滑、更直接的执行过程。这些结果表明，有效的 VLA 记忆并不仅仅在于提供更多 4D 上下文，而在于呈现可用于控制的运动一致性证据。

## Abstract
Vision-language-action (VLA) models increasingly condition robot policies on history, depth, or 4D features to resolve ambiguity in long-horizon manipulation. However, more spatiotemporal evidence is not necessarily better: when the injected evidence is not motion-consistent, it can introduce geometric drift, fragmented temporal cues, and unstable action generation. This raises a simple question: should a VLA remember past frames, or remember the motion that connects them? We introduce MotionVLA, a motion-history interface that converts a short past-only video window into compact, time-continuous trajectory-field tokens. Instead of treating history as a sparse set of ndependently lifted frames, MotionVLA represents recent observations as physically coherent motion evidence. Current visual tokens query this history to retrieve task-relevant motion information, which is then recoupled into the VLA stream under trajectory-grounded supervision. Experiments across simulation benchmarks and preliminary real-robot rollouts show that MotionVLA improves long-horizon manipulation while producing smoother and more direct executions. These results suggest that effective VLA memory is not just about providing more 4D context, but about exposing motion-consistent evidence that is usable for control.

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 核心思想

MotionVLA 提出一种**运动历史接口**，将短时过去视频窗口转换为紧凑的、时间连续的**轨迹场 token（trajectory-field tokens）**。  
这些 token 不直接作为大量历史帧拼接进 VLA，而是作为一个可查询的运动记忆库。当前视觉 token 通过 cross-attention 查询该记忆，提取与当前任务状态相关的运动信息，再重新注入 VLA 主干用于动作生成。

其核心设计可以概括为：

> 用连续轨迹场表示过去运动，用当前视觉特征按需检索历史运动，再通过轨迹监督将运动信息重新耦合到 VLA 控制流中。

---

### 2.2 基础 VLA 框架：π0 风格策略

论文以 π0 风格 VLA 为基础。机器人在时间步 `t` 接收：

- 多视角 RGB 图像：  
  `o_t = {I_t^(k)}`
- 机器人本体状态：  
  `s_t`
- 自然语言指令：  
  `x`

模型通过图像编码器、文本编码器和状态 token 构造前缀：

```text
H_t = LLM([Enc_img(o_t); Enc_txt(x); Tok(s_t)])
```

动作生成采用 flow-based action expert。给定条件上下文 `H_t`，模型输出长度为 `H` 的连续动作块：

```text
a_t ∈ R^(H × d_a)
```

训练时使用 flow-matching 行为克隆目标，将噪声逐步映射到专家动作。

---

### 2.3 轨迹场 4D 表示

MotionVLA 使用冻结的预训练轨迹提取器 **TraceAnything**。  
给定一段头部相机历史帧序列，轨迹提取器为像素构建连续时间 3D 轨迹：

```text
T: (i, u, v) -> x_i,u,v(·) ∈ C([0,1], R^3)
```

含义是：某个输入帧中的某个像素点在归一化时间区间 `[0,1]` 内对应一条连续 3D 轨迹。

不过，MotionVLA 并不直接将密集轨迹场全部送入 VLA，而是取轨迹提取器内部的最终 decoder token：

```text
Z = TrajEnc_tok(I_1:T^head)
```

这些 token 被视为紧凑的 4D 运动特征。

---

### 2.4 构建仅包含过去信息的运动历史

在时间步 `t`，MotionVLA 明确排除当前帧，只使用过去头部相机帧构建历史窗口：

```text
W_t = [I_{t-1-(T_hist-1)Δ}, ..., I_{t-1-Δ}, I_{t-1}]
```

其中：

- `T_hist`：历史帧数量；
- `Δ`：时间采样步长；
- 如果 episode 刚开始历史不足，则用最早可用帧做左侧 padding；
- 训练与推理使用完全相同的 past-only 历史定义；
- 推理时使用 FIFO buffer 在线维护历史帧。

默认设置为：

- `T_hist = 5`
- `Δ = 2`
- 头部相机流采样 10 FPS
- 有效覆盖约 10 个原始相机帧

构建流程大致为：

1. 从 episode 或 FIFO buffer 中取过去帧；
2. 若历史不足，用最早帧补齐；
3. 按 stride 采样得到历史窗口；
4. resize 与 normalize；
5. 输入冻结轨迹提取器；
6. 得到轨迹场 token；
7. 投影到 VLA hidden size；
8. 加位置编码，形成运动记忆 `M_kv`。

---

### 2.5 Decouple-then-Recouple：先解耦再重耦合

论文的核心模块是 **Decouple-then-Recouple 4D Fusion**。

#### 2.5.1 Decouple：当前视觉查询历史运动

设当前头部相机视觉 token 为：

```text
V_head ∈ R^(B × S_img × E)
```

运动历史 token 为：

```text
M_kv ∈ R^(B × S_4D × E)
```

MotionVLA 使用当前视觉 token 作为 query，历史运动 token 作为 key/value，进行 cross-attention：

```text
M = MHA(V_head, LN(M_kv), LN(M_kv))
```

得到的 `M` 与当前视觉 token 数量相同，但每个 token 都融合了与当前视觉状态相关的历史运动信息。

这一设计区别于直接拼接历史帧或 4D token。它不是让模型被动接收全部历史，而是让当前状态主动读取有用的运动证据。

---

#### 2.5.2 轨迹监督：防止运动 token 退化为外观摘要

仅靠 cross-attention，检索出的 token 可能只学习到静态外观信息，而不是真正的运动进展。  
因此论文加入辅助轨迹重建损失。

训练总目标为：

```text
L = L_action + α L_traj
```

其中 `L_action` 是 π0 风格 flow-matching 行为克隆损失。给定专家动作 `a`、噪声 `ε` 和时间变量 `τ`：

```text
a_τ = (1 - τ)ε + τa
```

模型预测速度场 `v_θ(a_τ, τ; prefix)`，目标为：

```text
a - ε
```

动作损失为：

```text
L_action = E[ ||v_θ(a_τ, τ; prefix) - (a - ε)||² ]
```

辅助轨迹损失则用一个轻量预测头 `g_φ` 从运动 token 中重建历史动作序列：

```text
â_hist = g_φ(M)
```

损失为：

```text
L_traj = 1/(T_hist d_a) ||â_hist - a_hist||²
```

论文选择重建历史动作，而不是重建像素或密集轨迹，因为历史动作更直接反映机器人控制状态与运动进展。

---

#### 2.5.3 Recouple：重新注入 VLA 流

在得到运动增强 token `M` 后，MotionVLA 将其与当前多视角视觉 token `V_mv` 拼接，并通过轻量 MLP 融合：

```text
F = MLP([M; V_mv])
```

然后将融合后的视觉 token、文本 token 和机器人状态 token 一起输入 VLA 主干，使动作专家基于当前感知和历史运动证据生成动作。

---

### 2.6 两阶段训练

论文采用 generic-to-specific 训练策略：

- **Stage I：通用运动历史接口对齐**
  - 使用约 100 小时异构机器人操作数据；
  - 同时优化动作损失和辅助轨迹损失；
  - 目的是让轨迹场 token 与 VLA embedding 空间对齐。

- **Stage II：目标任务适配**
  - 在具体 benchmark 任务上微调；
  - 辅助轨迹 head 被 detach；
  - 主要优化动作生成能力。

附录实验显示，Stage I 的对齐步数对性能有影响，在 `stack blocks two` 设置中约 60k steps 达到较好效果；过短会导致轨迹场特征利用不足，过长可能轻微损害下游适配。

#### 路径效率 PE

为了评估运动一致性，论文提出路径效率：

```text
PE = L_policy / L_expert
```

其中轨迹长度为：

```text
L = Σ ||p_{t+1} - p_t||₂
```

含义是策略执行的末端执行器路径长度与专家示范路径长度之比。

- `PE = 1` 表示路径长度与专家一致；
- PE 越低越好；
- PE 高说明执行中存在更多绕路、抖动或无效修正。

论文强调 PE 只在成功 episode 上统计，并且必须与 SR 一起报告，否则可能存在样本选择偏差。

## 6. 主要结论与发现

论文的主要结论包括：

- **运动一致的历史表示比简单历史堆叠更重要。**  
  直接加入历史帧或离散 4D token 可能因几何漂移和时间碎片化导致性能下降。

- **轨迹场 token 能为 VLA 提供更稳定的历史运动证据。**  
  MotionVLA 将过去视频窗口编码为时间连续的轨迹场 token，使当前视觉状态可以查询近期物理运动进展。

- **Decouple-then-Recouple 设计有效。**  
  当前视觉 token 先查询运动历史，再将运动信息重新注入 VLA，比直接拼接历史 token 更有效。

- **辅助轨迹监督有助于让 token 保持控制相关性。**  
  通过重建历史动作序列，模型学到的运动 token 更关注机器人动态进展，而不是静态外观。

- **在长程任务上收益更明显。**  
  RoboTwin2.0 的长程 Touch 任务中，π0 从 19% 提升到 41%；LIBERO-LONG 中，π0 从 85.2% 提升到 91.2%。

- **MotionVLA 产生更直接、更平滑的轨迹。**  
  在 LIBERO-GOAL / LONG 上，MotionVLA 的 PE 为 1.05 / 1.10，优于 π0 的 1.23 / 1.19，也优于 4D-VLA 的 1.36 / 1.32。

- **真实机器人实验显示初步迁移价值。**  
  Agilex Piper 上平均成功率从 π0 的 28.7% 提升到 41.9%，平均完成步数从 338 降至 282。

## 7. 优点与亮点

### 7.1 方法设计亮点

- **问题切入清晰**  
  论文没有简单追求更多历史或更大上下文，而是指出历史证据本身的运动一致性才是关键。

- **轨迹场作为 VLA 记忆接口具有新意**  
  将连续时间 3D 轨迹场用于 VLA 历史记忆，使历史信息从离散帧变为连续运动证据。

- **past-only 设置严谨**  
  构建历史时明确排除当前帧，训练和推理使用相同接口，避免信息泄漏。

- **query-conditioned retrieval 合理**  
  当前视觉 token 主动查询历史运动，而不是盲目拼接全部历史，减少 token 爆炸和无关历史干扰。

- **辅助监督目标贴近控制**  
  使用历史动作重建而非图像重建，使运动 token 更直接服务于控制。

- **轻量融合设计**  
  通过 cross-attention 和 MLP 融合，避免将密集轨迹场直接输入 VLA，控制了计算开销。

---

### 7.2 实验设计亮点

- **不仅看成功率，也看路径效率**  
  PE 指标很好地支撑了论文关于“更平滑、更直接执行”的核心论点。

- **覆盖长程、中程、短程任务**  
  RoboTwin2.0 任务按 episode 长度分组，有助于观察方法在不同时间跨度下的收益。

- **消融较完整**  
  证明收益不是来自简单加历史或简单加 4D，而是来自对齐、查询、轨迹监督和重耦合设计。

- **包含真实机器人验证**  
  虽然规模有限，但真实 Agilex Piper 实验增强了方法的应用可信度。

- **考虑鲁棒性**  
  对历史分支进行受控污染，分析轨迹提取器失败对性能的影响，比较诚实地暴露潜在风险。

---

## 8. 不足与局限

### 8.1 方法局限

- **依赖冻结轨迹提取器**  
  MotionVLA 的运动历史质量依赖 TraceAnything。如果出现遮挡、快速运动、低纹理、反光或动态模糊，轨迹场可能不可靠。

- **额外计算开销明显**  
  默认配置下推理 FPS 从无历史的 12.3 降到约 7.1；更长历史窗口甚至可能降到 3.1 或 0.9 FPS，影响实时部署。

- **历史并非总是有益**  
  论文也承认，当当前观测足以完成任务时，运动历史可能中性甚至轻微有害。

- **历史窗口设计仍较手工**  
  默认 `T_hist=5, Δ=2` 是实验选出的较优配置，但不同任务、机器人和相机频率下可能需要重新调参。

- **长历史可能引入过时信息**  
  消融显示更长历史并不总是更好，可能带来陈旧或任务无关的早期运动线索。

---

### 8.2 实验局限

- **真实机器人验证规模有限**  
  仅在 Agilex Piper 上测试 3 个任务，每个任务数据量相对有限，不能证明其在多机器人、多场景中的通用部署能力。

- **RoboTwin2.0 包含自定义任务**  
  `blocks touching rgb` 是作者自定义长程任务，虽然有助于测试时序推理，但也可能与方法假设更匹配。

- **对比方法训练条件不完全透明**  
  论文没有完整说明所有 baselines 是否在相同数据、相同训练预算和相同实现框架下复现。

- **外部预训练资源带来的优势未完全剥离**  
  TraceAnything 本身是强大的预训练轨迹提取器，MotionVLA 的提升部分来自该外部模型。论文虽有消融，但对外部预训练资源公平性的讨论仍不足。

- **路径效率只在成功 episode 上统计**  
  虽然论文明确与 SR 一起报告，但不同方法成功的 episode 子集不同，PE 仍可能存在一定选择偏差。

- **缺少大规模真实世界长期部署结果**  
  论文主要证明方法在 benchmark 和初步真实任务上有效，但尚未展示长期运行稳定性、跨光照环境、跨物体类别和跨相机配置泛化能力。

---

## 总体评价

MotionVLA 是一篇针对 VLA 长程操作中历史信息注入问题的有价值工作。它的关键贡献不在于简单加入更多 4D 特征，而在于指出并解决“历史证据是否运动一致”这一被忽视的问题。通过轨迹场 token、当前视觉查询、轨迹监督和重耦合机制，MotionVLA 在仿真和初步真实机器人实验中都表现出更好的成功率与路径效率。

从研究意义上看，该工作为 VLA 记忆设计提供了一个重要方向：未来机器人策略的历史模块不应只是存储过去观测，而应显式建模“从过去到现在的运动过程”。不过，方法仍依赖外部轨迹提取器，计算开销和真实世界泛化仍需进一步验证。
