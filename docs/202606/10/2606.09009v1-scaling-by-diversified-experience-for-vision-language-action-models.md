## 摘要
视觉-语言-动作模型在现实世界部署中面临重大挑战，原因在于高层推理与低层控制相互纠缠，以及策略优化的不稳定性。本文提出 SyVLA，一种利用多样化经验训练的鲁棒 VLA 模型。我们提出一种意图解耦算法，用于从推理上下文中分离与控制相关的特征，并提出一种由相似样本引导的强化学习流程，以稳定策略更新并缓解分布偏移。在真实世界机器人任务和多模态基准上的大量实验表明，与现有方法相比，SyVLA 实现了更高的任务成功率和更强的分布外泛化能力，同时有效保留了核心视觉-语言能力。代码和数据集已发布于 \href{https://sy-vla.github.io/}{项目页面}。

## Abstract
Vision-Language-Action models face significant challenges in real-world deployment due to the entanglement of high-level reasoning with low-level control, and the instability of policy optimization. In this paper, we introduce SyVLA, a robust VLA model trained with diversified experiences. We propose an Intention Decoupling algorithm to isolate control-relevant features from reasoning contexts and a similar-sample guided RL pipeline to stabilize policy updates and mitigate distribution shift. Extensive experiments on real-world robotic tasks and multi-modal benchmarks demonstrate that SyVLA achieves superior task success rates and stronger out-of-distribution generalization compared to existing methods, while effectively preserving core vision-language capabilities. Codes and Datasets is released on \href{https://sy-vla.github.io/}{project page}.

## 1. 核心问题与整体含义：研究动机和背景

本文关注视觉-语言-动作模型（Vision-Language-Action, VLA）在真实机器人部署中的两个关键瓶颈：

- **高层推理与低层控制耦合**
  - 近年来 VLA 模型通常基于大型视觉语言模型（VLM）构建，利用其视觉理解、语言理解和推理能力指导机器人动作。
  - 但在实际训练中，模型往往过度关注动作模仿，导致原有 VLM 的视觉语言能力和逻辑推理能力退化，甚至出现灾难性遗忘。
  - 另一方面，如果显式引入 Chain-of-Thought（CoT）或任务分解，让模型以“Think-Before-Act”的方式执行任务，高层推理信息又可能泄漏到低层动作控制表征中，造成动作犹豫、不精确或行为停滞。

- **模仿学习目标与真实任务成功率不一致**
  - 传统 VLA 多依赖 imitation learning（IL）学习专家轨迹，即逐步拟合专家动作。
  - 但真实机器人执行是闭环过程，小误差会在数百个动作步骤中不断累积，使观测逐渐偏离训练分布，最终导致任务失败。
  - 强化学习（RL）理论上可以直接优化真实任务成功率，但 VLA 模型参数量巨大、动作空间连续且高维，在真实机器人上进行 RL 容易产生策略漂移、训练不稳定甚至能力崩溃。

因此，本文的整体目标是：

> 在保持 VLM 原有视觉语言理解与推理能力的同时，提升 VLA 模型在真实机器人任务中的动作精度、长程稳定性和分布外泛化能力。

为此，作者提出 **SyVLA**，一个通过多样化经验训练的 VLA 框架，并重点设计了两个机制：

1. **Intention Decoupling（意图解耦）**：减少高层推理信息对低层控制表征的干扰。
2. **Similar-Sample Guided RL（相似样本引导强化学习）**：稳定真实机器人上的 RL 更新，缓解策略漂移。

## 2. 方法论：核心思想、关键技术与算法流程

## 2.1 SyVLA 总体架构

SyVLA 采用双系统结构：

- **VLM 模块**
  - 使用 **Qwen2.5-VL-3B** 作为视觉语言主干。
  - 负责视觉感知、语言理解、高层推理、任务分解和控制意图生成。
  - 支持“Think-Before-Act”推理，即先生成简短 CoT，再生成控制相关表征。

- **Action Expert 模块**
  - 使用基于 Transformer 的 **Flow Matching** 模型。
  - 参数量约 **0.69B**。
  - 根据视觉观测、机器人状态、当前噪声动作、解码时间步以及 VLM 传来的控制意图生成连续动作。
  - 采用 action chunk 机制，一次预测多个动作步，文中使用 chunk length = 50。

二者之间通过 **Feature Query Tokens** 连接：

- 在 VLM 输入序列末尾追加一组可学习的 Feature Query Tokens。
- VLM 前向传播后，取这些 tokens 的最后隐藏状态，称为 **Feature Query States**。
- Feature Query States 经过 MLP Adapter 映射后，作为 Action Expert 的控制条件。
- 相比 Pi0 使用 KV Cache 的方式，Feature Query Tokens 更轻量，推理延迟更低，并且更容易支持 VLM 与 Action Expert 的异步推理。

  - COCO 2017
  - VQA-v2
  - Pixmo Point Explanations
  - Pixmo Points
  - Cambrian

- 针对每个真实机器人任务收集数百条专家遥操作轨迹。

- 引入相似样本引导机制，稳定策略更新并避免能力崩溃。

## 2.3 Intention Decoupling：意图解耦算法

### 问题动机

在 Think-Before-Act 模式下，VLM 的推理信息可能混入 Feature Query States 中。

这会导致 Action Expert 在生成动作时受到高层语言推理信息干扰，表现为：

- 动作不精确；
- 执行犹豫；
- 在多个潜在子目标之间摇摆；
- 真实任务中长程稳定性下降。

### 核心思想

作者认为：

> Feature Query States 中并非所有 token 都对控制意图有用，一些 token 主要携带与控制无关或冗余的推理信息。可以通过梯度大小识别控制相关性，并屏蔽低相关 token。

### 算法流程

对每个训练样本执行两次 Action Expert 前向过程：

1. **第一次前向**
   - 将原始 Feature Query States 输入 Action Expert。
   - 计算动作损失 \(L_{action}\)。

2. **计算梯度**
   - 对每个 Feature Query State \(h_i^{raw}\) 计算动作损失梯度：
     \[
     g_i = \frac{\partial L_{action}}{\partial h_i^{raw}}
     \]
   - 计算每个梯度的 L2 norm：
     \[
     \|g_i\|_2
     \]

3. **选择低相关 token**
   - 根据梯度范数分布设定阈值。
   - 文中默认使用 **5% 分位数**作为阈值。
   - 若某个 token 的梯度范数低于阈值，则认为它对动作控制贡献较小。

4. **屏蔽 token**
   - 对这些低相关 token 的隐藏状态置零：
     \[
     h_i^{masked} = h_i^{raw} \cdot \mathbb{I}(\|g_i\|_2 \ge \tau)
     \]

5. **第二次前向与更新**
   - 使用屏蔽后的 Feature Query States 再次输入 Action Expert。
   - 重新计算动作损失，并进行模型更新。

### 理论解释

作者用简化的单层 Transformer Action Expert 分析了为什么梯度范数可以反映 token 的控制相关性。

简化地看，Action Expert 通过注意力聚合 Feature Query States：

\[
\hat{a} = \psi \left(
\frac{
\sum_i \kappa(q, W_K h_i) W_V h_i
}{
\sum_i \kappa(q, W_K h_i)
}
\right)
\]

其中：

- \(h_i\)：第 \(i\) 个 Feature Query State；
- \(q\)：动作查询向量；
- \(W_K, W_V\)：注意力中的 key/value 映射矩阵；
- \(\kappa\)：注意力核函数；
- \(\hat{a}\)：预测动作。

作者推导出损失对 \(h_i\) 的梯度主要受两个因素影响：

1. **注意力分数大小**
   - 如果某个 token 的注意力分数很小，说明模型认为它与动作决策关系较弱。

2. **该 token 的 value 向量与注意力输出的距离**
   - 如果它与整体注意力输出非常接近，说明它的信息已被其他 token 覆盖，可能是冗余信息。

因此，低梯度范数 token 要么与控制无关，要么是冗余信息。屏蔽它们有助于降低高层推理和低层控制之间的非因果耦合。

## 2.4 Similar-Sample Guided RL：相似样本引导强化学习

### 问题动机

直接对 VLA 做真实机器人 RL 存在明显风险：

- 高维连续动作空间导致探索不稳定；
- 参数量巨大，策略更新容易漂移；
- 稀疏奖励下训练样本效率低；
- 小幅策略变化可能导致真实机器人碰撞或任务完全失败。

### 核心思想

在 RL 更新时，不只使用真实 rollout 数据，还从 imitation learning 专家数据集中检索与当前 rollout 观测相似的样本，将它们一起加入 batch。

这样可以让策略：

- 从失败或成功 rollout 中学习任务级反馈；
- 同时保持与专家行为接近；
- 避免更新过大导致策略崩溃。

### 相似样本检索

对每个样本，使用图像编码器提取多视角观测特征。

两个样本 \(x, y\) 的相似度定义为：

\[
sim(x,y)=\sum_{v \in V} w_v \cdot sim_{cos}(E(O_x^{(v)}), E(O_y^{(v)}))
\]

其中：

- \(V\)：相机视角集合；
- \(O_x^{(v)}\)：样本 \(x\) 在视角 \(v\) 下的图像；
- \(E(\cdot)\)：图像编码器；
- \(sim_{cos}\)：余弦相似度；
- \(w_v\)：不同视角权重。

实现中使用 CLIP 编码图像特征。真实机器人有三个视角：

- 中心相机权重：0.83；
- 左右腕部相机权重：各 0.083。

### 专家样本 advantage 设计

作者发现：

- 如果对专家数据也使用标准 PPO/GAE 估计 advantage，训练很快出现梯度爆炸。
- 甚至只用专家数据、不用 rollout 数据，也会在 1000 步内发生不稳定。

作者认为原因是：

> value model 对专家行为分布估计不准，导致 advantage 赋值错误，使模型错误地区分专家数据中的“好动作”和“坏动作”。

因此，本文对专家数据采用固定 advantage：

\[
A_{expert}=1.0
\]

这等价于鼓励策略最大化专家动作似然，从而稳定更新。

同时，对 rollout 数据的 GAE 进行裁剪：

\[
A_{rollout} \in [-0.8, 0.8]
\]

以降低早期 value model 不准确带来的噪声。

## 3.4 对比方法

- **Pi0 from scratch**
  - 作者基于相同训练数据从头训练的 Pi0。
  - 用于尽量剥离 Pi0 大规模私有预训练数据带来的优势。

- **ChatVLA**
  - 使用 MoE 架构缓解动作数据与多模态数据之间的冲突。
  - 视觉语言能力较强，但精细操作能力不足。

此外，RL 部分还在附录中与以下方法做了比较：

- **SimpleVLA-RL**
- **Pi-RL**

- 部分基线可能未完全适配作者的数据、任务或训练流程，公平性仍可能受影响。
- Pi0 pretrained 使用了远大于 SyVLA 的私有预训练数据，而 Pi0 from scratch 又可能没有达到原始 Pi0 最优训练条件，因此比较仍需谨慎解释。

## 6. 主要结论与发现

论文的主要发现包括：

- **SyVLA 在真实机器人任务上整体优于强基线**
  - In-domain 平均成功率：
    - SyVLA：0.73；
    - Pi0 pretrained：0.63；
    - OpenVLA-oft：0.45；
    - GR00T：0.38；
    - Pi0 from scratch：0.38；
    - ChatVLA：0.21。
  - OoD 平均成功率：
    - SyVLA：0.64；
    - Pi0 pretrained：0.48；
    - Pi0 from scratch：0.26；
    - OpenVLA-oft / GR00T：0.24；
    - ChatVLA：0.09。

- **SyVLA 具有更强 OoD 泛化能力**
  - 在衣服位置偏移、未见算术指令、未见模糊语言指令下，性能下降较小。

- **Intention Decoupling 对动作能力至关重要**
  - 在 Fold Shirt 任务上：
    - 完整 SyVLA：0.86；
    - 去除 Intention Decoupling：0.43。
  - 表明高层推理与低层控制耦合确实会显著损害精细操作。

- **RL 能进一步提升长程任务表现**
  - 去除 RL 后 Fold Shirt 成功率为 0.71；
  - 完整模型为 0.86。
  - 说明 RL 能弥合 IL 目标和真实任务成功率之间的差距。

- **相似样本引导能稳定 RL**
  - 不使用专家数据时，成功率下降到 0.21。
  - 对专家数据使用标准 advantage 时，成功率为 0，出现训练崩溃。
  - 固定专家样本 advantage = 1.0 是稳定训练的关键。

- **SyVLA 能较好保留多模态能力**
  - 在 MME 和 HallBench 上超过 ChatVLA；
  - 在 DocVQA 和 MMMU 上略低于 ChatVLA；
  - 相比 Pi0、GR00T 等不具备 VQA 能力的模型，SyVLA 明显更完整。

## 7. 优点与亮点

## 7.1 方法设计亮点

- **兼顾推理能力和控制能力**
  - 不是单纯追求动作成功率，而是明确尝试保留 VLM 的视觉语言理解与推理能力。
  - 对 VLA 走向通用具身智能具有较强意义。

- **Feature Query Tokens 连接方式轻量**
  - 比传递完整 KV Cache 或整段隐藏状态更高效。
  - 有利于降低推理延迟，并支持异步推理。

- **Intention Decoupling 思路简洁有效**
  - 利用动作损失对 query token hidden state 的梯度范数作为控制相关性指标。
  - 不需要额外标注。
  - 计算开销主要在 Action Expert 上，作者报告训练时间只增加约 10%。

- **Similar-Sample Guided RL 具有实践价值**
  - 针对真实机器人 RL 的不稳定问题，直接利用已有 IL 数据约束策略更新。
  - 与单纯加 BC loss 不同，该方法通过语义相似样本构造更贴近当前 rollout 状态的专家约束。

- **关注真实机器人部署**
  - 任务包括柔性衣物折叠、毛巾包裹、布袋装物等，较有现实难度。
  - 不是只在仿真或标准桌面 pick-and-place 上验证。

---

## 7.2 实验设计亮点

- **同时评估机器人能力与多模态能力**
  - 避免只看任务成功率而忽略 VLM 灾难性遗忘。
  - 也避免只看 VQA benchmark 而忽略真实控制能力。

- **设计 OoD 测试**
  - 包括位置变化、未见语言指令、未见模糊需求。
  - 能更好检验泛化，而非训练任务记忆。

- **对 Pi0 做了较公平的补充比较**
  - 既比较官方 pretrained Pi0，也比较相同数据下从头训练的 Pi0。
  - 有助于揭示大规模私有数据对性能的影响。

- **消融实验较完整**
  - 对 CoT、意图解耦、RL、专家数据、相似检索、advantage 设置均做了验证。
  - 能较清楚定位各模块贡献。

## 8. 不足与局限

## 8.1 理论局限

- 作者提供了基于单层 Transformer 的理论分析，但这只是简化模型。
- 真实 Action Expert 是多层 Transformer Flow Matching 模型，且包含跨注意力、视觉编码、动作编码等复杂模块。
- 因此，Intention Decoupling 的理论保证仍不严格。
- 论文也承认，目前缺乏严谨理论证明。

---

## 8.2 实验覆盖局限

- 真实机器人任务数量有限，仅 3 个。
- 每个模型每个任务的评估次数较少，统计稳定性有限。
- 实验平台单一，缺少跨机器人、跨夹爪、跨相机布局的验证。
- 主要关注桌面近距离操作，没有覆盖移动机器人、导航、双臂协作、大范围空间操作等场景。
- 柔性物体任务主要是衣服和布袋，尚不足以证明对各种复杂可变形物体的广泛泛化。

---

## 8.3 基线公平性风险

- 不同 VLA 模型架构差异较大，训练配方、数据适配难度和推理机制都不同。
- 某些基线可能没有经过最适合其架构的超参数调优。
- Pi0 pretrained 使用了超过 10k 小时的私有大规模数据，而 SyVLA 使用约 450 小时数据；二者数据规模不同，比较时需要谨慎。
- 虽然作者加入了 Pi0 from scratch，但从头训练版本是否充分复现 Pi0 的最佳能力仍不完全确定。

---

## 8.4 RL 实践限制

- RL 阶段仍需要真实机器人在线 rollout，成本较高。
- 需要人工监督并提供 0/1 成功奖励。
- 尽管作者认为人工负担较轻，但对于大规模任务扩展仍可能成为瓶颈。
- RL 过程中仍存在真实机器人碰撞、损坏或安全风险。
- 相似样本引导依赖已有高质量 IL 数据，如果专家数据覆盖不足，检索到的样本可能无法提供有效约束。

---

## 8.5 方法适用性限制

- Intention Decoupling 依赖 Feature Query Tokens 架构，对其他 VLA 架构未必可直接适用。
- 使用梯度范数筛选 token 的策略虽然简单，但可能误删某些低频但关键的控制信息。
- mask 阈值虽然在 3%–15% 范围内较稳定，但超过 20% 后性能明显下降，说明该机制仍需谨慎调参。
- Think-Before-Act 模式会带来额外推理延迟，在高频实时控制任务中可能受限。

---

## 总体评价

本文提出的 SyVLA 是一个较完整、工程导向明确的 VLA 框架。其核心贡献不只是模型架构，而是围绕真实机器人部署中的两个关键问题——**推理-控制耦合**与**RL 训练不稳定**——提出了相对简洁且有效的解决方案。

从实验结果看，SyVLA 在真实机器人任务、OoD 泛化和多模态能力保留之间取得了较好的平衡。尤其是在使用远少于 Pi0 的预训练数据情况下，仍达到或超过多个强基线，说明其训练策略和解耦机制具有较高实践价值。

不过，论文仍需要更大规模、更跨平台、更高统计置信度的真实机器人实验来进一步验证方法的普适性。同时，Intention Decoupling 的理论基础和 Similar-Sample Guided RL 的适用边界也仍有待深入研究。
