## 摘要
我们提出了 \textbf{Echo-Memory}，这是一项关于动作条件世界模型中记忆机制的受控研究。这类模型根据首帧、文本提示和相机动作序列生成多段视频，但其核心失败往往并非局部图像合成，而是记忆：当相机离开后再返回时，场景或显著物体可能会悄然改变。现有记忆设计难以比较，因为性能提升与骨干网络、训练、检索和评估差异相互纠缠。Echo-Memory 固定了动作到视频的接口，仅改变历史信息由生成器存储和读取的方式。在共享的视频扩散骨干网络、优化器、相机动作表示、采样器和评估流程下，我们比较了原始上下文、基于压缩的记忆、具有不同读出路径的空间摘要，以及状态空间递归。这个匹配矩阵分离了四个原本相互混淆的维度：\emph{容量}、\emph{压缩}、\emph{读出}和\emph{递归}。我们还通过三分支协议评估记忆：重放质量、域内循环重访，以及开放域返回探测。这些分支的结果经常不一致，表明重放保真度并不足以作为记住一个世界的代理指标。由此得到三点发现。原始上下文是一个强大的容量基线，它对开放域返回的提升远大于对重放指标的提升。紧凑性并不能无代价地替代容量：激进的空间记忆和混合压缩记忆会丢失返回所需的显著证据。最后，分块状态空间递归是我们矩阵中最强的开放域返回机制，说明隐式记忆的结构与是否使用隐式记忆同样重要。这些结果提供了一个紧凑的协议，用于研究动作世界模型中的记忆，而不局限于孤立的重放指标。

## Abstract
We present \textbf{Echo-Memory}, a controlled study of memory mechanisms in action-conditioned world models. These models generate multi-segment videos from a first frame, text prompt, and camera-action sequence, but their central failure is often memory rather than local image synthesis: after the camera leaves and returns, the scene or salient object may silently change. Existing memory designs are hard to compare because gains are entangled with backbone, training, retrieval, and evaluation differences. Echo-Memory fixes the action-to-video interface and varies only how history is stored and read by the generator. Under a shared video diffusion backbone, optimizer, camera-action representation, sampler, and evaluation pipeline, we compare raw context, compression-based memory, spatial summaries with different read-out paths, and state-space recurrence. This matched matrix separates four otherwise conflated axes: \emph{capacity}, \emph{compression}, \emph{read-out}, and \emph{recurrence}. We also evaluate memory through a three-branch protocol: replay quality, in-domain loop revisit, and open-domain return probes. The branches routinely disagree, showing that replay fidelity is not a sufficient proxy for remembering a world. Three findings follow. Raw context is a strong capacity baseline and improves open-domain return far more than it improves replay metrics. Compactness is not a free substitute for capacity: aggressive spatial and hybrid-compression memories lose the salient evidence needed for return. Finally, block-wise state-space recurrence is the strongest open-domain return mechanism in our matrix, showing that the structure of implicit memory matters as much as the decision to use it. These results provide a compact protocol for studying memory in action world models beyond isolated replay metrics.

  - 现有长视频 / 世界模型中的记忆方法很多，包括原始上下文检索、压缩记忆、空间记忆、状态空间递归等。

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 核心思想

Echo-Memory 的方法论可以概括为：

> 固定动作到视频生成系统的其他所有变量，仅改变“历史信息如何被存储、压缩、读取和递归传递”，从而公平比较不同记忆机制。

论文将动作世界模型中的记忆机制划分为四类：

1. **Context Memory / 原始上下文记忆**
   - 直接检索历史帧或历史 latent 作为上下文输入。
   - 通过改变上下文长度 \(K\) 研究容量影响，例如 \(K=1,5,20\)。

2. **Compression Memory / 压缩记忆**
   - 对历史上下文进行压缩，减少 token 或时间维度成本。
   - 包括 token weighting、temporal length compression、hybrid compression 等。

3. **Spatial Memory / 空间记忆**
   - 将历史信息压缩为空间场景摘要或空间 token。
   - 重点分析“存储了空间 token”是否等于“生成器能在返回时读出并利用这些 token”。

4. **State-Space Memory / 状态空间递归记忆**
   - 不显式附加长历史帧，而是通过隐式 recurrent / state-space 状态携带历史。
   - 比较 legacy hybrid recurrence 与 block-wise recurrence。

### 2.2 统一视频扩散骨干

- 所有实验使用共享的视频 Diffusion Transformer，文中基于 Wan 系列视频 DiT。
- 视频片段表示为：

\[
x_{1:T} \in \mathbb{R}^{T \times H \times W \times C}
\]

其中：
- \(T=81\) 帧；
- 分辨率为 \(352 \times 640\)。

- 模型学习条件速度场：

\[
v_\theta(z_t; t, c_{\text{text}}, c_{\text{ctx}}, c_{\text{act}})
\]

其中：
- \(z_t\)：VAE latent 加噪后的表示；
- \(c_{\text{text}}\)：文本条件；
- \(c_{\text{ctx}}\)：记忆上下文；
- \(c_{\text{act}}\)：动作 / 相机条件；
- \(t\)：flow-matching 时间步。

训练目标为 rectified-flow 回归损失：

\[
L(\theta)=\mathbb{E}_{t,x,c}\left\|v_\theta(z_t;t,c_{\text{text}},c_{\text{ctx}},c_{\text{act}})-v^\star\right\|_2^2
\]

其中：

\[
v^\star=z_1-z_0
\]

论文只在目标帧上计算监督损失，历史上下文只作为条件输入，不作为预测目标。

### 2.4 上下文构造

对于当前生成片段 \([s,e]\)，上下文集合定义为：

\[
C_{s:e}=\{(x_{f_k},p_{f_k})\}_{k=0}^{K-1}
\]

其中：
- \(f_0=s\)，即当前片段的 anchor frame；
- 其余 \(f_1,\dots,f_{K-1}\) 来自历史帧检索器；
- \(p_{f_k}\) 是对应相机姿态 / 动作元数据；
- 检索策略基于视场重叠；
- 训练中以 \(p_{\text{drop}}=0.1\) 的概率丢弃检索到的历史帧，只保留 anchor，用于模拟 cold-start。

### 3.2 评测 benchmark：三分支协议

论文提出三分支评测协议，用于区分“会生成连续视频”和“真的记住世界”。

#### 1. Replay / 重放质量

- 使用真实数据集相机轨迹。
- 生成多段视频，与 GT 轨迹对应帧比较。
- 指标：
  - PSNR；
  - SSIM；
  - LPIPS。
- 作用：
  - 检查模型能否跟随相机轨迹；
  - 检查长时段 chunk 生成的低层图像质量；
  - 但论文认为它只是“健康信号”，不是最终记忆指标。

#### 2. In-domain return / 域内循环重访

- 使用数据集中有 GT 图像和 pose 的场景。
- 构造 180° loop / return 轨迹。
- 比较返回时图像是否与对应 GT 或镜像帧一致。
- 指标：
  - PSNR；
  - SSIM；
  - LPIPS；
  - 部分 VLM 语义一致性评分。

#### 3. Open-domain return / 开放域返回探测

- 用 Qwen Edit 构造开放域首帧。
- 对 20 个游戏风格环境 prompt，每个生成 8 个变体，共 \(20 \times 8\) 个首帧。
- 每个首帧中插入一个具有辨识度的 toy-like identity anchor object。
- 构造 45° leave-and-return 相机运动。
- 没有像素级 GT，因此主要使用 VLM 判断：
  - 物体外观是否保持；
  - 物体是否仍存在；
  - 视角是否一致；
  - 背景场景是否一致。

开放域 VLM 分数由 Qwen3-VL-30B-A3B 评估，打分公式为：

\[
0.45s_{\text{appearance}}+0.25s_{\text{presence}}+0.20s_{\text{view}}+0.10s_{\text{scene}}
\]

然后从 \([0,5]\) 缩放到 \([0,100]\)。

论文还用 Claude Opus 4.6、GPT-5.5 和人工标注做 cross-judge sanity check，发现与 Qwen3-VL 的相关性均超过 0.90，说明 VLM judge 的排序相对稳定。

## 5. 实验数量与充分性

### 5.1 实验组数概览

论文实验覆盖面较广，主要包括：

#### 1. 代表性记忆家族比较

比较 7 个主要 row：

- I2V baseline；
- Spatial Memory baseline；
- Compression weight-only；
- State-Space legacy hybrid；
- State-Space block-wise；
- Context K=5；
- Context K=20。

#### 2. Spatial read-out 消融

比较 4 个空间读出方式：

- Spatial baseline；
- inject-none；
- concat-text；
- cross-attn read-out。

#### 3. Compression 消融

比较 5 个压缩机制：

- weight-only；
- length \(r=2\)；
- length \(r=4\)；
- hybrid \(r=2+\) weight；
- hybrid \(r=4+\) weight。

#### 4. State-Space 消融

比较 2 个递归结构：

- legacy hybrid；
- block-wise recurrence。

#### 5. Context length 消融

比较 3 个上下文长度：

- \(K=1\)；
- \(K=5\)；
- \(K=20\)。

此外还有：
- replay 过程诊断；
- open-domain VLM judge 跨模型一致性检查；
- replay PSNR 与训练成本效率分析。

整体来看，核心实验大约包含十余个模型变体，并在三类评测分支上统一测试。

---

### 5.2 实验是否充分

总体上，实验设计具有较强充分性和公平性：

- **公平性强**：
  - 固定 backbone；
  - 固定 optimizer；
  - 固定训练步数；
  - 固定相机动作表示；
  - 固定采样器；
  - 固定评估流程；
  - 只改变记忆 profile。

- **评测维度完整**：
  - Replay 评估低层图像质量；
  - In-domain return 评估受控闭环一致性；
  - Open-domain return 评估分布外语义记忆。

- **消融逻辑清楚**：
  - 分别隔离容量、压缩、读出、递归结构等因素；
  - 避免将不同因素混在一起解释。

但也存在一定不足：

- open-domain 每个 row 的样本量文中提到是当前 material snapshot 中约 8 cases per row，规模相对有限；
- 只使用一个主要训练数据集；
- 训练步数固定为 5k，不能说明更大训练预算下排序是否稳定；
- VLM 评估虽做了 sanity check，但仍可能存在模型偏差。

## 6. 主要结论与发现

### 6.1 Replay 不是最终记忆指标

论文最重要的结论之一是：

> 低层 replay 质量和长期语义重访一致性并不单调相关。

例如：
- Spatial Memory 在 replay PSNR 上表现很好，但 open-domain VLM 分数很低；
- Block-wise State-Space replay PSNR 较低，但 open-domain VLM 分数最高。

这说明：
- replay 可以检查模型是否“健康”；
- 但不能证明模型真的记住了世界。

---

### 6.2 原始上下文是强容量基线

上下文长度从 \(K=1\) 增加到 \(K=5\)、\(K=20\) 后：

- Open-domain VLM：
  - \(K=1\)：12.25；
  - \(K=5\)：50.75；
  - \(K=20\)：58.63。

这说明：
- 原始历史观测对语义返回帮助极大；
- 其提升远大于 replay PSNR 的提升；
- 因此所有紧凑记忆机制都应该与 Context K=20 对比，而不是只与 I2V baseline 对比。

---

### 6.3 紧凑记忆不是容量的免费替代

论文发现：
- Spatial Memory 虽然在 replay 上表现不错，但 open-domain VLM 只有 6.00；
- Hybrid compression 的 open-domain 分数也很低；
- 紧凑空间摘要或混合压缩容易丢失返回时所需的 identity evidence。

因此：
- 压缩不能只追求 token 更少；
- 空间记忆不能只保留粗略布局；
- 需要保留显著物体、关键视角和可重访证据。

---

### 6.4 读出路径和存储同样重要

Spatial read-out 消融表明：
- 即使模型存储了空间 token，如果生成器无法在返回时有效读出，也不能形成真正记忆。
- inject-none 在 replay 上甚至最好，说明 replay 提升可能来自正则化，而不是真正使用了记忆。
- dedicated cross-attention read-out 能提升 open-domain VLM，但仍远低于 raw context 和 block-wise State-Space。

结论是：
- “存了什么”和“怎么读出来”必须分别评估；
- 只报告一个 replay 分数会掩盖这两个问题。

---

### 6.5 压缩方式影响很大

Compression 消融显示：

- Weight-only open-domain VLM：22.38；
- Length \(r=4\)：43.25；
- Hybrid \(r=2+\) weight：8.75；
- Hybrid \(r=4+\) weight：9.00。

这说明：
- 保留更多 temporal positions 不一定更好；
- 较短但更干净的上下文可能更容易被生成器利用；
- 混合压缩可能先破坏关键帧，再对破坏后的表示加权，导致身份信息丢失。

---

### 6.6 Block-wise State-Space 是最强开放域返回机制

State-Space 两种结构对比：

- Legacy hybrid open-domain VLM：34.75；
- Block-wise open-domain VLM：69.00。

Block-wise recurrence 虽然 replay PSNR 较低，但开放域返回最好。

论文认为：
- 隐式记忆是否有效取决于递归结构是否足够深度集成；
- 如果 recurrent state 可以被网络绕过，就可能只起到平滑作用；
- block-wise recurrence 使状态在多个 DiT block 中反复读写，更像真正的长期记忆。

---

## 7. 优点与亮点

### 7.1 受控实验设计非常清晰

论文最大优点是控制变量严谨：

- 同一视频 DiT backbone；
- 同一数据接口；
- 同一相机 RT 表示；
- 同一训练预算；
- 同一采样与评测流程；
- 只改变 memory profile。

这使得不同结果更能归因于记忆机制本身，而不是系统工程差异。

---

### 7.2 将“动作遵循”和“世界记忆”分开评估

三分支评测协议很有价值：

- Replay：模型是否能跟随轨迹；
- In-domain return：熟悉分布中的闭环一致性；
- Open-domain return：真正的语义重访能力。

这种设计揭示了传统 replay 指标无法发现的记忆失败。

---

### 7.3 提供强 baseline：Raw Context K 曲线

论文没有只用 I2V 作为 baseline，而是加入了 \(K=5\)、\(K=20\) 原始上下文。

这点很重要：
- I2V 是弱 baseline；
- Context K=20 是强容量基线；
- 可以防止夸大压缩记忆或新机制的收益。

---

### 7.4 消融实验具有解释性

每个消融都对应一个明确问题：

- Spatial read-out：存储和读取是否分离？
- Compression：压缩是否保留关键证据？
- State-Space：递归结构是否重要？
- Context length：容量增加带来什么收益？

论文不是简单排名，而是从不同指标冲突中分析机制差异。

---

### 7.5 开放域 VLM 评估设计有现实意义

通过 Qwen Edit 插入 distinctive identity anchor object，使得重访任务更接近真实交互世界模型需求：

- 相机离开；
- 再返回；
- 判断是不是同一个物体和同一个场景。

这种 probe 对“世界一致性”比单纯 pixel metric 更敏感。

---

## 8. 不足与局限

### 8.1 数据集覆盖有限

论文主要在 Context-as-Memory dataset 上训练和评测。

局限包括：
- 数据分布单一；
- 相机轨迹类型有限；
- 结论可能受该数据集场景、纹理和 pose 质量影响；
- 在更复杂动态环境、真实机器人视角或更噪声动作条件下，排序可能变化。

---

### 8.2 开放域评测样本规模偏小

文中指出部分 open-domain revisit 数字来自每个 row 约 8 个 cases 的 material snapshot。

因此：
- 小于 5 分左右的 VLM 差异不应过度解释；
- 更大规模的人类评估会更稳健；
- 对不同场景类别、物体类别、长 horizon 的覆盖还不够。

---

### 8.3 VLM judge 存在偏差风险

虽然论文做了 cross-judge sanity check，并发现 Qwen3-VL、Claude、GPT 和人工之间相关性较高，但 VLM 评估仍可能存在：

- 对某些物体类别更敏感；
- 对颜色 / 形状的偏好；
- 对背景一致性判断不稳定；
- 无法完全替代人工系统性标注。

---

### 8.4 训练预算较小，不能代表充分收敛

所有模型仅训练 5k steps。

这有助于公平对比，但也带来限制：

- 某些复杂记忆机制可能需要更长训练才能发挥；
- Spatial Memory 或 Compression 可能在更大模型 / 更长训练下改善；
- Block-wise State-Space 的优势是否在更大规模下保持仍需验证。

---

### 8.5 没有提出真正新的记忆模块

Echo-Memory 的贡献主要是评测框架和受控分析，而不是新的 SOTA 模型。

因此：
- 它更像 benchmark / empirical study；
- 对实际系统部署的直接提升有限；
- 未来仍需设计更强的对象级记忆、选择性压缩或可读写空间记忆模块。

---

### 8.6 主要关注静态场景与相机返回

论文重点是相机离开后返回时的场景 / 物体一致性。

但真实世界模型还需要处理：
- 动态物体状态变化；
- 交互导致的环境变化；
- 多智能体；
- 长时间物理因果；
- 用户编辑后的 persistent world update。

这些问题在本文中不是重点。

---

## 总结

Echo-Memory 的核心价值在于，它把动作世界模型中的“记忆”从混杂系统因素中剥离出来，构建了一个受控比较矩阵。论文证明：高 replay 图像质量并不等于模型记住了世界；原始上下文是强大的容量基线；压缩和空间摘要若没有对象级保留和有效读出，会丢失重访所需的关键证据；而 block-wise State-Space recurrence 展现出当前最强的开放域语义返回能力。整体上，这篇论文为后续研究长期一致、可重访、可交互世界模型提供了清晰的评测框架和设计准则。
