## 摘要
自动驾驶需要推理自车动作如何塑造未来世界的演化，而不仅仅是将观测映射为动作。然而，大多数端到端方法依赖直接的状态到动作模仿，而现有世界模型往往与下游策略生成的对齐较弱。我们提出 Discrete-WAM，一个统一的离散视觉—动作世界—策略框架，在共享令牌空间中表示视觉观测、未来状态、高层决策和自车动作。基于这种离散对齐，Discrete-WAM 通过多任务和多阶段预训练联合训练世界建模、世界—策略建模和策略建模，使动作条件下的未来预测能够直接支持策略生成。对于下游规划，Discrete-WAM 进一步将策略生成分解为分层决策预测和并行动作令牌编辑，其中决策令牌提供高层规划骨架，基于置信度的调度则高效地细化密集未来动作。在大规模自动驾驶基准上的实验表明，Discrete-WAM 在取得强规划性能的同时，还支持可控未来生成、反事实评估、基于惊讶度的世界模型分析以及高效的并行策略解码。这些结果表明，离散表征对齐、统一世界—策略训练和分层令牌编辑为物理 AI 提供了一种有前景的设计范式。

## Abstract
Autonomous driving requires reasoning about how ego actions shape future world evolution, rather than merely mapping observations to actions. However, most end-to-end methods rely on direct state-to-action imitation, while existing world models often remain weakly aligned with downstream policy generation. We introduce Discrete-WAM, a unified discrete vision-action world-policy framework that represents visual observations, future states, high-level decisions, and ego actions within a shared token space. Built on this discrete alignment, Discrete-WAM jointly trains world modeling, world-policy modeling, and policy modeling through multi-task and multi-stage pretraining, allowing action-conditioned future prediction to directly support policy generation. For downstream planning, Discrete-WAM further decomposes policy generation into hierarchical decision prediction and parallel action-token editing, where the decision token provides a high-level planning skeleton and confidence-based scheduling refines dense future actions efficiently. Experiments on large-scale autonomous-driving benchmarks show that Discrete-WAM achieves strong planning performance while supporting controllable future generation, counterfactual evaluation, surprise-based world-model analysis, and efficient parallel policy decoding. These results suggest that discrete representation alignment, unified world-policy training, and hierarchical token editing provide a promising design paradigm for physical AI.

## 2.2 模型结构

Discrete-WAM 包含四个主要组件：

- **视觉 VQ Tokenizer**
  - 使用 VQ-VAE 风格 tokenizer 将前视图像编码为离散视觉 token。
  - 视觉 codebook 大小为 `KV = 16384`。
  - 图像被划分为非重叠 patch，并编码为视觉 token 序列。

- **动作 Tokenizer**
  - 将未来轨迹转换为自车坐标系下的二维加速度序列：
    - 纵向加速度 `ax`
    - 横向加速度 `ay`
  - 每个加速度维度被划分为 60 个 bin。
  - 因此动作词表大小为：
    - `60 × 60 = 3600`
  - 动作 token 表示二维加速度原型。

- **上下文编码器**
  - 注入历史视觉观测、自车状态、导航命令等信息。

- **Decoder-only Transformer**
  - 统一处理视觉 token、动作 token、决策 token 和上下文 token。
  - 模型规模约为 **1B 参数**。
  - Transformer 配置：
    - hidden size：2048
    - 18 层
    - 16 个 attention heads
    - 8 个 key-value heads
    - 使用 grouped-query attention
    - 使用 Qwen3-MRoPE 位置编码。

## 2.3 动作离散化与软标签插值

论文认为，直接将连续动作硬量化为单个离散 token 会产生确定性量化误差，因此提出 **soft-label action interpolation**。

- 对连续加速度 `(ax, ay)`：
  - 找到其所在二维加速度网格单元；
  - 使用周围四个相邻加速度原型进行双线性插值；
  - 生成一个分布式 soft label，而不是 one-hot token。

- 训练时：
  - 动作头使用 soft-label cross-entropy 进行优化。

- 推理时：
  - 可以将预测分布对加速度词表求加权和，恢复连续加速度。

- 理论上：
  - 如果模型完全预测出 soft target 分布，则在网格单元内可精确恢复原始连续加速度；
  - 剩余误差来自模型分布预测误差，而不是硬量化误差。

## 2.4 三类统一训练任务

Discrete-WAM 通过统一 token 编辑接口组织三类任务。

### 1. 世界建模：World Modeling

- 目标：给定上下文和未来动作，预测未来视觉。
- 形式为：

```text
pθ(Vt+1:t+H | Ct, At+1:t+H)
```

- 含义：
  - 输入历史场景上下文 `Ct` 和未来动作 token；
  - 预测未来视觉 token；
  - 学习动作条件下的世界演化。

---

### 2. 策略建模：Policy Modeling

- 目标：先预测高层决策，再预测未来动作。
- 形式为：

```text
pψ,θ(At+1:t+H, Dt | Ct)
= pψ(Dt | Ct) pθ(At+1:t+H | Ct, Dt)
```

- 其中：
  - `Dt` 是高层决策 token；
  - 表示低频规划结构，例如：
    - 机动意图；
    - 目标车道；
    - 粗略路径；
    - 速度趋势；
    - 交互优先级。

- 作用：
  - 高层决策 token 提供规划骨架；
  - 低层动作 token 负责生成平滑、连续、细粒度轨迹。

---

### 3. 世界—策略建模：World-Policy Modeling

- 目标：联合预测未来动作和未来视觉。
- 将未来动作 token 和未来视觉 token 交替排列：

```text
Yt+1:t+H = [At+1, Vt+1, ..., At+H, Vt+H]
```

- 建模形式为：

```text
pθ(Yt+1:t+H | Ct)
= Πh pθ(At+h | Ct, Yt+1:t+h-1)
     pθ(Vt+h | Ct, Yt+1:t+h-1, At+h)
```

- 含义：
  - 动作生成和未来世界预测不再分离；
  - 模型学习“生成某个动作后，世界将如何变化”。

---

## 2.5 Token 编辑与注意力掩码

- Discrete-WAM 使用 token-editing 形式训练：
  - 部分 token 被扰动或替换；
  - 模型学习将 corrupted token 编辑回 clean token。
- 不同任务使用不同 attention mask：
  - 世界建模：未来动作作为干净条件，未来视觉作为编辑目标；
  - 策略建模：动作 token 块内部可双向注意，便于并行优化整段轨迹；
  - 世界—策略建模：视觉和动作 token 都可作为编辑目标，但 clean target 与 noisy token 隔离，避免信息泄漏。

---

## 2.6 损失函数

总体损失是多个任务相关损失的加权和：

```text
L = λv Lv_cls
  + λa La_cls
  + λacc Lacc
  + λtraj Ltraj
  + λpos Lpos_cls
  + λs Ls_cls
  + λdec Ldec
```

各项含义：

- `Lv_cls`：视觉 token 分类损失；
- `La_cls`：动作 token 分类损失；
- `Lacc`：加速度回归损失；
- `Ltraj`：轨迹级回归损失；
- `Lpos_cls`：辅助位置分类损失；
- `Ls_cls`：特殊 token 分类损失；
- `Ldec`：决策 token 分类损失。

---

## 2.7 模式感知连续动作解码

论文指出，多模态动作分布如果直接取全分布期望，可能会产生不合理的“平均动作”。

例如：
- 左转和直行都是可能动作；
- 直接平均可能得到一个既不像左转也不像直行的轨迹。

为缓解该问题，作者提出 **mode-aware decoding**：

- 对动作分布拟合 1、2、3 个 Gaussian mixture；
- 选择拟合误差最低的 GMM；
- 通过 top-p sampling 选择一个 mode；
- 只在该 mode 内重新归一化并计算连续加速度；
- 该连续加速度用于轨迹回归和辅助监督。

---

## 2.8 多阶段训练流程

论文采用三阶段训练：

- **阶段一：视觉预训练**
  - 使用世界建模和世界—策略建模任务；
  - 未来动作通过 teacher forcing 作为条件；
  - 只使用视觉预测损失；
  - 目标是让视觉表示对未来世界演化敏感。

- **阶段二：视觉和动作联合训练**
  - 同时激活视觉预测和动作预测损失；
  - 学习动作生成与未来视觉演化之间的耦合关系。

- **阶段三：动作 LoRA 微调**
  - 使用 LoRA 对策略部分进行微调；
  - 重点优化未来动作生成和 refinement；
  - 同时尽量保留前面学到的世界表示。

---

## 2.9 后训练：GRPO 强化学习微调

为提升长尾、安全关键场景表现，论文引入 RL post-training。

- 对每个场景上下文 `Ct`：
  - 采样多组候选轨迹；
  - 使用在线 reward 函数评估，例如 PDMS / EPDMS；
  - 使用 GRPO 风格目标优化策略。

- 同时优化：
  - 高层决策分布；
  - 低层动作 token 分布。

- 论文给出的目标形式近似为：

```text
Eτ~π Σk∈{A,D} [
  1/G Σi min(ρki Ai, clip(ρki, 1-ε, 1+ε) Ai)
  - KL(πkθ,ψ || πk_ref)
]
```

其中：
- `A` 表示动作；
- `D` 表示决策；
- `Ai` 是基于轨迹 reward 的 advantage；
- `ρki` 是重要性采样比；
- KL 项用于约束策略不要偏离 reference policy 太远。

## 6. 主要结论与发现

- **规划性能强**：
  - 在 NAVSIM-v2 上，Discrete-WAM 达到 **90.4 EPDMS**；
  - 在 NAVSIM-v1 上，达到 **92.2 PDMS**；
  - 在多个安全、舒适、进展相关指标上表现具有竞争力。

- **世界生成质量高**：
  - 在世界生成对比中取得：
    - FID：6.6；
    - FVD：80.0；
  - 优于 Vista、DrivingWorld 等对比方法。

- **统一世界—策略预训练有效**：
  - LoRA-SFT 优于从头训练和 full finetuning；
  - 说明保留预训练世界—策略表示对规划有帮助。

- **高层决策 token 有明显作用**：
  - Base-Dt 相比 Base 显著提升 EPDMS；
  - 决策 token 能作为低频规划骨架，降低动作 token 之间的残余依赖。

- **RL 后训练进一步提升性能**：
  - 从 SFT-Dt 到 RL，EPDMS 进一步提升；
  - 说明轨迹级 reward 优化可改善安全、舒适和决策选择。

- **置信度调度有助于并行 token 编辑**：
  - full_replace 多轮编辑可能破坏已有合理预测；
  - confidence-based selective replacement 能保留高置信 token，只细化不确定 token，从而改善 L2 误差。

- **surprise 可反映安全风险**：
  - 反事实动作导致越界、碰撞等情况时，surprise 明显升高；
  - surprise 与 PDMS 呈负相关，说明世界模型捕捉到一定动作条件因果动态。

## 7. 优点与亮点

- **统一离散 token 空间设计清晰**
  - 将视觉、动作、未来状态、决策放入统一序列建模框架；
  - 有利于跨模态对齐和共享 Transformer 计算。

- **动作离散化处理较细致**
  - 使用二维加速度词表；
  - 引入 soft-label interpolation，减少硬量化误差；
  - 给出了理论解释和误差界。

- **世界模型与策略模型深度耦合**
  - 不只是用世界模型作为辅助模块；
  - 而是通过 world-policy modeling 联合建模动作和未来视觉。

- **分层策略建模合理**
  - 高层决策 token 捕捉低频行为结构；
  - 低层动作 token 负责密集轨迹生成；
  - 符合自动驾驶规划的层次性。

- **并行 token 编辑具有推理效率优势**
  - 相比自回归逐步生成，离散扩散式并行编辑在中低轮数下延迟更低。

- **实验分析维度丰富**
  - 不仅有 benchmark 分数；
  - 还有注意力可视化、反事实分析、调度动态、推理延迟和多项消融。

- **后训练同时优化决策和动作**
  - 不只是在固定决策下优化轨迹；
  - 而是通过 GRPO 同时更新高层决策与低层动作策略。

---

## 8. 不足与局限

- **真实闭环验证不足**
  - 论文主要在 NAVSIM 伪仿真 / benchmark 上评估；
  - 缺少真实车辆闭环测试或更高保真仿真验证。

- **benchmark 适配风险**
  - RL post-training 使用 PDMS / EPDMS 风格 reward，可能对 NAVSIM 指标有较强适配；
  - 泛化到其他评测协议或真实道路时仍需验证。

- **数据分布较集中**
  - 预训练和评估主要依赖 nuPlan / NAVSIM；
  - 对其他国家、城市、天气、传感器配置、交通规则的泛化能力尚不清楚。

- **世界生成 horizon 较短**
  - 虽然 FID / FVD 表现好，但生成时长为 4 秒 / 8 帧；
  - 一些对比世界模型支持更长 rollout；
  - 长时累积误差和交互一致性仍待研究。

- **模型和训练资源成本较高**
  - 约 1B 参数；
  - 使用 32 张 NVIDIA H20 GPU；
  - 对普通研究团队复现存在较高门槛。

- **推理延迟比较不完全代表部署性能**
  - AR baseline 未使用 KV-cache；
  - 离散扩散也未使用额外工程优化；
  - 因此实验更像理论解码范式比较，而不是完整部署速度评估。

- **注意力解释仍有限**
  - 论文观察到天空区域存在稳定 attention；
  - 作者解释为 attention sink 或 implicit register，但这仍是推测；
  - 注意力图不能直接证明因果依赖。

- **决策 token 构造依赖人工设计**
  - 决策空间由横向路径候选和纵向速度 profile 的笛卡尔积构成；
  - 虽然有效，但仍带有一定手工结构先验；
  - 对更复杂交互行为的表达能力可能受限。

- **多模态动作处理仍存在近似**
  - mode-aware decoding 需要 GMM 拟合和 mode 选择；
  - 这会引入额外近似误差；
  - 在高度多模态或长尾场景中稳定性仍需进一步验证。
