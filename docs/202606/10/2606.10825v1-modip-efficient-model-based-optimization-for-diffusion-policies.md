## 摘要
扩散策略（DP）已成为机器人学习中一种富有表达力的策略表示，常与行为克隆（BC）等模仿学习方法结合使用。然而，尽管其成功主要局限于行为克隆，直接进行强化学习（RL）微调仍然具有挑战性，因为动作是通过多步去噪过程生成的。在本文中，我们提出 MODIP，一个用于扩散策略离线到在线微调的框架。MODIP 并不直接将强化学习应用于扩散策略，而是利用世界模型（WM）来指导策略适应，同时保持行为克隆的简洁性与稳定性。我们采用模型预测控制（MPC）在世界模型内生成高质量轨迹，并将其作为监督目标来微调扩散策略。为提高 MPC 规划效率，MODIP 使用终端状态价值而非依赖策略的状态—动作价值，从而减少推理时间。此外，MODIP 使用与策略无关的 TD 目标训练评论器，降低训练时间。在 D4RL（MuJoCo、Kitchen）和 RoboMimic 任务上的实验表明，MODIP 能够在行为克隆的基础上进一步提升扩散策略的性能，并且与扩散策略的强化学习微调方法以及 TD-MPC2 等强基于模型基线相比具有竞争力或表现更优。

## Abstract
Diffusion policies (DPs) have emerged as expressive policy representations for robot learning, often used with imitation learning methods such as behavioral cloning (BC). However, while their success has largely been confined to BC, direct reinforcement learning (RL) fine-tuning remains challenging because actions are generated through a multi-step denoising process. In this work, we propose MODIP, a framework for the offline-to-online fine-tuning of DPs. Rather than directly applying RL to the DPs, MODIP leverages a world model (WM) to guide policy adaptation and keeps the simplicity and stability of BC. We utilize model predictive control (MPC) to generate high-quality trajectories within the WM, and use them as supervised targets for fine-tuning the DP. To make MPC planning efficient, MODIP uses a terminal state value instead of a policy-dependent state-action value, reducing inference time. Additionally, MODIP trains critics with policy-independent TD targets, reducing training time. Experiments on D4RL (MuJoCo, Kitchen) and RoboMimic tasks show that MODIP improves diffusion policies beyond BC, and is competitive with or outperforms diffusion policy RL fine-tuning methods and strong model-based baselines such as TD-MPC2.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体思路

MODIP 的核心思想是：

- 先用离线数据训练一个扩散策略和一个潜在世界模型；
- 在线阶段不直接用 RL 更新扩散策略，而是：
  1. 用扩散策略作为 MPC 的动作序列先验；
  2. 在世界模型中通过 MPPI / MPC 搜索更优动作序列；
  3. 将 MPC 生成的高质量轨迹存入在线 buffer；
  4. 将离线数据和在线规划数据混合；
  5. 用标准扩散行为克隆损失继续训练扩散策略。

因此，MODIP 将策略优化拆解为：

- **规划器负责改进动作质量**；
- **扩散策略负责模仿规划器生成的数据分布**。

---

### 2.2 模型组件

MODIP 包含五类主要学习模块：

- **编码器** \(h_\eta\)：将观测状态 \(s_t\) 编码为潜在状态 \(z_t\)。
- **潜在动力学模型** \(f_\theta\)：根据当前潜在状态和动作预测下一潜在状态。
- **奖励模型** \(\hat{R}_\xi\)：预测短时域 rollout 中的即时奖励。
- **终端状态价值函数** \(V_\psi(z)\)：估计 MPC 规划时域之后的长期回报。
- **扩散策略** \(\pi_\phi\)：作为多模态动作序列先验，同时在训练中被 MPC 轨迹蒸馏更新。

---

### 2.3 MPC-to-policy distillation：用 MPC 轨迹监督微调扩散策略

MODIP 的策略更新仍然采用标准扩散模型的去噪行为克隆目标：

\[
L_{DP}(\phi)=\mathbb{E}_{a,k,\epsilon}\left[\|\epsilon-\epsilon_\phi(a^k,k,h_\eta(s_t))\|_2^2\right]
\]

与普通 BC 不同的是：

- 普通 BC 的动作来自离线专家数据；
- MODIP 在线阶段的动作主要来自 **MPC 改进后的轨迹**。

因此，扩散策略不是通过 RL loss 直接提升，而是通过更强的 MPC 轨迹数据分布实现提升。

此外，论文还引入 frozen BC regularization：

\[
L_\pi = L_{DP}(a_{MPC}) + \lambda(t)L_{DP}(a_{reg})
\]

其中：

- \(a_{MPC}\)：MPC 生成的动作；
- \(a_{reg}\)：冻结的原始 BC 扩散策略生成的动作；
- \(\lambda(t)\)：随训练逐渐退火，早期防止策略偏离离线先验过快。

---

### 2.4 DP-guided MPC：用扩散策略作为 MPC 的动作先验

在在线交互时，MODIP 不直接执行扩散策略，而是用它辅助 MPC 搜索。

MPC 优化目标为：

\[
A^*=\arg\max_A
\left[
\sum_{k=0}^{H_{mpc}-1}\gamma^k\hat{R}_\xi(\hat{z}_{t+k},a_{t+k})
+
\gamma^{H_{mpc}}V_\psi(\hat{z}_{t+H_{mpc}})
\right]
\]

其中：

- 通过潜在动力学模型 rollout：
  \[
  \hat{z}_{t+k+1}=f_\theta(\hat{z}_{t+k},a_{t+k})
  \]
- MPC 短时域奖励由奖励模型估计；
- 长期回报由终端状态价值 \(V(z)\) 估计。

MPPI 采样时使用混合 proposal：

- 一部分候选动作序列由扩散策略生成；
- 一部分候选动作序列从 Gaussian proposal 采样；
- 之后通过 MPPI 选择 elite trajectories，并更新采样分布。

这种设计的意义是：

- 扩散策略提供高质量、多模态、任务相关的动作序列先验；
- Gaussian 采样保留局部探索；
- 世界模型 MPC 对动作序列进行短时域优化。

---

### 2.5 使用终端状态价值 \(V(z)\) 替代 \(Q(z,\pi(z))\)

传统 hybrid MPC 常用终端估计：

\[
Q(z_{t+H}, \pi(z_{t+H}))
\]

但如果 \(\pi\) 是扩散策略，则每次计算终端动作都要执行多步去噪，代价很高。

MODIP 改用：

\[
V(z_{t+H})
\]

优势是：

- 只需一次前向传播；
- 避免在每个候选轨迹终端状态调用扩散策略；
- 显著降低 MPC 推理时间；
- 实验中还带来更好的性能。

---

### 2.6 策略无关 critic 学习

标准 actor-critic 中 TD target 常依赖当前策略，例如：

\[
Q(s',\pi(s'))
\]

这对扩散策略非常昂贵，因为每次 \(\pi(s')\) 都需要多步 denoising。

MODIP 使用策略无关 TD target：

\[
y_t = r_t + \gamma(1-d_t)\min_{j\in\{1,2\}}\bar{V}_{\psi_j}(\bar{h}_\eta(s_{t+1}))
\]

然后训练 Q 函数：

\[
L_Q(\omega)=
\mathbb{E}_{(s,a,r,s')\sim B}
\left[
(Q_\omega(h_\eta(s),a)-y_t)^2
\right]
\]

特点：

- TD target 只依赖下一状态的 value，不需要查询当前扩散策略；
- 减少训练时对 DP 的调用；
- 降低 critic 训练时间；
- 降低策略误差传递到 critic target 的风险。

此外，论文还训练：

- 两个状态价值头 \(V_{\psi_1}, V_{\psi_2}\)；
- 两个优势函数头 \(A_{\zeta_1}, A_{\zeta_2}\)；
- 用 value-advantage 分解恢复策略无关的状态价值估计。

---

### 2.7 离线到在线训练流程

整体流程为：

1. **离线预训练阶段**：
   - 用离线数据训练扩散策略；
   - 训练潜在世界模型、奖励模型、Q / V / A critic。
2. **在线 fine-tuning 阶段**：
   - 当前状态输入编码器得到潜在状态；
   - 扩散策略生成动作序列候选；
   - MPPI / MPC 在世界模型中优化轨迹；
   - 执行优化序列中的第一个动作或少量动作；
   - 将真实交互轨迹存入在线 replay buffer；
   - 按时间变化比例混合离线数据与在线数据；
   - 用混合数据继续训练世界模型、critic 和扩散策略。
3. **数据混合策略**：
   - 在线样本比例 \(\beta(t)\) 从较小值逐渐增加；
   - 早期依赖离线数据稳定训练；
   - 后期更多使用 MPC 改进后的在线数据。

- **MODIP 与直接 RL 微调 DP 的方法相比具有竞争力甚至更优**：
  - 在 Kitchen 任务上表现尤其强；
  - 在 MuJoCo 上也整体优于或接近最强 baseline。
- **DP 作为 MPC prior 比 MLP prior 更有效**：
  - 消融显示，将 DP prior 替换成 MLP policy 后，性能明显下降，尤其在 Hopper 和 Kitchen 上。
- **终端状态价值 \(V(z)\) 比 \(Q(z,\pi(z))\) 更高效且性能更好**：
  - 不仅推理速度提升约 2.9 倍；
  - 性能也在多个任务上优于 Q-based terminal estimate。
- **策略无关 critic target 能显著降低训练成本**：
  - 避免在 TD target 中调用扩散策略；
  - 总训练时间从约 40 小时降到约 25 小时。
- **MPC-to-policy distillation 是有效的策略改进机制**：
  - 通过规划器生成更好数据，再用 BC 蒸馏，可以绕开直接 RL 优化扩散策略的不稳定性。

## 7. 优点与亮点

### 7.1 方法设计亮点

- **避免直接 RL 优化扩散策略**：
  - 保留扩散策略训练的稳定监督学习目标；
  - 减少 policy-gradient 或 Q-guided diffusion training 的不稳定性。
- **将扩散策略作为 MPC 的多模态动作序列先验**：
  - 能利用 DP 对复杂动作分布的建模能力；
  - 比 Gaussian / MLP prior 更适合多模态机器人任务。
- **MPC-to-policy distillation 形成闭环改进**：
  - MPC 生成更优轨迹；
  - DP 模仿 MPC；
  - 更强 DP 又为下一轮 MPC 提供更好 prior。
- **终端状态价值设计非常实用**：
  - 直接解决 DP 在 MPC 终端状态处调用昂贵的问题；
  - 同时提升速度和性能。
- **策略无关 critic 学习降低训练成本**：
  - 对扩散策略这类高成本 actor 特别有意义；
  - 与 IQL / AFU 一类思想有相通之处，但被用于 model-based DP optimization。
- **离线到在线数据混合合理**：
  - 初期稳定依赖离线数据；
  - 后期逐渐转向在线 MPC 改进数据；
  - 有助于避免早期在线数据质量差导致训练崩溃。

### 7.2 实验设计亮点

- 覆盖多个任务类型：
  - dense reward locomotion；
  - sparse / long-horizon kitchen manipulation；
  - state-based robotic manipulation。
- 对比方法较全面：
  - 包括 DP RL、critic-guided policy improvement、model-based TD-MPC2。
- 消融实验紧扣方法贡献：
  - 能较清楚地说明每个组件的作用。

## 8. 不足与局限

### 8.1 输入模态限制

- 论文明确指出，所有实验都使用**低维状态向量**作为输入。
- 虽然扩散策略常用于视觉机器人学习，但本文没有在图像输入或像素观测任务上验证。
- 因此，MODIP 是否能扩展到视觉控制仍未被证明。

### 8.2 世界模型误差风险

- MODIP 依赖 learned latent world model 进行 MPC。
- 如果世界模型不准确，MPC 可能利用模型误差，生成在真实环境中表现不佳的轨迹。
- 虽然论文通过短时域 MPC、receding horizon 和 offline initialization 缓解该问题，但模型偏差仍是潜在风险。

### 8.3 硬件与复现信息不足

- 未报告 GPU 型号、数量和详细硬件设置。
- 虽然报告了部分训练时间，但缺少硬件上下文，难以评估实际算力门槛。
- 对实际机器人系统部署的实时性也缺少更细致分析。

### 8.4 与 baseline 的完全公平性仍有疑问

- 部分 baseline 结果来自已有文献。
- 不同方法的实现、调参、计算预算、动作 chunk 设置可能存在差异。
- 尽管论文称固定计算预算，但具体统一程度仍需更多细节支持。

### 8.5 真实机器人验证缺失

- 实验均为模拟环境或 benchmark。
- 未在真实机器人平台上测试，因此：
  - sim-to-real 鲁棒性未知；
  - 传感噪声、延迟、动力学偏差下的效果未知；
  - 在线 fine-tuning 的安全性也未被验证。

### 8.6 方法复杂度较高

- MODIP 包含 DP、world model、reward model、Q/V/A critic、MPC planner、online buffer 等多个模块。
- 相比单纯 BC 或 model-free DP fine-tuning，系统实现复杂度更高。
- 超参数也较多，例如 MPC horizon、MPPI samples、online mix ratio、BC regularization annealing 等，实际应用可能需要较多调参。
