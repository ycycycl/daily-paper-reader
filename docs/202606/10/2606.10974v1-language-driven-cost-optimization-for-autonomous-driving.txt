# Language-Driven Cost Optimization for Autonomous Driving 

Diego Martinez-Baselga 1, Khaled Mustafa 1, Javier Alonso-Mora 1

Abstract — The driving behavior of autonomous vehicles is typically governed by the cost function of their motion planner, which encodes objectives such as speed tracking, smoothness, lane keeping, and collision avoidance. However, tuning the parameters that shape this cost function is a challenging task that requires technical expertise, limiting the vehicle’s ability to adapt to evolving traffic scenarios or end-user preferences. This work presents a language-driven framework for adaptive cost design in autonomous driving. A Large Language Model (LLM) interprets structured scenario descriptions and natural language user queries to generate the parameters applied to a risk-aware Model Predictive Path Integral (MPPI) controller. The system incorporates a human-in-the-loop validation stage in which the proposed behavioral changes are described in non-technical language and confirmed prior to deployment. Users may additionally provide feedback either before or after deployment, enabling iterative refinement of the vehicle’s motion behavior. The framework is evaluated across multiple queries in realistic driving scenarios to assess its effectiveness. Simulation results demonstrate that our method successfully induces behavioral changes that align with the intended re-quirements in an intuitive manner, thereby bridging the gap between intelligent vehicle control systems and end users. 

I. I NTRODUCTION 

Motion planning plays a central role in autonomous driv-ing systems. It is responsible for computing safe and dy-namically feasible trajectories that account for the vehicle’s dynamics, surrounding agents, and context. In many systems, motion behavior is encoded through a weighted cost function that is minimized by an optimal control or sampling-based planner, such as Model Predictive Control (MPC) or Model Predictive Path Integral (MPPI) control [1], [2]. The relative weighting of individual cost terms determines the resulting driving style. However, these weights are typically tuned offline through empirical evaluation and remain fixed during deployment. As a result, the planner lacks the ability to adapt its behavior online to scenario-specific contextual informa-tion, such as traffic density, road geometry, or interaction patterns with other agents. Passenger driving preferences may vary depending on urgency, comfort requirements, physical condition, or traffic conditions. While such preferences can be naturally ex-pressed in everyday language, translating qualitative instruc-tions into quantitative modifications of a motion planner’s cost function remains challenging. In this work, we propose a language-driven motion plan-ning framework that adapts cost function parameters based on scenario descriptions and natural language user queries. This design allows intuitive and flexible control of a risk-aware autonomous driving planner without requiring expertise in optimal control or cost design. The contributions of this work are threefold. First, we introduce an adaptive language-driven motion planning framework where the vehicle behavior adapts to diverse environments and user requirements via a rich risk-aware cost function optimized through MPPI, enabling an LLM module to adjust parameters on the fly while ensuring real-time safety. Second, we propose a human-in-the-loop vali-dation and refinement scheme that allows users to review, confirm, and iteratively refine LLM-generated parameter updates through a structured feedback loop to mitigate mis-understandings. Finally, we demonstrate the effectiveness of our framework through evaluation in a realistic simulator

Recent efforts explore interpretable end-to-end driving via LLM-based reasoning [26] and con-versational interfaces [27]. Our contribution advances this direction by incorporating a validation loop in which the LLM-generated behavioral modifications are translated into non-technical language and explicitly confirmed by the user prior to deployment, enhancing transparency and mitigating misalignment in motion planning behavior personalization. A. MPPI formulation 

Model Predictive Path Integral (MPPI) control is asampling-based approach for stochastic optimal control that leverages importance sampling to iteratively refine control inputs [28], [29]. At each planning step, MPPI generates 

K candidate control sequences {Vk}Kk=1 by sampling per-turbations around the nominal sequence. Our motion planning framework, presented in Fig. 1, leverages the reasoning capabilities of an LLM in order to interpret st and qt and tune the cost function of an MPPI planner. The MPPI planner runs in a high-frequency comput-ing the commands that minimize Jst,q t . The LLM module changes the MPPI cost function on demand, whenever the vehicle is in a new scenario or a new query is received. The user validates the changes before they are applied, and can modify the applied behavior by requesting new queries. It is therefore involved in two different loops to give feedback to the motion planner, always communicating with it using natural language. The MPPI cost formulation is explained in Sec. A. Cost Formulation 

A key component of MPPI is the design of the cost function Sk associated with each sampled control sequence. We design our cost function as: 

Sk = ωtCtracking + ωsCspeed + ωst Csteering + Ccontrol + Crisk (4) where ωt, ω s, ω st ≥ 0 are tunable weights. The tracking cost, Ctracking , penalizes the lateral deviation between the ego-vehicle and a reference path, encouraging lane-keeping or lane-changing behavior depending on the proposed reference path. The speed cost, Cspeed , penalizes deviations from a desired reference velocity profile. The steering cost, Csteering ,regularizes the steering angle to discourage excessive curva-ture, while Ccontrol penalizes control effort, i.e., longitudinal acceleration and steering rate respectively weighted by ωa ≥

0 and ωstr ≥ 0, thereby encouraging smooth and comfortable maneuvers. The risk term, Crisk , captures the probability of collision between the ego vehicle and surrounding agents along each sampled MPPI trajectory. At prediction timestep

B. LLM module 

The LLM module runs on demand and is decoupled from the MPPI control loop. The primary objective of this module is to interpret the scenario features st and user queries qt and to adapt the cost function parameters 

ω = {ωt, ω s, ω st , ω a, ω str , ω soft } accordingly. To this end, we designed a structured prompt, represented in Fig. 2, to guide the LLM toward reliable and interpretable parameter tuning. To facilitate correct interpretation, we include intuitive descriptions of each cost term and its behavioral implications, as depicted in Fig. 3, and a set of behavioral guidelines as in-context learning [31]. To promote coherent reasoning prior to numerical parameter selection, we adopt a Chain-of-Thought 

prompting strategy [32], [33]. The LLM is first instructed to provide a concise motivation explaining the intended behavioral adjustments. Only after this reasoning step, we ask the model to output the updated weight values in a predefined format, enabling automatic parsing and validation. Furthermore, we use a general purpose LLM, allowing the system to handle diverse natural language queries without ad-ditional fine-tuning. To ensure numerical stability and exploit the model’s pre-trained ordinal reasoning capabilities, we discretize the cost function weight space of the LLM into an integer scale. To mitigate these risks and enhance alignment between intended and executed behavior, we close the loop with the user in two different ways. 

1) Validation Before Deployment: Before the updated parameters are applied to the MPPI controller, the system enters a validation loop , illustrated in Fig. 1. In this loop, the user is presented only with the natural language description of the changes requested in the last step of the LLM query (Fig. 2), without specific information about the parameters or cost function terms. This description uses non-technical language, and conceptually explains the changes expected in the behavior of the vehicle. The aim of this loop is to ensure that the LLM has understood the query properly and find a confirmation of the changes from the user without requiring knowledge of control design. In case the user identifies inconsistencies or is not satisfied with the proposed changes, they may provide corrective feedback. This feedback is appended to the existing conversation history, which already includes the system prompt, user query, and LLM response. The LLM is then requested to complete the task again, but with its previous response and feedback to refine its answer. The process iterates until the user explicitly confirms the be-havioral modification. Only then are the updated parameters applied to the motion planner. This paper presented a motion planning framework that integrates a Large Language Model (LLM) with a risk-aware Model Predictive Path Integral (MPPI) motion planner to enable adaptive autonomous driving behavior. The MPPI planner maintains high-frequency control execution, while the LLM, based on the provided context or user intent on demand, proposes modifications to the cost function parameters of the MPPI controller. A human-in-the-loop validation mechanism enables users to confirm and refine behavioral changes prior to deployment, supporting trans-parent and reliable personalization. Simulation results in interactive driving scenarios demonstrate that the proposed method systematically adapts motion behavior to user intent and contextual requirements, including variations in smooth-ness, aggressiveness, efficiency, and safety prioritization. By enabling structured natural-language interaction with the motion planning layer, the framework helps bridge the gap between intelligent vehicle control systems and non-expert end users. The performance of our approach could be improved with a tighter integration with perception and scene-understanding modules, enabling automatic extraction of contextual sce-nario descriptions without requiring explicit textual input.
