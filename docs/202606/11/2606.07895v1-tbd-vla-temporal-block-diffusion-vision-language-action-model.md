## 摘要
离散视觉-语言-动作（VLA）模型通常将动作生成表述为在离散化动作空间上的下一个 token 预测，并以自回归方式使每个 token 以先前上下文为条件。尽管这一范式有效，但其推理延迟较高，并且在很大程度上忽视了动作轨迹中固有的时间结构。近期研究引入并行解码以提高效率，从而实现更快的推理，但缺乏用于建模 token 依赖关系的显式机制。我们提出 TBD-VLA，这是一种基于离散 token 的 VLA 框架，引入块扩散以实现时间动作生成。我们将动作序列划分为时间块，并在每个块内执行掩码离散扩散，同时在块之间保持自回归生成。该设计统一了时间自回归与并行动作解码，在实现强时间一致性的同时提升了推理速度。此外，显式的时间建模通过时间修复支持动作块的异步执行（例如实时分块，Real-Time Chunking）。TBD-VLA 在仿真和真实世界操作任务中均显著优于以往的 VLA 方法，为构建快速、具备时间感知能力的离散 VLA 模型提供了一条可扩展路径。项目网页：https://tbd-vla.github.io/

## Abstract
Discrete Vision-Language-Action (VLA) models typically formulate action generation as next-token prediction over discretized action spaces, conditioning each token autoregressively on prior context. While effective, this paradigm incurs high inference latency and largely ignores the temporal structure inherent in action trajectories. Recent efforts introduce parallel decoding to improve efficiency, enabling faster inference, but lack explicit mechanisms for modeling token dependencies. We introduce TBD-VLA, a discrete token-based VLA framework that incorporates block diffusion to enable temporal action generation. We partition action sequences into temporal blocks and perform masked discrete diffusion within each block, while maintaining autoregressive generation across blocks. This design unifies temporal autoregression and parallel action decoding, achieving both strong temporal coherence and improved inference speed. In addition, the explicit temporal modeling enables asynchronous execution of action chunks (e.g., Real-Time Chunking) via temporal in-painting. TBD-VLA significantly outperforms prior VLA approaches in both simulation and real-world manipulation tasks, offering a scalable path toward fast, temporally aware, discrete VLA models. Project webpage: https://tbd-vla.github.io/

## 1. 核心问题与整体含义：研究动机与背景

### 1.1 背景：VLA 模型中的动作生成范式

视觉-语言-动作模型（Vision-Language-Action, VLA）旨在让机器人根据视觉观测、语言指令和自身状态直接生成可执行动作，是通用机器人策略学习的重要方向。当前 VLA 模型大致有两类动作生成方式：

- **连续动作头方法**  
  例如 π0.5、GR00T-N1.5 等，在 VLM 骨干网络之上接入连续动作专家，通常采用 flow matching 等方式生成连续动作序列。
  - 优点：天然适合连续、多模态动作空间。
  - 缺点：动作生成模块与 VLM 主体相对解耦，难以分析 VLM 表征本身如何贡献于机器人动作泛化能力。

- **离散动作 token 方法**  
  将机器人动作离散化为 token，让 VLM 像生成文本一样直接生成动作 token。
  - 优点：动作生成与 VLM 的 token 生成范式统一，更容易利用语言模型能力。
  - 缺点：若采用逐 token 自回归生成，长动作块推理延迟很高，不适合高频闭环控制。

### 1.2 核心问题

论文关注的问题是：

> 如何在离散 VLA 框架中，同时实现低延迟动作生成、显式时间依赖建模和较强的动作轨迹一致性？

现有方法主要存在两类不足：

- **逐 token 自回归方法推理慢**  
  例如 OpenVLA、MolmoAct、π0-FAST 等需要按顺序生成动作 token，长动作 chunk 会导致较高延迟。

- **并行解码或离散扩散方法时间建模不足**  
  OpenVLA-OFT、Discrete Diffusion VLA 等通过并行解码降低延迟，但通常缺乏对动作序列中时间结构的显式建模，可能影响长时序动作一致性。

### 1.3 论文整体含义

TBD-VLA 提出一种折中框架：

- **块内并行生成**：在每个时间块内部使用掩码离散扩散并行预测动作 token。
- **块间自回归生成**：不同时间块之间保持顺序依赖，显式建模时间结构。

因此，TBD-VLA 试图在以下三者之间取得平衡：

- VLM 原生 token 生成能力；
- 并行解码带来的低延迟；
- 动作轨迹中的时间一致性和因果依赖。

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体思路

TBD-VLA 的核心思想是将动作序列划分为多个时间块，并使用 **Temporal Block Diffusion** 进行生成：

- 将未来动作 chunk 按时间划分为若干 block；
- 每个 block 内部使用 masked discrete diffusion 并行去噪；
- block 与 block 之间采用自回归方式生成；
- 从而结合：
  - 自回归模型的时间依赖建模能力；
  - 扩散模型的并行 token refinement 能力。

---

## 2.2 动作 token 化

论文将连续机器人动作离散化为 token：

- 设动作预测 horizon 为 \(H_p\)；
- 动作维度为 \(D_a\)；
- 每个动作特征被离散化为 \(N_b\) 个 bins；
- 一个动作 chunk 表示为长度为：

\[
L_t = H_p \cdot D_a
\]

的 token 序列。

论文中主要设置为：

- 动作预测 horizon：\(H_p = 16\)
- 时间块大小：\(m = 4\)
- 动作 bins 数量：\(N_b = 512\)

输入提示模板为：

> “State: {state tokens}, Task: {instruction}, Actions: {placeholder tokens}”

其中：

- state tokens 表示机器人 proprioception；
- instruction 表示语言任务；
- placeholder tokens 用于提示模型需要生成多少动作 token。

---

## 2.3 时间块自回归建模

动作序列被划分为 \(K\) 个时间块：

\[
K = H_p / m
\]

论文将动作生成概率分解为块级自回归形式：

\[
p(a_{1:H_p} | o, g)
=
\prod_{k=0}^{K-1}
p_\theta(a_{km+1:(k+1)m} | o, g, a_{1:km})
\]

其中：

- \(o\)：视觉观测和机器人状态；
- \(g\)：语言任务指令；
- \(m\)：每个时间块包含的时间步数；
- 第 \(k\) 个 block 的生成依赖于此前所有 block。

这使模型不是简单地一次性并行生成整个动作序列，而是在时间维度上保留了因果结构。

---

## 2.4 块内掩码离散扩散

对于每个时间块，TBD-VLA 采用 masked discrete diffusion：

- 给定干净动作 token block \(x_k^0\)；
- 随机采样每个 token 的 mask 概率；
- 将部分 token 替换为 `[MASK]`；
- 模型学习在上下文和当前 corrupt block 条件下恢复原始 token。

前向噪声过程可理解为：

- 对 block 内每个 token 独立决定是否 mask；
- 被 mask 的 token 作为模型预测目标；
- 未被 mask 的 token 作为当前 block 的局部条件。

训练损失为 masked token 上的交叉熵：

\[
L_\theta =
-
\frac{
\sum_{k,i}
\mathbf{1}[x^t_{k,i} = [MASK]]
\log p_\theta(x^0_{k,i} | z_k, x^t_k, o, g)
}{
\sum_{k,i}
\mathbf{1}[x^t_{k,i} = [MASK]]
}
\]

其中：

- \(x^t_k\)：被 mask 后的 corrupt block；
- \(x^0_k\)：干净 block；
- \(z_k\)：预测第 \(k\) 个 block 时的历史上下文；
- 第一个 block 使用全 mask anchor block 作为起始条件；
- 后续 block 使用此前 clean blocks 作为上下文。

---

## 2.5 Temporal-level Token Shift

由于 VLM 骨干本身是自回归 next-token prediction 模型，而离散扩散通常是自重构式任务，二者目标存在不匹配。

论文提出 **temporal-level token shift**：

- 当前动作 block 的 logits 由前一个 block 的位置生成；
- 使训练目标更接近 VLM 原本的自回归预测范式；
- 在 block 级别对齐 VLM 的 next-token 机制和 diffusion denoising 机制。

这相当于在时间块层面上做“shifted prediction”。

---

## 2.6 Doubled-layout 与自定义注意力掩码

为提高训练效率，论文采用 doubled-layout trick：

- 将 clean action sequence 和 corrupt action sequence 拼接输入模型；
- 两者共享相同 RoPE 位置编码；
- 通过自定义 attention mask 控制可见信息。

对于第 \(n\) 个 block 的预测，模型可以看到：

- 视觉和语言前缀；
- 之前的 clean action blocks；
- 当前的 corrupt action block；

但不能看到未来 clean blocks。

该设计允许模型在一次 forward 中并行学习多个 block 的去噪任务，同时保持块间自回归约束。

---

## 2.7 推理流程

推理时，TBD-VLA 从 fully masked action blocks 开始生成：

1. 输入视觉、状态和语言指令；
2. 根据需要生成若干 action blocks；
3. 对每个 block 执行 \(n_d\) 步离散扩散去噪；
4. 每一步预测所有 masked token；
5. 按置信度从高到低 commit token；
6. 当前 block 生成完成后进入下一个 block；
7. 最终将动作 token 反离散化为连续动作。

论文默认设置：

- 时间块大小 \(m = 4\)；
- 每个 block 扩散步数 \(n_d = 2\)。

---

## 2.8 Decode-as-needed

为了降低延迟，TBD-VLA 不一定生成完整预测 horizon 的所有 blocks，而是只生成当前执行需要的部分。

若实际 rollout 执行 horizon 为 \(H_a\)，则只生成：

\[
K_{exec} = \lceil H_a / m \rceil
\]

个 blocks。

因此推理所需 VLM forward 次数约为：

\[
K_{exec} \cdot n_d
\]

这显著减少了无用未来动作的生成。

---

## 2.9 Prefix KV Cache

推理中，视觉 token、语言 prompt token 和已生成 action blocks 在多轮 diffusion steps 中保持不变。

论文使用 KV cache 缓存这些上下文的 key-value states，从而避免重复计算，进一步降低推理延迟。

---

## 2.10 动作解码：Expectation Sampling

不同于直接取最大概率 token 的 argmax 解码，论文提出 expectation sampling：

对于动作维度 \(j\)、时间步 \(h\)，连续动作值为：

\[
a_{t+h,j}
=
\sum_{x \in V}
p_{\theta,h,j}(x)c_j(x)
\]

其中：

- \(p_{\theta,h,j}(x)\)：模型对 token \(x\) 的预测概率；
- \(c_j(x)\)：token 对应 bin 的连续动作值。

这种方式利用完整概率分布，比 argmax 更细粒度，有助于提升动作精度。

---

## 2.11 Real-Time Chunking，RTC

TBD-VLA 还支持 Real-Time Chunking：

- 机器人执行当前动作时，模型异步生成未来动作；
- 对由于推理延迟造成的 action tail 进行冻结和复用；
- 使用 in-painting 方式补全后续动作。

由于 TBD-VLA 训练时本身就是 masked block diffusion，因此天然适配 partial context completion，与 RTC 的推理需求对齐。

## 6.2 RTC 的作用

在 LIBERO 延迟实验中：

- 无 RTC 时，延迟 \(L=4\) 下成功率下降到 72.3%；
- 有 RTC 时，成功率保持在 93.2%。

说明 TBD-VLA 的 masked block diffusion 与时间补全机制很好地适配异步执行。

在真实世界中：

- TBD-VLA w/o RTC：60.0%；
- TBD-VLA w/ RTC：67.1%。

RTC 带来约 7.1 个百分点提升。

---

## 6.3 消融结论

论文发现：

- \(m = 1\) 时接近完全时间自回归，推理较慢，性能没有明显优势；
- \(m = 16\) 时接近整个 horizon 的标准离散扩散，时间建模不足；
- \(m = 4\) 在性能和延迟之间取得较好平衡；
- \(n_d = 1\) 虽然更快，但性能下降；
- expectation sampling 显著优于 argmax；
- decode-as-needed、KV cache、VLM compile 均有效降低推理延迟。

# 7. 优点与亮点

## 7.1 方法设计上的亮点

- **统一自回归和扩散解码**  
  不是简单选择自回归或并行扩散，而是在时间块层面结合二者优势。

- **显式建模动作时间结构**  
  block 间自回归使模型能够捕捉动作轨迹的时间依赖。

- **块内并行去噪降低延迟**  
  相比逐 token 自回归，块内 token 可并行生成，推理更快。

- **与 VLM 生成范式兼容**  
  temporal-level token shift 缓解了 VLM next-token prediction 与 diffusion denoising 之间的目标错配。

- **自然支持 RTC**  
  由于训练中已学习 partial masked completion，推理时进行时间补全和异步动作生成更加自然。

- **Expectation sampling 设计实用**  
  利用完整 token 概率分布恢复连续动作，比 argmax 更适合机器人控制。

---

## 7.2 实验设计上的亮点

- 覆盖仿真和真实机器人；
- 覆盖多个机器人平台：
  - Franka Panda；
  - Franka Research 3；
  - Google Robot；
  - Widow-X；
- 包含多种鲁棒性扰动；

- 不仅报告成功率，也报告推理速度和 forward pass 数；
- 消融较完整，能支撑主要设计选择。

# 8. 不足与局限

## 8.1 方法局限

- **仍然需要离散化动作空间**  
  动作被离散为固定 bins，可能带来量化误差，尤其对精细操作任务有影响。

- **block size 需要手动选择**  
  \(m = 4\) 是实验中较优配置，但不同任务、机器人和控制频率下可能需要重新调参。

- **块间仍是自回归**  
  虽然比逐 token 自回归快，但生成多个 block 时仍存在顺序依赖，无法完全并行。

- **扩散步数带来额外计算**  
  每个 block 需要多步 denoising，速度与质量之间仍需权衡。

---

## 8.2 实验局限

- **真实世界任务数量有限**  
  真实实验只有三个 tabletop 任务，仍不足以证明广泛真实世界泛化能力。

- **真实 baseline 较少**  
  真实机器人主要与 π0.5 对比，缺少 OpenVLA、UniVLA、Discrete Diffusion VLA 等更多真实实验对比。

- **预训练数据影响较大**  
  TBD-VLA 使用了大规模混合机器人数据，性能提升可能部分来自数据规模，而非完全来自模型结构。

- **部分 benchmark 对比可能非同等训练条件**  
  不同方法的预训练数据、模型规模、微调策略可能不完全一致。

- **某些 OOD 场景仍会失败**  
  例如真实世界 Transfer Liquid 任务在 camera viewpoint shift 下成功率为 0，说明视觉视角变化对精细操作仍是重大挑战。

---

## 8.3 应用限制

- 对高精度视觉定位任务仍较敏感；
- 对相机视角变化、训练数据覆盖不足的任务可能失败；
- 推理虽已优化，但在更高频控制或更低算力边缘设备上仍可能不足；
- 需要较大预训练数据和较高训练算力，不一定适合资源受限场景。

# 总体评价

TBD-VLA 是一篇针对离散 VLA 动作生成效率和时间建模问题的较强工作。其核心贡献在于将 block diffusion 引入动作 token 生成，使模型在块内实现并行扩散解码，在块间保留时间自回归结构，从而兼顾推理速度、时间一致性和泛化性能。实验覆盖仿真、鲁棒性、真实机器人、延迟补偿和消融分析，整体证据较充分。其主要不足在于真实世界任务规模仍有限，baseline 训练条件不完全统一，并且在某些视觉 OOD 场景下仍存在明显失败模式。
