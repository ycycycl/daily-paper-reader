## 摘要
大多数视觉-语言-动作（VLA）模型将观测直接映射为动作，而缺乏显式的中间规划，这限制了其在长时程任务中的性能，因为早期错误会不断累积。我们提出 Coarse-to-Control，一种规划-执行式 VLA，在动作标记空间中原生引入规划。其核心思想是让策略首先预测一段紧凑的粗粒度动作标记序列，用以概括预期的未来轨迹，然后在该规划的条件下生成可执行的动作标记。由于规划与执行共享统一的离散动作词表，规划能够保持贴近控制流形，并提供可直接执行的指导，而不是需要再转换回运动指令的抽象提示。在 LIBERO、SimplerEnv-WidowX 以及真实世界操作任务上的实验表明，动作标记规划相较于直接生成动作具有稳定提升，其中在长时程多阶段任务上的收益最大。

## Abstract
Most vision-language-action (VLA) models map observations directly to actions without explicit intermediate planning, which limits performance on long-horizon tasks where early mistakes compound. We propose Coarse-to-Control, a plan-execute VLA that introduces planning natively in the action-token space. The key idea is to let the policy first predict a compact sequence of coarse action tokens that summarize the intended future trajectory, and then generate executable action tokens conditioned on this plan. Because both planning and execution share a unified discrete action vocabulary, the plan stays close to the control manifold and provides directly actionable guidance rather than an abstract hint that must be translated back to motor commands. Experiments on LIBERO, SimplerEnv-WidowX, and real-world manipulation tasks show that action-token planning consistently improves over direct action generation, with the largest gains on long-horizon multi-stage tasks.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 核心思想

- 论文提出 **Coarse-to-Control**，一个“先规划、再执行”的 VLA 框架。
- 其基本流程是：
  1. 模型先根据当前观测、语言指令和机器人状态，生成一段 **粗粒度动作规划 token**；
  2. 再以这些规划 token 为条件，生成 **可执行动作 token**；
  3. 推理时只将可执行动作 token 解码为真实机器人动作，规划 token 仅作为模型内部的中间引导。

- 与传统 CoT 的区别：
  - 文本 CoT：中间推理是语言；
  - 视觉 CoT：中间推理是图像或视觉子目标；
  - Coarse-to-Control：中间推理是 **粗粒度动作轨迹的离散 token**。

---

## 2.2 粗粒度动作规划的构造

- 给定未来一段长时程动作序列：

  \[
  A_{t:t+H_p-1}
  \]

  其中 \(H_p\) 是规划源时域长度。

- 方法将这段长动作序列压缩成 \(K\) 个粗粒度规划步骤，每个粗粒度步骤对应一个 action chunk。

- 若 chunk 大小为：

  \[
  c = H_p / K
  \]

- 对运动维度，使用 chunk 内相对动作求和，得到该段的净位移或净运动趋势：

  \[
  \bar{a}^{motion}_{t+i}
  =
  \sum_{j=0}^{c-1} a^{motion}_{t+ic+j}
  \]

- 对夹爪维度，保留该 chunk 最后一步的夹爪状态：

  \[
  \bar{a}^{gripper}_{t+i}
  =
  a^{gripper}_{t+(i+1)c-1}
  \]

- 这样得到的粗粒度计划保留：
  - 未来运动方向；
  - 阶段性意图；
  - 抓取/放置等夹爪状态变化；
  - 长时程任务的局部阶段结构。

- 同时丢弃：
  - 高频控制噪声；
  - 细微的低层动作细节。

---

## 2.3 联合规划-执行动作 tokenizer

- 本文设计了一个 **joint-mode residual-VQ action tokenizer**。
- 它有两种模式：
  - 执行模式 \(m=0\)：编码短时域可执行动作；
  - 规划模式 \(m=1\)：编码粗粒度长时域规划动作。

- 两种模式共享同一个离散动作词表。
- 这点是论文的重要设计：
  - 规划 token 和执行 token 位于相同的动作语义空间；
  - 规划不会变成抽象提示，而是和可执行控制更紧密对齐；
  - 执行分支无需学习从“规划词表”到“执行词表”的额外翻译。

- 目标 token 定义为：

  \[
  z^{plan}_t = Q(\bar{A}_t, m=1)
  \]

  \[
  z^{exec}_t = Q(A_{t:t+H_e-1}, m=0)
  \]

  其中：
  - \(Q\) 是 mode-conditioned tokenizer；
  - \(H_e\) 是可执行动作时域；
  - \(\bar{A}_t\) 是压缩后的粗粒度规划轨迹。

- tokenizer 训练目标包括：
  - 动作重建损失；
  - VQ codebook 损失；
  - commitment loss。

- 论文中主模型使用：
  - 3 个 residual VQ codebooks；
  - 每个 codebook 有 4096 个 entries；
  - codec latent dimension 为 64；
  - 动作维度为 7；
  - commitment weight 为 0.25。

## 2.4 Plan-Execute 自回归 VLA

- 在 VLA 策略训练中，模型预测的动作后缀由两部分拼接而成：

  \[
  z_t = [z^{plan}_t, z^{exec}_t]
  \]

- 自回归建模形式为：
  - 先生成规划 token；
  - 再基于规划 token 生成执行 token。

- 概念上可写为：

  \[
  z^{plan}_t \sim p_\theta(z^{plan} \mid o_t, l, s_t)
  \]

  \[
  z^{exec}_t \sim p_\theta(z^{exec} \mid o_t, l, s_t, z^{plan}_t)
  \]

- 训练时使用 teacher forcing 的 next-token prediction：

  \[
  L_{VLA} = - \sum_k \log p_\theta(z_{t,k} \mid x_t, z_{t,<k})
  \]

- 推理阶段：
  1. 输入图像、语言和本体状态；
  2. 生成粗粒度规划 token；
  3. 再生成可执行动作 token；
  4. 只解码执行 token 为连续动作；
  5. 规划 token 不直接发送给机器人执行，只作为内部上下文。

  - Put Spoon：100.0%
  - Put Carrot：95.8%
  - Stack Block：79.2%
  - Put Eggplant：58.3%
  - Overall：83.3%

- 在真实机器人任务中：
  - 平均成功率为 62.5%；
  - 在四个任务中的三个任务上表现最好；
  - 在多阶段任务中相比 π0、π0-fast、faster 更稳健。

- 规划时域越长通常越有利：
  - LIBERO overall 从无规划的 96.45% 提升到 \(H_p=160\) 时的 97.90%；
  - Long suite 从 94.20% 提升到 95.00%。

- 共享 token 空间很重要：
  - Faster-AR 无规划 baseline：95.40%；
  - Separate tokenizer：96.60%；
  - Joint-mode tokenizer：97.90%。
- 这说明：
  - 仅添加规划已经有帮助；
  - 但规划和执行共享动作词表能进一步减少接口错配。

- 真实世界子任务分析显示：
  - plan-based policy 更能保持中间进展；
  - 长时程任务中最终成功率更高；
  - 成功 rollout 的平均步数更少，说明轨迹更高效。

## 7. 优点与亮点

- **表示选择合理**
  - 将 CoT 从文本或视觉空间转移到动作 token 空间，避免了语义/视觉中间表示到控制动作之间的转换鸿沟。

- **规划与执行统一**
  - 使用共享离散动作词表，让规划 token 和执行 token 位于同一个控制相关空间中。

- **无需额外 planner-controller 接口**
  - 规划 token 只是自回归策略内部前缀；
  - 推理时只执行动作 token；
  - 系统实现相对简洁。

- **对长时程任务特别有效**
  - 在 LIBERO Long 和真实多阶段任务上收益明显；
  - 能缓解早期误差累积问题。

- **实验覆盖较全面**
  - 包含 LIBERO、SimplerEnv-WidowX、真实机器人；
  - 与 No CoT、Textual CoT、Visual CoT、Action CoT 多类方法比较；
  - 有较多消融实验验证核心设计。

- **效率上具有优势**
  - 相比 visual CoT 需要生成较长视觉前缀，本文 action-token planning 使用更短的 token 前缀；
  - runtime 对比显示 Coarse-to-Control 平均任务评估时间低于 CoT-VLA-style visual reasoning baseline。

---

## 8. 不足与局限

- **动作空间规划形式仍较固定**
  - 当前方法采用“压缩未来动作轨迹”为粗粒度计划；
  - 规划结构是预定义的固定时域与 chunk 化形式；
  - 尚未探索更动态、自适应或层次化的动作规划方式。

- **对 demonstration 数据依赖较强**
  - 粗粒度规划 token 来自未来 demonstration 动作序列压缩；
  - 若 demonstrations 质量不高，计划监督也可能受影响。

- **真实世界实验规模有限**
  - 每个任务 50 条 demonstrations、20 次 rollout；
  - 虽然对机器人实验来说有一定成本合理性，但统计规模仍有限；
  - 任务类型集中在桌面物体操控，泛化到移动操作、双臂操作、复杂开放环境仍未知。

- **模型和预训练数据影响未完全剥离**
  - 方法基于 π0-FAST 初始化和 PaliGemma-3B backbone；
  - 与其他方法的比较可能受到模型规模、数据来源、训练细节差异影响；
  - 并非所有 baseline 都在完全相同的训练设置下重新训练。

- **算力报告不完整**
  - 仅明确说明 tokenizer 预训练使用 8×H200；
  - 缺少完整训练耗时、策略训练 GPU 配置和总体计算成本。

- **共享 token 空间仍有进一步研究空间**
  - 论文自己也指出，目前 joint tokenizer 只是规划和执行统一的一种实现；
  - 如何更自然地统一粗粒度规划和细粒度执行，仍是开放问题。

# 总体评价

- 这篇论文提出了一个很有启发性的观点：**对于 VLA 模型，中间推理最好不要只停留在语言或视觉层面，而应尽量靠近动作控制层面。**
- Coarse-to-Control 通过在统一动作 token 空间中先生成粗粒度计划、再生成可执行动作，有效提升了长时程和多阶段操控任务的成功率。
- 方法设计简洁，实验覆盖较全面，消融较有说服力。
- 其主要局限在于真实世界规模有限、算力报告不完整，以及规划形式仍相对固定。
- 总体而言，本文为 VLA 中的动作空间推理和长时程机器人规划提供了一个清晰且有效的方向。
