## 摘要
单体式视觉-动作模型代表了自动驾驶领域一种新兴范式。然而，当为复杂交互编码扩展的时间上下文时，该架构会产生迅速超出实时计算预算的令牌序列。尽管线性 Transformer 和外部记忆等方法试图使上下文更加轻量化，但令牌压缩与该架构最为兼容，因为它不需要修改骨干网络。然而，现有压缩方法采用基于规则的启发式策略，如时间衰减，与规划过程解耦，因而可能丢失对决策至关重要的信息。我们提出 COMPACT-VA，这是一种基于条件 VQ-VAE 构建的规划对齐工作记忆框架，可将扩展上下文压缩为有界表示。压缩过程同时以历史轨迹和学习得到的规划意图为条件，其中后验编码器在训练期间从未来轨迹中蒸馏该规划意图，而先验编码器则学习从压缩观测中预测该意图。压缩后的记忆与预测的潜变量拼接后输入策略网络，以进行端到端优化，从而在保留决策关键信息的基础上进行规划。我们在高信号动态场景中进行评估，在这些场景中历史上下文对行为正确性最为关键（例如停止、让行或通行），并相应设计了行为指标。在可比的令牌预算下，我们在成功率上实现了超过 6% 的提升（达到 68.3%），并在各项指标上取得一致增益。消融实验验证了规划对齐耦合的有效性。闭环评估表明，与未压缩处理相比，COMPACT-VA 在保持整体驾驶性能的同时，实现了 3.3 倍加速和 2.7 倍内存降低。

## Abstract
Monolithic vision-action models represent an emerging paradigm in autonomous driving. However, this architecture produces token sequences that quickly exceed real-time computational budgets when encoding extended temporal context for complex interactions. While approaches like linear transformers and external memory try to make the context lightweight, token compression is most compatible with the architecture as it requires no backbone modifications. Yet existing compression adopts rule-based heuristics like temporal decay, decoupled from planning, risking loss of decision-critical information. We propose COMPACT-VA, a planning-aligned working memory framework built on conditional VQ-VAE, compressing extended context into bounded representations. Compression is conditioned on both historical trajectory and a learned planning intent that the posterior encoder distills from future trajectories during training, while the prior encoder learns to predict it from compressed observations. The compressed memory, concatenated with the predicted latent, feeds the policy for end-to-end optimization, planning with retained decision-critical information. We evaluate on high-signal dynamic scenarios where historical context is most critical for behavior correctness (e.g., stop, yield, or proceed), and accordingly design behavioral metrics. Under comparable token budgets, we achieve $>$6% improvement (68.3%) on success rates with consistent gains across metrics. Ablations validate planning-aligned coupling effectiveness. Closed-loop evaluation confirms that COMPACT-VA maintained general driving performance with 3.3* speedup and 2.7* memory reduction over uncompressed processing.

## 2. 方法论：COMPACT-VA 的核心思想与技术细节

### 2.1 基础框架：统一 Vision-Action 模型

- 论文基于 Alpamayo VA 模型进行扩展。
- 输入包括：
  - 多摄像头历史图像；
  - 时间位置嵌入；
  - 相机嵌入；
  - ego 历史轨迹 token。
- 输出为未来轨迹 token，采用自回归方式生成。

原始视觉 token 数量为：

\[
N_{raw} = T \times N_{cam} \times N_{img}
\]

其中：
- \(T\)：历史时间步数；
- \(N_{cam}\)：摄像头数量；
- \(N_{img}\)：每张图像产生的视觉 token 数。

在论文实验中，5 秒历史、40 张图像、每张 160 个 token，会产生：

\[
N_{raw}=6400
\]

这对实时推理非常昂贵。

---

### 2.2 分层时序压缩记忆：Hierarchical FIFO Memory Bank

COMPACT-VA 首先构建一个分层历史记忆库，将不同时间距离的帧采用不同压缩率处理：

- **最近帧**：保留高分辨率 token；
- **中间帧**：中等压缩；
- **较远帧**：强压缩。

压缩后的 token 数量为：

\[
N_{compressed} =
\sum_{k=1}^{K} n_k \cdot N_{cam} \cdot \left\lfloor \frac{N_{img}}{r_k} \right\rfloor
\]

其中：
- \(K\)：压缩层数；
- \(n_k\)：第 \(k\) 层包含的帧数；
- \(r_k\)：第 \(k\) 层累计压缩率。

实验设置中使用 3 层结构：

- Layer 1：4 帧，压缩率 \(r_1=1\)，不压缩；
- Layer 2：5 帧，压缩率 \(r_2=16\)；
- Layer 3：11 帧，压缩率 \(r_3=80\)。

最终将 6400 个视觉 token 压缩到 **1424 个 token**，约为 **4.5 倍压缩**。

---

### 2.3 Q-former 学习式压缩

- 每一层压缩通过 **Q-former** 实现。
- Q-former 使用可学习 query token 与原始视觉 token、时间嵌入、相机嵌入、历史轨迹信息一起进行注意力交互。
- query token 学习从原始观测中聚合任务相关信息。

与简单规则压缩不同，Q-former 压缩是端到端学习的，可以根据驾驶任务自动选择保留哪些视觉特征。

---

### 2.4 规划对齐压缩：Conditional VQ-VAE

论文的关键创新在于：仅有分层压缩还不够，压缩过程必须与规划目标耦合。

因此作者引入 **条件 VQ-VAE**，用未来轨迹蒸馏出“驾驶意图” latent，并要求压缩后的历史观测能够预测该意图。

#### 后验编码器 Q-net：训练时使用

- 输入：
  - 当前观测；
  - 真实未来轨迹。
- 作用：
  - 从未来轨迹中提取驾驶意图 latent。
- 处理方式：
  - 将未来轨迹转换为 unicycle 控制序列，包括加速度 \(a\) 和曲率 \(\kappa\)；
  - 经过正则化与量化；
  - 通过 MLP 和小型 Transformer 输出高斯分布参数：

\[
q_\phi(z|o,\tau_{future})
\]

其中 \(z\) 表示驾驶意图潜变量。

#### 先验编码器 P-net：训练和推理均使用

- 输入：
  - 压缩后的观测 token。
- 作用：
  - 仅依靠历史压缩记忆预测驾驶意图 latent：

\[
p_\theta(z|o_{compressed})
\]

推理时没有未来轨迹，因此只能使用先验编码器。

#### 向量量化 VQ

- 后验 latent 和先验 latent 都映射到共享离散 codebook。
- 论文中 codebook 大小为 20。
- 通过最近邻选择 code：

\[
i^* = \arg\min_k \|z-c_k\|
\]

并使用 straight-through estimator 进行反向传播。

---

### 2.5 策略输入与训练目标

- 压缩后的历史 token 与预测出的驾驶意图 latent 拼接。
- 驾驶意图 latent 被作为一个特殊 token 输入 VA policy Transformer。
- 训练损失由三部分组成：

\[
L = L_{traj}
+ \lambda_{KL}D_{KL}
\left(
q_\phi(z|o,\tau_{future})
\| 
p_\theta(z|o_{compressed})
\right)
+ \lambda_{commit}L_{commit}
\]

其中：
- \(L_{traj}\)：未来轨迹 token 的交叉熵损失；
- KL 项：让先验编码器学习匹配后验意图分布；
- \(L_{commit}\)：VQ codebook 的 commitment loss。

### 2.6 方法流程概括

1. 多摄像头历史图像经视觉编码器得到原始视觉 token；
2. 按时间远近放入分层 FIFO 记忆库；
3. Q-former 对不同层进行不同比例压缩；
4. 训练时，后验编码器从未来轨迹中提取驾驶意图；
5. 先验编码器从压缩观测中预测该意图；
6. 通过 KL 与轨迹预测损失约束压缩记忆必须保留规划相关信息；
7. 推理时仅使用压缩观测和先验编码器；
8. 压缩记忆与意图 token 输入策略 Transformer，输出未来轨迹。

### 3.2 重点评估场景

论文主要关注三类“高信号动态场景”：

#### 1. 四向停车交叉口

- 多车在 all-way stop 路口交互；
- 正确行为依赖车辆到达顺序；
- 需要记忆数秒前谁先到达。

#### 2. 停车 / 让行标志下的动态遮挡

- 车辆或行人可能曾经可见，随后被遮挡；
- 模型需要记住被遮挡目标，而不是只看当前画面。

#### 3. 无保护转弯

- ego 车辆需要穿越对向车流；
- 需要判断对向车辆是否减速让行；
- 依赖数秒历史轨迹趋势。

这些场景共同特点是：  
**行为正确性主要取决于离散决策，例如停、等、走，而不仅仅是轨迹回归误差。**

### 3.3 评价指标

论文认为传统 minADE 等轨迹误差指标不足以评估这类场景，因此设计了行为指标：

- **Go Success Rate / Go SR**：  
  车辆在应当前进时是否正确起步或通行。

- **Stop Success Rate / Stop SR**：  
  车辆在需要停车时是否真正停下。

- **Roll-through Rate**：  
  是否出现滚动停车，即没有完全停住就通过。

- **Stop Position Error**：  
  停车位置与目标停车线的偏差。

- **Stop Duration Error**：  
  停车持续时间与真实驾驶行为的偏差。

论文尤其强调 **Go SR**，因为它最能反映模型是否利用了长期记忆来判断何时通行。

- 加规划对齐的 COMPACT-VA。

#### 5. 压缩率消融

测试多种分层压缩 token 分配方式，例如：
- 最近帧保留 160 token；
- 中间帧保留 10 或 20 token；
- 远帧保留 2 token。

#### 6. 历史长度消融

比较：
- 5s 20 images；
- 5s 40 images；
- 5s 60 images；
- 5s 80 images。

#### 7. VQ 技能利用率分析

- 使用 codebook 大小 20；
- 分析训练过程中有多少 skill code 被激活；
- 结果显示约 15–17 个 skill 被使用，利用率约 80%。

---

### 5.2 实验是否充分

总体来看，实验设计较有针对性，尤其适合验证“长历史记忆是否有助于自动驾驶决策”这一核心问题。

**充分性较强的方面**：

- 不只报告轨迹误差，而是设计了行为正确性指标；
- 覆盖了开环、闭环和效率评估；
- 对比了短历史、稀疏长历史、密集长历史、无规划对齐压缩等合理 baseline；
- 消融实验能较好地验证各组件的贡献；
- token 数量基本控制在可比范围内，比较相对公平。

**仍然不足的方面**：

- 主要实验集中在筛选出的特定高信号场景，泛化性仍需更多验证；
- 闭环评估主要是通用驾驶场景，而非大量 stop-controlled intersection 场景，因为 Alpasim 中缺少足够重建；
- 训练算力和训练细节不够透明；
- 论文没有在更多公开自动驾驶 benchmark 上验证，例如 nuScenes、Waymo Open Motion 等；
- 对极端复杂多车交互、长时间遮挡、异常驾驶行为的覆盖仍有限。

---

## 6. 主要结论与发现

- **规划对齐压缩显著优于规则式或无对齐压缩**。  
  COMPACT-VA 在相近 token 预算下取得最高 Go SR，说明让压缩过程与未来规划目标对齐确实有助于保留决策关键信息。

- **更多 token 不一定更好**。  
  Dense Obs w/ Long Hist 使用 6400 token，但表现反而低于压缩方法。这说明直接堆叠历史视觉 token 可能造成模型注意力负担和时序推理困难。

- **稀疏采样长历史并不可靠**。  
  Sparse Obs w/ Long Hist 与短上下文 baseline 相比没有提升，甚至略差，说明关键行为线索可能出现在被跳过的帧中。

- **分层压缩本身有效，但规划对齐进一步提升性能**。  
  Compression w/o plan-align 的 Go SR 为 65.6%，COMPACT-VA 达到 68.3%，说明 cVAE 意图对齐机制提供了额外收益。

- **COMPACT-VA 在效率上优势明显**。  
  相比未压缩长上下文方案，实现 3.3 倍推理加速和 2.7 倍显存降低。

- **闭环通用驾驶性能基本保持**。  
  在 Alpasim 的 910 个通用场景中，COMPACT-VA 与 baseline 的安全和驾驶指标基本相当，没有明显牺牲通用驾驶能力。

---

## 7. 优点与亮点

### 7.1 方法设计上的亮点

- **压缩目标与规划任务对齐**：  
  不是仅根据时间远近或视觉相似度压缩，而是让压缩保留对未来驾驶行为有用的信息。

- **兼容现有 VA/VLA 主干**：  
  方法主要作用于输入 token 压缩，不要求修改 Transformer 主干结构，工程落地友好。

- **引入驾驶意图 latent**：  
  通过条件 VQ-VAE 从未来轨迹中蒸馏驾驶意图，再要求历史压缩记忆预测该意图，使压缩具有任务导向性。

- **离散 skill code 具有一定可解释性**：  
  VQ codebook 可被视为不同驾驶意图或行为模式的离散表示。

- **分层 FIFO 记忆设计合理**：  
  最近帧保留高分辨率，远帧进行强压缩，符合驾驶中“近处细节更重要、远处历史需保留摘要”的直觉。

---

### 7.2 实验设计上的亮点

- **关注传统指标难以体现的行为正确性**：  
  使用 Go SR、Stop SR、Roll-through Rate 等指标，比单纯 minADE 更能反映真实驾驶安全问题。

- **选择了记忆依赖强的场景**：  
  四向停车、动态遮挡、无保护转弯都非常适合测试长上下文能力。

- **对比设置较完整**：
  - 短上下文；
  - 稀疏长上下文；
  - 密集长上下文；
  - 无规划对齐压缩；
  - 规划对齐压缩。

- **效率评估有实际意义**：  
  报告了推理时间和显存，并显示压缩不仅提升性能，还降低计算开销。

---

## 8. 不足与局限

### 8.1 实验覆盖的局限

- **主要开环实验集中在筛选场景**：  
  数据来自高信号 stop-controlled intersection 子集，虽然适合验证核心假设，但可能高估方法在普通驾驶场景中的增益。

- **闭环评估没有充分覆盖核心记忆场景**：  
  作者也承认 Alpasim 中缺少足够 stop-controlled intersection 重建，因此闭环部分主要验证一般驾驶性能，而非最关键的记忆依赖决策场景。

- **缺少跨数据集验证**：  
  论文没有在 Waymo、nuScenes、Argoverse 等广泛使用的公开 benchmark 上展示结果，外部泛化性仍不明确。

---

### 8.2 方法上的局限

- **依赖未来轨迹监督学习驾驶意图**：  
  训练阶段需要高质量未来轨迹作为后验蒸馏信号，若数据中人类驾驶行为存在偏差，latent 也可能学习到这些偏差。

- **历史窗口仍然有限**：  
  方法主要面向 5–10 秒工作记忆，不适合需要更长时空记忆的任务，例如跨路口、多阶段导航或长期地图状态维护。

- **分层压缩结构仍含人工设计**：  
  虽然 Q-former 是学习式压缩，但层数、帧数比例、压缩率仍需要人工设定。

- **驾驶意图 latent 的语义解释有限**：  
  虽然 codebook 利用率较高，但论文没有充分展示每个 skill code 对应的具体驾驶语义。

---

### 8.3 资源透明度不足

- 论文未报告完整训练算力：
  - GPU 数量未知；
  - 训练时长未知；
  - batch size 和训练成本不完整；
  - 复现成本难以准确评估。

---

### 8.4 应用限制与风险

- **极端长尾场景仍需验证**：  
  如多车同时争抢路权、严重遮挡、异常交通参与者行为等。

- **端到端压缩可能带来不可见的信息丢失风险**：  
  虽然规划对齐能降低风险，但在安全关键系统中，压缩模块错误丢弃信息仍可能导致危险行为。

- **开环成功不等价于真实道路安全**：  
  行为指标更合理，但仍需要大规模闭环仿真和实车验证才能证明安全性。

---

## 总体评价

COMPACT-VA 是一篇针对自动驾驶长上下文 VA/VLA 模型的高价值工作。它提出的核心观点非常清晰：**长历史不是简单地保留更多 token，而是要保留对规划决策真正有用的信息**。通过分层 Q-former 压缩和条件 VQ-VAE 驾驶意图对齐，论文在有限 token 预算下提升了停车、让行、通行等关键行为决策表现，并显著降低推理开销。

其主要贡献不只是一个压缩模块，而是将 token 压缩从“视觉或时间启发式问题”重新定义为“规划任务对齐问题”。不过，论文仍需要更丰富的闭环记忆场景验证、更透明的训练资源报告，以及更多公开数据集上的泛化实验，才能进一步证明其在真实自动驾驶系统中的可靠性。
