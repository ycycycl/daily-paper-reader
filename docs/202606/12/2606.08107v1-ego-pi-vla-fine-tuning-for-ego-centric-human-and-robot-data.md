## 摘要
机器人学面临数据稀缺这一根本性挑战。不同于语言或视觉研究，机器人操作领域并不存在互联网规模的数据集。一条有前景的前进路径是利用自我中心的人类数据；这类数据更易收集，覆盖范围更广，并且可扩展到更大规模。为此，我们以 $π_{0.5}$ 模型为基础，研究在配备灵巧五指手的人类与类人机器人具身之间进行学习的关键设计选择。我们的结果表明，人类数据能够使机器人在没有相应机器人数据的情况下学习新的任务语义，并将已有技能组合成新的行为。论文网站如下：https://egopipaper.github.io/

## Abstract
Robotics faces a fundamental challenge of data scarcity. Unlike language or vision research, there is no internet-scale dataset for robotic manipulation. A promising path forward is to leverage egocentric human data, which can be collected more easily, with greater breadth, and at a larger scale. Towards this end, we investigate key design choices for learning across human and humanoid embodiments equipped with dexterous five-finger hands, using the $π_{0.5}$ model as a foundation. Our results show that human data enables robots to learn new task semantics and compose existing skills into novel behaviors without corresponding robot data. ## 2. 方法论：Ego-Pi 的核心思想与关键技术

### 2.1 总体框架

- 论文提出 **Ego-Pi**，一个用于将预训练 VLA 模型微调到**自我中心人类数据与人形机器人数据**上的框架。
- 基础模型采用 **π0.5**，这是一类基于视觉-语言-动作模型的机器人控制模型。
- 输入包括：
  - 语言指令；
  - 头部/自我中心图像；
  - 机器人左右腕部相机图像；
  - 本体状态；
- 输出包括：
  - 可选的子任务语言预测；
  - 短时域连续动作序列。

### 2.2 基于 π0.5 的流匹配动作预测

- π0.5 使用 **flow matching** 动作头，而不是传统自回归动作 token 预测。
- 训练目标是让模型预测从噪声动作到真实专家动作的速度场。
- 论文中的目标函数可概括为：

\[
L = \mathbb{E}_{\tau,\omega}
\left[
\left\|
\omega - a_{t:t+H} - f_\theta(a^{\tau,\omega}_{t:t+H}, o_t, \ell_t)
\right\|^2
\right]
\]

其中：

- \(\tau \in [0,1]\)：流时间；
- \(\omega \sim \mathcal{N}(0,I)\)：高斯噪声；
- \(a_{t:t+H}\)：专家动作序列；
- \(a^{\tau,\omega}_{t:t+H}\)：真实动作与噪声之间的插值；
- \(o_t\)：观测，包括头部图像、腕部图像和机器人状态；
- \(\ell_t\)：语言指令。

### 2.3 人手与机器人手的动作对齐

- 传统方法通常预测：
  - 手腕位置；
  - 手腕姿态；
  - 五个指尖位置；
  然后通过逆运动学转换为机器人关节角。
- 但对于高自由度灵巧手，例如 Tesollo hand 有 20 个主动关节，逆运动学容易产生：
  - 自碰撞；
  - 不自然手势；
  - 不稳定控制。
- 因此，Ego-Pi 采用**机器人中心的动作表示**：
  - 先将人手关键点或 Manus glove 捕获到的人手关节角转换为人手每个关节的角度；
  - 再通过偏移和缩放映射到机器人手关节空间。

动作表示为：

\[
a = \{p, r, q\} \in \mathbb{R}^{29}
\]

其中：

- \(p \in \mathbb{R}^3\)：手腕位置；
- \(r \in \mathbb{R}^6\)：6D 旋转表示的手腕姿态；
- \(q \in \mathbb{R}^{20}\)：机器人手关节角；
- 对于 Inspire hand，关节数较少，为 \(q \in \mathbb{R}^6\)。

### 2.4 双手灵巧动作的 token 交错表示

- π0.5 原本主要面向夹爪机器人，最大动作维度为 32。
- 但双手灵巧控制动作维度很高：
  - 单手约 29 维；
  - 双手共约 58 维；
  - 超出 π0.5 原有动作输出容量。
- 直接扩展输出投影层会破坏预训练权重，导致训练损失更高。
- 因此，论文提出 **interleaved action sequence formulation**：
  - 将左手动作和右手动作拆成两个动作 token；
  - 每个 token 仍不超过原模型动作维度限制；
  - 不修改 π0.5 原始动作头参数。

原本一个时间步动作：

\[
a_t = \{a_t^L, a_t^R\}
\]

标准预测序列为：

\[
a_{t:t+H} = \{a_t, a_{t+1}, ..., a_{t+H}\}
\]

交错后变为：

\[
a_{t:t+H/2}
=
\{a_t^L, a_t^R, ..., a_{t+H/2}^L, a_{t+H/2}^R\}
\]

- 优点：
  - 保留预训练模型结构；
  - 避免重新初始化动作投影层；
  - 能处理双手高维动作。
- 代价：
  - 有效预测 horizon 从 \(H\) 降为 \(H/2\)。

### 2.5 跨具身视觉对齐：手部骨架叠加

- 人手和机器人手虽然都有五指，但视觉外观差异很大。
- 为降低跨具身视觉差异，作者在输入图像上叠加手部 skeleton：
  - 每个手指使用固定颜色；
  - 人手和机器人手保持相同颜色对应；
  - 使用深度感知的遮挡绘制，使靠近相机的手指覆盖远处手指。
- 目的：
  - 帮助模型理解人手与机器人手各手指的对应关系；
  - 减少视觉域差异。

### 2.6 子任务预测辅助损失

- 对部分任务，作者加入 **subtask prediction**：
  - 给数据标注当前子任务；
  - 让 VLM 部分先生成子任务文本；
  - 再通过 flow-matching 动作头预测动作。
- 直觉是让策略在行动前先进行显式的“中间语义推理”。
- 这在需要技能组合和复杂顺序控制的任务中尤其关键。

## 6. 主要结论与发现

- **人类第一视角数据可以有效传递高层任务语义**。
  - 在番茄分拣任务中，人机联合训练达到约 92% 成功率。
  - 在包装任务中，人机联合训练达到约 90% 成功率。
- **仅使用机器人数据效果较差**。
  - 番茄分拣中，机器人只学会单碗放置，面对双碗时随机放置，成功率约 40%。
  - 包装任务中，仅机器人数据成功率约 10%，经常先抓玩偶或动作混乱。
  - 装盒任务中，仅机器人数据成功率约 20%，无法正确先开盒再放物体。
- **简单联合训练并不总是足够**。
  - 对番茄分拣和包装任务，简单 human-robot co-training 已经效果很好。
  - 对装盒任务，简单联合训练只有约 27% 成功率。
- **子任务预测是技能组合任务的关键因素**。
  - 在装盒任务中加入 subtask prediction 后，成功率从约 27% 提升到约 93%。
  - 这说明对于需要顺序依赖和前置条件的任务，显式中间语义预测有助于策略规划。
- **Skeleton overlay 作用有限**。
  - 虽然设计上有助于跨具身视觉对齐，但实验中对性能提升不明显。
- **机器人手形态会影响迁移效果**。
  - Inspire hand 在装盒任务中优于 Tesollo hand。
  - 可能原因是 Inspire hand 尺寸更接近人手，视觉和运动形态差距较小。
- **腕部相机对机器人执行仍然重要**。
  - 虽然人类数据没有腕部相机，但机器人在测试时移除腕部相机会明显降低性能。
  - 例如番茄分拣中，没有腕部图像时机器人难以稳定抓取番茄。

## 7. 优点与亮点

- **问题设定有价值**：
  - 聚焦机器人数据稀缺这一核心瓶颈；
  - 探索用人类第一视角数据传递高层任务语义，而不仅是提升已有任务泛化。
- **方法设计务实**：
  - 基于已有 VLA 基础模型 π0.5 微调，而非从零训练；
  - 更贴近未来通用机器人模型的实际发展路径。
- **动作交错机制巧妙**：
  - 在不修改预训练动作头的情况下支持双手高维灵巧控制；
  - 保留预训练参数，降低训练不稳定风险。
- **机器人中心动作对齐合理**：
  - 避免高自由度灵巧手逆运动学带来的不自然姿态和自碰撞问题；
  - 直接将人手动作映射到机器人关节空间，更适合实际控制。
- **子任务预测结果有启发性**：
  - 表明 VLA 在复杂任务中可能需要显式中间语义层；
  - 对未来“语言推理 + 动作控制”结合的机器人策略有参考价值。
- **实验任务设计清晰**：
  - 三个任务分别对应排序逻辑、技能组合和规则顺序；
  - 能较直接验证人类数据带来的语义迁移能力。
- **真实机器人验证**：
  - 实验不是纯仿真，而是在真实人形机器人和灵巧手上完成，更具说服力。

## 8. 不足与局限

- **任务复杂度有限**：
  - 三个任务都属于相对短时域、桌面、pick-and-place 风格任务；
  - 尚未覆盖更复杂的长程任务、工具使用、柔性物体操作或精细灵巧操作。
- **环境变化有限**：
  - 实验主要在固定相机和固定场景中进行；
  - 对光照、背景、物体外观、桌面布局变化的鲁棒性验证不足。
- **人类数据主要传递高层语义，而非低层技能**：
  - 论文假设机器人已经具备必要低层技能；
  - 人类数据主要用于“串联”和“规则学习”；
  - 尚未解决从人类示范直接学习机器人低层灵巧控制的问题。
- **数据规模仍较小**：
  - 每个任务人类和机器人示范数量在几十到一百多条；
  - 虽然足以验证概念，但还不能证明方法在大规模开放任务中的可扩展性。
- **缺乏充分统计分析**：
  - 没有详细报告每个成功率对应的测试次数、方差、置信区间；
  - 结果稳定性和显著性仍需进一步验证。
- **算力与训练成本信息不足**：
  - 未报告 GPU、训练时间和资源消耗；
  - 不利于复现和成本评估。
- **跨具身视觉对齐仍不充分**：
  - Skeleton overlay 的提升有限，说明简单视觉标注可能不足以解决人手与机器人手外观差异。
- **机器人硬件形态依赖明显**：
  - Tesollo hand 在部分任务中明显不如 Inspire hand；
  - 方法可能对机器人手尺寸、自由度和形态较敏感。
- **benchmark 仍偏作者自定义**：
  - 缺少与标准化机器人基准或更多公开数据集的对比；
  - 泛化到其他机器人平台和任务还需要进一步验证。
