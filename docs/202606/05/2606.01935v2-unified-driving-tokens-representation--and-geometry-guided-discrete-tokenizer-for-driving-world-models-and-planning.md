## 摘要
离散视觉词元应为自动驾驶中的基于词元的世界建模与规划提供紧凑表征。然而，大多数词元化器继承自图像生成任务，主要针对像素重建进行优化，这可能导致易于生成的内容与对驾驶决策有用的可解码信息之间存在差距。我们提出了一种表征引导且几何增强的词元化器，在联合监督下学习离散词元。该词元化器通过特征解码将其离散瓶颈与冻结的 DINO 特征空间对齐，同时通过结合感知损失和对抗损失的 RGB 重建来保留外观信息。为注入与几何状态相关的线索，我们在训练过程中加入相邻帧深度和相对位姿监督，并采用多码本量化来稳定联合目标。我们使用轻量级规划读出模块和 GPT 风格的下一词元世界模型评估相同的学习词元。在 NAVSIM 上的实验表明，该方法提升了重建保真度和表征一致性，在固定解码器下取得了具有竞争力的规划性能，并在匹配设置下获得了更好的生成质量。

## Abstract
Discrete visual tokens should provide a compact representation for both token-based world modeling and planning in autonomous driving. However, most tokenizers are inherited from image generation and are optimized mainly for pixel reconstruction, which may leave a gap between what is easy to generate and what is useful to decode for driving decisions. We present a representation-guided and geometry-enhanced tokenizer that learns discrete tokens under joint supervision. The tokenizer aligns its discrete bottleneck with a frozen DINO feature space through feature decoding, while preserving appearance via RGB reconstruction with perceptual and adversarial losses. To inject geometric state-related cues, we add adjacent-frame depth and relative-pose supervision during training and stabilize joint objectives with multi-codebook quantization. We evaluate the same learned tokens with a lightweight planning readout and a GPT-style next-token world model. Experiments on NAVSIM show improved reconstruction fidelity and representation consistency, competitive planning performance under a fixed decoder, and better generative quality under matched settings.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体框架

论文提出的 tokenizer 建立在 **VQ-VAE / VQGAN 式离散瓶颈**之上，但不只做 RGB 重建，而是引入多种监督信号：

- **RGB 重建监督**：保持视觉外观；
- **DINO 表征重建监督**：保留高层语义与结构信息；
- **相邻帧深度监督**：注入空间几何；
- **相邻帧相对位姿监督**：注入运动状态；
- **多码本量化**：缓解多目标联合训练下的离散瓶颈容量不足。

整体目标是让同一套 token 同时具备：
- 可生成性；
- 可规划性；
- 语义一致性；
- 几何感知能力。

---

### 2.2 输入与表征提取

给定前视 RGB 图像：

\[
I_t \in \mathbb{R}^{H \times W \times 3}
\]

将其划分为非重叠 patch，得到长度为 \(L\) 的 patch 序列。

论文使用冻结的 DINO 模型提取 patch 级语义特征：

\[
F_t = \Phi(I_t) \in \mathbb{R}^{L \times d_{dino}}
\]

其中：
- \(\Phi\) 是冻结的 DINO / DINOv3 模型；
- 训练 tokenizer 时不反向传播到 DINO；
- DINO 特征作为语义表征监督目标。

---

### 2.3 Encoder：DINO 表征 + RGB 细节分支

作者认为，仅使用 DINO 高层特征会丢失纹理、边缘等低层细节，因此 encoder 同时使用：

- **RGB detail branch**：从原始图像中提取局部纹理和边界；
- **DINO feature branch**：提供语义和实例级结构信息。

具体流程：

1. RGB 细节编码：

\[
R_t = E_{rgb}(I_t)
\]

2. 将 RGB 细节特征与 DINO 特征拼接：

\[
X_t = W_f [R_t; F_t]
\]

3. 经过带 RoPE 位置编码的 Transformer encoder：

\[
H_t = E_g(X_t)
\]

4. 投影到量化空间：

\[
Z_t = P_{in}(H_t)
\]

---

### 2.4 离散量化：VQ 瓶颈与 EMA 更新

对于每个 patch 的连续特征 \(z_{t,l}\)，找到最近的 codeword：

\[
k_{t,l} = \arg\min_k \|z_{t,l} - c_k\|_2^2
\]

得到离散 token index \(k_{t,l}\)，以及对应的 codeword embedding：

\[
e_{t,l} = c_{k_{t,l}}
\]

训练中使用：
- **straight-through estimator** 让梯度绕过硬量化；
- **commitment loss** 约束 encoder 输出靠近选中的 codeword；
- **EMA codebook update** 更新码本；
- **dead-code reinitialization** 重置长期未使用的 code；
- **弱正交正则** 减少 codeword 冗余，提高码本利用率。

---

### 2.5 Decoder：同时重建 RGB 与 DINO 特征

量化后的 token embedding 经过 post-transformer：

\[
\tilde{H}_t = P_{out}(E_t)
\]

\[
S_t = E_{post}(\tilde{H}_t)
\]

然后分为两个 decoder：

1. RGB 图像重建：

\[
\hat{I}_t = D_{img}(S_t)
\]

2. DINO 特征重建：

\[
\hat{F}_t = D_{dino}(S_t)
\]

这样 token 不只要能还原图像，还要能还原冻结基础视觉模型的语义表征。

### 2.6 损失函数设计

tokenizer 的基础损失为：

\[
L_{tok} = \lambda_{rec}L_{rec} + \lambda_{sem}L_{sem} + \lambda_{gan}L_{gan} + \lambda_{vq}L_{vq}
\]

其中：

- **RGB 重建损失**：

\[
L_{rec} = \|\hat{I}_t - I_t\|_2^2 + \lambda_{lpips} LPIPS(\hat{I}_t, I_t)
\]

包括像素级 L2 与 LPIPS 感知损失。

- **语义表征损失 \(L_{sem}\)**：  
  对齐解码出的 DINO 特征 \(\hat{F}_t\) 与冻结 DINO 目标特征 \(F_t\)，使用：
  - cosine similarity loss；
  - MSE loss。

- **对抗损失 \(L_{gan}\)**：  
  使用判别器提升图像重建质量。

- **VQ 相关损失 \(L_{vq}\)**：  
  包括 commitment loss、码本正则等。

### 2.7 跨帧几何监督：深度与相对位姿

为了让 token 包含几何和运动信息，作者加入相邻帧监督。

给定相邻帧：

\[
I_t, I_{t+1}
\]

取两帧经过 tokenizer 后的 post-quantization 特征：

\[
\tilde{H}_t, \tilde{H}_{t+1}
\]

为每帧加入 ego token，并通过一个 temporal aggregator：

\[
\{U_t, \bar{g}_t, U_{t+1}, \bar{g}_{t+1}\} = A_\psi(\{[g_t; \tilde{H}_t], [g_{t+1}; \tilde{H}_{t+1}]\})
\]

该聚合器交替进行：
- frame-wise attention；
- cross-frame attention。

然后进行两个几何预测任务：

#### 1）深度预测

使用 DPT-style dense prediction head：

\[
\hat{D}_t, \hat{C}_t = D_\omega(U_t)
\]

其中：
- \(\hat{D}_t\) 是深度；
- \(\hat{C}_t\) 是置信度图。

深度损失包括：
- 有效像素 mask 下的回归损失；
- 可选置信度加权；
- 平滑正则。

#### 2）相对位姿预测

根据两帧 ego token 预测相对位姿：

\[
\hat{p}_{t \rightarrow t+1} = [\hat{t}_{t \rightarrow t+1}, \hat{q}_{t \rightarrow t+1}]
\]

其中包括：
- 平移向量；
- 四元数旋转。

位姿损失包括：
- translation L1 loss；
- sign-invariant quaternion loss。

最终几何增强目标为：

\[
L = L_{tok} + \lambda_{geo}(\lambda_{depth}L_{depth} + \lambda_{pose}L_{pose})
\]

几何分支只在 tokenizer 训练阶段使用，推理时不需要深度或位姿标签。

### 2.8 多码本量化 Multi-Codebook Quantization

作者发现，在同一离散瓶颈中同时保留：
- RGB 外观；
- DINO 语义；
- 深度几何；
- 相对位姿；

会造成容量竞争，导致重建质量下降或码本不稳定。

因此，借鉴 UniTok，本文采用多码本机制：

- 原始单码本：
  - \(K = 16384\)

- 几何增强版本：
  - \(M = 4\) 个 codebook；
  - 每个 codebook 大小为 4096；
  - 总体离散容量增加，但 patch token 网格分辨率不变。

流程为：

1. 将每个 patch 的特征通过 attention-based splitter 分成 \(M\) 个 head-specific vectors；
2. 每个 head 独立量化；
3. 将 \(M\) 个 codeword 拼接；
4. 通过 merge projection 融合为一个 embedding 供后续 decoder 使用。

这样可以在不增加空间 token 数量的前提下提高表示容量。

### 2.9 下游任务一：基于 token 的规划解码器

为了验证 token 是否适合规划，作者冻结 tokenizer，训练一个轻量级规划 head。

输入：
- 当前帧 token 特征；
- ego status vector，维度为 11。

输出：
- 多条未来 ego 轨迹：

\[
Y = \{y_\tau\}_{\tau=1}^{T}, \quad y_\tau = (x_\tau, y_\tau, \psi_\tau)
\]

规划模块结构：
- token features 经 scene encoder 压缩为少量 scene tokens；
- ego 状态经过 embedding；
- 通过 shallow attention 让 ego token attend 到 scene tokens；
- MLP 预测多条候选轨迹；
- scoring head 预测每条轨迹的 PDM-style 指标；
- 推理时选择最高分轨迹。

训练损失：
- 对最接近 ground truth 的轨迹使用 L1 regression；
- score head 使用 binary cross entropy 监督。

### 2.10 下游任务二：自回归世界模型

作者冻结 tokenizer，将每帧图像转成离散 token 序列，然后训练 GPT-style Transformer：

- 输入：历史 token 序列；

   - 评估 PDMS 及其五个子指标。

4. **规划 tokenizer 消融**
   - 比较 Rep、Geo、MCB 对规划指标的影响。

5. **世界模型生成实验**
   - 与 DrivingGPT 比较；
   - 使用 FID / FVD；
   - 展示视觉 rollout。

6. **定性可视化**
   - RGB reconstruction；
   - DINO feature reconstruction；
   - depth reconstruction；
   - planning failure / success case；
   - generated rollout。

整体上至少包含：
- tokenizer 质量实验；
- 表征与几何消融；
- 规划 benchmark；
- 生成 benchmark；
- 定性可视化分析。

---

### 5.2 实验是否充分

总体来说，实验设计较为完整，覆盖了本文主张的三个核心方向：

- **重建质量**：证明 token 仍能保留外观；
- **语义一致性**：证明 DINO 表征对齐有效；
- **规划能力**：证明 token 对决策模块有用；
- **生成能力**：证明 token 仍适合自回归世界模型建模。

特别是作者采用：
- 冻结 tokenizer；
- 固定轻量 planner；
- 相同训练协议；

来评估 token 的“可消费性”，这在实验设计上较为公平，能较好地隔离 tokenizer 本身的贡献。

---

### 5.3 公平性与客观性

实验有若干公平设计：

- tokenizer 对比使用相同图像分辨率 288 × 512；
- Orbis 使用官方 checkpoint 并在同一评估 pipeline 下测试；
- 规划消融使用相同 planning decoder；
- world model 对比采用 matched setting；
- 所有结果在 NAVSIM test split 上报告。

但也存在一些潜在公平性问题：

- 与端到端方法比较时，传感器设置不完全一致：
  - 有些方法使用多相机；
  - 有些使用 LiDAR；
  - 本文 token-based planner 使用单视角前视相机。
- 与 token-based 方法比较时，虽然同属 frozen token input，但不同方法的 planner / world model 训练策略可能不同。
- tokenizer 的训练数据、训练预算、超参数细节在正文中不够完整，可能影响复现和完全公平比较。
- 实验主要集中在 NAVSIM，缺少跨数据集验证。

## 6. 主要结论与发现

- **DINO 表征监督显著改善 token 质量**：
  - Ours-Rep 在 rFID、PSNR、SSIM 和 DINO feature discrepancy 上均表现优秀；
  - 表明 token 不仅重建像素，也更好地保留了语义结构。

- **几何监督让 token 更适合规划**：
  - 加入深度和相对位姿监督后，规划指标尤其是 Ego Progress 提升明显；
  - 说明几何状态信息对下游轨迹预测有帮助。

- **多码本量化缓解容量瓶颈**：
  - 单码本同时承载 RGB、DINO、深度、位姿会导致 PSNR 下滑；
  - 多码本设计能恢复重建质量，同时保留几何能力；
  - 证明问题主要是离散瓶颈容量不足，而非任务目标完全冲突。

- **同一组 token 可以兼顾规划与世界模型生成**：
  - 在规划任务中，本文方法在 frozen visual token 方法中取得最高 PDMS；
  - 在生成任务中，world model 的 FID / FVD 优于 DrivingGPT baseline；
  - 支持“统一驾驶 token”这一核心观点。

- **规划性能表现突出**：
  - 本文方法在单视角 frozen-token 设定下达到 PDMS 91.8；
  - 高于 Epona、PWM、DriveLaW、DriveVLA-W0 等 token-based 方法；
  - 在所有列出方法中也具有较强竞争力，仅部分端到端强方法如 DriveSuprim 更高。

## 7. 优点与亮点

- **问题定义清晰且重要**  
  论文指出自动驾驶 tokenizer 不能只关注像素重建，而应服务于规划和世界模型，这是一个非常关键且现实的问题。

- **统一视角有价值**  
  将离散 token 视为“驾驶世界语言”，同时支持：
  - 世界模型预测；
  - 规划读出；
  - 图像生成；
  - 语义和几何解码。

- **方法设计合理**  
  同时结合：
  - DINO 高层表征；
  - RGB 细节分支；
  - RGB reconstruction；
  - DINO feature reconstruction；
  - depth / pose geometry supervision；
  - multi-codebook quantization。

  这些设计彼此逻辑一致，围绕“让 token 更可用于驾驶”展开。

- **消融实验有说服力**  
  Naive → +Rep → +Geo → +MCB 的路线清楚展示了每个模块的贡献：
  - 表征监督提升语义和规划；
  - 几何监督提升规划；
  - 多码本缓解容量竞争。

- **评估维度较全面**  
  不只看图像重建，也看：
  - DINO feature consistency；
  - depth / pose；
  - planning PDMS；
  - FID / FVD；
  - 定性可视化。

- **下游规划评估设计较干净**  
  冻结 tokenizer，使用固定轻量 planner，可以更直接衡量 token representation 的质量，而非 planner 架构本身的能力。

- **对世界模型友好**  
  尽管加入了规划相关监督，生成质量并没有明显受损，反而在 FID / FVD 上优于 baseline。

## 8. 不足与局限

- **数据集覆盖有限**  
  实验主要集中在 NAVSIM / OpenScene，缺少在其他自动驾驶数据集上的验证，例如：
  - nuScenes；
  - Waymo Open Dataset；
  - Argoverse；
  - nuPlan 原始闭环评估。

  因此跨数据集泛化能力尚不明确。

- **主要是 non-reactive benchmark**  
  NAVSIM 是 data-driven non-reactive simulation，无法完全反映真实闭环交互规划中的复杂反馈问题。  
  方法在真正 closed-loop、multi-agent interactive simulation 中是否仍然有效，需要进一步验证。

- **传感器设置较单一**  
  本文主要使用单视角前视相机 token。虽然这展示了方法在低输入条件下的潜力，但自动驾驶实际系统通常依赖：
  - 多相机；
  - LiDAR；
  - Radar；
  - HD map；
  - BEV representation。

  方法如何扩展到多模态、多视角设置尚未充分讨论。

- **算力和训练细节不足**  
  论文未明确给出 GPU 数量、型号、训练时间、batch size、训练步数等关键信息。  
  对于包含 1B 世界模型和复杂 tokenizer 训练的系统，这会影响复现性和工程可评估性。

- **几何监督依赖伪标签**  
  深度监督来自 DVGT 产生的伪标签，并通过 radar 对齐和后处理得到。  
  如果伪标签存在偏差或噪声，可能影响 tokenizer 学到的几何信息。

- **规划模块仍然较简单**  
  虽然轻量 planner 有助于测试 token 质量，但也限制了方法上限。  
  尚不清楚将该 tokenizer 接入更强的规划架构后，性能提升是否仍然稳定。

- **与端到端强方法比较不完全公平**  
  表 3 中不同方法使用不同传感器配置与模型范式。  
  本文方法在 frozen-token 单视角设置下表现突出，但与多相机或 LiDAR 方法直接比较时，需要谨慎解读。

- **世界模型评估仍较有限**  
  生成评估主要包括 FID / FVD 和短时 rollout，条件为过去 3 帧生成未来 8 帧。  
  缺少更长时域、多场景交互、可控生成、规划闭环使用等实验。

- **token 可解释性仍有限**  
  尽管可以解码 RGB、DINO、depth，但每个离散 token 的语义含义、码本结构、不同 codebook 之间的功能分工没有深入分析。
