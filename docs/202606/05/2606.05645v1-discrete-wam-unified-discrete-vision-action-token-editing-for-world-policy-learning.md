## 摘要
自动驾驶需要推理自车动作如何塑造周围世界的演化。然而，大多数端到端方法依赖于直接的状态到动作映射，捕捉相关性却没有显式建模动作条件动态。相反，连续潜空间世界模型往往缺乏用于跨反事实未来进行因果推理的组合结构。我们提出 Discrete-WAM，这是一种统一的潜在视觉-动作世界策略，将未来视觉状态和自车动作表示为对齐的离散 token，从而支持跨替代未来的组合式因果推理。基于这种统一的离散对齐，Discrete-WAM 建立了一个具有统一生成任务的共享离散扩散框架，联合表述世界建模、世界-动作策略以及支持层次化决策的策略，支持在多样化驾驶场景中的组合泛化。在大规模自动驾驶基准上的实验表明，Discrete-WAM 在取得具有竞争力性能的同时，支持可控生成和反事实推理，为实现更可靠的决策提供了一条原则性路径。

## Abstract
Autonomous driving requires reasoning about how ego actions shape the evolution of the surrounding world. However, most end-to-end methods rely on direct state-to-action mappings, capturing correlations without explicitly modeling action-conditioned dynamics. Conversely, continuous-latent world models often lack compositional structure for causal reasoning across counterfactual futures. We introduce Discrete-WAM, a unified latent vision-action world policy that represents future visual states and ego actions as aligned discrete tokens, enabling compositional causal reasoning across alternative futures. Built upon this unified discrete alignment, Discrete-WAM establishes a shared discrete diffusion framework with unified generative tasks, jointly formulating world modeling, world-action policy, and hierarchical decision-enabled policy, supporting compositional generalization across diverse driving scenarios. Experiments on large-scale autonomous-driving benchmarks show that Discrete-WAM achieves competitive performance while supporting controllable generation and counterfactual reasoning, offering a principled path toward more reliable decision-making.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体框架

Discrete-WAM 是一个统一的 **vision-action world-policy model**，主要由四个部分组成：

- **视觉 VQ tokenizer**
  - 将摄像头图像编码为离散视觉 token。
  - 使用类似 VQ-VAE 的离散视觉表示。
  - 视觉 codebook 大小为 `KV = 16384`。
  - 每张图像被划分为 `HV × WV = 16 × 16` 的 patch token。

- **动作 tokenizer**
  - 将未来自车轨迹转换为离散动作 token。
  - 不直接离散化位置，而是离散化 ego-centric 的二维加速度 `(ax, ay)`。
  - 纵向和横向加速度范围均裁剪到 `[-4, 4] m/s²`。
  - 每个方向划分为 `60` 个 bin，因此动作词表大小为：
    - `60 × 60 = 3600`。

- **上下文编码器**
  - 注入历史视觉观测、自车状态、导航命令等上下文信息。

- **共享 decoder-only Transformer**
  - 统一处理视觉 token、动作 token、决策 token 和上下文 token。
  - 模型约 `1B` 参数。
  - 结构包括：
    - hidden dimension：2048；
    - Transformer layers：18；
    - attention heads：16；
    - key-value heads：8；
    - 使用 grouped-query attention 和 Qwen3-MRoPE 位置编码。

## 2.2 动作离散化与软标签插值

论文的动作 token 设计是方法中的关键点之一。

- 给定未来 `H` 步轨迹，首先用三次样条拟合平滑轨迹。
- 然后用二阶有限差分计算每个未来时刻的二维加速度：
  - 纵向加速度 `ax`；
  - 横向加速度 `ay`。
- 加速度被映射到二维网格词表中。

与硬量化不同，论文使用 **soft-label interpolation**：

- 对每个连续加速度 `(ax, ay)`，找到其所在网格单元的四个相邻 prototype。
- 使用双线性插值为四个邻近离散动作 token 分配权重。
- 训练时，动作头不是学习 one-hot 标签，而是学习这个软分布。

其意义是：

- 避免硬量化带来的确定性误差；
- 在理想情况下，如果模型预测分布等于 soft target，则用词表加权求和可以精确恢复连续加速度；
- 剩余误差主要来自模型预测分布与目标分布之间的 mismatch。

## 2.3 统一的 world-policy 建模任务

论文将训练任务统一为 token editing，并设计三类任务。

### 任务一：World Modeling

目标是 **动作条件未来视觉预测**。

- 输入：
  - 当前场景上下文 `Ct`；
  - 未来动作 token `At+1:t+H`。
- 输出：
  - 未来视觉 token `Vt+1:t+H`。

可写为：

```text
pθ(Vt+1:t+H | Ct, At+1:t+H)
```

作用：

- 学习给定动作序列下，未来世界如何演化；
- 支持反事实未来生成，例如改变动作后观察未来图像如何变化。

---

### 任务二：Policy Modeling

目标是 **分层决策条件动作生成**。

论文将策略建模拆成两层：

1. 从场景上下文预测高层决策 skeleton `Dt`；
2. 在 `Dt` 条件下生成未来动作 token 序列。

形式为：

```text
pψ,θ(At+1:t+H, Dt | Ct)
= pψ(Dt | Ct) pθ(At+1:t+H | Ct, Dt)
```

高层决策 `Dt` 表示低频、稀疏的驾驶结构，例如：

- 机动意图；
- 目标车道；
- 粗略 waypoint；
- 速度趋势；
- 交互优先级。

在附录中，论文进一步说明决策 token 空间由：

- lateral path candidates；
- longitudinal speed profiles；

的笛卡尔积构成，总决策词表大小为：

```text
|D| = 400
```

论文还用条件总相关 `TC` 分析了为什么高层决策有助于降低未来动作 token 之间的残余依赖。

核心直觉是：

- 如果没有高层决策，未来动作 token 之间存在强耦合；
- 如果 `Dt` 能解释主要的低频驾驶意图，那么剩余动作 token 只需建模局部细节；
- 因而动作生成更稳定，也更适合并行 token editing。

---

### 任务三：World-Policy Modeling

目标是 **联合动作预测与未来视觉预测**。

- 将未来动作 token 和视觉 token 交错排列：

```text
Yt+1:t+H = [At+1, Vt+1, ..., At+H, Vt+H]
```

- 模型按时间前缀建模：

```text
pθ(Yt+1:t+H | Ct)
= Πh pθ(At+h | Ct, previous tokens)
    pθ(Vt+h | Ct, previous tokens, At+h)
```

作用：

- 让模型同时学习“生成动作”和“动作导致怎样的世界变化”；
- 避免世界模型和策略模型互相割裂；
- 为反事实推理和可控生成提供统一基础。

---

## 2.4 离散扩散式 Token Editing

Discrete-WAM 不是传统自回归逐 token 解码，而是采用离散扩散 / masked editing 风格的 token 编辑。

核心机制：

- 将未来视觉 token 或动作 token 部分 corruption；
- 模型根据上下文和可见条件恢复 clean token；
- token editing loss 作用于所有 editable positions，包括：
  - 被破坏的位置：学习恢复；
  - 未破坏的位置：学习保持不变。

这样带来的好处是：

- 模型不仅学习“如何修改错误 token”，也学习“何时不需要修改”；
- 提供隐式停止信号；
- 支持并行更新多个 token，提高推理效率。

论文还针对不同任务设计了不同 attention mask：

- **world modeling mask**
  - 未来动作作为 clean 条件；
  - 未来视觉作为编辑目标。

- **policy modeling mask**
  - 历史上下文采用 causal attention；
  - 未来动作 block 内部允许双向 attention；
  - 支持整段轨迹并行 refinement。

- **world-policy modeling mask**
  - 动作和视觉均可作为编辑目标；
  - clean target block 与 noisy prediction block 隔离，避免信息泄漏。

---

## 2.5 损失函数

总损失为多项加权组合：

```text
L = λv Lcls_v
  + λa Lcls_a
  + λacc Lacc
  + λtraj Ltraj
  + λpos Lcls_pos
  + λs Lcls_s
  + λdec Ldec
```

各项含义：

- `Lcls_v`：视觉 token 分类交叉熵；
- `Lcls_a`：动作 token 分类交叉熵；
- `Lacc`：加速度回归损失；
- `Ltraj`：由预测加速度积分得到轨迹后的轨迹回归损失；
- `Lcls_pos`：辅助位置分类损失；
- `Lcls_s`：特殊 token 分类损失；
- `Ldec`：高层决策分类损失。

此外，论文指出简单对整个动作分布取期望可能导致 **mode averaging**：

- 多个可能驾驶模式被平均后，可能得到物理上不合理的中间动作。
- 因此论文引入 **mode-aware decoding**：
  - 用 GMM 拟合动作分布；
  - 在 1、2、3 个 Gaussian component 中选择拟合误差最低者；
  - 通过 top-p 选择一个 mode；
  - 在该 mode 内重新归一化并计算连续加速度。

这一过程主要用于连续运动重构损失，不改变离散 token 分类目标。

---

## 2.6 训练流程

论文采用多阶段训练：

- **阶段一：视觉预训练**
  - 使用 world modeling 和 world-policy modeling；
  - 未来动作通过 teacher forcing 输入；
  - 只启用视觉预测损失；
  - 目标是学习动作条件的未来视觉表示。

- **阶段二：视觉-动作联合训练**
  - 同时训练未来视觉 token 和动作 token；
  - 加强世界演化和策略生成之间的耦合。

- **阶段三：动作 LoRA 微调**
  - 使用 LoRA adapter 进行动作策略微调；
  - 保留前面学到的视觉世界表示；
  - 聚焦未来动作生成和 refinement。

- **阶段四：RL post-training**
  - 使用 GRPO 风格的 post-training；
  - 对多个候选轨迹进行 reward 评估；
  - 同时优化高层决策分布和低层动作 token 生成。

整体而言，实验较为充分，原因包括：

  - 统一预训练；
  - 决策 skeleton；
  - RL post-training；
  - 选择性 token editing；
  - 离散动作表示。
- 提供了反事实分析和 surprise metric，展示模型不只是拟合轨迹，还能对动作变化导致的未来后果产生响应。

但也存在一些公平性与充分性方面的保留：

- 多数指标是 **open-loop / pseudo-simulation** benchmark，不能完全代表真实闭环部署表现。
- 不同对比方法的训练数据、模型规模、后训练策略可能并不完全一致。
- 世界生成比较中，不同方法支持的生成时长不同，例如有的方法可生成更长视频，而 Discrete-WAM 主要是较短 4 秒 / 8 帧设置，因此 FID / FVD 对比需要结合生成时长理解。
- 推理延迟实验为了比较解码范式，关闭了工程优化；因此结果不能直接等价于真实部署 latency。
- 没有给出跨数据集泛化、真实车测试或闭环仿真长时稳定性实验。

## 6. 主要结论与发现

- **规划性能强**
  - 在 NAVSIM-v2 上，Discrete-WAM 达到 `90.4 EPDMS`，优于多种强基线。
  - 在 NAVSIM-v1 上，达到 `92.2 PDMS`，也处于领先水平。

- **世界生成质量好**
  - 在世界生成比较中，Discrete-WAM 获得：
    - `FID = 6.6`；
    - `FVD = 80.0`；
  - 优于文中对比的 Vista、DrivingWorld 等方法。

- **统一离散 world-policy pretraining 有帮助**
  - LoRA-SFT 比从头训练和 full finetuning 更优；
  - 说明预训练得到的世界-策略知识需要被保留，而不是被下游规划目标完全覆盖。

- **高层决策建模有效**
  - 引入 `Dt` 后，EPDMS 从 `84.7` 提升到 `87.2`；
  - 表明决策 skeleton 可以提供强行为先验，并改善低层轨迹生成。

- **RL post-training 进一步提升性能**
  - 从 SFT-Dt 的 `90.0 EPDMS` 提升到 RL 后的 `90.4 EPDMS`；
  - 同时改善舒适性和安全相关指标。

- **选择性 token editing 优于全量替换**
  - full_replace 多轮编辑可能扰动已有稳定 token，导致轨迹误差增加；
  - confidence-based 和 JS-based selective replacement 能更好保留稳定 token、细化不确定 token。

- **模型具备一定反事实推理能力**
  - 当 lateral action perturbation 导致越界或碰撞时，surprise 值明显上升；
  - surprise 与 PDMS 呈现明显负相关；
  - 表明模型捕捉到部分动作条件安全关键动态。

---

## 7. 优点与亮点

- **统一离散视觉-动作 token 空间**
  - 将未来视觉状态和动作统一为离散 token，增强了可组合性和可编辑性。
  - 相比连续潜变量，更利于结构化建模和反事实分析。

- **世界模型与策略模型深度耦合**
  - 不是把世界模型作为外部辅助模块，而是在同一 Transformer 和 token-editing 框架中联合建模世界演化与动作生成。

- **动作 token 设计较细致**
  - 基于二维加速度而非直接位置；
  - 使用 soft-label interpolation 减少硬量化误差；
  - 通过连续加速度和轨迹重构损失保证物理一致性。

- **分层决策-规划结构合理**
  - 高层决策处理多模态驾驶选择；
  - 低层动作扩散编辑生成平滑轨迹；
  - 理论上通过条件总相关分析解释其作用。

- **离散扩散并行编辑具有推理效率优势**
  - 相比自回归逐步生成，动作 token 可在每轮中并行更新；
  - 中低编辑轮数下延迟更低。

- **实验覆盖面较广**
  - 包含规划、世界生成、消融、反事实、注意力分析和推理延迟。
  - 不只报告最终分数，也分析了模型行为机制。

- **支持可控生成和反事实安全分析**
  - 可以输入不同动作条件生成不同未来；
  - surprise metric 可作为潜在风险评估信号。

---

## 8. 不足与局限

- **真实闭环验证不足**
  - 主要评估基于 NAVSIM 的 open-loop / pseudo-simulation 指标；
  - 尚未证明在真实闭环仿真或实车部署中的长期稳定性。

- **长时世界建模能力有限**
  - Discrete-WAM 的世界生成主要面向短时 4 秒 / 8 帧；
  - 相比 DrivingWorld 等支持更长 rollout 的方法，长时交互仿真能力仍有待验证。

- **对 benchmark 依赖较强**
  - 方法在 NAVSIM 上表现很好，但缺少更多跨 benchmark 评估；
  - 例如 nuScenes、Waymo、CARLA closed-loop 或真实 log replay 的泛化实验没有充分展开。

- **训练成本较高**
  - 使用约 1B 参数模型和 32 张 H20 GPU；
  - 虽然 LoRA 降低了微调成本，但整体预训练仍需要较大资源。

- **决策 token 构造仍带有人为设计**
  - 高层决策空间由 lateral path primitives 和 longitudinal speed profiles 组合得到；
  - 决策监督通过 EPDMS 子分数 winner-take-all 生成；
  - 这说明方法并非完全端到端自动发现高层决策结构。

- **反事实推理仍主要是模型内部一致性**
  - surprise 与 PDMS 的相关性说明模型能感知异常未来；
  - 但这不等价于严格的因果保证；
  - 生成模型可能仍受到训练数据偏差和视觉预测误差影响。

- **推理延迟比较存在限制**
  - AR baseline 未使用 KV-cache；
  - 离散扩散也未使用额外工程优化；
  - 因此延迟实验更适合作为解码范式分析，而非最终部署性能结论。

- **attention map 解释存在不确定性**
  - 天空区域 attention 被解释为 attention sink 或 implicit register；
  - 但 attention 本身不能严格证明因果依赖；
  - 需要更强的因果干预实验来支持解释性结论。

- **安全关键长尾场景覆盖仍有限**
  - 虽然有 navhard 和反事实实验，但真实世界长尾交通风险远更复杂；
  - 对极端天气、遮挡、多智能体博弈、复杂交通法规等场景的评估仍不足。

# 总体评价

Discrete-WAM 的核心贡献在于把自动驾驶中的世界建模、动作生成和高层决策统一到一个离散 token editing 框架中。它不仅追求更高的规划分数，还试图让模型具备动作条件未来生成、反事实比较和安全感知能力。论文方法设计完整，实验覆盖较广，在 NAVSIM benchmark 上取得了强性能。不过，其结论仍主要建立在公开 benchmark 和短时生成任务上，真实闭环泛化、长时交互建模和因果可靠性仍需要进一步验证。
