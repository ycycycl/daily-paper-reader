## 摘要
视觉-语言-动作（Vision-Language-Action，VLA）模型在自动驾驶中展现出有前景的能力，凸显了统一多模态架构在联合建模感知与规划方面的潜力。然而，当前基于 VLA 的驾驶行为如何根植于视觉信息仍缺乏充分理解。现有评估协议主要关注总体性能指标，缺乏结构化且实用的诊断方法来量化视觉—行为依赖关系。在本研究中，我们提出了一种结构化的多层级视觉扰动框架，用于系统分析基于 VLA 的驾驶模型中的视觉—行为依赖关系。该框架沿三个互补维度组织受控视觉扰动：通道层级退化、信息层级破坏和结构层级修改。我们将其应用于基于 VLA 的驾驶系统，并在开环轨迹预测和交互式闭环安全评估两种设置下评估其行为响应。实验结果揭示了依赖于评估方式的依赖模式，以及不同抽象层级上不均衡的视觉扎根现象。这些发现表明，有必要对 VLA 驾驶模型开展更结构化的分析并进行有原则的设计，以更好地理解视觉信息如何塑造行为，并开发更安全、更鲁棒的系统。

## Abstract
Vision-Language-Action (VLA) models have demonstrated promising capability in autonomous driving, highlighting the potential of unified multimodal architectures for jointly modeling perception and planning. However, how current VLA-based driving behavior is grounded in visual information remains poorly understood. Existing evaluation protocols mainly focus on aggregate performance metrics, lacking structured and practical diagnostics to quantify visual-behavior dependency. In this work, we introduce a structured multi-level visual perturbation framework to analyze visual-behavior dependency in VLA-based driving models systematically. The framework organizes controlled visual perturbations along three complementary dimensions: channellevel degradation, information-level disruption, and structurelevel modification. We apply it to VLA-based driving systems and evaluate behavioral responses under both open-loop trajectory prediction and interactive closed-loop safety evaluation. Experimental results reveal evaluation-dependent dependency patterns and uneven visual grounding across abstraction levels. These findings call for more structured analyses and principled design of VLA driving models to better understand how visual information shapes behavior and develop safer, more robust systems.

# 2. 方法论：多层级视觉扰动框架

## 2.1 核心思想

- 作者提出一个**结构化多层级视觉扰动框架**，用于分析 VLA 模型的“视觉—行为依赖关系”。
- 基本思路是：
  1. 给定一个 VLA 驾驶模型；
  2. 在推理阶段对视觉输入或视觉 token 表征施加可控扰动；
  3. 保持其他输入如自车状态、历史信息、文本提示不变；
  4. 比较干净视觉输入与扰动视觉输入下的轨迹预测或闭环驾驶表现；
  5. 根据性能变化判断模型对视觉信息的依赖程度。

## 2.2 问题形式化

- 论文将 VLA 驾驶策略表示为：

\[
a_t = f_\theta(I_t, S_t)
\]

其中：

- \(f_\theta\)：VLA 驾驶模型；
- \(I_t\)：当前时刻视觉观测，例如前视 RGB 图像；
- \(S_t\)：辅助信息，例如自车状态、历史轨迹、雷达或文本提示；
- \(a_t\)：模型输出的未来轨迹或驾驶动作。

- 对视觉输入施加扰动后：

\[
\tilde{a}_t = f_\theta(\tilde{V}_t, S_t)
\]

其中：

- \(\tilde{V}_t\)：被扰动后的视觉图像或视觉 token 表征。

- 通过比较 \(a_t\) 与 \(\tilde{a}_t\) 对应的 benchmark 表现，衡量视觉扰动对驾驶行为的影响。

## 2.3 三类视觉扰动

论文将扰动划分为三个层级，对应视觉处理过程中的不同抽象程度。

### 2.3.1 通道层级扰动：Channel-level Perturbation

- 作用对象：原始图像空间。
- 目标：测试视觉通道是否整体上对模型行为有影响。
- 特点：改变底层像素信号或完全移除视觉输入。

具体操作包括：

- **Gaussian Replacement, GR**：
  - 用随机高斯图像替换原始输入图像；
  - 保持图像尺寸和通道数不变；
  - 去除真实语义内容，但保留某种低层统计形式。

- **Image Removal**：
  - 推理时完全移除图像输入；
  - 模型只能依赖文本提示、自车状态和历史轨迹等非视觉信息。

---

### 2.3.2 信息层级扰动：Information-level Perturbation

- 作用对象：图像空间或视觉编码后的 token 空间。
- 目标：测试模型对视觉语义密度和信息量的依赖。
- 特点：减少视觉细节或删除部分视觉 token，但尽量保留整体输入结构。

具体操作包括：

- **图像下采样再上采样**：
  - 将图像按一定比例降低分辨率，再恢复到原始尺寸；
  - 破坏细节和语义信息质量，但保持输入尺寸。

- **随机视觉 token 裁剪 / 删除**：
  - 在视觉 token 进入跨模态融合前，随机丢弃一定比例的 token；
  - 比例包括 50%、75%、90%。

- **FastV token pruning**：
  - 使用 FastV 方法基于重要性准则删除视觉 token；
  - 同样测试 50%、75%、90% 的 token 剪枝比例。

---

### 2.3.3 结构层级扰动：Structure-level Perturbation

- 作用对象：视觉编码器输出的 token 表征。
- 目标：测试模型是否依赖视觉空间结构和位置关系。
- 特点：保留视觉内容和 token 数量，但破坏空间组织或位置编码。

具体操作包括：

- **Global Shuffling**：
  - 随机打乱所有视觉 token 的特征顺序；
  - 位置编码索引保持不变；
  - 破坏视觉内容与空间位置的对应关系。

- **Positional Shuffling**：
  - 打乱视觉 token 的位置编码索引；
  - token 特征本身不变；
  - 用于测试模型对 RoPE 等位置机制的依赖。

- **Block-wise Shuffling**：
  - 将相邻视觉 token 分成局部块，例如 block-4、block-8；
  - 块内 token 顺序不变；
  - 块之间随机交换；
  - 用于分析中尺度空间结构被扰动后的影响。

## 2.4 视觉依赖度量

- 论文使用相对性能变化来衡量视觉扰动造成的影响：

\[
D(T) = \frac{M(f_\theta(\tilde{V}, S)) - M(f_\theta(V, S))}{|M(f_\theta(V, S))|}
\]

其中：

- \(T\)：扰动算子；
- \(M(\cdot)\)：benchmark 指标；
- \(V\)：干净视觉输入或表征；
- \(\tilde{V}\)：扰动后的视觉输入或表征。

- 对于不同任务，指标含义不同：
  - 在开环轨迹预测中，使用 Mean L2 Error，数值越低越好；
  - 在闭环安全评估中，使用 NCAP Score，数值越高越好。

## 2.5 算法流程

论文给出的整体流程可以概括为：

1. 选择一个 VLA 驾驶模型 \(f_\theta\)；
2. 在 benchmark 上使用干净视觉输入得到基线性能；
3. 依次选择通道层级、信息层级和结构层级扰动；
4. 对视觉输入或视觉 token 表征施加扰动；
5. 在相同 benchmark 上重新评估模型；
6. 计算相对性能变化；
7. 比较不同扰动和不同评估环境下的视觉—行为依赖模式。

# 3. 实验设计：数据集、benchmark 与对比设置

## 3.1 使用的模型

  - 可复现；
  - 与视觉 token 扰动分析兼容；
  - 具有较典型的 VLA 自动驾驶架构特征。

## 3.2 使用的数据集和场景

### nuScenes

- 用于开环轨迹预测评估。
- 论文使用 Impromptu-VLA 的评估设置，在 **NavTest split** 上测试。
- 主要衡量模型在离线场景中预测未来轨迹的准确性。

### NeuroNCAP

- 用于闭环安全评估。
- NeuroNCAP 是一个照片级真实感闭环安全测试环境。
- 它能够模拟车辆与环境的交互反馈，因此比纯开环评估更能反映安全关键行为。
- 每个场景执行 **30 次运行**，报告平均结果，以降低随机性影响。

## 3.3 评估指标

### nuScenes 开环指标

- 使用 1 秒、2 秒、3 秒预测轨迹的 L2 位移误差；
- 报告三者算术平均值：

\[
Mean\ L2\ Error(1-3s)
\]

- 单位为米；
- 数值越低表示轨迹预测越准确。

### NeuroNCAP 闭环指标

- **NCAP Score**：
  - 主要安全指标；
  - 越高表示驾驶行为越安全；
  - 作者对原始 NeuroNCAP 协议做了更严格处理：  
    将驶出可行驶区域的情况也作为碰撞失败处理，对应 episode 记为 0 分。

- **Collision Rate, CR**：
  - 碰撞率；
  - 越低越好。

- **Out-of-Bounds Rate, OOB**：
  - 驶出可行驶区域比例；
  - 越低越好。

# 6. 主要结论与发现

## 6.1 开环轨迹预测对视觉扰动不敏感

- 在 nuScenes 开环评估中，各类扰动带来的 Mean L2 Error 变化普遍较小。
- 即使完全移除图像输入，Mean L2 Error 也只从 0.311 m 增加到 0.333 m，约 +7.1%。
- 这说明：
  - 开环轨迹预测可能强烈依赖历史轨迹、自车状态和数据集中的运动先验；
  - 视觉输入对开环轨迹误差的影响没有想象中明显；
  - 开环指标可能低估视觉 grounding 的重要性。

## 6.2 闭环安全评估更能暴露视觉依赖

- 在 NeuroNCAP 闭环评估中，视觉扰动对 NCAP Score 影响明显更大。
- 完全移除图像输入时，NCAP Score 从 1.555 降至 1.328，下降约 14.6%。
- 信息层级扰动中，严重下采样导致更大安全性能下降：
  - Downsample 75%：NCAP 下降 27.2%；
  - Downsample 90%：NCAP 下降 31.5%。

- 这表明：
  - 闭环交互场景中，模型需要持续根据视觉反馈调整行为；
  - 视觉 grounding 的缺陷会更直接反映为安全性能下降；
  - 闭环评估比开环轨迹误差更适合诊断安全关键的视觉依赖。

## 6.3 不同视觉抽象层级的作用不均衡

- 通道层级扰动影响中等；
- 信息层级扰动，尤其是编码前图像下采样，对闭环安全影响最大；
- token pruning 的影响相对小于图像下采样，说明模型可能并不需要密集视觉 token，而是依赖少量关键语义特征；
- 结构层级扰动中，位置扰动比单纯内容打乱更有破坏性。

## 6.4 空间位置编码对 VLA 驾驶行为很重要

- Positional shuffling 使闭环 NCAP Score 下降 15.8%，比 global shuffling 的 7.0% 更严重。
- 这说明：
  - VLA 模型不仅依赖视觉内容，还依赖视觉内容与空间位置的正确对应；
  - Transformer 中的位置编码和跨模态对齐机制对驾驶行为具有重要影响。

## 6.5 部分结构扰动可能导致“保守行为”

- Block-8 shuffling 在闭环中反而使 NCAP Score 上升 8.0%。
- 作者认为这不代表模型理解更好，而可能是结构扰动使模型感知到更复杂或不确定的环境，从而采取更保守策略，降低碰撞率。
- 因此，性能提升不一定意味着更强视觉理解，需要结合行为分析解释。

# 7. 优点与亮点

## 7.1 问题切入重要

- 论文关注 VLA 自动驾驶模型是否真正依赖视觉信息，这是端到端自动驾驶安全性和可解释性的核心问题。
- 它指出高 benchmark 分数不等于强视觉 grounding，这一点对当前 VLA 自动驾驶研究具有现实意义。

## 7.2 方法框架清晰

- 将视觉扰动分为：
  - 通道层级；
  - 信息层级；
  - 结构层级。
- 这种划分与视觉处理过程的不同抽象阶段对应，具有较强解释性。

## 7.3 同时覆盖开环和闭环评估

- 论文没有只依赖 nuScenes 开环轨迹误差，而是引入 NeuroNCAP 闭环安全测试。
- 这使得论文能够揭示：
  - 开环指标对视觉扰动不敏感；
  - 闭环安全指标更能暴露视觉 grounding 缺陷。

## 7.4 扰动设计较全面

- 论文覆盖了从像素输入到视觉 token，再到空间结构和位置编码的多个层面。
- 特别是结构层级扰动，如 positional shuffling 和 block-wise shuffling，能够更细致分析视觉空间组织对行为的影响。

## 7.5 推理期分析成本较低

- 所有扰动均在推理阶段完成，不需要重新训练模型。
- 这使得该框架可作为一种通用诊断工具，用于后续不同 VLA 模型的分析。

---

# 8. 不足与局限

## 8.1 只评估了一个 VLA 模型

- 实验仅使用 Impromptu-VLA 3B 版本。
- 因此结论可能受到该模型架构、训练数据、输入格式和 Qwen2.5-VL backbone 的影响。
- 尚不能确定这些发现是否适用于：
  - 更大规模 VLA 模型；
  - diffusion planner；
  - reinforcement fine-tuned VLA；
  - 纯视觉端到端驾驶模型；
  - 模块化自动驾驶系统。

## 8.2 缺少跨模型横向比较

- 论文没有比较 AutoVLA、OpenDriveVLA、Drive-R1、RecogDrive 等其他 VLA 方法。
- 因此难以判断不同架构在视觉依赖上的差异。

## 8.3 闭环场景覆盖信息不足

- 虽然使用了 NeuroNCAP，但文中没有充分说明：
  - 总共测试了多少具体场景；
  - 场景类型分布；
  - 是否包含行人、车辆切入、恶劣天气、复杂路口等多样情况。
- 定性分析只展示了一个 stationary obstacle 场景，案例覆盖较有限。

## 8.4 扰动可能引入分布外输入

- Gaussian replacement、全局 token shuffle、位置编码打乱等操作可能产生模型训练时从未见过的输入分布。
- 因此性能下降可能同时来自：
  - 视觉依赖；
  - 分布外鲁棒性不足；
  - 模型对异常 token 排列的不适应。
- 这使得“视觉 grounding”与“OOD 鲁棒性”之间存在一定混淆风险。

## 8.5 缺少对内部表征的深入分析

- 论文主要基于最终行为指标判断视觉依赖。
- 没有进一步分析：
  - attention map；
  - token importance；
  - 跨模态对齐权重；
  - 中间层表征变化；
  - 语言 token 与视觉 token 的交互方式。
- 因此对模型内部机制的解释仍有限。

## 8.6 算力和复现成本信息不完整

- 论文没有报告具体 GPU、推理时长或仿真成本。
- 对希望复现实验的研究者而言，闭环 NeuroNCAP 测试的实际资源需求仍不清楚。

## 8.7 评价指标仍可能存在解释歧义

- 某些扰动下 NCAP Score 提升并不一定代表模型更好，而可能代表模型更保守。
- 仅凭 NCAP、CR、OOB 等指标难以区分：
  - 更安全的理解；
  - 更保守的规划；
  - 偶然的行为偏移。
- 因此仍需要更细粒度的行为解释指标。

---

# 总结

本文提出了一套结构化多层级视觉扰动框架，用于诊断自动驾驶 VLA 模型的视觉—行为依赖关系。实验表明，开环轨迹预测对视觉扰动普遍不敏感，而闭环安全评估更能揭示视觉 grounding 的重要性。模型对视觉信息的依赖并非均匀分布：低层像素完整性影响有限，语义形成和空间位置结构对闭环行为更关键。论文的主要价值在于提醒研究者，不能仅凭开环轨迹误差判断 VLA 驾驶模型是否真正“看见”并理解环境，未来需要更系统的闭环、多模型、多场景诊断来提升端到端自动驾驶系统的安全性和可靠性。
