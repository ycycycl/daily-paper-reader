## 摘要
真实世界中的家用机器人需要视觉-语言-动作（Vision-Language-Action, VLA）基础模型，以便在多样化物体、任务条件和家庭环境中获得可复用的操作技能。变形物体折叠是一项具有代表性的挑战，要求机器人能够在不同类别、几何形状、材料和场景下，从随机初始状态处理衣物。然而，现有 VLA 系统通常针对不同物体类别训练独立策略，而朴素的混合多任务训练往往会受到任务干扰并导致性能下降。为突破类别特定折叠策略的限制，我们提出 DeMaVLA，一种面向可泛化变形物体操作的 VLA 基础模型。DeMaVLA 采用带有动作专家的 VLM 骨干网络，并使用流匹配来表述连续动作生成。为提高效率，动作专家通过剪除每隔一层的 Transformer 层来构建，同时保持与 VLM 骨干网络的逐层对齐，从而降低训练和推理成本。DeMaVLA 首先在约 5,000 小时经过筛选的真实世界双臂示范数据上进行预训练，以获得通用操作先验。随后，通过人在环数据聚合（DAgger）流程，在混合折叠数据上进行后训练；该数据汇集了多个折叠任务中的自采集示范以及来自真实机器人失败案例的纠正轨迹。实验表明，DeMaVLA 在 RoboTwin 上取得了具有竞争力的性能，并在我们的家庭折叠基准上获得了强劲的真实世界结果。这些结果凸显了可扩展真实世界数据、高效动作生成以及纠正学习对于变形物体操作中通用 VLA 策略的价值。

## Abstract
Real-world household robots require Vision-Language-Action (VLA) foundation models that can acquire reusable manipulation skills across diverse objects, task conditions, and household environments. Deformable-object folding is a representative challenge, requiring robots to handle clothing items from random initial states across varying categories, geometries, materials, and scenes. However, existing VLA systems commonly train separate policies for different object categories, while naively mixed multi-task training often suffers from task interference and degraded performance. To move beyond category-specific folding policies, we introduce DeMaVLA, a VLA foundation model for generalizable Deformable Manipulation. DeMaVLA adopts a VLM backbone with an action expert and formulates continuous action generation using flow matching. To improve efficiency, the action expert is constructed by pruning every other transformer layer while preserving layer-wise alignment with the VLM backbone, reducing training and inference cost. DeMaVLA is first pre-trained on approximately 5,000 hours of selected real-world dual-arm demonstrations to acquire general manipulation priors. It is then post-trained on mixed folding data that aggregates self-collected demonstrations and corrective trajectories from real-robot failures across multiple folding tasks through a human-in-the-loop Data Aggregation~(DAgger) pipeline. Experiments show that DeMaVLA achieves competitive performance on RoboTwin and strong real-world results on our household folding benchmark. These results highlight the value of scalable real-world data, efficient action generation, and corrective learning for general-purpose VLA policies in deformable-object manipulation.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体框架

- DeMaVLA 是一个面向双臂机器人操作的 VLA 模型。
- 输入包括：
  - 多视角图像观测；
  - 自然语言指令；
  - 机器人本体状态；
  - 带噪声的未来动作块。
- 输出是：
  - 用于生成连续动作块的向量场，即通过 flow matching 逐步生成机器人未来动作。

论文将观测上下文定义为：

\[
O_t = [I_t^1, I_t^2, I_t^3, \ell_t, q_t]
\]

其中：

- \(I_t^1, I_t^2, I_t^3\)：三个操作视角图像；
- \(\ell_t\)：语言指令；
- \(q_t\)：机器人本体状态。

未来动作块定义为：

\[
A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]
\]

其中：

- \(H\)：动作预测时域；
- 每个 \(a_t\)：连续双臂控制命令。

---

### 2.2 VLM 骨干与动作专家

- DeMaVLA 使用 **Qwen3-VL** 作为视觉-语言骨干模型。
- 图像和语言被编码为视觉-语言 token，为动作生成提供语义和空间上下文。
- 由于标准 VLM 不直接处理机器人本体状态和连续动作预测，因此论文引入了一个 **action expert，动作专家**：
  - 将机器人本体状态 \(q_t\) 通过线性层投影到 Transformer embedding 空间；
  - 将带噪声的未来动作块表示为 \(H\) 个动作 token；
  - 动作专家在 VLM 上下文条件下处理这些机器人相关 token；
  - 最终输出 flow matching 所需的向量场，用于连续动作去噪和生成。

---

### 2.3 Layer-Aligned Pruned Action Expert：逐层对齐剪枝动作专家

- 论文的一个关键结构设计是 **layer-aligned pruned action expert**。
- 动作专家复用 Qwen3-VL 中 LLM 的 Transformer 结构，以保持与视觉-语言骨干的逐层对应关系。
- 但完整复制 LLM 作为动作专家成本很高，尤其 flow matching 在推理时需要多次前向传播。
- 因此作者采用剪枝策略：
  - 保留部分 Transformer 层；
  - 剪除约一半层，即“每隔一层剪除”；
  - 保留层仍与 VLM 骨干中的对应层保持层级对齐；
  - 被剪除层之间直接跳连，将前一保留层输出传给后一保留层。
- 这样既降低了训练和推理成本，又尽量保留了视觉-语言层级条件信息。

模型参数规模如下：

| 模块 | 参数量 |
|---|---:|
| Vision Encoder | 0.4B |
| Base Language Model | 4.0B |
| Action Expert | 2.2B |
| Total | 6.6B |

### 2.4 Flow Matching 连续动作生成

- DeMaVLA 不将动作离散化为语言 token，而是采用 **conditional flow matching** 建模连续动作块。
- 核心思想：
  - 给定观测上下文 \(O_t\)；
  - 对真实动作块 \(A_t\) 加噪；
  - 训练动作专家预测从噪声动作到真实动作的连续向量场；
  - 推理时通过多步去噪生成平滑的双臂动作序列。
- 这种方式适合双臂操作，因为：
  - 连续动作空间不需要离散化损失；
  - 可以表示多模态动作分布；
  - 生成动作更平滑，有利于长时程控制。

### 2.6 Training-time RTC：训练时实时动作块条件化

- 真实部署时，模型生成下一个动作块需要时间，而机器人还在执行前一个动作块。
- 为处理异步执行延迟，论文使用 **training-time real-time chunking，RTC**。
- 做法：
  - 将一个动作块分为：
    - committed prefix：已经确定并将被执行的动作前缀；
    - predicted postfix：模型需要预测的未来动作后缀。
  - 训练时：
    - 前缀保持干净，用作条件；
    - 只对后缀加噪；
    - loss 只计算在后缀动作上。
  - 推理时：
    - 模型根据已承诺动作生成剩余未来动作；
    - 不需要额外的 inference-time inpainting 或反向优化引导。

---

### 2.7 Human-in-the-Loop DAgger：人类在环纠错

- 论文采用类似 HG-DAgger 的人类门控交互模仿学习。
- 流程：
  1. 先用行为克隆训练初始策略；
  2. 将混合折叠策略部署到真实机器人；
  3. 人类观察机器人执行；
  4. 当机器人进入不安全、不稳定或明显次优状态时，人类接管；
  5. 人类纠正机器人动作，使其回到合理任务状态；
  6. 纠正完成后释放控制权；
  7. 将人类干预数据加入训练集；
  8. 继续后训练模型。

形式化表示为：

\[
\pi_{exec}(o_t) = g_t \pi_H(o_t) + (1 - g_t)\pi_\theta(o_t)
\]

其中：

- \(\pi_\theta\)：当前 DeMaVLA 策略；
- \(\pi_H\)：人类专家策略；
- \(g_t \in \{0,1\}\)：人类是否介入；
- \(g_t=1\) 时由人类控制，\(g_t=0\) 时由模型控制。

记录的干预数据为：

\[
D_{int} = \{(o_t, a_t^H) | g_t = 1\}
\]

并进行数据聚合：

\[
D \leftarrow D \cup D_{int}
\]

- 这种方式重点收集模型真实失败状态附近的数据，比单纯离线示范更能缓解 covariate shift。

  - 对比了多个强 VLA 基线；
  - 进行了预训练规模分析，证明数据规模的影响；
  - 真实世界中使用相同硬件、控制频率和初始化协议，公平性较好。

- **不足之处**：
  - 缺少更细粒度的 ablation study，例如：
    - 不使用剪枝动作专家；
    - 使用完整动作专家；
    - 不使用 DAgger；
    - 不使用 training-time RTC；
    - 不使用 flow matching；
    - 不同 action horizon；
    - 不同相机视角数量。
  - 真实世界任务类别只有 4 类衣物，虽然有代表性，但对“通用变形物体操作”的覆盖仍有限。
  - 每类任务 20 次 trial，真实机器人实验成本高可以理解，但统计置信度仍有限。
  - DAgger 数据收集顺序可能影响结果，后续任务受益于前序任务训练，这有利于展示迁移，但也可能引入任务顺序偏差。
  - 论文没有报告失败类型统计、人类干预次数变化、DAgger 每轮数据量等细节，难以全面评估纠错学习的贡献。

## 6. 主要结论与发现

- DeMaVLA 能够用单一 checkpoint 完成多类别家庭折叠任务，表现优于强 VLA baseline。
- 在 RoboTwin 仿真基准上，DeMaVLA 取得最高平均成功率：
  - Clean：88.42%；
  - Randomized：86.78%。
- 在真实多任务家庭折叠 benchmark 上，DeMaVLA 平均成功率达到 **92.5%**，明显高于 \(\pi_0\) 的 **76.3%**。
- DeMaVLA 在毛巾折叠上提升尤其显著：
  - \(\pi_0\)：55.0%；
  - DeMaVLA：100.0%。
- 大规模真实世界预训练数据显著提升性能：
  - 500 小时预训练后 SR 为 55.0%；
  - 5,000 小时预训练后 SR 达到 100.0%。
- 人类在环 DAgger 有助于针对真实机器人失败状态收集高价值纠错数据，从而提升多任务折叠鲁棒性。
- 高效动作专家设计和 training-time RTC 对真实部署中的长时程异步动作执行具有重要意义。

## 7. 优点与亮点

- **问题选择有现实价值**：
  - 柔性物体折叠是家庭机器人中非常典型且困难的任务，具有明确应用意义。

- **单 checkpoint 多任务折叠**：
  - 相比每个衣物类别训练一个策略，DeMaVLA 更接近通用家庭机器人需求。

- **结合大规模真实数据与纠错数据**：
  - 5,000 小时真实双臂预训练数据提供通用操作先验；
  - DAgger 纠错数据专门覆盖模型失败状态，增强鲁棒性。

- **动作生成方式合理**：
  - 使用 flow matching 直接建模连续动作块，避免动作离散化带来的精度损失。

- **动作专家设计兼顾性能与效率**：
  - 通过逐层对齐剪枝动作专家减少计算负担，同时保持 VLM 层级信息条件化。

- **真实 benchmark 设计较有挑战性**：
  - 衣物从篮子中随机初始状态开始，而非预先平铺，任务更贴近真实家庭场景。

- **评估覆盖仿真与真实世界**：
  - RoboTwin 展示一般双臂任务能力；
  - 真实折叠 benchmark 展示目标应用场景能力。

## 8. 不足与局限

- **算力与训练细节不透明**：
  - 没有报告 GPU 配置、训练时长、batch size、学习率、训练 step 等关键复现信息。

- **消融实验不足**：
  - 论文提出多个关键组件，但没有系统量化每个组件的单独贡献。
  - 特别是缺少：
    - 剪枝动作专家 vs 完整动作专家；
    - DAgger vs 无 DAgger；
    - RTC vs 无 RTC；
    - flow matching vs 其他连续动作生成方式；
    - 大规模预训练中折叠数据占比的影响。

- **真实任务覆盖仍有限**：
  - 虽然包括衬衫、裙子、裤子、毛巾，但仍主要集中在衣物折叠。
  - 对其他变形物体，如被子、床单、袋子、软包装、线缆等，没有实验验证。

- **真实机器人试验数量有限**：
  - 每类 20 次 trial，统计上仍可能受物体选择、初始状态采样和环境条件影响。

- **数据集未完全公开信息不足**：
  - 预训练数据由公开和自采集数据组成，但论文没有详细说明数据来源比例、采集流程、质量筛选标准等。
  - 如果数据不公开，外部复现难度较高。

- **DAgger 过程细节不足**：
  - 没有报告每轮人类介入频率、干预时长、纠错轨迹数量、不同任务的 DAgger 数据占比。
  - 因此难以判断交互学习的数据效率和人工成本。

- **泛化边界尚不清楚**：
  - 论文展示了跨衣物类别泛化，但对未知衣物类别、极端材质、复杂遮挡、不同机器人平台、不同家庭环境的泛化能力仍未充分验证。

- **可能存在任务顺序偏差**：
  - DAgger 数据按衬衫、裙子、裤子、毛巾顺序增量收集，后续任务受益于前面任务学习到的先验。
  - 这体现了迁移能力，但也可能使不同任务间的比较不完全独立。
