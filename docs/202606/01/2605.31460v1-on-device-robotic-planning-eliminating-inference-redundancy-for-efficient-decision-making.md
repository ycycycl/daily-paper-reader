## 摘要
基于推理的机器人策略利用大语言模型和视觉-语言模型实现了强大的语义规划能力，但大多存在较高的推理延迟，从而限制了其实时实际部署。在本文中，我们观察到机器人推理工作负载中存在大量时间冗余，即连续观测常常会产生相同的动作和子目标。基于这一洞察，我们提出了 REIS，一种受人类认知启发的机器人决策框架，旨在在保持语义适应性的同时尽量减少不必要的推理。REIS 结合了轻量级场景门控、KV 引导的可供性路由以及审慎推理，以在具身约束下加速机器人控制。在 ALFRED 和真实世界机器人任务上的实验表明，REIS 在保持具有竞争力的任务性能的同时，显著降低了推理开销。

## Abstract
Reasoning-based robotic policies using large language and vision-language models achieve strong semantic planning capabilities but mostly suffer from a high inference latency that limits practical real-time deployment. In this work, we observe that robotic reasoning workloads contain substantial temporal redundancy, where consecutive observations frequently produce identical actions and subgoals. Based on this insight, we present REIS, a human cognition inspired robotic decision-making framework that minimizes unnecessary reasoning while preserving semantic adaptability. REIS combines lightweight scene gating, KV-steered affordance routing, and deliberative reasoning to accelerate robotic control under embodied constraints. Experiments on ALFRED, and real-world robotic tasks demonstrate that REIS significantly suppresses reasoning overhead while maintaining competitive task performance.

### 1.1 研究背景

近年来，基于大语言模型（LLM）、视觉语言模型（VLM）和视觉语言动作模型（VLA）的机器人策略逐渐形成“reason-and-act”的决策范式。此类方法通过显式的中间推理、语义理解和高层任务规划，使机器人能够更好地处理长时程任务、语言指令、复杂场景和语义泛化。

例如，CoT-VLA、DiffusionVLA 等方法表明，引入视觉链式思维、扩散式动作生成或语言规划，可以提升机器人在导航和操作任务中的表现。然而，这类推理型机器人策略通常存在一个关键瓶颈：**推理延迟过高**。

机器人控制往往需要在 10–100 ms 级别做出响应，但基于 VLM / LLM 的推理通常涉及自回归文本生成、多步链式思维或迭代式扩散推理，导致延迟远高于实时控制需求。尤其在动态环境中，模型推理完成时，当前观测可能已经过时，从而导致规划失效或反应迟缓。

### 1.2 核心问题

论文关注的问题是：

> 如何在端侧机器人平台上，减少基于 VLM / LLM 的高层语义推理延迟，同时尽量保持机器人任务执行的语义规划能力和适应性？

论文提出的核心观察是：

- 机器人连续观测之间存在大量时间冗余；
- 相邻帧往往对应相同动作、相同子目标或相同高层规划状态；
- 因此，并不需要对每一帧都进行完整的 VLM / LLM 推理。

### 1.3 整体含义

该论文的意义在于，将机器人推理系统的优化目标从“如何更快地执行每次推理”转向“如何避免不必要的推理”。这是一种更贴近端侧部署约束的思路。

## 2. 方法论：核心思想、关键技术与流程

论文提出的方法名为 **REIS**，即 **Redundancy Elimination for Inference-efficient Systems**，意为“面向高效推理系统的冗余消除”。

### 2.1 总体框架：双系统决策架构

REIS 采用类似人类认知中“双系统”的设计：

- **System One：快速直觉系统**
  - 负责轻量级视觉变化检测、动作保持判断、子目标状态验证；
  - 尽量避免调用大模型完整推理；
  - 适合处理连续、重复、低变化的状态。

- **System Two：审慎推理系统**
  - 负责完整的高层语义规划、多步推理和重新规划；
  - 只在 System One 判断当前状态复杂、不确定、异常或需要子目标切换时触发；
  - 保留语义适应性和长时程规划能力。

整体流程可以概括为：

1. 机器人接收实时视觉观测和任务指令；
2. System One 判断当前观测是否与之前状态足够相似；
3. 若无显著变化，则继续执行当前动作或子目标；
4. 若发生关键状态变化，则通过 Affordance Router 判断是否需要动作切换、子目标更新或完整重规划；
5. 只有在必要时才触发 System Two 执行完整 VLM / LLM 推理。

### 2.2 关键技术一：EMA-HSVS 轻量视觉场景门控

论文提出 **EMA-HSVS**，即 **Ego-Motion Aware Head-Selective Vision Similarity**，用于快速判断当前视觉观测是否发生了重要结构变化。

其作用是：

- 在导航任务中检测场景级变化，例如转角、路口、墙面结构变化、开放空间出现等；
- 如果当前帧与上一次触发推理的帧高度相似，则跳过完整推理；
- 如果相似度低于阈值，则触发后续 Affordance Router 或 System Two。

#### 技术细节

EMA-HSVS 不是直接使用全部视觉 token 或全部注意力头，而是选择对结构变化敏感的视觉编码器 attention heads。

其流程大致包括：

1. 使用视觉 Transformer 编码输入图像；
2. 从不同 attention heads 中提取特征；
3. 通过贪心搜索选择最能区分“场景变化 / 未变化”的 head 子集；
4. 对选中 head 的特征进行池化和 L2 归一化；
5. 计算当前帧与参考帧之间的余弦相似度；
6. 结合指数滑动平均（EMA）平滑相似度变化；
7. 若相似度低于阈值，则认为出现结构性变化并触发后续推理。

论文还引入 **Self-Mask** 机制，用于屏蔽机器人自身机械臂或自运动造成的像素变化，减少由于机器人本体遮挡导致的误触发。

#### 任务差异

- 在 **导航任务** 中，EMA-HSVS 被启用，因为宏观场景结构变化是重要触发信号；
- 在 **操作任务** 中，EMA-HSVS 被关闭，因为局部物体状态变化很细微，过强的视觉门控可能漏掉关键变化。

### 2.3 关键技术二：KV-Steered Affordance Router

REIS 的第二个核心模块是 **KV-steered Affordance Router**，用于快速判断当前执行状态是否仍然有效，而不进行完整自回归推理。

它主要回答几个二值问题，例如：

- 当前子目标是否已经完成？
- 是否可以逻辑上切换到下一个子目标？
- 是否应该继续当前低层动作？
- 是否出现严重偏离，需要重新规划？

论文将这些判断组合为四类执行状态：

| 状态 | 含义 |
|---|---|
| C1 | 当前子目标未完成，当前动作仍可继续，跳过 System Two |
| C2 | 当前子目标未完成，但低层动作需要改变 |
| C3 | 当前子目标已完成，且可切换到下一个子目标 |
| C4 | 状态不一致、异常或需要高层重规划 |

其中：

- C1 通常直接继续执行；
- C2、C3、C4 会触发不同程度的 System Two 验证或重规划；
- System One 不会直接最终确认语义子目标完成，关键边界事件仍交由 System Two 确认，以避免错误切换。

### 2.4 关键技术三：KV Cache Steering

为减少在线推理成本，论文使用 **KV cache steering** 来注入先验推理信息。

核心思想是：

- 不让模型每次都重新生成完整 reasoning；
- 而是将“带推理路径”和“不带推理路径”的隐藏状态差异提取出来；
- 用这个差异作为 steering vector，在后续推理时注入到模型 KV cache 中；
- 从而使模型在不生成大量文本 token 的情况下，获得类似推理上下文的决策偏置。

#### 公式说明

论文对每一层 Transformer 的 key 和 value 分别计算 steering vector：

\[
S_k^l = \frac{1}{N}\sum_{j=1}^{N}(K^+_{l,j} - K^-_{l,j})
\]

\[
S_v^l = \frac{1}{N}\sum_{j=1}^{N}(V^+_{l,j} - V^-_{l,j})
\]

其中：

- \(K^+\)、\(V^+\)：来自包含详细推理过程的正样本 prompt；
- \(K^-\)、\(V^-\)：来自直接输出答案、不包含推理过程的负样本 prompt；
- \(S_k^l\)、\(S_v^l\)：第 \(l\) 层的 KV steering 向量；
- \(N\)：对比样本数量。

这种方式使模型在回答 affordance 判断问题时，不需要生成长链式思维文本，而是通过隐藏状态引导快速输出 yes / no 概率。

### 2.5 执行模块

REIS 的执行模块负责将高层规划转化为实际机器人动作：

- 导航任务中，使用符号函数调用和轻量视觉跟踪；

- Base + Reasoning 延迟约 **43.59 秒**；
- REIS Affordance Routing 延迟约 **406.21 ms**；
- CAG / RAG 类增强方法延迟约 **1 秒左右**；
- 高分辨率 1920×1080 下 Affordance Routing 延迟约 **4392.50 ms**；
- 224×224 下延迟约 **406.21 ms**；
- 112×112 下延迟约 **362.73 ms**；
- 32×32 下延迟约 **339.71 ms**。

这些结果说明，REIS 的核心优势不是单次视觉前向极致加速，而是通过跳过冗余推理显著降低整体任务耗时。

---

## 5. 实验数量与充分性分析

### 5.1 实验数量概览

论文实验较多，覆盖了多个角度：

#### 冗余性分析

- ALFRED 上统计动作和子目标冗余；
- 真实世界导航数据上统计动作冗余。

主要结果：

- ALFRED 子目标冗余率：**87.10%**
- ALFRED 动作冗余率：**46.89%**
- 真实世界动作冗余率：**83.04%**

#### 整体系统性能实验

在三个主要任务 / 数据集上比较：

- Realworld Navigation；
- Realworld Manipulation；
- ALFRED。

对比：

- Naive；
- Only EMA-HSVS；
- Only Affordance Router；
- REIS。

#### EMA-HSVS 实验

包括：

- 场景级变化检测；
- 物体级变化检测；
- 混淆矩阵比较；
- 不同视觉相似度方法比较；
- head 数量贪心搜索；
- self-mask 消融；
- 不同视觉编码器泛化测试。

#### Affordance Router 实验

包括：

- Base + Direct；
- Base + Reasoning；
- REIS；
- 单帧、两帧、三帧重复验证；
- CAG / RAG / KV cache steering 对比；
- 不同分辨率下延迟与准确率对比。

#### 模型泛化实验

测试多个 3B–4B 模型在三类任务上的表现。

---

### 5.2 实验是否充分

整体来看，论文实验覆盖较广，较充分地支持了“机器人推理存在大量冗余，且可通过视觉门控和 affordance routing 降低推理开销”这一核心论点。

优点包括：

- 同时覆盖仿真与真实机器人；
- 同时覆盖导航和操作；
- 在端侧硬件上真实测量延迟；
- 有消融实验区分 EMA-HSVS 和 Affordance Router 的贡献；
- 有多模型、多视觉编码器测试；
- 有对推理延迟、任务成功率、变化检测准确率的综合评估。

### 5.3 公平性与客观性

论文在整体对比中强调：

- 使用相同 VLM backbone；
- 输入观测流、任务指令和输出 schema 保持一致；
- 方法差异主要来自推理调度策略。

这有助于保证对比公平。

但仍存在一些潜在不足：

- 真实世界数据规模相对有限；
- 真实机器人实验更像可行性验证，而非大规模系统评测；
- ALFRED 上进行了 SFT，而真实场景未 fine-tune，导致不同场景下模型适配程度不一致；
- 部分任务中的机械硬件异常被排除，可能使实际部署难度被低估；
- 对比方法主要是系统变体和固定频率推理，并未与更多最新端侧 VLA 加速系统做完整横向比较。

## 6. 主要结论与发现

### 6.1 机器人推理存在大量时间冗余

论文首先验证了一个关键事实：连续机器人观测中，动作和子目标常常不变。

例如：

- ALFRED 中子目标冗余率达到 **87.10%**；
- 真实世界导航中动作冗余率达到 **83.04%**。

这说明每帧都调用大模型进行完整语义推理是严重浪费的。

---

### 6.2 REIS 可以显著提升端侧执行速度

在整体任务中，REIS 相比 Naive 基线实现了平均约 **15.09×** 的速度提升。

具体来看：

- 真实导航任务速度从 **0.28 cm/s** 提升到 **7.66 cm/s**；
- 真实操作任务速度从 **0.714 cm/s** 提升到 **1.67 cm/s**；
- ALFRED 中速度从 **0.408 frames/s** 提升到 **6.357 frames/s**。

其中导航任务加速最明显，达到约 **27.35×**，说明宏观移动任务中视觉和时间冗余尤其严重。

---

### 6.3 任务成功率下降较小

虽然 REIS 大幅减少推理调用，但任务准确率下降有限。

论文报告：

- Realworld Navigation：从 0.680 降到 0.641；
- Realworld Manipulation：从 0.760 降到 0.729；
- ALFRED：从 0.554 降到 0.512。

平均宏观准确率下降约 **3.73%**。

这说明 REIS 在效率和性能之间取得了较好的折中。

---

### 6.4 KV-Steered Affordance Router 显著优于直接判断和完整 reasoning

在 System Two routing 任务中：

- Base + Direct 准确率：0.762；
- Base + Reasoning 准确率：0.751；
- REIS Affordance Routing 准确率：0.913。

同时，延迟方面：

- Base + Reasoning：43.59 秒；
- REIS：406.21 ms。

这说明 KV steering 能在不生成长文本推理的情况下提供有效的隐藏状态引导。

---

### 6.5 EMA-HSVS 能有效检测结构变化

EMA-HSVS 在变化检测中表现较好：

- 物体级 true-different recall：97.17%；
- 场景级 true-different recall：98.65%；
- 对冗余帧过滤的 precision 约 94.6%。

这表明基于选定 attention heads 的视觉相似度度量，比简单的 CLS、patch mean 或 L2 距离更适合作为机器人推理触发器。

---

## 7. 优点与亮点

### 7.1 问题定义切中端侧机器人痛点

论文没有单纯追求更大模型或更复杂 reasoning，而是抓住端侧部署中的核心矛盾：

- 机器人需要实时反应；
- VLM / LLM 推理很慢；
- 连续观测存在大量冗余。

这一问题设定具有很强的现实意义。

---

### 7.2 从“减少推理次数”而非“加速每次推理”入手

REIS 的核心思路是避免不必要推理，而不是只优化单次模型前向。这对机器人任务尤其有效，因为物理世界连续变化，很多帧并不需要重新规划。

---

### 7.3 双系统架构设计合理

System One / System Two 的分工清晰：

- System One 处理快速、重复、低风险判断；
- System Two 处理高层语义、异常和边界确认。

这种设计兼顾了实时性和安全性。

---

### 7.4 KV Cache Steering 用法具有创新性

论文将 KV steering 用于机器人 affordance routing，避免在线生成 chain-of-thought，同时保留 reasoning 的隐式引导效果。这一思路在端侧场景中尤其有价值。

---

### 7.5 实验覆盖较全面

论文不仅在 ALFRED 上测试，还做了真实机器人导航和操作实验，并包含：

- 消融实验；
- 不同模块对比；
- 不同模型对比；
- 不同视觉编码器对比；
- 不同分辨率延迟分析；
- 真实端侧硬件部署。

---

### 7.6 端侧部署可信度较高

所有主要评估均在 Jetson Orin NX 上进行，而不是只在高端服务器 GPU 上模拟。这增强了论文对实际机器人部署的说服力。

---

## 8. 不足与局限

### 8.1 真实世界实验规模有限

论文虽然包含真实机器人实验，但规模仍然有限：

- 导航环境数量有限；
- 操作任务主要是 Pick-and-Place；
- 真实数据更多用于可行性验证，而非大规模泛化评估。

因此，REIS 在更复杂真实场景中的鲁棒性仍需进一步验证。

---

### 8.2 主要仍依赖高层 function-calling 范式

REIS 当前主要优化的是高层语义规划和动作 / 子目标切换，并非端到端连续控制。对于 π0、Diffusion Policy、OpenVLA 等连续动作生成式 VLA 系统，REIS 尚未完整集成。

作者也指出，未来需要将 affordance routing 扩展到扩散策略和连续轨迹生成中。

---

### 8.3 真实场景未进行模型微调

论文在 ALFRED 上进行了 10 epochs SFT，但真实机器人部署中未进行 fine-tuning。这可能导致真实场景中的语义理解能力受限，也使不同场景间的比较存在适配差异。

---

### 8.4 对异常和安全性的处理仍较初步

虽然论文设计了 boundary safety invariant，即 System One 不直接最终确认子目标完成，但真实机器人安全问题远比子目标切换复杂，例如：

- 动态障碍物；
- 人机共处；
- 抓取失败；
- 碰撞风险；
- 传感器误差；
- 硬件老化。

论文对这些安全问题的覆盖较有限。

---

### 8.5 可能存在门控误判风险

EMA-HSVS 和 Affordance Router 本质上都是近似判断模块，存在：

- 漏检关键变化；
- 错误过滤重要帧；
- 错误保持当前动作；
- 错误延迟重规划。

尽管实验中 recall 较高，但在开放世界环境中，误判风险仍可能影响任务安全和成功率。

---

### 8.6 训练与算力细节不完整

论文明确说明了推理硬件和模型规模，但训练资源报告不足：

- 未说明 SFT 所用 GPU；
- 未说明训练时间；
- 未说明 batch size 和优化细节；
- 未说明 KV steering 数据生成规模和成本。

这使得方法复现仍有一定困难。

---

### 8.7 对比对象仍可扩展

论文主要比较了 Naive、EMA-HSVS-only、Affordance-only、Base+Reasoning、CAG/RAG 等方法，但与一些最新机器人推理调度或 VLA 低延迟系统的直接对比还不充分。

例如：

- 未来状态异步推理；
- 强化学习式“何时思考”调度；
- VLA 专用推理加速框架；
- diffusion policy step pruning 方法。

如果能与这些系统进行统一 benchmark 比较，结论会更有说服力。

---

## 总体评价

这篇论文提出了一个面向端侧机器人规划的实用框架 REIS，核心贡献在于利用机器人连续观测中的时间冗余，减少不必要的大模型推理。方法上，REIS 将视觉变化门控、KV-steered affordance routing 和按需审慎推理结合起来，在 Jetson Orin NX 等资源受限平台上实现了显著加速，同时保持较小的任务性能损失。

论文的实验设计较完整，覆盖仿真、真实导航、真实操作、消融、多模型和端侧延迟分析，能够较好支撑其主要结论。其不足主要在于真实世界实验规模有限、仍局限于高层函数调用式规划、训练资源细节不足，以及对开放环境安全性和复杂连续控制的验证尚不充分。

总体而言，REIS 是一项具有较强实际部署价值的工作，为低延迟 VLM / LLM 机器人规划提供了一个清晰且有效的系统设计方向。
