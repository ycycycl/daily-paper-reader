## 摘要
逐帧动作控制的图像到视频生成是交互式世界模拟的一种有前景的范式，其中每个控制信号都应引发即时的视觉响应。然而，在长程自回归展开过程中保持视觉保真度和三维一致性仍然具有挑战性。现有的三维感知方法往往会因两个障碍而遭受灾难性漂移：其一是潜在—RGB循环（Latent--RGB Cycling）导致的信息损失，即生成的潜在表示被反复解码为 RGB 并重新编码以用于后续条件输入；其二是由无误差假设（error-free hypothesis）引发的训练—推理差距，即干净的训练记忆无法匹配受到预测误差污染的推理记忆。为应对这些挑战，我们提出了 Robust Dreamer，这是一种记忆增强框架，围绕如何设计三维记忆以及如何稳健地使用该记忆而构建。首先，我们引入潜在高斯记忆（Latent Gaussian Memory），将生成过程中继承的扩散潜在表示锚定到高斯基元，并通过潜在空间高斯溅射对其进行召回。这在避免反复 VAE 转换所造成的累积退化的同时，提供了密集、几何感知且视图对齐的条件信息。其次，我们提出了带动态偏差档案的偏差学习（Deviation Learning with Dynamic Deviation Archive），通过一步近似合成由展开引起的潜在偏差，按自回归阶段和去噪时间戳对其进行存储，并在训练过程中将其注入历史记忆。这使生成器能够接触到真实的受污染记忆状态，并在推理前学习内部校正。ScanNet、DL3DV 和 OmniWorldGame 上的实验表明，该方法在长时域性能方面达到了最先进水平。

## Abstract
Frame-wise action-controlled image-to-video generation is a promising paradigm for interactive world simulation, where each control signal should elicit an immediate visual response. However, maintaining visual fidelity and 3D consistency over long autoregressive rollouts remains challenging. Existing 3D-aware methods often suffer from catastrophic drift due to two impediments: information loss from \textit{Latent--RGB Cycling}, where generated latents are repeatedly decoded to RGB and re-encoded for future conditioning, and the training--inference gap induced by the \textit{error-free hypothesis}, where clean training memory fails to match prediction-corrupted inference memory. To address these challenges, we present \textbf{Robust Dreamer}, a memory-augmented framework built around how to design 3D memory and how to use it robustly. First, we introduce \textbf{Latent Gaussian Memory}, which anchors diffusion latents inherited from the generation process to Gaussian primitives and recalls them via latent-space Gaussian splatting. This provides dense, geometry-aware, view-aligned conditioning while avoiding accumulated degradation from repeated VAE conversion. Second, we propose \textbf{Deviation Learning with Dynamic Deviation Archive}, which synthesizes rollout-induced latent deviations through a one-step approximation, stores them by autoregressive stage and denoising timestamp, and injects them into historical memory during training. This exposes the generator to realistic corrupted memory states and teaches internal correction before inference. Experiments on ScanNet, DL3DV, and OmniWorldGame demonstrate state-of-the-art long-horizon performance.

## 1. 核心问题与整体含义：研究动机和背景

本文关注的是**逐帧动作控制的图像到视频生成**，即给定初始图像和一系列用户动作 / 控制信号，模型需要一步一步生成后续视频帧。该问题是交互式世界模拟、具身智能、游戏世界建模和机器人规划中的重要基础能力。

与一次性生成一段短视频不同，本文强调的是：

- **逐帧自回归生成**：每个动作输入都要立即触发下一帧视觉响应；
- **长时程 rollout**：模型需要连续生成几十帧甚至数百帧；
- **三维一致性**：当相机视角变化、回访旧区域或发生遮挡 / 显露时，场景结构应保持稳定；
- **交互性**：不能依赖预先确定的整段相机轨迹，而是要允许每一步动作动态输入。

论文指出，现有 3D-aware 自回归视频生成方法在长时间推演中容易发生**灾难性漂移**，主要原因有两个：

### 1.1 Latent–RGB Cycling 导致信息损失

许多已有方法将历史生成帧存入 3D memory 时，会经历如下过程：

1. 视频扩散模型生成 latent；
2. latent 被 VAE 解码成 RGB 图像；
3. RGB 图像被写入 3D 记忆；
4. 后续需要作为条件时，再将 RGB 渲染结果重新编码回 latent。

这种反复的 **latent → RGB → latent** 转换会引入量化误差和细节损失。单次转换可能影响不大，但在逐帧自回归生成中会被反复累积，最终导致：

- 颜色漂移；
- 纹理模糊；
- 结构损坏；
- 几何不一致。

### 1.2 训练—推理不匹配导致误差累积

训练阶段通常使用干净的真实历史帧构建 memory，而推理阶段的 memory 来自模型自己生成的帧。由于生成帧必然带有误差，推理时 memory 会逐步被污染。

这导致典型的 **error-free hypothesis** 问题：

- 训练时：模型看到的是干净历史；
- 推理时：模型看到的是带有预测误差的历史；
- 结果：模型没有学会如何从污染 memory 中恢复，误差不断放大。

因此，本文的核心意义在于：  
**将 3D memory 从 RGB 空间推进到 diffusion latent 空间，并显式建模自回归误差，使长时程动作控制视频生成更加稳定。**

## 2. 方法论：核心思想、关键技术细节与流程

论文提出了 **Robust Dreamer**，一个面向长时程动作控制视频生成的记忆增强框架。其核心由两个部分组成：

1. **Latent Gaussian Memory**：在潜空间中维护几何感知的 3D 高斯记忆；
2. **Deviation Learning with Dynamic Deviation Archive**：通过动态偏差档案模拟推理时的污染 memory，训练模型纠错。

---

### 2.1 整体自回归推理流程

给定初始帧 `I0` 及其 latent `X0`，模型在每一步接收控制信号 `ci`，生成下一帧。

推理过程大致如下：

1. 使用当前 Latent Gaussian Memory 根据目标视角进行 splatting；
2. 得到视图对齐的 latent memory 条件；
3. 将以下信息拼接作为 Dreamer 的条件输入：
   - 初始 clean anchor latent：`X0`
   - 当前上一帧 latent：`Xi`
   - 从 memory 召回的 latent：`X̂Mi`
   - 当前动作控制信号：`ci`
4. DiT denoiser 生成下一帧 latent `Xi+1`；
5. 将 `Xi+1` 解码为 RGB 图像；
6. 将生成的 latent 直接写回 Latent Gaussian Memory；
7. 重复该过程完成长时程 rollout。

论文中条件形式可概括为：

```text
yi = concat(X0, Xi, X̂Mi, ci)
```

其中 `yi` 是生成下一帧时的条件输入。

### 2.2 Latent Gaussian Memory：潜空间高斯记忆

这是论文的第一个核心贡献。

传统 3D memory 通常存储 RGB 或 RGB-derived features，而本文直接存储扩散模型生成过程中的 latent feature。

每个 memory primitive 被表示为一个 Gaussian：

```text
gs = (μs, os, qs, ss, xs)
```

其中：

- `μs`：高斯中心的 3D 坐标；
- `os`：不透明度；
- `qs`：旋转；
- `ss`：各向异性尺度；
- `xs`：绑定在该高斯上的 latent feature。

这种设计将：

- **几何属性**：位置、尺度、旋转、不透明度；
- **内容属性**：扩散 latent feature；

显式解耦。

#### 写入阶段：Pixel-Aligned Latent Writing

当某一帧 `Ii` 生成后，系统会将其 latent `Xi` 写入 memory。

具体流程是：

1. 使用基于 CUT3R / DUSt3R 风格的 feed-forward reconstruction backbone 预测几何；
2. 图像经过 ViT 编码得到图像 token 和 pose token；
3. DPT head 预测：
   - 全局 pointmap，即每个像素对应的 3D 点；
   - 每个像素对应的 Gaussian 参数；
4. 将生成 latent `Xi` resize 到图像分辨率；
5. 对每个像素位置，将对应 latent feature 绑定到该像素生成的 Gaussian primitive 上。

也就是说，memory 中的内容不是重新编码得到的，而是**直接继承视频扩散模型生成时的 latent**。

#### 召回阶段：Latent-Space Gaussian Splatting

当需要生成下一帧时，系统从目标视角渲染 memory，但渲染的不是 RGB，而是 latent feature。

对目标视角中的某个像素 `k`，其 recalled latent 为：

```text
X̂Mi(k) = Σj∈N(k) Tj wj xj
```

其中：

- `N(k)`：投影到该像素的高斯集合；
- `wj`：该高斯对该像素的 splatting 权重；
- `Tj`：前方高斯累积透射率；
- `xj`：该高斯存储的 latent feature。

这样可得到一个与目标视角对齐的 dense latent map，直接作为下一帧生成条件。

#### 动态场景扩展

对于动态场景，论文进一步引入 4D Gaussian 表示：

- 高斯不仅有 3D 空间属性；
- 还包含时间相关的旋转、尺度等属性；
- 召回时根据目标时间戳切片得到动态内容。

### 2.3 Dynamic Deviation Archive：动态偏差档案

这是论文的第二个核心贡献。

为了缓解训练—推理不匹配，作者提出在训练中人为构造“被误差污染的历史 latent memory”，让模型学习从污染 memory 中恢复。

#### 基本思想

训练时不再只使用 clean history，而是对历史 latent 注入模拟的自回归偏差：

```text
X̃i = Xi + Ii Di
```

其中：

- `Xi`：干净历史 latent；
- `Di`：从偏差档案中采样的偏差；
- `Ii`：Bernoulli 随机变量，控制是否注入偏差；
- `p`：偏差注入概率，文中设置为 0.9。

最终，模型仍然被监督生成干净目标 `Xn+1`。  
也就是说，训练目标是：

> 输入污染历史，输出干净未来。

这迫使模型学习内部纠错能力。

---

### 2.4 一步偏差合成 One-Step Deviation Synthesis

直接完整模拟自回归 diffusion rollout 成本极高，因此作者使用一步近似生成偏差。

给定带噪 latent：

```text
Xti = tXi + (1 - t)ε
```

DiT 预测 velocity：

```text
V̂ti = DiT(Xti, yi-1, t; θ)
```

然后用一步 Euler 近似得到终点 latent：

```text
X̄i = Xti + (1 - t)V̂ti
```

偏差定义为：

```text
Di = X̄i - Xi
```

这个偏差反映了当前模型在该噪声时间步、该自回归阶段下可能产生的生成误差。

---

### 2.5 Archive 的组织方式

Dynamic Deviation Archive 按两个维度组织：

1. **自回归阶段 index `i`**；
2. **去噪时间戳 `t`**。

形式上：

```text
A = {Ai}
Ai = {Ati}
Ati = {Di,t,z}
```

其中：

- `Ai`：第 `i` 帧对应的偏差档案；
- `Ati`：第 `i` 帧、时间步 `t` 对应的偏差集合；
- `z`：该 cell 中的偏差样本索引；
- 每个 cell 容量为 `Z`，文中设置为 500。

这样的结构可以捕捉：

- 早期 rollout 中较轻微的外观 / 位姿误差；
- 后期 rollout 中累积的颜色漂移、结构畸变和语义不一致；
- 不同 diffusion 时间步下的误差模式。

#### 多样性保持更新

当某个 archive cell 已满时，作者不随机替换，而是替换掉与新偏差最相似的旧偏差：

```text
z* = arg minz ||Di - Di,t,z||2
```

这样可以减少冗余偏差，保留罕见但重要的错误模式。

---

### 2.6 训练目标

训练时，模型使用 flow matching objective。目标是让 DiT 在偏差污染的历史条件下，仍然预测干净未来 latent 的 velocity。

整体目标可以概括为：

```text
L = E || DiT(Xtn+1, yn, t; θ) - (Xn+1 - ε) ||²
```

其中：

- `Xtn+1` 是目标 latent 加噪后的状态；
- `yn` 是由 clean anchor、污染前驱 latent、污染 memory 召回结果和控制信号拼接而成的条件；
- `Xn+1` 是干净未来 latent；
- `ε` 是噪声。

- 用于测试泛化能力和 3D 一致性。

#### 3. OmniWorldGame

- 面向长时程交互式 world simulation 的数据集；
- 包含动态场景、游戏式环境和复杂对象运动；
- 用于评估动态内容下的时序一致性和动作控制能力。

## 4. 资源与算力

论文在附录中给出了较详细的实现和算力信息。

### 4.1 硬件

实验使用：

- **8 张 NVIDIA H200 GPU**
- 每张 GPU 显存：**144 GB**

### 4.2 训练时间

训练分为两个阶段：

#### 阶段一：Latent Gaussian Memory 构建 / 重建模块训练

- 冻结 backbone；
- 优化 Gaussian head 和 global head；
- 训练时间约：**31 小时**。

#### 阶段二：视频扩散模型 / Dreamer 训练

- 基于 **Wan2.1-I2V-14B-480P**；
- 使用 LoRA 微调；
- 训练时间约：**31 小时**。

因此总训练时间约为：

- **8×H200 环境下约 62 小时**，不含预实验和调参成本。

### 4.3 推理时间

文中报告：

- 生成 81 帧视频约需 **14 分钟**；
- 生成 301 帧视频约需 **55.3 分钟**；
- 单帧平均推理时间约：
  - 81 帧：10.4 秒 / 帧；
  - 301 帧：11.0 秒 / 帧。

### 4.4 Memory 和 Archive 开销

#### Latent Gaussian Memory

对于 81 帧、512×288 分辨率：

- GPU memory：约 **100 MB**；
- 平均 runtime：约 **76.34 ms / frame**。

相比标准 per-pixel 3D-aware memory：

- 标准方法约 **500 MB GPU memory**；
- Robust Dreamer 降低约 **81% memory**；
- runtime 降低约 **18%**。

#### Dynamic Deviation Archive

- 存储在 CPU；
- CPU memory 约 **22 GB**；
- 每帧从 CPU 传到 GPU 的 deviation 约 **0.6 MB**；
- 一步偏差近似耗时约 **0.266 ms / step**；
- archive 维护最多增加约 **350 ms / training iteration**；
- 相对于单次训练迭代约 11.1 秒，额外开销约 **3.2%**。

## 6. 主要结论与发现

论文的主要结论如下：

1. **在 latent 空间维护 3D memory 比 RGB memory 更适合长时程自回归生成。**  
   Latent Gaussian Memory 避免了反复 VAE decode / encode 引入的信息损失，显著减少颜色漂移和结构退化。

2. **训练时显式暴露污染 memory 能显著提升推理稳定性。**  
   Deviation Learning 让模型学会在历史条件不完美的情况下恢复干净未来帧。

3. **自回归误差不是简单随机噪声。**  
   实验显示 Gaussian noise、blur、color shift 等简单扰动均不如模型合成 deviation，说明真实 rollout 误差具有结构性、阶段性和模型相关性。

4. **Robust Dreamer 在长视频生成中优势更加明显。**  
   在 ScanNet 300 帧长期生成中，相比 baseline，Robust Dreamer 在 PSNR、SSIM、LPIPS 和 FID 上均表现最好。

5. **方法可扩展到动态场景。**  
   在 OmniWorldGame 上，Robust Dreamer 也优于其他方法，说明其 4D Gaussian memory 和 deviation-aware training 对动态场景有一定适应能力。

6. **额外 memory 开销可控。**  
   通过 2D 下采样和 voxel pruning，Latent Gaussian Memory 的 GPU memory 和 runtime 相对较低。

## 7. 优点与亮点

### 7.1 方法设计上的亮点

- **直接在 diffusion latent 空间构建 3D memory**  
  避免传统 RGB memory 的反复 VAE 转换损耗。

- **将 Gaussian Splatting 与视频扩散 latent 结合**  
  使 memory recall 具备视角对齐、几何感知和 dense conditioning 能力。

- **区分 memory 设计和 memory 使用鲁棒性**  
  不仅提出更好的 memory 表示，还专门解决如何在污染 memory 下训练生成器。

- **Dynamic Deviation Archive 设计合理**  
  按自回归阶段和 denoising timestamp 组织偏差，较好建模误差的非平稳性。

- **一步偏差近似降低训练成本**  
  避免完整 diffusion rollout 带来的高昂计算开销。

- **多样性保持的 archive 更新策略**  
  替换最相似偏差，减少冗余，保留罕见错误模式。

---

### 7.2 实验设计上的亮点

- 覆盖 ScanNet、DL3DV、OmniWorldGame 三类场景；
- 同时评估短期和长期生成；
- 使用多个指标综合评估；
- 与多种类型 baseline 对比；
- 消融实验直指两个核心问题；
- 提供了 runtime、GPU memory、CPU memory 和 scaling 分析；
- 包含域外定性结果，展示一定泛化能力。

该方法对以下方向具有启发：

- 交互式世界模型；
- embodied AI 中的模拟环境生成；
- 基于动作的场景探索；

特别是文中提出的“latent memory + deviation-aware training”范式，可能对其他自回归生成任务也有参考价值。

## 8. 不足与局限

### 8.1 依赖 3D 重建质量

Latent Gaussian Memory 的基础是可用的 3D / 4D Gaussian reconstruction。如果几何预测失败，memory 也会出错。

容易失败的场景包括：

- 严重遮挡；
- 反光 / 透明表面；
- 快速运动；
- 纹理贫乏区域；
- 视角覆盖不足；
- 非刚体大形变；
- 拓扑变化场景，如水、火、烟雾等。

论文也承认，当可恢复几何本身不可用时，该方法仍可能失败。

---

### 8.2 对极端动态和非刚体场景支持有限

虽然论文引入 4D Gaussian memory，但其动态建模能力仍有限。作者明确指出：

- 当前主要覆盖静态场景和相对结构化的动态场景；
- 对强非刚体运动没有专门设计；
- 动态 deviation 与真实场景运动可能难以区分；
- 需要更多可靠 pose 标注和动态数据集支持。

---

### 8.3 训练和推理成本较高

尽管 memory 模块本身开销较小，但整体系统依赖 14B 级视频扩散模型：

- 训练需要 8 张 H200；
- 两阶段训练约 62 小时；
- 生成 81 帧仍需约 14 分钟；
- 单帧约 10 秒级别。

因此，虽然论文面向交互式 world simulation，但当前实现还难以达到真正实时交互。

---

### 8.4 代码未开源，复现仍有难度

论文 checklist 中明确说明：

- 当前没有提供开源代码；
- 没有完整 command-level 复现实验说明；
- 使用了多个大模型组件和数据集；
- 训练资源要求较高。

这会影响外部研究者复现和验证结果。

---

### 8.5 统计显著性不足

论文未报告：

- 多随机种子结果；
- error bars；
- confidence intervals；
- statistical significance tests。

因此，虽然结果数值优势明显，但仍缺少对实验稳定性的系统量化。

---

### 8.6 baseline 公平性仍需进一步说明

论文比较了多个强 baseline，但仍有若干不确定点：

- baseline 是否均在同等数据和同等分辨率下训练；
- baseline 是否拥有相同规模 backbone；
- 是否对每个 baseline 做了充分调参；
- 是否所有方法都适配逐帧动作控制设定。

这些因素可能影响比较的严格公平性。

---

### 8.7 Archive 偏差分布仍是近似

Dynamic Deviation Archive 使用一步近似来模拟 rollout-induced deviation。论文指出该近似约达到 ODE-based simulation 误差幅度的 58.7%。

这意味着：

- 一步偏差并不能完全代表真实多步采样误差；
- 极端错误或 hallucination 仍可能超出 archive 分布；
- 如果推理时误差超出训练中 archive 覆盖范围，模型仍可能漂移。

---

## 总体评价

Robust Dreamer 是一篇针对长时程动作控制视频生成中“记忆漂移”问题的系统性工作。它的贡献不只是提出一个新的 3D memory，而是同时从两个层面解决问题：

1. **如何设计 memory**：使用 Latent Gaussian Memory 避免 Latent–RGB Cycling；
2. **如何鲁棒使用 memory**：使用 Dynamic Deviation Archive 缓解训练—推理差距。

实验表明，该方法在多个数据集、多个指标和长时程 rollout 中都取得显著提升，尤其在 300 帧级别生成中展现出较强优势。其主要不足在于计算资源要求高、复现门槛高、统计显著性不足，以及对极端复杂动态场景的适应能力仍有限。

整体而言，这篇论文对交互式世界模型和 3D-aware 自回归视频生成具有较高参考价值。
