## 摘要
近期的无感知端到端（E2E）自动驾驶方法通过将密集的图像块令牌压缩为紧凑的场景令牌，以供下游轨迹生成与评分使用，从而绕过显式感知输出。尽管这些场景令牌为规划器形成了紧凑的视觉瓶颈，但它们仅从规划目标中获得监督，对所编码的视觉信息约束有限。为解决这一局限，我们提出神经令牌重建（Neural Token Reconstruction, NTR），这是一种表示学习框架，用于直接约束无感知驾驶中的紧凑场景令牌瓶颈。NTR 引入了一种自蒸馏的掩蔽潜在重建目标，仅使用紧凑场景令牌作为重建记忆来重建被掩蔽的图像块级潜在特征。这迫使重建梯度完全通过场景令牌瓶颈传播，鼓励场景令牌为规划保留更丰富且冗余更少的视觉表示。我们进一步引入由基础模型标注导出的语义先验，作为一种弱语义接口，使重建目标偏向与驾驶相关的结构，同时不引入显式感知头。所有辅助重建组件在推理时均被移除，使部署的规划器保持不变。NTR 在三个公开自动驾驶基准上取得了最先进的性能，包括在 Waymo E2E 上达到 8.0461 RFS，以及在 NavSim1&2 上达到 94.1 PDMS / 90.9 EPDMS。学习到的场景令牌表现出更低的两两冗余和更高的有效秩，表明有效的瓶颈监督能够同时提升紧凑视觉表示学习与规划性能。

## Abstract
Recent perception-free end-to-end (E2E) autonomous driving methods bypass explicit perception outputs by compressing dense image patch tokens into compact scene tokens for downstream trajectory generation and scoring. While these scene tokens form a compact visual bottleneck for the planner, they receive supervision solely from the planning objective, providing limited constraints on the encoded visual information. To address this limitation, we introduce Neural Token Reconstruction (NTR), a representation learning framework to directly constrain the compact scene-token bottleneck in perception-free driving. NTR introduces a self-distillation masked latent reconstruction objective that reconstructs masked patch-level latent features using only compact scene tokens as reconstruction memory. This forces reconstruction gradients to pass exclusively through the scene-token bottleneck, encouraging scene tokens to preserve richer and less redundant visual representations for planning. We further introduce semantic priors derived from foundation-model annotations as a weak semantic interface biasing reconstruction targets toward driving-related structures without introducing explicit perception heads. All auxiliary reconstruction components are removed at inference time, leaving the deployed planner unchanged. NTR achieves state-of-the-art performance on three public autonomous driving benchmarks, including 8.0461 RFS on Waymo E2E and 94.1 PDMS / 90.9 EPDMS on NavSim1&2. The learned scene tokens exhibit lower pairwise redundancy and higher effective rank, indicating that effective bottleneck supervision improves both compact visual representation learning and planning performance.

## 1. 核心问题与整体含义：研究动机与背景

### 1.1 背景：无感知端到端自动驾驶中的“场景 token 瓶颈”

本文关注的是**无感知端到端自动驾驶**（perception-free end-to-end autonomous driving）中的视觉表示学习问题。

传统自动驾驶系统通常包含显式感知模块，例如：

- 目标检测；
- 语义 / 实例分割；
- 车道线检测；
- 占用栅格预测；
- 轨迹预测；
- BEV 表示构建等。

而近年来的无感知端到端方法尝试绕过这些显式中间表示，直接从多摄像头图像输入生成未来轨迹或驾驶动作。这类方法的优势是系统更简洁，减少人工设计模块和密集标注需求；但问题在于，模型必须自己学会哪些视觉信息对规划重要。

近期较强的无感知端到端方法，例如 DrivoR 一类方法，会将密集的 ViT patch token 压缩成少量紧凑的 **scene tokens**，再将这些 scene tokens 交给下游规划器进行：

- 轨迹候选生成；
- 轨迹评分；
- 最终轨迹选择。

因此，scene tokens 成为视觉 backbone 与规划器之间的唯一视觉接口，也就是论文所说的 **scene-token bottleneck**。

### 1.2 核心问题：scene token 只受规划损失监督，视觉约束不足

现有方法通常只用最终规划目标监督 scene tokens，例如轨迹回归损失、轨迹评分损失等。这带来一个关键问题：

> 少量 scene tokens 需要压缩大量图像 patch 信息，但它们没有被直接约束去保留哪些视觉信息，只能从稀疏的规划损失中间接学习。

这会导致以下现象：

- 不同 scene tokens 学到高度相似的内容；
- token 之间冗余较高；
- 注意力区域重叠严重；
- 对道路结构、交通参与者、交通灯等细粒度驾驶相关信息保留不足；
- 紧凑表示没有被充分利用。

论文认为，这不是单纯靠增加 token 数量或调整 token 结构就能解决的问题，而是一个**瓶颈表示学习问题**。

### 1.3 整体含义

本文的核心贡献在于提出：

> 在不引入显式感知头、不改变推理阶段规划器结构的前提下，直接监督 scene-token bottleneck，使其保留更丰富、低冗余、对驾驶规划更有用的视觉信息。

为此，论文提出 **Neural Token Reconstruction，NTR**，通过训练时的掩码潜特征重建任务，让 scene tokens 被迫承担重建局部视觉特征的责任，从而改善压缩后的视觉表示质量。

## 2. 方法论：NTR 的核心思想与关键技术

## 2.1 基础架构：紧凑 scene-token 规划器

论文以 DrivoR 风格的无感知端到端规划器作为基础架构。

输入包括：

- 多摄像头图像：  
  \[
  I = \{I_v\}_{v=1}^{V}
  \]
- 当前自车状态：  
  \[
  e = [v, a, \psi, c]
  \]
  其中包括速度、加速度、航向角和高层导航指令。

每个摄像头图像被切分为 patch，并由 ViT backbone 编码为 patch tokens：

\[
P_v \in \mathbb{R}^{H_p W_p \times D}
\]

同时，每个视角引入少量可学习的 scene tokens：

\[
S_v \in \mathbb{R}^{N_s \times D}
\]

其中：

\[
N_s \ll H_p W_p
\]

scene tokens 与 patch tokens 一起输入 ViT backbone：

\[
\tilde{S}_v, \tilde{P}_v = f_{\theta}(S_v, P_v)
\]

但下游规划器只使用压缩后的 scene tokens：

\[
\tilde{S} = \text{Concat}_{v=1}^{V}(\tilde{S}_v)
\]

密集 patch tokens 会被丢弃。因此，该过程形成了从密集 patch token 到少量 scene token 的压缩瓶颈：

\[
P_v \rightarrow \tilde{S}_v
\]

规划器根据 scene tokens 和自车状态输出多条候选轨迹及其评分：

\[
\{(\hat{\tau}_k, \hat{s}_k)\}_{k=1}^{N_q} = D_{\phi}(\tilde{S}, e)
\]

NTR 不改变这个规划器结构，而是在训练阶段额外添加重建监督。

---

## 2.2 NTR 的核心思想：只用 scene tokens 重建被遮挡 patch 的潜特征

NTR 的关键设计是：

> 在训练时遮挡部分图像 patch，让在线分支通过压缩后的 scene tokens 去重建这些被遮挡位置的教师网络潜特征。

也就是说，重建解码器不能直接访问密集 patch tokens，只能访问 scene tokens。这样一来，如果要完成重建任务，scene tokens 就必须保留足够丰富的局部视觉信息。

### 在线分支

在线 encoder 接收被 mask 的 patch tokens 和 scene tokens：

\[
\tilde{S}_{v,m}, \tilde{P}_{v,m} = f_{\theta}(S_v, P_{v,m})
\]

其中：

- \(P_{v,m}\) 是被 mask 后的 patch tokens；
- \(\tilde{S}_{v,m}\) 是压缩后的 scene tokens；
- \(\tilde{P}_{v,m}\) 不作为重建记忆使用。

### 教师分支

教师网络是在线网络的 EMA 版本，输入未遮挡的干净图像 patch tokens：

\[
\tilde{S}_{v,*}, \bar{Z}_v = f_{\xi}(S_v, P_v)
\]

教师 patch 输出作为 stop-gradient 的重建目标：

\[
Z_{v,*} = \text{sg}(\bar{Z}_v)
\]

教师参数通过 EMA 更新：

\[
\xi \leftarrow \mu \xi + (1-\mu)\theta
\]

其中论文默认：

- EMA momentum：\(\mu = 0.999\)

### 重建位置选择

对每个视角，选择一部分 patch 作为重建目标：

\[
\Omega^{v}_{rec} \subset \{1, ..., H_pW_p\}
\]

重建比例为：

\[
\rho_{rec}
\]

论文默认：

- \(\rho_{rec} = 0.3\)

### 重建解码器

对于被选中的 patch 位置，构造位置查询：

\[
Q^{pos}_{\Omega^v_{rec}} = q_{rec} + E_{pos}[\Omega^v_{rec}]
\]

其中：

- \(q_{rec}\) 是共享可学习重建 query；
- \(E_{pos}\) 是 patch 位置编码。

轻量 Transformer 解码器 \(g_{\omega}\) 使用这些 query，并且只 cross-attend 到 scene tokens：

\[
\hat{Z}^{v}_{\Omega^v_{rec}} = g_{\omega}(Q^{pos}_{\Omega^v_{rec}}, \tilde{S}_{v,m})
\]

重要的是，解码器不能访问 dense online patch outputs，只能访问 scene tokens。因此重建损失的梯度必须经过 scene-token bottleneck：

\[
\frac{\partial L_{rec}}{\partial \theta}
=
\sum_v
\frac{\partial L_{rec}}{\partial \tilde{S}_{v,m}}
\frac{\partial \tilde{S}_{v,m}}{\partial \theta}
\]

这正是 NTR 的核心机制：  
**通过重建任务直接约束 scene tokens，而不是只依赖规划损失间接监督。**

## 2.3 语义先验：用基础模型引导重建目标分配

随机选择重建 patch 虽然能提供稠密监督，但驾驶场景中很多区域并不关键，例如：

- 天空；
- 远处背景；
- 路边纹理；
- 无关建筑物；
- 静态杂乱区域。

而对规划更关键的是：

- 车辆；
- 行人；
- 骑行者；
- 可行驶区域；
- 道路边界；
- 交通灯；
- 交通标志等。

因此，论文使用 foundation segmentation model，文中提到 SAM3，基于文本 prompt 生成语义 mask，作为重建位置选择的弱语义先验。

给定类别 prompt 集合：

\[
P = \{p_k\}_{k=1}^{K}
\]

基础分割模型为每个摄像头视角生成语义 mask：

\[
U \in \{0,1\}^{V \times K \times H \times W}
\]

然后池化到 patch 网格：

\[
G = \text{Pool}(U)
\]

语义先验进一步被转成重建得分图：

\[
A(v,i,j) =
\sum_{k=1}^{K} \alpha_k G(v,k,i,j) + \tau \epsilon_{vij}
\]

其中：

- \(\alpha_k\) 是不同语义类别权重；
- \(\tau\) 控制随机探索强度；
- \(\epsilon_{vij} \sim \mathcal{N}(0,1)\)。

之后选择得分最高的 patch 作为重建目标：

\[
\Omega^v_{rec} = \text{Top-m}(A(v,:,:))
\]

其中：

\[
m = \lfloor \rho_{rec} H_p W_p \rfloor
\]

论文默认：

- \(\tau = 0.4\)

这使得重建任务更关注驾驶相关结构，而不是平均地浪费在背景区域上。

## 2.4 总训练目标

整体训练目标为：

\[
L = L_{plan} + \lambda_{rec} L_{rec}
\]

其中：

- \(L_{plan}\)：原始规划损失，包括轨迹监督和轨迹评分监督；
- \(L_{rec}\)：NTR 的潜特征重建损失；
- \(\lambda_{rec}\)：重建损失权重。

论文设置：

- Waymo 上 \(\lambda_{rec} = 1.0\)；
- NavSim 上 \(\lambda_{rec} = 0.5\)。

重建损失定义在教师 latent feature 空间中：

\[
L_{rec}
=
\frac{1}{|\Omega_{rec}|D}
\sum_{(v,i)\in\Omega_{rec}}
\left\|
\hat{z}_{vi} - z^{*}_{v,i}
\right\|_1
\]

其中：

- \(\hat{z}_{vi}\)：重建出的 latent feature；
- \(z^{*}_{v,i}\)：EMA teacher 输出的 stop-gradient latent target；
- \(\Omega_{rec}\)：所有视角的重建 patch 位置集合。

### 推理阶段

推理时会移除：

- 重建解码器；
- EMA teacher；
- 语义先验模块；
- foundation segmentation model 相关组件。

因此，部署阶段的规划器结构和计算开销保持不变。

## 6. 主要结论与发现

论文的主要结论可以概括为：

- 无感知端到端驾驶中的 scene tokens 是一个关键视觉瓶颈；
- 仅依靠规划损失监督 scene tokens 会导致 token 表示冗余和信息利用不足；
- 通过 masked latent reconstruction 直接监督 scene-token bottleneck，可以显著提升规划性能；
- 使用 EMA teacher 比 frozen target 更有效；
- 使用语义先验选择重建区域比随机重建更有利于驾驶相关信息学习；
- NTR 可以降低 scene token 两两相似度，提高 effective rank；
- 表示多样性和低冗余与规划性能提升存在一致关系；
- 所有辅助模块只在训练时使用，推理阶段不会增加额外计算成本或改变部署架构。

从结果上看，NTR 在多个公开 benchmark 上取得了领先表现：

- Waymo E2E：最高 8.0461 RFS；
- NavSim V1：94.1 PDMS；
- NavSim V2：90.9 EPDMS。

## 7. 优点与亮点

### 7.1 问题定位清晰

论文没有简单追求更大的模型或更多 token，而是指出了无感知端到端驾驶中的一个关键瓶颈：

> 压缩后的 scene tokens 缺乏直接监督，导致视觉信息保留不足。

这个问题具有较强的普遍性，尤其适用于采用 token compression 或 register token 的端到端驾驶模型。

### 7.2 方法设计简洁有效

NTR 的设计较为优雅：

- 不引入显式感知头；
- 不要求 dense perception labels；
- 不改变原始规划器；
- 推理阶段零额外开销；
- 只通过训练时辅助目标改善瓶颈表示。

### 7.3 重建路径设计合理

论文特别强调重建解码器只能访问 scene tokens，而不能访问 dense patch tokens。这一点很关键，因为它保证了重建监督真正作用在瓶颈表示上，而不是被 dense features 绕过。

### 7.4 自蒸馏目标稳定

使用 EMA teacher 生成 stop-gradient latent targets，避免直接重建像素带来的低层纹理偏置，也比固定 teacher 更能适应端到端规划训练过程。

### 7.5 语义先验设计具有驾驶任务针对性

利用 foundation segmentation model 只在训练时引导重建目标分配，让模型更关注：

- agents；
- drivable area；
- traffic control elements。

这比随机重建更符合自动驾驶任务需求。

### 7.6 实验覆盖较广

论文覆盖：

- Waymo E2E；
- NavSim V1；
- NavSim V2；
- 消融实验；
- token 诊断；
- 定性分析；
- 声称包含真实车辆部署验证。

整体实验体系较完整。

## 8. 不足与局限

### 8.1 依赖 foundation-model 语义先验

NTR 的最佳版本依赖 SAM3 等 foundation segmentation model 生成语义先验。虽然推理阶段不需要这些模块，但训练阶段仍存在问题：

- 语义 mask 质量可能受域偏移影响；
- rare scenario 中 foundation model 可能标注不稳定；
- 不同天气、光照、国家道路环境下泛化能力未充分验证；
- 语义类别权重 \(\alpha_k\) 可能需要人工设计或调参。

### 8.2 训练和预处理成本增加

虽然推理阶段零额外成本，但训练阶段增加了：

- EMA teacher forward；
- reconstruction decoder；
- mask sampling；
- semantic prior 生成；
- foundation segmentation preprocessing。

论文没有详细报告这些额外成本，因此实际训练资源开销不够透明。

### 8.3 主要改善 scene-token bottleneck，而非 backbone 本身

论文也承认，NTR 主要约束从 dense patch tokens 到 compact scene tokens 的压缩过程，而不是改善底层视觉 backbone。它与 MAE、iBOT、JEPA 等 backbone-level self-supervised pretraining 是互补关系。

这意味着：

- 如果 backbone dense features 本身质量不足，NTR 只能改善压缩后的保留方式；
- 对视觉表征源头的提升有限。

### 8.4 与其他正则化方法比较不足

论文主要与规划方法对比，但对以下方法缺少系统比较：

- token diversity regularization；
- contrastive token decorrelation；
- attention diversity loss；
- information bottleneck regularization；
- alternative masked modeling at bottleneck；
- reconstruction 到 BEV / semantic latent 的方法。

因此还不能完全证明 NTR 是最优的 bottleneck supervision 方案。

### 8.5 真实车部署细节不足

摘要和正文提到在私有大规模数据集和真实车辆规划栈中验证，但给定文本中没有详细说明：

- 私有数据规模；
- 场景分布；
- 实车测试里程；
- 安全接管率；
- 闭环在线表现；
- 与原系统的对照实验；
- 失败案例统计。

因此这部分更像是补充性声明，公开可验证性有限。

### 8.6 Leaderboard 对比存在潜在公平性问题

Waymo 表中不同方法可能存在：

- 是否使用 ensemble 的差异；
- 模型规模差异；
- 是否使用额外数据；
- 是否使用 VLM / world model；
- 训练策略和后处理不同。

NTR Single 已经很强，但 NTR Ensemble 与其他 single/ensemble 方法之间的严格公平性仍需谨慎解读。

---

## 总结

NTR 是一篇从表示学习角度切入无感知端到端驾驶的工作。它指出紧凑 scene tokens 是规划器的信息瓶颈，而现有方法仅靠规划损失监督这一瓶颈，容易导致 token 冗余和视觉信息丢失。为此，论文提出通过自蒸馏 masked latent reconstruction 直接监督 scene tokens，并利用 foundation-model 语义先验将重建目标集中到驾驶相关区域。该方法训练时增加辅助监督，推理时不改变模型结构，也不增加部署开销。

实验显示，NTR 在 Waymo E2E、NavSim V1 和 NavSim V2 上均取得领先结果，同时显著降低 scene token 相似度、提高 effective rank，说明其确实改善了紧凑视觉表示的质量。整体而言，论文问题定义明确、方法设计简洁、实验结果有说服力，是端到端驾驶中 token bottleneck 表示学习方向的一项有价值工作。但其对 foundation model 标注质量、训练成本、真实部署细节和替代正则化方法的比较仍有进一步完善空间。
