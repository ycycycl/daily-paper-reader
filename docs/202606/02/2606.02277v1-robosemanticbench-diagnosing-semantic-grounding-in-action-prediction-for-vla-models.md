## 摘要
视觉-语言-动作（VLA）模型建立在这样一个前提之上：来自预训练语言或视觉-语言骨干网络的语义理解应当引导机器人动作预测。然而，机器人微调通常被优化为对任务特定动作分布的模仿，并且许多评测可以通过视觉捷径或指令-动作捷径来解决。我们提出 RoboSemanticBench（RSB），这是一个用于诊断动作预测中语义落地能力的具身基准：即后训练的 VLA 模型是否能够利用复杂指令语义来选择并操控正确的物理目标。在每个回合中，机器人会收到一道多选数学题或通识知识题，观察候选答案积木，并必须抓取与正确答案对应的积木。RSB 覆盖受控算术、小学阶段数学理解，以及常识或事实理解，并包含四选项和十选项测试套件。在具有代表性的 VLA 模型上，我们发现，在控制抓取成功率之后，许多策略虽然学会了抓取候选积木，但选择语义上正确积木的比例接近随机甚至低于随机水平，这揭示了骨干网络层面的语义能力与动作预测之间持续存在的差距。

## Abstract
Vision-language-action (VLA) models are built on the premise that semantic understanding from pretrained language or vision-language backbones should guide robot action prediction. Yet robot fine-tuning is optimized as imitation over task-specific action distributions, and many evaluations can be solved through visual or instruction-action shortcuts. We introduce RoboSemanticBench (RSB), an embodied benchmark for diagnosing semantic grounding in action prediction: whether post-trained VLA models can use complex instruction semantics to select and manipulate the correct physical target. In each episode, a robot receives a multiple-choice math or general-knowledge question, observes candidate answer blocks, and must grasp the block corresponding to the correct answer. RSB covers controlled arithmetic, grade-school mathematical understanding, and commonsense or factual understanding under four-choice and ten-choice suites. Across representative VLA models, we find that many policies learn to grasp candidate blocks but select the semantically correct block at near-random or below-random rates after controlling for grasp success, revealing a persistent gap between backbone-level semantic competence and action prediction.

    - 指令-动作统计相关性；

    - 模型是否具备低层控制能力；

## 2. 方法论：核心思想、关键技术细节与指标

### 2.1 核心思想

- RoboSemanticBench 将传统的多选语义问题转化为机器人具身操作任务。
- 每个 episode 中：
  - 机器人接收一道多选问题；
  - 场景中摆放若干候选答案方块；
  - 每个方块对应一个选项；
  - 机器人需要抓取“正确答案对应的方块”，并放入灰色答案区域。
- 操作原语保持简单且固定：  
  **pick the correct option block and place it in the gray answer zone**。
- 难度主要来自语义理解和语义到动作目标的绑定，而不是复杂运动控制。

### 2.2 任务形式化

每个 episode 包含：

- 问题：\( q \)
- 候选选项集合：  
  \[
  O = \{o_1, ..., o_N\}
  \]
- 可见答案方块集合：  
  \[
  B = \{b_1, ..., b_N\}
  \]
- 选项到方块的映射：  
  \[
  m: O \rightarrow B
  \]

模型需要：

1. 理解问题 \( q \)；
2. 判断正确答案属于哪个选项；
3. 根据选项-方块映射找到对应物理方块；
4. 生成动作并抓取正确方块；
5. 将其放入答案区域。

### 2.3 三类语义任务

RSB 包含三个语义子集：

- **RSB-Math**
  - 程序化生成的简单算术题；
  - 包括两位数加法、两位数减法、一位数乘两位数；
  - 干扰项通常接近正确答案，避免模型依赖表面模式。

- **RSB-HardMath**
  - 基于 GSM8K 的小学数学应用题；
  - 需要从自然语言题干中抽取数量关系并进行多步推理；
  - 更强调组合式语言理解和数学推理。

- **RSB-General**
  - 包含常识、事实知识和 MMLU 风格多选题；
  - 涉及日常工具、地点、功能、常识知识等；
  - 用于测试非数学类语义理解。

### 2.4 四选与十选设置

- 每个语义子集都有两种选择规模：
  - **4-choice**：A, B, C, D；
  - **10-choice**：A, B, C, D, E, F, G, H, I, K。
- 十选设置跳过 J，以避免与 I 视觉混淆。
- 十选任务更难，因为随机猜中概率从 25% 降至 10%。

### 2.5 防止泄漏与捷径

- 指令模板只暴露：
  - 问题；
  - 选项；
  - 选项与方块的对应关系。
- 不直接暴露正确答案。
- 训练问题与测试问题不重合，避免模型记忆问答对。
- 方块布局和选项映射随机化，避免模型依赖固定位置、颜色或轨迹。

### 2.6 评价指标

论文提出三个核心指标：

#### 1. Task Success Rate, TSR

- 表示机器人是否抓取了语义上正确答案对应的方块。
- 这是最终任务成功率。

#### 2. Grasp Success Rate, GSR

- 表示机器人是否成功抓取了任意候选方块。
- 不要求该方块语义正确。
- 用于衡量低层运动控制和抓取能力。

#### 3. Normalized Semantic Grounding, nSG

用于剥离低层抓取能力，衡量“在已经抓到候选方块的前提下，是否抓对了语义目标”。

公式为：

\[
nSG = \frac{TSR/GSR - 1/N}{1 - 1/N}
\]

其中：

- \( N \) 是候选选项数量；
- \( TSR/GSR \) 表示在成功抓取候选方块条件下，抓中正确目标的概率；
- \( 1/N \) 是随机选择基线。

解释：

- \( nSG = 0 \)：等同随机选择；
- \( nSG > 0 \)：语义选择优于随机；
- \( nSG < 0 \)：语义选择劣于随机；
- \( nSG = 1 \)：所有成功抓取都抓对语义目标。

  - GSR 仍较高；

L = 0.9 L_{FM} + 0.1 L_{CoT}
\]

其中：

- \( L_{FM} \)：动作生成流匹配损失；
- \( L_{CoT} \)：CoT 文本生成 next-token 损失。

- 在机器人示范微调中加入 RoboVQA 的 VQA 监督。
- 目标是保留 VLM backbone 的语义能力。
- 训练目标：

\[
L_{cotrain} = L_{action} + 0.1 L_{VQA}
\]

结果显示这种 cotraining 反而降低了 TSR。

- 使用 GSR 与 TSR 分离低层抓取能力和语义目标选择；
- 引入 nSG 对抓取能力进行归一化，避免把运动控制失败误判为语义失败；
- 训练 / 测试问题不重合，减少记忆风险；
- 对方块代理进行了 Beyond Blocks 验证。

### 5.3 公平性与客观性

- 公平性较强：
  - 所有模型使用相同训练-测试协议；
  - 使用相同仿真环境和任务定义；
  - 尽量匹配训练预算；
  - 每个 suite 评测 500 episodes。
- 但也存在潜在限制：
  - 不同 VLA 模型原始预训练数据、架构和动作接口差异较大，完全公平很难保证；
  - 各模型使用官方默认设置，可能没有针对 RSB 进行充分超参调优；
  - 实验主要在仿真环境中完成，真实机器人泛化尚未验证。

## 6. 主要结论与发现

### 6.1 多数 VLA 模型语义目标选择接近随机

- 主实验显示，大多数 VLA 模型虽然能抓到候选方块，但抓到正确答案方块的比例较低。
- 许多模型的 nSG 接近 0 或为负，说明在成功抓取候选目标后，其语义选择能力接近随机甚至劣于随机。

### 6.2 GSR 高但 TSR 低，说明问题不只是控制失败

- 例如 π0、π0.5、GR00T N1.7、QwenGR00T 等模型在不少 suite 上 GSR 接近或超过 95%。
- 但这些模型的 TSR 明显低得多。
- 这说明它们学会了“抓一个候选物体”，但没有稳定学会“根据语义抓正确物体”。

### 6.3 π0.5 表现最好但仍不充分

- π0.5 在主评测中平均 TSR 最高，约为 21.8%。
- 其平均 nSG 也是唯一较明显为正的模型之一，约为 5.2%。
- 论文推测 π0.5 的 subtask annotations 或预训练信号可能帮助其将高层语义分解传递给动作模块。
- 但其绝对表现仍然不高，说明距离可靠语义落地仍有较大差距。

### 6.4 十选任务更能暴露语义 grounding 问题

- 在 10-choice suite 中，随机基线为 10%。
- 多数模型 TSR 低于或接近这一水平。
- 随着候选目标增多，依赖捷径或视觉先验更难奏效，语义落地缺陷更明显。

### 6.5 CoT 推理不能根本解决问题

- ReasoningVLA 通过显式生成 CoT 推理提升了 TSR：
  - QwenGR00T 平均 TSR：10.7%
  - ReasoningVLA 平均 TSR：16.0%
- 但提升有限，仍远未达到可靠水平。
- 错误分析显示，ReasoningVLA 很多时候 CoT 判断正确，但动作仍抓错目标。
- 这说明“语言空间会推理”不等于“动作路径会执行推理结果”。

### 6.6 VQA cotraining 反而降低表现

- QwenGR00T cotraining 后平均 TSR 从 10.7% 降至 8.2%。
- 说明简单加入语言或 VQA 辅助监督不一定能增强动作中的语义 grounding，甚至可能干扰动作学习。

## 7. 优点与亮点

- **问题定义清晰**  
  论文明确区分了：
  - 语义理解；
  - 低层抓取；
  - 语义到动作目标的绑定。  
  这比单纯报告任务成功率更有诊断价值。

- **Benchmark 设计简洁但有效**  
  通过固定操作原语、变化语义问题，使任务难点集中在语义 grounding，而不是复杂运动控制。

- **指标设计合理**  
  GSR / TSR / nSG 三个指标能够有效分解：
  - 是否会抓；
  - 是否抓对；
  - 抓对是否超过随机语义选择。

- **覆盖多种语义类型**  
  包含简单算术、GSM8K 数学应用题、MMLU 风格常识和事实知识，语义覆盖面较广。

- **多模型系统评测**  
  评估了 9 个代表性 VLA 模型，结果更有说服力。

- **有负结果分析**  
  论文不仅提出 benchmark，还尝试了 ReasoningVLA 和 VLA Cotrain 两种自然改进方向，并报告负面或有限结果，有助于明确未来研究难点。

- **错误分析有洞察力**  
  通过分析“抓取成功但任务失败”的 cases，论文证明主要错误来自目标选择，而非放置失败或抓取失败。

- **控制捷径较充分**  
  通过随机布局、随机选项映射、训练测试问题分离、十选设置等手段，尽量减少颜色、位置、模板和记忆捷径。

## 8. 不足与局限

- **主要在仿真中评测**
  - 论文使用物理仿真环境和 Aloha-AgileX 双臂设定；
  - 尚未充分验证真实机器人环境中的表现；
  - sim-to-real 下视觉噪声、物体遮挡、控制误差可能进一步影响结果。

- **任务形式仍较人工化**
  - 多选题 + 抓答案方块是一种高度抽象的具身任务；
  - 虽然适合诊断语义 grounding，但与真实家庭机器人任务仍有差距；
  - 真实任务往往涉及更长时序、更复杂约束和开放词汇目标。

- **方块 / 选项机制可能与实际机器人语义任务不同**
  - 虽然 Beyond Blocks 实验证明日常物体替换后趋势类似；
  - 但主 benchmark 仍主要依赖字母方块作为受控代理；
  - 对开放世界物体识别、属性理解、空间关系推理覆盖有限。

- **语义任务偏向 QA 式理解**
  - RSB 主要把语义理解转化为多选问答；
  - 对更自然的指令理解，例如含糊指令、用户偏好、长期目标、交互式澄清等覆盖不足。

- **不同模型训练公平性仍有限**
  - 虽然作者尽量匹配训练 steps × batch size；
  - 但不同模型的预训练语料、动作表示、网络规模和优化方式差异很大；
  - 有些模型可能需要专门超参调优才能发挥最佳性能。

- **未报告完整训练成本**
  - 论文说明使用 8 张 H100，但没有给出具体训练时长、GPU-hours 和评测耗时；
  - 难以评估复现实验的实际成本。

- **ReasoningVLA 的 CoT 数据依赖外部教师模型**
  - CoT 标注由 Gemini 3 Flash 蒸馏生成；
  - 标注质量、格式约束和教师模型偏差可能影响结果；
  - 该方案也增加了数据构造复杂度。

- **没有充分证明 backbone 原本语义能力是否保留**
  - 论文假设 Qwen3 等 backbone 在 GSM8K、MMLU 上具备较强能力；
  - 但没有系统报告微调前后 backbone 在纯文本 / VQA 任务上的能力变化；
  - 因此难以精确区分：
    - 语义能力遗忘；
    - 语义-动作接口失败；
    - 动作模块忽略语义表征。

---

## 总体评价

RoboSemanticBench 是一个针对 VLA 模型语义落地能力的高价值诊断 benchmark。它通过简单、受控的 pick-and-place 任务，把复杂语义理解转化为物理目标选择，从而揭示了当前 VLA 模型中一个关键问题：模型可能会抓东西，但并不真正知道应该根据指令抓哪个东西。论文的指标设计和实验分解较有说服力，尤其是 GSR-TSR gap 和 nSG 指标很好地区分了控制能力与语义 grounding 能力。其主要局限在于任务仍偏合成和仿真，且多选 QA 形式与真实开放机器人任务存在差距，但作为诊断工具具有明确价值。
