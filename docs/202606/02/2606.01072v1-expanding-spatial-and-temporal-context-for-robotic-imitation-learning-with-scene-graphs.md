## 摘要
模仿学习使机器人能够通过观察学习如何执行任务。然而，家庭和办公室等真实世界环境由于空间尺度较大，往往存在严重的部分可观测性。此外，许多任务涉及执行一系列子任务，要求自主机器人在较长的时间范围内进行推理。为应对这些挑战，我们提出在模仿学习中使用场景图作为一种显式且结构化的记忆机制。通过维护一个动态场景图来捕获以物体为中心的关系及其随时间的演化，我们的方法使智能体能够在任务执行过程中保留相关的历史上下文，从而基于逐步累积的场景信息进行高效推理。我们在模拟移动操作和真实世界桌面操作上的实验表明，我们的方法显著提升了策略性能，尤其是在需要长期推理以及在部分可观测条件下实现鲁棒泛化的场景中。

## Abstract
Imitation learning enables robots to learn how to execute tasks via observation. However, real-world environments like homes and offices are often severely partially observed due to their large spatial scales. In addition, many tasks involve executing a series of subtasks requiring autonomous robots to reason over extended time horizons. To address these challenges, we propose using scene graphs as an explicit and structured memory mechanism in imitation learning. By maintaining a dynamic scene graph that captures object-centric relationships and their evolution over time, our method allows the agent to retain relevant historical context during task execution to efficiently reason over incrementally accrued scene information. Our experiments on simulated mobile manipulation and real-world tabletop manipulation demonstrate that our approach substantially improves policy performance, particularly in settings that demand long-term reasoning and robust generalization under partial observability.

# 2. 方法论：核心思想、关键技术细节与算法流程

## 2.1 核心思想

- 作者提出在模仿学习策略中维护一个**动态任务相关场景图**。
- 场景图中的每个节点表示一个任务相关物体，节点包含：
  - **视觉嵌入**：表示物体外观语义；
  - **2D 边界框**：表示物体在当前图像中的位置；若物体当前不可见，则置零；
  - **3D 质心坐标**：表示物体在世界坐标系或机器人基座坐标系中的空间位置。
- 该场景图随着机器人执行任务不断更新，形成一种显式记忆，使策略可以利用过去观察到但当前不可见的物体信息。

## 2.2 问题建模

- 论文将任务视为部分可观测马尔可夫决策过程，即 POMDP。
- 每条专家示范轨迹表示为：

  \[
  \tau_i = [(I_0, a_0), ..., (I_T, a_T)]
  \]

  其中：
  - \(I_t\)：RGB-D 图像观测；
  - \(a_t\)：专家动作；
  - \(s_t\)：真实环境状态，但机器人不能完整观测到。

- 理论上，最优策略可能需要完整历史观测-动作序列，但这在长时序任务中难以直接学习。因此，作者用场景图压缩历史信息。

## 2.3 任务相关物体枚举

- 输入自然语言任务描述，例如：
  - “Heat up a cup of tea”
  - “Put pineapple in bowl”
  - “Clean table”
- 使用大语言模型，例如 GPT-4 / GPT-5，从任务描述中提取任务相关物体名称。
- 例如：
  - 加热茶任务中需要关注“microwave”“cup”；
  - 清理桌面任务中需要关注“fruit toys”“drawer”等。
- 论文补充材料中说明，作者使用简单 prompt：

  > Imagine you are a robot performing the task “任务名称”, what are the task-relevant objects?

- 在本文任务中，GPT-4 对相关实体识别达到 100% 准确率。

## 2.4 场景图节点表示

每个物体节点表示为：

\[
n_i = [e_i, b_i, c_i]
\]

其中：

- \(e_i\)：物体视觉嵌入，由 DINO-v2 特征在物体 mask 区域内平均池化得到；
- \(b_i\)：2D bounding box，表示当前图像中物体位置；若不可见则设为 \((0,0,0,0)\)；
- \(c_i\)：3D centroid，通过深度图、相机内参、相机外参和机器人里程计反投影得到。

此外，场景图还包含一个**场景级根节点**：

- 使用 DINO-v2 ViT 的 CLS token 表示当前全局图像上下文；
- 其边界框设为 \([0,1,0,1]\)；
- 3D 坐标设为 \([0,0,0]\)。

整体上，论文实现的是一个浅层两级树结构：

- 第一级：场景根节点；
- 第二级：任务相关物体节点。

虽然没有显式定义边类型，但物体之间的空间关系可以通过共享坐标系中的 3D 位置和 2D 图像位置隐式表达。

## 2.5 场景图构建与维护流程

场景图的维护主要包括以下步骤：

1. **初始检测**
   - 在 episode 的第一帧中，用 Grounding DINO 检测任务相关实体。
   - 结合 segmentation mask 初始化物体节点。

2. **物体特征提取**
   - 使用 DINO-v2 提取物体区域视觉特征；
   - 计算 2D bounding box；
   - 使用 RGB-D 深度信息和相机参数估计 3D 质心。

3. **跟踪已有节点**
   - 使用 XMem 等视频目标分割 / 跟踪方法维护物体身份。
   - 如果物体仍在视野中，更新其视觉特征、2D box 和 3D 坐标。
   - 如果物体离开视野：
     - 2D box 置零；
     - 3D 坐标保持不变，假设物体未发生移动，直到再次被观察到。

4. **发现新物体**
   - 每个时间步继续运行检测器以发现新出现的任务相关物体。
   - 将新检测 mask 与已有节点 mask 做重叠匹配。
   - 如果重叠低于阈值 \(\tau_m\)，则初始化为新节点并加入场景图。

## 2.6 场景图条件化扩散策略

- 作者将场景图编码为 token 序列，输入 Transformer encoder。
- 每个节点的最终表示为：

\[
E_i = [e_i, MLP(b_i), MLP(c_i)]
\]

- 场景根节点表示为：

\[
E_{scene} = [e_{scene}, MLP(b_{scene}), MLP(c_{scene})]
\]

- 完整场景图表示为：

\[
R_{sg} = [E_{scene}, E_1, ..., E_i, ...]
\]

- Transformer 对所有节点进行联合自注意力处理，使策略能够基于物体之间的隐式空间关系和当前全局上下文做决策。

## 2.7 扩散模仿学习目标

- 作者使用 Diffusion Policy 作为下游动作生成模型。
- 训练时，对专家动作序列加入噪声，并训练模型预测噪声：

\[
L(\theta) = \mathbb{E}_{a_0,k,\epsilon}
\left[
\|\epsilon - \epsilon_{\theta}(a_k, R_{sg}, k)\|^2
\right]
\]

- 测试时，从高斯噪声开始，通过 DDIM 采样逐步去噪，生成短时动作序列。
- 因此，策略输出不是单步动作，而是一个短 horizon 的动作 chunk。

## 3.4 对比方法

论文主要比较以下方法：

### RGB

- 标准 Diffusion Policy。
- 输入为机器人相机 RGB 图像和本体状态。
- 不使用显式物体记忆。

### Visible-Objects

- 使用当前视野中的物体中心表示。
- 只利用当前相机可见物体，不维护历史记忆。
- 相当于“无长期记忆的物体表示”。

### PTP

- 基于 Past-Token Prediction 的长上下文扩散策略思想。
- 增加历史帧输入，将历史帧从 1 增加到 10。
- 同时预测过去 10 步和未来 32 步动作。
- 该方法只在模拟实验中比较；真实实验中由于模拟结果显示其很难推进到后续阶段，因此未纳入真实机器人对比。

## 3.5 消融实验

论文做了两个关键消融：

### Ours w/o 3D

- 移除 3D 质心 \(c_i\)。
- 可见物体只用视觉嵌入和 2D box 表示；
- 不可见物体只保留视觉嵌入。
- 也就是说，系统知道某个物体曾经存在，但不知道它在空间中哪里。

### Ours w/o object locations

- 移除所有位置信息，包括 2D box 和 3D centroid。
- 每个物体仅由视觉嵌入表示。
- 用于验证空间信息对长时序任务的重要性。

- 所有实验总算力消耗。

因此，论文对算力资源的说明是**部分充分**的：真实机器人训练和推理硬件较清楚，但模拟控制器训练和总体计算预算不完整。

# 5. 实验数量与充分性

## 5.2 充分性评价

- **充分之处**：
  - 同时包含模拟与真实机器人实验，增强了方法可信度。
  - 任务设计针对性强，均围绕部分可观测和长时序依赖展开。
  - 对比了纯 RGB、当前可见物体表示和长上下文方法，能较好说明结构化记忆的作用。
  - 消融实验直接验证了 3D 位置和物体空间信息的重要性。
  - 报告了失败分析和运行时分析，不仅关注成功率，也讨论系统瓶颈。

- **不足之处**：
  - 实验仍然主要是作者自定义任务，没有使用统一公开 benchmark。
  - 没有报告精确数值表格，主要通过堆叠柱状图展示成功率；从文本中难以提取精确数值。
  - PTP 没有进入真实机器人实验对比，虽然作者解释其在模拟中表现较差，但从公平性角度看，真实实验缺少该长上下文基线。
  - 任务相关物体由 LLM 提取，但任务集较简单，无法充分证明在复杂自然语言指令下的鲁棒性。
  - 场景动态性有限，大多假设不可见物体不会移动，这在真实家庭环境中可能不成立。

## 5.3 客观性与公平性

- 总体来看，对比是比较合理的：
  - RGB 代表传统视觉策略；
  - Visible-Objects 代表无记忆的物体中心表示；
  - PTP 代表扩展历史上下文的方法；
  - 消融版本验证位置记忆的重要性。

- 但也存在潜在公平性问题：
  - 本方法使用了外部感知模块，如 LLM、GroundingDINO、DINO-v2、XMem，而 RGB baseline 主要端到端从图像学习，先验信息不同。
  - 如果 baseline 也引入强大的显式地图或 SLAM 记忆，比较结果可能更全面。
  - 没有与更强的 VLA 模型或开放词汇 3D scene graph planning 方法做直接对比。

# 6. 主要结论与发现

- **动态任务相关场景图显著提升长时序模仿学习性能**。
  - 在模拟移动操作和真实桌面操作中，本文方法相比 RGB、Visible-Objects 和 PTP 有更稳定的任务完成能力。

- **显式记忆比简单增加历史帧更有效**。
  - PTP 增加历史帧可以改善部分短期推理，但面对数百甚至上千步的长时序任务时仍然不足。
  - 原因在于，长历史图像并不等于有效记忆，策略仍需从冗余历史中挖掘关键信息。

- **物体中心表示本身有帮助，但不够**。
  - Visible-Objects 在一些任务中优于 RGB，说明物体中心感知确实提升操作精度。
  - 但它只能使用当前可见物体，无法处理物体离开视野后的任务依赖。

- **空间位置信息对后续子任务尤其关键**。
  - 消融实验显示，移除 3D centroid 或所有 object location 后，早期子任务影响不大，但后续子任务和最终成功率明显下降。
  - 说明长时序任务中，仅记住“物体存在”不够，还必须记住“物体在哪里”。

- **失败主要来自策略长时序决策，而非感知模块**。
  - 在模拟中，作者没有观察到感知或跟踪失败。
  - 在真实 64 个 episode 中，分割失败 4 次，主要由遮挡或边界模糊导致。
  - LLM 物体枚举和初始化后的跟踪没有观察到失败。
  - 大部分失败来自 imitation policy 在长时序阶段的决策错误。

# 7. 优点与亮点

## 7.1 方法设计亮点

- **结构化显式记忆**
  - 与直接输入长图像历史不同，场景图只保留任务相关物体信息，更紧凑、更可解释。

- **任务相关过滤**
  - 使用 LLM 从任务描述中提取相关物体，避免追踪整个场景中的所有物体，降低表示复杂度。

- **结合语义、2D 与 3D 信息**
  - 节点同时包含：
    - DINO-v2 视觉嵌入；
    - 当前图像中的 2D 位置；
    - 世界坐标系下的 3D 位置。
  - 这种设计兼顾语义识别、图像定位和空间记忆。

- **动态增量更新**
  - 场景图不是静态输入，而是随机器人运动和观测不断维护。
  - 即使物体离开视野，也保留其 3D 位置作为记忆。

- **易于接入现有模仿学习策略**
  - 作者没有完全重写策略学习框架，而是将场景图作为 Transformer encoder 输入，再条件化 Diffusion Policy。
  - 这使方法具有一定模块化和可扩展性。

## 7.2 实验设计亮点

- **覆盖模拟与真实环境**
  - 同时验证移动操作和桌面操作，证明方法不是只适用于单一平台。

- **任务设计紧扣问题本身**
  - 每个任务都包含部分可观测或长时序依赖，不是简单短程抓取任务。

- **消融实验明确**
  - 去除 3D 和去除所有位置信息的实验很好地证明了空间记忆的必要性。

- **包含运行时和失败分析**
  - 论文不仅展示成功率，也分析了感知失败、跟踪失败、策略失败以及执行频率。

# 8. 不足与局限

## 8.1 方法局限

- **依赖感知模块准确性**
  - 方法依赖 LLM、GroundingDINO、DINO-v2、XMem 等外部模块。
  - 如果物体检测、分割或跟踪失败，场景图会被污染，从而影响策略。

- **假设不可见物体静止**
  - 当物体离开视野时，系统保持其 3D 坐标不变。
  - 这一假设在受控实验中合理，但在真实家庭环境中，人或其他机器人可能移动物体，导致记忆过时。

- **场景图结构较浅**
  - 当前实现主要是“场景根节点 + 物体节点”的两层结构。
  - 没有显式建模房间、容器、支撑关系、开闭状态、物体间功能关系等更丰富层级。
  - 对多房间、多楼层、多代理或复杂动态环境的适用性仍需验证。

- **没有显式边建模**
  - 论文称通过共享坐标系隐式编码关系，但没有显式边类型，如“inside”“on top of”“near”“opened by”等。
  - 对需要复杂关系推理的任务，隐式关系可能不足。

## 8.2 实验局限

- **缺少标准公开 benchmark**
  - 实验任务均为作者构建，虽然针对性强，但难以和其他方法进行广泛可复现比较。

- **成功率缺少详细数值表**
  - 图中展示堆叠柱状图，但正文没有提供完整数值表格。
  - 这会影响结果复核和精确比较。

- **真实实验规模有限**
  - 每个任务 300 条示范已经较可观，但真实评估 episode 数量有限。
  - 文中提到真实失败分析基于 64 episodes，规模仍不算大。

- **任务语义相对简单**
  - LLM 只需从简单任务名称中提取物体。
  - 尚未证明该方法能处理复杂自然语言、多条件约束或模糊指令。

- **没有和更强记忆模型充分比较**
  - 对比了 PTP，但没有比较：
    - 显式 SLAM 地图策略；
    - 3D occupancy / semantic map 输入策略；
    - recurrent policy；
    - memory transformer；
    - 大型 VLA 模型加历史上下文。
  - 因此，虽然证明了场景图记忆有效，但尚未完全覆盖所有强基线。

## 8.3 应用限制

- **高度动态或拥挤环境中可能退化**
  - 多物体遮挡、相似物体、多实例频繁出现时，身份维护和 mask overlap 匹配可能出错。

- **实时性仍受感知模块制约**
  - Diffusion policy 本身达到约 3.5–3.8 Hz，但 mask propagation 每帧 0.19 秒，是主要开销。
  - 对高速操作任务或更高频控制任务可能不足。

- **泛化性仍需更大规模验证**
  - 论文展示了跨任务、跨平台有效性，但是否能扩展到开放世界家庭环境、多房间布局、自然用户指令，还需要进一步实验。

# 总结

本文提出了一种将**动态任务相关场景图**作为机器人模仿学习显式记忆的方法，用于解决部分可观测和长时序任务中的历史信息遗忘问题。该方法通过 LLM 枚举任务相关物体，利用 GroundingDINO、DINO-v2、RGB-D 和 XMem 构建并维护物体节点，再将场景图编码为 Transformer token 条件化 Diffusion Policy。实验表明，该方法在模拟移动操作和真实桌面操作中均显著优于 RGB、当前可见物体表示和长上下文基线，尤其在需要记住不可见目标位置、跟踪任务进度和跨越多个子任务时优势明显。其主要价值在于提供了一种紧凑、可解释、可动态更新的机器人策略记忆形式，但仍受限于感知模块准确性、动态环境假设、实验 benchmark 覆盖范围和复杂开放场景泛化能力。
