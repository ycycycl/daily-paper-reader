## 摘要
离散视觉令牌应为自动驾驶中基于令牌的世界建模与规划提供紧凑表征。然而，大多数分词器继承自图像生成，并主要针对像素重建进行优化，这可能在易于生成的内容与有助于解码驾驶决策的内容之间造成差距。我们提出了一种表征引导且几何增强的分词器，在联合监督下学习离散令牌。该分词器通过特征解码将其离散瓶颈与冻结的 DINO 特征空间对齐，同时利用带有感知损失和对抗损失的 RGB 重建来保持外观。为注入与几何状态相关的线索，我们在训练过程中加入相邻帧深度和相对位姿监督，并通过多码本量化稳定联合目标。我们使用轻量级规划读出模块和 GPT 风格的下一令牌世界模型评估相同的学习令牌。在 NAVSIM 上的实验表明，该方法提高了重建保真度和表征一致性，在固定解码器下实现了具有竞争力的规划性能，并在匹配设置下获得了更好的生成质量。

## Abstract
Discrete visual tokens should provide a compact representation for both token-based world modeling and planning in autonomous driving. However, most tokenizers are inherited from image generation and are optimized mainly for pixel reconstruction, which may leave a gap between what is easy to generate and what is useful to decode for driving decisions. We present a representation-guided and geometry-enhanced tokenizer that learns discrete tokens under joint supervision. The tokenizer aligns its discrete bottleneck with a frozen DINO feature space through feature decoding, while preserving appearance via RGB reconstruction with perceptual and adversarial losses. To inject geometric state-related cues, we add adjacent-frame depth and relative-pose supervision during training and stabilize joint objectives with multi-codebook quantization. We evaluate the same learned tokens with a lightweight planning readout and a GPT-style next-token world model. Experiments on NAVSIM show improved reconstruction fidelity and representation consistency, competitive planning performance under a fixed decoder, and better generative quality under matched settings.

### 研究背景

本文关注自动驾驶中的一个关键中间表示问题：**离散视觉 token 是否能够同时服务于世界模型生成与下游规划决策**。

近年来，自动驾驶世界模型逐渐从连续潜空间、扩散式预测，发展到类似语言模型的 **token 序列建模范式**。在这种范式中，视觉观测首先被离散 tokenizer 压缩为 token 序列，再由 GPT-style Transformer 等模型进行下一 token 预测，从而实现未来场景生成、反事实想象和规划辅助。

然而，现有很多自动驾驶 token pipeline 直接继承图像生成领域的 tokenizer，例如 VQ-VAE、VQGAN 或 LlamaGen 风格 tokenizer。这类 tokenizer 的主要目标通常是：

- 尽可能重建 RGB 像素；
- 提高视觉生成质量；
- 降低重建误差或提升感知质量。

但自动驾驶规划所需要的信息并不只是“像素好看”，还包括：

- 车辆、行人、道路边界等语义结构；
- 场景中物体的空间布局；
- 深度、相对运动、ego-motion 等几何状态信息；
- 对未来轨迹决策有用的可解码表示。

因此，论文指出一个核心矛盾：

> 面向图像生成优化的视觉 token，不一定是面向自动驾驶规划最有用的 token。

### 论文试图解决的问题

本文试图学习一种 **统一驾驶离散 token 表示**，使其同时满足两类需求：

1. **世界模型友好**  
   token 可以作为 GPT-style 自回归世界模型的预测目标，用于生成未来驾驶场景。

2. **规划任务友好**  
   token 可以作为冻结视觉表示，被轻量级规划解码器读取，用于预测未来 ego 轨迹。

## 2. 方法论：表征引导 + 几何增强 + 多码本量化

论文提出一种 **Representation- and Geometry-Guided Discrete Tokenizer**，基于 VQ-VAE 离散量化框架，但在监督信号和量化机制上进行了扩展。

整体方法可以分为四个核心部分：

1. 基于 DINO 特征的表征引导编码；
2. RGB 与 DINO 特征联合重建；
3. 相邻帧深度与相对位姿几何监督；
4. 多码本量化以缓解离散瓶颈容量不足。

---

### 2.1 Tokenizer 输入与编码流程

给定前视相机 RGB 图像：

\[
I_t \in \mathbb{R}^{H \times W \times 3}
\]

模型将图像切分为非重叠 patch，得到 patch 序列，长度为：

\[
L = H_p W_p
\]

其中 \(H_p = H/P\)，\(W_p = W/P\)。

#### 冻结 DINO 特征

作者使用冻结的 DINOv3-B 视觉基础模型提取 patch-level 特征：

\[
F_t = \Phi(I_t) \in \mathbb{R}^{L \times d_{dino}}
\]

这里 \(\Phi\) 完全冻结，不参与训练。

DINO 特征提供较强的语义与结构信息，例如道路、车辆、物体布局等。

#### RGB detail 分支

由于 DINO 特征偏高级语义，可能弱化低层纹理、边缘和细节，作者额外引入 RGB detail encoder：

\[
R_t = E_{rgb}(I_t)
\]

然后将 RGB detail 特征与 DINO 特征拼接：

\[
X_t = W_f [R_t; F_t]
\]

再输入带 RoPE 位置编码的 Transformer encoder：

\[
H_t = E_g(X_t)
\]

这一设计的目的在于：

- DINO 分支提供语义结构；
- RGB detail 分支补充纹理和边界；
- Transformer 建模全局空间关系。

---

### 2.2 离散向量量化

编码后的连续特征 \(H_t\) 被投影到量化空间：

\[
Z_t = P_{in}(H_t)
\]

每个 patch token 通过最近邻查找分配到 codebook 中的一个 codeword：

\[
k_{t,l} = \arg\min_k \|z_{t,l} - c_k\|^2_2
\]

得到离散索引 \(k_t\) 和对应量化 embedding \(E_t\)。

作者采用：

- hard vector quantization；
- straight-through estimator；
- EMA codebook 更新；
- commitment loss；
- dead-code reinitialization；
- active codeword orthogonality regularization。

这样做是为了避免多目标训练下常见的：

- codebook collapse；
- codebook 利用率低；
- 表征容量不足；
- 多个监督目标相互干扰。

---

### 2.3 RGB 与 DINO 特征联合解码

量化后的 token 经过 post-Transformer：

\[
S_t = E_{post}(P_{out}(E_t))
\]

然后分别通过两个解码头：

1. RGB 图像解码器：

\[
\hat{I}_t = D_{img}(S_t)
\]

2. DINO 特征解码器：

\[
\hat{F}_t = D_{dino}(S_t)
\]

损失函数包括：

#### RGB 重建损失

\[
L_{rec} = \|\hat{I}_t - I_t\|^2_2 + \lambda_{lpips} LPIPS(\hat{I}_t, I_t)
\]

即像素级 L2 损失加 LPIPS 感知损失。

#### 语义表征对齐损失

作者将解码出的 DINO 特征 \(\hat{F}_t\) 与冻结 DINO 特征 \(F_t\) 对齐，使用 cosine similarity 与 MSE 的组合，记为：

\[
L_{sem}
\]

#### 对抗损失

引入 discriminator，对 RGB 重建结果使用标准 GAN loss：

\[
L_{gan}
\]

整体 tokenizer 损失为：

\[
L_{tok} = \lambda_{rec}L_{rec} + \lambda_{sem}L_{sem} + \lambda_{gan}L_{gan} + \lambda_{vq}L_{vq}
\]

### 2.4 相邻帧几何监督

为了让 token 不仅表示单帧外观和语义，还携带几何与运动信息，作者加入相邻帧监督。

给定相邻帧 \(I_t\) 和 \(I_{t+1}\)，从 tokenizer 得到 post-quantization token 特征：

\[
\tilde{H}_t, \tilde{H}_{t+1}
\]

作者为每帧加入 ego token，并通过一个时序聚合器 \(A_\psi\) 进行 frame-wise attention 与 cross-frame attention：

\[
\{U_t, \bar{g}_t, U_{t+1}, \bar{g}_{t+1}\} = A_\psi(\{[g_t; \tilde{H}_t], [g_{t+1}; \tilde{H}_{t+1}]\})
\]

然后解码两个几何任务：

#### 深度预测

使用 DPT-style dense prediction head 输出深度与置信度：

\[
\hat{D}_t, \hat{C}_t = D_\omega(U_t)
\]

深度损失使用有效像素 mask，并可结合置信度权重和平滑正则，记为：

\[
L_{depth}
\]

#### 相对位姿预测

从两个更新后的 ego token 预测相邻帧相对位姿：

\[
\hat{p}_{t \rightarrow t+1} = [\hat{t}_{t \rightarrow t+1}, \hat{q}_{t \rightarrow t+1}]
\]

其中包含平移和四元数旋转。

相对位姿损失为：

\[
L_{pose} =
\|\hat{t}_{t \rightarrow t+1} - t_{t \rightarrow t+1}\|_1
+
\min(
\|\hat{q}_{t \rightarrow t+1} - q_{t \rightarrow t+1}\|_1,
\|\hat{q}_{t \rightarrow t+1} + q_{t \rightarrow t+1}\|_1
)
\]

最终加入几何监督后的总损失为：

\[
L = L_{tok} + \lambda_{geo}(\lambda_{depth}L_{depth} + \lambda_{pose}L_{pose})
\]

需要注意的是，几何解码分支只在 tokenizer 训练阶段使用，推理时不需要深度或位姿标签。

### 2.5 多码本量化 Multi-Codebook Quantization

作者观察到，当同一个离散瓶颈需要同时保留：

- RGB 外观；
- DINO 语义表征；
- 深度；
- ego-motion 几何信息；

单一 codebook 容量可能不足，导致 RGB 重建质量下降或 codebook 不稳定。

因此，作者引入多码本量化。

对于每个 patch 的预量化特征 \(z_{t,l}\)，先通过 attention-based splitter 产生 \(M\) 个 head-specific 向量：

\[
V_{t,l} = P_{attn}(z_{t,l})
\]

然后每个 head 使用独立 codebook 进行量化，得到多个 codeword：

\[
e^{(1)}_{t,l}, ..., e^{(M)}_{t,l}
\]

最后通过 merge projection 融合：

\[
e_{t,l} = P_{merge}([e^{(1)}_{t,l}; ...; e^{(M)}_{t,l}])
\]

论文主要实验中：

- naive tokenizer 和 Rep tokenizer 使用单码本：\(K = 16384\)；
- Rep+Geo tokenizer 使用 4 个 codebook，每个大小为 4096；
- 总体 token 空间容量提高，但 patch 网格分辨率保持不变。

### 2.6 下游规划解码器

为了评估 token 是否“可被规划使用”，作者冻结 tokenizer，只训练一个轻量级规划头。

输入包括：

- 当前帧视觉 token 特征；
- ego status 向量 \(s_t \in \mathbb{R}^{11}\)。

模型先用 learnable registers 和小型 Transformer 将 patch token 压缩成 scene tokens：

\[
Z^{scene}_t = f_{scene}(P_t)
\]

然后将 ego 状态 embedding 与 scene tokens 交互，输出多条候选未来轨迹：

\[
Y = \{y_\tau\}_{\tau=1}^{T}, \quad y_\tau = (x_\tau, y_\tau, \psi_\tau)
\]

训练时：

- 使用 L1 轨迹回归；
- 只监督与 ground-truth 最接近的候选轨迹；
- 额外训练一个 scoring head，预测 PDM-style 指标结果；
- 推理时选择 score 最高的轨迹。

作者强调该规划头较轻量，约 20M 参数，目的是评估 token 表征质量，而不是优化 planner 架构。

### 2.7 自回归世界模型

为了评估 token 是否适合世界模型生成，作者冻结 tokenizer，训练 GPT-style 自回归 Transformer。

流程如下：

1. 每帧得到 \(L\) 个离散 token；
2. 按固定扫描顺序线性化；
3. 多帧拼接成 token stream；
4. Transformer 进行 next-token prediction；
5. 使用 teacher forcing 和 cross-entropy loss；
6. 可通过 AdaLN 注入动作、ego state 等条件信息。

世界模型规模为约 1B 参数。

- RGB 重建；
- DINO 特征可视化；
- depth 重建；
- 规划失败 / 成功案例；
- 世界模型 rollout 可视化。

### 5.2 实验充分性评价

整体来看，实验设计较完整，覆盖了作者主张的三个核心维度：

1. token 是否能重建图像；
2. token 是否包含语义和几何；
3. token 是否能用于规划与世界模型生成。

其中，消融实验比较清晰，能较好说明：

- DINO 表征监督提升规划与重建；
- 几何监督提升规划中的 ego progress；
- 多码本量化缓解联合监督下的容量冲突。

### 5.3 公平性与客观性

实验有较强公平性的一面：

- 下游规划 decoder 固定；
- tokenizer 冻结；
- 对不同 tokenizer 使用相同规划读出协议；
- 世界模型比较中声称采用 matched setting；
- NAVSIM test split 不与训练数据重叠。

但也存在一些需要谨慎看待的地方：

- 规划对比中，端到端方法使用的传感器不同，包括多相机和 LiDAR，因此跨传感器比较并不完全公平；
- 本文方法主要是单前视相机，与多相机系统的实际场景覆盖能力不同；
- 世界模型生成对比主要与 DrivingGPT 对比，缺少更多 tokenizer 或生成模型基线；
- 所有主要实验集中在 NAVSIM，缺少 nuScenes、Waymo、CARLA 或真实闭环 benchmark 的跨数据集验证；
- 深度监督依赖 DVGT 伪标签，其噪声或偏差可能影响 token 学习。

## 6. 主要结论与发现

论文的主要发现可以概括如下。

### 6.1 表征监督能显著改善 token 质量

加入 DINO 特征重建后，token 不仅提升 RGB 重建质量，也在 DINO feature space 中更接近原图。

例如在 Table 1 中，Ours-Rep 相比 LlamaGen 和 Orbis：

- rFID 更低；
- PSNR 更高；
- DINO feature discrepancy 更小。

这说明冻结基础模型表征可以作为有效监督，引导离散 token 学到更结构化、更语义化的驾驶场景表示。

### 6.2 几何监督使 token 更适合规划

加入相邻帧深度和相对位姿监督后，token 能够预测深度和 ego-motion 信息。

在规划消融中：

- Naive tokenizer 的 PDMS 为 85.5；
- +Rep 后提升到 89.4；
- +Rep + Geo 后提升到 90.9；
- +Rep + Geo + MCB 后达到 91.8。

这表明规划任务确实受益于语义和几何信息。

### 6.3 单码本存在容量瓶颈，多码本能缓解冲突

Table 2 显示，加入几何监督后，如果仍使用有限离散瓶颈，PSNR 会明显下降，从 26.51 降到 23.90。

但加入多码本后，PSNR 恢复到 26.33，同时仍保持较好的深度和位姿性能。

这说明问题不是语义、几何和外观目标天然冲突，而是单一离散瓶颈容量不足。

### 6.4 统一 token 同时支持规划和生成

在 NAVSIM 上，本文方法在基于冻结 token 的单视角规划方法中取得最高 PDMS：

- Ours：91.8；
- DriveVLA-W0：90.2；
- DriveLaW：89.1；
- PWM：88.1；
- Epona：86.2。

同时，在世界模型生成中，本文 token 训练的 GPT-style 模型相较 DrivingGPT 获得更好的 FID 和 FVD。

这支持作者的核心论点：

> 一个经过语义表征和几何监督训练的离散 tokenizer，可以作为自动驾驶世界模型和规划任务的统一接口。

## 7. 优点与亮点

### 7.1 问题定义有价值

论文没有单纯追求图像重建或视频生成，而是明确指出自动驾驶 token 的特殊性：

- token 是世界模型的预测目标；
- token 也是规划模块的输入表示。

这种“统一驾驶语言”的视角较有启发性。

### 7.2 监督信号设计合理

方法同时使用：

- RGB 重建；
- LPIPS 感知损失；
- GAN 对抗损失；
- DINO 特征重建；
- 深度监督；
- 相对位姿监督。

这些监督覆盖了：

- 外观；
- 语义；
- 几何；
- 运动状态。

相比单纯像素重建 tokenizer，更符合自动驾驶任务需求。

### 7.3 DINO 特征对齐具有实用性

使用冻结 DINO 特征作为语义表征目标，可以避免直接依赖昂贵的语义标签，同时提供较强的 object-level 和 layout-level 结构信息。

这使得 tokenizer 能够在不显式标注检测框或分割 mask 的情况下学习更有语义的离散表示。

### 7.4 几何分支只在训练时使用

深度和位姿监督只在 tokenizer 训练中使用，推理时不引入额外传感器或几何模型。

这意味着部署时仍只需要 RGB 输入和 tokenizer 编码，方法复杂度相对可控。

### 7.5 多码本量化针对性强

作者通过实验发现联合监督会造成容量竞争，并用多码本量化解决。这一设计与实验结论吻合较好，不是简单堆模块。

### 7.6 下游评估较全面

论文同时评估：

- 重建；
- DINO 表征一致性；
- 深度；
- 位姿；
- 规划；
- 世界模型生成。

这比只报告图像重建或规划结果更全面。

### 7.7 固定 planner 评估 token 质量

作者冻结 tokenizer，并使用相同轻量级 planner head 比较不同 token，较好地隔离了 tokenizer 表征能力的影响。

## 8. 不足与局限

### 8.1 算力与训练细节披露不足

论文未明确说明：

- GPU 型号；
- GPU 数量；
- 训练耗时；
- batch size；
- tokenizer 和世界模型训练成本；
- 1B 世界模型的训练资源。

这会影响方法的可复现性和实际部署成本评估。

### 8.2 主要实验集中在 NAVSIM

所有核心实验基本都在 NAVSIM 上完成，缺少跨数据集验证。

尚不清楚该 tokenizer 在以下数据或场景上的泛化能力：

- nuScenes；
- Waymo Open Dataset；
- Argoverse；
- CARLA 闭环仿真；
- 真实车端长尾场景；
- 多城市、多天气、多光照条件。

### 8.3 使用单前视相机，感知范围有限

本文主要采用单视角前视相机设置。虽然这使实验更轻量，但自动驾驶实际规划往往需要：

- 多相机环视；
- LiDAR；
- Radar；
- HD map；
- 历史多帧信息。

单前视 token 对侧向车辆、后方车辆、复杂路口全局拓扑的表达可能不足。

### 8.4 几何监督依赖伪标签

深度标签来自 DVGT 伪标签，而非真实高精度深度。虽然经过 radar 对齐和后处理，但仍可能存在：

- 远距离深度误差；
- 动态物体误差；
- 遮挡区域误差；
- 伪标签系统性偏差。

这些偏差可能被 tokenizer 学入离散表示中。

### 8.5 规划评估是非反应式 benchmark

NAVSIM 是 data-driven non-reactive simulation，不能完全反映真实闭环驾驶中的交互效应。

例如：

- ego 车辆动作不会真正改变其他交通参与者行为；
- 长时间闭环误差累积无法充分评估；
- 安全关键的交互场景可能覆盖不足。

因此，规划结果虽然有参考价值，但不能直接等同于真实道路闭环性能。

### 8.6 世界模型生成评估仍较有限

世界模型实验主要比较 FID 和 FVD，并与 DrivingGPT 对比。

但对于自动驾驶世界模型，更重要的可能还包括：

- 几何一致性；
- 物体运动物理合理性；
- 交通规则一致性；
- 长时序 rollout 稳定性；
- 与规划结果的耦合收益；
- counterfactual action-conditioned generation 质量。

论文在这些方面还没有展开充分评估。

### 8.7 没有真正联合训练规划与世界模型

虽然标题和动机强调统一 token 同时服务规划和世界模型，但当前 pipeline 中：

- tokenizer 先训练；
- planner 冻结 tokenizer 后单独训练；
- world model 冻结 tokenizer 后单独训练。

规划和世界模型并未在一个统一架构中端到端联合优化。作者也在结论中承认，未来工作包括统一规划与世界模型架构。

### 8.8 与更强端到端系统比较需谨慎

Table 3 中本文方法在单视角 frozen-token 方法中表现很强，但与多相机或 LiDAR 端到端方法比较时，由于传感器设置不同，不能直接说明本文方法整体优于这些系统。

---

## 总体评价

本文提出的 Unified Driving Tokens 是一篇围绕自动驾驶离散视觉表示展开的较完整工作。它的核心贡献不在于某个单独模块，而在于重新定义了驾驶 tokenizer 的目标：**token 不应只为图像重建服务，而应同时服务于世界模型预测和规划决策**。

通过 DINO 表征对齐、RGB 重建、几何监督和多码本量化，作者构建了一个兼顾外观、语义和几何的离散 token 学习框架。NAVSIM 实验显示，该 token 在重建、规划和生成三个维度上均有收益，尤其在 frozen-token 规划方法中表现突出。

不过，论文仍存在算力信息缺失、实验集中于单一 benchmark、缺少真实闭环验证和多传感器扩展等局限。整体而言，这篇论文对自动驾驶世界模型中的 tokenizer 设计具有较强参考价值，尤其适合关注“离散视觉 token 如何服务驾驶规划”的研究方向。
