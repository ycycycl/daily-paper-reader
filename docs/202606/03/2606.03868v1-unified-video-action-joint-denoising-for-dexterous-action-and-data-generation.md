## 摘要
近期的世界动作模型通过将广泛的视觉动态先验与可执行的机器人动作对齐，来利用视频基础模型。我们从分布视角重新审视这种对齐。现有公式通常将对齐后的先验收窄为以观测为条件的未来动作策略分布。相比之下，我们通过在多种条件设定下建模交互视频与可执行手部轨迹的联合空间，使该分布保持更广。我们提出 Donk，一种面向灵巧手的统一视频-动作去噪模型。在给定语言、初始图像和初始手部状态的条件下，Donk 可采样未来视频和双手 MANO 轨迹，作为一种动作策略。在不使用图像条件的情况下，同一去噪架构可从文本条件分布中采样成对的视频-动作 rollout，从而将对齐的视频先验转化为数据引擎。在动作、视频以及仅文本生成评估中，Donk 提高了灵巧轨迹精度，保持了较强的视频保真度，并在同一统一训练方案下生成平滑的文本条件动作 rollout。

## Abstract
Recent world action models leverage video foundation models by aligning broad visual-dynamics priors with executable robot actions. We revisit this alignment from a distributional perspective. Existing formulations typically narrow the aligned prior into an observation-conditioned policy distribution over future actions. In contrast, we keep the distribution broader by modeling the joint space of interaction videos and executable hand trajectories under multiple conditioning regimes. We propose Donk, a unified video-action denoising model for dexterous hands. With language, an initial image, and the initial hand state, Donk samples future videos and bimanual MANO trajectories as an action policy. Without the image condition, the same denoising architecture samples paired video-action rollouts from a text-conditioned distribution, turning the aligned video prior into a data engine. Across action, video, and text-only generation evaluations, Donk improves dexterous trajectory accuracy, preserves strong video fidelity, and produces smooth text-conditioned action rollouts under the same unified training recipe.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体方法：Donk

- 论文提出 **Donk**，一个统一的视频-动作联合去噪模型。
- Donk 基于视频扩散 Transformer，具体初始化自 **Wan2.2 TI2V-5B** 视频生成模型。
- 模型同时生成：
  - 交互视频；
  - 与视频在时间和空间上对齐的双手 MANO 手部动作轨迹。

- MANO 参数用于表示灵巧手动作，包括手腕姿态、手指关节等结构化手部状态。

---

### 2.2 统一条件分布建模

论文将任务写成统一条件生成问题：

\[
p_\theta(V_{0:T}, A_{1:T} \mid c, I^\star, g_0)
\]

其中：

- \(V_{0:T}\)：视频序列；
- \(A_{1:T}\)：未来手部动作轨迹；
- \(c\)：语言指令；
- \(I^\star\)：可选图像条件，可能是初始图像 \(I_0\)，也可能为空；
- \(g_0\)：初始手-相机几何锚点，包括初始 MANO 手状态和相机内参。

模型有两种推理模式：

#### 1）TI2VA 策略模式

当给定初始图像时：

\[
p_\theta(V_{1:T}, A_{1:T} \mid c, I_0, g_0)
\]

模型根据语言、初始图像和初始手状态预测未来视频和动作，可作为动作策略使用。

#### 2）T2VA 数据引擎模式

当没有初始图像时：

\[
p_\theta(V_{0:T}, A_{1:T} \mid c, \tilde{g}_0)
\]

模型仅根据文本和一个初始化的合理手-相机锚点，生成完整交互视频和动作轨迹。

---

### 2.3 视频与动作 token 联合建模

Donk 将视频 latent 和动作 token 拼接为统一序列：

\[
z = [z_{\text{video}}, z_{\text{action}}, z_{\text{anchor}}]
\]

其中：

- 视频通过 Wan 的 VAE 编码到视频 latent 空间；
- 动作通过轻量动作编码器转换为 action token；
- 初始几何锚点通过 anchor encoder 编码为 anchor token。

模型使用 Transformer denoiser 对视频 token 和动作 token 进行联合去噪。

---

### 2.4 Video-Preserving Joint Attention：保护视频先验的联合注意力

直接让视频 token 和动作 token 完全双向注意力交互，可能破坏预训练视频模型的生成能力。

因此作者设计了**视频保护式联合注意力掩码**：

- 视频 token 只能 attend 到视频 token；
- 动作 token 和 anchor token 可以 attend 到完整序列，包括视频 token、动作 token 和 anchor token。

这样做的目的：

- 保留 Wan 视频生成模型原有的视频先验；
- 让动作 token 从生成的视频动态中读取信息；
- 避免新加入的动作分支干扰视频质量；
- 实现视频-动作对齐，同时尽量不牺牲视频生成稳定性。

---

### 2.5 Anchor-Map Controller：初始手-相机几何控制

为了让生成的手部动作和图像空间位置对齐，Donk 引入了初始几何锚点：

\[
g_0 = (s_0, K)
\]

其中：

- \(s_0\)：初始 MANO 手状态；
- \(K\)：相机内参。

然后通过渲染函数生成初始 anchor map：

\[
M_0 = R(g_0)
\]

该 anchor map 是一个颜色编码的 MANO 骨架图像，用来提供第一帧手部位置和姿态的几何提示。

接着：

- 使用冻结的 Wan VAE 编码 anchor map；
- 通过轻量 anchor-map adapter 转换到 Wan token 空间；
- 在选定的 Transformer 层中，将 anchor hint 注入第一帧视频 token。

注入形式为：

\[
z^{\ell,0}_{\text{video}} \leftarrow z^{\ell,0}_{\text{video}} + \gamma_\ell H_\ell
\]

其中：

- \(H_\ell\)：第 \(\ell\) 层的 anchor hint；
- \(\gamma_\ell\)：可学习门控参数，初始化为 0。

这样可以保证模型训练初期不破坏预训练 Wan 的行为。

---

### 2.6 文本条件初始锚点生成器

在 T2VA 文本生成模式中，没有真实初始图像和手部状态。

因此作者额外训练了一个轻量的文本条件 initializer，用于从文本中采样合理的初始手-相机配置 \(\tilde{g}_0\)。

需要注意的是：

- 这个 initializer 只提供第一帧几何 scaffold；
- 它不提供未来动作计划；
- 后续视频和动作轨迹仍由统一的 Donk denoiser 生成。

---

### 2.7 训练目标

Donk 使用 flow matching 训练视频和动作的速度场预测。

主要损失包括：

#### 1）视频 flow matching 损失

\[
L_{\text{video}} = \|\hat{v}_x - v_x\|^2_2
\]

#### 2）动作 flow matching 损失

\[
L_{\text{action}} =
\frac{\|M_a \odot (\hat{v}_a - v_a)\|^2_2}{\max(\sum M_a, 1)}
\]

其中 \(M_a\) 是有效手部维度 mask，用于处理缺失或无效手部数据。

#### 3）手部区域加权视频损失

作者使用 \(L_{\text{gaze}}\) 强化手-物交互区域的视频监督，让模型更关注手部附近的视觉动态。

#### 4）教师先验正则

使用冻结的 Wan teacher 预测视频速度 \(\hat{v}^{tea}_x\)，并约束 Donk 的视频预测不要偏离预训练视频模型：

\[
L_{\text{prior}} = \|\hat{v}_x - \hat{v}^{tea}_x\|^2_2
\]

该项只在保留图像条件时使用，避免文本-only 分支错误模仿需要图像条件的 teacher。

#### 5）总损失

\[
L_{\text{Donk}} =
\lambda_v(L_{\text{video}} + \lambda_g L_{\text{gaze}})
+ \lambda_a L_{\text{action}}
+ \lambda_p L_{\text{prior}}
\]

## 6. 主要结论与发现

- Donk 成功将视频生成和动作生成统一在一个联合去噪框架中。
- 在 TI2VA 策略模式下，Donk 在 OakInk2 上取得最好的手部姿态和手腕平移轨迹表现。
  - Hand RMSE 从 DreamZero-alike 的 0.262 降至 0.238；
  - ADE、FDE、DTW 等指标均优于对比方法；
  - wrist rotation 上略逊于 VITRA，但整体动作精度更强。

- 在视频质量方面，Donk-TI2VA 没有明显破坏预训练视频模型能力。
  - 在 EgoDex/LOME 评估中，Donk 获得最高 PSNR、SSIM、最低 LPIPS 和最高 CLIP-I；
  - FVD 也优于 Wan2.2-TI2V-5B，但不如部分纯视频模型在个别指标上的表现。

- 在 T2VA 模式下，Donk 可以仅根据文本生成成对的视频和手部动作 rollout。
  - 相比 Wan2.2-5B-I2V，Donk-T2VA 的 FVD 从 306.2 降至 191.1；
  - VLM judge 从 1.59 提升到 2.37；
  - CLIP-S 也略有提升；
  - tLPIPS 稍差，说明时间平滑性仍有提升空间。

- 消融实验表明：
  - State conditioning 对动作精度提升明显；
  - Gaze/hand-focused supervision 进一步改善手部 RMSE、轨迹误差和旋转误差。

## 7. 优点与亮点

- **统一建模视角清晰**  
  将 TI2VA 策略学习和 T2VA 数据生成统一为同一个条件生成问题，而不是设计两个独立系统。

- **分布视角具有启发性**  
  论文从条件分布角度指出现有 WAM 过度局限于 observation-conditioned policy，并提出更宽泛的视频-动作联合分布。

- **较好利用视频基础模型先验**  
  基于 Wan 视频生成模型初始化，使模型继承强大的视频动态和视觉生成能力。

- **视频保护式注意力设计合理**  
  通过非对称 attention mask 避免动作 token 干扰视频 token，降低了联合训练破坏视频生成质量的风险。

- **显式几何锚点有助于空间对齐**  
  Anchor-map controller 提供初始手-相机几何 scaffold，有助于解决手部位置和图像空间对齐问题。

- **动作表示结构化**  
  使用 MANO 参数表示双手动作，适合灵巧手建模，也便于与人手数据和仿真/机器人手重定向结合。

- **实验同时覆盖动作和视频两侧**  
  不仅报告动作误差，也报告视频生成质量，符合“世界动作模型”的双重目标。

- **具备数据生成潜力**  
  T2VA 模式为从文本生成配对视频-动作数据提供了初步可行路径，对灵巧操作数据稀缺问题有潜在价值。

## 8. 不足与局限

- **缺少真实机器人闭环验证**  
  论文主要是离线动作预测和生成质量评估，没有展示 Donk 生成的动作能否直接驱动真实机器人完成任务。

- **T2VA 动作质量缺少系统定量评估**  
  虽然论文展示了文本生成视频-动作 rollout，但由于缺少可对比基线，T2VA 动作轨迹主要依赖定性展示，缺少大规模定量验证。

- **生成数据是否提升下游策略未被充分证明**  
  论文声称 T2VA 可作为数据引擎，但没有充分展示用这些生成数据训练或增强机器人策略后的实际收益。

- **依赖 MANO 表示，跨机器人 embodiment 仍需额外处理**  
  MANO 是人手模型，若用于真实灵巧机器人手，需要进行动作重定向，可能引入误差和约束不匹配。

- **对初始 anchor initializer 的细节披露较少**  
  文本条件初始化器如何训练、效果如何、失败模式如何，论文中说明相对有限。

- **算力需求较高**  
  训练使用 64 张 96GB NVIDIA Hopper GPU，说明该方法对计算资源要求较高，不易被普通实验室复现。

- **best-of-K 评估可能高估策略实用性**  
  在真实机器人控制中通常只能执行一个动作样本，而 best-of-5 / best-of-10 会选择最接近 ground truth 的结果，可能不能完全反映部署表现。

- **视频指标与机器人任务成功率之间仍有差距**  
  PSNR、LPIPS、FVD、CLIP-S 等指标可以衡量视频质量，但不一定直接对应物理可执行性或任务成功率。

- **开放场景泛化仍需更多验证**  
  T2VA 展示了野生动物、灭火等开放文本场景，但这些结果主要是定性展示，缺少系统性安全性、物理合理性和动作可执行性分析。
