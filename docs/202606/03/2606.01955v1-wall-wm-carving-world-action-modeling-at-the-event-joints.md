## 摘要
WALL-WM 是一种世界动作模型（World Action Model），它将视频—动作学习从以片段为中心的优化转向以事件为基础的视觉—语言—动作（Vision-Language-Action, VLA）预训练，并以语义连贯的动作事件作为学习的原子单元。现有 WAM 通常从多模态或视频基础模型初始化，随后在当前观测和指令的直接条件下优化固定长度的动作片段。尽管这种做法较为便捷，但以片段为中心的表述会造成根本性的粒度不匹配。语言描述的是语义目标和事件，视觉随连续的场景动态而演化，动作则运行在控制层面的时间尺度上；将三者强行纳入同一个固定长度预测窗口，会把 VLA 训练转化为短时域相关性拟合。WALL-WM 通过围绕语义事件组织监督信号和数据来解决这一不匹配问题。具体而言，它将事件基础的 VLA 预训练与由事件级字幕和聚类均衡采样构建的数据生态相结合，从而能够在多样化的行为、场景和任务结构上进行可扩展学习。基于同一个事件预训练骨干网络，WALL-WM 支持两种互补的推理模式。事件模式接收下一事件描述，并支持可变长度的执行片段；统一模式则使用带有阶梯式解码（Staircase Decoding）的 VLM，在保持梯度连续的 VLA 路径的同时，对传统固定长度片段推理进行条件化。结合基于 Muon 优化器的大规模预训练基础设施，WALL-WM 为通用 WAM 提供了一套实用的规模化方案。实验表明，WALL-WM 能够在语言、场景和任务上实现广泛泛化，并在大规模真实世界泛化评测中达到最先进性能。

## Abstract
WALL-WM is a World Action Model that shifts video-action learning from chunk-centric optimization to event-grounded Vision-Language-Action pretraining, using semantically coherent action events as the atomic unit of learning. Existing WAMs commonly initialize from multimodal or video foundation models and then optimize fixed-length action chunks conditioned directly on the current observation and instruction. Although convenient, this chunk-centric formulation creates a fundamental granularity mismatch. Language describes semantic goals and events, vision evolves through continuous scene dynamics, and actions operate at control-level timescales; forcing all three into the same fixed-length prediction window turns VLA training into short-horizon correlation fitting. WALL-WM addresses this mismatch by organizing both supervision and data around semantic events. Specifically, it pairs event-grounded VLA pretraining with a data ecosystem built from event-level captions and cluster-balanced sampling, enabling scalable learning over diverse behaviors, scenes, and task structures. From the same event-pretrained backbone, WALL-WM supports two complementary inference modes. The event mode consumes next-event descriptions and enables variable-length execution chunks, while the unified mode uses a VLM with Staircase Decoding to condition conventional fixed-length chunk inference while preserving a gradient-continuous VLA path. Together with Muon-optimizer-based large-scale pretraining infrastructure, WALL-WM provides a practical scale-up recipe for general-purpose WAMs. Experiments show that WALL-WM generalizes broadly across language, scenes, and tasks, achieving state-of-the-art performance in large-scale real-world generalization evaluation.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 核心思想

WALL-WM 的核心建模目标是：

\[
p_\theta(V_e, a_e \mid V_0, s, c_e)
\]

其中：

- \(V_0\)：当前多视角观测，一般是每个相机的一帧关键图像；
- \(s\)：当前机器人本体状态；
- \(c_e\)：下一语义事件描述；
- \(V_e\)：该事件对应的未来多视角视频；
- \(a_e\)：该事件对应的末端执行器动作轨迹；
- \(e\)：一个语义事件，长度可变。

也就是说，模型不是预测“接下来固定 1 秒动作”，而是预测：

> 在当前状态下，根据下一事件描述，生成该事件完整持续期间的未来视频和动作轨迹。

---

## 2.2 整体架构

WALL-WM 是一个 **视频塔 + 动作塔** 的双塔结构。

### 视频塔：Multi-View Video DiT

- 继承自 Wan 系列文本到视频模型；
- 主要负责未来多视角视频 latent 的去噪生成；
- 保留原有单视角视频生成能力；
- 新增多视角建模机制。

### 动作塔：Action Transformer / Action DiT

- 随机初始化；
- 与视频塔深度相同；
- 每一层动作 block 都 cross-attend 到对应视频 block 的特征；
- 负责对动作轨迹进行 flow matching 去噪。

### 单向层级耦合

论文强调视频塔和动作塔不是简单拼接，而是采用 **layer-wise one-way coupling**：

- 视频塔生成每层视觉特征；
- 动作塔每层读取对应视频层的 key/value；
- 动作塔更新动作表示；
- 视频塔不被动作塔反向修改。

这样做的意图是：

- 保留视频基础模型的视觉—语义先验；
- 避免动作监督破坏视频生成模型；
- 让动作预测从视频动态特征中获得时空和接触信息。

---

## 2.3 多视角视觉事件建模

论文对 Wan 单视角视频模型进行了多视角扩展，关键技术包括：

### 2.3.1 Multi-View Adaptation

在每个 DiT block 中加入跨视角注意力分支：

- 每个相机视角仍按单视角方式进入原 Wan self-attention；
- 然后将同一时间帧下多个相机的空间 token 拼接；
- 在跨视角 token 上执行 self-attention；
- 输出经过零初始化 projector 后加回原特征。

其设计重点是：

- 保留预训练 Wan 的单视角能力；
- 用零初始化分支避免训练初期破坏原有先验；
- 逐步学习跨视角信息交换。

### 2.3.2 Camera RoPE

为不同相机引入可学习的旋转位置编码：

- 不依赖推理时相机标定；
- 每个相机有独立 learnable view embedding；
- 支持不同机器人、不同相机数量和布局。

### 2.3.3 Cross-View Geometric Masking

训练时使用几何约束来提升跨视角一致性，包含两类 mask：

#### Sight-cone attention masking

- 根据相机内外参把图像 patch 反投影为视锥；
- 如果两个 patch 的视锥在 3D 空间中有交集，则允许跨视角注意力；
- 否则在 attention bias 中加入 \(-\infty\)，禁止注意力连接。

作用：

- 防止跨视角 attention 变成无约束特征混合；
- 强化物理上合理的跨视角对应关系。

#### Tube patch masking

- 随机选择一个视角中的空间窗口；
- 在所有 latent 帧上遮蔽该窗口；
- 要求模型从其他视角恢复被遮蔽内容；
- 对遮蔽区域的预测损失可加权。

作用：

- 主动制造跨视角信息恢复需求；
- 迫使模型学习真实跨视角对应。

论文说明主训练 recipe 中主要保留 border masking 和 sight-cone supervision，tube masking 更多用于强化跨视角对应的实验设置。

## 2.4 视频与动作的 flow matching 训练

### 视频侧

视频塔采用 Wan 风格的 \(v\)-prediction flow matching。

给定干净视频 latent \(z_0^V\)、噪声 \(\epsilon^V\)、时间步 \(t_V\)，目标为：

\[
C_V^\star = \epsilon^V - z_0^V
\]

损失为预测 flow 和目标 flow 的均方误差。

训练时还使用：

- border masking：排除黑边、无效区域；
- EMA；
- resolution / length bucketing；
- 事件长度截断；
- caption drop schedule。

### 动作侧

动作塔也采用 flow matching，主要使用 \(v\)-prediction：

\[
v_A^\star = \epsilon^A - a_0
\]

动作损失是动作轨迹预测速度和目标速度之间的 MSE。

论文还提到可选：

- \(x\)-prediction：直接预测 clean action；
- DCT 辅助损失：强调轨迹整体形状，减少抖动；
- 但主实验默认使用 \(v\)-prediction，关闭 DCT。

## 2.5 视频—动作时间对齐

WALL-WM 区分两种窗口布局。

### 事件中心窗口

用于事件级预训练和 event-mode 推理：

- 一个事件 caption 对应一个事件视频和动作；
- 事件长度可变；
- 动作 token 按照视频 latent 帧分组；
- 使用相对时间 embedding \(E_\tau\) 对齐视频帧和动作片段。

### 观测中心窗口

用于 fixed-chunk unified mode：

- 输入包含历史帧、当前 anchor、未来帧；
- 使用相对时间 embedding \(E_\tau\) 和绝对窗口 embedding \(E_{abs}\)；
- 通过历史窗口解决全局指令下局部 chunk 语义不明确的问题。

## 2.6 非对称视频—动作去噪步映射

论文提出两种视频动作 denoising 对齐方式：

### 对称 1-to-1

- 动作去噪第 \(j\) 步读取视频第 \(j\) 步特征；
- 适合小数据联合训练。

### 非对称 1-to-\(N_d\)

主 recipe 使用该方案：

- 视频塔冻结；
- 只运行一次视频 forward 到固定 anchor step \(s^\star\)；
- 所有动作去噪步都读取这个固定视频特征；
- 论文默认：
  - 总动作去噪步 \(N_d=50\)；
  - anchor \(s^\star=45\)；
  - 训练时可采样 \(K=6\) 个动作噪声水平复用同一个视频 forward。

这样做的动机是：

- 避免高噪声视频特征和真实动作监督不匹配；
- 降低训练成本；
- 保持视频 prior 稳定。

## 2.7 语言推理与 Staircase Decoding

WALL-WM 还引入了基于 Qwen3.5-9B 的语言推理模块。

### VLM text conditioner

输入：

- 当前多视角观测；
- 语言指令。

输出：

- DiT 条件特征；
- 下一事件描述；
- 当前事件剩余时间估计。

训练目标包括：

- 对齐 VLM 输出特征和原 T5 text embedding；
- 下一事件 caption 的交叉熵损失；
- 剩余时间的 Huber 回归损失。

### Staircase latent CoT

传统 CoT 逐 token 自回归生成，延迟高。WALL-WM 提出 **Staircase latent decoding**：

- 不生成离散文本 CoT；
- 生成一组连续 latent reasoning states；
- 低层 Transformer 只计算一次共享视觉语言 grounding；
- 高层通过 staggered / parallel 方式并行生成多个 latent CoT；
- latent CoT 直接注入 WAM cross-attention。

训练时使用 frozen lightweight Qwen3.5-0.8B 做 latent-to-text reconstruction：

- 将 latent CoT 投影成 soft prefix；
- 冻结小语言模型重建文本 CoT；
- 只训练 staircase reasoning branch 和 prefix projector。

## 2.8 两种推理模式

### Event Mode

- 每一步由人、VLM 或 agent 提供下一事件描述；
- WALL-WM 执行该事件对应的可变长度视频—动作段；
- 执行完成后重新观测并规划下一事件。

优点：

- 更接近自然任务阶段；
- 避免固定 chunk 切断语义动作；
- 适合长时程任务和事件级 agent 接口。

### Unified Mode

- 仍然执行固定长度 action chunk；
- 但通过历史窗口和 Staircase latent CoT 提供事件化上下文；
- 兼容传统 VLA benchmark 和固定频率控制接口。

#### Diverse Manipulation

测试常规桌面操作：

- 倒三角摆杯；
- 勺子放入碗；
- 眼镜放到木架；
- 环套杆；
- 彩色积木放到对应区域；
- 倒水；
- 拿物体放篮子。

#### Reasoning Manipulation

测试语言和语义推理：

- 耳机分类；
- 按形状分类；
- 按顺序按按钮；
- 配对物体；
- 指定水果放篮子。

#### Dexterous Manipulation

测试细粒度接触控制：

- 插线；
- 文具放入盒中。

#### Generalization

测试复杂场景泛化与指令切换：

- 盘子放入收纳槽；
- 给锅盖盖子；
- 把清洁布推到桌边；
- 螺丝刀插入杯中。

### 指标

使用 **Task Progress** 而非单纯成功率：

- 每个任务按 10 分制细分中间步骤；
- 再归一化到 0–100；
- 能反映部分完成、接触进展、语义选择等。

### 对比方法

真实机器人对比包括：

- \(\pi_0.5\)；
- DreamZero；
- LingBot-VA；
- WALL-WM-U-Scratch；
- WALL-WM event mode；
- 另有 pretrained fixed-length baseline without VI-SA 的消融。

## 5. 实验数量与充分性

## 5.1 实验数量概览

论文实验覆盖面较广，大致包括：

### 视频生成实验

- 1 个 embodied video generation benchmark；
- ID / OOD 两类任务；
- 与 Wan2.1、Wan2.2 对比；
- 多项视觉、运动、语义和物理指标。

### 3D / 多视角表征实验

- CO3Dv2 上的 3D awareness probing；
- cross-view correspondence error；
- 与 DINOv2、V-JEPA、CogVideoX、Aether、Open-Sora、WAN 等对比。

### 真实机器人实验

- 4 个真实机器人 benchmark suite；
- 合计约 18 个任务；
- 每个任务有 Task Progress rubric；
- 与多种 VLA/WAM baseline 比较。

### 消融实验

至少包含：

- event-conditioned execution + VI-SA 的联合消融；
- WALL-WM-U-Scratch vs event-mode；
- pretrained fixed-length baseline without VI-SA vs event-mode；
- 推理模式差异的间接比较。

### 系统压缩实验

论文提到：

- DMD distillation；
- FP8 quantization；
- 去噪步数压缩；
- 若移除 action loss，action MAE 下降 53%。

但这部分的详细表格和系统性消融相对有限。

## 5.2 实验是否充分

### 充分之处

- 既评估视频生成，又评估真实机器人控制；
- 覆盖常规操作、推理操作、灵巧接触、泛化场景；
- 使用 dense Task Progress，比二元成功率更细致；
- 有 ID / OOD 视频生成任务；
- 有 3D awareness 和多视角一致性评估；
- 有事件模式与从头统一模式的比较；
- 有一定事件建模和跨视角建模消融。

### 不足之处

实验虽然丰富，但仍存在客观性和公平性问题：

- 真实机器人平台是作者内部平台；
- WALL-WM 的数据与平台高度对齐；
- baseline 可能没有获得同等规模、同等平台匹配的数据和调参资源；
- 消融中有些因素耦合在一起，例如 event execution 和 VI-SA 同时变化，难以单独归因；
- 没有完整公开训练算力和训练细节；
- 没有充分展示不同数据规模、事件标注质量、caption 粒度、cluster sampler 对最终性能的独立影响；
- 缺少跨第三方机器人平台的大规模验证。

因此，论文实验展示了强结果，但严格意义上的公平比较和可复现性仍有限。

## 6. 主要结论与发现

论文主要结论包括：

- **事件级建模比固定 chunk 更适合对齐语言、视觉和动作**。
- 以动作事件为训练原子，可以更好保留视频基础模型的视觉—语义先验。
- 事件级 caption + 视频 + 动作配对，使未来视频预测和动作生成拥有一致语义边界。
- 多视角视频 DiT 与动作 DiT 的层级耦合可以同时提升：
  - 未来视频生成；
  - 动作控制；
  - 接触动态建模；
  - 多视角一致性。
- 真实机器人结果表明，event-mode WALL-WM 在大多数评估套件上获得最高平均 Task Progress：
  - Diverse Manipulation：75.86；
  - Reasoning Manipulation：71.60；
  - Dexterous Manipulation：32.00；
  - Generalization：53.75。
- 在视频生成评估中，WALL-WM 相比 Wan 系列提升了 embodied-relevant 指标，如运动质量、语义一致性和物理合理性。
- 对于推理和泛化任务，事件条件执行和跨视角交互对性能提升明显。
- 对于精密插入等灵巧任务，event mode 相比 unified scratch 的优势较小，说明低层接触精度仍是难点。

## 7. 优点与亮点

## 7.1 方法层面

- **问题定义清晰**：准确指出固定长度 chunk 与语言/视觉/动作多时间尺度之间的粒度错配。
- **事件作为学习原子**：比传统 chunk 更符合机器人任务结构。
- **统一视频和动作建模**：同时预测未来视频和动作，不只是 reactive policy。
- **保留视频模型先验**：通过冻结/单向耦合/零初始化跨视角分支，降低动作训练破坏视觉先验的风险。
- **多视角设计较完整**：
  - Camera RoPE；
  - cross-view attention；
  - sight-cone mask；
  - tube mask。
- **支持两种推理接口**：
  - event mode 适合 agent 规划；
  - unified mode 兼容传统固定 chunk 控制。
- **Staircase latent CoT** 兼顾推理能力和低延迟，避免逐 token CoT 的高成本。
- **数据组织方法扎实**：
  - 时间同步；
  - 多级 caption；
  - cluster-balanced sampling；
  - recovery data；
  - 无具身采集。

## 7.2 实验层面

- 不只做离线 benchmark，还做了真实机器人实验；
- Task Progress 指标比 success rate 更细粒度；
- 覆盖任务类型较多；
- 包含视频生成、3D awareness、多视角一致性和机器人控制多个维度；
- 对系统部署延迟、压缩和 FP8 做了工程讨论。

## 7.3 工程层面

- 提出 DMuon、kernel fusion、multi-event packing、通信计算 overlap 等大规模训练工程方案；
- 推理侧通过 distillation + FP8 + CUDA Graph 达到 10Hz；
- 体现了从预训练到真实机器人部署的完整系统视角。

---

## 8. 不足与局限

## 8.1 可复现性不足

论文没有明确披露：

- GPU 型号和数量；
- 训练时长；
- 总计算量；
- 详细超参数；
- 完整数据规模；
- 每个阶段的训练预算。

对于一个大规模 WAM 系统，这些缺失会影响外部复现和公平比较。

---

## 8.2 平台与数据偏差

作者也承认：

- 真实机器人评估在内部平台完成；
- WALL-WM 训练数据与该平台高度对齐；
- baseline 不一定有同等平台适配、数据规模和工程调参；
- 因此真实机器人性能可能包含平台对齐优势。

这使得结果虽然强，但不一定能直接外推到其他机器人形态和相机布局。

---

## 8.3 消融归因不够细

部分关键设计没有被完全拆开评估，例如：

- event-conditioned execution；
- VI-SA / cross-view attention；
- hierarchical caption；
- cluster-balanced sampling；
- recovery augmentation；
- Staircase latent CoT；
- asymmetric denoising anchor；
- VLM next-event predictor。

论文有一定消融，但许多因素仍然耦合，难以精确判断每个模块的独立贡献。

---

## 8.4 事件标注依赖较强

当前方法高度依赖：

- 事件边界；
- 多级 caption；
- action-boundary segmentation；
- caption 质量控制。

这带来两个问题：

- 标注和后处理成本高；
- 如果事件边界不准确，可能影响训练目标质量。

论文也在 Future Work 中指出，未来需要转向更自监督的事件发现和事件级预训练。

---

## 8.5 灵巧操作仍然困难

在 Dexterous Manipulation 中：

- WALL-WM event mode 平均分为 32.00；
- WALL-WM-U-Scratch 为 31.25；
- 两者差距很小。

这说明对于窄容差插入、精细接触、复杂手指操作等任务，事件级语义建模帮助有限，低层控制、力觉反馈和接触建模仍是瓶颈。

---

## 8.6 推理复杂度与系统依赖较高

虽然论文通过压缩达到 10Hz，但整体系统依赖复杂：

- 多塔 DiT；
- VLM reasoning；
- Staircase decoder；
- flow matching denoising；
- FP8 PTQ；
- DMD distillation；
- 自定义 kernel；
- CUDA Graph；
- 分布式优化。

这意味着部署门槛较高，不是一个轻量级 VLA 策略。

---

## 8.7 对外部 benchmark 的覆盖有限

论文主要真实机器人实验在内部场景中完成。虽然引用并对比了若干方法，但缺少：

- 标准公共真实机器人 benchmark；
- 跨实验室复现；
- 多 embodiment 外部平台验证；
- 公开统一评测协议下的大规模比较。

---

# 总体评价

WALL-WM 是一篇系统性很强的世界动作模型论文。它的核心贡献在于把机器人视频—动作学习的基本单位从固定长度 chunk 转换为 **语义事件**，并围绕这一思想构建了模型架构、数据生态、训练流程、推理模式和基础设施。论文在真实机器人和 embodied video generation 上都展示了强性能，尤其在推理操作和复杂场景泛化中优势明显。

不过，论文也存在大规模系统论文常见的问题：训练资源披露不足、内部平台优势明显、部分消融不够拆分、外部可复现性有限。总体而言，它更像是一篇提出“下一代通用 WAM 训练范式”的技术报告，而不仅是单一模型论文。
