## 摘要
视觉-语言-动作（VLA）模型的实时推理对于机器人控制至关重要。尽管视觉 Token 剪枝在加速推理方面展现出巨大潜力，但现有大多数方法主要基于浅层线索进行剪枝决策，因而存在丢弃深层所需视觉信息的风险。为解决这一问题，我们提出 SAFE-Pruner，这是一种即插即用的剪枝框架，将未来层的注意力线索纳入剪枝决策。具体而言，我们识别出语义注意力一致性，即 VLA 模型在不同执行步骤中倾向于将注意力概率质量集中在同一语义实体上。基于这一观察，我们设计了一种前瞻性策略，用于预测深层中的 Token 显著性，从而避免关键 Token 被过早移除，并实现更稳定的加速。我们进一步引入一种自适应子任务划分策略，以检测突发的注意力转移，从而提高预测精度和剪枝可靠性。在仿真和真实环境中的大量实验表明，我们的方法在成功率下降不到 1.7% 的情况下最高可实现 1.89 倍加速，并且相较于最先进方法最高提升 1.9%。

## Abstract
Real-time inference of vision-language-action (VLA) models is essential for robotic control. While visual token pruning has shown strong potential for accelerating inference, most existing methods mainly base pruning decisions on shallow-layer cues and risk discarding visual information required by deep layers. To address this issue, we propose SAFE-Pruner, a plug-and-play pruning framework that incorporates attention cues of future layers into pruning decisions. Specifically, we identify semantic attention consistency, the tendency that VLA models concentrate their attention probability mass on the same semantic entity across execution steps. Based on this observation, we design a forward-looking strategy to forecast the token saliency in deep layers, which prevents the premature removal of critical tokens and leads to more stable acceleration. We further introduce an adaptive subtask division strategy to detect abrupt attention shifts, thereby improving forecasting accuracy and pruning reliability. Extensive experiments in simulation and real-world settings demonstrate that our method achieves up to 1.89x speedup with a minimal degradation in success rate of less than 1.7%, while outperforming state-of-the-art methods by up to 1.9%.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想

SAFE-Pruner 的核心思想是：

> 不只根据当前浅层注意力决定剪枝，而是结合“当前浅层显著性”和“预测的未来深层显著性”进行 token 保留决策。

论文观察到两个关键现象：

- **层内阶段差异**  
  在同一控制 timestep 内：
  - 浅层注意力较宽泛、分散；
  - 深层注意力更集中，通常聚焦于任务关键区域；
  - 因此仅用浅层注意力剪枝容易误删深层需要的 token。

- **跨时间步的语义注意力一致性**  
  在同一子任务内，即使目标物体或机械臂位置发生变化，VLA 模型往往仍然关注具有相同任务语义的区域，例如夹爪、目标物体、容器等。  
  这使得模型可以利用历史关键帧的深层注意力来预测当前帧的深层显著性。

---

### 2.2 注意力显著性计算

在剪枝层 $\ell_s$，方法使用 action token 作为 query，视觉 token 作为 key，计算 query-to-visual attention。

- 设：
  - $H^{\ell_s}_{vis}$ 为视觉 token 隐状态；
  - $H^{\ell_s}_{query}$ 为查询 token 隐状态；
  - $N_h$ 为注意力头数；
  - $L_{query}$ 为 query token 数；
  - $L_{vis}$ 为视觉 token 数。

- 缩放点积注意力为：

\[
A^{\ell_s} = Softmax\left(\frac{Q^{\ell_s}(K^{\ell_s})^\top}{\sqrt{d}}\right)
\]

- 对所有注意力头和 query token 聚合，得到每个视觉 token 的浅层显著性：

\[
S^{\ell_s}_v =
\frac{1}{N_h L_{query}}
\sum_{h=1}^{N_h}
\sum_{t=1}^{L_{query}}
A^{\ell_s}_{h,t,v}
\]

该分数越高，表示视觉 token 越被 action token 关注。

---

### 2.3 未来感知 token 剪枝

SAFE-Pruner 在若干关键 timestep 上不进行剪枝，而是完整推理并缓存每一层的 saliency map。

对于当前非关键 timestep $t$：

1. **选择参考关键帧**
   - 从过去关键帧中选择最近的一个：

\[
\tau_{ref} = \max \{\tau \in T_{key}: \tau < t\}
\]

2. **建立当前帧与参考帧的 token 对应关系**
   - 使用视觉 token embedding 的余弦相似度；
   - 对当前 token 找到参考关键帧中语义最接近的 token；
   - 将参考帧中缓存的层级 saliency 转移到当前帧。

3. **预测深层显著性**
   - 对深层集合 $L_d$ 中的若干层显著性取平均：

\[
\hat{S}^{L_d}_t =
\frac{1}{|L_d|}
\sum_{\ell \in L_d}
\hat{S}^{\ell}_t
\]

4. **融合浅层显著性与预测深层显著性**
   - 最终 token 重要性分数为：

\[
s_t =
(1-\lambda)S^{\ell_s}_t
+
\lambda \hat{S}^{L_d}_t
\]

其中 $\lambda$ 是融合权重，论文实现中设为 $0.5$。

5. **Top-K 保留**
   - 根据融合后的 $s_t$ 选择 top-K 视觉 token；
   - 其余视觉 token 被剪除；
   - 后续 Transformer 层只处理保留 token。

---

### 2.4 自适应子任务划分与关键帧选择

固定间隔选关键帧存在问题：当任务从“抓取”切换到“放置”等子任务时，注意力语义可能突变，历史关键帧的 saliency prior 会变得过时。

因此，SAFE-Pruner 引入自适应子任务划分：

- 使用当前浅层 saliency $S^{\ell_s}_t$ 和预测浅层 saliency $\hat{S}^{\ell_s}_t$ 的余弦相似度：

\[
\kappa_t =
\frac{
\langle S^{\ell_s}_t, \hat{S}^{\ell_s}_t \rangle
}{
\|S^{\ell_s}_t\|_2
\|\hat{S}^{\ell_s}_t\|_2
}
\]

- 若：

\[
\kappa_t < \gamma
\]

则认为当前 timestep 出现注意力模式突变，是新的子任务边界。

- 对于这种关键 timestep：
  - 不剪枝；
  - 执行完整 token 推理；
  - 缓存新的层级 saliency；
  - 更新关键帧集合。

---

### 2.5 整体算法流程

SAFE-Pruner 的推理流程可以概括为：

1. 第一个 timestep 执行完整推理，缓存各层 saliency；
2. 对每个后续 timestep：
   - 计算浅层 saliency；
   - 从最近关键帧预测当前帧各层 saliency；
   - 计算当前 saliency 与预测 saliency 的相似度；
   - 若相似度低于阈值，则判定为子任务边界，执行完整推理并更新缓存；
   - 否则融合浅层显著性和预测深层显著性；
   - 根据融合分数剪枝视觉 token；
   - 使用剪枝后的 token 执行动作生成。

该方法是 **training-free、plug-and-play** 的，不需要重新训练 VLA 模型，也不改变模型结构。

总体来看，实验相对充分：

- 覆盖了多个 VLA 架构；
- 包含仿真和真实机器人；
- 同时报告成功率、FLOPs 和 latency；
- 与多个已有剪枝方法对比；
- 进行了关键组件消融；
- 对核心假设“语义注意力一致性”做了定量验证。

---

### 5.3 公平性与客观性

论文声称对 baseline 进行了充分超参数调优，并在相同实验设置下比较，整体上具有较好的公平性。

但仍有几点需要注意：

- 不同方法的工程实现可能影响 latency；
- 真实机器人实验只和 FastV 做了对比，没有覆盖所有 baseline；
- 真实任务每个仅 20 次试验，统计稳定性有限；
- 部分超参数如剪枝率、阈值 $\gamma$ 是通过 grid search 调得，可能对特定 benchmark 有一定适配性。

## 6. 主要结论与发现

- **语义注意力一致性是 VLA 模型中的重要现象**  
  在连续机器人操控过程中，同一子任务内模型往往持续关注相同语义实体，而不只是固定空间位置。

- **浅层显著性不足以可靠指导高比例剪枝**  
  当剪枝率超过 70% 后，仅基于浅层 attention 的方法容易显著丢失深层核心 token。

- **未来感知剪枝能改善准确率与效率平衡**  
  SAFE-Pruner 通过预测深层 saliency，在高剪枝率下仍能保留任务关键 token。

- **自适应子任务划分提升可靠性**
  通过检测 attention shift，方法能够避免在子任务切换时使用过时的历史注意力先验。

- **整体性能优于现有方法**
  在多个 benchmark 上，SAFE-Pruner 实现了较低 FLOPs 和 latency，同时保持接近 vanilla 的成功率。

- **实际加速效果明显**
  论文报告最高可达到约 **1.89×** 加速，成功率下降小于 **1.7%**。

---

## 7. 优点与亮点

- **问题定义清晰**
  - 明确指出现有 token 剪枝方法的“短视性”；
  - 将 VLA 推理中的层级注意力演化纳入剪枝决策。

- **核心观察有启发性**
  - 提出并验证了“语义注意力一致性”现象；
  - 该观察不仅用于剪枝，也可能启发未来 VLA 模型压缩、缓存和动态推理研究。

- **方法简单、即插即用**
  - 不需要重新训练；
  - 不修改模型结构；
  - 可部署在多种 VLA 架构上。

- **兼顾浅层与深层信息**
  - 不是单纯依赖浅层 attention；
  - 通过历史关键帧预测当前深层 saliency，降低误删关键 token 风险。

- **考虑子任务切换**
  - 自适应关键帧选择比固定间隔更合理；
  - 能处理长时程任务中注意力模式突变的问题。

- **实验覆盖较全面**
  - 包含 LIBERO、SIMPLER 和真实机器人；
  - 覆盖 OpenVLA、OpenVLA-OFT、CogACT、π0.5 等多种架构；
  - 对比了多种主流剪枝方法。

- **效率收益实际可见**
  - 不仅报告 FLOPs，也报告实际 latency；
  - 在真实机器人上也展示了明显延迟下降。

---

## 8. 不足与局限

- **依赖语义注意力一致性假设**
  - 方法成立的关键前提是相邻 timestep 或同一子任务内 attention 语义较稳定；
  - 在快速变化、高动态、多目标频繁切换的任务中，该假设可能不稳定。

- **子任务边界检测较简单**
  - 使用浅层 saliency 的余弦相似度和固定阈值 $\gamma$；
  - 阈值可能对模型、任务、场景敏感；
  - 更复杂的子任务切换可能不一定能被该指标准确捕捉。

- **真实机器人实验规模有限**
  - 真实任务只有 3 类；
  - 每个任务 20 次试验；
  - 对比方法只包括 FastV 和原始 π0.5，未覆盖全部 baseline；
  - 统计显著性和泛化结论仍需更大规模验证。

- **对复杂开放世界任务验证不足**
  - LIBERO 和 SIMPLER 虽然标准化，但仍属于受控环境；
  - 对遮挡、光照剧烈变化、多个相似物体、动态干扰等现实挑战覆盖有限。

- **预测机制可能引入额外开销**
  - 需要维护关键帧缓存；
  - 需要进行 token embedding 相似度匹配；
  - 虽然整体加速明显，但论文没有非常详细地拆分该模块自身的开销。

- **超参数依赖仍存在**
  - 包括剪枝层 $\ell_s$、深层集合 $L_d$、剪枝率 $\rho$、融合权重 $\lambda$、子任务阈值 $\gamma$；
  - 不同模型和数据集可能需要调参。

- **缺少训练阶段资源报告**
  - 方法本身无需训练，但真实机器人任务涉及 π0.5 微调；
  - 文中没有详细报告微调成本，影响复现和资源评估。
