## 摘要
在机器人操作与导航等具身视觉语言决策任务中，视觉语言模型（VLMs）和视觉语言动作模型（VLAs）是具有不同优势的强大工具：VLMs 更擅长长期规划，而 VLAs 更擅长反应式控制。然而，它们的性能受到同一感知瓶颈的限制：由于模型无法区分与任务相关的物体和干扰物，因而会产生视觉幻觉。原则上，准确识别并聚焦关键物体，同时过滤无关物体，是突破这一限制的关键。一种直接的解决方案是一步式聚焦：直接关注关键物体。然而，这种方法被证明效果不佳，因为有效聚焦本质上需要深层场景理解。为此，我们提出 SceneDiver，一种面向 VLMs 的由粗到细的焦点计划生成方法，利用其长期规划能力，首先构建整体场景图以建立初步理解，随后通过识别、理解与分析的迭代循环，将任务逐步分解为更简单的子问题。为实现反应式控制，我们还设计了一个轻量级适配器，用于将审慎的聚焦能力蒸馏到 VLAs 中。在标准具身 AI 基准上的评估表明，我们的方法在 VLMs 和 VLAs 中均显著减少了视觉幻觉，同时在需要快速执行的任务中保持了计算效率。我们的代码和数据已发布于：https://future-item.github.io/SceneDiver。

## Abstract
In embodied vision-language decision making tasks such as robotic manipulation and navigation, Vision-Language and Vision-Language-Action Models (VLMs & VLAs) are powerful tools with different benefits: VLMs are better at long-term planning, while VLAs are better at reactive control. However, their performance is limited by the same perceptual bottleneck: visual hallucinations arise due to the models' inability to distinguish task-relevant objects from distractors. In principle, accurate identification and focus on critical objects while filtering out irrelevant ones is the key to break this limitation. A straightforward solution is one-step focus: directly attending to essential objects. However, this approach proves ineffective because effective focus inherently requires deep scene understanding. To this end, we propose SceneDiver, a coarse-to-fine focus plan generation method for VLMs leveraging their long-term planning abilities, that first constructs a holistic scene graph to establish initial comprehension, then progressively decomposes the task into simpler sub-problems through an iterative cycle of recognition, understanding, and analysis. To enable reactive control, we also design a lightweight adapter for distilling the deliberate focus ability into VLAs. Evaluations on standard embodied AI benchmarks confirm that our method substantially reduces visual hallucinations for both VLMs and VLAs, while preserving computational efficiency in tasks requiring fast execution. Our code and data are released at: https://future-item.github.io/SceneDiver.

## 2. 方法论：SceneDiver 的核心思想与技术细节

### 2.1 核心思想

- 论文提出 **SceneDiver**，一种由粗到细的焦点计划生成方法。
- 它不是让模型一次性从整张图中找目标，而是让模型像“进入场景中探索”一样：
  1. 先构建整体场景图，获得全局结构理解；
  2. 再基于任务指令推理出可能相关的局部子场景；
  3. 对每个局部子场景进行放大、验证、搜索；
  4. 最终生成任务相关物体集合；
  5. 用该集合生成焦点图，对图像进行软调制；
  6. 将调制后的图像输入 VLM 或 VLA 进行最终决策。

### 2.2 粗粒度阶段：场景图推理

- 使用 **OvSGTR** 作为场景图生成后端。
- 场景图中：
  - 节点表示物体；
  - 边表示空间关系；
  - 每个物体包含：
    - `<ref>`：唯一物体标识；
    - `<box>`：物体边界框；
    - `<pred>`：物体间关系谓词。

- 场景图以文本格式输入 VLM，让 VLM 根据任务指令在图结构上进行推理。
- 这一阶段的目标是：
  - 从复杂全局场景中筛选出可能相关的节点；
  - 将整体场景分解为若干较简单的局部子场景；
  - 为后续细粒度验证提供候选区域。

- 论文强调，场景图不是最终真值，而是推理引导：
  - 如果图和图像不一致，VLM 可以选择邻近物体；
  - 可以丢弃无关目标；
  - 可以保留不确定候选进入下一阶段。

### 2.3 细粒度阶段：验证与探索

粗阶段给出候选子场景后，SceneDiver 会让 VLM 逐个“靠近观察”。

该阶段包含三类智能体式探索行为：

- **目标匹配与确认**
  - 如果局部视野中的对象与任务和粗计划一致，则确认其为有效候选。
  - 例如确认某个杯子、积木或导航目标确实存在。

- **语义缩放 / Semantic Zoom**
  - 如果目标存在但视觉证据不清晰，例如类别、颜色或形状模糊，则进一步缩小视野、放大局部区域。
  - 用于解决细粒度属性判断问题。

- **局部空间搜索 / Local Spatial Search**
  - 如果场景图中候选节点与图像位置存在偏差，或目标被场景图漏检，模型会在邻近区域继续搜索。
  - 这使得方法对场景图错误具有一定恢复能力。

最终该阶段得到一个任务相关候选物体集合：

- 记为 \( C \)，其中包含被确认需要关注的目标或关键环境对象。

### 2.4 焦点调制：将候选物体转化为图像提示

- 对候选集合 \( C \) 中的边界框进行栅格化，生成像素级焦点分数图：

\[
s_{u,v} = I[\exists k \in C : (u,v) \in b_k]
\]

其中：
- \( s_{u,v} \in [0,1] \)；
- \( b_k \) 是第 \( k \) 个候选物体的边界框；
- 如果像素属于候选物体区域，则焦点值为 1，否则为 0。

- 然后对图像进行软调制，而不是简单裁剪或遮挡背景。

调制过程包括两步：

\[
I_{dim} = I \odot (\beta + (1-\beta)s)
\]

\[
I_{out} = s \odot I_{dim} + (1-s) \odot B_\sigma(I_{dim})
\]

含义是：
- 目标区域保持清晰和明亮；
- 背景区域被适度变暗并高斯模糊；
- \(\beta\) 是可见性下限，避免背景完全消失；
- 这样既保留场景上下文，又降低干扰物对模型注意力的影响。

### 2.5 SceneDiver Adapter：蒸馏到 VLA 的轻量适配器

由于完整的场景图推理和多轮探索计算成本较高，论文进一步提出 **SceneDiver Adapter**，用于将显式焦点规划能力蒸馏到 VLA 中。

- 插入位置：
  - 位于 VLA 的视觉 projector 之后；
  - 工作在视觉-语言对齐空间中。

- 核心模块包括：

#### Task-Guided Slot Attention

- 将视觉 token 分解为若干对象中心的 slot。
- 与标准 Slot Attention 不同，本文使用任务条件初始化：
  - 先将任务文本 token 池化成全局任务向量 \(v_{task}\)；
  - 再用该向量初始化 slot 的均值；
  - 使 slot 更倾向于捕捉任务相关实体，而非任意纹理。

公式形式为：

\[
S_{init} \sim N(\mu(v_{task}) + \delta, \sigma_{global})
\]

#### Scene-Aware Mask Prediction Module

- 分为粗细两级：
  - 粗阶段预测每个 slot 的任务相关性得分；
  - 细阶段将 slot 语义反传到 patch 级别，生成空间掩码。

掩码预测公式为：

\[
M_{pred} = \sigma \left( \sum_k r_k \cdot A_{k,:} + \alpha \cdot \Delta_{patch} \right)
\]

其中：
- \(r_k\)：第 \(k\) 个 slot 的相关性；
- \(A_{k,:}\)：slot 到 patch 的注意力；
- \(\Delta_{patch}\)：patch 级残差修正；
- \(\alpha\) 初始接近 0，使训练早期更依赖 slot 级预测。

#### 蒸馏目标与安全机制

- 使用 Hungarian Matching 将 slot 与场景图物体对齐。
- 训练包含两个主要损失：
  - **Structure Loss**：监督 slot 学习结构化场景表示；
  - **Mask Loss**：监督掩码预测模块学习空间定位。
- 引入基于熵的不确定性门控：
  - 如果模型对某些区域过于不确定，则绕过掩码，直接输入原始图像；
  - 避免错误遮蔽导致灾难性失败。

  - 目标物体更小；
  - 更容易被遮挡；
  - 更容易与相似物体混淆。

- 包含四类子任务：
  1. **Base**：直接指令；
  2. **Common-Sense**：常识推理式指令；
  3. **Complex-Instruction**：长文本复杂指令，需要提取真正目标；
  4. **Visual-Appearance**：通过颜色、形状、外观描述目标。

- 对比模型包括开源和闭源 VLM：
  - Qwen2.5-VL-7B-Instruct-AWQ；
  - Qwen2.5-VL-32B-Instruct-AWQ；
  - InternVL2.5-8B；
  - gpt-4o-mini；
  - gemini-2.5-flash；
  - doubao-seed-1.6-flash。

- 对比方法包括：
  - **Base Model**：原始模型直接决策；
  - **SoM / Set-of-Mark Prompting**：基于视觉标记的物体提示；
  - **Multi-Res**：多分辨率输入；
  - **VCD / Visual Contrastive Decoding**：视觉对比解码，缓解幻觉；
  - **Thinking**：具备思考模式的模型；
  - **Ours / SceneDiver**。

### 3.3 VLA 鲁棒性实验：LIBERO-plus

- 使用 **LIBERO-plus** 评估 VLA 的鲁棒性。
- 基础模型：
  - **OpenVLA-OFT**。

- 训练数据：
  - 只使用原始、无干扰的 LIBERO 数据；
  - 包括 libero spatial、libero object、libero goal、libero long 的混合训练。

- 测试场景包含五类干扰：
  1. Objects Layout：物体布局变化；
  2. Camera Viewpoints：相机视角变化；
  3. Robot Initial States：机器人初始状态变化；
  4. Background Textures：背景纹理变化；
  5. Light Conditions：光照变化。

- 测试任务套件：
  - Spatial；
  - Object；
  - Goal；
  - Long。

### 3.4 消融实验与压力测试

论文还设计了多类附加实验：

- **Ablation 1：仅使用文本场景图**
  - 检查场景图作为文本辅助输入是否足够。

- **Ablation 2：直接聚焦 vs 分步聚焦**
  - 检查是否需要粗到细的两阶段规划。

- **Ablation 3：去掉细粒度验证**
  - 检查细粒度探索是否必要。

- **场景图压力测试**
  - 人为删除部分场景图节点；
  - 删除比例包括 10%、20%、30%、50%；
  - 评估恢复率、幻觉率和漏检率。

- **感知错误分析**
  - 统计漏检、误检、属性绑定错误、多实例错误、目标漏检等。

- **Vanilla EmbodiedBench 补充实验**
  - 在原始导航和操作任务上验证方法仍有提升。

- **端到端 baseline 对比**
  - 在 LIBERO-plus 上比较 SceneDiver Adapter 与直接端到端预测焦点的模型。

## 4. 资源与算力

- 论文明确报告了 **推理延迟测试所用 GPU**：
  - 使用单张 **NVIDIA RTX 4090 GPU**；
  - 对 SceneDiver Adapter 在 OpenVLA-OFT 上的推理延迟进行测试；
  - 测试了 10,000 次 action generation。

- 延迟结果：
  - 总动作生成平均时间：114.45 ms；
  - Slot Attention 平均耗时：2.18 ms；
  - MaskNet 平均耗时：0.83 ms；
  - SceneDiver Adapter 总额外开销约 3.01 ms；
  - 占总动作生成时间约 2.64%。

- 论文没有明确说明：
  - 训练使用了多少张 GPU；
  - 训练总时长；
  - 训练显存消耗；
  - 场景图生成器和 adapter 的完整训练成本；
  - 不同模型调用闭源 API 的成本。

因此，论文在**推理开销**方面报告较清楚，但在**训练算力成本**方面信息不完整。

## 6. 主要结论与发现

- **SceneDiver 能显著提升具身决策成功率**
  - 在机器人操作任务中，不同 VLM 的成功率普遍提升约 10%–15%。
  - 例如 gemini-2.5-flash 从 38.7% 提升到 46.7%。

- **在复杂导航任务中，SceneDiver 优于现有视觉聚焦和幻觉缓解方法**
  - 在开源和闭源模型上均取得更高成功率；
  - 对常识指令、复杂指令和视觉外观描述均有效；
  - 相比 SoM、Multi-Res、VCD 和 Thinking 模型都有优势。

- **仅靠更强推理并不足以解决感知瓶颈**
  - Thinking-enhanced 模型仍然不如 SceneDiver；
  - 说明长链推理不能替代可靠的视觉聚焦和场景验证。

- **SceneDiver Adapter 能增强 VLA 鲁棒性**
  - 在 LIBERO-plus 中，OpenVLA-OFT 加入 adapter 后在所有任务套件和干扰类型上均有提升；
  - 特别是在 Long-horizon 任务、物体布局变化、光照变化和背景纹理变化下提升明显。

- **计算开销较低**
  - Adapter 推理额外开销约 2.64%；
  - 适合实时控制任务。

- **细粒度验证是关键组件**
  - 消融实验显示，单纯输入场景图或直接聚焦效果有限；
  - 由粗到细的验证和探索机制是性能提升的关键。

## 7. 优点与亮点

- **问题定位清晰**
  - 论文准确指出 VLM 和 VLA 在具身决策中的共同瓶颈是任务相关感知失败，而不仅是语言推理不足。

- **方法设计符合具身任务需求**
  - 先全局理解，再局部探索；
  - 与人类观察复杂场景的方式较接近；
  - 特别适合存在遮挡、干扰物和相似物体的场景。

- **粗到细机制设计合理**
  - 场景图提供结构化先验；
  - VLM 负责推理和纠错；
  - 细阶段允许语义缩放和局部搜索，增强对场景图错误的鲁棒性。

- **不是硬裁剪，而是软调制图像**
  - 保留背景上下文；
  - 降低无关区域干扰；
  - 避免完全遮蔽导致空间关系丢失。

- **兼顾 VLM 和 VLA**
  - 对 VLM 提供显式焦点计划；
  - 对 VLA 通过 adapter 蒸馏隐式聚焦能力；
  - 使方法既能用于高层规划，也能用于低延迟控制。

- **实验覆盖面较广**
  - 包括机器人操作、房间导航、LIBERO-plus 鲁棒性；
  - 包含开源和闭源模型；
  - 有消融、压力测试和延迟分析。

- **推理效率较好**
  - Adapter 只增加约 3 ms 延迟；
  - 对实时机器人控制较友好。

## 8. 不足与局限

- **依赖场景图生成质量**
  - 虽然论文通过压力测试证明对节点丢失有一定恢复能力，但场景图仍是核心先验。
  - 在真实世界中，动态物体、遮挡严重、光照复杂、类别开放度更高时，场景图质量可能下降。

- **完整训练成本未充分报告**
  - 论文只明确报告了 RTX 4090 上的推理延迟；
  - 未说明训练 adapter、训练或微调场景图生成器所需 GPU 数量、训练时长和总体成本。

- **真实机器人验证不足**
  - 主要实验集中在 MuJoCo、AI2-THOR / EmbodiedBench 和 LIBERO-plus 等模拟或基准环境；
  - 尚未充分证明在真实机器人硬件、真实室内场景中的稳定性。

- **改造 benchmark 可能带来选择偏差**
  - 作者构建了更困难的导航变体，这有助于检验感知瓶颈；
  - 但任务难度和场景选择由作者控制，可能更符合 SceneDiver 的优势设定。

- **多轮 VLM 推理成本较高**
  - 显式 SceneDiver 需要场景图构建、粗推理、细粒度验证和多次局部探索；
  - 对高频在线决策不够友好，因此才需要 adapter 蒸馏。
  - 但 adapter 的训练和泛化依赖显式流程提供监督。

- **对动态场景支持有限**
  - 论文也承认，对于快速运动物体或高度动态环境，场景图生成和焦点计划可能滞后。
  - 未来需要结合动态跟踪或时序记忆机制。

- **复杂长程任务仍有扩展空间**
  - SceneDiver 提升了长程任务中的感知稳定性；
  - 但它本身不是完整的层级任务规划框架；
  - 对更复杂的多阶段任务，可能还需要与层级规划、记忆模块或世界模型结合。

- **闭源模型实验复现性有限**
  - gpt-4o-mini、gemini-2.5-flash、doubao 等闭源模型版本和推理策略可能变化；
  - 具体 prompt、采样参数和 API 行为会影响结果复现。
