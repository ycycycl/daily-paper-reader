## 2. 方法论：核心思想与关键技术

## 2.1 总体框架

- Cosmos 3 使用统一的 **Mixture-of-Transformers（MoT）** 架构。
- 模型同时包含两个功能路径：
  - **Reasoner / 自回归理解路径**：处理语言、ViT 编码视觉 token，用于多模态理解和文本生成。
  - **Generator / 扩散生成路径**：处理 VAE 编码的图像、视频、音频和动作 token，用于去噪生成。
- 推理时：
  - 语言通过 next-token prediction 生成；
  - 图像、视频、音频、动作通过 rectified flow / diffusion denoising 生成。

---

## 2.2 模态编码器

### 视觉编码

- **理解用视觉编码器**：
  - 使用 ViT，patch size 为 16×16；
  - 经过 MLP 合并 2×2 token 并投影到 Transformer 隐空间；
  - ViT 与主干共同训练。
- **生成用视觉编码器**：
  - 使用 Wan2.2-TI2V-5B 的视频 VAE；
  - 时间压缩 4×，空间压缩约 32×32；
  - VAE 冻结，仅训练投影层和生成主干。

### 音频编码

- 使用 audio VAE；
- 输入为 48 kHz 立体声音频；
- hop size 为 1920，因此约 25 个 audio token / 秒；
- audio VAE 冻结。

### 动作编码

- 论文将 **action** 作为核心模态，而非附属控制变量。
- 支持多类具身形态：
  - 自动驾驶车辆；
  - 相机运动；
  - 单臂 / 双臂机器人；
  - 人形机器人；
  - egocentric human motion。
- 动作被映射为统一几何表示：
  - ego pose；
  - effector pose；
  - grasp state。
- 姿态变化使用相对 SE(3) 变换：
  - 给定连续位姿 \(T_{t-1}\)、\(T_t\)，相对运动为  
    \[
    \Delta T_t = T_{t-1}^{-1} T_t
    \]
  - 平移用 3D 表示，旋转用 6D rotation representation。
- 不同具身形态动作维度不同，因此使用 **domain-aware projection**：
  \[
  z = W^{(k)}_{\text{in}}x + b^{(k)}_{\text{in}}
  \]
  \[
  \hat{x} = W^{(k)}_{\text{out}}z + b^{(k)}_{\text{out}}
  \]
  其中 \(k\) 表示动作域或具身类型。

## 2.3 Token 排布与生成模式

Cosmos 3 将输入序列组织为两个子序列：

1. **AR subsequence**
   - 包含语言 token 和 ViT 视觉 token；
   - 用于理解和推理；
   - 自回归因果注意力。

2. **DM subsequence**
   - 包含 VAE 图像 / 视频 token、音频 token、动作 token；
   - 用于扩散式去噪生成；
   - 使用双向全注意力，并可关注 AR 上下文。

支持的生成模式包括：

- Language / VLM 模式；
- Text-to-Image；
- Text-to-Video；
- Text-to-Video + Audio；
- Image-to-Video；
- Video-to-Video continuation；
- Video transfer，例如 edge、depth、segmentation、blur、world-scenario-map 到 RGB 视频；
- Action 模式：
  - **Forward dynamics**：给定视频上下文和动作，预测未来视觉状态；
  - **Inverse dynamics**：给定视觉状态转移，推断动作；
  - **Policy / joint video-action prediction**：同时预测动作和未来视频。

## 2.4 Mixture-of-Transformers 架构

- 每个 Transformer decoder layer 内包含两套参数：
  - Reasoner tower；
  - Generator tower。
- 两个 tower 结构相同，但参数独立。
- 两者均从预训练 VLM 初始化，使生成路径继承视觉语言语义能力。

### 双流联合注意力

- AR token 只对 AR token 做因果自注意力：
  \[
  O_{\text{AR}} = \text{Attn}_{\text{causal}}(Q_{\text{AR}}, K_{\text{AR}}, V_{\text{AR}})
  \]
- DM token 对 AR + DM token 做全注意力：
  \[
  O_{\text{DM}} = \text{Attn}_{\text{full}}(Q_{\text{DM}}, [K_{\text{AR}}; K_{\text{DM}}], [V_{\text{AR}}; V_{\text{DM}}])
  \]
- 关键设计点：
  - 生成路径可以读取理解路径的上下文；
  - 理解路径不被生成 token 反向影响；
  - 保留自回归语言能力，同时实现扩散生成条件化。

---

## 2.5 多模态位置编码

- 使用扩展版 **3D MRoPE**，为 token 分配 \((t, h, w)\) 坐标。
- 语言 token 使用 \(t=h=w\)，退化为 1D RoPE。
- 视频 token 使用时间、高度、宽度三维坐标。
- 音频和动作 token 仅使用时间坐标，空间坐标设为 0。
- 为对齐不同 FPS、音频采样率和动作频率，引入 **absolute temporal modulation**：
  \[
  \delta t = \frac{\text{TPS}_{\text{base}}}{\text{TPS}}
  \]
  其中 TPS 表示每秒时间 token 数。
- 论文中设定：
  \[
  \text{TPS}_{\text{base}} = \frac{24}{4} = 6
  \]
  因为视频 VAE 时间压缩率为 4，且 24 FPS 是基础帧率。
- 还在 AR 与 DM token 之间插入固定 temporal gap，防止文本和视觉 token 位置过近导致视频初帧过饱和和 checkerboard artifact。

## 2.6 训练目标

- Generator 使用 **rectified flow matching**。
- 对目标 latent \(x_0\)，噪声 latent 定义为：
  \[
  x_\sigma = \sigma \epsilon + (1-\sigma)x_0
  \]
  其中 \(\epsilon \sim \mathcal{N}(0, I)\)。
- 模型预测速度：
  \[
  v^* = \epsilon - x_0
  \]
- 损失为 masked MSE，条件 token 不参与 loss。
- 图像、视频、音频、动作各模态可独立采样噪声时间。

## 4. 资源与算力

论文披露了较多训练与基础设施细节。

## 4.1 模型规模

Cosmos 3 有三个规模：

- **Cosmos3-Edge**
  - 约 4B MoT 参数；
  - 基于 2B dense transformer；
  - 28 层，hidden dim 2048。
- **Cosmos3-Nano**
  - 约 16B MoT 参数；
  - 基于 8B dense transformer；
  - 36 层，hidden dim 4096。
- **Cosmos3-Super**
  - 约 64B MoT 参数；
  - 基于 32B dense transformer；
  - 64 层，hidden dim 5120。

## 4.2 Generator 训练算力

论文明确说明：

- Generator pre-training：
  - Cosmos3-Nano：
    - 使用 **1024 NVIDIA GB200 GPUs**；
    - 训练 **31.05T tokens**。
  - Cosmos3-Super：
    - 使用 **2048 NVIDIA GB200 GPUs**；
    - 训练 **17.86T tokens**。
- Generator mid-training：
  - Cosmos3-Nano：
    - 使用 **1024 NVIDIA GB200 GPUs**；
    - 训练 **2.4T tokens**。
  - Cosmos3-Super：
    - 使用 **2048 NVIDIA GB200 GPUs**；
    - 训练 **1.9T tokens**。

## 4.3 吞吐指标

论文报告了 GB200 上的训练吞吐：

- Cosmos3-Nano：
  - iteration time：约 7.1 秒；
  - 约 507 iterations/hour；
  - per-GPU 约 520 TFLOPS；
  - MFU 约 0.23。
- Cosmos3-Super：
  - iteration time：约 19.5 秒；
  - 约 185 iterations/hour；
  - per-GPU 约 673 TFLOPS；
  - MFU 约 0.30。

## 4.4 其他训练资源

- Reasoner 训练使用 AdamW，预训练两轮，最大序列长度 16K。
- 机器人策略后训练中：
  - Cosmos3-Nano-Policy-DROID 推理部署在 **2 张 NVIDIA RTX Pro 6000 GPUs** 上。
- Edge LLM：
  - 从零训练 15T tokens；
  - 长上下文扩展阶段训练 90B tokens；
  - 但该部分未完整披露使用 GPU 数量和总训练时长。

## 4.5 未明确披露的信息

- 论文没有完整披露所有阶段的 wall-clock 训练时间。
- Reasoner 预训练 / SFT 的 GPU 数量和总训练耗时未在主要内容中完整给出。
- 大规模数据处理和合成数据生成也未给出完整总算力成本。

## 6. 主要结论与发现

- **统一全模态架构可行**：
  - Cosmos 3 在一个架构中同时完成理解、生成、动作预测和世界模拟。
- **Reasoner 能增强 Generator**：
  - 消融显示，用 Cosmos3 Reasoner 初始化理解路径比 Qwen3-VL 更有利于 Physical AI 领域生成。
- **动作作为核心模态是有效的**：
  - 统一 action token 支持 forward dynamics、inverse dynamics 和 policy；
  - 中训练后的 MT-init 在多个动作任务上显著优于 PT-init。
- **跨动作域存在迁移和协同**：
  - egocentric motion、camera motion、robotics、autonomous vehicle 之间存在可迁移的动作先验；
  - egocentric human data 可帮助机器人动作适应。
- **合成数据对 Physical AI 有价值**：
  - SDG 数据覆盖真实预训练分布中的长尾区域；
  - 不同合成数据源提升不同能力；
  - 混合 SDG-All 通常更平衡。
- **后训练能有效专门化模型**：
  - Cosmos3-Super-Text2Image 在开源 T2I 排行中领先；
  - Cosmos3-Super-Image2Video 在开源 I2V 排行中领先；
  - Cosmos3-Nano-Policy-DROID 在 RoboLab 和 RoboArena 上表现强。
- **视频、音频与动作联合建模有助于物理一致性**：
  - Cosmos 3 在 Physics-IQ、HUE、SoundBench、RBench 等任务上展现较强物理合理性与时间一致性。

## 7. 优点与亮点

## 7.1 方法设计亮点

- **统一性强**：
  - 将 VLM、视频生成、音视频生成、world-action model、forward / inverse dynamics 统一为一个序列建模框架。
- **MoT 架构清晰有效**：
  - Reasoner 与 Generator 参数分离但通过注意力交互；
  - 保留语言自回归能力，同时支持扩散生成。
- **动作模态设计较系统**：
  - 使用共享几何组件统一不同具身动作；
  - domain-aware projection 兼容不同动作维度；
  - 同时支持动作条件生成、动作推断和策略预测。
- **位置编码考虑真实时间轴**：
  - 通过 FPS / TPS modulation 对齐视频、音频和动作的物理时间。
- **训练课程完整**：
  - Reasoner 预训练 + SFT；
  - Generator 预训练 + 中训练 + 后训练；
  - 逐步引入图像、视频、音频、动作、transfer control。
- **Prompt upsampling 设计实用**：
  - 使用结构化 JSON prompt 将稀疏用户意图转换为高密度场景规格；
  - 对图像、视频、音频和动作生成都有帮助。

## 7.2 实验设计亮点

- **评测覆盖极广**：
  - 从通用多模态理解到真实机器人执行，覆盖 Physical AI 的多个关键环节。
- **真实机器人验证有说服力**：
  - RoboArena 和 RoboLab 结果说明模型不仅是生成器，也可作为策略模型基础。
- **基础设施贡献明显**：
  - SILA 数据处理平台；
  - Joint DataLoader；
  - 两路 flat attention；
  - Ulysses context parallelism；
  - tokenizer AOT 编译；
  - 异步 checkpointing；
  - vLLM-Omni serving。
- **开源资源丰富**：
  - 代码；
  - 权重；
  - 合成数据集；
  - benchmark；
  - 推理与服务框架。

## 8. 不足与局限

## 8.1 可复现性限制

- 虽然论文开源了大量资源，但训练依赖极大规模数据和算力：
  - 数千张 GB200；
  - 数十万亿 token；
  - 亿级图像和视频；
  - 大量内部数据。
- 外部研究者很难完整复现训练过程。

## 8.2 数据偏差与内部数据依赖

- 许多关键能力来自内部策划数据、专有 driving logs、内部机器人或 egocentric 数据。
- 公开数据与内部数据比例、过滤细节、潜在分布偏差难以完全审计。
- 合成数据虽然补充长尾，但也可能引入 sim-to-real gap。

## 8.3 评测偏差风险

- 一些评测由作者自建，可能更贴合 Cosmos 3 的训练分布或设计目标。
- VLM-as-Judge 可能存在偏差，尤其在物理合理性、音视频同步、复杂空间关系上。
- 使用 Claude / GPT / Gemini 进行 prompt upsampling 或评测，可能使不同模型对比受到 prompt 工程影响。
- 闭源模型的设置不可完全控制，公平性存在天然限制。

## 8.4 模型复杂度与部署成本

- Cosmos3-Super 规模巨大，实际部署成本高。
- 虽然有 Nano 和 Edge，但论文主要开放和重点评测的是 Nano / Super。
- 实时机器人策略虽然已在 DROID 上部署，但需要多 GPU 和多步扩散采样优化，边缘部署仍有挑战。

## 8.5 安全性与可控性问题

- 世界模型可生成高保真图像、视频、音频和动作，存在潜在误用风险。
- 机器人策略若部署于真实世界，需要额外安全约束和验证。
- 生成环境作为训练场景时，错误物理规律可能被智能体学习并放大。

## 8.6 长期交互与闭环能力仍有限

- 论文展示了动作预测、策略和视频生成，但真正长期闭环交互环境仍是未来方向。
- 长时间、多智能体、可持续一致的物理模拟仍未完全解决。
- 当前视频生成多为短时片段，虽然最长支持较多帧，但稳定长时世界记忆仍需进一步验证。

# 总体评价

Cosmos 3 是一篇系统工程规模极大的 Physical AI 世界模型技术报告。其最大贡献不只是单个 benchmark 上的性能提升，而是提出并实现了一个将 **理解、生成、模拟、动作和策略** 统一起来的全模态世界模型框架。论文在架构、数据、训练、基础设施、评测和开源方面都非常完整，代表了从“多模态模型”向“可行动世界模型”发展的重要方向。不过，由于其高度依赖超大规模算力、内部数据、自建评测和复杂工程基础设施，外部复现、公平比较和真实世界安全部署仍是主要挑战。
