## 摘要
在大规模数据上训练的视觉-语言-动作（Vision-Language-Action, VLA）模型已取得显著进展，但在部署时仍容易受到分布偏移的影响。近期的 VLA 模型表明，提示可以作为引导策略行为的高效接口，但现有基于提示的引导通常依赖外部指导。这自然引出了一个问题：VLA 的测试时训练（test-time training, TTT）能否通过优化提示来实现，使得这一引导接口本身能够从交互中学习并适应？我们通过 TTT-VLA 回答这一问题。TTT-VLA 是一个基于潜在提示优化（Latent Prompt Optimization, LPO）的测试时训练框架。在训练过程中，潜在提示通过一个额外的代理任务进行学习，为策略学习提供额外的已学习条件信号。在测试时，通过从当前环境中收集交互数据，并利用代理任务的自监督信号仅在这些数据上优化潜在提示来执行 TTT，而不修改策略本身。在 SimplerEnv 上的实验表明，所提出的方法在单具身和多具身设置下均能持续提高任务成功率。进一步分析显示，性能提升主要来自对少数关键决策的纠正，而非对策略行为的全局性改变。这些结果表明，LPO 为基础操作策略在部署时的改进提供了一条有效且实用的途径。

## Abstract
Vision-Language-Action (VLA) models trained on large-scale data have made remarkable progress, but they remain vulnerable to distribution shifts at deployment time. Recent VLA models suggest that prompts can serve as an efficient interface for steering policy behavior, but existing prompt-based steering typically relies on external guidance. This raises a natural question: can test-time training (TTT) for VLA be achieved by optimizing a prompt, so that the steering interface itself can be learned and adapted from interaction? We address this question with TTT-VLA, a test-time training framework based on Latent Prompt Optimization (LPO). During training, the latent prompt is learned with an additional proxy task, providing an extra learned conditioning signal for policy learning. At test time, TTT is performed by collecting interaction data from the current environment and optimizing only the latent prompt on those data using the proxy task's self-supervised signal, without modifying the policy itself. Experiments on SimplerEnv demonstrate that the proposed method consistently improves task success rates in both single- and multi-embodiment settings. Further analysis shows that the gains arise primarily from correcting a small number of critical decisions rather than globally altering policy behavior. These results suggest that LPO provides an effective and practical pathway for deployment-time improvement of foundation manipulation policies.

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 核心思想：Latent Prompt Optimization（LPO）

- TTT-VLA 的核心是引入一个可学习的**潜在提示** \(z\)。
- 该潜在提示作为 VLA 策略的额外条件输入，与视觉观测、语言指令、机器人状态等共同决定动作输出。
- 训练阶段：
  - 潜在提示与策略一起学习。
  - 同时通过一个额外的自监督代理任务，使潜在提示捕获对控制有用的空间、状态和具身信息。
- 测试阶段：
  - 固定 VLA 主干、动作专家和状态专家等模型参数。
  - 仅使用当前环境交互数据更新潜在提示 \(z\)。
  - 通过这种方式实现部署时自适应。

### 2.2 VLA 动作生成目标

论文基于 flow matching 形式建模动作 chunk 的生成。

- 给定：
  - 当前观测 \(o\)
  - 条件上下文 \(c\)，其中包括语言、状态、图像等显式条件，也包括潜在提示 \(z\)
  - 目标动作 chunk \(a_1 = a\)
  - 噪声动作 \(a_0\)
  - 插值时间 \(t \sim U(0,1)\)

- 构造带噪动作轨迹：

\[
a_t = (1-t)a_1 + t a_0
\]

- 目标向量场：

\[
u_t = a_0 - a_1
\]

- 策略预测动作空间中的向量场，并用均方误差训练：

\[
L_{act}(\theta,z) = \mathbb{E}\left[\|v_\theta(a_t,t;o,c)-u_t\|_2^2\right]
\]

其中 \(\theta\) 是策略参数，\(z\) 是潜在提示。

### 2.3 测试时训练目标

由于测试阶段没有专家动作监督，不能直接用动作模仿损失更新提示，因此论文引入代理任务：

\[
L_{train}(\theta,z) = L_{act}(\theta,z) + L_{proxy}(\theta,z)
\]

测试时只更新潜在提示：

\[
z \leftarrow z - \eta \nabla_z L_{proxy}
\]

关键点：

- 策略主干冻结。
- 动作专家冻结。
- 状态专家冻结。
- 只更新 latent prompt。
- 代理任务在训练和测试阶段保持一致。

### 2.4 代理任务：State Grounding

论文当前实现中使用的代理任务是**状态 grounding**。

- 任务目标：
  - 根据当前观测、语言条件和潜在提示，预测机器人末端执行器位置与夹爪状态。
- 状态 grounding 也采用 flow matching 形式。
- 给定目标机器人状态 \(s_1=s\)，噪声状态 \(s_0\)，构造：

\[
s_t = (1-t)s_1 + t s_0
\]

\[
u_t^s = s_0 - s_1
\]

- 状态专家预测状态空间中的向量场：

\[
L_{sg}(\theta,z) = \mathbb{E}\left[\|v_\theta^s(s_t,t;o,c)-u_t^s\|_2^2\right]
\]

论文选择 state grounding 的理由：

- 机器人状态在训练和部署时通常可获得，不需要额外人工标注。
- 准确预测机器人末端状态需要理解视觉空间、机器人具身结构和环境布局，因此是控制相关的自监督信号。
- 该任务可以促使潜在提示吸收与空间和具身相关的上下文。

### 2.5 网络结构与训练策略

- 模型基于 **Mixture-of-Transformers（MoT）** 架构。
- 包含两个 flow-matching 专家：
  - **动作专家**：预测动作向量场。
  - **状态 grounding 专家**：预测机器人状态向量场。
- 使用 pretrained **π0.5 checkpoint** 初始化。
- 潜在提示形式：
  - \(z \in \mathbb{R}^{n \times d}\)
  - \(n\) 是 prompt token 数量。
  - \(d\) 是策略主干隐藏维度。
- 多具身设置：
  - 为不同机器人具身分配不同 latent prompt。
  - 使用 hard router 选择对应提示。
- 单具身设置：
  - 由于没有天然的具身划分，论文采用两阶段训练和 random-drop 策略。
  - 第一阶段完全阻断 latent prompt 到动作专家的连接。
  - 第二阶段以 0.5 概率随机阻断该连接。
  - 目的是避免潜在提示直接干扰动作学习，让其先通过 state grounding 学到控制相关上下文。
- 梯度约束：
  - state grounding 损失只更新状态专家和 latent prompt。
  - 动作损失主要更新 VLM backbone 和动作专家。
  - 这样可以强化潜在提示作为代理任务适应接口的角色。

## 6. 主要结论与发现

### 6.1 主结果

#### WidowX 单具身结果

- π0.5 baseline 平均成功率：51.1%
- π0.5 + SG-LP：63.5%
- π0.5 + SG-LP + TTT：67.4%

说明：

- 单纯加入 state-grounded latent prompt 已显著提升性能。
- 测试时继续优化 latent prompt 还能进一步提升。

#### Google Robot Visual Matching

- π0.5：67.5%
- π0.5 + SG-LP：68.9%
- π0.5 + SG-LP + TTT：72.4%

#### Google Robot Variant Aggregation

- π0.5：58.1%
- π0.5 + SG-LP：58.6%
- π0.5 + SG-LP + TTT：60.1%

#### 多具身 WidowX 结果

- π0.5：22.8%
- π0.5 + SG-LP：28.5%
- π0.5 + SG-LP + TTT：31.6%

### 6.2 行为层面的发现

论文发现 TTT 的增益主要来自所谓 **critical decision steering**：

- TTT 并不是全局改变整条动作轨迹。
- 多数情况下，TTT 只在少数关键决策点改变行为。
- 例如：
  - 抓取成功后是否应该抬起机械臂。
  - 堆叠方块时是否已经对齐到可以释放。
  - 物体被夹住后是否还继续向下运动。
- 这些关键节点的小变化会导致后续轨迹产生显著差异，从而决定任务成败。

### 6.3 Online TTT 的发现

- 完全在线的 batch size 1 prompt 更新不稳定。
- 可能导致 latent prompt collapse。
- 当前更可靠的做法是离线收集 buffer 后批量优化。

## 7. 优点与亮点

### 7.1 方法设计上的亮点

- **轻量适应**：
  - 测试时只更新 latent prompt，不更新 VLA 主干参数。
  - 降低了部署时适应的风险和成本。
- **自监督信号**：
  - 使用机器人状态 grounding 作为代理任务，无需人工奖励或额外标注。
- **兼容基础 VLA 模型**：
  - 方法建立在 π0.5 这类强 VLA checkpoint 之上，而不是从零设计特定架构。
- **提示接口可学习**：
  - 与人类手写提示不同，latent prompt 可以通过训练和测试时交互自动优化。
- **适用于单具身和多具身**：
  - 多具身中可为不同机器人学习不同 prompt。
  - 单具身中通过 random-drop 和两阶段训练增强 latent prompt 的作用。
- **冻结主干带来的安全性**：
  - 不直接修改策略主体，理论上更不容易破坏预训练模型已有能力。

### 7.2 实验设计上的亮点

- 使用 SimplerEnv 进行统一评测，覆盖 WidowX 和 Google Robot。
- 对比了大量公开 VLA 和机器人策略方法。
- 内部消融较完整，区分了：
  - latent prompt 训练带来的收益。
  - 测试时 prompt 优化带来的额外收益。
  - 不同 TTT 策略和学习率的影响。
- 对行为变化进行了定性和定量分析，不只报告成功率。
- 明确报告了训练和测试时训练资源，有助于理解方法实际部署成本。

## 8. 不足与局限

### 8.1 没有真实世界部署验证

- 论文只在 SimplerEnv 中评测，没有在真实机器人上进行完整测试时训练和部署。
- 真实部署中需要考虑：
  - 交互数据采集成本。
  - 错误动作带来的安全风险。
  - 机器人硬件稳定性。
  - 测试时训练期间的失败轨迹是否可接受。
- 因此，方法的真实应用可行性仍需进一步验证。

### 8.2 代理任务选择较单一

- 当前只使用 state grounding 作为代理任务。
- 尚未比较其他潜在自监督任务，例如：
  - 未来状态预测。
  - 未来图像预测。
  - 视觉表征预测。
  - 逆动力学预测。
  - 接触状态预测。
- 方法效果可能强依赖代理任务质量，state grounding 是否是最优选择尚不明确。

### 8.3 Online TTT 不稳定

- 论文发现在线逐样本更新 latent prompt 容易失败。
- 当前主方法依赖 offline buffer，意味着部署前需要先收集一定数量交互轨迹，再进行优化。
- 这限制了真正实时、持续适应的能力。

### 8.4 性能提升主要是局部修正

- 作者指出，当前 TTT 带来的增益主要是修正少数关键决策点。
- 它并不能显著重塑长时程策略行为。
- 因此对于需要复杂长期纠错、全局重新规划或新技能学习的任务，提升上限可能有限。

### 8.5 缺少与同类 TTT-VLA 方法的直接对比

- 论文提到 WorldAgen 是接近的并行工作，但由于代码和 SimplerEnv 结果不可用，没有做直接比较。
- 因此尚不清楚 TTT-VLA 相比其他测试时训练策略在相同设置下的优势幅度。

### 8.6 部署成本仍然存在

- 虽然只更新 latent prompt，但仍需要：
  - 收集当前环境交互数据。
  - 使用多张 H100 GPU 进行 15–30 分钟优化。
- 对资源受限或需要快速上线的机器人系统来说，这仍可能是负担。

### 8.7 评测仍可能受随机性影响

- Flow matching 采样和 SimplerEnv 配置有限会带来随机性。
- 作者通过增加 episodes 和固定初始 seed 缓解该问题，但策略采样噪声仍可能影响比较结果。
