## 摘要
分块视觉-语言-动作（VLA）策略预测多步机器人控制，并且每次更新仅以当前视觉观测为条件。然而，机器人动作会引发接触、遮挡和物体运动，后续决策所依赖的几何结构可能会在下一次视觉更新到来之前发生变化。空间 VLA 改善当前帧几何。时间 VLA 聚合过去帧。二者都没有跨块维护一个经动作更新的场景先验。我们主张在控制调用之间保持一个持久的、经动作更新的场景状态，并提出 EvoScene-VLA。其循环场景前缀在各个块之间携带具备几何感知能力的场景状态。在每次视觉-语言模型（VLM）调用时，VLM 将来自当前观测的场景信息与上一块经动作更新的先验相结合；动作解码器同时输出下一段动作块和一个紧凑的场景更新。该更新成为下一次先验，并在下一次调用到来时由 VLM 根据新的观测进行校正。因此，每次控制调用都从一个同时反映近期动作和最新视觉证据的场景先验开始。在训练期间，\textbf{Scene Predictor} 提供未来场景 token 目标，Geometric Anchor 将场景槽与冻结的深度和 3D 教师模型对齐。部署时我们丢弃这两个模块。在 31 个 RoboTwin 任务上，EvoScene-VLA 在固定评测中将平均成功率从 87.2% 提高到 89.1%，在随机化评测中从 86.1% 提高到 88.5%。在 Galaxea R1-Lite 真实机器人上，EvoScene-VLA 优于所有基线方法。

## Abstract
Chunked vision-language-action (VLA) policies predict multi-step robot controls, conditioning each update on the current visual observation alone. Yet robot actions cause contact, occlusion, and object motion, and the geometry that later decisions depend on can change before the next visual update arrives. Spatial VLAs improve current-frame geometry. Temporal VLAs aggregate past frames. Neither maintains an action-updated scene prior across chunks. We argue for a persistent action-updated scene state across control calls, and introduce EvoScene-VLA. Its recurrent scene prefix carries a geometry-aware scene state across chunks. At each vision-language model (VLM) call, the VLM combines scene information from the current observation with the action-updated prior from the previous chunk; the action decoder outputs both the next action chunk and a compact scene update. This update becomes the next prior, which the VLM corrects against the new observation when the next call arrives. Each control call therefore starts from a scene prior that reflects both recent actions and fresh visual evidence. During training, \textbf{Scene Predictor} supplies future scene-token targets, and Geometric Anchor aligns scene slots with frozen depth and 3D teachers. We discard both modules at deployment. On 31 RoboTwin tasks, EvoScene-VLA raises average success from 87.2% to 89.1% in fixed evaluation and from 86.1% to 88.5% in randomized evaluation. On the Galaxea R1-Lite real robot, EvoScene-VLA outperforms all baselines.

- **研究对象**：本文关注分块式视觉-语言-动作策略（chunked VLA policies）在机器人多步控制中的问题。此类策略通常一次预测一个较长的动作块，例如连续 50 步控制命令，从而提升执行效率。
- **核心问题**：现有 VLA 策略大多在每次控制调用时只依赖“当前视觉观测”来规划接下来一段动作，但机器人在执行动作块的过程中会主动改变场景，例如：
  - 抓取导致物体移动；
  - 擦拭导致表面状态变化；
  - 关闭抽屉或柜门导致遮挡；
  - 抬起杯子后原位置变为空。
- **关键矛盾**：在一个动作块内部，视觉观测不会持续更新，策略可能基于过时场景进行后续动作预测；在块与块之间，即使有新图像输入，模型也没有显式记录“刚刚执行的动作如何改变了场景”。
- **现有方法的不足**：
  - **空间 VLA**：通过深度、3D 表征、多视角特征改善当前帧几何理解，但通常只处理当前观测，不维护动作后的场景状态。
  - **时间 VLA**：保存历史图像或记忆 token，但过去观测本身并不能说明最近动作已经如何改变场景。
  - **动作条件预测方法**：可能预测未来状态，但往往只在当前决策中使用，不跨控制块持久保留。
- **本文整体含义**：作者认为，分块机器人控制需要一种**跨动作块持久存在、由动作更新、并能被新观测校正的场景先验**。EvoScene-VLA 正是围绕这一思想设计：让动作解码器不仅输出动作，还同时输出对未来场景状态的紧凑更新，使下一次控制调用能从“动作更新后的场景信念”开始。

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体框架

- EvoScene-VLA 基于 LingBot-VLA 扩展而来。
- 其核心设计是引入一个 **recurrent scene prefix（循环场景前缀）**，用于在不同控制块之间传递场景状态。
- 每次控制调用包含两个信息来源：
  1. 当前多视角图像观测；
  2. 上一个动作块执行后由动作解码器生成的场景先验。
- VLM 将两者融合，得到当前块的场景表征；动作解码器随后联合预测：
  - 下一段动作 chunk；
  - 与这些动作对应的未来场景 token。
- 执行动作后，最后一个关键帧的场景 token 被写回，作为下一次调用的先验。

---

### 2.2 循环场景前缀：Recurrent Scene Prefix

- 输入包括：
  - 多视角图像 `x_t`；
  - 语言指令 `ℓ`；
  - 每个视角的 observation slots；
  - 来自上一动作块的 prior slots。
- 前缀形式为：

```text
[ 图像 tokens, 每视角 observation slots, prior slots, 语言 tokens ]
```

- 其中：
  - **observation slots**：从当前图像中提取每个视角的几何信息；
  - **prior slots**：继承上一个动作块后的场景状态。
- 第一帧或第一个控制块时，prior slots 用可学习 embedding 初始化；之后则由动作解码器输出的场景 token 更新。

#### 非对称注意力掩码

作者设计了非对称 attention mask：

- 图像与语言 token 不读取 scene slots，以保护预训练 VLM 的图文通路；
- observation slots 读取对应视角图像和同组 observation slots；
- prior slots 读取所有 observation slots 和自身；
- 其他 token 不反向读取 prior slots。

这样做的目的：

- 避免破坏原始 VLM 的图像-语言理解能力；
- 让场景状态通过一个受控瓶颈吸收当前观测信息；
- 使 prior slots 成为一个可被动作更新、又可被新观测校正的场景状态容器。

---

### 2.3 Two-Level Geometric Anchor：两级几何锚定

由于循环场景 token 如果没有额外约束，可能只学习到普通外观特征，而不是几何信息，因此作者引入训练时使用的 **Geometric Anchor**，包括两部分。

#### 2.3.1 Local Anchor：局部深度锚定

- 目标：让 observation slots 学习每个视角的几何深度信息。
- 方法：
  - 对某个视角的图像 token 进行 mask；
  - 要求模型根据其他视角和当前场景表征恢复该视角的深度表征；
  - 深度监督来自冻结的 Monocular Depth Teacher。
- 损失形式为 SmoothL1：

```text
L_geo = 平均 SmoothL1(预测深度特征, 深度教师特征)
```

- 作用：
  - 防止模型直接复制目标视角图像特征；
  - 强迫场景 token 聚合跨视角几何信息。

#### 2.3.2 Global Anchor：全局 3D 基础模型锚定

- 目标：让场景表征与全局 3D 几何语义对齐。
- 方法：
  - 使用一个轻量 view-conditioned decoder 从 scene representation 中解码 3D 特征；
  - 将其与冻结的 3D foundation model 输出特征对齐。
- 损失为 L1：

```text
L_rep = 平均 |解码后的场景特征 - 3DFM 教师特征|
```

- 作用：
  - 将场景 token 约束到更稳定的 3D 表征空间；
  - 提升多视角几何理解能力。

---

### 2.4 Scene Predictor：训练时未来场景预测器

- Scene Predictor 是训练阶段使用的模块，推理时移除。
- 它的功能是：根据当前场景状态和真实动作序列，预测未来若干关键帧的场景 latent。
- 输入包括：
  - 当前机器人状态 `r_t`；
  - 当前场景表征 `s_p`；
  - 动作序列 `a_t:t+H`；
  - 若干关键帧 query。
- 使用 causal Transformer，保证第 `k_i` 个未来场景预测只依赖该时刻之前已经执行的动作。
- 预测结果与未来帧的 3D foundation model 特征对齐：

```text
L_pred = 平均 |预测未来场景解码特征 - 未来帧 3DFM 特征|
```

- 作用：
  - 为动作解码器提供未来场景 token 的训练目标；
  - 帮助模型学习“动作如何导致场景演化”。

---

### 2.5 Joint Action–Scene Denoising：动作与场景联合去噪

- 本文最关键的机制是：动作解码器同时去噪动作序列和未来场景 token。
- 训练时使用 flow matching：
  - 对动作目标加入噪声；
  - 对未来场景 latent 加入噪声；
  - 使用同一个时间变量 `τ`；
  - 模型预测从噪声状态到真实目标的 velocity。
- 两类损失：
  - `L_actFM`：动作 flow matching 损失；
  - `L_sceneFM`：场景 token flow matching 损失。
- 这样，动作解码器学到的不只是“如何输出动作”，还包括“执行这些动作后场景应如何变化”。

#### 推理流程

推理时不再使用 Geometric Anchor 和 Scene Predictor，只保留：

- VLM；
- recurrent scene prefix；
- action expert / action decoder。

每个控制块流程为：

1. 输入当前多视角图像、语言指令、机器人状态和上一个 prior；
2. VLM 输出校正后的当前场景表征；
3. action expert 通过 flow matching 同时生成动作 chunk 和未来场景 token；
4. 机器人执行动作 chunk；
5. 将最后关键帧场景 token 写入 prior buffer；
6. 下一个控制块重新观测并校正该 prior。

---

### 2.6 总训练目标

整体损失为：

```text
L = L_actFM + λ1 L_geo + λ2 L_rep + λ3 L_pred + λ4 L_sceneFM
```

其中：

- `L_actFM`：动作生成损失；
- `L_geo`：局部深度几何锚定；
- `L_rep`：当前帧全局 3D 表征锚定；
- `L_pred`：未来场景预测监督；
- `L_sceneFM`：将未来场景目标蒸馏进动作解码器。

## 5. 实验数量与充分性分析

### 5.1 实验数量

论文实验大致包括以下几类：

1. **RoboTwin 主实验**
   - 31 个任务；
   - Clean 和 Rand 两种设置；
   - 对比 4 种方法；
   - 报告每个任务和平均成功率。

2. **定性分析**
   - 展示 microwave、block stacking、cabinet placement 等任务中的失败恢复案例；
   - 分析 chunked baseline 因场景过时而失败的情况。

3. **轨迹质量分析**
   - 比较 EvoScene-VLA 与 LingBot-VLA 的 3D 末端执行器轨迹；
   - 结果显示本文方法轨迹更平滑。

4. **消融实验**
   - 在 RoboTwin-5Task 上逐步加入：
     - `L_pred + L_rep`；
     - `L_geo`；
     - inference-time recurrent prior；
   - 分析每个模块贡献。

5. **真实机器人实验**
   - Galaxea R1-Lite；
   - 3 个清洁任务；
   - 每个任务 100 次闭环 rollout；
   - 与 π0.5、LingBot-VLA、LingBot-VLA\* 对比。

### 5.2 是否充分

整体来看，实验较为充分，原因包括：

- 同时包含仿真和真实机器人实验；
- 使用 31 个 RoboTwin 任务，任务覆盖范围较广；
- 评估了固定初始条件和随机初始条件；
- 包含强基线 LingBot-VLA\*；
- 做了模块级消融，能支持方法设计的必要性；
- 提供了成功率、定性案例和轨迹可视化。

### 5.3 客观性与公平性

实验设计总体较公平：

- EvoScene-VLA 从 LingBot-VLA checkpoint fine-tune，与 baseline 共享相同训练数据、chunk 长度和计算预算；
- 与深度增强版本 LingBot-VLA\* 比较，有助于排除“只是加入深度监督”的影响；
- 训练时使用的辅助模块在推理时移除，因此推理开销控制较好。

但仍有一些需要注意的点：

- 真实机器人任务只有 3 个，且集中在清洁场景；
- 消融实验只在 5 个任务上做，没有覆盖全部 31 个任务；
- 部分教师模型和未来 3D 特征质量可能影响结果，但没有深入分析教师错误传播；
- 成功率提升在 RoboTwin 上是稳定但不算巨大，Clean 从 87.2% 到 89.1%，Rand 从 86.1% 到 88.5%。

## 6. 主要结论与发现

- **核心结论**：分块式 VLA 控制不仅需要当前图像几何或历史观测记忆，还需要一个由自身动作更新的、跨 chunk 持久存在的场景先验。
- **性能提升**：
  - RoboTwin 31 任务 Clean 设置：
    - LingBot-VLA\*：87.2%
    - EvoScene-VLA：89.1%
    - 提升 +1.9%
  - RoboTwin 31 任务 Rand 设置：
    - LingBot-VLA\*：86.1%
    - EvoScene-VLA：88.5%
    - 提升 +2.4%
  - Galaxea R1-Lite 真实机器人：
    - LingBot-VLA\* 平均 37.3%
    - EvoScene-VLA 平均 42.0%
- **重要现象**：
  - Rand 设置下提升更大，说明动作更新场景 prior 对姿态和布局变化更鲁棒；
  - 在真实清洁任务中，表面状态会随擦拭动作不断变化，本文方法更能处理这类场景演化；
  - 联合去噪动作和场景 token 有助于生成更平滑的末端执行器轨迹。
- **消融发现**：
  - 未来场景监督与全局 3D anchor 有明显贡献；
  - 局部深度 anchor 进一步提升性能；
  - 推理时真正跨 chunk 传递 recurrent prior 继续带来增益。

## 7. 优点与亮点

### 方法层面

- **问题定义清晰**：准确指出 chunked VLA 中“动作执行导致场景变化，但模型缺乏动作更新场景记忆”的关键缺口。
- **设计具有针对性**：不是简单增加历史帧或深度监督，而是构造“动作更新 + 新观测校正”的闭环场景状态。
- **动作解码器承担场景演化建模**：
  - 动作解码器本身最清楚即将执行什么动作；
  - 因此让它同时预测场景变化是合理且高效的。
- **推理阶段轻量**：
  - Geometric Anchor 和 Scene Predictor 都是训练时模块；
  - 部署时只保留 recurrent scene prefix 和 action-scene co-denoising；
  - 不需要额外在线 world model。
- **几何监督较完整**：
  - Local Anchor 提供局部深度约束；
  - Global Anchor 提供全局 3D foundation model 对齐；
  - 两者共同增强 scene slots 的几何含义。
- **兼容现有 VLA 框架**：
  - 基于 LingBot-VLA 扩展；
  - 保留预训练 VLM 的图像-语言路径；
  - 使用非对称 attention mask 降低对原模型能力的破坏。

### 实验层面

- 同时在仿真和真实机器人上验证；
- 使用 31 个 RoboTwin 任务，覆盖较多操作类型；
- 包含随机初始条件评估，更接近真实部署挑战；
- 真实机器人采用闭环 rollout，而不是离线回放；
- 消融实验能够较好支持各模块的有效性。

## 8. 不足与局限

### 8.1 方法局限

- **场景状态不可解释性较强**：
  - recurrent prior 是 latent token；
  - 很难直接判断其是否真的表示几何结构；
  - 论文主要通过下游成功率和消融间接验证。
- **依赖冻结教师模型质量**：
  - Local Anchor 依赖 depth teacher；
  - Global Anchor 和 Scene Predictor 依赖 3D foundation model；
  - 如果教师模型在复杂遮挡、反光、透明物体或真实场景中表现不佳，可能限制上限。
- **未来场景监督只在固定关键帧上进行**：
  - 训练时只预测固定 key-frame offsets；
  - 如果部署时重新观测周期不同，prior 可能存在时间错位。
- **chunk 长度影响未充分探索**：
  - 更长 chunk 会带来更大的场景变化，可能更需要 recurrent prior；
  - 但也会让未来场景预测更难；
  - 论文未系统分析 chunk length 对性能的影响。

### 8.2 实验局限

- **真实机器人任务数量较少**：
  - 只有 3 个清洁任务；
  - 都属于 indoor-cleaning 场景；
  - 尚不能证明对更广泛家庭操作、工具使用、装配任务等场景的泛化能力。
- **真实平台单一**：
  - 只在 Galaxea R1-Lite 双臂平台上验证；
  - 不清楚是否能直接迁移到不同机器人形态，例如单臂机械臂、移动操作平台等。
- **消融覆盖有限**：
  - 消融只在 RoboTwin-5Task 上做；
  - 没有在完整 31 任务上全面消融；
  - 对每个模块在不同任务类型上的贡献分析还不够细。
- **提升幅度中等**：
  - RoboTwin 平均成功率提升约 2 个百分点；
  - 虽然稳定，但并非压倒性提升；
  - 需要更多任务和统计分析来确认显著性。
- **可能存在 benchmark 适配风险**：
  - 方法针对 chunked control 中场景变化问题设计；
  - 在短动作、频繁重观测或场景变化很小的任务中，收益可能有限。

## 总体评价

EvoScene-VLA 的核心贡献在于，将“动作导致的场景演化”显式引入分块 VLA 控制流程中。它不只是增强当前图像几何，也不只是保存历史观测，而是让动作解码器同时产生动作和未来场景状态，并将该状态作为下一控制块的 prior。这个思想对长时程、接触丰富、遮挡频繁或场景状态持续变化的机器人任务具有较强启发意义。

实验结果表明，该方法在 RoboTwin 仿真和 Galaxea 真实机器人上均优于强基线，尤其在随机布局和清洁类动态表面任务中更有优势。不过，方法仍依赖 latent scene token 和冻结 3D 教师模型，真实实验覆盖也较有限。整体而言，这是一篇问题切入明确、系统设计完整、实验验证较充分的分块 VLA 控制论文。
