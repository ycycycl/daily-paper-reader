## 摘要
通过视觉-语言-动作（VLA）模型实现端到端自动驾驶，需要在高保真轨迹规划与高效推理之间取得微妙平衡。现有范式通常存在不足：自回归（AR）VLA 在边缘硬件上受限于内存带宽，并且容易受到曝光偏差导致的漂移影响；而全序列扩散模型无法复用 KV 缓存，并会遭受“逻辑泄漏”，从而违反基本的先感知后规划因果性。我们提出 Fast-dDrive，这是一种块扩散 VLA，能够在语义单元内部进行双向细化，同时在语义单元之间强制执行严格的因果顺序。基于驾驶 VLA 通常输出结构化类 JSON 结果这一观察，Fast-dDrive 将结构标记冻结为章节脚手架，并采用一种章节感知训练方案，以优先强化安全关键的规划。我们进一步引入脚手架投机解码，以在显著更高吞吐量下实现与 AR 相当的质量。最后，我们提出一种低开销的测试时扩展方案：从单个共享前缀 KV 缓存中分叉出 N 个随机轨迹滚动预测并对其求平均，从而以部分计算成本有效抑制预测方差。实验结果表明，Fast-dDrive 重新定义了驾驶智能体的速度-精度前沿。在 WOD-E2E 测试集上，Fast-dDrive 实现了 SOTA 的 ADE@3s 和 ADE@5s，并在基于扩散的 VLA 中取得最高的 RFS；在 nuScenes 上，其将平均 L2 误差降低至 0.32 米（提升 22%）。当与 SGLang 集成时，我们的框架相较于 AR 基线实现了 12 倍吞吐量提升，缩小了高容量 VLA 与车载实时部署效率需求之间的差距。

## Abstract
End-to-end autonomous driving via Vision-Language-Action (VLA) models demands a precarious balance between high-fidelity trajectory planning and efficient inference. Existing paradigms typically fall short: autoregressive (AR) VLAs are memory-bandwidth-bound on edge hardware and prone to exposure-bias drift, while full-sequence diffusion models preclude KV-cache reuse and suffer from "logical leakage" that violates the fundamental perceive-then-plan causality. We present Fast-dDrive, a block-diffusion VLA that performs bidirectional refinement within semantic units while enforcing strict causal ordering across them. Leveraging the observation that driving VLAs often emit structured JSON-like outputs, Fast-dDrive freezes structural tokens into a section scaffold and employs a section-aware training recipe that prioritizes safety-critical planning. We further introduce Scaffold Speculative Decoding to achieve AR-equivalent quality at significantly higher throughput. Finally, we propose a low-overhead test-time scaling scheme: by forking $N$ stochastic trajectory rollouts from a single shared-prefix KV cache and averaging them, we effectively suppress prediction variance at a fractional computational cost. Empirical results demonstrate that Fast-dDrive redefines the speed-accuracy frontier for driving agents. On the WOD-E2E test set, Fast-dDrive achieves SOTA ADE@3s and ADE@5s, alongside the highest RFS among diffusion-based VLAs; on nuScenes, it reduces average L2 error to $0.32$m (a $22\%$ improvement). When integrated with SGLang, our framework delivers $12\times$ throughput speedup over the AR baseline, narrowing the gap between high-capacity VLAs and the efficiency demands of real-time on-vehicle deployment.

#### 自回归 VLA 的问题

多数驾驶 VLA 继承通用 VLM / LLM 的自回归解码方式，即一个 token 接一个 token 生成。

其问题包括：

- **推理慢**：batch size 为 1 时，每生成一个 token 都要加载一次大模型权重，受限于 GPU 内存带宽。
- **无法充分利用并行计算**：现代 GPU 的矩阵计算能力没有被充分发挥。

已有扩散式驾驶 VLA，如 dVLM-AD，将整个结构化输出作为一个整体进行双向去噪。

其优点是可以利用全局上下文，但存在两类结构性问题：

- **无法复用 KV-cache**：每一步扩散都需要重新计算整段输出，推理开销大。
- **逻辑泄漏**：因为整个响应序列双向可见，后面的轨迹可能反过来影响前面的感知和解释，破坏“先感知、再决策、再规划”的因果顺序。

## 2. 方法论：核心思想、关键技术细节与算法流程

## 2.1 总体框架

Fast-dDrive 是一个 **块扩散式视觉-语言-动作模型**，基于：

- **Qwen2.5-VL-3B** 作为 backbone；
- **Fast-dVLM block-diffusion 架构**；
- 自动驾驶专用的结构化 JSON 输出格式；
- 联合训练的扩散分支与自回归分支。

模型输入包括：

- RGB 摄像头图像；
- ego state，如速度、加速度、yaw、yaw rate 等；
- 高层导航指令，如左转、直行等；
- 系统 prompt 和输出 JSON schema。

模型输出为四个语义 section：

1. `critical_objects`：关键物体检测与风险项；
2. `explanation`：自然语言解释；
3. `future_meta_behavior`：未来高层驾驶行为；
4. `trajectory`：未来 5 秒轨迹坐标。

---

## 2.2 核心思想一：Section-Aligned Block Diffusion

Fast-dDrive 不像全序列扩散那样对整个输出双向去噪，而是将输出划分为多个语义 section：

```text
critical_objects → explanation → future_meta_behavior → trajectory
```

其注意力结构为：

- **section / block 内部：双向注意力**  
  允许同一个语义单元内部的 token 相互参考，提高局部一致性。

- **section / block 之间：因果顺序**  
  后面的 trajectory 不能反向影响前面的 perception 或 explanation。

这样同时获得：

- 扩散模型的并行生成能力；
- 自回归模型的因果一致性；
- KV-cache 复用能力。

论文认为这可以缓解全序列扩散中的“逻辑泄漏”问题。

---

## 2.3 核心思想二：冻结 JSON 结构 token 作为 Scaffold

自动驾驶 VLA 的输出通常是固定 JSON schema，例如：

```json
{
  "critical_objects": {...},
  "explanation": "...",
  "future_meta_behavior": {...},
  "trajectory": "[[x1,y1], ...]"
}
```

其中有大量 token 是确定性的，例如：

- key 名；
- 大括号；
- 中括号；
- 冒号；
- 逗号；
- 引号；
- 固定字段名。

Fast-dDrive 将这些确定性 token 视为 **frozen scaffold**：

- 推理前直接预填；
- 训练时不 mask；
- 解码时不需要模型预测；
- 只对 value token 做扩散去噪或验证。

根据论文统计，scaffold token 约占总输出 token 的 **30%**。

这带来三个好处：

- **结构合法性 100% 保证**；
- **减少约 30% 的生成负担**；
- **让模型容量集中在真正需要预测的内容上**，如轨迹坐标、行为标签和解释文本。

---

## 2.4 核心思想三：SASD，Section-Aware Structured Diffusion

论文提出 **Section-Aware Structured Diffusion（SASD）**，包含两个训练机制。

### 2.4.1 Section-Importance-Weighted Loss

不同 section 的安全重要性不同。

例如：

- 轨迹坐标错误可能导致碰撞；
- meta behavior 错误会影响规划方向；
- explanation 的文字小错误通常不会直接导致危险。

因此论文对不同 section 赋予不同 loss 权重。

形式上，训练 loss 是各 section 的加权 masked cross-entropy：

```text
L_train = 对每个 section 的 masked token CE 按权重加权求和
```

文中给出的权重顺序为：

- trajectory：3.0；
- future_meta_behavior：2.0；
- critical_objects：1.5；
- explanation：1.0。

也就是说，安全关键的轨迹和行为决策获得更强训练信号。

### 2.4.2 Section-Adaptive Noise Schedule

论文还为不同 section 使用不同的 Beta 噪声分布。

其目的在于：

- 对不同难度、不同语义类型的 section 采用不同 mask 强度；
- 让安全关键 section 面对更适合的去噪训练难度。

文中使用的 Beta 参数为：

- trajectory：Beta(2, 1)；
- future_meta_behavior：Beta(1, 1.5)；
- critical_objects：Beta(1, 2)；
- explanation：Beta(1, 1)。

这两个机制都只发生在训练阶段，**没有额外推理开销**。

## 2.5 核心思想四：联合 AR 与 Diffusion 训练

Fast-dDrive 同时保留：

- MDM diffusion branch；
- AR branch。

总损失为：

```text
L = α L_train + β L_AR
```

其中论文设置：

```text
α = β = 0.5
```

这样做的目的：

- diffusion branch 学习块内并行去噪；
- AR branch 保留自回归验证能力；
- 同一个模型可以支持两种推理模式：
  - Section Diffusion；
  - Scaffold Speculative Decoding。

---

## 2.6 推理模式一：Section Diffusion

Section Diffusion 是纯扩散式推理。

流程为：

1. 预填 JSON scaffold；
2. 只 mask value token；
3. 按 section 顺序逐块去噪；
4. 每个 block 内使用双向注意力；
5. 已生成 section 的 KV-cache 可以复用；
6. 不调用 AR head。

这种方式较快，但实验显示其轨迹精度略低于带 AR 验证的 Scaffold Speculative Decoding。

---

## 2.7 推理模式二：Scaffold Speculative Decoding

这是论文的关键推理加速方法。

每个 block 的流程如下：

### Step 1：自动接受 Scaffold token

所有结构 token 直接接受，不进入 draft-verify 流程。

### Step 2：MDM Head 并行草拟 value token

扩散分支用 block-bidirectional attention 一次性预测当前 block 内所有 value token。

### Step 3：AR Head 因果验证

AR 分支按从左到右顺序验证 draft token。

规则为：

- 如果 AR head 的 argmax token 与 MDM draft 一致，则接受；
- 如果出现第一个不匹配，则用 AR token 替换 draft token；
- 在 rejection 位置额外接受一个 bonus token；
- 后续 draft token 丢弃，进入下一轮。

论文认为这种方式具有：

- 接近 AR 的输出质量；
- 明显高于 AR 的吞吐量；
- 结构 token 零成本；
- section-aligned block 提高 draft 接受率。

## 2.8 测试时扩展：Shared-Prefix Multi-Trajectory Rollouts

论文还提出一种低成本 test-time scaling 方法。

核心观察是：

- 前三个 section：
  - `critical_objects`
  - `explanation`
  - `future_meta_behavior`
  
  通常后验分布尖锐，适合贪心确定性生成。

- `trajectory` section 存在多模态不确定性，适合采样。

因此流程为：

1. 先确定性解码前三个 section；
2. 保存共享前缀 KV-cache；
3. 从同一个 KV-cache 分叉出 N 条轨迹 rollout；
4. 只在 trajectory section 的 AR verifier 中开启采样；
5. 得到 N 条轨迹；
6. 对轨迹进行 Jerk-Minimizing Trajectory 插值；
7. 对 N 条轨迹求平均作为最终输出。

论文默认采用：

```text
N = 4
```

这种方法的直觉是：

- 每条随机轨迹有一定预测方差；
- 平均 N 条轨迹可以将残差方差降低约 `1/N`；
- 因为前缀只算一次，所以额外计算成本低于完整跑 N 次模型。

## 6. 主要结论与发现

## 6.1 WOD-E2E 结果

在 WOD-E2E test set 上：

- Fast-dDrive Scaffold Spec：
  - ADE@5s：2.907；
  - ADE@3s：1.254；
  - RFS：7.823；
  - TPS：210.4；
  - Tok/Step：4.90。

- 加入 inference scaling，N=4：
  - ADE@5s：2.821；
  - ADE@3s：1.240；
  - RFS：7.827；
  - TPS：114.7；
  - Tok/Step：2.76。

与 dVLM-AD 相比：

- ADE 更低；
- RFS 更高；
- 吞吐明显更高。

与 AR baseline 相比：

- 规划误差更低；
- 吞吐约提升 4×；
- 加 SGLang 后最高约 11.8× 端到端加速。

## 6.2 nuScenes 结果

在 nuScenes val set 上，Fast-dDrive 达到：

- 1s L2：0.12 m；
- 2s L2：0.33 m；
- 3s L2：0.50 m；
- Avg L2：0.32 m。

优于列出的 VLM / VLA 方法，也达到或超过若干传统训练式规划方法。

## 6.3 效率结果

在 WOD-E2E val set 上：

- AR Baseline：
  - latency：7855 ms；
  - TPS：51.6；
  - Tok/Step：1。

- dVLM-AD：
  - latency：9575 ms；
  - TPS：35.2。

- Fast-dDrive + Scaffold Spec：
  - latency：1919 ms；

  - Tok/Step：4.90。

  - latency：665 ms；

  - speedup：11.8×。

说明：

- block diffusion + KV-cache reuse 有效；
- scaffold auto-accept 能进一步降低开销；
- 系统优化与算法加速可以叠加。

## 6.4 消融实验发现

SASD 消融结果显示：

- 但 ADE@5s 的变化并非完全单调，说明 RFS 和 ADE 优化目标并不完全一致。

## 7. 优点与亮点

## 7.1 方法设计紧贴任务结构

论文没有把驾驶输出当作普通文本，而是利用了 JSON schema 的确定结构：

- scaffold token 直接冻结；
- value token 才参与生成；
- section 与驾驶语义阶段对应。

这使模型结构、训练目标和输出格式高度一致。

---

## 7.2 同时解决扩散与 AR 的关键痛点

Fast-dDrive 兼顾了：

- AR 的因果顺序和 KV-cache；
- 扩散的块内并行与双向上下文；
- speculative decoding 的高效验证；
- structured output 的确定 schema 先验。

这是论文最重要的方法贡献。

---

## 7.3 推理效率提升显著

论文不仅报告精度，还详细报告：

- latency；
- TPS；
- Tok/Step；
- batch size = 1；
- 单张 H100；
- SGLang 集成效果。

这些指标对自动驾驶部署很关键。

---

## 7.4 测试时扩展设计合理

shared-prefix rollout 的设计比较巧妙：

- 不对整个输出采样，避免破坏感知和解释；
- 只对 trajectory section 采样；
- 共用前三个 section 的 KV-cache；
- 通过轨迹平均降低方差。

这种方法比简单 Best-of-N 或全输出采样更低成本。

---

## 7.5 实验覆盖较全面

论文同时覆盖：

- WOD-E2E 长尾驾驶场景；
- nuScenes 常规城市驾驶；
- 与 AR、diffusion、传统规划方法比较；
- 效率和精度双维度评估；
- 消融和定性案例。

整体实验体系较完整。

## 8. 不足与局限

## 8.1 依赖固定 JSON schema

Fast-dDrive 的 scaffold 构建依赖固定输出格式。

如果任务发生变化，例如：

- critical object 类别改变；
- 输出更多传感器诊断信息；
- 需要更复杂的行为层级；
- 轨迹点数量或时间间隔变化；

则需要手动修改 schema 和 scaffold 模板。

这限制了方法的通用性。

---

## 8.2 主要是 open-loop 评估

论文只在 open-loop benchmark 上验证。

open-loop 指标不能完全反映真实闭环驾驶表现，因为：

- 模型预测不会影响未来环境；
- 无法评估动态交互；
- 无法验证错误恢复能力；
- 无法观察长期驾驶安全性；
- 无法衡量舒适性和交通规则遵守的闭环效果。

自动驾驶最终仍需要 closed-loop simulation 或实车验证。

---

## 8.3 缺少训练成本的完整披露

虽然论文说明使用 8×H100 训练 3 epochs，但没有给出：

- 总训练时间；
- batch size；
- learning rate；
- 显存峰值；
- 推理时视觉 token 数量细节；
- 完整超参数表；
- 多次运行方差。

这会影响复现和成本评估。

---

## 8.4 测试时扩展仍增加计算

shared-prefix rollout 虽然比完整 N 次推理便宜，但仍需要额外计算。

在极低延迟车端环境中：

- N=4 可能仍过重；
- 需要根据实时约束动态选择 N；
- 额外采样和轨迹平均可能增加系统复杂度。

---

## 8.5 对 scaffold 的加速收益依赖输出格式

论文中的 scaffold 约占 30% token。

如果未来任务的 value token 占比更高，或者输出格式更自由：

- scaffold 自动接受带来的收益会下降；
- section-aligned block 的边界也可能不明显；
- 方法优势可能减弱。

---

## 8.6 对比方法之间训练条件并非完全一致

虽然论文努力提供相同 backbone 的 AR baseline，但一些外部方法仍可能存在：

- 不同模型规模；
- 不同训练数据；
- 不同后训练策略；
- 不同输入模态；
- 不同工程优化。

因此跨方法比较仍需谨慎。

---

## 8.7 RFS 与安全性之间仍有距离

RFS 是人类偏好相关指标，但它并不等价于真实驾驶安全。

例如：

- RFS 高不一定保证无碰撞；
- ADE 低也不一定保证舒适或合规；
- 对长尾极端场景的覆盖仍有限。

论文缺少对碰撞率、越界率、交通规则违反率等闭环安全指标的评估。

---

## 总体评价

Fast-dDrive 是一篇针对自动驾驶 VLA 推理效率与规划精度矛盾的系统性工作。它的核心贡献在于将 **结构化 JSON 输出、section 级因果顺序、块扩散、scaffold 先验、推测解码和共享前缀测试时扩展** 有机结合，形成了一个既快又准的端到端驾驶 VLA 框架。

从实验结果看，Fast-dDrive 在 WOD-E2E 和 nuScenes 上均取得较强表现，并在单张 H100 上显著提升推理吞吐。它尤其适合结构化输出明确、语义阶段清晰、对低延迟有强需求的自动驾驶 VLA 场景。

不过，论文仍主要停留在 open-loop 评估，且依赖固定 schema，缺少闭环验证和完整训练成本披露。未来若能在 closed-loop 仿真、真实车端部署、动态 schema 适配和安全指标评估上进一步验证，其应用价值会更有说服力。
