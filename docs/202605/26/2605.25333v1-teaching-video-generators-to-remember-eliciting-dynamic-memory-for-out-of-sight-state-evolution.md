## 摘要
视频世界模型应当在证据不可观测时维持不断演化的状态，然而当前生成器在中断时往往会冻结隐藏状态。这并不只是容量问题：预训练的视频扩散 Transformer 已经具备能够进行非局部检索的 KV-cache 机制，但它们很少被训练来将其用作动态记忆。我们提出 ReMind，这是一个通过面向记忆的数据、事件感知训练和缓存适配来引出动态记忆行为的框架。围绕包含 100 余类动态事件的分类体系，我们构建了带相机标注的训练混合数据，结合了经 VLM 过滤的真实视频、生成的困难动态样本、合成相机循环以及记忆中断增强。每个片段被转换为一个帧图，其中包含受保护锚点、退化区间和显式时间间隔。一个节点结构化课程，包括节点丢弃、噪声记忆、前沿续写和参考缓存训练，迫使模型跨中断检索相关过去状态，而不是仅依赖局部连续性。PM-RoPE 是一种优雅的相机相位 RoPE 扩展，在保持预训练通路的同时，以单次注意力成本释放时空检索能力。ReMind 在 STEVO-Bench 和恢复任务上取得了最佳总体得分。此外，通用图像到视频评测证实，该课程避免了灾难性遗忘。我们将开源代码、数据和模型。

## Abstract
Video world models should maintain evolving states when evidence is unobserved, yet current generators often freeze hidden states upon interruption. This is not simply a capacity problem: pretrained video diffusion transformers already possess KV-cache mechanisms capable of non-local retrieval, but they are rarely trained to use them as dynamic memory. We introduce ReMind, a framework eliciting dynamic memory behavior via memory-oriented data, event-aware training, and cache adaptation. Organized around a taxonomy of 100+ dynamic events, we build a camera-annotated training mixture combining VLM-filtered real videos, generated hard dynamics, synthetic camera loops, and memory-interruption augmentations. Each clip is converted into a frame graph with protected anchors, degraded intervals, and explicit temporal gaps. A node-structured curriculum, including node-drop, noisy memory, frontier continuation, and reference-cache training, forces the model to retrieve relevant past states across interruptions rather than relying solely on local continuity. PM-RoPE, an elegant camera-phase RoPE extension, unlocks spatiotemporal retrieval at a single-attention cost while preserving pretrained pathways. ReMind achieves the best overall scores on STEVO-Bench and recovery tasks. Furthermore, general image-to-video evaluations confirm this curriculum avoids catastrophic forgetting. We will open-source our code, data, and models.

## 2. 方法论：ReMind 的核心思想与关键技术

### 2.1 总体思想

论文提出 **ReMind**，目标是将预训练自回归视频扩散 Transformer 中已有的 KV-cache 重新训练为一种 **动态记忆模块**。

其核心思想可以概括为：

> 不仅让模型记住最近上下文，而是让模型学会在遮挡、黑暗、摄像机看向别处等中断之后，主动检索“最后一个可靠状态锚点”，并在此基础上推断隐藏状态的持续演化。

ReMind 包含三条主线：

1. **面向记忆的数据构建**  
   构造带有动态事件、遮挡 / 光照 / 摄像机中断、恢复节点的训练视频。

2. **节点结构化训练课程**  
   通过节点丢弃、噪声记忆、前沿续写、参考缓存等方式，迫使模型利用历史锚点，而不是仅依赖最近帧。

3. **PM-RoPE 与非连续 KV-cache**  
   在保持预训练注意力路径的同时，为缓存帧提供相机感知的时空地址，使模型能够更准确地检索历史状态。

### 2.2 面向记忆的数据构建

作者从多个来源构建训练数据，包括：

- OpenVid；
- DL3DV；
- Pexels 检索视频；
- 经 VLM 过滤的真实动态视频；
- 生成模型合成的困难动态样本；
- Neoverse 渲染的相机循环数据；
- 遮挡、灯光切换、摄像机移动等增强样本。

数据构建围绕一个包含 **100 多种动态事件类型** 的 taxonomy 展开，包括：

- 倒水 / 填充；
- 燃烧；
- 融化；
- 生长 / 开花；
- 扩散；
- 人体运动；
- 物体交互；
- 天气和光照变化；
- 周期运动；
- 离散状态变化，如开门、翻页、开关等。

每个视频片段会被转换为一个 **frame graph / 帧图**，其中包含：

- **Anchor node / 状态锚点**：中断前最后一个可靠状态；
- **Interruption node / 中断节点**：遮挡、黑暗、摄像机看走等退化区域；
- **Recovery node / 恢复节点**：中断结束后需要恢复状态的位置；
- **Memory edge / 记忆边**：显式标记恢复节点应回溯到哪些历史锚点。

这种图结构的意义是：  
普通视频训练通常将帧看作连续链条，而 ReMind 将训练样本组织为带有非局部记忆边的图，使模型学习“恢复帧需要参考哪个历史状态”。

### 2.3 Projective Memory RoPE：PM-RoPE

论文提出 **PM-RoPE**，即 Projective Memory Rotary Positional Embedding，是一种相机感知的 RoPE 扩展。

普通 RoPE 主要编码时间 / 空间位置，但当发生遮挡或摄像机运动时，仅凭时间距离很难判断哪个历史帧更可靠。比如：

- 最近帧可能是遮挡物；
- 更早的帧反而是最后一个干净可见状态；
- 摄像机回到原视角时，需要识别与当前视角几何对应的历史帧。

PM-RoPE 将相机参数注入注意力机制，使缓存条目具备更明确的时空地址。

具体地，每一帧构造一个相机描述符：

\[
c_i = [vec(R_i), t_i, \log f_{x_i}, \log f_{y_i}]
\]

其中：

- \(R_i\)：相机旋转；
- \(t_i\)：相机平移；
- \(f_x, f_y\)：焦距；
- 平移和内参会被归一化。

然后用一个零初始化 MLP \(h(\cdot)\) 将相机描述符映射为 RoPE 相位偏移：

\[
\delta_i = h(c_i)
\]

并将其加入标准 RoPE 相位：

\[
\tilde{q}_i = RoPE_{\theta_i + \delta_i}(q_i)
\]

\[
\tilde{k}_j = RoPE_{\theta_j + \delta_j}(k_j)
\]

这样，注意力中的 Query / Key 不只知道时间位置，还能感知相机姿态关系。

此外，PM-RoPE 还在 Value 和 Output 路径中加入零初始化残差项，用于注入相机几何信息。由于这些新增模块都是零初始化，微调开始时模型行为与原始预训练模型一致，从而降低破坏预训练能力的风险。

PM-RoPE 的主要优点是：

- 不需要额外双分支注意力；
- 保持单次 self-attention 成本；
- 与预训练注意力路径兼容；
- 能够联合编码时间、相机位姿和历史记忆地址。

### 2.4 非连续 Streaming KV-cache

ReMind 将视频 latent 分成多个 chunk，并使用 chunk-causal attention。

在推理时，每个生成的 chunk 会将自己的 Key / Value 写入 KV-cache，后续 chunk 可以读取这些历史缓存。

关键设计是：

> 缓存中的历史 chunk 保留其原始时间位置，而不是被压缩成连续位置。

例如：

- 干净参考 chunk 位于位置 \(0, ..., m-1\)；
- 目标视频可能在间隔 \(G\) 个 chunk 后才开始；
- 因此目标 chunk 的位置可能是 \(Gm, ..., Gm+T-1\)。

这样做可以避免旧参考帧被误认为是最近邻帧，也让模型知道历史状态与当前状态之间存在真实时间间隔。

### 2.5 节点结构化训练课程

ReMind 使用多个训练 regime 来迫使模型学习动态记忆。

主要包括：

#### 1. All-history training

- 每个 chunk 可以看到所有之前 chunk；
- 模拟基本的流式生成过程。

#### 2. Noisy-memory training

- 将部分历史 chunk 加入高噪声；
- 让模型学会区分可靠历史与不可靠上下文。

#### 3. Node-drop training

- 将遮挡、中断等节点替换为纯噪声 latent；
- 同时保留至少一个可靠 anchor；
- 迫使模型跨越退化区间检索历史锚点。

#### 4. V2V suffix / frontier training

- 保留一个干净或退化前缀；
- 只监督恢复后缀；
- 对齐自回归视频续写场景。

#### 5. Reference-cache training

- 将未退化视频中的干净参考 chunk 放入较早位置；
- 目标视频从一个采样的时间间隔之后开始；
- 显式训练模型从非连续缓存中检索旧状态。

这些训练方式共同使模型不再只依赖连续前缀，而是学会比较多个记忆候选：

- 哪个视觉更可靠；
- 哪个相机姿态更匹配；
- 哪个时间位置对应中断前状态；
- 哪个历史状态更适合作为恢复依据。

### 2.6 动态辅助损失

论文还加入了 **Dynamic Temporal-Delta Loss**，解决动态区域较小、标准 flow-matching 损失容易被静态背景主导的问题。

标准 flow-matching 中：

\[
x_t = (1-\sigma_t)x_0 + \sigma_t \epsilon
\]

模型预测：

\[
u_t = \epsilon - x_0
\]

但对于倒水、填充、燃烧等局部动态事件，状态变化区域可能只占画面很小一部分。为此作者加入帧间差分损失：

\[
L_\Delta = \mathbb{E}_{t,\epsilon} \left[
\|(\hat{x}_{0,i} - \hat{x}_{0,i-1}) - (x_{0,i} - x_{0,i-1})\|^2
\right]
\]

总损失为：

\[
L = L_{flow} + \lambda_{adapt} L_\Delta
\]

其中自适应权重：

\[
\lambda_{adapt} = \alpha \cdot \exp(-\gamma \cdot \sigma_{batch})
\]

当视频整体变化较小但局部状态演化重要时，提升 delta loss 权重；当大范围摄像机运动或场景变化较大时，降低该权重，避免破坏整体结构稳定性。

论文中设置：

- \(\alpha = 0.2\)
- \(\gamma = 5.0\)
- 辅助损失在微调前 2000 次迭代后启用。

## 3. 实验设计：数据集、场景、Benchmark 与对比方法

### 3.1 主要评测任务

论文从两个方面评估 ReMind：

#### 1. 隐藏状态恢复能力

使用 **STEVO-Bench**。

该 benchmark 专门评估视频世界模型在以下情况下是否能保持状态演化：

- 遮挡；
- 关灯 / 黑暗；
- 摄像机看向别处；
- 摄像机再转回来；
- 目标在不可见期间仍然应继续运动或变化。

STEVO-Bench 评估指标包括：

- **State Progress**：隐藏状态是否正确持续演化；
- **Physical Plausibility**：物理动态是否合理；
- **Coherence**：时序和场景一致性；
- **Observation Control**：是否执行了遮挡、黑暗等观测中断；
- **Action Control**：是否完成了提示中的动作；
- **Total**：总分。

#### 2. 通用图生视频质量

使用 **VBench**。

目的是验证 ReMind 的记忆训练不会导致通用 I2V 能力灾难性遗忘。

### 3.3 定量结果

#### STEVO-Bench 结果

ReMind 在总分上最高：

论文进行了多组消融。

比较：

#### KV-importance 分析

作者可视化了注意力 / KV 重要性热力图：

- 基础模型主要关注时间对角线，即最近帧；
- ReMind 在遮挡或摄像机回环后，会显著关注较早的干净 anchor；
- 说明模型确实学会了非局部历史检索。

## 6. 主要结论与发现

论文主要结论包括：

- 当前视频生成模型在视野外状态演化上存在明显缺陷，常出现状态冻结、重置或不合理恢复；
- 这一问题可以被表述为 **动态记忆检索问题**，而不仅是长上下文建模问题；
- 预训练视频扩散 Transformer 的 KV-cache 已具备潜在记忆能力，但需要专门训练才能被用作动态记忆；
- ReMind 通过面向记忆的数据、节点结构化训练和 PM-RoPE，可以显著提升隐藏状态恢复能力；
- 在 STEVO-Bench 上，ReMind 获得最高总分和最高 State Progress；
- 在 VBench 上，ReMind 没有出现灾难性遗忘，通用 I2V 指标反而优于 InSpatio-World；
- KV-importance 可视化表明，ReMind 确实学会了检索远距离、语义相关的历史 chunk，而不是只关注最近帧。

## 7. 优点与亮点

### 7.1 问题定义清晰

论文准确抓住了视频世界模型中的一个关键短板：  
**可见时生成合理，不可见时状态不会继续演化。**

该问题对真实世界建模、自动驾驶仿真、机器人预测等任务非常重要。

### 7.2 方法设计系统

ReMind 不是单一模块，而是一个完整框架，包括：

- 数据构建；
- 帧图标注；
- 中断增强；
- 训练课程；
- KV-cache 机制；
- PM-RoPE；
- 动态辅助损失。

各部分围绕“动态记忆”这一核心目标形成闭环。

### 7.3 PM-RoPE 兼容预训练模型

PM-RoPE 通过零初始化方式接入原模型：

- 初始时不破坏预训练能力；
- 微调中逐步学习相机感知记忆地址；
- 不需要额外双注意力分支；
- 计算成本更低。

这是一个实用性较强的设计。

### 7.4 数据构建具有针对性

论文不是简单扩大数据，而是围绕 100 多类动态事件构建记忆导向数据，并显式标注：

- anchor；
- interruption；
- recovery；
- memory edge。

这种数据组织方式对训练动态记忆非常关键。

### 7.5 实验验证较全面

论文同时评估：

- 隐藏状态恢复；
- 通用 I2V 质量；
- 架构消融；
- 损失消融；
- 数据消融；
- 注意力检索诊断。

因此不仅证明“结果更好”，也尝试解释“为什么更好”。

## 8. 不足与局限

### 8.1 仍未解决全部物理推理问题

作者明确指出，ReMind 主要针对：

- 遮挡；
- 黑暗；
- 摄像机 lookaway；
- loop closure；
- 局部状态恢复。

它并不等价于完整物理世界模型，仍可能无法处理复杂因果交互、长期物理约束、多物体遮挡关系等问题。

### 8.2 依赖相机和深度估计质量

PM-RoPE 依赖相机位姿、内参和深度信息。对于网络视频，论文使用 Depth Anything 3 等方法估计深度和相机轨迹。

这会带来风险：

- 位姿估计错误会削弱监督信号；
- 动态物体多、镜头复杂的视频可能估计不稳定；
- 训练数据可能偏向相机轨迹可靠的样本。

### 8.3 数据构建成本较高

ReMind 需要：

- VLM 过滤；
- 动态事件 taxonomy；
- 相机 / 深度标注；
- 中断增强；
- 合成困难动态；
- 渲染相机循环。

整体 pipeline 较复杂，复现成本高。

### 8.4 评估仍有自动评分偏差

消融实验中使用 Qwen3-VL 评分，虽然流程较规范，但 VLM 评分可能存在：

- 对视觉质量偏好不稳定；
- 对物理状态演化判断不够精确；
- 对不同场景类别存在偏差。

因此，高层语义指标仍需人工评估或更严格物理指标补充。

### 8.5 算力需求较高

主实验最高使用 64 张 A100 80GB，说明方法训练成本不低。对于普通实验室或小团队，完整复现可能困难。

### 8.6 开源细节尚未完全给出

论文承诺开源代码、数据和模型，但当前文本中仍缺少：

- 完整超参数；
- 总训练步数；
- 总 GPU-hours；
- 数据 manifest 细节；
- 所有生成样本配置。

因此，最终可复现性仍取决于后续发布质量。
