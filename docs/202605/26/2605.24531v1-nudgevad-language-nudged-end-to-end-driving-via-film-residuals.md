## 摘要
自然语言指令有望实现可控的端到端驾驶，但当规划器已经接收到可靠的高层命令时，其益处可能会被掩盖。我们提出 NudgeVAD，这是一种冻结规划器的残差框架，将语言作为对 VAD 轨迹的校准式引导。通过恒等初始化的 FiLM 和零初始化的残差头，NudgeVAD 在初始化时等价于冻结的规划器，因此学习到的偏差仅来自语言条件化的残差。我们沿命令可靠性轴评估 NudgeVAD。在命令可靠的情况下，语言能够改进初始规划器，但当与 VAD-FT（UNCOND）——一个在计算量匹配条件下、无需语言进行微调的 VAD 模型——相比时，语言几乎变得冗余。然而，在命令随机的情况下，语言变得至关重要：移除文本会使 ADE6s 退化至 3.166 m，而带文本的 NudgeVAD 可恢复至 2.806 m，并比 VAD-FT（UNCOND）优 0.312 m。这些结果表明，语言并非普遍具有叠加增益；当类别命令通道不可靠时，语言最具价值。

## Abstract
Natural-language instructions promise controllable end-to-end driving, but their benefit can be hidden when planners already receive reliable high-level commands. We propose NudgeVAD, a frozen-planner residual framework that uses language as a calibrated nudge to a VAD trajectory. With identity-initialized FiLM and a zero-initialized residual head, NudgeVAD is equivalent to the frozen planner at initialization, so learned deviations arise only from language-conditioned residuals. We evaluate NudgeVAD along a command-reliability axis. With reliable commands, language improves the initial planner but becomes nearly redundant once compared against VAD-FT (UNCOND), a compute-matched VAD model fine-tuned without language. With random commands, however, language becomes essential: detaching text degrades ADE6s to 3.166 m, while NudgeVAD with text recovers 2.806 m and outperforms VAD-FT (UNCOND) by 0.312 m. These results show that language is not universally additive; it is most valuable when the categorical command channel is unreliable.

## 2. 方法论：NudgeVAD 的核心思想与技术细节

### 2.1 核心思想

- 论文提出 **NudgeVAD**，即一种“冻结规划器 + 语言残差修正”的端到端驾驶框架。
- 它不是直接联合微调整个规划器，也不是让语言完全替代原有规划模块，而是：
  - 保持预训练 VAD 规划器不变；
  - 使用语言分支预测一个轨迹残差；
  - 将该残差加到 VAD 原始轨迹上；
  - 语言只作为对原规划轨迹的“小幅校准”或“nudge”。

### 2.2 基本预测形式

给定：

- 自车历史轨迹 \(H\)
- 传感器与地图上下文 \(S\)
- 高层命令 \(c\)
- 自然语言指令 \(t\)

目标是预测未来 6 秒自车轨迹：

\[
Y \in \mathbb{R}^{T_f \times 2}
\]

其中：

- \(T_f = 12\)
- 采样频率为 2 Hz
- 因此预测 6 秒未来轨迹

NudgeVAD 将最终预测分解为：

\[
\hat{Y} = f_\theta(H, S, c) + g_\phi(e, t)
\]

其中：

- \(f_\theta\)：冻结的 VAD 规划器；
- \(g_\phi\)：语言条件残差分支；
- \(e\)：VAD 中的自车规划特征；
- \(\theta\)：冻结，不参与训练；
- \(\phi\)：语言残差分支参数，参与训练。

### 2.3 冻结的 3 模式 VAD 规划器

- 论文使用 **VAD-Tiny** 作为基础规划器。
- VAD 会根据场景编码得到自车规划特征 \(e\)，并输出三个命令条件轨迹：

\[
\hat{Y}_0 = EgoFutDec_\theta(e) \in \mathbb{R}^{3 \times T_f \times 2}
\]

- 三个轨迹通常对应：
  - 左转
  - 右转
  - 直行

- 最终由命令 \(c\) 选择其中一条轨迹：

\[
\hat{y}_0 = \hat{Y}_0[c]
\]

- 这也是本文分析语言冗余的关键：
  - 如果 \(c\) 是可靠的，命令已经提供了机动类别；
  - 如果 \(c\) 是随机的，语言就变成剩余的关键意图信号。

### 2.4 语言分支：LLaMA + LoRA + FiLM + 残差 MLP

语言分支包括以下步骤：

#### 1. 指令编码

- 使用冻结的 **LLaMA-7B encoder** 编码自然语言指令。
- 使用可训练的 **LoRA adapters** 进行轻量微调。
- 对最终 token hidden states 做 masked mean pooling，得到句子级嵌入 \(u\)。

#### 2. 投影到规划特征空间

\[
v = W_p u
\]

- 将语言嵌入投影到与规划器 ego feature 兼容的特征空间。

#### 3. FiLM 调制

论文使用 FiLM 对 VAD 的 ego feature 进行语言条件调制：

\[
\gamma = W_\gamma v + 1
\]

\[
\beta = W_\beta v
\]

\[
\tilde{e} = \gamma \odot e + \beta
\]

其中：

- \(\gamma\)：缩放参数；
- \(\beta\)：平移参数；
- \(\tilde{e}\)：语言调制后的自车规划特征。

#### 4. 残差预测

调制后的特征经过轻量 MLP，输出三种命令模式下的轨迹残差：

\[
\Delta = W_2 GELU(LN(W_1 \tilde{e})) \in \mathbb{R}^{3 \times T_f \times 2}
\]

最终预测为：

\[
\hat{Y} = \hat{Y}_0 + \Delta
\]

\[
\hat{y} = \hat{Y}[c]
\]

### 2.5 恒等初始化设计

为了避免语言分支在训练初期破坏冻结规划器的已有能力，作者设计了恒等初始化：

\[
W_\gamma = 0,\quad W_\beta = 0,\quad b_\gamma = 1,\quad b_\beta = 0,\quad W_2 = 0
\]

因此初始化时：

\[
\gamma = 1,\quad \beta = 0,\quad \Delta = 0
\]

所以：

\[
\hat{Y} = \hat{Y}_0
\]

即 NudgeVAD 在训练第零步与原冻结 VAD 完全等价。

这一设计的意义是：

- 初始时不会扰动原规划器；
- 后续性能变化更容易归因于语言条件残差；
- 有助于隔离语言信息的真实贡献。

### 2.6 命令可靠性探针

论文设计了两种命令模式：

#### Reliable command

- 使用基于过去轨迹和 HD map lanelet 几何推断得到的命令。
- 注意：作者强调没有使用 ground-truth future command，以避免未来信息泄露。

#### Random command

- 将命令替换为随机 one-hot 向量：

\[
c = OneHot(Uniform\{0,1,2\})
\]

- 训练和推理阶段都使用随机命令。
- 目的：
  - 移除命令通道中的机动意图；
  - 强迫模型依赖自然语言恢复驾驶意图。

### 2.7 训练目标

- 只训练语言残差分支。
- 损失函数为加权 L1 轨迹损失：

\[
L_{traj} = \sum_{t=1}^{T_f} w_t \|\hat{y}_t - Y_t\|_1 + \lambda_{end} \|\hat{y}_{T_f} - Y_{T_f}\|_1
\]

- 没有使用辅助语言损失。
- 论文还提到三点稳定训练策略：
  - 移除 scalar residual gate；
  - 增大 residual MLP 容量；
  - 冻结感知模块保持 evaluation mode，避免 BatchNorm drift。

## 3. 实验设计：数据集、benchmark 与对比方法

### 3.1 数据集与任务

- 使用 **doScenes** 数据集。
- doScenes 建立在 **nuScenes** 之上，并包含自然语言驾驶指令。
- 任务是：
  - 给定场景、历史自车轨迹、地图信息、命令和语言指令；
  - 预测未来 6 秒自车轨迹；
  - 每条轨迹包含 12 个 waypoints，频率为 2 Hz。

### 3.2 评价指标

论文主要报告：

- **ADE**：Average Displacement Error，平均位移误差；
- **FDE**：Final Displacement Error，终点位移误差；
- **∆ADE**：

\[
\Delta ADE = ADE_{no-text} - ADE_{with-text}
\]

用于衡量同一模型、同一命令下文本带来的收益。

### 3.3 对比方法

论文区分了两个重要 baseline：

#### 1. VAD-INIT

- NudgeVAD 所使用的初始 VAD checkpoint。
- 用来观察语言分支是否能改进初始规划器。

#### 2. VAD-FT (UNCOND)

- 无语言条件的 VAD 微调模型。
- 使用和 NudgeVAD 相同的数据与额外训练预算，继续训练 60 epochs。
- 目的是建立计算量匹配的无语言 baseline。

这一区分非常关键，因为：

- 如果只和 VAD-INIT 比，语言可能看起来有效；
- 但如果和同等训练预算的无语言 VAD-FT 比，语言的真实边际价值可能变小。

### 3.4 命令模式对比

实验主要包含两种命令可靠性场景：

#### Reliable commands

- 使用基于过去 lanelet 几何推断的命令。
- 测试语言是否能在可靠命令之外提供额外信息。

#### Random commands

- 将命令随机化。
- 测试当类别命令通道不可靠时，语言是否能恢复驾驶意图。

### 3.5 消融实验

论文还进行了 adapter 设计消融，尤其是在 random command 下比较：

- VAD-STAGE 1 no-cmd base
- VAD-FT (UNCOND)
- Plain text residual
- Large residual MLP
- NudgeVAD with FiLM
- Stop override

这些实验用于分析：

- 单纯文本残差是否足够；
- 增大残差 MLP 容量是否有效；
- FiLM 调制是否是关键；
- stop override 是否贡献了主要提升。

## 6. 主要结论与发现

### 6.1 可靠命令下，语言收益有限

在 reliable command 场景中：

- VAD-INIT：
  - ADE = 3.146
  - FDE = 7.238

- NudgeVAD on VAD-INIT：
  - ADE = 2.957
  - FDE = 6.763
  - 说明语言残差确实能改进初始规划器。

但与 compute-matched 无语言微调 baseline 相比：

- VAD-FT (UNCOND)：
  - ADE = 2.729
  - FDE = 6.396

- NudgeVAD on VAD-FT：
  - ADE = 2.726
  - FDE = 6.376
  - ADE 只提升约 0.003 m。

这说明：

- 在可靠命令已经提供机动意图时，语言相对冗余；
- 初始模型上的语言收益可能只是因为模型尚未充分微调；
- 与公平的无语言微调模型相比，语言边际收益很小。

### 6.2 随机命令下，语言变得关键

在 random command 场景中：

- NudgeVAD w/o text：
  - ADE = 3.166
  - FDE = 6.722

- NudgeVAD w/ text：
  - ADE = 2.806
  - FDE = 6.148
  - ∆ADE = 0.36

- VAD-FT (UNCOND)：
  - ADE = 3.118
  - FDE = 6.517

因此，带文本的 NudgeVAD 比无语言微调模型好：

- ADE 提升 0.312 m。

这说明：

- 当命令通道随机、不可靠时，语言能够恢复机动意图；
- 语言在此时不是锦上添花，而是关键控制信号。

> 注：正文某处写到 “with-text model reducing ADE from 3.166 to 2.086”，但表格和摘要均为 2.806，推测 2.086 是排版或笔误。

- Plain text residual：
  - a@6s = 3.170
  - 没有超过 VAD-FT (UNCOND)

- Large residual MLP：
  - a@6s = 2.961
  - 明显改善

- NudgeVAD (FiLM)：
  - a@6s = 2.806
  - 进一步提升

- Stop override：
  - a@6s = 2.774
  - 只带来 0.032 m 左右的边际改善

说明：

- 单纯加入文本残差不够；
- 更强残差 MLP 和 FiLM 调制是主要收益来源；
- stop override 并不是主要贡献来源。

---

## 7. 优点与亮点

### 7.1 问题定义有洞察力

- 论文没有简单地假设“语言一定有用”，而是指出语言与已有命令之间可能存在信息冗余。
- 这一点对语言条件自动驾驶研究非常重要，因为很多 benchmark 可能无意中让语言重复导航命令。

### 7.2 冻结规划器 + 残差设计清晰

- 保持 VAD 主干冻结，有助于：
  - 保留已有几何规划能力；
  - 避免语言分支破坏预训练模型；
  - 更清楚地分析语言带来的增量作用。

### 7.3 恒等初始化设计合理

- 初始化时 NudgeVAD 与原 VAD 完全一致。
- 任何后续偏移都来自训练出的语言残差。
- 这使方法更加稳定，也更易解释。

### 7.4 命令可靠性轴设计有价值

- 通过 reliable / random command 对照，论文直接验证：
  - 命令可靠时语言冗余；
  - 命令不可靠时语言关键。
- 这种评估方式比单一 benchmark 分数更能揭示语言模块的真实价值。

### 7.5 计算量匹配 baseline 更公平

- 使用 VAD-FT (UNCOND) 作为 compute-matched baseline 是重要亮点。
- 它避免了把额外训练轮数带来的收益误判为语言带来的收益。

### 7.6 消融实验能支持方法设计

- Plain residual、large MLP、FiLM、stop override 的逐步对比说明：
  - 模块设计不是任意堆叠；
  - FiLM 调制确实是关键因素之一；
  - 后处理并非主要收益来源。

---

## 8. 不足与局限

### 8.1 数据集覆盖有限

- 论文只在 doScenes 上评估。
- doScenes 基于 nuScenes，场景复杂度和分布仍有限。
- 没有跨数据集验证方法泛化性。

### 8.2 缺少闭环评估

- 论文使用 open-loop trajectory prediction。
- ADE / FDE 低不代表闭环驾驶一定安全、舒适或符合指令。
- 自动驾驶规划最终需要闭环仿真或真实系统验证。

### 8.3 指标不能充分衡量语言遵循

- 作者自己也承认：
  - ADE 不是完整的 instruction following 指标。
  - 更低 ADE 不一定意味着更好的语义合规性。
- 例如：
  - 指令要求“跟随白色卡车”，ADE 低未必说明真的跟随了目标车辆；
  - 指令要求“靠边停车”，ADE 低未必说明行为语义正确。

### 8.4 语言类型分析不足

- 没有细粒度分析不同语言指令类别：
  - 转向类；
  - 停车类；
  - 跟车类；
  - 避让类；
  - 车道选择类；
  - 目标物体引用类。
- 因此不清楚 NudgeVAD 最擅长或最不擅长哪些语言能力。

### 8.5 随机命令设定较理想化

- random command 是一个清晰的诊断实验，但现实中命令错误通常不是完全均匀随机。
- 更真实的命令不可靠可能包括：
  - 延迟命令；
  - 部分错误命令；
  - 模糊导航指令；
  - 地图匹配错误；
  - route planner 局部失败。
- 论文没有覆盖这些更真实的噪声模式。

### 8.6 算力和实现细节报告不足

- 未报告 GPU、batch size、训练时长、显存、推理速度。
- 使用 LLaMA-7B 可能带来较高资源成本。
- 对部署到车端系统而言，计算开销是重要问题，但论文没有深入讨论。

### 8.7 缺少统计稳定性分析

- 没有报告多 seed 结果。
- 没有置信区间或显著性检验。
- 一些提升幅度较小，例如 reliable command 下的 0.003 m，可能在统计噪声范围内。

### 8.8 语言分支仍然依赖原 VAD 的表示质量

- NudgeVAD 冻结主规划器，因此语言只能通过残差修正已有轨迹。
- 如果基础 VAD 对场景理解错误较大，语言残差可能难以完全纠正。
- 该方法更适合“小幅校准”，不一定适合需要大幅重新规划的复杂指令。

---

## 总结

NudgeVAD 的核心贡献不只是提出一个语言融合模块，而是系统性地指出：自然语言在端到端驾驶规划中的价值取决于已有命令通道的可靠性。通过冻结 VAD 规划器、使用恒等初始化 FiLM 和零初始化残差头，论文构造了一个较干净的分析框架，使语言只通过残差方式影响轨迹。实验表明，在可靠命令下，语言相对于计算量匹配的无语言微调模型几乎没有额外收益；而在随机命令下，语言能显著恢复性能，并超过无语言 baseline。该工作对未来语言条件驾驶 benchmark 的设计具有启发意义：评估时应显式控制命令可靠性，并同时衡量轨迹误差和语义指令遵循能力。
