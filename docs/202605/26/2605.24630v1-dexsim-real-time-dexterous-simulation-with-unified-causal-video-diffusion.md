## 摘要
视频扩散模型的最新进展已使对物理世界进行大规模仿真成为可能。然而，涉及手—物交互的仿真仍较少受到探索。我们提出 DexSIM，一个用于实时仿真灵巧操作的灵巧仿真框架。以往利用视频扩散和三维重建的方法主要关注导航，而灵巧操作仍较为受限；但它在创建与仿真世界交互的体验以及为机器人生成合成数据方面具有广泛应用。现有方法缺乏实时交互性以及长期空间一致性和记忆能力。我们为 DexSIM 提出一个两阶段训练框架。首先，我们通过将手部动作轨迹与视频联合嵌入到统一特征空间中，训练一个双向视频扩散模型。我们采用高斯热图手部编码，以获得更准确的手部表征。随后，我们进行基于滚动展开的自回归训练，并使用更新的空间缓存作为注意力汇聚点以提供空间记忆，从而提升长期一致性和具备三维感知的灵巧操作仿真能力。DexSIM 在像素和语义相似性、运动保真度以及手部投影精度方面均优于基线方法。它还支持手部运动迁移等新应用，并以 15.24 FPS 实现实时交互。

## Abstract
Recent progress of video diffusion models have enabled extensive simulation of the physical world. While simulation with hand object interaction has been less explored. We propose DexSIM, a dexterous simulation framework for simulating dexterous manipulation in real-time. While previous works utilizing video diffusion and 3D reconstruction focus on navigation, dexterous manipulation has been limited while it has extensive applications for creating interactive experiences with the simulated world and for generating synthetic data for robotics. Existing methods lack real-time interactivity and long-term spatial consistency and memory. We propose a 2-stage training framework for DexSIM. First we train a bi-directional video diffusion model by jointly embedding the hand action trajectory and video in a unified feature space. We utilize gaussian heatmap hand encoding for more accurate hand representation. Then we conduct a roll-out based autoregressive training with updated spatial cache as attention sink for spatial memory, which improves long-term consistency and 3D aware dexterous manipulation simulation. DexSIM outperforms the baseline on pixel and semantic similarity, motion fidelity, and hand projection accuracy. It also allows new applications such as hand motion transfer and runs at 15.24 FPS real-time interactivity.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体框架

DexSIM 采用两阶段训练流程：

1. **第一阶段：训练双向视频扩散模型**
   - 输入初始图像、手部动作轨迹、初始深度信息；
   - 一次性预测整段未来视频；
   - 目标是获得一个表达能力较强的 teacher / base model。

2. **第二阶段：训练因果自回归模型**
   - 将双向模型蒸馏为可实时运行的因果模型；
   - 通过 roll-out 训练缩小训练和推理之间的分布差距；
   - 引入可更新的空间缓存，增强长期空间记忆和 3D 一致性；
   - 最终得到低 NFE、低延迟、可流式交互的 DexSIM Causal 模型。

---

### 2.2 任务定义

论文将状态表示为图像经过编码后的 latent：

- 当前状态：  
  \( s_t = enc(I_t) \)

第一阶段双向生成目标为：

\[
p(s_{1:T} \mid s_0, c_{1:T}, d_0)
\]

其中：

- \(s_0\)：初始图像状态；
- \(c_{1:T}\)：未来手部动作条件；
- \(d_0\)：初始深度 latent；
- \(s_{1:T}\)：未来视频状态序列。

第二阶段因果生成目标为：

\[
p(s_t \mid s_{<t}, c_{\leq t}, d)
\]

其中：

- 模型根据历史帧、当前及历史手部动作，以及空间缓存 \(d\) 自回归预测下一帧；
- 使用滑动窗口注意力，并保留初始帧作为 attention sink。

### 2.3 统一特征表示

DexSIM 的一个核心设计是将不同模态条件放入统一特征空间，包括：

- 视频 latent；
- 噪声 latent；
- 手部动作轨迹；
- 初始深度图；
- 空间缓存 token。

与 ControlNet 式双网络结构不同，DexSIM 更强调在 DiT 主干内部进行统一建模，以降低参数和显存开销，并提升条件与视频生成之间的对齐能力。

---

### 2.4 手部动作表示：高斯热图编码

论文没有直接使用复杂的 MANO 3D 手模型，而是采用更容易获取、对遮挡更鲁棒的 2D 手部关键点表示。

- 使用 MediaPipe 提取双手关键点；
- 双手共 42 个关键点；
- 每个关键点投影为一个高斯热图：

\[
H = \mathcal{N}(x \mid p, \sigma^2)
\]

得到：

\[
h_t \in \mathbb{R}^{42 \times h \times w}
\]

随后通过卷积编码器得到手部动作 latent：

\[
l_t \in \mathbb{R}^{c \times h' \times w'}
\]

文中设置通道数 \(c = 1536\)。

这种表示相比单纯 2D mask 更精细，能够更准确表达手指位置、手势和动作轨迹。

---

### 2.5 深度与空间感知

为了增强模型对场景三维结构的理解，DexSIM 将初始深度图作为条件输入。

- 初始深度图经过卷积编码器映射到 DiT latent 空间；
- 深度 latent 被用于提供空间布局、物体距离、手与物体相对位置等信息；
- 这有助于减少仅使用 2D 手部条件带来的空间歧义。

第一阶段训练使用标准 latent diffusion 损失：

\[
L_{LDM} =
\mathbb{E}_{z_0,t,\epsilon}
\left[
\|\epsilon - \epsilon_\theta(z_t,t,c_d,c_h)\|_2^2
\right]
\]

其中：

- \(c_d\)：深度条件；
- \(c_h\)：手部动作条件。

---

### 2.6 自回归 roll-out 与 DMD 蒸馏

第二阶段中，DexSIM 将双向模型转化为因果模型。

主要训练策略包括：

- **teacher forcing 预训练**：
  - 先用真实历史帧作为条件训练因果注意力模型；
- **self roll-out 训练**：
  - 让模型使用自己生成的历史帧继续预测未来；
  - 减少训练时和推理时的分布差距；
- **Distribution Matching Distillation, DMD**：
  - 使用第一阶段的双向 DexSIM 作为 teacher；
  - 将因果学生模型蒸馏成少步数生成模型；
  - 最终推理只需 4 个 denoising steps / NFE。

DMD 的核心思想是利用 teacher 分布和 fake / student 分布之间的 score 差异来更新生成器，使因果模型更接近双向模型的生成分布。

---

### 2.7 空间缓存 Spatial Cache

空间缓存是 DexSIM 解决长期一致性的关键机制。

其作用包括：

- 保存初始场景的 3D 空间信息；
- 在自回归生成过程中不断更新；
- 作为 attention sink，为后续帧生成提供稳定的空间参考；
- 缓解长视频生成中的物体漂移、空间遗忘和交互错位。

具体设计：

- 初始深度图被编码为深度 latent；
- 论文中使用约 **256 个 spatial cache tokens**；
- 每个 token 通道数为 **1536**；
- 空间缓存通过 cross-attention 与初始帧 latent 和后续生成帧 latent 交互并更新；
- 同时保留初始场景 attention sink；
- 总共引入约 **512 个额外条件 token** 用于因果帧生成。

该机制使模型不仅依赖最近几帧，还能维持对整个场景结构的长期记忆。

## 8. 不足与局限

- **实验数据集单一**
  - 主要基于 SSV2；
  - 缺少跨数据集验证；
  - 难以判断在复杂真实环境、机器人操作场景或长时任务中的泛化能力。

- **对比方法不足**
  - 主要与 InterDyn 对比；
  - 缺少与更多现代视频世界模型、因果视频扩散模型和灵巧操作模型的系统比较。

- **消融实验不够完整**
  - 对 spatial cache 有消融，但对其他关键组件缺乏系统分析；
  - 难以量化每个模块的独立贡献。

- **输入条件可能比 baseline 更强**
  - DexSIM 使用手部关键点、深度和空间缓存；
  - InterDyn 使用 2D mask；
  - 因此对比可能不是完全同等条件下的公平比较。

- **物理真实性仍有限**
  - 视频扩散模型主要学习视觉相关性，不显式建模接触力、摩擦、质量、刚体动力学；
  - 生成结果可能视觉上合理，但未必物理上严格正确。

- **长期生成仍可能误差累积**
  - 虽然 spatial cache 缓解了问题，但因果自回归模型仍天然存在累积误差；
  - 论文没有充分展示非常长时间交互下的稳定性。

- **实时性仍有应用门槛**
  - 15.24 FPS 已接近实时，但对高交互 AR/VR 或机器人闭环控制而言可能仍不够；
  - TTFF 2.12 秒在某些低延迟场景中仍偏高。

- **资源需求较高**
  - 训练需要 8 张 H100；
  - 基于 1.3B 视频模型；
  - 对普通研究团队或实际部署仍有较高门槛。

- **论文承认模型规模和数据规模限制**
  - 作者指出当前模型在整体外观真实感和运动平滑性上仍受限；
  - 未来需要更强 base model、更大数据和更好的蒸馏策略。
