## 摘要
通过视觉-语言-动作（Vision-Language-Action, VLA）模型实现端到端自动驾驶，需要在高保真轨迹规划与高效推理之间取得微妙平衡。现有范式通常存在不足：自回归（AR）VLA 在边缘硬件上受限于内存带宽，并且容易受到曝光偏差漂移的影响；而全序列扩散模型无法复用 KV 缓存，并存在违反基本“先感知后规划”因果性的“逻辑泄漏”问题。我们提出 Fast-dDrive，这是一种块扩散 VLA，能够在语义单元内部进行双向细化，同时在语义单元之间强制执行严格的因果顺序。基于驾驶 VLA 往往生成结构化、类似 JSON 输出的观察，Fast-dDrive 将结构性 token 冻结为分段脚手架，并采用一种分段感知的训练策略，以优先优化安全关键的规划。我们进一步引入脚手架推测解码（Scaffold Speculative Decoding），以显著更高的吞吐量实现与 AR 相当的质量。最后，我们提出一种低开销的测试时缩放方案：通过从单个共享前缀 KV 缓存分叉出 $N$ 个随机轨迹 rollout 并对其求平均，我们以较低的计算成本有效抑制了预测方差。实证结果表明，Fast-dDrive 重新定义了驾驶智能体的速度—精度前沿。在 WOD-E2E 测试集上，Fast-dDrive 取得了最先进的 ADE@3s 和 ADE@5s，并在基于扩散的 VLA 中获得最高的 RFS；在 nuScenes 上，它将平均 L2 误差降低至 $0.32$m（提升 $22\%$）。当与 SGLang 集成时，我们的框架相较 AR 基线实现了 $12\times$ 的吞吐量加速，缩小了高容量 VLA 与车载实时部署效率需求之间的差距。

## Abstract
End-to-end autonomous driving via Vision-Language-Action (VLA) models demands a precarious balance between high-fidelity trajectory planning and efficient inference. Existing paradigms typically fall short: autoregressive (AR) VLAs are memory-bandwidth-bound on edge hardware and prone to exposure-bias drift, while full-sequence diffusion models preclude KV-cache reuse and suffer from "logical leakage" that violates the fundamental perceive-then-plan causality. We present Fast-dDrive, a block-diffusion VLA that performs bidirectional refinement within semantic units while enforcing strict causal ordering across them. Leveraging the observation that driving VLAs often emit structured JSON-like outputs, Fast-dDrive freezes structural tokens into a section scaffold and employs a section-aware training recipe that prioritizes safety-critical planning. We further introduce Scaffold Speculative Decoding to achieve AR-equivalent quality at significantly higher throughput. Finally, we propose a low-overhead test-time scaling scheme: by forking $N$ stochastic trajectory rollouts from a single shared-prefix KV cache and averaging them, we effectively suppress prediction variance at a fractional computational cost. Empirical results demonstrate that Fast-dDrive redefines the speed-accuracy frontier for driving agents. On the WOD-E2E test set, Fast-dDrive achieves SOTA ADE@3s and ADE@5s, alongside the highest RFS among diffusion-based VLAs; on nuScenes, it reduces average L2 error to $0.32$m (a $22\%$ improvement). When integrated with SGLang, our framework delivers $12\times$ throughput speedup over the AR baseline, narrowing the gap between high-capacity VLAs and the efficiency demands of real-time on-vehicle deployment.

## 2. 方法论：核心思想、关键技术与算法流程

## 2.1 总体思路

Fast-dDrive 是一种 **block-diffusion VLA**。

核心思想是：

- 在一个语义 section 内使用双向扩散 denoising，使模型可以并行生成并利用局部全局上下文；
- 在不同 section 之间保持严格的因果顺序，避免规划信息泄漏到感知或解释阶段；
- 利用驾驶 VLA 输出通常是 JSON-like 结构的特点，将固定结构 token 预填为 frozen scaffold；
- 只对真正需要预测的 value token 做 diffusion 或 verification；
- 使用投机解码和共享 KV cache 进一步提升速度。

模型基于 **Qwen2.5-VL-3B**，并转换为 Fast-dVLM 风格的块扩散架构。

---

## 2.2 结构化输出与 Section 设计

论文将自动驾驶输出定义为固定 JSON schema，包含四个语义部分：

1. `critical_objects`  
   - 识别关键对象，例如车辆、行人、交通灯、天气、道路危险等；
2. `explanation`  
   - 输出自然语言解释；
3. `future_meta_behavior`  
   - 输出高层行为决策，例如加速、减速、保持车道、左转、右转；
4. `trajectory`  
   - 输出未来 5 秒轨迹点。

这些 section 按照如下因果顺序生成：

```text
critical_objects → explanation → future_meta_behavior → trajectory
```

这样可以保证模型先完成感知与解释，再生成行为和轨迹，避免未来轨迹反向影响感知结论。

---

## 2.3 Frozen Scaffold：冻结结构 token

论文观察到 JSON 输出中有大量 token 是确定性的，例如：

- `{`
- `}`
- `"`
- `:`
- `,`
- 字段名；
- 数组括号；
- 固定 schema 标记。

这些 token 并不需要模型预测。

因此 Fast-dDrive 将它们作为 **scaffold token** 预填充，并且在训练和推理中永不 mask。

模型只预测 value token，例如：

- `"yes"` / `"no"`；
- explanation 文本；
- `"slow down"` / `"left turn"`；
- 轨迹坐标值。

根据论文统计，scaffold token 约占总 decoded tokens 的 **30%**，因此冻结 scaffold 可以直接减少约三成生成负担，并且天然保证 JSON 结构合法。

---

## 2.4 Block-Causal Masked Diffusion

传统 masked diffusion 对整段序列做随机 mask，再通过 denoising 预测被 mask 的 token。

Fast-dDrive 采用 block-causal diffusion：

- 将输出划分为多个 block；
- block 内部使用双向 attention；
- block 之间使用 causal attention；
- 当前 block 可以看见 prompt 和之前 block；
- 当前 block 不能看见未来 block；
- 之前 block 的 KV cache 可以复用。

这使模型同时获得：

- diffusion 的并行 denoising 能力；
- AR 的因果顺序；
- KV cache 复用带来的推理效率。

---

## 2.5 SASD：Section-Aware Structured Diffusion

论文提出 **Section-Aware Structured Diffusion（SASD）**，主要包含两个训练时设计。

### 2.5.1 Section-Importance-Weighted Loss

不同 section 对安全和规划的重要性不同。

例如：

- trajectory token 直接决定车辆未来运动，安全影响最大；
- future_meta_behavior 影响行为决策；
- explanation 虽然重要，但轻微语言错误不一定直接影响安全。

因此论文对不同 section 的 loss 赋予不同权重。

训练目标可理解为：

```text
总 loss = 各 section 中 masked value token 的交叉熵加权和
```

权重越高，模型越关注该 section。

文中 WOD-E2E 设置中，section 权重为：

- trajectory：3.0
- future_meta_behavior：2.0
- critical_objects：1.5
- explanation：1.0

即越安全关键的部分权重越高。

### 2.5.2 Section-Adaptive Noise Schedule

不同 section 的难度和 token 分布不同，因此论文不使用统一 diffusion 噪声调度，而是对每个 section 使用不同的 Beta 分布采样噪声程度。

论文给出的 Beta 参数为：

- trajectory：Beta(2, 1)
- future_meta_behavior：Beta(1, 1.5)
- critical_objects：Beta(1, 2)
- explanation：Beta(1, 1)

该设计希望让不同 section 在训练时接触到更适合自身特点的 mask / noise 难度。

## 2.6 联合 AR 与 Diffusion 训练

Fast-dDrive 同时训练两个分支：

1. **Diffusion branch**
   - 用于块内并行 denoising；
2. **AR branch**
   - 保留自回归生成能力，用于后续投机解码中的验证。

总 loss 为：

```text
L = 0.5 * L_diffusion + 0.5 * L_AR
```

这样同一个模型可以支持两种推理方式：

- Section Diffusion；
- Scaffold Speculative Decoding。

---

## 2.7 推理模式一：Section Diffusion

Section Diffusion 是纯 diffusion 推理模式。

流程为：

1. 预填 frozen scaffold；
2. 按照 section 顺序处理；
3. 每个 section 内部对 value token 进行迭代 denoising；
4. 已完成 section 的 KV cache 可复用；
5. 不调用 AR head。

优点是并行性较好，推理速度快；  
缺点是没有 AR verifier，精度略低于 Scaffold Speculative Decoding。

---

## 2.8 推理模式二：Scaffold Speculative Decoding

这是论文的关键加速方法之一。

对于每个 block，其流程为：

1. **Auto-accept scaffold**
   - scaffold token 是固定结构，直接接受；
   - 不参与 draft，也不参与 verify；

2. **MDM Draft**
   - diffusion head 使用双向 attention；
   - 并行预测当前 block 内所有 value token；

3. **AR Verify**
   - AR head 使用 causal attention；
   - 按从左到右顺序验证 draft token；
   - 如果 AR argmax 与 diffusion draft 一致，则接受；
   - 如果不一致，则用 AR token 替换，并丢弃后续 draft；
   - 在 rejection 处额外接受一个 bonus token。

该方法的关键优势是：

- scaffold token 不需要推理；
- value token 可以被并行 draft；
- AR verifier 保证质量接近纯 AR；
- 每个 block 只需要固定的 draft + verify forward；
- section-aligned block 比任意固定长度 block 有更高语义一致性。

---

## 2.9 测试时缩放：Shared-Prefix Multi-Trajectory Rollouts

论文还提出一种低开销 test-time scaling。

核心观察：

- 前三个 section，即感知、解释、行为，通常 posterior 很尖锐，不适合采样；
- 轨迹 section 具有一定多模态和不确定性，适合采样；
- 前三个 section 是确定性生成，因此它们的 KV cache 可以共享。

流程为：

1. 使用 Scaffold Speculative Decoding 确定性生成：
   - critical_objects；
   - explanation；
   - future_meta_behavior；

2. 保存共享 prefix KV cache；

3. 从这个 KV cache 分叉出 N 个 trajectory rollout；

4. 只在 trajectory section 中启用 stochastic sampling；

5. 得到 N 条轨迹后进行 JMT 插值到 20 个 waypoint；

6. 对 N 条轨迹取平均作为最终输出。

论文认为，根据均值方差原理，平均 N 条 rollout 可将随机残差方差降低约 `1/N`，同时额外计算成本远小于完整重复 N 次推理。

默认使用：

```text
N = 4
```

## 6. 主要结论与发现

## 6.1 WOD-E2E 结果

在 WOD-E2E test set 上，Fast-dDrive 取得：

- 最低 ADE@3s；
- 最低 ADE@5s；
- 在 diffusion-based VLA 中最高 RFS；
- Scaffold Spec 单次推理已经超过多数 AR 和 diffusion baselines；
- 使用 N = 4 inference scaling 后，ADE 进一步下降。

关键结果：

- Fast-dDrive Scaffold Spec：
  - ADE@5s = 2.907
  - ADE@3s = 1.254
  - RFS = 7.823
  - TPS = 210.4

- 加入 inference scaling N=4：
  - ADE@5s = 2.821
  - ADE@3s = 1.240
  - RFS = 7.827
  - TPS = 114.7

相比 dVLM-AD：

- ADE 更低；
- RFS 更高；
- TPS 大幅提升。

## 6.2 nuScenes 结果

在 nuScenes val 上，Fast-dDrive 达到：

- 1s L2 error：0.12 m；
- 2s L2 error：0.33 m；
- 3s L2 error：0.50 m；
- Avg L2 error：0.32 m。

相比 dVLM-AD 平均 0.41 m，有明显提升。  
也优于或接近一些传统训练式规划方法，如 UniAD、VAD-Base、BEV-Planner。

## 6.3 效率结果

在 WOD-E2E val 上：

- AR Baseline：
  - Latency：7855 ms；
  - TPS：51.6；

- Fast-dDrive + Scaffold Spec：
  - Latency：1919 ms；
  - TPS：210.4；
  - 约 4.1× latency speedup；

- Fast-dDrive + SGLang：
  - Latency：665 ms；
  - TPS：608.5；
  - 相比 AR Baseline 约 11.8× 加速。

说明 block diffusion + scaffold + speculative decoding 可以显著突破 AR 单 token 解码瓶颈。

## 6.4 消融结论

SASD 消融显示：

- Section-Importance-Weighted Loss 对 RFS 提升更明显；
- Section-Adaptive Noise Schedule 单独使用提升较小，甚至 ADE 不一定更好；
- 二者结合时 RFS 最好；
- 说明安全关键 section 的训练重加权对自动驾驶 VLA 很重要。

Test-time scaling 结果显示：

- rollout 数量 N 增加时，ADE@5s 单调下降；
- 多轨迹平均可以降低预测方差；
- N = 4 是较好的速度—精度折中。

## 7. 优点与亮点

## 7.1 方法设计亮点

- **很好地利用了自动驾驶输出的结构性**  
  将 JSON schema 中固定 token 作为 frozen scaffold，减少无效生成，并保证结构合法。

- **兼顾 diffusion 和 AR 的优点**  
  block 内双向 denoising 提供全局上下文，block 间因果生成避免 logical leakage。

- **解决 full-sequence diffusion 无法复用 KV cache 的问题**  
  通过 section-aligned block diffusion，实现 KV cache 复用。

- **Scaffold Speculative Decoding 设计自然且高效**  
  scaffold token 自动接受，value token 才进入 draft-verify，大幅降低推理开销。

- **测试时缩放成本较低**  
  只在 trajectory section 采样，且共享前三个 section 的 KV cache，比完整 Best-of-N 更经济。

- **训练目标与安全重要性相关联**  
  对 trajectory 和 future_meta_behavior 更高加权，符合自动驾驶任务需求。

## 7.2 实验设计亮点

- 同时验证精度和速度；
- 同时比较 AR、full-sequence diffusion、block diffusion；
- 使用真实自动驾驶 benchmark；
- 在单 H100、batch size = 1 下报告效率，贴近车端低 batch 推理场景；
- 提供定性案例展示 reasoning 与 trajectory 的一致性；
- 包含 SGLang 系统集成结果，说明方法可与实际 serving 优化结合。

## 7.3 实际应用价值

Fast-dDrive 对自动驾驶 VLA 的实用意义较强：

- 大幅降低推理延迟；
- 保留可解释 JSON / CoT 输出；
- 提升轨迹精度；
- 减少结构化输出错误；
- 更接近车端实时部署要求。

## 8. 不足与局限

## 8.1 依赖固定 JSON Schema

Fast-dDrive 的 scaffold construction 强依赖预定义结构。

如果任务发生变化，例如：

- critical object 类型增加；
- reasoning 格式变化；
- 轨迹点数量变化；
- 行为标签集合变化；

则需要重新设计模板，甚至重新训练或微调模型。

这限制了方法的灵活性。

## 8.2 主要是 open-loop 评估

论文实验集中在 WOD-E2E 和 nuScenes 的 open-loop benchmark。

open-loop 只能评估模型预测与专家轨迹的接近程度，不能完全反映：

- 闭环控制稳定性；
- 与其他交通参与者交互时的反应能力；
- 长时间 rollout 后的安全性；
- 极端 corner case 下的恢复能力。

论文也承认未来需要 closed-loop simulation 验证。

## 8.3 数据和 baseline 公平性仍有潜在问题

虽然作者尽量使用相同 backbone 的 AR baseline，但跨方法比较仍可能受到影响：

- 不同方法训练数据规模不同；
- Fast-dDrive 混合了额外 60k trajectory-only 样本；
- 部分 baseline 可能有强化学习后训练或轨迹池差异；
- RFS 等指标可能受到 benchmark 内部评价机制影响。

因此，SOTA 结论需要结合具体实验条件理解。

## 8.4 测试时缩放仍增加计算

虽然共享 KV cache 使多轨迹 rollout 成本较低，但 N = 4 仍会带来额外计算。

在极低延迟车端环境下：

- 额外 rollout 可能不可接受；
- 需要根据硬件和安全需求权衡 N；
- 实际部署中可能只能使用单次 Scaffold Spec。

## 8.5 训练和系统细节不够完整

论文没有明确提供：

- 总训练时长；
- GPU-hour；
- 显存使用；
- 完整优化器配置；
- SGLang 集成细节；
- 不同硬件平台上的效率表现。

这会影响复现和工程评估。

## 8.6 感知输入相对有限

论文不使用：

- LiDAR；
- Radar；
- HD Map；
- surround-view 全视角信息。

虽然这有利于轻量化和公平比较，但也意味着：

- 对侧后方交互场景感知可能不足；
- 对复杂路口和遮挡场景的鲁棒性可能有限；
- 与实际自动驾驶全栈传感器配置存在差异。

# 总体评价

Fast-dDrive 是一篇围绕自动驾驶 VLA 推理效率和轨迹精度展开的较完整工作。其核心贡献不只是将 diffusion 用于驾驶，而是进一步利用驾驶输出的结构化特点，将 **section-level causality、frozen scaffold、block diffusion、speculative decoding 和 shared-prefix rollout** 组合成一个高效系统。

论文的主要价值在于证明：  
**当模型输出具有强结构时，把这种结构显式编码进生成过程，可以同时提升速度、稳定性和精度。**

从实验结果看，Fast-dDrive 在 WOD-E2E 和 nuScenes 上均取得了较强表现，并在单 H100 上实现显著推理加速，是面向实时自动驾驶 VLA 的一个有代表性的方向。
