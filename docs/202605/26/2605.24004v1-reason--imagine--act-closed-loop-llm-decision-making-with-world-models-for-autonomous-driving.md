## 摘要
大语言模型（LLMs）在自动驾驶领域具有良好前景，但仅基于语义的决策策略在动态交通中可能产生物理上不安全的行为。现有方法要么进行在线语言推理而缺乏显式的动力学验证，要么主要在离线流程中使用世界模型，从而在决策时刻的语义意图与物理可行性之间留下了缺口。我们提出 Reason—Imagine—Act（RIA），一种闭环框架，将 LLM 推理器与动作条件世界模型相结合，用于在线安全验证。在每个步骤中，LLM 提出一个动作模板和候选子动作，世界模型执行短时域 rollout，安全评分器选择最安全的可执行动作，并将反馈提供给下一步推理。在统一的 CARLA 点目标协议（1000 个 episode）下，RIA 实现了 80.05% 的路线完成率、51.10% 的到达率和 0.20% 的碰撞率。在相同的闭环接口下，RIA 在核心闭环指标上持续优于包括 CARLA TM 和 MADA 在内的免训练基线。为便于复现，代码可见于 https://github.com/pku-smart-city/source_code/tree/main/RIA。

## Abstract
Large language models (LLMs) are promising for autonomous driving, but semantics-only decision policies can yield physically unsafe behavior in dynamic traffic. Existing methods either perform online language reasoning without explicit dynamics verification or use world models mainly in offline pipelines, leaving a gap between semantic intent and physical feasibility at decision time. We propose Reason--Imagine--Act (RIA), a closed-loop framework that couples an LLM reasoner with an action-conditioned world model for online safety verification. At each step, the LLM proposes an action template and candidate sub-actions, the world model performs short-horizon rollouts, and a safety scorer selects the safest executable action with feedback to the next reasoning step. Under a unified CARLA point-goal protocol (1000 episodes), RIA achieves 80.05% route completion, 51.10% arrival rate, and 0.20% collision rate. Under the same closed-loop interface, RIA consistently outperforms training-free baselines, including CARLA TM and MADA, on core closed-loop metrics. For reproducibility, code is available at https://github.com/pku-smart-city/source_code/tree/main/RIA.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 总体框架：Reason–Imagine–Act

RIA 框架由三个阶段组成：

1. **Reason：推理**
   - LLM 读取当前结构化状态、环境信息和短期驾驶记忆。
   - 输出一个高层动作模板，例如：
     - `stop`
     - `speed up`
     - `speed down`
     - `maintain speed`
     - `lane changing left`
     - `lane changing right`
     - `normal behavior`
   - 同时生成该模板下的一组候选子动作，例如不同强度的加速、减速、刹车、变道参数等。

2. **Imagine：想象**
   - 动作条件世界模型针对每个候选子动作进行短时域 rollout。
   - 世界模型预测未来若干控制步中的车辆状态、周围交通交互和潜在风险。
   - 重点不是生成高质量图像或长时域视频，而是用于短时域安全排序。

3. **Act：行动**
   - 安全评分器根据 rollout 结果对候选动作打分。
   - 系统选择最安全的可执行动作，并交给 CARLA Traffic Manager（TM）作为底层控制器执行。
   - 执行结果和物理验证摘要会被写入记忆，在下一轮 LLM 推理中继续使用。

### 2.2 形式化描述

在每个控制时刻 \(t\)：

- 原始观测为 \(o_t\)，经过特征构建得到结构化状态 \(s_t\)。
- LLM 输出：
  - 动作模板 \(g_t \in G\)
  - 候选子动作集合：

\[
U(g_t)=\{u_t^{(j)}\}_{j=1}^{J}
\]

- 世界模型对每个候选动作进行预测，得到未来轨迹：

\[
\hat{\tau}_t^{(j)}=\{\hat{s}_{t+1}^{(j)}, \ldots, \hat{s}_{t+H}^{(j)}\}
\]

- 安全评分器计算分数：

\[
S(\hat{\tau}_t^{(j)})
\]

- 最终选择：

\[
u_t^\star = \arg\max_j S(\hat{\tau}_t^{(j)})
\]

其中 \(H\) 是短时域 rollout 长度，\(J\) 是候选子动作数量。

### 2.3 世界模型设计

论文使用一种受 RSSM 启发的动作条件潜变量动态模型。

#### 状态表示

结构化状态 \(s_t\) 包括：

- **自车特征**
  - 位置、速度、航向角、加速度、车道中心偏移、路线进度等；
- **道路特征**
  - 车道拓扑、路线方向、限速、交通灯状态、距离停止线距离、距离车道边界距离等；
- **社会交互特征**
  - 周围车辆和行人的相对位置、相对速度等。

这种设计使得红灯风险、越界、偏离车道、前车距离坍缩等安全指标可以直接从预测状态中计算。

#### 社会注意力编码

为建模周围交通参与者与自车之间的交互，论文使用注意力机制计算社会上下文特征：

\[
\alpha_i = \text{softmax}_i \left( \frac{(W_q x_e)^\top (W_k x_i)}{\sqrt{d}} \right)
\]

\[
c_t = \sum_{i=1}^{N} \alpha_i W_v x_i
\]

其中：

- \(x_e\) 是自车 token；
- \(x_i\) 是邻近交通参与者 token；
- \(c_t\) 是社会上下文；
- 该设计允许模型关注具有高交互风险的车辆，而不只是距离最近的车辆。

#### 潜变量动态模型

模型维护隐状态 \(h_t\)：

\[
h_t = f_{\text{dyn}}(h_{t-1}, z_{t-1}, a_{t-1})
\]

同时使用 GRU 编码时序特征：

\[
\tilde{h}_t = \text{GRU}(x_t, \tilde{h}_{t-1})
\]

其中：

\[
x_t = [s_t; c_t]
\]

解码器预测残差动态：

\[
\hat{s}_{t+1} = s_t + g_{\text{dec}}(h_t, z_t, a_t)
\]

残差预测比直接预测绝对状态更稳定，也更适合闭环 rollout。

### 2.4 世界模型训练

训练数据由专家策略和扰动探索结合得到：

- 专家策略主要来自 CARLA Traffic Manager；
- 加入 \(\epsilon=0.2\) 的噪声探索；
- 特别保留碰撞、越界、车道边界违规等失败事件前的短窗口数据，以增强模型对安全边界附近状态转移的学习。

训练目标为多步预测损失：

\[
\mathcal{L}_{WM}
=
\frac{1}{H}
\sum_{k=1}^{H}
\gamma^k
\|\hat{s}_{t+k} - s_{t+k}\|_2^2
+
\lambda \mathcal{L}_{latent}
\]

其中：

- \(\gamma\) 是长时域误差折扣因子；
- \(\mathcal{L}_{latent}\) 是潜变量正则项，例如 KL 损失；
- \(\lambda\) 用于平衡预测误差和潜变量正则。

### 2.5 安全评分机制

世界模型对每个候选子动作 rollout 后，计算风险代价：

\[
C(\hat{\tau}_t^{(j)}) = p_{crit} + p_{dist} + p_{lane}
\]

其中：

- 近距离临界碰撞风险：

\[
p_{crit}=100 \cdot \mathbf{1}[d_{min}<3.0]
\]

- 距离风险：

\[
p_{dist} = \max(0, 8.0 - d_{min})^2
\]

- 车道偏移风险根据是否为变道动作分开计算：
  - 变道动作：

\[
p_{lane}^{lc} = 2.0 \max(0, l_{max}-2.8)
\]

  - 非变道动作：

\[
p_{lane}^{nlc} = 8.0 \max(0, l_{max}-1.5)
\]

最终安全分数定义为：

\[
S(\hat{\tau}) = -C(\hat{\tau})
\]

也就是说，代价越小，安全分数越高。

---

### 2.6 硬规则与底层控制

- 系统包含两类硬安全机制：
  - rollout 前的短路规则：如果检测到即将发生前向碰撞，直接触发安全 fallback；
  - rollout 后的覆盖规则：如果所有候选动作都违反关键阈值，则强制选择安全动作。

- 底层执行不是直接输出连续油门、刹车、转向，而是通过 **CARLA Traffic Manager** 执行：
  - RIA 输出高层模板和子动作；
  - 子动作映射为 TM 的目标速度、跟车距离、变道偏好、保守/激进程度等参数；
  - TM 最终生成可执行控制命令。

- TM 还使用世界模型给出的四类提示：
  - 前向间隙；
  - 前向 TTC；
  - 归一化车道偏移；
  - 静态障碍物前方距离。

根据这些提示计算风险分数，并切换低、中、高风险控制模式。

## 6. 主要结论与发现

### 6.1 世界模型验证显著提升 LLM 驾驶决策

与不使用世界模型的 LLM baseline 相比，RIA 取得明显提升：

| 方法 | RC ↑ | AR ↑ | ColR ↓ |
|---|---:|---:|---:|
| LLM w/o WM verification | 61.21% | 30.20% | 0.40% |
| LLM+WM / RIA | 80.05% | 51.10% | 0.20% |

说明世界模型 rollout 验证是性能提升的主要来源。

### 6.2 RIA 在闭环任务表现上优于训练免费基线

与 CARLA TM 和 MADA 相比，RIA 的核心指标更好：

| 方法 | RC ↑ | AR ↑ | ColR ↓ |
|---|---:|---:|---:|
| CARLA TM | 46.47% | 13.40% | 16.00% |
| MADA baseline | 21.88% | 22.40% | 53.00% |
| RIA | 80.05% | 51.10% | 0.20% |

论文认为这说明：将 LLM 的语义推理与世界模型的物理预测结合，可以有效提升闭环驾驶任务中的完成度与安全性。

### 6.3 安全与舒适性指标也有改善

RIA 在多个行为质量指标上优于 LLM-only：

| 指标 | LLM w/o WM | RIA |
|---|---:|---:|
| OffR | 0.28 | 0.19 |
| HB | 207.41 | 24.58 |
| HJ | 87.85 | 54.77 |
| MAJ | 97.43 | 31.21 |

说明世界模型不仅减少碰撞，也降低了急刹、高 jerk 等不舒适行为。

### 6.4 存在一定 trade-off

- 相比 CARLA TM，RIA 的离路事件为 0.19，而 TM 为 0.00。
- 但 RIA 在路线完成率、到达率和碰撞率方面明显优于 TM。
- 因此论文认为 RIA 在整体任务表现与安全性之间取得了更好平衡。

## 7. 优点与亮点

### 7.1 方法层面

- **闭环耦合 LLM 与世界模型**
  - 不是简单用 LLM 输出决策，也不是单独使用世界模型预测，而是将两者放入实时决策循环。

- **“想象后再行动”的安全验证机制**
  - 在动作执行前对多个候选子动作做短时域 rollout，有助于提前发现潜在碰撞或越界风险。

- **动作条件世界模型**
  - 世界模型根据不同候选动作预测不同未来，而非被动预测单一路径。
  - 这使其更适合用于候选动作排序。

- **结构化状态设计实用**
  - 使用自车、道路、社会交互等结构化特征，使安全规则可以直接计算，避免图像生成式世界模型带来的复杂后处理。

- **反馈机制**
  - 世界模型验证结果被反馈给 LLM 的下一轮推理，有助于减少反复变道、震荡决策等问题。

- **安全评分简洁可解释**
  - 评分项包括最小距离、车道偏移、临界碰撞阈值等，易于理解和调试。

### 7.2 实验层面

- **闭环评估**
  - 相比开环预测指标，闭环 CARLA 驾驶任务更接近真实部署场景。

- **评估规模较大**
  - 1000 个 episode 的随机点目标导航比小规模案例展示更有说服力。

- **对比包含 LLM-only 消融**
  - 能较清楚地证明世界模型验证模块的增益。

- **同时关注安全与舒适性**
  - 不只报告成功率，也报告碰撞、红灯、离路、急刹、高 jerk 等指标。

## 8. 不足与局限

### 8.1 实验覆盖有限

- 仅在 CARLA 仿真环境中评估，没有真实道路数据或实车实验。
- 没有在更标准化的大规模自动驾驶 benchmark 上验证，例如 nuPlan、Waymo Open Motion Dataset、CARLA Leaderboard 官方完整设置等。
- 场景虽然包含多个 CARLA Town，但复杂长尾场景覆盖仍有限，例如：
  - 高密度交通；
  - 非保护左转；
  - 无信号交叉口博弈；
  - 施工区；
  - 异常行为行人；
  - 极端天气和传感器噪声。

### 8.2 感知假设较理想

- 所有方法共享特权仿真状态，而不是从摄像头、LiDAR 或多传感器感知结果中获取状态。
- 实际部署中，感知误差、遮挡、检测延迟会显著影响世界模型 rollout 和安全评分。

### 8.3 消融分析不足

论文没有充分回答以下问题：

- 如果去掉物理反馈记忆，仅保留 WM 排序，性能如何？
- 如果去掉硬规则 fallback，碰撞率是否仍然很低？
- 不同 rollout horizon 对性能影响如何？
- 不同候选动作数量是否会显著改变结果？
- 世界模型误差增大时，系统是否会被错误 rollout 误导？
- 社会注意力模块和近失败窗口采样分别贡献多少？

### 8.4 算力和复现细节不足

- 未报告 GPU、训练时间、训练数据规模、推理延迟和 API 成本。
- 对于实时自动驾驶而言，延迟非常关键，但论文没有系统分析 LLM 调用加 WM rollout 的实时性。

### 8.5 对比方法范围有限

- 论文只比较了 training-free 设置下的方法。
- 没有与训练型闭环驾驶策略、强化学习方法、经典规划器或现代端到端 VLM 驾驶方法进行全面比较。
- 因此结论更适合解释为：RIA 在相同无训练闭环接口下优于若干基线，而不是已经全面优于现有自动驾驶规划方法。

### 8.6 底层控制依赖 CARLA TM

- RIA 并不直接输出连续控制，而是通过 TM 执行。
- 这降低了方法实现难度，但也意味着性能部分依赖 TM 的底层控制能力。
- 在真实车辆控制器或不同仿真器中，动作模板到控制参数的映射可能需要重新校准。

---

## 总体评价

本文提出的 RIA 框架抓住了 LLM 自动驾驶中的关键问题：**语言模型会推理，但不一定懂物理后果；世界模型懂动态，但缺乏语义意图。** 论文通过“推理—想象—行动”的闭环流程，将 LLM 高层决策与世界模型短时域安全验证结合，在 CARLA 点目标导航任务中显著提升了路线完成率、到达率，并降低碰撞和不舒适行为。

整体来看，方法概念清晰、工程实现务实、实验结果有较强说服力。但论文仍存在消融不足、算力细节缺失、依赖特权状态和 CARLA TM、缺少真实感知与实车验证等限制。其主要贡献更适合作为一种 **LLM 驾驶决策的在线安全验证框架**，而不是完整替代自动驾驶规划控制系统的端到端方案。
