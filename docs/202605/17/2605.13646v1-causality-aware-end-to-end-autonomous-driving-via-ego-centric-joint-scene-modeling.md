## 摘要
端到端自动驾驶通过直接从传感器输入预测未来轨迹，绕过传统的模块化流程，近年来取得了显著进展。然而，现有方法往往忽视自车规划中的因果相互依赖关系，未能考虑自车与周围交通参与者之间的相互作用。这种因果层面的疏忽会导致轨迹预测不一致且不可靠，尤其是在需要联合推理自车决策与邻近交通参与者行为的交互关键场景中。为解决这一局限，我们提出 CaAD，一种因果感知的端到端自动驾驶框架，用于在共享的潜在场景表征中捕获这些依赖关系。首先，我们提出一个以自车为中心的联合因果建模模块，该模块建立在边缘预测分支之上，学习自车与交互相关交通参与者之间的因果依赖。其次，我们采用一个通过联合模态嵌入实现的因果感知策略对齐阶段，将随机自车策略与基于周围交通和地图上下文计算得到的面向规划的闭环反馈进行对齐。在 Bench2Drive 和 NAVSIM 基准上，CaAD 展现出强大的闭环规划性能，在 Bench2Drive 上取得了 87.53 的驾驶得分和 71.81 的成功率，并在 NAVSIM 上取得了 91.1 的 PDMS。

## Abstract
End-to-end autonomous driving, which bypasses traditional modular pipelines by directly predicting future trajectories from sensor inputs, has recently achieved substantial progress. However, existing methods often overlook the causal inter-dependencies in ego-vehicle planning, ignoring the reciprocal relations between the ego vehicle and surrounding agents. This causal oversight leads to inconsistent and unreliable trajectory predictions, especially in interaction-critical scenarios where ego decisions and neighboring agent behaviors must be reasoned about jointly. To address this limitation, we propose CaAD, a Causality-aware end-to-end Autonomous Driving framework that captures these dependencies within a shared latent scene representation. First, we propose a ego-centric joint-causal modeling module that builds on the marginal prediction branch, and learns causal dependencies between the ego vehicle and interaction-relevant agents. Second, we employ a causality-aware policy alignment stage implemented with joint-mode embeddings to align the stochastic ego policy with planning-oriented closed-loop feedback computed from surrounding traffic and map context. On the Bench2Drive and NAVSIM benchmarks, CaAD demonstrates strong closed-loop planning performance, achieving a Driving Score of 87.53 and Success Rate of 71.81 on Bench2Drive, and a PDMS of 91.1 on NAVSIM.

- 周围车辆预测虽然单独合理，但未必与自车决策形成协调的交通场景；

论文认为，自动驾驶规划不应只预测“自车会怎么走”，还应建模“在某个自车意图下，关键交互车辆会如何响应”。因此，作者提出 **CaAD：Causality-aware End-to-End Autonomous Driving**，其核心含义是：

> 通过以自车为中心的联合场景建模，将自车轨迹与交互相关智能体轨迹组织到同一个场景假设中，从而提升端到端规划在交互密集场景中的一致性、安全性和闭环可靠性。

## 2. 方法论：核心思想、关键技术细节与流程

CaAD 建立在 query-based 端到端自动驾驶框架之上，主要包含两个核心模块：

1. **以自车为中心的联合因果建模模块**；
2. **基于联合模态嵌入的因果感知策略对齐阶段**。

整体流程可以理解为：

> 先用监督学习构建自车条件下的联合场景表示，再用强化学习式策略对齐优化自车闭环行为。

---

### 2.1 基础架构

CaAD 基于类似 HiP-AD 的稀疏 query-based 端到端规划器，统一处理：

- 检测；
- 地图建模；
- 周围智能体运动预测；
- 自车规划。

基础模型中包含多类 task queries，并通过统一 decoder 进行迭代更新。模型原本会得到：

- 自车规划嵌入：\(x_e\)；
- 第 \(i\) 个周围车辆的边缘预测嵌入：\(x^a_i\)。

但这些边缘嵌入主要面向单个 actor 的预测，不能显式表示“同一个场景假设下自车与他车的联合未来”。

---

### 2.2 以自车为中心的联合因果建模

CaAD 引入 \(M\) 个以自车为中心的联合场景模态。每个模态 \(m\) 同时预测：

- 自车未来轨迹：\(\hat{\tau}^e_m\)；
- 所有关联 agent 的未来轨迹：\(\hat{\tau}^a_{i,m}\)。

一个联合场景假设可表示为：

\[
S_m = (\hat{\tau}^e_m, \hat{\tau}^a_{1,m}, \ldots, \hat{\tau}^a_{N,m})
\]

其中，同一个模态索引 \(m\) 表示同一个自车条件下的未来场景假设。

---

### 2.3 Marginal–Joint Embedding Interaction：边缘嵌入与联合模态嵌入交互

作者为自车和每个 agent 构建联合模态嵌入：

- 自车联合模态嵌入：\(z^e_m\)；
- agent 联合模态嵌入：\(z^a_{i,m}\)。

然后将边缘嵌入与联合模态嵌入拼接成 mode stack：

\[
U_e = [x_e; z^e_1; \ldots; z^e_M]
\]

\[
U^a_i = [x^a_i; z^a_{i,1}; \ldots; z^a_{i,M}]
\]

之后通过 **Agent-Mode Attention** 在模态维度上进行 attention：

\[
\bar{U}_e = Attn_{AM}(U_e, U_e, U_e)
\]

\[
\bar{U}^a_i = Attn_{AM}(U^a_i, U^a_i, U^a_i)
\]

其作用是：

- 让边缘 actor-specific 信息注入 joint-mode embedding；
- 让每个联合模态获得与该实体相关的场景上下文；
- 最终形成能同时解码自车和关键 agent 未来轨迹的联合场景表示。

---

### 2.4 交互相关智能体选择

CaAD 不对所有周围 agent 强制进行联合建模，因为许多远处车辆与自车意图无关。作者只选择与自车运动路径可能发生交互的 agent。

具体做法是：

- 使用自车 ground-truth 空间轨迹 \(\tau^e_{sp}\)；
- 使用每个 agent 的边缘多模态预测轨迹；
- 如果某个 agent 的某个预测轨迹与自车空间轨迹发生碰撞或交叉，则将其加入交互集合：

\[
A_{int} = \{ i \mid \exists k, collision(\tilde{\tau}^a_{i,k}, \tau^e_{sp}) = true \}
\]

作者强调使用 **空间轨迹** 而不是时间采样轨迹的原因是：

- 空间轨迹更清楚反映自车打算走哪条路径；
- 时间轨迹同时包含速度和时序信息，容易混淆交互意图；
- 实际交通交互是稀疏的，只需重点建模少数相关 agent。

---

### 2.5 自车中心的模态分配

传统联合预测方法通常根据所有 actor 轨迹的聚合 L2 误差选择最佳 joint mode：

> 哪个模态整体预测所有车辆最接近真实值，就选择哪个。

但 CaAD 认为，在端到端驾驶中，联合场景应围绕自车规划目标组织。因此，它采用 **ego-centric winner-takes-all** 模式选择：

\[
m^\star = \arg\min_m d(\hat{\tau}^e_m, \tau^e_{tp})
\]

即根据自车预测轨迹与自车真实时间轨迹之间的距离选择最佳联合模态。

之后：

- 对交互相关 agent，使用同一个 ego-selected mode \(m^\star\) 进行监督；
- 对非交互相关 agent，仍然使用各自边缘预测分支的最佳模态进行监督。

这使得联合建模聚焦于“某个自车意图下，关键 agent 如何响应”，而不是强行让全场景所有 agent 服从同一个联合模态。

---

### 2.6 联合分类与回归损失

联合分类损失包含三部分：

\[
L_{joint\_cls} =
L^e_{cls}(m^\star)
+
\sum_{i \in A_{int}} L^a_{cls,i}(m^\star)
+
\sum_{i \notin A_{int}} L^a_{cls,i}(k^\star_i)
\]

含义是：

- 自车使用 ego-selected joint mode；
- 交互相关 agent 使用同一个 joint mode；
- 非交互 agent 使用自身边缘预测的最佳模态。

回归部分则被改造成高斯负对数似然形式。模型不仅预测均值 \(\mu\)，还预测标准差 \(\sigma\)：

\[
L_{reg}(\tilde{m}) =
\frac{1}{2}
\left(
\frac{(\mu_{\tilde{m}} - \tau_{gt})^2}{\sigma_{\tilde{m}}^2}
+
2 \log \sigma_{\tilde{m}}
+
\log 2\pi
\right)
\]

这样做的目的是将确定性轨迹回归头转化为随机策略头，为后续强化学习式策略对齐提供基础。

---

### 2.7 因果感知策略对齐：Causality-Aware Policy Alignment

监督式模仿学习虽然能学习专家轨迹，但未必直接优化闭环指标，如：

- 安全性；
- 舒适性；
- 地图合规；
- 长期行驶进度；
- 与周围车辆的交互一致性。

因此，CaAD 在联合场景表示上进一步进行策略对齐。其关键点是：

> 强化学习不是作用在孤立的自车轨迹上，而是作用在 ego-centric joint-mode embedding 上，使每个自车 rollout 都在对应联合场景假设下被评估。

具体流程：

1. 对每个联合模态 \(m\)，采样 \(G\) 条自车 rollout；
2. 使用类似 NAVSIM PDMS 的 reward 评价每条 rollout；
3. reward 考虑周围交通和地图上下文；
4. 使用 GRPO 进行 group-relative 策略优化；
5. 只优化自车策略，周围 agent 预测仍保持监督学习。

reward 采用 penalty-plus-quality 结构，包含：

- NC：无责任碰撞；
- DAC：可行驶区域合规；
- DD：行驶方向合规；
- EP：自车进度；
- TTC：碰撞时间裕度；
- C：舒适性。

总体形式为：

\[
r =
\left(
\prod_{p \in \{NC,DAC,DD\}} score_p
\right)
\times
\frac{
\sum_{w \in \{EP,TTC,C\}} weight_w \cdot score_w
}{
\sum_{w \in \{EP,TTC,C\}} weight_w
}
\]

其中 EP、TTC、Comfort 的权重分别为 5.0、5.0、2.0。

GRPO 中，每个模态内的 reward 进行归一化，形成相对优势：

\[
A_{m,g} =
\frac{
r_{m,g} - mean(r_m)
}{
std(r_m)
}
\]

若发生碰撞，则优势被截断为 -1；否则取非负优势：

\[
\tilde{A}_{m,g} =
\begin{cases}
-1, & \text{if collision occurs} \\
\max(0, A_{m,g}), & \text{otherwise}
\end{cases}
\]

然后使用 clipped GRPO objective 更新策略。

最终训练目标为：

\[
L =
L_{e2e}
+
\lambda_{joint}(L_{joint\_reg} + L_{joint\_cls})
+
\lambda_{rl} L_{GRPO}
\]

其中：

- \(L_{e2e}\)：原始端到端监督损失；
- \(L_{joint}\)：联合预测损失；
- \(L_{GRPO}\)：策略对齐损失。

## 5. 实验数量与充分性

论文实验较为丰富，主要包括以下几类：

### 5.1 主结果实验

至少包含三个主要 benchmark / setting：

1. Bench2Drive closed-loop；
2. NAVSIM closed-loop / planning evaluation；
3. nuScenes open-loop。

其中 Bench2Drive 和 NAVSIM 是核心结果，nuScenes 是补充实验。

---

### 5.2 消融实验

论文做了多组消融，主要围绕以下因素：

- 是否使用 temporal cue 选择交互 agent；
- 是否使用 spatial cue 选择交互 agent；
- 是否进行 selected agents 的 joint prediction；
- 是否采用 ego-centric mode assignment；
- policy alignment 作用于 marginal representation 还是 joint scene representation；
- ego-centric policy alignment 与 all-agent policy alignment 的比较。

在 Bench2Drive-mini 上，作者构建了多个模型变体：

- HiP-AD Base；
- Model A：仅 marginal policy alignment；
- Model B：temporal cue + selected joint prediction + ego-centric assignment；
- Model C：spatial selection + joint prediction，但无 ego-centric assignment；
- Model D：spatial selection + selected joint prediction + ego-centric assignment；
- Model E：完整 CaAD。

此外，附录中还在完整 Bench2Drive routes 上比较：

- baseline；
- all-agent policy alignment；
- ego-centric policy alignment。

---

### 5.3 定性实验

论文提供了多组可视化结果：

- 复杂交互场景中 CaAD 能穿过车辆间隙，避免死锁；
- 在让行场景中先等待交互车辆通过，再继续前进；
- 模块消融可视化表明 joint-causal scene modeling 和 policy alignment 分别改善不同失败模式；
- NAVSIM 中展示 CaAD 相比 DiffusionDrive 更能避开未来冲突对象。

---

### 5.4 实验充分性与公平性评价

总体来看，实验较充分，理由包括：

- 覆盖两个主流闭环 benchmark；
- 与大量 SOTA 方法比较；
- 单独比较了 RL-based 方法，能验证其策略对齐部分是否优于已有 RL refinement；
- 做了较细粒度的 ability metrics，特别适合验证交互场景能力；
- 消融实验能较清楚地分离空间 agent 选择、联合预测、自车中心模态分配、策略对齐的贡献。

但也存在一些客观限制：

- 消融实验主要在 Bench2Drive-mini 上进行，不是所有消融都在完整 220 routes 上验证；
- 部分 baseline 结果来自论文或复现，完全一致的训练预算和实现细节未必能保证；
- NAVSIM 是 non-reactive simulation，背景车辆不响应自车行为，不能完全验证真实多智能体闭环交互；
- 论文强调因果建模，但实验主要体现为联合轨迹一致性和闭环指标提升，并没有严格因果识别或反事实验证。

---

## 6. 主要结论与发现

论文的主要结论包括：

- **显式建模自车与交互相关 agent 的联合未来，有助于提升闭环自动驾驶表现**；
- 仅仅做边缘预测或仅在 marginal representation 上做策略优化，效果有限；
- 自车中心的联合模态选择比传统 all-actor aggregate selection 更适合端到端规划；
- 使用空间轨迹选择交互相关 agent 优于使用时间轨迹；
- 策略对齐应作用在 joint scene representation 上，而不是孤立的 ego-only 或 marginal representation 上；
- 只对自车策略做 RL alignment 比对所有 agent 都做 alignment 更稳定；
- CaAD 在交互关键能力上提升明显，尤其是并线和超车。

主要定量结果包括：

### Bench2Drive

CaAD 达到：

- Driving Score：87.53；
- Success Rate：71.81%。

相比 HiP-AD：

- Driving Score 从 86.77 提升到 87.53；
- Success Rate 从 69.09 提升到 71.81。

在 ability metrics 上：

- Merging：60.00；
- Overtaking：86.67；
- Emergency Brake：83.33；
- Give Way：50.00；
- Traffic Sign：71.34；
- Mean：70.27。

该 Mean 高于 HiP-AD 的 65.98，也明显高于 DIVER 和 ReCogDrive。

### NAVSIM

CaAD 达到：

- NC：99.4；
- DAC：98.3；
- EP：84.9；
- TTC：95.9；
- PDMS：91.1。

PDMS 超过 ReCogDrive 的 90.8、DriveDPO 的 90.0 和 DiffusionDrive 的 88.1。

### nuScenes Open-loop

相比复现的 HiP-AD，CaAD：

- 平均 L2 从 0.67 降至 0.59；
- 平均碰撞率从 0.06 升至 0.08。

这说明 CaAD 在 open-loop 碰撞指标上并非全面占优，但闭环交互表现更强，也反映 open-loop 与 closed-loop 评估之间存在目标不一致。

## 7. 优点与亮点

### 7.1 方法设计上的亮点

- **明确指出端到端驾驶中的边缘预测局限**  
  论文抓住了现有方法中自车规划与周围 agent 预测“松耦合”的问题。

- **提出以自车为中心的联合场景模态**  
  不再将所有 agent 统一做全局 joint prediction，而是围绕自车意图组织关键交互对象。

- **交互 agent 选择具有实际合理性**  
  只对可能与自车路径发生冲突或交互的 agent 做联合监督，避免无关车辆带来噪声。

- **空间轨迹选择机制有效**  
  使用空间路径而不是时间轨迹筛选交互对象，更符合驾驶意图表达。

- **自车中心 winner-takes-all 模态分配合理**  
  使 joint mode 的定义更贴近规划目标，而非被全场景轨迹误差主导。

- **策略对齐与联合场景表示结合紧密**  
  RL 不只是简单优化 ego trajectory，而是在 joint-mode scene hypothesis 下评估 rollout。

- **只优化自车策略，保持 agent 预测监督稳定**  
  避免对所有 agent 使用 noisy reward 导致 credit assignment 不稳定。

---

### 7.2 实验设计上的亮点

- 同时覆盖 Bench2Drive 和 NAVSIM 两个重要闭环 benchmark；
- 与多类方法对比，包括 imitation-based、generative、world-model、RL-based 方法；
- 提供 ability-level 指标，能更清楚验证交互能力；
- 消融实验较系统，能支撑核心模块有效性；
- 定性结果直观展示了复杂交互场景中的行为改善。

---

## 8. 不足与局限

### 8.1 方法层面的局限

- **“因果”概念仍偏工程化**  
  论文使用 causality-aware、joint-causal 等表述，但主要实现是 ego-conditioned joint prediction 和 policy alignment，并没有严格的因果图建模、因果干预或反事实推断。

- **依赖交互 agent 选择质量**  
  如果关键 agent 没有被选入 \(A_{int}\)，联合建模效果会下降。

- **使用 ground-truth ego spatial trajectory 进行训练期 agent selection**  
  这在训练中可用，但可能引入训练与推理之间的分布差异。论文说明 joint prediction 分支主要用于训练监督，但这种设计仍可能限制泛化解释。

- **联合分支是 training-only**  
  推理阶段主要使用均值轨迹和原规划输出，联合预测如何持续影响推理行为需要通过训练后参数间接体现，解释性略弱。

- **奖励设计仍依赖手工指标**  
  PDMS-style reward 包含多个人工权重和阈值，可能影响不同场景下的行为偏好。

---

### 8.2 实验覆盖上的局限

- **真实世界验证不足**  
  实验主要在仿真和数据驱动 benchmark 上完成，没有真实车或真实闭环部署结果。

- **NAVSIM 是 non-reactive simulation**  
  背景 agent 不响应自车，因此不能完全验证交互式多智能体因果响应。

- **消融实验主要使用 Bench2Drive-mini**  
  虽然 mini 覆盖场景类型，但仍少于完整 benchmark，统计稳定性有限。

- **训练成本和推理效率报告不完整**  
  文中给出 Bench2Drive 的 GPU 数量和训练阶段，但缺少训练总时长、参数量、FPS、延迟等部署关键信息。

- **长尾社会行为仍困难**  
  作者明确指出，CaAD 仍不能可靠处理为紧急车辆让路等罕见社会行为。

---

### 8.3 偏差与应用风险

- benchmark 中的交互模式可能与真实城市驾驶存在差异；
- 模型可能学习到数据集中专家策略或场景分布的隐藏偏差；
- 在未覆盖的复杂社会规范场景中，策略对齐可能无法产生正确行为；
- 如果 reward 过度强调某些指标，可能牺牲人类驾驶员难以量化的礼让性或自然性；
- 闭环高分不等同于真实道路安全认证。

---

## 总体评价

CaAD 是一篇针对端到端自动驾驶交互建模问题的高质量工作。它的核心贡献在于将自车规划与交互相关 agent 预测组织到同一个 ego-centric joint scene representation 中，并进一步在该表示上进行策略对齐。实验结果显示，该设计在 Bench2Drive 和 NAVSIM 上均取得了领先性能，尤其在并线、超车、让行等交互关键场景中提升明显。

不过，论文中“因果”更多体现为自车条件下的联合场景建模，而非严格因果推断；同时，实验仍主要局限于仿真 benchmark，真实道路泛化、长尾社会行为处理、推理效率和部署成本仍需进一步验证。
