## 摘要
自动驾驶已从模块化流水线迈向端到端统一，而视觉-语言-动作（Vision-Language-Action, VLA）模型是这一进程在视觉到动作（Vision-to-Action, VA）之外的自然延伸。在实践中，驾驶 VLA 在规划质量上往往落后于 VA，这表明困难并非单纯源于模型规模，而在于语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的是 VLA 以孤立子任务改进的方式构建、无法组合成连贯驾驶能力的问题，而非 VLA 本身的问题。我们提出 MindVLA-U1，这是首个用于自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干网络在一次前向传播中基于同一共享表征生成自回归（AR）语言 token（可选）和基于流匹配的连续动作轨迹，同时保留各模态的自然输出形式。完整的流式设计以逐帧方式处理驾驶视频，而非在代价高昂的时间 VLM 建模下采用固定的视频-动作块。规划轨迹在帧间平滑演化，同时一个学习得到的流式记忆通道承载并更新时间上下文。该统一架构通过灵活的自注意力上下文管理，在密集与稀疏 MoT 骨干网络上支持快/慢系统，并为动作提供可度量的语言-控制路径：语言预测的驾驶意图通过无分类器引导（CFG）引导动作扩散，将语言侧意图转化为连续动作规划的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越有经验的人类驾驶员（8.20 RFS vs. 8.13 GT RFS），仅需 2 个扩散步骤；其规划 ADE 相较以往 VA/VLA 方法大幅达到最先进水平；并在保持用于人车交互的自然语言接口的同时，达到与 VA 相当的延迟（16 FPS vs. RAP 在 1B 规模下的 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces AR language tokens (optional) and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A full streaming design processes the driving video framewise rather than as fixed video-action chunks under costly temporal VLM modeling. Planned trajectories evolve smoothly across frames while a learned streaming memory channel carries temporal context and updates. The unified architecture enables fast/slow systems on dense & sparse MoT backbones via flexible self-attention context management, and exposes a measurable language-control path for action: language-predicted driving intents steers the action diffusion via classifier-free guidance (CFG), turning language-side intent into control signals for continuous action planning. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA by large margins, and matches VA latency (16 FPS vs. RAP's 18 FPS at 1B scale) while preserving natural language interfaces for human-vehicle interaction.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体架构：统一流式 VLA

MindVLA-U1 的核心是一个 **统一共享 VLM 骨干网络**，在一次前向传播中同时处理：

- 多视角图像 token；
- ego-state 历史状态；
- 语言 query / answer token；
- 流式 memory token；
- noisy action token。

输出包括：

- 可选的自回归语言回答；
- 基于 flow matching 的连续动作轨迹。

其关键设计原则是：

- **语言保持自回归形式**；
- **动作保持连续形式**；
- **语言与动作共享同一个 VLM 表征空间**；
- **动作 token 直接进入 Transformer 自注意力，而不是通过外部 action expert 处理**。

## 2.2 联合 AR 语言建模与 Flow Matching 动作生成

对于单帧输入，模型接收：

- 多视角视觉 token：\(I\)
- ego-state 历史：\(e\)
- 语言问题：\(q\)
- 语言答案 token：\(a = (a_1, ..., a_L)\)
- noisy action token：\(x_t = (x_{t,1}, ..., x_{t,N})\)

### 语言损失

语言部分使用标准自回归交叉熵：

\[
L_{AR} = - \sum_{l=1}^{L} \log p_\theta(a_l | I, e, q, a_{<l})
\]

语言 token 采用 causal mask。

### 动作损失

动作部分使用 flow matching。给定真实轨迹 \(x_0=w\)，噪声 \(\epsilon \sim \mathcal{N}(0,I)\)，构造 noisy action：

\[
x_t = t\epsilon + (1-t)x_0
\]

其中 \(t \sim Beta(1.5, 1.0)\)，约定 \(t=0\) 为数据、\(t=1\) 为噪声。

动作头预测速度场：

\[
L_{FM} = \|v_\theta(x_t,t;I,e,q,a) - (\epsilon - x_0)\|^2
\]

总损失为：

\[
L = L_{AR} + L_{FM}
\]

推理时，通过 ODE / Euler integration 从噪声逐步恢复轨迹。论文主要结果使用 **2 个 diffusion / flow matching 步骤**。

## 2.3 Intent-CFG：语言到动作的可度量桥接

论文提出 **Intent-CFG** 作为语言侧语义状态到连续动作规划的显式路径。

流程如下：

1. VLM 语言头预测当前驾驶意图 token，例如：
   - left；
   - right；
   - go straight；
   - 或 MindLabel 扩展的 20 类意图。

2. 将预测意图 token 的 embedding 加入动作 MLP 的时间 embedding。

3. 训练时以一定概率将 intent 替换为 unconditional token \(\varnothing\)，形成 classifier-free guidance 训练。

4. 推理时分别计算：
   - 条件速度场 \(v_\theta(x_t,t;z)\)
   - 无条件速度场 \(v_\theta(x_t,t;\varnothing)\)

5. 使用 CFG 混合：

\[
v_{cfg}(x_t,t) = v_\theta(x_t,t;\varnothing) + s \left(v_\theta(x_t,t;z) - v_\theta(x_t,t;\varnothing)\right)
\]

其中论文实现中 guidance scale 为 \(s=1.5\)。

该机制的重要意义在于：

- 语言不只是“存在一个语言头”；
- 语言预测出的驾驶意图能实际改变连续轨迹；
- 可以通过实验量化语言对动作规划的贡献。

## 2.4 流式记忆机制：替代多帧 VLM 输入

MindVLA-U1 不使用固定视频 chunk 或多帧堆叠输入，而是逐帧处理驾驶视频。

每一帧输入：

- 当前多视角图像；
- 当前 ego-state；
- FIFO memory channel 中的历史 memory token。

### 记忆读取

历史 memory token 存储在过去帧的 ego 坐标系中。读取时通过 SE(2) 变换对齐到当前 ego 坐标系。

相对位姿为：

\[
T_{j \to i} = P_i \cdot P_j^{-1}
\]

然后使用旋转、平移和时间偏移构成 motion-aware modulation，对 memory token 进行调制。

### 记忆更新

每帧前向之后，使用 Q-Former 风格的 propagation transformer：

- 从当前 backbone 输出中压缩出新的 memory token；
- 加入 FIFO memory；
- 超出容量时删除最老 memory。

默认实现：

- 每帧写入 \(N_m=128\) 个 memory token；
- FIFO 保存 \(N_g=2\) 个历史帧条目；
- 总 memory 容量为 256 token。

这种设计避免了多帧视频 token 冗余输入，同时保持长期时序连续性。

## 2.5 快 / 慢推理系统与 MoT

论文通过注意力 mask 和 token 顺序管理实现多种推理模式：

- `vqa_first`：先语言推理，再动作规划；
- `action_first`：先规划动作，再生成语言解释；
- `action_only`：跳过语言生成，快速规划；
- `vqa_only`：只做语言问答。

其意义是：

- 慢模式用于复杂、安全关键场景；
- 快模式用于常规驾驶控制；
- 同一个 checkpoint 支持不同推理路径。

此外，论文提出稀疏 **Mixture-of-Transformers（MoT）** 变体：

- context group：视觉和语言 token；
- action group：memory、ego-state 和 action token；
- 两组共享 self-attention K/V 池；
- FFN 使用不同专家。

推荐分组为：

\[
\{V,L\} + \{M,S,A\}
\]

其中：

- \(V\)：vision；
- \(L\)：language；
- \(M\)：memory；
- \(S\)：ego-state；
- \(A\)：action。

默认主模型使用：

- backbone：Qwen3-VL-2B；
- GPU：8 × NVIDIA H200；
- 训练精度：BF16；
- 分布式训练：DeepSpeed ZeRO-2；
- 训练轮数：50 epochs；
- 训练时间：约 7 小时。

优化器与超参数：

- AdamW；
- 学习率 \(1 \times 10^{-4}\)；
- weight decay 0.1；
- warmup 1,000 iterations；
- cosine annealing；
- flow matching component weights：
  - position：1.0；
  - velocity：0.5；
  - acceleration：0.5。

### 推理资源

推理延迟在：

- batch size = 1；
- 1 × H200；

上测试。

主要结果：

- Qwen3-VL-2B dense slow path：约 2,594 ms，0.39 FPS；
- Qwen3-VL-2B action_only fast path：约 103 ms，9.70 FPS；
- Qwen3-VL-2B vqa_first_fast，2 steps：约 108 ms，9.26 FPS；
- Qwen3-VL-2B vqa_first_fast，1 step：约 63 ms，15.92 FPS；
- InternVL-2 1B fast 配置：64 ms，15.55 FPS；
- 对比 RAP-DINO：57 ms，17.68 FPS。

论文指出慢速路径主要瓶颈是自回归 VQA 解码，约占 94% 延迟。

## 5.2 实验充分性评价

- 从自动驾驶 VLA 论文角度看，实验覆盖较充分：
  - 有验证集和官方测试集；
  - 有 VA 与 VLA 两类强基线；
  - 有关键组件消融；
  - 有速度、显存、延迟分析；
  - 有 scaling 分析；
  - 有 benchmark 分布偏移诊断。

- 但仍存在若干限制：
  - 所有 headline 结果集中在 WOD-E2E 一个数据集；

## 6. 主要结论与发现

- MindVLA-U1 在 WOD-E2E 上显著提升 VLA 自动驾驶规划性能。

- 在 validation split 上：
  - MindVLA-U1 dense：RFS 7.83；
  - MindVLA-U1 + Intent-CFG：RFS 7.92；
  - MindVLA-U1 + MoT：RFS 7.92；
  - MindVLA-U1 + RL：RFS 8.20；
  - 人类 logged trajectory 的 RFS 为 8.13。

- 在 official test split 上：
  - MindVLA-U1：RFS 7.77；
  - MindVLA-U1 + RL：RFS 7.87；
  - 高于此前 VA / VLA 方法。

- Intent-CFG 显示语言侧意图可以实际影响连续轨迹：
  - no intent baseline：RFS 7.83；
  - NTP-predicted Intent-CFG：RFS 7.92。

- 流式记忆有效优于多帧 VLM 堆叠：
  - chunk-wise single frame：RFS 7.69；
  - chunk-wise + 4-frame image sequence：RFS 7.61；
  - streaming no memory：RFS 7.73；
  - streaming + memory：RFS 7.83。

- MoT 能在保持规划性能的同时提供快 / 慢系统结构：
  - 最好 MoT 变体可达 RFS 8.01；
  - 推荐的 \(\{V,L\}+\{M,S,A\}\) 达到 RFS 7.92，并具有更好的推理结构分离。

- VLM 规模并非简单单调提升规划性能：
  - Qwen3.5-VL 2B 在默认训练预算下最好；
  - 9B 延长训练后有所恢复；
  - 说明当前实验不能证明清晰的 VLA scaling law。

- 语言质量和动作质量存在结构性解耦：
  - base VLM 的 BLEU / ROUGE 明显下降；
  - 但 RFS 几乎保持；
  - 说明动作控制更多依赖共享场景表征和 intent embedding，而非表面语言生成质量。

## 7. 优点与亮点

- **统一架构设计清晰**  
  将语言生成和连续动作规划统一在同一个 VLM backbone 中，而不是用语言 token 离散化动作，也不是外接独立 action expert。

- **动作保持连续形式**  
  通过 flow matching 直接生成连续轨迹，避免语言 token 化带来的精度瓶颈。

- **语言到动作路径可度量**  
  Intent-CFG 明确展示语言侧意图可以控制动作扩散，是论文相较许多 VLA 工作的重要亮点。

- **流式范式适合自动驾驶**  
  逐帧处理 + FIFO memory 更符合真实驾驶流，而不是固定视频块和动作 chunk。

- **兼顾性能与实时性**  
  同一个 checkpoint 支持慢速语言推理和快速 action-only 推理，缓解 VLA 推理成本问题。

- **实验覆盖丰富**  
  包括主对比、组件消融、速度测试、RL 后训练、backbone scaling、分布偏移诊断和定性分析。

- **MindLabel 数据管线有扩展价值**  
  自动生成 scene QA、intent、dreamed trajectories 和 trajectory-evaluation QA，为后续 VLA 训练提供了更丰富监督形式。

- **对 benchmark 分布偏移有自我诊断**  
  论文没有只报告分数，也分析了 WOD-E2E val/test intent distribution shift，这一点较为客观。

## 8. 不足与局限

- **仅为 open-loop 评估**  
  WOD-E2E 是离线 logged benchmark。RFS 和 ADE 不能完全代表闭环驾驶安全性、交互性和长期稳定性。

- **单一主 benchmark**  
  所有核心结果主要来自 WOD-E2E，缺少 nuScenes、NAVSIM、CARLA closed-loop 或真实车部署验证。

- **“超过人类驾驶员”需谨慎解读**  
  论文中的人类参考是 logged GT trajectory 在 RFS 下的评分，不等价于真实道路闭环驾驶能力超过人类。

- **MindLabel 自动标注可能有偏差**  
  标注依赖 Qwen3-VL 和 Qwen3.5-Plus，可能继承这些模型的视觉理解偏差、语言偏差或幻觉问题。

- **基线公平性仍有复杂因素**
  不同方法的 backbone、外部 AD 预训练、数据规模、推理模式和是否使用 RL 并不完全一致。虽然论文标注了 AD-PT，但完全公平比较仍较困难。

- **Intent-CFG 当前主要使用 3 类 GT intent**
  left / right / straight 的意图空间较粗，难以覆盖 waiting、yielding、emergency stop、parking 等复杂长尾场景。

- **推理实时性仍依赖 fast path**
  完整慢速 VQA decoding 路径非常慢，约 0.39 FPS。实际部署中需要异步推理、缓存、蒸馏或降低语言更新频率。

- **scaling law 尚不明确**
  论文显示 VLM 参数规模增大并不必然提升规划性能，但受训练 schedule、数据规模、action head 容量等因素影响，不能得出普适结论。

- **RL 后训练可能优化 rater 偏好而非真实安全**
  GRPO 只用 RFS reward，可能让模型更贴近 rater panel 偏好，但不一定等价于闭环安全性或法规合规性。
