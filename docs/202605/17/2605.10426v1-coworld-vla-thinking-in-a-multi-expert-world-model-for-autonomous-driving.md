## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景的范式。然而，现有推理机制仍难以提供面向规划的中间表示：文本形式的思维链（Chain-of-Thought, CoT）无法保留连续的时空结构，而潜在世界推理仍难以作为动作生成的直接条件。本文提出 CoWorld-VLA，一个用于自动驾驶的多专家世界推理框架，其中世界表示作为显式条件来指导动作规划。CoWorld-VLA 通过多源监督提取互补的世界信息，并将其编码为 VLA 内的专家 token，从而提供规划器可访问的条件信号。具体而言，我们构建了四类 token：语义交互 token、几何结构 token、动态演化 token 和自车轨迹 token，它们分别建模交互意图、空间结构、未来时间动态和行为目标。在动作生成过程中，CoWorld-VLA 采用基于扩散的分层多专家融合规划器，并在联合去噪过程中始终与场景上下文耦合，以生成连续的自车轨迹。实验表明，CoWorld-VLA 在 NAVSIM v1 基准上的未来场景生成和规划任务中均取得了具有竞争力的结果，在避碰和轨迹精度方面表现出较强性能。消融研究进一步验证了专家 token 的互补性，以及它们作为动作生成规划条件的有效性。代码将发布于 https://github.com/potatochip1211/CoWorld-VLA。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for end-to-end autonomous driving. However, existing reasoning mechanisms still struggle to provide planning-oriented intermediate representations: textual Chain-of-Thought (CoT) fails to preserve continuous spatiotemporal structure, while latent world reasoning remains difficult to use as a direct condition for action generation. In this paper, we propose CoWorld-VLA, a multi-expert world reasoning framework for autonomous driving, where world representations serve as explicit conditions to guide action planning. CoWorld-VLA extracts complementary world information through multi-source supervision and encodes it into expert tokens within the VLA, thereby providing planner-accessible conditioning signals. Specifically, we construct four types of tokens: semantic interaction, geometric structure, dynamic evolution, and ego trajectory tokens, which respectively model interaction intent, spatial structure, future temporal dynamics, and behavioral goals. During action generation, CoWorld-VLA employs a diffusion-based hierarchical multi-expert fusion planner, which is coupled with scene context throughout the joint denoising process to generate continuous ego trajectories. Experiments show that CoWorld-VLA achieves competitive results in both future scene generation and planning on the NAVSIM v1 benchmark, demonstrating strong performance in collision avoidance and trajectory accuracy. Ablation studies further validate the complementarity of expert tokens and their effectiveness as planning conditions for action generation. Code will be available at https://github.com/potatochip1211/CoWorld-VLA.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 总体框架

CoWorld-VLA 采用三阶段训练流程：

1. **Stage 1：动作条件未来世界模型预训练**
   - 使用 Wan 视频生成模型的潜在空间学习未来场景演化。
   - 输入历史图像与文本化的自车意图、速度、导航和轨迹提示。
   - 输出未来视频 latent，用于学习动态世界模型。

2. **Stage 2：多专家世界表征学习**
   - 以 Qwen3-VL 作为 VLM backbone。
   - 在 VLM 输入序列中插入四类专家 token：
     - 语义交互 token
     - 几何结构 token
     - 动态演化 token
     - 自车轨迹 token
   - 通过多源监督使这些 token 学习不同类型的世界知识。

3. **Stage 3：分层多专家融合规划器**
   - 使用 Hierarchical Multi-Expert Fusion，简称 HMEF。
   - 以专家 token 和场景上下文为条件，在扩散去噪过程中生成连续轨迹。

## 2.2 问题形式化

论文将自动驾驶轨迹规划表示为条件动作生成问题：

- 输入包括：
  - 当前前视图像 \(o_t\)
  - 导航指令 \(\kappa_t\)
  - 历史自车位姿
  - 当前速度和加速度

- 标准 VLA 形式为：

\[
p_\theta(A_{t+1:t+T} \mid o_t, c_t)
\]

即直接根据观测和条件预测未来轨迹。

- CoWorld-VLA 引入多专家潜在世界表示：

\[
Z = \{z_{sem}, z_{geo}, z_{dyn}, z_{traj}\}
\]

因此轨迹生成变为：

\[
p_\theta(A_{t+1:t+T} \mid o_t, c_t, Z)
\]

其中：
- \(z_{sem}\)：语义交互知识
- \(z_{geo}\)：几何结构知识
- \(z_{dyn}\)：动态演化知识
- \(z_{traj}\)：自车轨迹行为先验

这些表示共同构成论文所称的 **planning-oriented Latent CoT**。

## 2.3 Stage 1：动作条件预测世界模型

- 使用冻结的 Wan VAE 将视频序列编码到 latent 空间：

\[
z = E_{vae}(x) = [z_h, z_f]
\]

其中：
- \(z_h\)：历史帧 latent
- \(z_f\)：未来帧 latent

- 对未来 latent 添加噪声：

\[
\tilde{z}_{f,\sigma} = (1-\sigma)z_f + \sigma \epsilon
\]

- 速度目标为：

\[
v_{target} = \epsilon - z_f
\]

- 历史 latent 保持无噪声，作为确定性上下文。

- 文本条件由四部分组成：

\[
P = [Scene] \oplus [Speed] \oplus [Navigation] \oplus [Trajectory]
\]

包括：
- 场景描述
- 粗粒度速度描述
- 导航命令，如左转、直行、右转
- 自车未来轨迹点

- 训练目标是 flow matching loss：

\[
L_{flow} = \mathbb{E}\left[\|F_\theta(\tilde{z}_\sigma,c,\sigma)_f - (\epsilon - z_f)\|_2^2\right]
\]

该阶段的作用是获得能够预测未来场景动态的基础世界模型。

## 2.4 Stage 2：多专家表示学习

该阶段的核心是将多种外部专家模型的知识注入 VLM 隐空间。

### 四类专家 token

- **Semantic Interaction Token**
  - 由 V-JEPA 提供监督。
  - 关注高层语义、交通参与者互动和预测性语义信息。

- **Geometric Structure Token**
  - 由 VGGT 提供监督。
  - 关注道路布局、空间结构、三维几何约束。

- **Dynamic Evolution Token**
  - 由 Stage 1 中训练的 Wan 世界模型提供监督。
  - 用于学习未来视觉动态和时间一致性。

- **Ego Trajectory Token**
  - 由真实未来自车轨迹监督。
  - 用于学习直接服务于自车行为目标的表示。

### VLM 表示生成

Qwen3-VL 接收图像、文本和专家 token，输出：

\[
\{H_{ctx}, H_{sem}, H_{geo}, H_{dyn}, H_{traj}\}
\]

其中：
- \(H_{ctx}\)：上下文 VLM hidden states
- \(H_{sem}\)：语义交互 token 表示
- \(H_{geo}\)：几何结构 token 表示
- \(H_{dyn}\)：动态演化 token 表示
- \(H_{traj}\)：轨迹 token 表示

### 多专家监督损失

- JEPA 语义监督：

\[
L_{sem} = \lambda_{l1} SmoothL1(\hat{Z}_{sem}, Z_{sem}) + \lambda_{cos}(1 - cos(\hat{Z}_{sem}, Z_{sem}))
\]

- VGGT 几何监督：

\[
L_{geo} = MSE(\hat{Z}_{geo}, Z_{geo})
\]

- Wan 动态世界模型监督：

\[
L_{dyn} = L_{flow}(\hat{o}_{fut}, o_{fut}; o_t, H_{dyn})
\]

- 自车轨迹监督：

\[
L_{traj} = MSE(\hat{A}_{t+1:t+T}, A_{t+1:t+T})
\]

- 总损失为：

\[
L_{total} = w_{dyn}L_{dyn} + w_{sem}L_{sem} + w_{geo}L_{geo} + w_{traj}L_{traj}
\]

论文中使用的权重为：
- \(w_{dyn}=1.0\)
- \(w_{sem}=0.1\)
- \(w_{geo}=0.1\)
- \(w_{traj}=1.0\)

## 2.5 Stage 3：分层多专家融合规划器 HMEF

HMEF 的目标是将 Stage 2 学到的专家 token 转化为最终可执行轨迹。

### 输入

- 场景 token：
  - VLM 上下文 token
  - 当前帧 JEPA token
  - 当前帧 VGGT token

- 专家 action token：

\[
H_{act} = \{H_{sem}, H_{geo}, H_{dyn}, H_{traj}\}
\]

### 场景压缩

- 由于场景 token 序列较长，使用 Perceiver-style compressor 将其压缩为固定数量的 latent context tokens。

### 专家编码

- 每个专家分支使用独立的双向 Transformer 编码。
- 将专家特征对齐到未来规划时间步。
- 得到每个时间步的专家特征 \(F_{e,t}\)。

### 扩散式轨迹生成

- 将真实轨迹归一化到 \([-1,1]\)。
- 训练时采样噪声强度 \(\tau\)，构造带噪轨迹：

\[
A_\tau = (1-\tau)\epsilon + \tau A_{norm}
\]

- Denoiser 采用双流结构：
  - clean scene stream
  - noisy expert-conditioned action stream

- 两个流之间进行 joint self-attention。

- 每个专家分支输出一个轨迹预测 \(\hat{A}_e\)，训练损失为：

\[
L_{diff} = \frac{1}{N_e}\sum_{e=1}^{N_e}\|\hat{A}_e - A_{norm}\|_2^2
\]

### 专家融合

- 模型学习全局融合权重：

\[
\alpha = softmax(w)
\]

- 融合轨迹为：

\[
\bar{A} = \sum_{e=1}^{N_e}\alpha_e \hat{A}_e
\]

- 总动作损失：

\[
L_{act} = L_{diff} + \lambda_{fusion}\|\bar{A} - A_{norm}\|_2^2
\]

- 推理时从高斯噪声开始，迭代去噪并融合专家轨迹，得到最终可执行自车轨迹。

## 6. 主要结论与发现

- CoWorld-VLA 在 NAVSIM navtest 上取得 **PDMS 89.8**，优于文中对比的多种端到端、世界模型和 VLA 方法。
- 在子指标上：
  - NC 达到 99.2，说明避碰能力较强。
  - EP 达到 83.6，说明在保证安全的同时保持较好的前进效率。
  - TTC、DAC 和 Comfort 也保持较高水平。
- 视频生成方面，CoWorld-VLA 的 FVD 为 **32.7**，优于 SVD、GenAD、DrivingGPT、Epona 和 DriveLaW 等方法。
- 消融实验显示：
  - 仅使用 Ego Trajectory Token 时 PDMS 为 83.7。
  - 加入 Geometry 后提升到 85.1。
  - 再加入 Semantic 后提升到 87.7。
  - 加入 Dynamic Evolution 后提升到 88.7。
  - 说明四类专家 token 具有互补性。
- HMEF 进一步提升规划性能：
  - Stage 2 VLM：PDMS 88.7
  - 加入 ReCogDrive action expert：PDMS 89.1
  - 使用完整 HMEF：PDMS 89.8
- 扩散去噪步数方面，10 steps 表现最好，说明其在精度和效率之间取得较好平衡。
- 学习到的专家融合权重显示：
  - 动态演化专家约 0.35
  - 轨迹专家约 0.31
  - 语义专家约 0.19
  - 几何专家约 0.15  
  表明模型在规划中更依赖动态演化和轨迹相关信息。

## 7. 优点与亮点

- **提出多专家 Latent CoT 机制**  
  相比文本 CoT，Latent CoT 更适合表达连续空间和时间信息。

- **将世界模型知识显式用于规划**  
  不是只把世界模型作为辅助训练分支，而是在推理时将专家 token 作为轨迹生成条件。

- **专家信息设计较全面**
  - 语义交互：处理交通参与者意图与高层语义。
  - 几何结构：处理道路布局和三维空间约束。
  - 动态演化：处理未来场景变化。
  - 自车轨迹：处理行为目标和可执行运动。

- **多源监督具有互补性**
  - V-JEPA 提供语义预测表征。
  - VGGT 提供几何结构先验。
  - Wan 提供视频动态世界模型。
  - 轨迹回归提供行为目标监督。

- **HMEF 规划器设计较有针对性**
  - 使用 Perceiver 压缩长场景 token，降低计算成本。
  - 每类专家使用独立编码器。
  - 通过扩散式去噪生成连续轨迹。
  - 学习专家融合权重，提高不同专家的协同能力。

- **实验结果较强**
  - 在 NAVSIM v1 上取得最高 PDMS。
  - 在视频生成 FVD 上表现优异。
  - 消融实验清楚展示了各模块贡献。

- **论文透明度较好**
  - 给出了训练步数、GPU 数量、训练时间、学习率等详细实现信息。
  - 附录进一步说明了 prompt 构造、token 组织、融合权重训练等细节。

---

## 8. 不足与局限

- **训练成本很高**
  - 三阶段训练总计使用大量 A800 GPU：
    - Stage 1：64 张 A800，约 74 小时
    - Stage 2：32 张 A800，约 70 小时
    - Stage 3：16 张 A800，约 20 小时
  - 对普通研究机构或工业快速迭代不够友好。

- **当前只使用单图像输入**
  - 论文明确指出当前模型限制在 single-image inputs。
  - 自动驾驶通常需要多帧、多视角甚至多模态信息，单帧前视图像可能不足以处理遮挡、复杂交互和长期动态。

- **跨数据集泛化验证有限**
  - 规划主结果集中在 NAVSIM v1。
  - 缺少在 nuScenes、Waymo、CARLA 或真实闭环环境中的系统验证。

- **视频生成对比的公平性有限**
  - 表 2 中不同方法对应的数据集不完全一致。
  - FVD 对比可能受到数据分布、分辨率、评估协议差异影响。

- **缺少真实闭环或实车部署实验**
  - NAVSIM 是 non-reactive open-loop protocol。
  - 该协议不能完全反映真实交通参与者会对自车行为产生反应的闭环环境。
  - 实际部署中的鲁棒性、安全性和异常场景处理能力仍未充分验证。

- **推理效率分析不足**
  - 虽然规划使用 10 步扩散采样，但论文没有系统报告推理延迟、显存占用、FPS 或车载部署可行性。
  - 对实时自动驾驶系统而言，这一点非常关键。

- **依赖多个大型基础模型**
  - 包括 Wan2.2-5B、Qwen3-VL-2B、V-JEPA、VGGT 等。
  - 系统复杂度较高，训练、调参和维护难度较大。
  - 不同专家模型的偏差可能传递到最终规划器。

- **专家 token 的可解释性仍有限**
  - 虽然论文称其为 Latent CoT，但这些 token 是连续隐变量。
  - 相比文本推理，其内部含义不易直接解释。
  - 对安全关键自动驾驶场景而言，仍需要更强的可验证性和可审计性。

- **规划安全性仍主要由 benchmark 指标衡量**
  - 虽然 NC、TTC 等指标较好，但缺少对极端场景、长尾场景、传感器噪声、分布外场景的深入分析。

# 总体评价

CoWorld-VLA 是一篇面向自动驾驶 VLA 规划的较完整工作。它针对文本 CoT 不适合连续时空规划、单一世界模型表征不足、世界模型与规划器耦合不强等问题，提出了多专家 Latent CoT 和扩散式融合规划器。方法设计系统，实验结果强，尤其在 NAVSIM v1 上展现出较好的避碰与轨迹精度。

不过，该方法也明显依赖大规模算力和多个基础模型，系统复杂度高，且目前主要在单帧前视相机和 NAVSIM benchmark 上验证。未来若能扩展到多摄像头、多帧、多模态输入，并补充真实闭环仿真、推理效率和长尾安全场景分析，其说服力和应用价值会进一步增强。
