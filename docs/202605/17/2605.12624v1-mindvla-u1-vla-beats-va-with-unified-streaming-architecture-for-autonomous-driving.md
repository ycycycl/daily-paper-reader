## 摘要
自动驾驶已从模块化流水线逐步迈向端到端统一，而视觉-语言-动作（Vision-Language-Action, VLA）模型是这一进程在视觉到动作（Vision-to-Action, VA）之外的自然延伸。在实践中，驾驶 VLA 在规划质量上常常落后于 VA，这表明困难并不单纯在于模型规模，而在于将语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的不是 VLA 本身的问题，而是 VLA 的构建方式——将其作为彼此孤立的子任务改进，未能组合成连贯的驾驶能力。我们提出 MindVLA-U1，这是首个用于自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干网络在一次前向传播中基于同一个共享表示，同时生成自回归语言 token 和通过流匹配得到的连续动作轨迹，从而保留各模态的自然输出形式。流式设计按帧处理驾驶视频，而不是将其作为固定的视频-动作块；同时，学习得到的记忆通道在帧间传递时间上下文，使规划轨迹能够平滑演化，而无需冗余的多帧 VLM 建模。该统一架构通过灵活的自注意力上下文管理，支持在稠密/稀疏 Mixture-of-Transformers（MoT）骨干网络上的快/慢执行，并揭示了一条可测量的语言到动作路径：由语言预测的驾驶意图通过无分类器引导（classifier-free guidance, CFG）引导动作扩散，将语言侧意图转化为连续轨迹生成的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越有经验的人类驾驶员（8.20 RFS 对比 8.13 GT RFS），且仅需 2 个扩散步；在规划 ADE 指标上以大幅优势超过此前的 VA/VLA 方法，达到最先进水平；同时在保留自然语言接口的情况下匹配 VA 级吞吐量（16 FPS 对比 RAP-DINO 的 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose into coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces autoregressive language tokens and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A streaming design processes the driving video framewise rather than as fixed video-action chunks, while a learned memory channel carries temporal context across frames so planned trajectories evolve smoothly without redundant multi-frame VLM modeling. The unified architecture admits fast/slow execution on dense/sparse Mixture-of-Transformers (MoT) backbones via flexible self-attention context management, and exposes a measurable language-to-action route: a language-predicted driving intent steers action diffusion through classifier-free guidance (CFG), turning language-side intent into a control signal for continuous trajectory generation. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA methods by large margins, and matches VA-class throughput (16 FPS vs. RAP-DINO's 18 FPS) while preserving natural-language interfaces.

### 1.2 论文核心观点

本文认为：  
**VLA 落后于 VA 并不是 VLA 范式本身的问题，而是现有 VLA 架构接口设计不合理的问题。**

## 2. 方法论：核心思想、关键技术与算法流程

## 2.1 总体架构：统一流式 VLA

MindVLA-U1 的核心是一个 **统一共享 VLM 骨干网络**。

在同一个 forward pass 中，模型同时处理：

- 多视角图像 token；
- ego-state 历史状态；
- 语言 query / answer token；
- streaming memory token；
- noisy action token。

并同时输出：

- 自回归语言 token；
- 连续动作轨迹的 flow-matching velocity field。

其关键理念是：

> 语言保持自回归，动作保持连续，两者共享同一个 VLM 表征，而不是将动作离散化为语言 token，也不是使用完全独立的动作专家。

---

## 2.2 统一共享骨干：AR 语言 + Flow Matching 动作

### 输入

在单帧时间步，模型输入包括：

- 多视角视觉 token：\(I\)
- ego-state 历史：\(e\)
- 语言 query：\(q\)
- 语言 answer token：\(a = (a_1, ..., a_L)\)
- noisy action token：\(x_t = (x_{t,1}, ..., x_{t,N})\)

这些 token 共同进入共享 VLM backbone \(f_\theta\)。

### 语言目标：自回归损失

语言部分使用标准 next-token prediction：

\[
L_{AR} = - \sum_{l=1}^{L} \log p_\theta(a_l | I, e, q, a_{<l})
\]

也就是说，语言头负责生成场景 QA、意图 token 等语言输出。

### 动作目标：Flow Matching 连续轨迹生成

动作部分使用 flow matching / diffusion-style 轨迹生成。

给定真实未来轨迹 \(x_0 = w\)，随机噪声 \(\epsilon \sim \mathcal{N}(0, I)\)，构造 noisy action：

\[
x_t = t\epsilon + (1-t)x_0
\]

其中 \(t \sim Beta(1.5, 1.0)\)，约定：

- \(t=0\)：真实数据；
- \(t=1\)：纯噪声。

动作头预测 velocity field：

\[
L_{FM} = \| v_\theta(x_t, t; I,e,q,a) - (\epsilon - x_0) \|^2
\]

训练总目标为：

\[
L = L_{AR} + L_{FM}
\]

推理时通过 ODE / Euler integration 从噪声逐步生成连续未来轨迹。

### 与既有 VLA 的区别

MindVLA-U1 与已有方法的核心区别是：

- 不把轨迹坐标离散化为语言 token；
- 不通过独立 action expert 脱离 VLM；
- noisy action token 直接进入共享 Transformer self-attention；
- 语言与动作共享每一层表征与梯度。

## 2.3 语言到动作桥接：Intent-CFG

论文提出 **Intent-CFG** 作为语言到连续动作的显式控制路径。

### 核心思路

模型先通过语言头预测驾驶意图 token，例如：

- left；
- right；
- go straight；
- yielding；
- stopping；
- waiting 等。

然后将该 intent embedding 加入动作扩散模型的 time embedding，用于引导 velocity field。

### CFG 机制

训练时，部分样本将 intent 替换为空条件 token \(\emptyset\)，类似 classifier-free guidance dropout。

推理时分别计算：

- 无条件 velocity field：\(v_\theta(x_t,t;\emptyset)\)
- 有条件 velocity field：\(v_\theta(x_t,t;z)\)

再按 guidance scale \(s\) 混合：

\[
v_{cfg}(x_t,t) =
v_\theta(x_t,t;\emptyset)
+
s \left(
v_\theta(x_t,t;z) - v_\theta(x_t,t;\emptyset)
\right)
\]

其中论文实现中 guidance scale 为 \(s=1.5\)。

### 意义

这一机制使得语言预测出的 intent 可以实际改变连续轨迹生成，因此不只是“模型有语言头”，而是存在可测量的语言到动作因果路径。

## 2.4 流式时序建模：Streaming Memory

MindVLA-U1 不使用固定 video chunk，而是逐帧处理驾驶视频。

每一帧输入：

- 当前多视角图像；
- 当前 ego-state；
- 从历史帧传来的 compact memory token。

### FIFO 记忆通道

系统维护一个有界 FIFO memory buffer：

- 每帧写入一组压缩 memory token；
- 超过容量后移除最旧 memory；
- 每一步计算成本保持固定。

默认实现中：

- 每帧写入 \(N_m=128\) 个 memory token；
- FIFO 保存 \(N_g=2\) 个 frame-step；
- 总 memory 容量为 \(256\) 个 token。

### Motion-aware alignment

历史 memory 是在旧 ego pose 下写入的，因此读取时要根据当前 ego pose 做 SE(2) 对齐。

论文使用相对变换：

\[
T_{j \to i} = P_i \cdot P_j^{-1}
\]

其中 \(P_i, P_j\) 是车辆位姿。

然后将：

- 旋转 \((\cos\phi, \sin\phi)\)
- 平移 \((\delta_x, \delta_y)\)
- 时间偏移

编码成调制向量，加到 memory token 上。

### Memory 更新

每帧 forward 后，使用 Q-Former-style propagation transformer：

- 从 backbone output 中压缩得到新 memory；
- 写入 FIFO；
- 下一帧继续读取。

重要的是，训练和推理都使用相同的 read-forward-write 流式机制，因此部署行为和训练行为一致。

## 2.5 快 / 慢执行与 MoT

MindVLA-U1 通过 attention mask 和 token 管理支持多种推理模式：

1. **vqa_first**
   - 先生成语言 / reasoning，再用语言条件生成动作；
   - 适合复杂、安全关键场景。

2. **action_first**
   - 先生成动作，再生成对该动作的语言解释；
   - 适合可解释驾驶。

3. **action_only**
   - 去掉语言解码，只执行动作规划；
   - 适合实时控制。

4. **parallel / vqa_only 等变体**
   - 用于不同部署需求。

### Sparse MoT 设计

论文还实现了 Mixture-of-Transformers 变体。

MoT 将 token 分成两组：

- context group：视觉、语言；
- action group：memory、ego-state、action。

两组拥有不同 Q/K/V/O 投影和 FFN expert，但共享 attention K/V pool，因此仍能跨模态融合。

优势是：

- 快路径可以物理移除语言 token；
- action group 更轻量；
- 便于未来扩展 reasoning、safety、world model 等 expert。

## 2.6 MindLabel 数据标注管线

论文还提出 MindLabel，用于补充驾驶 VLA 缺乏语言监督的问题。

MindLabel 包括三阶段：

1. **Scene Understanding Question Generation**
   - 生成 Common、Spatial、Temporal、Motion、Object-centric 等问题。

2. **Action Dreaming**
   - 基于 GT 意图生成多种替代轨迹；
   - 包括 Better、Alternative、Conservative、Worse 等候选；
   - 生成轨迹评价 QA。

3. **Unified Answer Generation**
   - 使用 Qwen3-VL 和 Qwen3.5-Plus 生成答案。

规模上，MindLabel 在 WOD-E2E 上生成：

- 约 18.8K clips；
- 约 3.8M VQA pairs；
- 约 250K dreamed trajectories。

但主实验只使用了基础 scene-grounded VQA 和 WOD-E2E 官方 3-class intent label，尚未充分利用全部 MindLabel 资源。

# 3. 实验设计

## 3.1 数据集与 Benchmark

论文主要使用 **Waymo Open Dataset End-to-End（WOD-E2E）**。

该 benchmark 特点：

- 真实道路数据；
- 长尾场景；
- 约 4,021 个 20 秒驾驶片段；
- 8 个摄像头组成 360° 视野；
- 长尾事件频率低于 0.003%。

官方划分：

- train：2,037 sequences；
- validation：479 sequences；
- test：1,505 sequences。

测试集只释放前 12 秒，后 8 秒用于官方评估。

## 3.2 评价指标

论文使用多类指标：

### 轨迹规划指标

- **RFS（Rater Feedback Score）**
  - 由多个 human rater 参考轨迹评分；
  - 取值范围约为 0 到 10；
  - 预测轨迹与最接近的人类参考轨迹匹配后评分。

- **RFS-GT ADE 3s / 5s**
  - 预测轨迹与 logged GT trajectory 之间的 ADE；
  - 是外部方法常用指标。

- **RFS-matched ADE 3s / 5s**
  - 与 RFS 匹配到的人类参考轨迹计算 ADE。

- **Sequence-level ADE**
  - 在 3 / 5 / 10 / 15 / 20 / 25 秒长时序上评估。

### 语言指标

- BLEU-4；
- ROUGE-L。

用于衡量 VQA answer head 的语言生成质量。

## 3.3 对比方法

论文对比了 VA 和 VLA 方法。

### VA 方法

包括：

- VAD；
- UniAD；
- RAP-DINO；
- ViT-Adapter-GRU；
- Swin-Trajectory；
- DiffusionLTF；
- UniPlan / DiffusionDrive。

其中 RAP-DINO 是论文中最强 VA 参考之一。

### VLA 方法

包括：

- Poutine-Base；
- Open-LLaMA；
- NaiveEMMA；
- AutoVLA；
- dVLM-AD；
- HMVLM；
- Alpamayo-R1 等相关方法在讨论中也被提及。

论文强调：很多已有 VLA 在推理阶段实际是 action-only，语言接口被丢弃或没有报告语言质量。

# 4. 资源与算力

论文较明确地报告了训练与推理资源。

## 4.1 训练算力

默认主模型：

- Backbone：Qwen3-VL-2B；
- GPU：8 × H200；
- 训练时长：约 7 小时；
- 训练轮数：50 epochs；
- 精度：BF16；
- 分布式策略：DeepSpeed ZeRO-2；
- 优化器：AdamW；
- 学习率：\(10^{-4}\)；
- weight decay：0.1；
- warmup：1,000 iterations。

## 4.2 推理算力

推理 benchmark 使用：

- 单张 H200；
- batch size = 1。

论文报告了详细 latency：

- 慢路径 vqa_first_decoding：
  - 总延迟约 2594 ms；
  - 约 0.39 FPS；
  - 其中 VQA 自回归解码约 2450 ms，占约 94% 延迟。

- action_only 快路径：
  - 延迟约 103 ms；
  - 约 9.70 FPS。

- template QA + 2-step：
  - 约 108 ms；
  - 约 9.26 FPS。

- template QA + 1-step：
  - 约 63 ms；
  - 约 15.92 FPS。

- InternVL-2 1B 版本：
  - 约 64 ms；
  - 约 15.55 FPS；
  - 接近 RAP-DINO 的 17.68 FPS。

## 4.3 资源总结

论文在资源披露上相对充分，提供了：

- GPU 型号；
- GPU 数量；
- 训练时长；
- 推理延迟；
- FPS；
- VRAM；
- stage-level latency。

但也有一点需要注意：

- 不同模型和不同 backbone 的总训练 FLOPs 没有完整报告；
- 大模型扩展实验中，作者承认尚未做到 compute-matched scaling。

# 6. 主要结论与发现

## 6.1 VLA 可以达到甚至超过 VA 规划质量

在 WOD-E2E validation 上：

- Human Driver RFS：8.13；
- MindVLA-U1 + RL：8.20；
- RAP-DINO：7.91；
- MindVLA-U1 + Intent-CFG：7.92；
- MindVLA-U1 dense：7.83。

在 official test 上：

- MindVLA-U1 + RL：
  - RFS：7.87；
  - RFS-GT ADE 3s / 5s：1.09 / 2.66；
- 超过此前 VLA 和 VA 方法。

因此论文认为，VLA 的规划性能劣势可以通过正确架构设计被消除。

## 6.2 连续动作接口非常关键

使用 flow matching 连续轨迹生成，比将轨迹离散化为语言 token 更适合自动驾驶控制。

论文强调：

- 动作应保持连续；
- 语言应保持显式；
- 二者应共享骨干表示，而不是相互割裂。

## 6.3 Intent-CFG 提供了可测量的语言到动作路径

NTP-predicted Intent-CFG 将 RFS 从 7.83 提升到 7.92。

这说明：

- 语言预测出的 intent 并非附属输出；
- 它可以通过 CFG 影响动作扩散；
- 语言侧状态能够成为连续控制信号。

## 6.4 Streaming Memory 优于直接多帧 VLM 输入

实验显示：

- chunk-wise single frame：RFS 7.69；
- chunk-wise + 4-frame image sequence：RFS 7.61；
- streaming no memory：RFS 7.73；
- streaming + memory：RFS 7.83。

说明简单堆叠多帧图像 token 并不能有效建模驾驶时序，学习式 streaming memory 更有效。

## 6.5 语言质量和动作质量存在一定解耦

Backbone scaling 实验显示：

- instruction-tuned VLM 的 BLEU / ROUGE 明显更高；
- base VLM 的语言质量下降约 2 倍；
- 但 RFS 基本保持。

作者据此认为：

> Intent-CFG 依赖的不是表面语言生成质量，而是共享 backbone 中可分类、可控制的场景表征。

## 6.6 快 / 慢路径使 VLA 接近 VA 吞吐

在约 1B scale 下，MindVLA-U1 可以达到约 15.55 FPS，接近 RAP-DINO 的 17.68 FPS。

说明 VLA 的高成本并非必然，可以通过架构路径设计和移除语言解码来缓解。

# 7. 优点与亮点

## 7.1 架构统一性强

论文不是简单拼接 VLM 与 planner，而是将：

- vision；
- language；
- memory；
- ego-state；
- action；

全部纳入同一个 shared Transformer 表征中。

这使得语言和动作之间有更紧密的表示耦合。

## 7.2 动作输出形式合理

连续 action diffusion / flow matching 避免了轨迹 token 化带来的精度损失。

这对自动驾驶尤为重要，因为轨迹规划需要米级甚至厘米级精度。

## 7.3 语言到动作路径可验证

Intent-CFG 是论文最重要的机制贡献之一。

它使得“语言是否影响动作”从抽象宣称变成可实验验证的问题。

## 7.4 流式记忆设计贴近真实驾驶

真实车辆输入是逐帧流式到达，而不是固定 video chunk。

MindVLA-U1 的 streaming memory：

- 成本有界；
- 支持长时序；
- 避免 chunk 边界不连续；
- 训练和推理一致。

## 7.5 实验体系较完整

论文不仅报告主指标，还包括：

- ablation；
- latency；
- scaling；
- RL；
- val/test distribution shift；
- qualitative visualization。

整体实证支撑比较充分。

## 7.6 对 benchmark 偏移进行了主动诊断

作者没有只报告 val/test gap，而是用 MindLabel intent 统计分析出：

- validation 偏 accelerating / starting；
- test 偏 waiting / following / yielding。

这对理解 WOD-E2E 结果有帮助。

## 7.7 MindLabel 数据管线有扩展价值

MindLabel 生成了大量：

- scene QA；
- object-centric QA；
- trajectory evaluation QA；
- dreamed trajectories；
- 20-class intent。

虽然主实验只部分使用，但该数据管线对后续驾驶 VLA 训练和偏好学习有潜在价值。

# 8. 不足与局限

## 8.1 仅开放环评估，不能代表真实闭环驾驶

论文最重要的局限是：

- 所有主结果基于 WOD-E2E 开放环 benchmark；
- 模型没有在真实车辆或 closed-loop simulator 中与环境交互；
- RFS 高并不等价于闭环安全驾驶能力强。

因此“超过人类驾驶”应理解为：

> 在 WOD-E2E 开放环 RFS 指标上超过 logged human driver reference，而不是现实道路闭环驾驶超过人类。

## 8.2 只验证了一个主要 benchmark

实验集中在 WOD-E2E。

尚未验证：

- nuScenes；
- NAVSIM；
- CARLA closed-loop；
- 真实车辆部署；
- 跨城市、跨传感器、跨数据分布迁移。

因此泛化能力仍需进一步证明。

## 8.3 与对比方法不完全同条件

虽然论文标注了 AD-PT 等信息，但不同方法之间仍存在差异：

- backbone 不同；
- 预训练数据不同；
- 是否使用外部自动驾驶数据不同；
- 是否有 RL post-training 不同；
- 官方 leaderboard 方法细节不完全可控。

这会影响严格公平性。

## 8.4 RL 直接优化 RFS，可能存在 benchmark 适配

MindVLA-U1 + RL 使用 RFS 作为唯一 reward。

这带来积极效果，但也可能导致：

- 更贴近 rater preference；
- 远离 logged GT；
- 对该 benchmark 指标产生一定过拟合风险。

论文也观察到：

- RFS 提升；
- rater-matched ADE 改善；
- 但 GT-matched ADE 略有变差。

## 8.5 MindLabel 大量数据尚未充分利用

虽然 MindLabel 很丰富，但主结果仅使用：

- basic scene-grounded VQA；
- 官方 3-class intent label。

尚未充分利用：

- dreamed trajectories；
- trajectory-evaluation QA；
- 20-class intent；
- chain-of-thought rationales；
- preference ranking。

因此 MindLabel 对最终模型性能的完整贡献尚未被验证。

## 8.6 Intent ontology 较粗

主模型使用 WOD-E2E 的 3-class intent：

- left；
- right；
- straight。

但真实驾驶意图远更丰富，如：

- waiting；
- yielding；
- braking；
- following；
- emergency stop；
- lane change；
- u-turn。

论文也指出 test 中 waiting 等类别占比显著更高，粗粒度 intent 可能限制 Intent-CFG 在复杂场景中的效果。

## 8.7 大模型 scaling 结论尚不充分

Backbone scaling 出现非单调：

- 0.8B：7.81；
- 2B：7.94；
- 4B：7.86；
- 9B：7.84；
- 9B 训练 200 epochs 后恢复到 7.91。

作者明确承认这不是完整 scaling law，因为没有严格控制：

- 训练 FLOPs；
- 数据规模；
- action head 容量；
- 训练 schedule；
- closed-loop 指标。

## 8.8 慢路径延迟仍然较高

如果完整执行 VQA 自回归解码，延迟约 2.6 秒，只有 0.39 FPS。

这对实时自动驾驶不可接受。

虽然 fast path 可接近实时，但这意味着：

- 语言推理不能每帧完整运行；
- 需要进一步研究 slow language pathway 与 fast action loop 的异步调度。

---

# 总体评价

MindVLA-U1 是一篇架构贡献较强、实验较完整的驾驶 VLA 论文。它的核心价值在于重新定义了自动驾驶 VLA 的接口设计：动作保持连续，语言保持显式，二者通过共享 backbone 与 Intent-CFG 产生可测量联系，同时用 streaming memory 替代低效多帧 VLM 输入。实验表明，该设计可以在 WOD-E2E 上达到甚至超过强 VA 方法，并在一定配置下接近 VA 吞吐。

不过，论文结论仍主要限于 WOD-E2E 开放环环境。其闭环安全性、跨数据集泛化、真实车部署、完整 scaling law，以及 MindLabel 全量数据利用效果仍需要后续工作验证。

（完）
