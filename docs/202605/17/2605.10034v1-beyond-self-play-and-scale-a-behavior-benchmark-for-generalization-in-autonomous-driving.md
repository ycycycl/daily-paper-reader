## 摘要
近年来，GigaFlow 和 PufferDrive 等自动驾驶（Autonomous Driving, AD）研究工作使得将大规模强化学习（Reinforcement Learning, RL）作为驾驶策略训练方法成为可能。然而，这类策略仍与既有基准脱节，使得大规模强化学习在标准化自动驾驶评测中的性能尚不明确。我们提出 BehaviorBench——一个综合性测试套件，从评测、复杂性和行为多样性三个维度弥合这一差距。在评测方面，我们提供了一个将 PufferDrive 连接到 nuPlan 的接口，首次使得通过大规模强化学习训练的策略能够在成熟的自动驾驶规划基准上进行评估。作为补充，我们还提供了一个评测框架，使规划器能够直接在 PufferDrive 仿真中以更少时间开销进行基准测试。在复杂性方面，我们观察到，当前的标准化基准过于简单，以至于仅通过带碰撞检测的直线车道跟随即可取得近乎完美的分数。我们从 Waymo Open Motion Dataset（WOMD）中提取了一个具有实际意义且交互丰富的数据划分，在该划分上，如果不具备多智能体推理能力，就不可能获得强性能。最后，我们关注行为多样性。现有基准通常使用单一的基于规则的交通模型——智能驾驶员模型（Intelligent Driver Model, IDM）来评估规划器。我们提供了一套多样化的交互式交通智能体，用于在异质行为条件下对策略进行压力测试，而不只是使用 IDM。总体而言，我们的基准分析揭示了如下洞见：尽管通过纯自博弈在标准奖励函数下训练的策略能够以涌现方式学习交互行为，但它们会过拟合于训练中的对手，且无法泛化到其他交通智能体行为。基于这一观察，我们提出了一种将 PPO 策略与基于规则的规划器相结合的混合规划器。

## Abstract
Recent Autonomous Driving (AD) works such as GigaFlow and PufferDrive have unlocked Reinforcement Learning (RL) at scale as a training strategy for driving policies. Yet such policies remain disconnected from established benchmarks, leaving the performance of large-scale RL for driving on standardized evaluations unknown. We present BehaviorBench -- a comprehensive test suite that closes this gap along three axes: Evaluation, Complexity, and Behavior Diversity. In terms of Evaluation, we provide an interface connecting PufferDrive to nuPlan, which, for the first time, enables policies trained via RL at scale to be evaluated on an established planning benchmark for autonomous driving. Complementarily, we offer an evaluation framework that allows planners to be benchmarked directly inside the PufferDrive simulation, at a fraction of the time. Regarding Complexity, we observe that today's standardized benchmarks are so simple that near-perfect scores are achievable by straight lane following with collision checking. We extract a meaningful, interaction-rich split from the Waymo Open Motion Dataset (WOMD) on which strong performance is impossible without multi-agent reasoning. Lastly, we address Behavior Diversity. Existing benchmarks commonly evaluate planners against a single rule-based traffic model, the Intelligent Driver Model (IDM). We provide a diverse suite of interactive traffic agents to stress-test policies under heterogeneous behaviors, beyond just using IDM. Overall, our benchmarking analysis uncovers the following insight: despite learning interactive behaviors in an emergent manner, policies trained via pure self-play under standard reward functions overfit to their training opponents and fail to generalize to other traffic agent behaviors. Building on this observation, we propose a hybrid planner that combines a PPO policy with a rule-based planner.

## 2.1 PufferDrive 评测接口

### 基本思想

PufferDrive 原本主要用于高速 RL 训练，但缺少独立评测框架。本文扩展它，使其可以：

- 在 PufferDrive 内直接进行 closed-loop benchmark；
- 将 PufferDrive 训练出的 planner 接入 nuPlan 的 `AbstractPlanner` 接口；
- 支持在 nuPlan 和 interPlan 中评测 RL planner。

这使得大规模 RL planner 第一次可以与传统自动驾驶规划 benchmark 中的方法进行直接比较。

### 仿真设置

每个场景最多包含：

- 128 个智能体；
- ego-centric observation：
  - ego 状态：维度 7；
  - 最近 31 个周围智能体：每个维度 8；
  - 128 条可见道路片段：每条维度 7；
- action 可以是：
  - 离散加速度 / 转向组合；
  - 或连续二维控制；
- 动力学模型为运动学自行车模型；
- 时间步长为：
  - \(\Delta t = 0.1s\)

行人和骑行者作为不可控对象，按日志轨迹 replay。

---

## 2.2 评测协议与总分公式

### 评测协议

在每个场景中：

- 一个智能体被指定为 ego，由待评测 planner 控制；
- 其他车辆由某种 traffic agent 控制；
- episode 在以下情况终止：
  - ego 到达目标；
  - ego 发生碰撞；
  - ego 离开可行驶区域。

### 评分公式

论文使用一个 per-scenario score，范围为 \([0,1]\)。它由两部分组成：

1. **结果指标 outcome indicator**
2. **行为质量 behavior score**

公式为：

\[
Score_{scenario} =
(1 - I[AF-Col])
(1 - I[Off-Road])
I[Goal]
\cdot
(w_{cmf}S_{cmf} + w_{align}S_{align} + w_{ctr}S_{ctr})
\]

其中：

- \(I[AF-Col]\)：是否发生 ego 责任碰撞；
- \(I[Off-Road]\)：是否驶出道路；
- \(I[Goal]\)：是否到达目标；
- \(S_{cmf}\)：舒适性；
- \(S_{align}\)：车道朝向对齐；
- \(S_{ctr}\)：车道中心保持；
- 权重为：
  - \(w_{cmf}=0.2\)
  - \(w_{align}=0.5\)
  - \(w_{ctr}=0.3\)

该公式的一个重要特点是：  
**碰撞、越界、未到达目标中任意一个失败，整体得分直接归零。**

这使得 benchmark 更强调“先安全完成任务，再评价驾驶质量”。

## 2.3 场景划分：Interactive1k 与 Random1k

论文从 **Waymo Open Motion Dataset, WOMD** 的验证集构建两个 1000 场景划分：

### Random1k

- 从 WOMD validation set 中均匀随机采样 1000 个场景；
- 代表较普通、较接近总体分布的驾驶情况。

### Interactive1k

- 从 WOMD 中筛选最具交互性的 1000 个场景；
- 包含：
  - 密集交通；
  - 并线；
  - 阻塞车道；
  - 无保护转弯；
  - 多车交互；
  - 需要主动协商的交通情形。

### 交互性评分

作者定义 interactivity score：

\[
S_{int} = \lambda_{lane} \sum_i w_i \cdot \min(\frac{c_i}{\tau_i}, 1)
\]

其中：

- \(\lambda_{lane}\)：车道变化或转弯 multiplier；
  - 如果需要变道或转弯，取 1.0；
  - 否则取 0.5；
- \(c_i\)：场景复杂性指标；
- \(w_i\)：权重；
- \(\tau_i\)：归一化阈值。

使用的复杂性指标包括：

- ego 轨迹与其他 agent 轨迹交叉的数量；
- ego 加速度变化；
- ego 转向变化；
- time-to-collision 低于 3 秒的步数；
- ego 周围 40 米内 agent 数量；
- ego 到目标的距离。

该设计使 Interactive1k 比 Random1k 更能测试交互推理与主动协商能力。

## 2.4 多样化交通智能体

BehaviorBench 不只使用 IDM，而是提供多种 traffic agent：

### 1. IDM

经典规则交通模型，根据：

- 当前速度；
- 目标速度；
- 与前车距离；
- 安全间距；

计算纵向加速度。横向行为沿车道中心线行驶。

### 2. PPO

使用 PPO 训练的去中心化策略，所有 agent 共享参数，通过 self-play 同时控制多个车辆。

该 PPO 使用简单 reward，主要包括：

- 避免碰撞；
- 避免 off-road；
- 到达目标。

### 3. Reward-conditioned PPO

该策略输入中包含 reward coefficient vector，使一个网络可以根据不同 reward 权重表现出不同驾驶风格。

训练时每个 episode 采样 reward 权重；测试时固定权重，得到不同交通行为：

- **Aggressive**：激进型；
- **Normal**：正常型；
- **Cautious**：谨慎型；
- **Mix**：一个场景中混合三种风格。

这部分思想类似 GigaFlow 的 reward conditioning，但用于构造行为多样的 traffic agent。

### 4. SMART

SMART 是一个基于自回归 decoder-only motion prediction 的学习型交通模型，在 Waymo Open Sim Agents Challenge 2024 中排名很高。

它通过 token 化运动 primitive 进行多智能体轨迹生成，每次预测 0.5 秒运动。

### 5. Expert / Log-Replay

周围 agent 直接 replay WOMD 真实轨迹。  
缺点是它是 open-loop，不会对 ego 的行为做出反应。

---

## 2.5 PDM+PPO 混合规划器

论文还提出一个 hybrid planner，将规则规划器 PDM 与 PPO 策略结合。

### 基本动机

- PDM 在普通场景中安全、稳定、舒适；
- 但 PDM 无法很好处理复杂交互，尤其不能主动变道；
- PPO 更激进，能在交互场景中更容易达成目标；
- 但 PPO 容易过拟合训练对手，且舒适性和车道保持较差。

因此作者提出：

> 常规情况下使用 PDM；当 PDM 所有候选轨迹都失败时，切换到 PPO fallback。

### 算法流程

每个 planning step：

1. 使用 constant-velocity prediction 预测周围车辆；
2. PDM 生成 15 条 IDM proposal；
3. 对 proposal 打分；
4. 如果存在得分大于 0 的 PDM proposal：
   - 执行最高分 proposal；
5. 如果所有 PDM proposal 都失败：
   - 从 PPO 策略中取 top-K action；
   - 对每个 action rollout 1 秒；
   - 使用 benchmark score 对 rollout 轨迹评分；
   - 执行最高分 action 对应轨迹。

论文中主要使用：

- \(K=8\)
- PPO rollout horizon 为 1 秒；
- PDM horizon 为 4 秒。

该方法把复杂行为质量要求从 reward engineering 转移到 planning-time scoring。

# 6. 主要结论与发现

## 6.1 当前标准 benchmark 过于简单

论文发现，在 nuPlan Val14 上：

- 简单规则规划器；
- 少量 IDM proposal；
- 车道跟随 + 碰撞检测；

即可取得很高分数，甚至超过一些学习型 planner。

这说明 Val14 不能充分测试复杂交互推理。

---

## 6.2 Interactive1k 更能区分 planner 能力

在 Interactive1k 上：

- PDM 这类保守规则方法 goal completion 明显下降；
- PPO 在复杂交互场景中更积极，表现更好；
- planner 之间差异更明显。

这说明 Interactive1k 是更具挑战性和区分度的场景划分。

---

## 6.3 纯 self-play RL 会过拟合训练对手

论文最重要的发现之一是：

> PPO 在与 PPO traffic 交互时碰撞率很低，但换成 IDM、Expert、SMART 或其他 conditioned traffic 后，碰撞率显著上升。

这表明 self-play RL 可能学到的是对训练 opponent 的适配，而不是真正泛化的驾驶交互策略。

即使训练到 \(10^{11}\) agent steps，跨 traffic agent 泛化问题仍没有自然消失。

---

## 6.4 行为多样性对评测非常关键

不同 traffic agent 会显著改变 planner 排名。

例如：

- PDM 在 Random1k 这类普通场景更稳；
- PPO 在 Interactive1k 中更能完成目标；
- SMART realism 较高，但 closed-loop 中存在 off-road 和对保守交通不适配的问题；
- Cautious traffic 反而可能导致某些 planner 更难处理，因为过度保守会制造阻塞和不确定交互。

因此，只用 IDM 评测会严重低估 planner 的泛化风险。

---

## 6.5 混合规划器 PDM+PPO 表现最好或接近最好

PDM+PPO 在多数设置中取得最佳表现：

- 保留 PDM 在普通场景中的安全性；
- 在 PDM 无法找到有效 proposal 时借助 PPO 的主动性；
- 使用 rollout scoring 约束 PPO 的粗糙行为。

但它仍在 comfort、lane alignment 等行为指标上有改进空间。

---

# 7. 优点与亮点

## 7.1 问题定位准确

论文指出了当前自动驾驶 RL 研究中的一个关键缺口：

> 训练 scale 提升了，但 evaluation 没有跟上。

这对自动驾驶规划研究很重要，因为没有可靠 benchmark，就难以判断 RL planner 的真实性能。

---

## 7.2 Benchmark 设计完整

BehaviorBench 同时覆盖：

- 标准 benchmark 兼容；
- 新的高交互场景；
- 多样 traffic agent；
- 综合指标；
- realism 评估。

不是只提出一个数据 split，而是构建了完整评测生态。

---

## 7.3 强调跨行为泛化

论文没有只看 planner 在单一交通模型中的分数，而是系统测试不同 traffic agent：

- 规则型；
- RL 型；
- 模仿型；
- log replay；
- reward-conditioned 多风格型。

这使得“是否泛化”能够被更直接地观察。

---

## 7.4 实验结论有实践意义

论文发现 self-play RL 会过拟合 opponent，这对自动驾驶 RL 训练非常重要。

它提醒研究者：

- 更大训练规模不一定解决泛化；
- 自博弈环境中的对手分布非常关键；
- 评测必须包含行为分布 shift。

---

## 7.5 混合规划器设计务实

PDM+PPO 是一个轻量但有效的 baseline：

- 不完全依赖复杂 reward engineering；
- 不需要 GigaFlow 级别极大训练成本；
- 利用规则规划器的稳定性和 PPO 的主动性；
- 在 benchmark 上表现强。

---

## 7.6 复现性较好

论文提供：

- benchmark 构建方式；
- scoring 公式；
- traffic agent 参数；
- PPO reward 细节；
- 网络结构；
- 训练资源；
- hybrid planner 伪代码；
- 开源代码链接。

整体复现信息比较充分。

# 8. 不足与局限

## 8.1 未包含交通灯和 stop sign

论文明确承认，目前 BehaviorBench 尚未集成：

- traffic lights；
- stop signs。

这会限制其对真实城市驾驶完整规则环境的覆盖。作者目前通过 agent 到达目标后移除来缓解，但这并不能替代真实交通控制逻辑。

---

## 8.2 reward-conditioned traffic agent 并不总是理想

虽然 Aggressive、Normal、Cautious 通常表现出预期风格，但论文也观察到：

- 某些 normal agent 左转失败；
- cautious agent 有时行为异常；
- aggressive agent 可能驶入错误车道。

作者认为这些异常也可作为压力测试，但从 benchmark 可控性角度看，这仍是潜在问题。

---

## 8.3 统计显著性不足

论文没有报告：

- error bars；
- confidence intervals；
- 多随机种子重复训练结果。

作者解释是重复训练成本高。  
但对于 RL 方法，随机种子和训练波动可能显著影响结果，因此这是一个明显局限。

---

## 8.4 场景来源主要依赖 WOMD

BehaviorBench 的两个核心 split 来自 WOMD validation set。

这带来两个问题：

- 场景分布主要反映 WOMD 覆盖的城市和驾驶文化；
- 对其他国家、道路规则、天气、传感器误差等因素的泛化尚未验证。

---

## 8.5 部分 planner 与 benchmark 指标可能存在适配关系

PDM+PPO 在 fallback rollout 时使用与 benchmark 类似的 score 进行轨迹选择。  
这虽然合理，但可能使其对该评测指标有一定“内生适配”优势。

---

## 8.6 感知与不确定性问题未覆盖

论文主要研究规划与仿真交互，默认输入为较理想的状态信息。它没有重点处理：

- 感知误差；
- 遮挡；
- 预测不确定性；
- 传感器噪声；
- 端到端感知-规划耦合。

因此其结论主要适用于 planning benchmark，而不是完整自动驾驶系统部署。

---

## 8.7 Expert replay 是 open-loop

Expert traffic agent 直接 replay 真实轨迹，不会响应 ego 行为。  
这可以测试分布外交互风险，但也可能产生不自然的 closed-loop 场景，因为真实人类本应对 ego 行为做出反应。

---

## 8.8 真实部署安全性仍不可直接推断

BehaviorBench 能更好评测泛化，但它仍是仿真 benchmark。  
即使 planner 在 BehaviorBench 上表现好，也不能直接说明其满足真实道路部署安全要求。

---

# 总结

本文提出的 **BehaviorBench** 是一个面向自动驾驶规划泛化能力的系统性 benchmark。它的核心贡献不是单一算法，而是指出并修复当前大规模 RL 自动驾驶评测中的关键短板：标准 benchmark 过于简单、traffic agent 行为过于单一、RL 训练环境与标准评测脱节。

实验表明，纯 self-play RL 即使在大规模训练下也可能严重过拟合训练对手；只有在复杂交互场景和多样交通行为下评测，才能真正暴露 planner 的泛化问题。论文提出的 PDM+PPO 混合规划器提供了一个强而实用的 baseline，但仍存在交通规则覆盖、统计显著性、真实部署外推等局限。
