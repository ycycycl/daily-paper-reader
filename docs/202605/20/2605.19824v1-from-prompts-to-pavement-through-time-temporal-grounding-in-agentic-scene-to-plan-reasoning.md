## 摘要
近年来，利用大型语言模型（LLM）与大型多模态模型（LMM）集成来支持自动驾驶车辆（AV）中的高层场景理解与规划的尝试，仍然将时间视为次要属性。这种缺乏时间锚定的问题会导致对连续动作推理的不一致，从而削弱安全性与可解释性。本研究探讨在智能体间通信中引入时间条件化，是否能够在不降低语义或逻辑一致性的情况下保持或增强连贯性。为此，我们提出了三种时间整合程度逐步提高的规划器架构，并在 BDD-X 数据集的精选子集上使用语义、句法和逻辑指标对其进行评估。结果表明，尽管时间条件化会重塑推理风格，但在基于标准 NLP 的正确性指标上并未带来统计显著的改进。然而，定性分析揭示了 Sentinel 中的预测性危险推理、稳定的纠正行为以及策略性分歧。这些发现阐明了基于提示的时间锚定的局限，并建立了首个用于时间化场景到规划推理的实证基准。

## Abstract
Recent attempts to support high-level scene interpretation and planning in Autonomous Vehicles (AVs) using ensembles of Large Language Models (LLMs) and Large Multimodal Models (LMMs) continue to treat time as a secondary property. This lack of temporal grounding leads to inconsistencies in reasoning about continuous actions, undermining both safety and interpretability. This work explores whether temporal conditioning within inter-agent communication can preserve or enhance coherence without introducing degradation in semantic or logical consistency. To investigate this, we introduce three planner architectures with progressively increasing temporal integration and evaluate them on curated subsets of the BDD-X dataset using semantic, syntactic, and logical metrics. Results show that while temporal conditioning reshapes reasoning style, it yields no statistically significant improvements in standard NLP-based correctness metrics. However, qualitative analysis reveals predictive hazard reasoning, stable corrective behavior, and strategic divergence in the Sentinel. These findings clarify the limits of prompt-based temporal grounding and establish the first empirical benchmark for temporal scene-to-plan reasoning.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体思路

论文基于此前的 **From Prompts to Pavement** 框架，构建了三种时间整合程度递增的多智能体规划器：

1. **Static Planner**：只看当前帧，不使用时间信息。
2. **Sentinel Planner**：引入上一时刻计划，进行短期时序对齐。
3. **Synthesizer Planner**：引入多个历史观测与历史计划，构建长期叙事记忆。

三者共享相似的多智能体骨架，但在规划阶段加入不同程度的时间条件化。

---

### 2.2 Agentic Backbone：基础多智能体框架

系统由若干 LMM 智能体组成，主要包括：

#### 1）Descriptor Agent

- 输入当前图像帧 `i`。
- 输出结构化文本观察：

  \[
  o = \langle \kappa, e, d \rangle
  \]

  其中：

  - \(\kappa\)：criticality key，表示场景关键性或危险程度；
  - \(e\)：issue explanation，对问题或风险的解释；
  - \(d\)：scene description，对场景的文本描述。

- 为增强空间理解，作者使用了 Chain-of-Symbols（CoS）式提示方法。
- 该模块沿用原框架，本文主要改动集中在 Planner。

#### 2）Planner Agent

- 输入 Descriptor 的观察结果 \(o\)。
- 输出高层行为计划：

  \[
  p = [g_1, g_2, ..., g_n]
  \]

- 其中每个 \(g_k\) 表示一个目标或动作步骤，并且具有顺序关系：

  \[
  g_k \rightarrow g_{k+1}
  \]

- 目标是生成有序、高层、可解释的驾驶规划。

#### 3）Generator Agent

- 原始 From Prompts to Pavement 框架中，Generator 会把高层计划转化为可执行的行为树。
- 本文为了专注研究规划阶段，移除了 Generator。
- 因此实验只评估从场景到高层计划的过程，而不验证实际控制执行效果。

---

### 2.3 Static Planner：无时间基线

Static Planner 是非时序基线，只使用当前观察 \(o(t)\)。

它由三个子智能体构成：

#### Initiator Agent

- 根据当前观察生成初始计划 \(p_0\)。
- 该计划是贪心生成的，可能存在不完整或不合理之处。

#### Critic Agent

- 对当前计划 \(p_i\) 进行批判性评估。
- 输出：

  \[
  c = \langle HALT, f \rangle
  \]

  其中：

  - `HALT = 1` 表示计划可接受；
  - `HALT = 0` 表示需要继续修改；
  - \(f\) 是反馈文本，用来指导后续 refinement。

#### Refiner Agent

- 如果 Critic 认为计划需要修改，Refiner 根据当前计划 \(p_i\) 和反馈 \(f\) 生成改进版：

  \[
  p_{i+1}
  \]

- 该循环持续到计划被接受，或达到最大迭代次数。

Static Planner 的作用是作为公平基线，确保后续 Sentinel 和 Synthesizer 的改进不是来自多轮 refinement，而是来自时间信息。

---

### 2.4 Sentinel Planner：短期时间对齐

Sentinel Planner 在 Static Planner 基础上加入一个新的 **Sentinel Agent**，用于比较：

- 当前观察 \(o(t)\)；
- 上一时刻计划 \(p(t-1)\)；
- 帧率 FPS；
- 时间间隔 \(\Delta t\)。

其形式化描述为：

\[
Sentinel(o(t), p(t-1), FPS, \Delta t) = s
\]

其中输出 \(s\) 包含：

- `ALIGNMENT ∈ {0,1}`：当前观察是否与上一计划一致；
- `justification`：解释一致或不一致的原因。

具体逻辑为：

- 如果 `ALIGNMENT = 1`：
  - 表示当前世界状态仍然符合上一时刻计划的预期；
  - Planner 可以继续基于原方向进行规划。
- 如果 `ALIGNMENT = 0`：
  - 表示当前场景与上一时刻计划不匹配；
  - Sentinel 会指出偏差来源；
  - 后续 Planner 需要生成纠正性计划。

Sentinel 的核心作用是提供**局部时间连续性检查**，使模型能够回答：

> 当前情况是否仍然符合我上一时刻打算做的事？

---

### 2.5 Synthesizer Planner：长期时序叙事

Synthesizer Planner 进一步加入长期历史信息，不只看上一帧，而是看多个历史观测和历史计划。

其核心模块是 **Synthesizer Agent**：

\[
Synthesizer(o(t-k:t-1), p(t-k:t-1), o(t)) = n
\]

其中：

- \(o(t-k:t-1)\)：过去 \(k\) 个时间步的观察序列；
- \(p(t-k:t-1)\)：对应的历史计划序列；
- \(o(t)\)：当前观察；
- \(n\)：生成的时序叙事表示，即 narrative representation。

该叙事表示会总结：

- 环境如何随时间变化；
- 车辆过去如何响应；
- 当前状态与历史计划之间的因果关系；
- 未来可能的发展方向。

然后该叙事文本被提供给 Initiator、Critic 和 Refiner，使其在规划时不仅考虑当前场景，还考虑长时序上下文。

Synthesizer 的目标是让 Planner 具有类似“故事线”的时间理解能力。

### 3.3 采样策略

- 防止模型看到过多未来信息；
- 避免“提前知道后续事件”；
- 保持规划任务的在线决策性质。

随机采样的目的是模拟真实驾驶中事件突然出现、模型必须快速响应的情况。

### 3.6 统计检验

论文设计了三个假设：

- H1：Static 在语义与逻辑指标上差于 Sentinel。
- H2：Static 在语义与逻辑指标上差于 Synthesizer。
- H3：Sentinel 在语义与逻辑指标上差于 Synthesizer。

为了处理语义指标分数高度集中在高分区的问题，作者使用了 heavy-sided log transform：

\[
t(s)=
\begin{cases}
-\log(1-s), & \epsilon \leq s \leq 1-\epsilon \\
-\log(\epsilon), & s > 1-\epsilon \\
-\log(1-\epsilon), & s < \epsilon
\end{cases}
\]

统计方法包括：

- 配对 permutation test；
- \(10^5\) 次置换；
- Cohen’s d 效应量；
- \(10^4\) 次 BCa bootstrap 置信区间；
- Benjamini–Hochberg 校正控制多重比较的 FDR。

这些设计使得论文不仅报告平均指标，也检验差异是否具有统计显著性。

## 6. 主要结论与发现

### 6.1 定量结论

- 时间条件化确实改变了规划器的推理风格。
- 但在标准 NLP 指标上没有带来统计显著提升。
- p-value 普遍高于 0.05。
- Cohen’s d 效应量很小，通常 \(|d| < 0.2\)。
- 因此，论文的核心假设没有在定量指标上得到强支持。

---

### 6.2 各规划器表现

#### Static Planner

- 生成速度最快；
- 语义和逻辑指标与时间化方法相近；
- 作为 baseline 表现稳定。

#### Sentinel Planner

- 在时间感知方法中最具实用性；
- token cost 最低；
- 语义指标与 Static 接近，差距通常约 \(\delta \approx 0.01 - 0.02\)；
- 定性上展现了较好的短期纠错和风险预测能力。

#### Synthesizer Planner

- 理论上具有更强长期时间建模能力；
- 但实际效果并未显著优于 Sentinel 或 Static；
- 可能存在 “compression drift”：
  - 历史信息被压缩成叙事后，可能丢失关键细节；
  - 长上下文总结反而降低计划质量；
  - 推理成本和时间更高。

---

### 6.3 定性发现

论文强调，模型与 ground truth 的差异不一定是错误。

例如在图 3 中：

- ground truth 关注公交车或车道限制；
- 模型提前注意到行人风险，并生成更保守、更安全的计划；
- 下一帧中 ground truth 才体现出该风险。

这表明 Sentinel 可能具备：

- predictive hazard identification：预测性危险识别；
- stable corrective behavior：稳定纠错行为；
- purposeful strategic divergence：有目的的策略性分歧。

也就是说，模型有时不是没有对齐标注，而是生成了比标注更完整或更安全的规划。

---

### 6.4 对指标的反思

论文指出，传统 NLP 指标存在明显局限：

- ROUGE 和 METEOR 偏词面重合，不适合评估开放式规划；
- BERTScore 只能衡量静态语义相似；
- NLI 可以评估蕴含或矛盾，但不擅长评估连续计划之间的时序因果一致性；
- 这些指标像是在评估“单帧质量”，而不是评估“整段影片的连贯性”。

因此，缺乏显著提升并不一定说明时间建模无效，也可能说明现有指标无法捕捉真正的时序规划质量。

## 7. 优点与亮点

### 7.1 问题定位清晰

- 论文明确指出当前 LMM 自动驾驶规划系统中的关键缺口：缺乏时间锚定。
- 这一问题对于自动驾驶安全性和可解释性都非常重要。

### 7.2 架构设计具有层级性

- Static、Sentinel、Synthesizer 三种方法形成清晰递进关系：
  - 无时间；
  - 短期时间；
  - 长期时间。
- 这种设计便于分析不同时间整合深度的影响。

### 7.3 多智能体流程较完整

- 使用 Initiator、Critic、Refiner 构建规划 refinement loop；
- Sentinel 和 Synthesizer 分别承担短期对齐与长期叙事建模；
- 方法设计符合 agentic reasoning 的趋势。

### 7.4 统计分析较严谨

- 不只报告均值；
- 使用 permutation test；
- 使用 bootstrap 置信区间；
- 使用 Cohen’s d；
- 使用 Benjamini–Hochberg 校正；
- 对无显著性结果进行了诚实讨论。

### 7.5 定性分析有启发性

- 论文没有简单将低 ROUGE/METEOR 解释为失败；
- 而是指出模型可能通过更高层次的安全推理产生与标注不同但合理的计划；
- 对自动驾驶数据集标注质量和 NLP 评估指标提出了有价值的批判。

### 7.6 建立初步 benchmark

- 作者声称这是首个面向 temporal scene-to-plan reasoning 的实证 benchmark。
- 即使结果未显示显著提升，也为后续研究提供了方法框架和负结果参考。

## 8. 不足与局限

### 8.1 缺乏真实闭环验证

- 本文只评估文本计划。
- 没有将计划输入控制器、行为树或仿真环境。
- 因此无法证明文本计划能带来更安全的实际驾驶行为。

### 8.2 指标与任务目标不完全匹配

- ROUGE、METEOR、BERTScore 和 NLI 都不是为自动驾驶规划专门设计的。
- 它们难以衡量：
  - 时序因果一致性；
  - 风险提前识别；
  - 计划可执行性；
  - 交通规则遵守；
  - 安全裕度；
  - 多帧行为稳定性。

### 8.3 数据集 ground truth 不完整

- BDD-X 标注往往较简短，只解释一个主要原因。
- 模型可能看到更多上下文并生成更完整计划，却被指标惩罚。
- 这使得自动指标评估存在偏差。

### 8.4 长期时间建模方式较弱

- Synthesizer 依赖文本叙事总结历史。
- 这种 prompt-based memory 容易出现：
  - 信息压缩过度；
  - 关键细节遗失；
  - 历史噪声累积；
  - compression drift。
- 没有真正的显式记忆模块或学习式时序状态表示。

### 8.5 算力与模型细节不足

- 未说明具体使用了哪个 LMM/LLM；
- 未说明模型参数规模；
- 未说明推理硬件；
- 未说明 prompt 模板细节；
- 复现实验存在困难。

### 8.6 实验覆盖仍有限

- 只使用 BDD-X；
- 只采样视频前 2 秒；
- 长时序驾驶任务验证不足；
- 没有跨数据集泛化实验；
- 没有极端天气、夜间、复杂交互等更细粒度压力测试。

### 8.7 实时性问题明显

- 平均生成时间达到数十秒；
- 对自动驾驶在线规划而言远远不够；
- 即使 Sentinel token cost 较低，也仍难以直接部署在实时系统中。

---

## 总体评价

这篇论文的价值不在于证明 temporal prompting 能显著提升自动驾驶规划性能，而在于系统性地证明了：**仅靠提示词拼接和文本级时间条件化，难以稳定获得可量化的规划提升**。Sentinel 的短期时间对齐显示出一定实用潜力，尤其在风险预测和纠错行为上较有启发；但 Synthesizer 的长期叙事压缩暴露了 prompt-based memory 的局限。论文较好地建立了一个初步 benchmark，并指出未来应转向显式记忆模块、事件触发式时序推理、闭环仿真验证以及更适合规划任务的时序因果评估指标。
