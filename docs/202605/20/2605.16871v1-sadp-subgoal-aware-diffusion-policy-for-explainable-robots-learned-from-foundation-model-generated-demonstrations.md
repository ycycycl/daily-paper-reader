## 摘要
可解释机器人不仅需要成功完成任务，还需要能够以用户友好的方式呈现其内部决策过程。然而，大多数模仿学习方法仅在任务级示范上进行训练，并未显式建模子目标结构或执行进度。标准机器人学习数据集中子目标级监督的稀缺进一步加剧了这一局限，限制了机器人在长时程操作过程中传达其正在执行的子任务的能力。为解决这一问题，本文提出了子目标感知扩散策略（Subgoal-Aware Diffusion Policy, SADP），该框架利用基础模型自动生成带有子目标标注的示范，并在这些数据集上训练扩散策略。SADP 通过同时以任务级和子目标级描述作为条件来生成动作，从而围绕人类可理解的子目标组织策略执行。一个轻量级辅助头进一步预测子目标完成状态，使机器人能够呈现其当前执行阶段并监测子目标进展。在 RLBench 仿真环境以及 UR5e 机器人的真实世界评估中的实验表明，SADP 相比强任务条件扩散基线实现了更高的任务成功率，同时提供了用于监测进度和诊断失败的子目标级执行信号。这些结果表明，内置式而非事后式可解释性可以与高任务性能并存。

## Abstract
Explainable robots require not only successful task execution but also the ability to expose internal decision-making process in a user-friendly manner. However, most imitation learning methods are trained solely on task-level demonstrations, without explicitly modeling subgoal structure or execution progress. This limitation is further exacerbated by the scarcity of subgoal-level supervision in standard robot learning datasets, which restricts the development of robots that can convey the subtasks they are executing during long-horizon manipulation. To address this issue, this paper proposes Subgoal-Aware Diffusion Policy (SADP), a framework that leverages foundation models to autonomously generate subgoal-annotated demonstrations and trains diffusion policies on these datasets. SADP structures policy execution around human-interpretable subgoals by conditioning action generation on both task-level and subgoal-level descriptions. A lightweight auxiliary head further predicts subgoal completion states, allowing the robot to expose its current execution stage and monitor subgoal progression. Experiments in RLBench simulations and real-world evaluations on a UR5e robot demonstrate that SADP achieves higher task success rates than strong task-conditioned diffusion baselines, while providing subgoal-level execution signals for monitoring progress and diagnosing failures. These results highlight that built-in, rather than post-hoc, interpretability can coexist with high task performance.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 方法名称与核心思想

- 论文提出 **SADP：Subgoal-Aware Diffusion Policy**，即“子目标感知扩散策略”。
- 核心思想是：
  - 使用基础模型将任务指令自动分解为有序的自然语言子目标；
  - 通过基础模型和启发式运动规划器自动生成带有子目标标签和完成信号的示范数据；
  - 训练一个扩散策略，使其动作生成不仅依赖观测和任务指令，还依赖当前子目标描述；
  - 同时训练一个轻量级完成预测头，用于判断当前子目标是否完成，从而触发切换到下一个子目标。

### 2.2 子目标标注示范生成流程

论文的数据生成流程大致如下：

1. **输入自然语言任务指令**
   - 例如：“Put rubbish in bin” 或 “Put item in drawer”。

2. **LLM 分解任务**
   - LLM 将完整任务拆解为一系列自然语言子目标。
   - 例如 PutRubbishInBin 被分解为：
     - 抓取垃圾；
     - 回到默认位姿；
     - 移动到垃圾桶上方；
     - 打开夹爪释放物体。

3. **LLM / VLM 提取 affordance 信息**
   - 对每个子目标，系统提取目标物体点云、体素价值图等 affordance 表示。
   - 仿真中使用 oracle object mask；
   - 真实环境中使用 GPT-4o、GroundingDINO、Segment Anything 2 等模型处理 RGB-D 观测。

4. **LLM 生成启发式运动规划器**
   - 系统根据每个子目标生成动作计划，并顺序执行子目标。

5. **轨迹分段与完成标签生成**
   - 每条长程任务轨迹被自然切分为多个子目标片段。
   - 每个子目标片段的最后一帧标记为 `0`，表示当前子目标完成；
   - 其他帧标记为 `1`，表示当前子目标仍在执行。
   - 因此一条完整轨迹的完成标签模式类似：
     - `1 ... 1 0 1 ... 1 0 ... 1 0`

6. **成功轨迹过滤**
   - 使用 LLM / VLM 过滤成功执行的轨迹，保留为训练数据。

### 2.3 SADP 策略结构

SADP 的策略由两个主要部分构成：

#### 2.3.1 子目标条件扩散策略

- SADP 基于 Diffusion Policy，使用条件去噪过程生成机器人动作。
- 条件输入包括：
  - 机器人本体状态历史；
  - 目标物体点云特征；
  - 任务级自然语言描述；
  - 当前子目标自然语言描述。
- 点云特征由轻量 MLP 编码器处理，参考 DP3。
- 文本特征由 CLIP B/32 文本编码器提取。
- 所有信息拼接成全局条件向量 `c`，并通过 FiLM 机制注入扩散模型。

扩散动作生成过程可以概括为：

- 从高斯噪声动作开始；
- 在条件向量 `c` 的引导下逐步去噪；
- 最终得到干净动作输出。

论文中给出的反向扩散更新公式为：

- 在第 `k` 步，根据当前噪声动作、噪声调度参数和预测噪声，得到上一步动作。
- 动作学习损失为真实噪声与模型预测噪声之间的均方误差。

即训练目标之一是：

- 最小化动作噪声预测误差 `L_action`。

#### 2.3.2 子目标完成预测头

- SADP 在扩散策略之外加入一个轻量级二分类头。
- 该完成预测头共享同一个条件向量 `c`。
- 输出当前子目标的完成概率分数。

论文中定义为：

- `p = sigmoid(W · c + b)`

其中：

- `p` 是预测分数；
- 分数越低，表示越确信当前子目标已经完成；
- 当 `p < τ` 时，系统认为当前子目标完成，并切换到下一个子目标。

由于大多数帧都是“子目标未完成”，完成帧较少，因此存在类别不均衡。作者使用 **Focal Loss** 训练完成预测头。

总体损失为：

- `L_total = L_action + λ L_completion`

其中：

- `L_action` 是扩散策略动作生成损失；
- `L_completion` 是完成预测损失；
- `λ` 从 0 逐渐增加到 0.1，以避免训练早期完成预测任务干扰动作学习。

### 2.4 推理阶段流程

推理时，SADP 的执行过程为：

1. 使用 LLM 根据任务指令生成子目标序列；
2. 当前子目标与任务描述、观测一起作为扩散策略条件；
3. 扩散策略生成动作；
4. 完成预测头实时判断当前子目标是否完成；
5. 若完成分数低于阈值 `τ = 0.2`，切换到下一个子目标；
6. 重复直到任务完成。

  - ToyInBoxClose：
    - DP3：40.0%
    - TARAD：56.7%
    - SADP：73.3%
  - ToyInDrawerClose：
    - DP3：46.7%
    - TARAD：76.7%
    - SADP：86.7%

- **子目标结构对长程任务特别有帮助。**
  - 在短任务 SpongeInPlate 上，TARAD 略优于 SADP；
  - 但在 ToyInBoxClose 和 ToyInDrawerClose 这类多阶段任务上，SADP 明显更强。

- **完成预测信号具有监控和诊断价值。**
  - 成功轨迹中，完成分数会在子目标转换附近下降；
  - 失败轨迹中，若机器人卡在某个子目标，当前子目标不会切换，从而可以定位失败阶段。

- **消融实验表明：**
  - 去掉子目标条件后，平均成功率下降到 **69.1%**；
  - 使用 oracle completion signal 后，平均成功率提升到 **87.1%**；
  - 说明子目标条件和准确的完成预测都很重要。

## 7. 优点与亮点

### 7.1 方法层面

- **内生式可解释性**
  - 不是在黑箱策略执行后再生成解释，而是将子目标结构直接嵌入动作生成过程。

- **利用自然语言子目标**
  - 子目标本身是人类可读的自然语言描述，便于用户理解和监控。

- **自动生成子目标标注数据**
  - 通过 LLM / VLM 和启发式 motion planner 自动生成示范，避免人工逐帧标注。

- **动作生成与进度预测联合学习**
  - 扩散策略负责生成动作；
  - 完成预测头负责判断子目标是否完成；
  - 两者共享表示，使解释信号与执行行为更紧密相关。

- **适合长程多阶段任务**
  - 当前子目标条件可以帮助策略保持时序一致性，减少过早进入后续阶段的问题。

### 7.2 实验层面

- **同时评估仿真和真实机器人**
  - 不只停留在仿真结果，也验证了真实 UR5e 上的可行性。

- **对比基线较合理**
  - DP3 和 TARAD 都是与本文任务相关的 diffusion policy 方法。

- **消融设计清晰**
  - 分别验证子目标 conditioning 和 completion signal 的贡献。

- **不仅报告成功率，也展示可解释信号**
  - 通过成功和失败轨迹中的 completion score，说明 SADP 可以用于进度监控和故障定位。

## 8. 不足与局限

### 8.1 实验覆盖有限

- 仿真任务数量为 6 个，真实任务数量为 3 个，虽然覆盖了一些典型长程操作，但仍不足以证明方法在开放世界或更复杂家庭场景中的泛化能力。
- 所有任务基本仍属于桌面操作或抽屉/盒子交互，场景复杂度有限。
- 没有测试大规模多物体、多机器人、多房间或动态人机协作环境。

### 8.2 数据规模偏小

- 每个任务仅收集 30 条示范。
- 虽然自动数据生成降低了人工成本，但论文没有展示数据规模扩大后性能如何变化。
- 也没有系统分析自动生成示范中的失败率、过滤比例和子目标标注噪声。

### 8.3 对基础模型依赖较强

- 子目标分解、affordance 提取、真实场景感知都依赖 LLM / VLM。
- 如果基础模型生成了错误子目标、漏掉必要步骤或识别错误物体，SADP 可能受到影响。
- 论文没有充分分析基础模型错误对最终策略的影响。

### 8.4 完成预测仍是主要瓶颈

- 消融实验显示，使用 oracle completion signal 可将平均成功率从 82.8% 提升到 87.1%。
- 这说明当前二分类完成预测并不完美。
- 短时长子目标样本较少，容易导致完成边界学习不稳定。
- 阈值 `τ = 0.2` 是经验设定，可能对任务和环境敏感。

### 8.5 可解释性评价不够量化

- 论文展示了 completion score 曲线和失败诊断案例，但缺少：
  - 用户研究；
  - 人类理解度评估；
  - 解释准确率指标；
  - 子目标边界检测精度；
  - 解释与真实因果执行过程的一致性度量。
- 因此，“可解释性提升”主要是结构和案例层面的论证。

### 8.6 算力和复现细节不足

- 未报告 GPU 型号、数量、训练耗时和数据生成耗时。
- 未明确说明基础模型调用频率和成本。
- 对希望复现系统的研究者而言，这部分信息不足。

### 8.7 与更强通用模型缺少直接比较

- 论文没有与 OpenVLA、π0、CoT-VLA 等大型 VLA 模型进行直接实验对比。
- 也没有与显式层级策略、行为树、符号规划结合学习的方法系统比较。
- 因此尚不能完全判断 SADP 相对更大规模通用机器人模型的优势边界。
