## 摘要
尽管自回归（AR）视觉-语言-动作（VLA）模型在机器人任务中展现出强大的推理能力，但其顺序解码过程往往会带来较高的推理延迟，并可能在长时域执行中放大误差累积。离散扩散语言模型（dLLMs）通过并行 token 细化提供了一种有前景的替代方案，但其在机器人领域的实际部署仍受限于重复的去噪函数评估（NFEs），以及难以将标准 KV 缓存直接应用于双向迭代解码。为弥合这些范式之间的差距，我们提出 BlockVLA，这是一个通过块扩散范式将预训练 AR 骨干适配为高效离散扩散策略的框架。BlockVLA 在块级别保持自回归依赖，同时支持每个块内的并行去噪，从而结合了全局因果一致性与局部并行生成。该设计能够在已完成块之间复用前缀 KV 缓存，降低迭代去噪的有效成本，并为从 AR 预训练过渡到基于扩散的策略微调提供更平滑的路径。我们在 LIBERO 和 SimplerEnv 基准上进行了广泛评估。实验结果表明，与标准离散扩散基线相比，BlockVLA 实现了 3.3 倍的推理加速。此外，我们的模型表现出更优的训练效率，其成功率收敛速度显著快于基线；这一优势在复杂的长时域任务中尤为明显，BlockVLA 在训练早期阶段取得了显著的性能提升。本工作确立了块扩散作为连接大规模预训练 AR 模型与高效、高频实时机器人控制之间的稳健桥梁。

## Abstract
While autoregressive (AR) Vision-Language-Action (VLA) models have demonstrated formidable reasoning capabilities in robotic tasks, their sequential decoding process often incurs high inference latency and may amplify error accumulation during long-horizon execution. Discrete Diffusion Language Models (dLLMs) provide a promising alternative through parallel token refinement, but their practical deployment in robotics remains limited by repeated denoising function evaluations (NFEs) and the difficulty of directly applying standard KV caching to bidirectional iterative decoding. To bridge these paradigms, we propose BlockVLA, a framework that adapts pretrained AR backbones into an efficient discrete diffusion policy through a block diffusion paradigm. BlockVLA maintains autoregressive dependencies at the block level while enabling parallel denoising within each block, thereby combining global causal coherence with local parallel generation. This design enables prefix KV-cache reuse across completed blocks, reduces the effective cost of iterative denoising, and provides a smoother transition from AR pretraining to diffusion-based policy fine-tuning. We conduct extensive evaluations on the LIBERO and SimplerEnv benchmarks. Experimental results demonstrate that our BlockVLA achieves a 3.3$\times$ inference acceleration over standard discrete diffusion baselines. Furthermore, our model exhibits superior training efficiency, with success rates converging substantially faster than baselines, a gain that is particularly pronounced in complex, long-horizon tasks, where BlockVLA achieves significant performance gains in the early stages of training. This work establishes Block Diffusion as a robust bridge between large-scale pretrained AR models and efficient, high-frequency real-time robotic control.

## 1. 核心问题与整体含义：研究动机与背景

### 1.1 研究背景

本文关注的是机器人视觉-语言-动作模型（Vision-Language-Action, VLA）中的动作生成效率问题。近年来，基于大规模视觉语言模型（VLM）的 VLA 系统，如 RT-2、OpenVLA、π0-FAST 等，通常将连续机器人动作离散化为 token，并使用自回归（Autoregressive, AR）方式逐 token 生成动作序列。这种范式能够很好地继承大规模预训练 VLM 的视觉-语言对齐、语义推理和任务泛化能力。

然而，自回归 VLA 在机器人控制中存在两个关键瓶颈：

- **推理延迟高**：动作 token 必须按顺序逐个生成，难以满足高频实时控制需求。
- **长时域误差累积严重**：长任务中早期预测错误会影响后续动作生成，可能导致任务失败。

### 1.2 离散扩散模型的潜力与问题

离散扩散语言模型（Discrete Diffusion Language Models, dLLMs）通过并行 token 去噪，可以在一定程度上缓解自回归模型的顺序解码瓶颈，并允许模型在迭代过程中修正错误。因此，它被认为是提升 VLA 推理速度和鲁棒性的有前景方向。

但标准离散扩散 VLA 也存在明显问题：

- **需要多次去噪函数评估（NFE）**：每次去噪都需要一次模型前向传播，实际推理速度仍然受限。
- **难以复用 KV Cache**：标准扩散解码通常是双向迭代细化整段序列，无法像自回归模型那样简单复用前缀缓存。
- **与 AR 预训练范式不匹配**：当前强大的 VLM / VLA backbone 多数是自回归预训练模型，直接改造成全双向扩散模型会带来优化和迁移困难。

### 1.3 论文整体含义

本文提出的 **BlockVLA** 试图在自回归模型和离散扩散模型之间建立一个折中范式：

- **块间保持自回归因果依赖**；
- **块内使用离散扩散并行去噪**；
- **已生成块可以作为前缀复用 KV Cache**。

因此，BlockVLA 的核心意义在于：  
它将大规模预训练 AR VLA 平滑适配为一种高效的块扩散策略，在保留 AR 模型因果推理能力和 KV-cache 效率的同时，获得扩散模型的局部并行生成能力，从而面向高频实时机器人控制。

## 2. 方法论：核心思想、关键技术与算法流程

## 2.1 核心思想

BlockVLA 的核心思想是 **Block Diffusion for VLA**，即将动作序列划分为多个连续块：

- 在 **块与块之间**，采用自回归依赖：
  - 第 `b` 个动作块只能依赖视觉-语言前缀和前面已经生成的动作块。
- 在 **每个块内部**，采用离散扩散去噪：
  - 同一个块内的动作 token 可以并行预测和细化。

这种设计使模型同时具备：

- 自回归模型的全局时间因果结构；
- 扩散模型的块内并行生成能力；
- KV Cache 可复用性；
- 对预训练 AR backbone 更友好的适配路径。

---

## 2.2 模型骨干与动作表示

BlockVLA 基于 OpenVLA / DDVLA 的多模态架构实现，主要包括：

- **视觉编码器**：
  - SigLIP；
  - DINOv2。
- **语言 / 多模态 backbone**：
  - Prismatic-7B；
  - Llama-style Transformer。
- **动作离散化方式**：
  - 每个连续动作维度被离散化为 256 个 bins；
  - 使用基于分位数的 binning，边界来自动作分布的 1% 到 99% 分位数；
  - gripper 开合动作作为独立 token。
- **单步动作 token 数**：
  - 3 个平移维度；
  - 3 个旋转维度；
  - 1 个夹爪维度；
  - 共 7 个 token。

给定动作 horizon `H` 和动作维度 `D`，动作序列被展平为长度 `H × D` 的一维 token 序列，并与多模态前缀拼接：

```text
x = [BOS, visual tokens, proprioceptive tokens, language tokens, action tokens, EOS]
```

其中多模态前缀记为：

```text
c = [BOS, v, p, l]
```

动作序列记为：

```text
a = [a1, a2, ..., aB]
```

其中每个 `ab` 是一个动作块。

## 2.3 离散扩散基础

论文采用 masked discrete diffusion 形式。对于原始 token 序列 `x0`，前向扩散过程会按时间步 `t` 将部分 token 替换为 `[MASK]`：

- token 以一定概率保留原值；
- 以一定概率变为 `[MASK]`。

模型训练目标是：  
给定被 mask 的序列 `xt`，预测原始 clean token `x0`。

损失函数本质上是 masked token 的交叉熵损失：

```text
L(θ) = - E [ 对所有被 mask 位置计算 log pθ(x0 | xt) ]
```

推理时，从全 `[MASK]` token 开始，模型反复执行：

1. 并行预测所有 masked token；
2. 根据置信度固定一部分 token；
3. 对仍不确定的 token 继续 mask；
4. 多轮迭代后得到最终序列。

---

## 2.4 Block Diffusion 形式

BlockVLA 将动作序列划分为多个 block，联合概率分解为：

```text
log pθ(x0) = Σb log pθ(xb | x<b)
```

即第 `b` 个块依赖前面所有块。

但与纯自回归不同，块内不是逐 token 生成，而是使用扩散并行去噪：

```text
pθ(ab0 | abtb, history, c)
```

其中：

- `ab0` 是第 `b` 个 clean 动作块；
- `abtb` 是第 `b` 个块的 noisy / masked 版本；
- `history` 是前面块的信息；
- `c` 是视觉-语言-本体状态前缀。

---

## 2.5 两种块级条件策略

论文讨论了两种 block-wise 训练策略。

### 2.5.1 Teacher Forcing

Teacher Forcing 中，当前块预测时依赖的是前面块的真实 clean token：

```text
pθ(ab0 | abtb, a<b0, c)
```

实现上，为了让模型访问 clean history，输入会拼接 noisy action sequence 和 clean action sequence：

```text
xinput = [c, at, EOS, a0]
```

其中：

- `at` 是所有块被不同程度 mask 后的 noisy action；
- `a0` 是 clean action sequence。

同时需要两个额外处理：

- **Loss masking**：
  - clean suffix `a0` 只是辅助上下文，不参与扩散损失；
  - 对应 CE target 设置为 ignore index。
- **Position-index alignment**：
  - clean suffix 不应被视为新的时间延续；
  - 它与 noisy sequence 中对应 token 使用相同 RoPE position id。

### 2.5.2 Diffusion Forcing

Diffusion Forcing 中，当前块依赖的是前面块的 noisy 或部分去噪版本，而不是 clean history：

```text
pθ(ab0 | abtb, a<bt', c)
```

输入为：

```text
xinput = [c, at, EOS]
```

这种方式更贴近推理时的情况，因为推理过程中前面已经生成的块可能包含残余误差。论文最终默认使用 **Diffusion Forcing**，因为它在实验中明显优于 Teacher Forcing。

## 2.6 非对称多模态掩码设计

VLA 任务与普通语言建模不同：  
视觉、语言、本体状态是输入条件，而动作 token 是生成目标。

因此，BlockVLA 设计了专门的非对称 attention mask：

- 多模态前缀 `c` 内部可以双向注意力交互，便于视觉-语言融合；
- 前缀 token 不允许 attend 到动作 token，避免未来动作泄漏；
- 动作块可以 attend 到完整前缀 `c`；
- 当前块可以 attend 到前面块；
- 当前块内部允许双向注意力；
- 当前块不能 attend 到未来块。

这一设计保证训练与推理的一致性。

---

## 2.7 Token Shift 设计

传统自回归模型使用 one-token shift：

- 位置 `n-1` 的 hidden state 预测位置 `n` 的 token。

DDVLA 延续了这一设计，但 BlockVLA 重新评估了 token shift 对 VLA diffusion finetuning 的影响。

实验表明，在 LIBERO-Object 上：

- 去掉 token shift 并不会降低性能；
- 反而在大多数训练阶段取得更高成功率。

因此，论文默认在 DDVLA baseline 和 BlockVLA 中都去掉 token shift，使输入位置与预测目标位置直接对齐。

## 2.8 训练流程

训练阶段流程如下：

1. 将动作序列 `a` 划分为 `B` 个连续 block；
2. 对每个 block 独立采样扩散时间步 `tb`；
3. 根据 forward masking process 得到 noisy block `abtb`；
4. 使用 block-wise causal attention mask；
5. 对所有 block 并行训练；
6. 仅对 masked action token 计算交叉熵损失；
7. 更新模型参数。

训练时由于整段动作序列已知，因此不使用 KV Cache，以保持并行 batch 训练效率。

---

## 2.9 推理流程

推理阶段采用块级自回归 + 块内扩散：

1. 编码视觉-语言-本体状态前缀 `c`；
2. 将其 KV states 写入 KV Cache；
3. 对第 `b` 个 block：
   - 初始化为全 `[MASK]`；
   - 在固定 KV Cache 条件下进行若干步并行去噪；
   - 得到当前 block 的最终 token；
4. 当前 block 固定后，再前向一次，生成该 block 的 KV states；
5. 将该 block 的 KV states 追加到缓存；
6. 继续生成下一个 block；
7. 直到所有 block 生成完成。

这种流程避免了标准全序列扩散每一步都重新计算整段序列的开销。

## 6. 主要结论与发现

## 6.1 性能提升

在 LIBERO 上，BlockVLA 在 50k training steps 下取得：

- Spatial：90.6%；
- Object：97.6%；
- Goal：93.2%；
- Long：85.2%；
- 平均：91.7%。

相比 DDVLA 在 50k steps 的平均 83.2%，BlockVLA 有明显提升。

尤其在 LIBERO-Long 上，提升非常显著：

- DDVLA 50k：53.6%；
- BlockVLA 50k：85.2%。

这说明块级因果结构对长时域任务帮助很大。

---

## 6.2 训练效率更高

BlockVLA 在早期训练阶段明显优于 DDVLA。例如 LIBERO-Long：

- 5k steps：
  - DDVLA：0.4%；
  - BlockVLA：35.8%。
- 15k steps：
  - DDVLA：3.8%；
  - BlockVLA：62.2%。

这表明 BlockVLA 从 AR 预训练 backbone 迁移到 diffusion policy 时更加平滑，收敛更快。

---

## 6.3 推理速度更快

论文报告 BlockVLA 在单块 RTX 4090 上达到：

- **186.7 tokens/s**。

相比 DDVLA：

- 实现 **3.3× 推理加速**。

相比 OpenVLA：

- 实现约 **8.0× 推理吞吐提升**。

主要原因是：

- 每个 block 只需要少量 denoising step；
- 已完成 block 的 KV Cache 可以复用；
- 避免标准扩散模型反复全序列去噪。

---

## 6.4 Diffusion Forcing 优于 Teacher Forcing

实验显示，在 LIBERO-Object 和 LIBERO-Long 上，Diffusion Forcing 明显优于 Teacher Forcing。

原因是：

- Teacher Forcing 训练时依赖 clean history；
- 推理时历史块可能包含预测误差；
- Diffusion Forcing 使用 noisy / imperfect history，更贴近推理分布；
- 因而更能缓解训练-推理不一致问题。

---

## 6.5 较大 block size 更好

论文比较了 block size 7 和 14：

- block size = 7 对应单步动作；
- block size = 14 可覆盖两个连续动作时间步。

结果显示 block size 14 在 LIBERO-Object 和 LIBERO-Long 上更好。作者认为这是因为较大 block 能在一次并行去噪中建模连续时间步之间的动态相关性，提高轨迹连续性和稳定性。

---

## 6.6 Denoising step 存在速度-性能权衡

在固定 block size = 14 时，论文比较了每块 1 到 4 个 denoising steps：

- 去噪步数增加通常有助于提高动作生成质量；
- 但会降低推理吞吐量；
- 实验中 2 或 3 steps 是较好的平衡点；
- 成功率在 3 steps 附近达到峰值，但 2 steps 推理更快。

---

## 7. 优点与亮点

## 7.1 方法设计亮点

- **兼顾 AR 与 Diffusion 优势**：
  - 块间自回归保证全局因果一致性；
  - 块内扩散保证局部并行生成。
- **支持 KV Cache 复用**：
  - 相比标准双向扩散，BlockVLA 可以缓存多模态前缀和已完成动作块。
- **适合从 AR backbone 迁移**：
  - 保留部分因果结构，降低从 AR 预训练到 diffusion fine-tuning 的范式鸿沟。
- **针对 VLA 非对称结构设计 mask**：
  - 区分条件输入和动作生成目标；
  - 避免将视觉-语言 token 与动作 token 作为完全同质序列处理。
- **Diffusion Forcing 提升推理一致性**：
  - 训练时模拟 noisy history，降低推理时误差累积影响。
- **去掉 token shift 的实证分析有价值**：
  - 指出 AR 语言建模中的 token shift 不一定适合 diffusion VLA。

---

## 7.2 实验亮点

- 覆盖 LIBERO 的四类任务，特别是包含 Long 长时域任务；
- 不只报告最终性能，也报告训练过程中多个 checkpoint；
- 对训练效率和推理吞吐量均有量化；
- 消融实验较完整，覆盖核心设计选择；
- SimplerEnv 结果增强了方法在更复杂仿真环境下的可信度。

---

## 7.3 实用价值

BlockVLA 对实际机器人部署具有较强意义：

- 降低动作生成延迟；
- 提升长任务成功率；
- 减少扩散模型多步去噪开销；
- 使大规模 AR VLA 更容易适配到高频实时控制。

## 8.3 训练细节披露不足

论文虽然提到：

---

## 8.4 对比方法范围有限

主要 baseline 是：

- OpenVLA；
- DDVLA。

虽然这两个 baseline 合理，但论文没有系统比较：

- 其他最新 diffusion VLA；
- 其他并行 decoding 方法；
- cache-based diffusion acceleration 方法；
- continuous diffusion policy；
- action chunking / latent action 方法；
- 更强的 AR VLA 模型。

因此，BlockVLA 相对于更广泛方法谱系的优势仍需进一步验证。

---

## 8.5 Block size 与任务结构可能相关

BlockVLA 的性能依赖 block size 选择：

- block 太小：并行收益有限，难以建模跨步动态；
- block 太大：可能降低因果细粒度控制能力，并增加块内扩散难度。

论文只主要比较了 7 和 14，尚未系统分析：

- 更大 block size；
- 不同动作 horizon；
- 不同动作 tokenization；
- 自适应 block size。

---

## 8.6 长时域误差并未完全消除

BlockVLA 块间仍是自回归的，因此：

- 块级错误仍可能传播；
- 只是相比逐 token AR 减轻了误差累积；
- 如果早期 block 生成明显错误，后续 block 仍会受影响。

Diffusion Forcing 能提高鲁棒性，但不能从根本上消除块级误差传播。

---

## 8.7 对离散动作 tokenization 有依赖

本文基于离散化动作 token：

- 每个动作维度量化为 256 bins；
- 这可能带来量化误差；
- 对高精度连续控制任务可能不够理想；
- 对不同机器人动作空间的适配效果尚未充分验证。

---

## 8.8 真实实时控制闭环仍需验证

论文报告了 token/s 推理吞吐量，但真实机器人控制中还涉及：

- 图像采集延迟；
- 视觉编码开销；
- 控制频率；
- 动作执行延迟；
- 传感器噪声；
- 闭环反馈稳定性。

因此，虽然 BlockVLA 显示出高吞吐潜力，但其端到端真实控制频率和稳定性仍需进一步实验验证。

---

# 总体评价

BlockVLA 是一篇将块扩散引入 VLA 动作生成的较有价值工作。它针对 AR VLA 推理慢、标准离散扩散难以 KV-cache 复用的问题，提出了块间自回归、块内扩散并行去噪的混合范式。实验显示，该方法在 LIBERO 和 SimplerEnv 上不仅提升最终成功率，还显著提高训练收敛速度和推理吞吐量，尤其在长时域任务中表现突出。

其最大贡献在于证明：**Block Diffusion 可以作为连接大规模自回归预训练 VLA 与高频实时机器人控制之间的有效桥梁**。不过，论文仍需要在真实机器人、多随机种子、更广泛 baseline、训练细节复现性和复杂开放环境中进一步验证。
