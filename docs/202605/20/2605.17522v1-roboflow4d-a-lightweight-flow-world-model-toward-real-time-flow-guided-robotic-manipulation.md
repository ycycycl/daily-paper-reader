## 摘要
在三维环境中进行规划与行动是机器人在真实世界中执行操作任务的一项基本能力。尽管已有研究探索了利用预测流规划器来引导三维操作，但现有方法通常依赖于堆叠多个子模型的模块化流水线，导致计算开销较高且实时性能受限。为应对这些挑战，我们提出 RoboFlow4D，这是一种轻量级流世界模型，通过估计物理三维空间中的时间运动来统一感知与规划。作为一个端到端框架，RoboFlow4D 可直接根据视觉观测和文本指令预测多帧三维流，并提供显式的基于流的规划以引导动作生成。该设计能够与通用动作策略无缝集成，形成高效的观测—规划—执行闭环。通过流预测与动作控制之间的慢—快协作，RoboFlow4D 实现了实时且资源高效的操作。仿真和真实场景中的大量实验表明，RoboFlow4D 持续提升了操作成功率和计算效率，推动了面向具身智能的流引导规划的发展。

## Abstract
Planning and acting in 3D environments is a fundamental capability for robotic manipulation in the real world. Although prior work has explored predictive flow planners to guide 3D manipulation, existing approaches often rely on modular pipelines stacking multiple submodels, resulting in high computational overhead and limited real-time performance. To address these challenges, we introduce RoboFlow4D, a lightweight flow world model that unifies perception and planning by estimating temporal motion in physical 3D space. As an end-to-end framework, RoboFlow4D directly predicts multi-frame 3D flows from visual observations and textual instructions, providing explicit flow-based planning to guide action generation. This design allows seamless integration with general action policies, forming an efficient observation-planning-execution closed loop. Through slow-fast collaboration between flow prediction and action control, RoboFlow4D enables real-time and resource-efficient manipulation. Extensive experiments in both simulation and real-world settings demonstrate that RoboFlow4D consistently improves manipulation success rates and computational efficiency, advancing flow-guided planning for embodied intelligence.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体框架

RoboFlow4D 的完整系统由三部分组成：

1. **RoboFlow4D 流世界模型**
   - 输入：历史 RGB 图像序列、可选的 2D 查询点、文本任务指令。
   - 输出：未来多帧 3D 流，即若干关键点在未来时间上的三维轨迹。

2. **Flow-Conditioned Policy Learning**
   - 将 RoboFlow4D 预测的 3D 流编码成条件向量。
   - 动作策略在当前观测、机器人本体状态和流规划的共同条件下生成动作。

3. **Closed-Loop Control**
   - RoboFlow4D 作为低频“慢规划器”。
   - 动作策略作为高频“快执行器”。
   - 二者组成观测—规划—执行闭环，能够在任务中反复重规划。

### 2.2 RoboFlow4D 的输入与多模态编码

RoboFlow4D 接收：

- RGB 图像序列：  
  \[
  I = \{I_1, I_2, ..., I_n\}
  \]
- 可选的 2D 查询点：  
  \[
  Q \in \mathbb{R}^{m \times 2}
  \]
- 文本任务指令。

对应地使用三个编码器：

- **Vision Encoder**
  - 使用 DINOv2 提取局部 patch token。
  - 使用 SigLIP 提取全局图像 token。
  - 局部 token 通过多头注意力聚合成上下文 token。
  - 全局 token 通过均值池化得到全局视觉表示。

- **Point Encoder**
  - 将可选 2D 查询点通过 MLP 映射为点 token。
  - 再通过注意力池化得到紧凑的点条件表示。
  - 如果没有查询点，则该条件置零。

- **Text Encoder**
  - 使用 SigLIP 的文本编码器提取任务语言表示。

### 2.3 3D Perceiver：从 2D 观测中注入 3D 先验

论文认为，仅从 RGB 图像预测 3D 流是有歧义的，因此设计了 **3D Perceiver** 来引入三维几何先验。

- 3D Perceiver 包含：
  - 一组可学习的 3D query。
  - 类似 Flamingo/Perceiver Resampler 的跨注意力模块。
  - 一个 MLP 投影层。

- 它从视觉上下文 token 中提取 3D-aware 表征。

- 为了增强 3D 感知能力，作者使用一个冻结的 3D 基础模型 **VGGT** 作为教师模型，对 RoboFlow4D 的 3D 表征进行蒸馏对齐。

- 对齐损失形式可概括为：
  \[
  L_{align} = \| g_\phi(c_{3D}) - h \|_2^2
  \]
  其中：
  - \(c_{3D}\) 是 RoboFlow4D 的 3D 条件特征。
  - \(h\) 是 VGGT 提取的教师 3D 表征。
  - \(g_\phi\) 是可学习投影器。

### 2.4 FlowDiT：基于扩散 Transformer 的 3D 流预测

RoboFlow4D 使用一个扩散式 Transformer，称为 **FlowDiT**，来预测未来多帧 3D 流。

- 多模态条件包括：
  - 全局图像 token。
  - 3D-aware token。
  - 点查询 token。
  - 文本 token。

- 这些 token 拼接后作为扩散模型的条件输入。

- FlowDiT 由多个 DiT block 组成，每个 block 包含：
  - AdaLN。
  - 时空多头注意力或时空交叉注意力。
  - MLP。

- 输出为：
  \[
  F \in \mathbb{R}^{n_{kp} \times K \times 3}
  \]
  其中：
  - \(n_{kp}\) 是关键点数量。
  - \(K\) 是未来关键帧数量。
  - 每个点有 3D 坐标。

### 2.5 训练目标

RoboFlow4D 采用条件去噪扩散训练，使用 v-prediction 参数化。

整体损失由三部分组成：

\[
L = L_{diff} + \lambda_{align} L_{align} + \lambda_{smooth} L_{smooth}
\]

其中：

- **扩散去噪损失 \(L_{diff}\)**  
  用于让模型从噪声轨迹中恢复真实 3D 流。

- **3D 对齐损失 \(L_{align}\)**  
  用于让模型从 2D 图像中学习更好的 3D 表征。

- **平滑损失 \(L_{smooth}\)**  
  通过惩罚轨迹的二阶时间差分，增强预测流的时间连续性和物理平滑性。

### 2.6 数据生成流程

论文的数据生成分为两个阶段：

1. **夹爪流跟踪**
   - 使用 Grounded-SAM2 对机器人夹爪进行分割。
   - 使用 SpatialTrackerV2 跟踪夹爪上的 3D 点轨迹。
   - 对轨迹进行过滤，去除静止点、异常点和不合理大位移点。

2. **原子任务分解与目标导向流采样**
   - 根据夹爪开合状态将长任务分解为多个原子任务。
   - 例如：
     - pre-grasp
     - grasp
     - transport
     - release
   - 对每个原子任务，从当前状态到该原子任务完成状态采样多帧 3D 流。
   - 采样时使用单调时间变换，使更多关键帧集中在原子任务结束附近，以更精细地捕捉接近目标时的轨迹变化。

### 2.7 流条件动作策略

RoboFlow4D 预测的 3D 流会被 Flow Encoder 编码为全局流条件。

流程为：

1. 用 MLP 提取每个关键点、每个时间帧的特征。
2. 用 attention pooling 聚合为全局流条件。
3. 将该流条件与当前状态编码得到的基础条件拼接。
4. 动作策略基于拼接后的条件生成 action chunk。

论文验证了 RoboFlow4D 可以作为可插拔模块，增强不同动作策略，包括：

- Diffusion Policy，DP。
- Diffusion Transformer Policy，DiT。

### 2.8 慢—快闭环控制机制

论文提出一种双频闭环控制：

- **慢系统**：RoboFlow4D
  - 低频运行。
  - 根据当前观测和任务指令预测一个原子任务级别的 3D 流规划。

- **快系统**：动作策略
  - 高频运行。
  - 在同一个流规划条件下连续执行多个 action chunk。

这种设计的意义是：

- 避免每一步都重新规划，降低延迟。
- 保持动作执行的实时性。
- 当环境或机器人状态偏离时，再由 RoboFlow4D 重新规划，实现恢复和纠错。

- RoboFlow4D 模型规模约为：
  - **0.76B 参数**。
- 相比依赖视频生成的模块化方法：
  - Dream2Flow 需要约 3–11 分钟生成 3D object flow。
  - NovaFlow 需要约 2 分钟。
  - 这些方法通常依赖超过 1B 参数的视频生成专家模型。
- RoboFlow4D 可以在 1 秒内完成一次 4D 流规划。

### 4.2 未明确说明的信息

- 训练 GPU 型号。
- 总训练时长。
- batch size。
- 训练轮数或总步数。
- 预训练与微调的具体计算成本。

因此，论文在**训练资源透明度**方面仍不充分。

## 6. 主要结论与发现

### 6.1 LIBERO 结果

RoboFlow4D 显著提升了轻量动作策略的性能。

- 对 DP：
  - Spatial 从 81.6% 提升到 89.8%。
  - Object 从 91.5% 提升到 93.2%。
  - Goal 从 78.4% 提升到 85.2%。
  - Long 从 64.0% 提升到 72.0%。
  - 平均成功率从 78.9% 提升到 85.1%，提升 6.2%。

- 对 DiT：
  - 平均成功率从 83.7% 提升到 87.7%，提升 4.0%。
  - 在 Spatial 和 Long 上提升尤其明显。

这说明显式 3D 流规划能够增强动作策略，尤其对空间泛化和长时序任务有帮助。

---

### 6.2 ManiSkill3 结果

在单第三视角图像输入的困难设置下，RoboFlow4D 也带来稳定提升。

- DP 平均成功率：
  - 从 12.3% 提升到 22.0%。
  - 提升 9.7%。

- DiT 平均成功率：
  - 从 12.7% 提升到 23.7%。
  - 提升 11.0%。

这说明在空间信息有限的单视角场景中，RoboFlow4D 的 3D 流先验尤其有价值。

---

### 6.3 真实机器人结果

在真实机器人任务中，RoboFlow4D 同样提升成功率并略微减少完成时间。

- DP + RoboFlow4D：
  - 平均成功率从 27.5% 提升到 40.0%。
  - 平均完成时间从 38.2 秒降到 36.8 秒。

- DiT + RoboFlow4D：
  - 平均成功率从 32.5% 提升到 43.8%。
  - 平均完成时间从 39.5 秒降到 38.3 秒。

- DiT + RoboFlow4D 的平均成功率 43.8%，超过 π0 的 41.3%。

这表明 RoboFlow4D 不仅在仿真中有效，也能在真实机器人操作中改善性能。

---

### 6.4 消融实验结论

- 去掉 context token、query points 或 3D alignment 都会使 3D 流预测误差上升。
- 去掉 3D alignment 的影响最大，说明从 VGGT 蒸馏 3D 几何知识很关键。
- 慢—快频率比 \(r=4,2,1\) 下成功率变化不大，说明双系统闭环对不同规划频率较鲁棒。

## 7. 优点与亮点

### 7.1 方法设计亮点

- **端到端 3D 流预测**
  - 避免视频生成、深度估计、分割、点跟踪、3D 重建等多模块堆叠。
  - 显著降低推理延迟。

- **显式 4D 运动规划**
  - 不只预测图像空间中的 2D 轨迹，而是预测三维空间中的多帧运动流。
  - 更适合真实机器人操作。

- **可插拔策略增强**
  - RoboFlow4D 可以作为独立规划模块接入 DP、DiT 等不同动作策略。
  - 不要求重新设计底层控制器。

- **慢—快双系统闭环**
  - 低频规划与高频控制结合。
  - 兼顾规划质量和执行效率。
  - 支持失败后的纠错重规划。

- **目标导向的原子任务流**
  - 不预测固定长度的长轨迹，而是根据当前状态到原子任务目标自适应预测。
  - 更适合长时序操作任务。

- **3D 知识蒸馏**
  - 通过 VGGT 对齐增强从 2D 图像恢复 3D 结构的能力。
  - 有助于解决 RGB 到 3D 轨迹预测的歧义。

---

### 7.2 实验设计亮点

- 覆盖仿真和真实机器人。
- 覆盖多个 benchmark 和多个任务类型。
- 同时报告成功率、完成时间、推理延迟。
- 与 VLA、大模型、传统策略和模块化流方法均有比较。
- 消融实验能较好验证关键模块的必要性。

---

## 8. 不足与局限

### 8.1 训练资源信息不透明

- 论文报告了部署时的 GPU 和推理延迟，但没有完整说明训练阶段的算力消耗。
- 缺少训练 GPU 数量、训练时长、batch size、总训练步数等细节。
- 因此难以全面评估其“轻量级”在训练阶段是否同样成立。

---

### 8.2 真实机器人实验规模仍有限

- 每个真实任务只收集 50 条示范。
- 每个任务平均评估 20 次。
- 虽然能初步证明可行性，但统计置信度和场景覆盖仍有限。
- 真实任务种类主要集中于桌面操作，尚未覆盖更复杂、更动态或更开放的真实环境。

---

### 8.3 依赖夹爪可见性与轨迹跟踪质量

- 数据生成依赖 Grounded-SAM2 分割夹爪和 SpatialTrackerV2 跟踪 3D 点。
- 如果夹爪被遮挡、反光、运动模糊或相机视角不佳，生成的监督流可能有噪声。
- 对于夹爪不可见的数据，论文提到使用 scene-level flow 作为 fallback，但这可能降低一致性。

---

### 8.4 主要建模的是夹爪中心的运动先验

- RoboFlow4D 主要预测 gripper-centric flow。
- 这种设计对机器人操作很实用，但对复杂物体形变、多物体交互、双臂协作、非刚体操作等任务可能不足。
- 它隐式建模机器人—物体关系，但没有显式建模物体动力学、接触力或约束。

---

### 8.5 与模块化 3D 流方法的性能对比不够完整

- 论文强调与 Dream2Flow、NovaFlow 的延迟差异，但主要是引用其报告的分钟级运行时间。
- 缺少在相同 benchmark、相同机器人任务、相同硬件条件下的直接成功率比较。
- 因此关于“全面优于模块化方法”的结论仍需进一步验证。

---

### 8.6 对动态环境和安全性的讨论有限

- 论文的 Impact Statement 提到预测错误、分布偏移和不安全自动化风险。
- 但方法本身尚未显式引入：
  - 不确定性估计。
  - 安全约束。
  - 碰撞检测。
  - 失败回退策略。
- 在动态、人机共处或安全关键场景中，还需要额外机制。

---

### 8.7 语言泛化能力尚不明确

- RoboFlow4D 接收文本指令，但实验主要是固定任务描述和特定任务集合。
- 尚不清楚其对开放词汇、复杂组合指令或未见任务语言的泛化能力如何。

---

## 总体评价

RoboFlow4D 是一篇围绕“实时 3D 流引导机器人操作”的系统性工作。它的核心贡献在于用端到端轻量模型替代重型模块化 3D 流生成流水线，并将多帧 3D 流作为显式规划接口接入动作策略。实验表明，该方法能在 LIBERO、ManiSkill3 和真实机器人任务中稳定提升成功率，并显著降低流规划延迟。其主要亮点是端到端 4D 流预测、3D 知识蒸馏、目标导向原子任务规划和慢—快闭环控制。不过，论文仍存在训练资源不透明、真实场景覆盖有限、对动态与安全约束考虑不足等局限。总体而言，该工作为低延迟、资源高效的流引导机器人操作提供了有价值的方向。
