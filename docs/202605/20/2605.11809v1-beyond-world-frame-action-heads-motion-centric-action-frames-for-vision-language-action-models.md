## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型凭借更强的骨干网络、更广泛的预训练以及更大规模的示范数据集取得了快速进展，然而其动作头仍然高度同质化：大多数方法直接在固定的世界坐标系中预测动作指令。我们提出了 \textbf{MCF-Proto}，一种轻量级动作头，为 VLA 策略配备以运动为中心的动作坐标系（Motion-Centric Action Frame, MCF）和基于原型的动作参数化。在每个步骤中，策略预测一个旋转 $R_t \in SO(3)$，在由一组原型构成的变换后局部坐标系中组合动作，并将其映射回世界坐标系进行端到端训练，且仅使用标准示范数据，无需辅助监督。这一简单设计诱导出稳定的涌现结构。在没有显式方向标签的情况下，学习到的局部坐标系形成了稳定的几何结构，其坐标轴与示范中的末端执行器运动高度兼容。与此同时，学习表示中的动作变得显著更加紧凑，其变化由更少的主导方向捕获，并通过共享原型以更规则的方式组织。这些结构特性带来了更强的鲁棒性，尤其是在几何扰动下。我们的结果表明，在动作头中加入轻量级的几何与组合结构，可以实质性地改善 VLA 策略组织和泛化机器人操作行为的能力。匿名代码仓库已在补充材料中提供。

## Abstract
Vision-Language-Action (VLA) models have advanced rapidly with stronger backbones, broader pre-training, and larger demonstration datasets, yet their action heads remain largely homogeneous: most directly predict action commands in a fixed world coordinate frame. We propose \textbf{MCF-Proto}, a lightweight action head that equips VLA policies with a Motion-Centric Action Frame (MCF) and a prototype-based action parameterization. At each step, the policy predicts a rotation $R_t \in SO(3)$, composes actions in the transformed local frame from a set of prototypes, and maps them back to the world frame for end-to-end training, using only standard demonstrations without auxiliary supervision. This simple design induces stable emergent structure. Without explicit directional labels, the learned local frames develop a stable geometric structure whose axes are strongly compatible with demonstrated end-effector motion. Meanwhile, actions in the learned representation become substantially more compact, with variation captured by fewer dominant directions and more regularly organized by shared prototypes. These structural properties translate into improved robustness, especially under geometric perturbations. Our results suggest that adding lightweight geometric and compositional structure to the action head can materially improve how VLA policies organize and generalize robotic manipulation behavior. An anonymized code repository is provided in the supplementary material.

## 2. 方法论：MCF-Proto 的核心思想与技术细节

论文提出的方法名为 **MCF-Proto**，即 **Motion-Centric Action Frame with Prototype-based Action Parameterization**。

其核心由两个部分组成：

1. **Motion-Centric Action Frame，MCF：以运动为中心的局部动作坐标系**
2. **Prototype-based Action Composition：基于原型字典的动作组合**

### 2.1 整体流程

给定当前观测 $o_t$，VLA 主干网络首先提取特征：

\[
h_t = f_{\text{backbone}}(o_t)
\]

其中观测可包括图像、语言指令、机器人本体状态等。

传统动作头会直接预测世界坐标系下的动作：

\[
a_t = (\Delta x_t, \Delta r_t, a_{t,\text{rest}})
\]

其中：

- $\Delta x_t \in \mathbb{R}^3$：末端执行器平移增量；
- $\Delta r_t \in \mathbb{R}^3$：末端执行器旋转增量；
- $a_{t,\text{rest}}$：其他控制维度，如夹爪动作。

MCF-Proto 则不是直接在世界坐标系中预测动作，而是：

1. 根据特征 $h_t$ 预测一个局部旋转矩阵 $R_t \in SO(3)$；
2. 在该局部坐标系中生成平移和旋转动作；
3. 将局部动作变换回世界坐标系；
4. 使用标准示范动作进行监督。

---

### 2.2 Motion-Centric Action Frame：学习局部动作坐标系

模型通过一个轻量 MLP 从策略特征 $h_t$ 中预测旋转 $R_t$。

- 旋转使用连续的 **6D rotation representation** 表示，以避免欧拉角或四元数可能带来的不连续问题。
- $R_t$ 定义了当前时间步的运动中心局部坐标系。

局部坐标系中的动作记为：

\[
\Delta x_t^{\text{mcf}}, \quad \Delta r_t^{\text{mcf}}
\]

再映射回世界坐标系：

\[
\Delta x_t = R_t \Delta x_t^{\text{mcf}}
\]

\[
\Delta r_t = R_t \Delta r_t^{\text{mcf}}
\]

最终输出动作：

\[
\hat{a}_t = (\Delta x_t, \Delta r_t, \hat{a}_{t,\text{rest}})
\]

该设计的作用是：

- 将场景、视角、机器人初始位姿等造成的几何变化部分吸收到局部坐标系中；
- 使相似的操作动作在局部坐标系中更容易对齐；
- 降低动作表示的分散程度。

---

### 2.3 Prototype-based Action Composition：局部坐标系中的原型组合动作

在 MCF 中，作者进一步使用共享原型字典生成局部动作，而不是直接回归每个动作维度。

分别为平移和旋转维护两个原型字典：

\[
B^{\text{trans}} = \{B_k^{\text{trans}}\}_{k=1}^{K_t}
\]

\[
B^{\text{rot}} = \{B_k^{\text{rot}}\}_{k=1}^{K_r}
\]

每个原型是一个矩阵：

\[
B_k \in \mathbb{R}^{3 \times d}
\]

它将低维潜变量映射到 3D 动作增量。

模型从特征 $h_t$ 中预测：

- 平移潜变量 $z^{\text{trans}} \in \mathbb{R}^d$；
- 旋转潜变量 $z^{\text{rot}} \in \mathbb{R}^d$；
- 平移原型门控权重 $\pi_t^x$；
- 旋转原型门控权重 $\pi_t^r$。

门控权重通过 softmax 得到。

局部平移动作由多个原型加权组合：

\[
\Delta x_t^{\text{mcf}}
=
\sum_{k=1}^{K_t}
\pi_t^x[k] B_k^{\text{trans}} z^{\text{trans}}
\]

局部旋转动作类似：

\[
\Delta r_t^{\text{mcf}}
=
\sum_{k=1}^{K_r}
\pi_t^r[k] B_k^{\text{rot}} z^{\text{rot}}
\]

### 该设计的直观含义

- MCF 负责把不同任务、不同场景中的动作对齐到更合适的局部坐标系；
- 原型字典负责在对齐后的局部空间中复用常见运动模式；
- 门控机制决定当前状态更依赖哪些原型；
- 潜变量 $z$ 调整动作幅度和细节。

作者认为二者是互补关系：

- 只有 MCF：可以做几何对齐，但缺少显式的共享组合结构；
- 只有 Proto：在世界坐标系中组合原型时，不同几何场景下相似动作仍然分散；
- MCF + Proto：先对齐，再组合，因此更容易形成紧凑、可复用的动作结构。

### 2.4 正则项与训练目标

#### 动作监督损失

训练仍使用标准示范数据，不需要额外标签，如：

- 物体位姿；
- 接触状态；
- 动作方向；
- 任务阶段标注。

动作监督损失为：

\[
L_{\text{act}}
=
\|\hat{a}_t[I_{\text{trans}}] - a_t^\star[I_{\text{trans}}]\|_1
+
\|\hat{a}_t[I_{\text{grip}}] - a_t^\star[I_{\text{grip}}]\|_1
+
\text{SmoothL1}(\hat{a}_t[I_{\text{rot}}], a_t^\star[I_{\text{rot}}])
\]

其中：

- 平移和夹爪维度使用 L1；
- 旋转维度使用 Smooth L1。

#### 原型正交正则

为了防止原型字典冗余，作者加入正交正则：

\[
L_{\text{ortho}}
=
\|(B^{\text{trans}})^\top B^{\text{trans}} - I\|_F^2
+
\|(B^{\text{rot}})^\top B^{\text{rot}} - I\|_F^2
\]

该项鼓励不同原型之间保持差异性，避免多个原型学习到相同模式。

#### MCF 时间平滑正则

为了防止局部坐标系在时间上剧烈跳变，作者加入相邻时间步旋转平滑项。

定义：

\[
\Delta R_t = R_t R_{t-1}^\top
\]

\[
\cos \theta_t = \frac{\text{tr}(\Delta R_t)-1}{2}
\]

平滑损失为：

\[
L_{\text{smooth}} = 1 - \cos \theta_t
\]

总损失：

\[
L =
L_{\text{act}}
+
\lambda_{\text{ortho}} L_{\text{ortho}}
+
\lambda_{\text{smooth}} L_{\text{smooth}}
\]

作者进一步分析 MCF 坐标轴是否与示范动作方向一致。

评价方式：

- 对每个时间步，计算示范末端执行器运动方向与 MCF 三个坐标轴之间的最小夹角；
- 夹角越小，说明至少有一个坐标轴与当前运动方向更兼容。

结果表明，MCF-Proto 明显优于只使用 MCF 的版本。

例如：

| 任务 | MCF only | MCF-Proto |
|---|---:|---:|
| place | 35.4° | 18.8° |
| door-close | 34.7° | 18.2° |
| knob-turn | 35.7° | 10.8° |
| drawer-close | 35.4° | 17.4° |
| insert | 32.8° | 13.2° |

这说明原型组合不仅提升预测性能，也促进了局部坐标系学到更有意义的运动方向结构。

### 3.6 消融实验

作者做了层级消融，比较不同组件贡献：

| 方法 | MCF | Proto | 正交正则 | 平滑正则 | Avg |
|---|---|---|---|---|---:|

## 6. 主要结论与发现

论文的主要结论可以概括为：

1. **动作头设计会显著影响 VLA 策略性能。**  
   不只是视觉语言骨干和数据规模重要，动作表示方式本身也很关键。

2. **学习到的 MCF 会形成稳定且有意义的局部几何结构。**  
   即使没有方向标签监督，MCF 坐标轴也会与示范中的末端执行器运动方向高度兼容。

3. **MCF 能使动作表示更紧凑。**  
   与世界坐标系相比，局部坐标系中的动作：
   - 协方差迹更低；
   - 平均距离更小；
   - PCA 前三主成分解释更多方差；
   - 有效秩大幅下降。

4. **原型组合与局部坐标系相互增强。**  
   MCF 对齐动作后，原型更容易捕捉可复用运动模式；原型组合又反过来促进 MCF 学到更稳定的方向结构。

5. **方法在几何扰动下提升尤其明显。**  
   在 LIBERO-plus 的 Camera 和 Robot 扰动下，MCF-Proto 的优势较大，说明其确实增强了对几何变化的鲁棒性。

6. **真实机器人实验支持方法的实际可用性。**  
   在 OpenArm 的多个长时程任务中，MCF-Proto 均优于基线。

## 7. 优点与亮点

### 方法层面

- **轻量级改动**  
  方法主要修改动作头，不需要重新设计整个 VLA 架构。

- **无需额外监督**  
  不需要物体位姿、接触点、方向标签或任务阶段标签，只使用标准示范动作训练。

- **几何归纳偏置明确**  
  将动作先表达在学习到的局部坐标系中，再映射回世界坐标系，符合机器人操作的局部几何特性。

- **组合式动作表示**  
  通过原型字典和门控机制，将动作表示成可复用运动模式的组合，有助于跨任务共享。

- **MCF 与 Proto 互补性强**  
  消融显示二者结合远优于单独使用，说明设计不是简单堆叠组件。

### 实验层面

- **benchmark 覆盖较全面**  
  同时测试 LIBERO、LIBERO-plus 和真实机器人任务。

- **鲁棒性分析有针对性**  
  特别考察 Camera 和 Robot 等几何扰动，与方法动机高度相关。

- **不仅报告成功率，还分析表示结构**  
  包括动作集中度、PCA 有效维度、坐标轴对齐角度、原型使用热图等。

- **真实平台验证增强可信度**  
  OpenArm 实验表明方法不只适用于仿真环境。

## 8. 不足与局限

### 实验覆盖局限

- 主要 benchmark 仍集中在 **LIBERO / LIBERO-plus**，虽然这些是常用机器人操作基准，但场景类型仍有限。
- 真实机器人实验只有三个任务，且平台为 OpenArm，尚不能充分证明跨平台、跨机器人形态的泛化能力。
- 尚未在更复杂场景中充分验证，例如：
  - 高动态操作；
  - 柔性物体操作；
  - 精细接触密集任务；
  - 多物体复杂长程规划；
  - 大范围移动操作。

### 方法表达局限

- 学到的 MCF 是隐式潜在坐标系，并不一定具有明确语义。
- 虽然坐标轴与运动方向对齐，但不能保证其对应物体坐标系、接触坐标系或任务语义坐标系。
- 对于需要明确物体几何、接触约束或力控制的任务，单纯 MCF 可能不足。

### 评估与公平性风险

- 不同基线的训练条件、模型容量、数据预处理是否完全一致，论文中未详细逐一说明。
- LIBERO-plus 每个任务评估 rollout 数较少，可能导致部分成功率估计不够稳定。
- 真实机器人实验样本规模有限，统计置信度仍有提升空间。

### 应用限制

- 方法仍属于行为克隆框架，依赖离线示范数据的覆盖度和质量。
- 若训练数据没有覆盖某些几何变化或失败恢复情形，MCF-Proto 仍可能无法泛化。
- 对于需要在线探索、闭环纠错或长程任务规划的场景，仍需结合强化学习、在线适应或层级规划方法。

### 理论分析有限

- 附录提供了局部紧凑表示偏好会诱导坐标轴对齐主方向的理想化解释。
- 但该分析基于椭圆对称分布和简化目标，与完整神经网络训练过程仍有差距。
- 因此理论更多是解释性支持，而非严格证明 MCF-Proto 在真实任务中的收敛或泛化性质。
