## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型在机器人操作中展现出显著潜力，但其高昂的计算成本阻碍了实时部署。现有的 token 剪枝方法面临一个根本性的权衡：采用剪枝进行激进压缩不可避免地会丢弃关键的几何细节，如接触点，从而导致严重的性能下降。这迫使模型在压缩率上作出妥协，限制了可实现的压缩程度以及由此带来的潜在加速。我们认为，打破这一权衡需要将压缩重新思考为视觉编码器中具有几何感知能力的连续 token 重采样。为此，我们提出了可微分网格采样器（Differentiable Grid Sampler, GridS），这是一个即插即用模块，可在 VLA 中执行任务感知的视觉 token 连续重采样。通过自适应地预测一组最小的显著坐标，并借助可微分插值提取特征，GridS 在实现大幅压缩（少于原始视觉 token 的 10%）的同时保留了必要的空间信息。在 LIBERO 基准和真实机器人平台上的实验表明，GridS 验证了迄今报道的最低可行视觉 token 数量，在成功率不下降的情况下将 FLOPs 减少了 76%。代码可在 https://github.com/Fediory/Grid-Sampler 获取。

## Abstract
Vision-Language-Action (VLA) models have shown remarkable promise in robotics manipulation, yet their high computational cost hinders real-time deployment. Existing token pruning methods suffer from a fundamental trade-off: aggressive compression using pruning inevitably discards critical geometric details like contact points, leading to severe performance degradation. This forces a compromise, limiting the achievable compression rate and thus the potential speedup. We argue that breaking this trade-off requires rethinking compression as a geometry-aware, continuous token resampling in the vision encoder. To this end, we propose the Differentiable Grid Sampler (GridS), a plug-and-play module that performs task-aware, continuous resampling of visual tokens in VLA. By adaptively predicting a minimal set of salient coordinates and extracting features via differentiable interpolation, GridS preserves essential spatial information while achieving drastic compression (with fewer than 10% original visual tokens). Experiments on both LIBERO benchmark and a real robotic platform demonstrate that validating the lowest feasible visual token count reported to date, GridS achieves a 76% reduction in FLOPs with no degradation in the success rate. The code is available at https://github.com/Fediory/Grid-Sampler.

## 1.1 论文关注的问题

本文研究的是 **Vision-Language-Action（VLA）模型在机器人操作中的高效部署问题**。VLA 模型能够将视觉观测、语言指令和机器人动作预测统一到一个模型中，在机器人泛化操作任务中表现出很强潜力，但其计算开销较大，尤其来自视觉编码器产生的大量视觉 token。

典型 VLA 模型会将输入图像通过 ViT / DINOv2 / SigLIP 等视觉编码器编码为密集 token 网格，例如：

- 单张图像可能产生 `16 × 16 = 256` 个视觉 token；
- 多视角输入时 token 数量进一步增加；
- 下游 Transformer 的注意力复杂度通常随序列长度近似二次增长；
- 大量 token 实际对应背景、桌面、无关物体等冗余区域。

因此，视觉 token 数量成为 VLA 实时部署的核心瓶颈。

## 1.2 现有方法的矛盾

现有视觉 token 剪枝方法通常存在一个关键 trade-off：

- **压缩率低**：保留较多 token，性能稳定但加速有限；
- **压缩率高**：大幅删 token，计算下降明显，但容易丢失关键几何细节，如：
  - 夹爪与物体接触点；
  - 物体边缘；
  - 抓取点；
  - 小物体位置；
  - 精细空间关系。

机器人操作不同于普通图像理解，它不仅需要识别“是什么”，还需要知道“在哪里接触”“如何对齐”“何时释放”等几何信息。因此，简单的语义显著性剪枝可能保留物体主体，却丢掉对控制最关键的边界和接触区域。

## 1.3 论文整体含义

本文的核心观点是：

> VLA 的视觉压缩不应被理解为在固定 patch 网格上“离散删 token”，而应被重新表述为一种任务感知、几何感知、连续坐标上的视觉特征重采样。

为此，作者提出 **Differentiable Grid Sampler（GridS）**，通过学习少量连续采样坐标，并用可微双线性插值从密集视觉特征图中提取稀疏 token，从而在极高压缩率下尽量保留关键几何信息。

# 2. 方法论：GridS 的核心思想与技术细节

## 2.1 方法总览

GridS 是一个即插即用模块，插入在：

> 视觉编码器输出的 dense visual tokens 与下游 VLA Transformer 之间。

标准流程为：

1. 输入图像经过视觉编码器，得到密集视觉特征图；
2. GridS 根据全局上下文预测少量连续二维坐标；
3. 在这些连续坐标处用双线性插值采样视觉特征；
4. 加入坐标位置编码，得到稀疏视觉 token；
5. 将稀疏 token 与语言 token、动作 token 一起输入下游 Transformer。

其目标是将原本的 `H × W` 个视觉 token 压缩为 `K` 个 token，其中：

- `K << H × W`
- 例如从 256 个 token 压缩到 16 个，甚至 4 个、1 个。

## 2.2 标准 VLA 的计算瓶颈

输入图像：

\[
I \in \mathbb{R}^{3 \times H_R \times W_R}
\]

经过视觉编码器后得到密集特征：

\[
T_{dense} \in \mathbb{R}^{H \times W \times C}
\]

其中：

- `H × W` 是视觉 token 数量；
- `C` 是特征维度；
- 例如 DINOv2 输出 `16 × 16 = 256` 个 token。

下游 Transformer 的计算复杂度与 token 数量密切相关，视觉 token 越多，计算和显存开销越大。

## 2.3 Global Coordinate Prediction：全局坐标预测

GridS 首先对 dense feature map 做全局平均池化，得到全局上下文向量：

\[
z = \frac{1}{H \times W} \sum_{i=1}^{H} \sum_{j=1}^{W} T^{(i,j)}_{dense}
\]

这个向量概括了当前视觉场景的全局信息。

随后使用一个轻量 MLP 预测 `K` 个连续二维坐标：

\[
P = \sigma(MLP(z))
\]

其中：

- \(P \in [0,1]^{K \times 2}\)；
- 每个坐标表示一个需要采样的空间位置；
- sigmoid 保证坐标落在合法范围内；
- `K` 是目标视觉 token 数量。

这一点与传统剪枝不同：

- 传统方法是在固定网格中选择已有 patch；
- GridS 预测连续坐标，可以落在 patch 之间。

## 2.4 Differentiable Bilinear Sampling：可微双线性采样

对于预测出的连续坐标 \(P(x,y)\)，它通常不会正好落在整数网格点上。

GridS 取该点周围四个邻近网格点进行双线性插值。设左上角点为：

\[
P_1 = (x_i, y_i) = (\lfloor x \rfloor, \lfloor y \rfloor)
\]

偏移为：

\[
\Delta x = x - x_i
\]

\[
\Delta y = y - y_i
\]

四个邻居的权重为：

\[
\omega_1 = (1-\Delta x)(1-\Delta y)
\]

\[
\omega_2 = \Delta x(1-\Delta y)
\]

\[
\omega_3 = (1-\Delta x)\Delta y
\]

\[
\omega_4 = \Delta x \Delta y
\]

最终采样特征为四邻居加权和：

\[
F_{sampled}(x,y) = \sum_{k=1}^{4} \omega_k \cdot P_k
\]

这种方式的意义是：

- 可以从 patch 之间的位置提取特征；
- 减少固定 patch 网格造成的空间量化误差；
- 更适合抓取点、边缘、接触点等亚 patch 级几何区域。

## 2.5 可微性与端到端优化

双线性插值对坐标 \(x,y\) 是可微的，因此下游任务损失可以反向传播到坐标预测 MLP：

\[
\frac{\partial L}{\partial x}
=
\sum_{k=1}^{4}
\frac{\partial L}{\partial F_{sampled}}
\cdot P_k \cdot
\frac{\partial \omega_k}{\partial x}
\]

这意味着 GridS 不是基于人工规则或测试时启发式剪枝，而是：

- 在 VLA fine-tuning 过程中端到端学习；
- 由动作预测误差直接驱动；
- 自动学习“哪些视觉位置对当前任务最重要”。

## 2.6 Geometry Injection：几何坐标注入

由于 GridS 采样后得到的是无规则排列的稀疏 token，原始二维网格结构被破坏。因此作者引入坐标编码器，将预测坐标 \(P\) 映射为位置嵌入：

\[
E_{pos} \in \mathbb{R}^{K \times C}
\]

最终稀疏 token 为：

\[
T_{spa} = F_{spa} + E_{pos}
\]

这样下游 Transformer 不仅知道 token 的视觉语义，还知道其空间位置。

## 2.7 训练方式

GridS 与 VLA 模型一起进行标准 fine-tuning：

- 不需要额外 attention 标注；
- 不需要显著性监督；
- 不需要 ground-truth 关键点；
- 只使用原本 VLA 的动作预测损失；
- 可适配 autoregressive VLA 和 flow-matching VLA。

作者强调，很多现有 pruning 方法是 training-free，并非真正优势，而是因为其离散选择机制不可微，无法端到端训练。

# 6. 主要结论与发现

## 6.1 GridS 可以极大减少视觉 token

在 LIBERO 上，GridS 将视觉 token 从 256 减少到 16，甚至 4 或 1，在很多任务中仍保持甚至超过 baseline。

典型结果包括：

- π0 baseline：256 tokens，平均成功率 94.4%；
- π0 + GridS 16 tokens：平均成功率 96.0%，提升 +1.6%；
- π0 + GridS 4 tokens：平均成功率 95.5%，提升 +1.1%；
- π0.5 baseline：平均成功率 96.7%；
- π0.5 + GridS 16 tokens：平均成功率 97.7%，提升 +1.0%。

附录中甚至报告：

- π0.5 + GridS `K=1`：平均成功率 96.6%；
- π0.5 baseline：96.7%。

这表明标准 VLA 的视觉 token 存在极大冗余。

## 6.2 GridS 显著降低计算成本

在 LIBERO 上：

- π0 baseline FLOPs：216.01G；
- π0 + GridS 16：51.65G；
- FLOPs 约降低 76%。

训练速度方面：

- π0 baseline：14.32 s/step；
- π0 + GridS：4.21 s/step；
- 加速 3.4×。

π0.5：

- baseline：12.83 s/step；
- GridS：4.41 s/step；
- 加速 2.9×。

## 6.3 单 batch 推理加速有限，但大 batch 更明显

作者发现：

- 单 batch 推理时 wall-clock speedup 约为 1.2×；
- 这是因为 JAX 编译后的 dense baseline 已高度优化，固定 kernel overhead 占比较高；
- batch size 增大后，GridS 的速度优势更明显，可达到约 3.2×。

## 6.4 GridS 提升真实世界和 OOD 鲁棒性

真实 SO100 机器人实验中，GridS 在多个任务上超过 SmolVLA baseline。

尤其在 Stack Cubes：

- baseline 成功率约 7.6%；
- GridS 成功率 60.0%；
- 提升 +52.4%。

在 OOD 场景中，GridS 也明显更稳健。作者报告整体 OOD 成功率提升约 +28.6%。

## 6.5 连续采样优于离散剪枝

在 LIBERO 上：

- Random Sampling 16 tokens：平均成功率 87.8%；
- Top-K Pruning 16 tokens：90.5%；
- GridS 16 tokens：96.0%。

这说明性能提升不只是因为减少 token、防止过拟合，而是来自可微、几何感知、连续重采样机制。

## 6.6 信息瓶颈可能带来泛化优势

作者认为 GridS 不只是加速模块，也是一种信息瓶颈：

- dense token 可能让模型过拟合背景、纹理、固定布局；
- GridS 迫使模型只关注最小充分视觉统计量；
- 适当丢弃背景信息反而提升 OOD 泛化。

信息保留图显示，在真实任务中 GridS 通常保留约 0.8–0.9 的信息，而非 1.0，作者认为这有助于过滤非因果噪声。

# 7. 优点与亮点

## 7.1 方法设计简洁且可插拔

GridS 只是在视觉编码器与 Transformer 之间加入一个轻量模块，不需要大规模改造 VLA 架构，适配性较好。

## 7.2 从“离散剪枝”转向“连续重采样”

这是论文最核心的创新点。相比固定 patch 上的离散选择，连续坐标采样更适合机器人操作中的：

- 接触点；
- 抓取点；
- 物体边界；
- 夹爪-物体相对几何关系。

## 7.3 可微分，能够端到端优化

双线性插值使采样坐标可以被任务损失直接监督，避免了传统剪枝中不可微、启发式、测试时后处理的问题。

## 7.4 实验覆盖较广

论文不仅在 LIBERO 上做实验，还包括：

- ALOHA；
- SO100 真实机器人；
- LIBERO-PLUS；
- RoboTwin 初步扩展；
- 多种 VLA backbone。

这比单一仿真 benchmark 更有说服力。

## 7.5 效率与性能同时提升

通常压缩会带来性能下降，但 GridS 在多个任务中同时实现：

- FLOPs 大幅降低；
- 训练速度提升；
- 推理延迟下降；
- 成功率不下降甚至提升；
- OOD 鲁棒性提升。

## 7.6 消融实验比较完整

作者系统分析了：

- token 数量；
- pooling；
- MLP 容量；
- 坐标注入；
- 插值方式；
- 随机采样和 Top-K；
- 视角缺失；
- 信息保留图。

这些有助于解释 GridS 为什么有效。

# 8. 不足与局限

## 8.1 单 batch 推理加速有限

尽管 FLOPs 降低显著，但在真实实时机器人中通常 batch size 为 1。论文也承认：

- 单 batch 推理加速约 1.2×；
- 远低于 FLOPs 降低比例；
- 实际速度受 kernel overhead、框架优化、硬件调度影响较大。

因此，GridS 的实际实时收益在某些部署条件下可能没有理论指标那么大。

## 8.2 token 数量是静态超参数

GridS 目前需要预设 `K`，例如 16 或 32。不同任务、场景复杂度、视角条件下最优 `K` 不同。

真实实验中出现明显倒 U 型趋势：

- `K=4` 信息不足；
- `K=16` 最优；
- `K=32` 反而因背景噪声增多而下降。

这说明未来需要动态预测 token 数量，而不是固定预算。

## 8.3 极端压缩结果需要谨慎解读

论文报告 `K=1` 在 LIBERO 上几乎不掉点，这一结果很有趣，但也可能受到以下因素影响：

- LIBERO 任务分布较规则；
- 模型可能利用语言和训练集先验；
- 部分任务可能存在过拟合或记忆化；
- 仿真环境视觉复杂度有限。

因此，不能简单推广为所有机器人任务都只需一个视觉 token。

## 8.4 对 PEFT / LoRA 兼容性不足

论文在局限中指出，GridS 与 LoRA 等参数高效微调方法兼容性较差：

- 使用 LoRA 时 GridS 比 dense baseline 低 8.3%；
- 原因可能是 GridS 改变了视觉 token 空间分布；
- 冻结的大模型注意力结构难以适应这种稀疏连续 token。

这限制了 GridS 在低成本微调场景中的实用性。

## 8.5 真实世界实验规模仍有限

真实机器人实验虽然有价值，但数据量和任务数量仍较小：

- 3 个任务；
- 每个任务几十条 demonstration；
- 一个低成本 SO100 平台；
- OOD 场景主要围绕这些任务构造。

对于更复杂机器人、更多物体、更长任务链、更动态环境，仍需进一步验证。

## 8.6 与 training-free 方法比较存在训练条件差异

FastV、SparseVLM 等方法是训练后剪枝，而 GridS 是端到端 fine-tuning。虽然这正是 GridS 的优势之一，但从严格公平比较角度看：

- GridS 获得了额外训练适配机会；
- training-free 方法没有；
- 因此性能差距部分可能来自训练机制，而非采样结构本身。

不过作者通过 Random Sampling / Top-K 的 full fine-tuning 对比，部分缓解了这一问题。

## 8.7 可能丢失细粒度语义或运动学信息

LIBERO-PLUS 中 GridS 在以下扰动上有所下降：

- Robot Initial States；
- Objects Layout；
- Language Instructions。

这说明压缩过强时，可能损失：

- 完整机械臂姿态；
- 小物体细节；
- 复杂语言 grounding 所需的上下文；
- 多目标精细区分信息。

# 总结

本文提出的 GridS 将 VLA 视觉 token 压缩从传统的离散剪枝转化为可微分、任务感知、连续坐标重采样。它通过全局上下文预测少量采样点，再用双线性插值从密集视觉特征中提取稀疏 token，并注入坐标位置编码。实验表明，GridS 可以在使用少于 10% 视觉 token 的情况下显著降低 FLOPs 和训练成本，同时在 LIBERO、ALOHA 和真实机器人任务中保持甚至提升成功率，尤其在 OOD 场景中表现出较强鲁棒性。

其主要价值在于证明：对于机器人操作，视觉输入中的大量 dense token 可能是冗余甚至有害的；一个经过端到端优化的信息瓶颈可以帮助模型聚焦关键几何结构，从而同时提升效率和泛化能力。不过，GridS 仍存在单 batch 实际加速有限、固定 token budget、真实实验规模有限、PEFT 兼容性不足等问题，后续仍需在更复杂、更动态、更大规模的机器人环境中验证。
