## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景范式。然而，现有推理机制仍难以提供面向规划的中间表征：文本形式的思维链（Chain-of-Thought, CoT）无法保留连续的时空结构，而潜在世界推理仍难以作为动作生成的直接条件。本文提出 CoWorld-VLA，一种用于自动驾驶的多专家世界推理框架，其中世界表征作为显式条件来指导动作规划。CoWorld-VLA 通过多源监督提取互补的世界信息，并将其编码为 VLA 中的专家令牌，从而提供规划器可访问的条件信号。具体而言，我们构建了四类令牌：语义交互令牌、几何结构令牌、动态演化令牌和自车轨迹令牌，分别用于建模交互意图、空间结构、未来时序动态和行为目标。在动作生成过程中，CoWorld-VLA 采用基于扩散的分层多专家融合规划器，并在联合去噪过程中始终与场景上下文耦合，以生成连续的自车轨迹。实验表明，CoWorld-VLA 在 NAVSIM v1 基准上的未来场景生成和规划任务中均取得了具有竞争力的结果，展现出较强的避碰能力和轨迹精度。消融研究进一步验证了专家令牌之间的互补性，以及它们作为动作生成规划条件的有效性。代码将在 https://github.com/potatochip1211/CoWorld-VLA 发布。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for end-to-end autonomous driving. However, existing reasoning mechanisms still struggle to provide planning-oriented intermediate representations: textual Chain-of-Thought (CoT) fails to preserve continuous spatiotemporal structure, while latent world reasoning remains difficult to use as a direct condition for action generation. In this paper, we propose CoWorld-VLA, a multi-expert world reasoning framework for autonomous driving, where world representations serve as explicit conditions to guide action planning. CoWorld-VLA extracts complementary world information through multi-source supervision and encodes it into expert tokens within the VLA, thereby providing planner-accessible conditioning signals. Specifically, we construct four types of tokens: semantic interaction, geometric structure, dynamic evolution, and ego trajectory tokens, which respectively model interaction intent, spatial structure, future temporal dynamics, and behavioral goals. During action generation, CoWorld-VLA employs a diffusion-based hierarchical multi-expert fusion planner, which is coupled with scene context throughout the joint denoising process to generate continuous ego trajectories. Experiments show that CoWorld-VLA achieves competitive results in both future scene generation and planning on the NAVSIM v1 benchmark, demonstrating strong performance in collision avoidance and trajectory accuracy. Ablation studies further validate the complementarity of expert tokens and their effectiveness as planning conditions for action generation. Code will be available at https://github.com/potatochip1211/CoWorld-VLA.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 核心思想

- CoWorld-VLA 的核心思想是：  
  **不再依赖纯文本 CoT 或单一隐式世界表示，而是在 VLM/VLA 的潜在空间中构建多专家 Latent CoT，并将这些专家 token 显式作为轨迹规划条件。**

- 论文设计了四类专家 token：
  - **Semantic Interaction Token / 语义交互 token**  
    用于建模交通参与者、场景语义和交互意图。
  - **Geometric Structure Token / 几何结构 token**  
    用于建模道路布局、空间结构、车道几何和 3D 约束。
  - **Dynamic Evolution Token / 动态演化 token**  
    用于建模未来场景变化、交通流演化和视觉动态。
  - **Ego Trajectory Token / 自车轨迹 token**  
    用于建模自车行为目标和轨迹先验。

## 2.2 整体三阶段训练流程

CoWorld-VLA 采用三阶段训练：

### Stage 1：动作条件化预测世界模型

- 使用 **Wan2.2-5B 视频生成模型**作为基础视频扩散 Transformer。
- 目标是学习从历史帧和自车意图条件预测未来视频 latent。
- 输入包括：
  - 历史视频帧；
  - 速度描述；
  - 导航指令；
  - 未来轨迹文本化描述；
  - 静态场景 prompt。
- 使用冻结的 Wan VAE 将视频编码为 latent：
  - 历史 latent 作为观测上下文；
  - 未来 latent 加噪并通过 flow matching 学习去噪。
- 损失函数为未来 latent 上的 flow matching loss，用于让模型学习未来场景演化。

### Stage 2：多专家世界表征学习

- 使用 **Qwen3-VL-2B** 作为 VLM 主干。
- 在 VLM 输入序列中插入四组可学习专家 token。
- VLM 输出对应隐藏状态：
  - `Hsem`：语义交互表示；
  - `Hgeo`：几何结构表示；
  - `Hdyn`：动态演化表示；
  - `Htraj`：自车轨迹表示；
  - `Hctx`：场景上下文表示。

- 多源监督包括：
  - **V-JEPA / JEPA 分支**  
    提供高层语义和预测性表示监督，用于训练 `Hsem`。
  - **VGGT 分支**  
    提供三维几何和空间结构监督，用于训练 `Hgeo`。
  - **Wan 世界模型分支**  
    使用 `Hdyn` 作为条件生成未来场景，通过 flow matching 监督动态 token。
  - **轨迹回归分支**  
    使用 `Htraj` 通过轻量 MLP 回归未来自车轨迹，用 MSE 监督。

- 总损失为多分支加权和：
  - 动态演化损失；
  - 语义对齐损失；
  - 几何对齐损失；
  - 轨迹回归损失。

- 论文给出的权重为：
  - `wdyn = 1.0`
  - `wsem = 0.1`
  - `wgeo = 0.1`
  - `wtraj = 1.0`

### Stage 3：层级多专家融合扩散规划器 HMEF

- Stage 3 设计了 **Hierarchical Multi-Expert Fusion, HMEF** 规划器。
- 目标是把 Stage 2 学到的专家 token 真正转化为可执行轨迹。

- 输入包括：
  - VLM 场景上下文 token；
  - 当前帧 JEPA token；
  - 当前帧 VGGT token；
  - 四类专家 action token；
  - 历史自车状态；
  - 当前速度、加速度等低层车辆状态。

- 关键设计：
  - 使用 **Perceiver-style compressor** 压缩较长的场景 token 序列，降低去噪计算成本。
  - 对每个专家分支使用独立双向 Transformer 编码，并按未来时间步对齐。
  - 在归一化轨迹空间中进行扩散去噪。
  - 每个专家分支预测一条轨迹，再通过可学习权重融合。

- 扩散过程：
  - 将真实轨迹归一化到 `[-1, 1]`。
  - 采样噪声和扩散时间步，构造 noisy action。
  - 去噪器采用双流结构：
    - clean scene stream；
    - noisy action stream。
  - 通过联合 self-attention 将场景上下文和专家动作条件耦合。
  - 最终输出各专家轨迹，并通过可学习权重融合为最终轨迹。

- 推理时：
  - 从高斯噪声开始；
  - 迭代去噪；
  - 融合专家轨迹；
  - 反归一化得到可执行自车轨迹。

  - 继续加入 Semantic Interaction token 后提升到 **87.7**。
  - 再加入 Dynamic Evolution token 后提升到 **88.7**。
  - 说明语义、几何、动态和轨迹信息具有互补性。

- **HMEF 扩散规划器进一步提升轨迹质量**
  - Stage 2 VLM 直接规划 PDMS 为 **88.7**。
  - 加入 ReCogDrive action expert 后为 **89.1**。
  - 加入 HMEF 后提升到 **89.8**。
  - 定性结果显示，HMEF 在车道保持、转弯和绕行场景中轨迹更接近 ground truth。

- **未来视频生成质量也明显改善**
  - CoWorld-VLA 在视频生成上 FVD 为 **32.7**，优于对比方法：
    - SVD：227.5；
    - GenAD：184.0；
    - DrivingGPT：142.6；
    - Epona：61.3；
    - DriveLaW：55.6。
  - 定性结果显示 Stage 2 比 Stage 1 更能保持转弯意图、车道结构和局部物体一致性。

- **动态演化和轨迹专家在规划中权重较高**
  - 学到的专家权重显示：
    - 动态演化专家约 0.35；
    - 轨迹专家约 0.31；
  - 说明未来运动趋势和自车行为目标对轨迹规划最关键。

## 7. 优点与亮点

- **方法设计有较强针对性**
  - 针对文本 CoT 不适合连续时空规划的问题，提出 Latent CoT，更符合自动驾驶任务需求。

- **多专家世界表示设计合理**
  - 将语义、几何、动态和自车轨迹四类规划相关信息显式拆分，避免单一 latent world representation 信息不足的问题。

- **较好地连接了世界模型与动作规划**
  - 不只是将世界模型作为辅助监督，而是将专家 token 在推理阶段直接作为规划条件使用。

- **使用多种基础模型知识**
  - JEPA 提供语义和预测表示；
  - VGGT 提供几何结构；
  - Wan 提供未来视频动态；
  - VLM 提供多模态场景理解；
  - HMEF 负责将这些信息转为可执行轨迹。

- **实验覆盖了生成与规划两个层面**
  - 同时证明模型能够改善未来场景生成和轨迹规划，增强了“世界模型有助于规划”的论证。

- **消融实验较清晰**
  - 多专家 token 的逐步加入带来持续性能提升，说明每个专家分支都有贡献。
  - HMEF 模块相较 Stage 2 回归头也有增益，证明扩散式融合规划器的必要性。

- **单帧前视相机设置下性能突出**
  - 在不使用 LiDAR 和多帧输入的情况下，取得较高 PDMS，说明 latent world reasoning 对输入受限场景有帮助。

## 8. 不足与局限

- **训练成本高**
  - 三阶段训练需要大量 A800 GPU：
    - Stage 1：64 张 A800，约 74 小时；
    - Stage 2：32 张 A800，约 70 小时；
    - Stage 3：16 张 A800，约 20 小时。
  - 对普通研究机构复现成本较高。

- **系统复杂度高**
  - 同时依赖 Wan、Qwen3-VL、V-JEPA、VGGT、HMEF 等多个模块。
  - 多阶段训练和多模型对齐增加了工程复杂度和调参难度。

- **输入模态有限**
  - 当前主要使用单帧前视相机输入。
  - 尚未扩展到多摄像头、多帧、多模态传感器融合。
  - 对实际自动驾驶中的环视感知和长时序建模仍有不足。

- **评估场景仍有限**
  - 轨迹规划主要在 NAVSIM v1 上评估。
  - 缺少在 nuScenes、Waymo、CARLA、真实车辆闭环等更多 benchmark 上的泛化验证。

- **open-loop / non-reactive 评估限制**
  - NAVSIM 是 non-reactive open-loop 协议，无法完全反映与其他交通参与者交互后的闭环行为。
  - 实际驾驶中，其他车辆会对自车动作产生反应，这一点在当前实验中体现有限。

- **公平性可能存在讨论空间**
  - 对比方法的输入传感器、帧数、训练数据规模和是否使用强化学习并不完全一致。
  - 论文虽然标注了 Camera / LiDAR、单帧 / 多帧等设置，但严格公平对比仍较困难。

- **未来信息监督可能带来训练-推理分布差异**
  - Stage 2 中 JEPA、VGGT 等专家监督来自未来观测，虽然推理时不使用未来信息，但训练时依赖未来 supervision。
  - 这种设计是合理的蒸馏式学习，但仍可能存在训练目标与推理可用信息之间的差异。

- **真实安全性仍需更多验证**
  - 虽然 NC、TTC 等指标较好，但自动驾驶安全需要更多长尾场景、异常行为、恶劣天气和复杂交互测试。
  - 论文没有系统评估鲁棒性、分布外泛化和失效案例。
