## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景的范式。然而，现有推理机制仍难以提供面向规划的中间表示：文本式思维链（Chain-of-Thought, CoT）无法保留连续的时空结构，而潜在世界推理仍难以作为动作生成的直接条件。本文提出 CoWorld-VLA，一个用于自动驾驶的多专家世界推理框架，其中世界表示作为显式条件来指导动作规划。CoWorld-VLA 通过多源监督提取互补的世界信息，并将其编码为 VLA 内的专家 token，从而提供规划器可访问的条件信号。具体而言，我们构建了四类 token：语义交互、几何结构、动态演化和自车轨迹 token，分别建模交互意图、空间结构、未来时间动态和行为目标。在动作生成过程中，CoWorld-VLA 采用基于扩散的分层多专家融合规划器，并在联合去噪过程中始终与场景上下文耦合，以生成连续的自车轨迹。实验表明，CoWorld-VLA 在 NAVSIM v1 基准上的未来场景生成和规划任务中均取得了具有竞争力的结果，在避碰和轨迹精度方面表现出强劲性能。消融研究进一步验证了专家 token 的互补性及其作为动作生成规划条件的有效性。代码将发布于 https://github.com/AFARI-Research/CoWorld-VLA。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for end-to-end autonomous driving. However, existing reasoning mechanisms still struggle to provide planning-oriented intermediate representations: textual Chain-of-Thought (CoT) fails to preserve continuous spatiotemporal structure, while latent world reasoning remains difficult to use as a direct condition for action generation. In this paper, we propose CoWorld-VLA, a multi-expert world reasoning framework for autonomous driving, where world representations serve as explicit conditions to guide action planning. CoWorld-VLA extracts complementary world information through multi-source supervision and encodes it into expert tokens within the VLA, thereby providing planner-accessible conditioning signals. Specifically, we construct four types of tokens: semantic interaction, geometric structure, dynamic evolution, and ego trajectory tokens, which respectively model interaction intent, spatial structure, future temporal dynamics, and behavioral goals. During action generation, CoWorld-VLA employs a diffusion-based hierarchical multi-expert fusion planner, which is coupled with scene context throughout the joint denoising process to generate continuous ego trajectories. Experiments show that CoWorld-VLA achieves competitive results in both future scene generation and planning on the NAVSIM v1 benchmark, demonstrating strong performance in collision avoidance and trajectory accuracy. Ablation studies further validate the complementarity of expert tokens and their effectiveness as planning conditions for action generation. Code will be available at https://github.com/AFARI-Research/CoWorld-VLA.

  > 如何构建一种既能保留连续时空结构、又能直接服务于轨迹规划的中间世界表示？

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 方法总览

- 论文提出 **CoWorld-VLA**，即一个面向自动驾驶的 **多专家世界模型推理框架**。
- 其核心思想是：
  - 不使用纯文本 CoT 作为推理中间状态；
  - 而是在 VLM/VLA 的隐藏空间中构建多个专家 token；
  - 这些 token 分别承载不同类型的世界知识；
  - 最终作为轨迹生成器的显式条件，用于扩散式动作规划。

CoWorld-VLA 的整体训练流程分为三个阶段：

1. **Stage 1：动作条件化预测世界模型训练**
2. **Stage 2：多专家世界表示学习**
3. **Stage 3：分层多专家融合扩散规划器训练**

---

## 2.2 输入与问题建模

- 输入包括：
  - 当前前视图像；
  - 导航指令；
  - 自车历史位姿；
  - 当前速度、加速度等状态。
- 目标是预测未来一段时间内的自车轨迹：

  \[
  A_{t+1:t+T}
  \]

- 标准 VLA 通常建模为：

  \[
  p_\theta(A_{t+1:t+T} \mid o_t, c_t)
  \]

- CoWorld-VLA 引入多专家潜在世界表示：

  \[
  Z = \{z_{sem}, z_{geo}, z_{dyn}, z_{traj}\}
  \]

  因而轨迹生成被改写为：

  \[
  p_\theta(A_{t+1:t+T} \mid o_t, c_t, Z)
  \]

其中四类专家表示分别对应：

- **Semantic Interaction Token**：语义交互、交通参与者意图、高层场景关系；
- **Geometric Structure Token**：道路布局、空间约束、3D 几何结构；
- **Dynamic Evolution Token**：未来场景动态演化；
- **Ego Trajectory Token**：自车行为目标和轨迹先验。

## 2.3 Stage 1：动作条件化预测世界模型

- 该阶段基于 **Wan2.2-5B 视频生成模型**构建预测式世界模型。
- 使用冻结的 Wan VAE 将视频序列编码为潜在表示：

  \[
  z = E_{vae}(x) = [z_h, z_f]
  \]

  其中：
  - \(z_h\)：历史帧潜变量；
  - \(z_f\)：未来帧潜变量。

- 模型只对未来部分施加噪声，并通过 flow matching 学习未来场景演化。
- 条件信息通过文本 prompt 构造，包括：
  - 场景描述；
  - 自车速度；
  - 导航命令；
  - 未来轨迹点。

- 文本 prompt 经 Wan 的冻结文本编码器编码后，用于引导未来视频生成。
- 该阶段的作用是学习一个能够根据历史图像和自车意图预测未来视觉场景的世界模型。

## 2.4 Stage 2：多专家世界表示学习

- Stage 2 使用 **Qwen3-VL-2B** 作为 VLM 主干。
- 输入包括：
  - 当前图像；
  - 驾驶任务 prompt；
  - 四组可学习专家 token。

VLM 输出隐藏状态：

\[
\{H_{ctx}, H_{sem}, H_{geo}, H_{dyn}, H_{traj}\}
\]

其中：

- \(H_{ctx}\)：上下文 token；
- \(H_{sem}\)：语义交互专家 token；
- \(H_{geo}\)：几何结构专家 token；
- \(H_{dyn}\)：动态演化专家 token；
- \(H_{traj}\)：轨迹专家 token。

### 多专家监督来源

- **语义专家监督**：
  - 使用冻结的 **V-JEPA** 提取未来观测中的高层语义和预测表示；
  - 通过 SmoothL1 和余弦相似度损失对齐 VLM 的语义 token。

- **几何专家监督**：
  - 使用冻结的 **VGGT** 提取未来观测中的 3D 几何结构特征；
  - 使用 MSE 损失对齐几何 token。

- **动态专家监督**：
  - 使用 Stage 1 中训练好的 Wan 世界模型；
  - 将 \(H_{dyn}\) 作为未来视频生成条件；
  - 通过 flow matching 损失使动态 token 学习未来场景演化。

- **轨迹专家监督**：
  - 将 \(H_{traj}\) 输入轻量 MLP 轨迹回归头；
  - 直接监督未来轨迹点；
  - 损失为 MSE。

整体训练目标为：

\[
L_{total} = w_{dyn}L_{dyn} + w_{sem}L_{sem} + w_{geo}L_{geo} + w_{traj}L_{traj}
\]

论文中设置：

- \(w_{dyn}=1.0\)
- \(w_{sem}=0.1\)
- \(w_{geo}=0.1\)
- \(w_{traj}=1.0\)

## 2.5 Stage 3：分层多专家融合扩散规划器 HMEF

- Stage 3 设计了 **Hierarchical Multi-Expert Fusion，HMEF** 规划器。
- 该模块的目标是将 Stage 2 学到的专家 token 真正用于动作生成。

输入包括：

- 场景上下文 token：
  - VLM 上下文 token；
  - 当前帧 JEPA token；
  - 当前帧 VGGT token。
- 专家动作 token：
  - \(H_{sem}\)
  - \(H_{geo}\)
  - \(H_{dyn}\)
  - \(H_{traj}\)

### 关键技术

- 使用 **Perceiver 查询机制**压缩长场景 token，降低扩散去噪计算成本。
- 每个专家分支使用独立的双向 Transformer 编码，并对齐到未来规划时间步。
- 轨迹在归一化动作空间中进行扩散建模。
- 训练时从真实轨迹和高斯噪声构造带噪轨迹：

  \[
  A_\tau = (1-\tau)\epsilon + \tau A_{norm}
  \]

- 模型学习从带噪轨迹中恢复干净轨迹。
- 每个专家分支都会预测一条轨迹，随后通过可学习权重进行融合：

  \[
  \bar{A} = \sum_e \alpha_e \hat{A}_e
  \]

- 推理时：
  - 从高斯噪声开始；
  - 迭代去噪；
  - 融合多专家轨迹；
  - 反归一化得到最终可执行自车轨迹。

## 6. 主要结论与发现

- CoWorld-VLA 在 NAVSIM v1 上取得了较强规划性能：
  - PDMS 达到 **89.8**；
  - 高于 Uni-World VLA、DriveLaW、ResWorld 等对比方法。
- 在关键子指标上表现突出：
  - NC 达到 **99.2**；
  - EP 达到 **83.6**；
  - 表明其在避碰和保持前进效率方面较强。
- 在未来视频生成任务中，CoWorld-VLA 获得 **FVD = 32.7**，优于 SVD、GenAD、DrivingGPT、Epona、DriveLaW 等方法。
- 消融实验显示：
  - 仅使用轨迹 token 时 PDMS 为 **83.7**；
  - 加入几何 token 后提升至 **85.1**；
  - 加入语义 token 后提升至 **87.7**；
  - 加入动态 token 后提升至 **88.7**；
  - 说明多专家 token 具有明显互补性。
- HMEF 进一步提升规划能力：
  - Stage 2 VLM：PDMS 88.7；
  - 加 ReCogDrive action expert：PDMS 89.1；
  - 加 HMEF：PDMS 89.8。
- 扩散去噪步数实验显示：
  - 10 步去噪在性能和效率之间最优。

## 7. 优点与亮点

- **推理表示设计有针对性**
  - 不依赖文本 CoT，而是在连续潜在空间中构建 Latent CoT，更适合表达自动驾驶中的空间和动态信息。

- **多专家世界建模清晰**
  - 将世界知识拆解为语义、几何、动态和轨迹四类专家表示，结构明确，符合自动驾驶规划任务的实际需求。

- **有效利用多个基础模型**
  - 使用 V-JEPA 提供语义预测先验；
  - 使用 VGGT 提供几何结构先验；
  - 使用 Wan 视频模型提供动态演化先验；
  - 使用 Qwen3-VL 作为统一 VLM 主干。

- **世界模型不只是辅助监督**
  - 论文强调将专家 token 作为推理时的规划条件，而不是仅在训练中辅助学习，这是相对于许多世界模型方法的重要改进。

- **规划器设计合理**
  - HMEF 通过扩散去噪生成连续轨迹；
  - 同时保留专家分支预测和全局融合权重；
  - 能将多专家表示显式转化为可执行动作。

- **实验覆盖较全面**
  - 同时评估视频生成和轨迹规划；
  - 包含主实验、消融实验、去噪步数分析和定性可视化；
  - 对方法各模块的贡献分析较充分。

- **在较弱传感器输入下表现强**
  - 论文强调其在单帧前视相机输入条件下取得优秀结果，说明模型对世界表示和规划条件的利用较有效。

## 8. 不足与局限

- **训练成本很高**
  - 三阶段训练总计需要大量 A800 GPU：
    - Stage 1：64 张 A800，约 74 小时；
    - Stage 2：32 张 A800，约 70 小时；
    - Stage 3：16 张 A800，约 20 小时。
  - 对学术复现和实际部署都构成较高门槛。

- **模型系统复杂**
  - 同时依赖 Wan、Qwen3-VL、V-JEPA、VGGT 和 HMEF；
  - 多模块、多阶段训练增加了工程复杂度和调参难度。

- **当前输入受限**
  - 论文明确指出当前模型主要使用单图像输入；
  - 尚未扩展到多摄像头、多帧历史、多传感器融合的完整自动驾驶设置。

- **评估场景仍有限**
  - 主要规划 benchmark 是 NAVSIM v1；
  - 缺少在 nuScenes、Waymo、CARLA 闭环仿真或真实车端环境中的系统验证。

- **真实闭环交互能力仍不确定**
  - NAVSIM 是 non-reactive open-loop 协议；
  - 其他交通参与者不会根据自车行为实时反应；
  - 因此无法完全验证模型在真实动态交互场景中的鲁棒性。

- **不同方法对比的公平性存在潜在风险**
  - 对比方法使用的传感器、训练数据、模型规模和强化学习策略不完全一致；
  - 虽然论文标注了 Camera / LiDAR、是否使用 RL 等信息，但仍难做到严格等条件对比。

- **对基础模型依赖较强**
  - 性能可能部分来自 Wan、Qwen3-VL、V-JEPA、VGGT 等强基础模型；
  - 多专家 token 本身的贡献虽有消融支持，但各外部专家模型质量对最终结果的影响仍值得进一步分析。

- **安全性验证仍不足**
  - 虽然 NC、TTC 等指标表现好，但缺少极端长尾场景、恶劣天气、传感器噪声、遮挡、罕见交通行为下的深入验证。
