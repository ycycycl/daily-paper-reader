## 摘要
自动驾驶已从模块化流水线逐步发展到端到端统一，而视觉-语言-动作（Vision-Language-Action, VLA）模型是这一进程在视觉到动作（Vision-to-Action, VA）之外的自然延伸。在实践中，驾驶 VLA 在规划质量上往往落后于 VA，这表明困难并不只是模型规模，而在于将语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的是 VLA 以孤立子任务改进的方式构建、未能组合成连贯驾驶能力的问题，而非 VLA 本身的问题。我们提出 MindVLA-U1，这是首个面向自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干在一次前向传播中基于同一共享表示生成自回归语言 token 和基于流匹配的连续动作轨迹，同时保留各模态的自然输出形式。流式设计以逐帧方式处理驾驶视频，而不是固定的视频-动作片段；同时，学习得到的记忆通道在帧间传递时间上下文，使规划轨迹能够平滑演化，而无需冗余的多帧 VLM 建模。该统一架构通过灵活的自注意力上下文管理，支持在稠密/稀疏混合 Transformer（Mixture-of-Transformers, MoT）骨干上的快/慢执行，并提供了一条可度量的语言到动作路径：由语言预测的驾驶意图通过无分类器引导（classifier-free guidance, CFG）来引导动作扩散，将语言侧意图转化为连续轨迹生成的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越经验丰富的人类驾驶员（RFS 为 8.20，相比 GT RFS 的 8.13），仅需 2 个扩散步骤；相较既有 VA/VLA 方法，在规划 ADE 上以大幅优势达到当前最优，并在保留自然语言接口的同时实现与 VA 同级的吞吐量（16 FPS，相比 RAP-DINO 的 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose into coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces autoregressive language tokens and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A streaming design processes the driving video framewise rather than as fixed video-action chunks, while a learned memory channel carries temporal context across frames so planned trajectories evolve smoothly without redundant multi-frame VLM modeling. The unified architecture admits fast/slow execution on dense/sparse Mixture-of-Transformers (MoT) backbones via flexible self-attention context management, and exposes a measurable language-to-action route: a language-predicted driving intent steers action diffusion through classifier-free guidance (CFG), turning language-side intent into a control signal for continuous trajectory generation. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA methods by large margins, and matches VA-class throughput (16 FPS vs. RAP-DINO's 18 FPS) while preserving natural-language interfaces.

## 2. 方法论：核心思想、关键技术细节与算法流程

## 2.1 总体框架：统一流式 VLA 架构

论文提出 **MindVLA-U1**，核心思想是：

- 使用一个统一的 VLM backbone 同时处理：
  - 多视角图像视觉 token；
  - ego-state 历史状态；
  - 语言 query / answer token；
  - 流式 memory token；
  - noisy action token。
- 在**同一次前向传播**中输出：
  - 自回归语言结果；
  - 基于 flow matching 的连续动作轨迹。
- 保留不同模态的自然输出形式：
  - 语言仍然是 autoregressive token；
  - 动作仍然是连续轨迹，而不是离散 token。

## 2.2 统一共享骨干：AR 语言 + Flow Matching 动作

对于单帧输入，模型接收：

- 多视角图像 token：\(I\)
- ego-state 历史：\(e\)
- 语言问题：\(q\)
- 语言答案 token：\(a = (a_1, ..., a_L)\)
- 带噪动作 token：\(x_t = (x_{t,1}, ..., x_{t,N})\)

共享 backbone \(f_\theta\) 同时处理这些 token。

### 语言损失

语言部分使用标准自回归交叉熵：

\[
L_{AR} = - \sum_{l=1}^{L} \log p_\theta(a_l | I, e, q, a_{<l})
\]

含义是：模型根据视觉、ego 状态和问题，逐 token 生成场景问答或驾驶解释。

### 动作损失

动作部分使用 flow matching / diffusion-style 连续轨迹生成。

给定真实轨迹 \(x_0 = w\)，噪声 \(\epsilon \sim \mathcal{N}(0, I)\)，构造：

\[
x_t = t\epsilon + (1-t)x_0
\]

其中 \(t=0\) 表示真实数据，\(t=1\) 表示纯噪声。

动作头预测速度场：

\[
L_{FM} = ||v_\theta(x_t, t; I,e,q,a) - (\epsilon - x_0)||^2
\]

总训练目标为：

\[
L = L_{AR} + L_{FM}
\]

推理时，通过 ODE / Euler integration 从噪声逐步还原未来轨迹。

## 2.3 语言到动作桥接：Intent-CFG

论文提出 **Intent-CFG**，用于建立可度量的语言到动作控制路径。

流程如下：

1. VLM 语言头先预测当前驾驶意图 token，例如：
   - left；
   - right；
   - go straight；
   - waiting；
   - yielding 等。
2. 将预测的 intent token embedding 加入 action MLP 的时间嵌入中。
3. 训练时随机将 intent 替换为空条件 token \(\emptyset\)，形成 classifier-free guidance 训练。
4. 推理时分别计算：
   - 条件速度场 \(v_\theta(x_t,t;z)\)
   - 无条件速度场 \(v_\theta(x_t,t;\emptyset)\)
5. 使用 CFG 混合：

\[
v_{cfg}(x_t,t) = v_\theta(x_t,t;\emptyset) + s \left(v_\theta(x_t,t;z) - v_\theta(x_t,t;\emptyset)\right)
\]

其中 \(s\) 是 guidance scale，文中使用 \(s=1.5\)。

这意味着语言侧预测出的“驾驶意图”可以直接引导连续轨迹扩散，而不是仅仅停留在文本解释层面。

## 2.4 流式时序建模：Streaming Memory

MindVLA-U1 不采用固定视频 chunk 输入，而是逐帧处理驾驶视频。

每一帧处理流程为：

1. 读取当前多视角图像；
2. 从 FIFO memory channel 中读取历史 memory token；
3. 对历史 memory 根据 ego pose 做 SE(2) 运动对齐；
4. 将 memory token 与当前视觉、语言、ego-state、action token 一起输入 backbone；
5. 生成当前语言和轨迹；
6. 使用 Q-Former-style propagation transformer 从 backbone 输出中压缩出新的 memory token；
7. 将新的 memory token 写入 FIFO，移除最旧 memory。

其特点是：

- 每帧只处理当前图像，避免重复输入多帧视频；
- memory 是可学习的压缩时序状态；
- memory 更新与语言和动作损失端到端联合训练；
- 能在长序列中保持轨迹平滑和上下文连续。

## 2.5 快慢系统与 MoT 设计

MindVLA-U1 支持多种推理模式：

- **vqa_first**：先生成语言解释 / 场景问答，再生成动作；
- **action_first**：先生成轨迹，再生成语言解释；
- **action_only**：只生成动作，不解码语言，用于快速控制；
- **vqa_only**：只做语言理解或问答。

这些模式通过 attention mask 和 token 顺序管理实现，不需要额外模型。

此外，论文提出 sparse **Mixture-of-Transformers（MoT）** 变体：

- context group：视觉、语言 token；
- action group：memory、ego-state、action token；
- 两组共享 self-attention K/V pool；
- FFN 使用不同专家处理不同功能。

这样可以在同一模型中实现：

- 慢速语义推理；
- 快速动作控制；
- 未来可扩展到 safety、world model、reasoning 等专家。

- Qwen3-VL-2B dense 慢路径 vqa_first_decoding：
  - 约 2,594 ms；
  - 约 0.39 FPS。
- Qwen3-VL-2B action_only 快路径：
  - 约 103 ms；
  - 约 9.70 FPS。
- Qwen3-VL-2B vqa_first_fast，template QA，1-step：
  - 约 63 ms；
  - 约 15.92 FPS。
- InternVL-2 1B 快路径：
  - 约 64 ms；
  - 约 15.55 FPS。
- 对比 RAP-DINO：
  - 约 57 ms；
  - 约 17.68 FPS。

需要注意的是：

- 慢路径主要瓶颈来自自回归 VQA decoding，约占 94% 延迟；
- 快路径通过移除语言 answer token 实现接近 VA 的吞吐。

---

## 5. 实验数量与充分性

论文实验较为丰富，主要包括以下几类：

## 5.1 主结果实验

- 在 WOD-E2E validation split 上比较 VA、VLA、人类参考和 MindVLA-U1；
- 在 WOD-E2E official test split 上比较多种 VA/VLA 方法；
- 验证 MindVLA-U1 在 RFS 和 ADE 上的优势。

## 5.2 Intent-CFG 消融

比较：

- 无 intent baseline；
- trajectory-derived intent；
- GT-supplied intent；
- NTP-predicted Intent-CFG。

结果表明：

- 简单轨迹派生或 GT 3 类 intent 提升有限；
- NTP-predicted Intent-CFG 提升明显，RFS 从 7.83 到 7.92。

## 5.3 MoT 消融

比较：

- dense backbone；
- MoT：{V,L,M,S}+{A}；
- MoT：{V,L}+{M,S,A}。

用于验证不同专家分组对规划质量和快路径结构的影响。

## 5.4 Streaming Memory 消融

比较：

- chunk-wise single frame；
- chunk-wise + 4-frame image sequence；
- streaming training without memory；
- streaming + memory。

结果显示：

- 简单多帧 VLM 输入反而降低 RFS；
- streaming training 有小幅提升；
- streaming memory 进一步提升 RFS 和长时序 ADE。

## 5.5 RL post-training 实验

- 使用 GRPO，以 RFS 为唯一 reward；
- 从 SFT checkpoint 继续优化；
- 验证 RFS 从 7.83 提升到 8.20。

## 5.6 Backbone scaling 实验

比较 Qwen3.5-VL 不同规模：

- 0.8B；
- 2B；
- 4B；
- 9B；
- 9B 200 epochs；
- base vs instruction-tuned。

发现：

- RFS 不随参数规模单调增长；
- 语言质量下降并不必然导致动作质量下降；
- action quality 与 surface language quality 存在结构性解耦。

## 5.7 Val/Test 分布诊断

使用 MindLabel 分析验证集和测试集 intent 分布差异：

- validation 中 accelerating、starting 占比较高；
- test 中 waiting 显著增多；
- 解释了 val/test RFS gap 的一部分来源。

### 实验充分性评价

总体而言，实验较充分：

- 包含主 benchmark、官方 test、消融、速度评测、scaling、RL、分布诊断；
- 对比了强 VA 和 VLA baseline；
- 不仅报告精度，也报告吞吐和语言指标；
- 对自身方法的局限也有较清晰讨论。

但仍有不足：

- 所有 headline 结果主要集中在 WOD-E2E；
- 缺少 nuScenes、NAVSIM 或真实车端闭环验证；
- RL 后超过人类 RFS 是 open-loop 指标，不等价于真实道路闭环优于人类；
- 部分对比方法是否完全同等训练数据和预训练条件并不完全一致。

---

## 6. 主要结论与发现

- **MindVLA-U1 显著缩小并反超了 VLA 与 VA 的规划差距。**
  - 在 WOD-E2E test 上，MindVLA-U1 + RL 达到最高 RFS 7.87；
  - 在 validation 上，MindVLA-U1 + RL 达到 8.20 RFS，超过人类 logged trajectory 的 8.13。
- **统一共享 backbone 可以同时保留语言能力和连续动作精度。**
  - 与离散 action token VLA 不同，MindVLA-U1 没有轨迹量化精度瓶颈。
- **语言到动作路径是可测量的。**
  - Intent-CFG 让语言预测出的驾驶意图直接影响动作扩散。
- **流式 memory 比多帧 VLM 输入更适合驾驶时序建模。**
  - 多帧 DeepStack 方式没有提升，反而降低 RFS；
  - streaming memory 提升 RFS 和长时序 ADE。
- **快慢路径可以共存在同一模型中。**
  - 慢路径保留语言推理；
  - 快路径接近 VA 吞吐。
- **动作质量与语言表面生成质量并不完全绑定。**
  - base VLM 的 BLEU/ROUGE 明显下降，但 RFS 基本保持；
  - 说明 action 主要依赖共享场景表征和 intent embedding，而不是语言流畅度本身。
- **模型规模并非当前瓶颈的唯一因素。**
  - 参数规模增大并没有带来单调提升；
  - 可能受训练数据、训练步数、action head 容量、评价指标等限制。

---

## 7. 优点与亮点

## 7.1 架构设计完整

- 同时解决了 VLA 自动驾驶中的多个关键接口问题：
  - 连续动作；
  - 语言保留；
  - 时序记忆；
  - 语言到动作控制；
  - 快慢执行；
  - 稀疏 MoT 扩展。

## 7.2 动作生成方式合理

- 使用 flow matching 生成连续轨迹，避免离散 token 坐标精度损失；
- 动作 token 进入 backbone self-attention，而不是在外部 action expert 中单独处理。

## 7.3 流式设计符合驾驶任务特性

- 驾驶是天然连续流，而非独立 clip；
- FIFO memory 和 SE(2) motion alignment 更贴合车辆运动和长时序上下文。

## 7.4 提供了真正的语言到动作机制

- Intent-CFG 是本文最重要亮点之一；
- 它将语言预测结果转化为扩散轨迹条件，而不是仅用语言作为装饰性输出。

## 7.5 实验维度丰富

- 包含 accuracy、RFS、ADE、语言质量、吞吐、消融、scaling、RL、分布诊断；
- 对 VA/VLA 的对比相对全面。

## 7.6 工程可部署性较强

- 明确报告 H200 上的 latency / FPS；
- 快路径在 1B 规模下接近 RAP-DINO 的 VA 吞吐；
- 说明架构不仅追求离线指标，也考虑实时控制约束。

---

## 8. 不足与局限

## 8.1 只在 open-loop benchmark 上验证

- WOD-E2E 是 logged dataset；
- RFS 和 ADE 衡量的是开放环轨迹质量；
- 模型并未在真实闭环环境中验证：
  - 无法观察车辆执行预测轨迹后环境如何反应；
  - 无法评估 compounding error；
  - 无法证明真实驾驶安全性超过人类。

## 8.2 数据集覆盖有限

- 主要实验集中在 WOD-E2E；
- 缺少跨数据集验证，例如：
  - nuScenes；
  - NAVSIM；
  - CARLA；
  - 真实车端场景。
- 因此方法泛化性仍需进一步确认。

## 8.3 Val/Test 存在明显分布偏移

- 论文自己指出：
  - validation 中 accelerating、starting 更多；
  - test 中 waiting 更多。
- 这说明 WOD-E2E split 本身存在 intent distribution shift；
- 虽然作者进行了诊断，但也意味着单一 benchmark 结果可能受到数据分布影响。

## 8.4 “超过人类驾驶”需要谨慎解读

- 8.20 vs 8.13 是在 RFS open-loop 指标上超过 logged GT；
- logged GT 只是一个人类实际轨迹，并不一定是最优轨迹；
- RFS 更接近 rater preference，而非真实闭环安全驾驶能力；
- 因此不能直接解释为系统真实道路驾驶能力全面超过人类。

## 8.5 MindLabel 的高质量与偏差风险

- MindLabel 大量依赖自动标注模型 Qwen3-VL / Qwen3.5-Plus；
- 可能引入：
  - VLM 标注偏差；
  - 语言描述幻觉；
  - intent 分类误差；
  - dreamed trajectory 的质量不稳定。
- 虽然主实验只使用基础 VQA 和 3 类 GT intent，但未来若使用完整 MindLabel，需要更严格的数据质量验证。

## 8.6 VLM scaling 结论尚不充分

- 论文发现参数规模不带来单调提升；
- 但作者也承认这不能构成 VLA scaling law 结论；
- 因为存在多个混杂因素：
  - 大模型可能 undertrained；
  - WOD-E2E 数据规模有限；
  - action head 容量固定；
  - open-loop RFS 可能饱和。

## 8.7 慢路径延迟仍然较高

- 完整 vqa_first_decoding 慢路径约 2.6 秒一帧，只有 0.39 FPS；
- 这对实时驾驶控制不可接受；
- 虽然快路径可达到 10–16 FPS，但语言推理频率如何与动作控制频率协调，仍需要进一步工程化。

## 8.8 复杂意图与多模态行为仍未充分利用

- 主实验中 intent 仍主要使用 WOD-E2E 的 3 类 intent；
- MindLabel 的 20 类 intent、dreamed trajectories、CoT rationale 尚未用于主模型；
- 对复杂场景如 emergency stop、yield、parking、u-turn 的泛化仍有限。

---

# 总体评价

MindVLA-U1 是一篇系统性较强的自动驾驶 VLA 架构论文。它的核心贡献不是单一模块，而是重新设计了自动驾驶 VLA 的整体接口：用统一 backbone 连接视觉、语言、记忆和连续动作，用 streaming memory 解决时序连续性，用 Intent-CFG 建立语言到动作的可测量控制路径，并用快慢执行和 MoT 兼顾推理能力与实时性。

实验结果显示，该方法在 WOD-E2E 上取得了当前非常强的 open-loop 规划表现，并在 RFS 上超过人类 logged trajectory 参考。不过，这一结论仍应限定在 WOD-E2E open-loop 评价框架内，尚不能直接代表真实闭环自动驾驶能力。总体上，论文对自动驾驶 VLA 的架构设计具有较高参考价值，也为后续结合世界模型、闭环 RL、复杂 intent、多模态轨迹规划提供了清晰路线。
