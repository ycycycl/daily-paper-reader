## 摘要
自回归视频扩散模型已经实现了实时的、动作条件化的世界生成。然而，维持一个持久世界——即重新访问先前见过的视角时能够产生一致的内容——仍然是一个未解决的问题。完整的 KV-cache 注意力机制可以保持这种一致性，但会破坏实时性约束：内存占用和注意力计算成本会随 rollout 长度线性增长。滑动窗口推理恢复了吞吐量，但牺牲了长期一致性。我们提出 WorldKV，这是一个无需训练的框架，包含两个组件：世界检索（World Retrieval）和世界压缩（World Compression）。世界检索将被逐出的 KV-cache 块存储在 GPU/CPU 内存中，并通过相机/动作对应关系选择性地检索与场景相关的块，将其无需重新编码地插回原生注意力窗口。世界压缩通过键-键相似性相对于锚帧剪枝每个块中的冗余 token，使每块存储量减半，从而在固定预算下容纳 2 倍以上的历史信息。在 Matrix-Game-2.0 和 LingBot-World-Fast 上，WorldKV 以约 2 倍吞吐量达到或超过完整 KV 的记忆保真度，并且在无需任何微调的情况下可与经过记忆训练的基线方法相竞争。项目页面：https://cvlab-kaist.github.io/WorldKV/

## Abstract
Autoregressive video diffusion models have enabled real-time, action-conditioned world generation. However, sustaining a persistent world, where revisiting a previously seen viewpoint yields consistent content, remains an open problem. Full KV-cache attention preserves this consistency but breaks real-time constraints: memory footprint and attention cost grow linearly with rollout length. Sliding window inference restores throughput but discards long-term consistency. We propose WorldKV, a training-free framework with two components: World Retrieval and World Compression. World Retrieval stores evicted KV-cache chunks in GPU/CPU memory and selectively retrieves scene-relevant chunks via camera/ action correspondence, inserting them back into the native attention window without re-encoding. World Compression prunes redundant tokens within each chunk via key-key similarity to an anchor frame, halving per-chunk storage to fit 2x more history under a fixed budget. On Matrix-Game-2.0 and LingBot- World-Fast, WorldKV matches or exceeds full-KV memory fidelity at roughly 2x the throughput, and is competitive with memory-trained baselines without any fine-tuning. ### 1.1 研究背景

本文关注的是**实时交互式视频世界模型**中的长期一致性问题。近年来，自回归视频扩散模型结合因果注意力和 KV-cache，已经能够根据用户动作或相机控制实时生成视频流，例如在游戏、机器人仿真、具身智能和虚拟环境中进行交互式世界生成。

这类模型的目标不只是生成局部上合理的连续帧，而是进一步构建一个**持久世界**：用户离开某个场景后再回到同一视角时，模型应该生成与第一次看到时一致的内容。例如，房间布局、物体位置、纹理外观等应当保持稳定。

### 1.2 核心问题

论文指出，现有推理方式存在明显矛盾：

- **Full KV-cache attention**
  - 让模型 attend 到完整历史 KV-cache。
  - 可以保留长期视觉记忆，重访时更一致。
  - 但显存和注意力计算量随 rollout 长度线性增长。
  - 长序列下容易导致显存爆炸，吞吐下降，难以保持实时性。

- **Sliding-window inference**
  - 只保留最近一小段 KV-cache。
  - 推理速度稳定，适合实时生成。
  - 但一旦早期场景对应的 KV-cache 被移出窗口，模型在重访时容易“失忆”，出现内容漂移或重新幻觉生成。

因此，论文要解决的问题是：

> 如何在不重新训练模型、不修改模型结构的情况下，让自回归视频世界模型既能保持实时推理速度，又能拥有长期场景记忆？

### 1.3 整体意义

本文的关键观察是：  
**KV-cache 本身已经隐含了世界记忆。**

作者发现，即便 Matrix-Game-2.0 这种只在短序列上训练、原生使用滑动窗口推理的模型，在推理时如果允许它 attend 到完整历史 KV-cache，也能够在一定程度上恢复过去见过的视角。这说明长期记忆并不一定完全来自专门训练的外部记忆模块，而可能已经潜藏在模型原生 KV-cache 中。

它把 KV-cache 从单纯的加速缓存，重新解释为一种可以被检索、压缩和复用的**世界记忆载体**。

## 2. 方法论：核心思想、关键技术与流程

WorldKV 是一个**training-free** 的推理时框架，不需要微调、不需要额外训练记忆模块，也不需要重新编码历史帧。它主要由两个组件构成：

1. **World Retrieval：世界检索**
2. **World Compression：世界压缩**

---

### 2.1 总体框架

WorldKV 建立在滑动窗口推理之上。它不保留完整 KV-cache 参与每一步注意力计算，而是：

- 将被滑动窗口逐出的历史 KV-cache chunk 存入 GPU/CPU 内存；
- 当模型当前视角接近某些历史视角时，选择相关历史 chunk；
- 将这些 chunk 插回当前原生注意力窗口；
- 对历史 chunk 内部进行 token 压缩，减少存储与注意力成本。

作者将当前注意力窗口划分为四个区域：

1. **Sink KV caches**
   - 来自初始帧，作为视觉锚点。
2. **Retrieved KV caches**
   - 从历史记忆中检索出的相关 chunk。
3. **Recent KV caches**
   - 最近生成的若干帧。
4. **Denoising chunk**
   - 当前正在生成或去噪的 chunk。

这种设计使模型既能看到最近上下文，也能访问与当前视角相关的长期记忆。

---

### 2.2 World Retrieval：基于相机 / 动作对应的历史 KV 检索

#### 核心动机

作者通过注意力图分析发现，模型在重访某些视角时，并不是只关注最近 KV-cache，而是会对与当前视角重叠的历史 chunk 分配更高注意力。

例如在动作序列：

> Right → Stop → Left → Stop → Right

中，当相机转回早期视角时，模型注意力会集中到最初对应视角的 KV-cache；当再次转向右侧时，注意力又会转移到之前相似视角的历史 chunk。

这说明：  
**视角相关性是选择历史 KV-cache 的有效信号。**

#### 检索机制

在推理过程中，每个被逐出的 KV chunk 都会被保存，并附带其生成时的相机或动作状态：

- 对于有明确相机轨迹的模型，保存相机平移和旋转；
- 对于键鼠动作模型，累计 WASD 和 yaw/pitch 输入，构造伪位移和伪旋转。

在当前时刻，给定当前相机 / 动作状态，WorldKV 从历史 chunk 中检索 top-k 个最相关的 chunk。

论文中用如下形式表示：

\[
R = Top-k(sim(a_{cur}, a_i) \mid i = 1, ..., M)
\]

其中：

- \(a_{cur}\)：当前相机或动作状态；
- \(a_i\)：第 \(i\) 个历史 chunk 的状态；
- \(M\)：存储的历史 chunk 数量；
- \(k\)：检索预算；
- \(sim(\cdot,\cdot)\)：相关性函数。

#### 相似度实现

主实验中采用相机 / 动作距离作为检索依据：

- 平移部分：使用平方 L2 距离；
- 旋转部分：使用 geodesic distance；
- 两者分别归一化后相加；
- 选取距离最小的历史 chunk。

作者还在附录中测试了 query-based retrieval，即根据当前 denoising query 对历史 KV 的注意力分数排序。结果显示 query-based retrieval 也优于滑动窗口，但相机 / 动作检索效果更好。

### 2.3 World Compression：基于 Key-Key 相似度的 KV 压缩

#### 核心动机

World Retrieval 需要保存大量被逐出的 KV-cache，但视频世界模型每个 chunk 的 KV-cache 体积非常大。

论文提到，在 LingBot-World-Fast 中，一个 3 个 latent frame 的 chunk 跨所有 Transformer 层大约占用 **3.4GB**。一分钟 rollout 会累积超过 **200GB** 的 KV-cache，甚至超过单张 B200 GPU 显存容量。

作者观察到，相邻帧之间通常存在大量冗余：

- 相机视角变化小；
- 场景布局相近；
- 物体外观相似；
- KV-cache 中包含许多近似重复的信息。

因此，需要对每个 chunk 内部进行压缩。

#### 压缩思想

World Compression 将每个 chunk 的第一帧作为 **anchor frame**，完整保留 anchor frame 的 token。对于同一 chunk 中后续帧的 token，则计算其 key 与 anchor frame keys 的相似度。

如果某个 token 的 key 与 anchor keys 非常相似，说明它大概率编码了 anchor 中已有的信息，可以视为冗余；  
如果相似度较低，则说明它可能对应新出现的区域、动态物体或视角变化带来的新增内容，应当保留。

#### 相似度计算

给定 anchor frame 的 key：

\[
K^{(a)} \in R^{T \times d}
\]

对于非 anchor 帧 \(f\) 中的某个 key \(k_j^{(f)}\)，计算它与所有 anchor keys 的平均余弦相似度：

\[
s_j^{(f)} = \frac{1}{T} \sum_{i=1}^{T}
\frac{k_j^{(f)\top} k_i^{(a)}}{\|k_j^{(f)}\| \cdot \|k_i^{(a)}\|}
\]

然后：

- 汇总所有非 anchor 帧 token 的相似度；
- 保留相似度最低的一部分 token；
- 剪掉与 anchor 高度相似的冗余 token。

#### 压缩比例

主实验中每个 chunk 包含 3 帧：

- 第 1 帧作为 anchor，完整保留；
- 第 2、3 帧只保留相似度最低的 25% token；
- 原本 3T 个 token 被压缩到约 1.5T 个 token；
- 即每个 chunk 约压缩为原来的一半。

这种设计使固定预算下可容纳约 2 倍历史 chunk。

#### 按层独立压缩

压缩在每个 Transformer 层独立进行，因为不同层中 token 的重要性可能不同。每一层都有自己的保留 token 集合，检索时也按层使用对应的压缩 KV-cache。

- **Sliding Window**
  - 只保留最近窗口；
  - 高效但容易失忆。

- **Full KV**
  - attend 到完整历史 KV-cache；
  - 记忆强但代价高。

#### 记忆训练型方法

- **WorldPlay**
  - 8B 参数；
  - 带有长期几何一致性和记忆机制；
  - 需要训练或蒸馏。

- **Yume-1.5**
  - 5B 参数；
  - 文本控制交互世界生成模型；
  - 带有显式长期交互能力。

#### WorldKV

- 不训练；
- 不修改模型结构；
- 在推理时对 KV-cache 做检索和压缩。

---

### 3.4 评价指标

论文使用重访帧与首次访问帧之间的相似性作为记忆保真度指标，包括：

- **LPIPS ↓**
  - 感知距离，越低越好；
- **PSNR ↑**
  - 像素级重建质量，越高越好；
- **SSIM ↑**
  - 结构相似性，越高越好；
- **FID ↓**
  - 分布距离，越低越好；
- **Throughput / FPS ↑**
  - 推理吞吐量，越高越好。

吞吐量在每个 rollout 最后一个 chunk 处测量，因为此时 Full KV 的历史最长、负担最大。

## 4. 资源与算力

### 4.1 推理硬件

论文明确报告了部分推理硬件：

- 主表默认吞吐量在 **4 × H200 GPU** 上测量；
- 括号中的 LingBot-World-Fast 吞吐量在 **4 × B200 GPU** 上测量；
- Figure 2 中提到：
  - LingBot-World-Fast 14B 在 **4 × B200** 上测试；
  - Matrix-Game-2.0 1.3B 在 **4 × H200** 上测试。

### 4.2 训练资源

WorldKV 是 training-free 方法，因此论文没有进行 WorldKV 自身训练。

文中没有报告：

- 额外训练时长；
- 训练 GPU 数量；
- 训练数据规模；
- 微调成本。

这也符合其核心设定：WorldKV 作为推理时 KV 管理方法，不需要训练或蒸馏。

### 4.3 显存与存储成本

论文给出了一些关键资源观察：

- LingBot-World-Fast 中，一个 3 latent frames chunk 的 KV-cache 约为 **3.4GB**；
- 一分钟 rollout 可累积超过 **200GB** KV-cache；
- Full KV 显存随时间线性增长，容易进入 OOM 区域；
- WorldKV 通过压缩使每个 chunk 存储约减半；
- CPU offloading 可以让 VRAM 消耗接近稳定，但当前 CPU-GPU 传输延迟仍影响实时性。

## 6. 主要结论与发现

### 6.1 KV-cache 具有涌现的世界记忆能力

即使模型没有专门接受长上下文训练，例如 Matrix-Game-2.0，只要在推理时允许访问完整历史 KV-cache，也能利用过去 KV-cache 恢复部分历史场景。这说明 KV-cache 中已经包含可复用的长期视觉记忆。

### 6.2 Full KV 有记忆但代价过高

Full KV-cache attention 能提升重访一致性，但：

- 显存线性增长；
- 注意力成本线性增长；
- 长 rollout 中 FPS 持续下降；
- 容易 OOM；
- 难以满足实时交互需求。

例如 LingBot-World-Fast 中，Full KV FPS 从约 8.87 降至 3.61，明显影响实时性。

### 6.3 WorldKV 接近或超过 Full KV，且速度约为其 2 倍

在 LingBot-World-Fast 上：

- Full KV：
  - FPS：2.36，B200 上 3.83；
  - LPIPS：0.441；
  - PSNR：15.901；
  - SSIM：0.472；
  - FID：85.705。

- WorldKV：
  - FPS：4.78，B200 上 7.71；
  - LPIPS：0.455；
  - PSNR：15.660；
  - SSIM：0.463；
  - FID：75.644。

WorldKV 以接近 2 倍速度达到接近 Full KV 的记忆质量，且 FID 更好。

在 Matrix-Game-2.0 上：

- Sliding Window：
  - FPS：18.87；
  - LPIPS：0.594；
  - PSNR：11.422；
  - SSIM：0.280；
  - FID：157.261。

- Full KV：
  - FPS：7.82；
  - LPIPS：0.529；
  - PSNR：13.748；
  - SSIM：0.364；
  - FID：124.912。

- WorldKV：
  - FPS：16.25；
  - LPIPS：0.462；
  - PSNR：14.101；
  - SSIM：0.405；
  - FID：93.561。

在该模型上，WorldKV 不仅比滑动窗口好，也全面超过 Full KV。

### 6.4 选择性检索有时优于完整历史注意力

作者发现 WorldKV 在某些情况下优于 Full KV。原因可能是：

- Full KV 中包含大量无关或冗余历史；
- 长上下文会稀释注意力；
- 模型可能难以从超长 KV 中准确利用相关内容；
- WorldKV 只保留与当前视角相关的 chunk，反而提供更干净的上下文。

### 6.5 中等压缩比例效果最好

压缩实验显示：

- 只保留 anchor frame，即 3 → 1.0，性能明显下降；
- 3 → 1.5 或 3 → 2.0 通常效果较好；
- 完全不压缩并不一定最优；
- 过度压缩，例如 9 → 3，会损失非 anchor 帧中的独特信息。

这说明 World Compression 不是单纯节省存储，也可能通过减少冗余改善注意力质量。

## 7. 优点与亮点

### 7.1 无需训练，部署成本低

WorldKV 最大亮点是 **training-free**：

- 不需要重新训练世界模型；
- 不需要额外记忆模块；
- 不需要蒸馏；
- 不需要改变网络结构；
- 可以直接应用于已有 KV-cache-based 自回归视频世界模型。

这使其工程可用性较强。

### 7.2 充分利用模型原生 KV-cache

论文没有引入复杂外部 memory bank 或 3D 重建模块，而是直接利用模型已经计算出的 KV-cache。这种思路简洁有效，将原有推理缓存转化为长期世界记忆。

### 7.3 检索依据符合交互世界模型特性

相机 / 动作状态与视角重访高度相关。World Retrieval 利用这一点选择历史 chunk，比纯最近窗口更适合交互式世界生成。

### 7.4 压缩方法简单且解释性强

Key-Key 相似度压缩的逻辑清楚：

- 高相似 token 多为冗余；
- 低相似 token 对应新出现区域或动态变化；
- 保留低相似 token 可以保存关键视觉差异。

附录可视化也显示，被保留 token 往往位于新露出的边界区域或动态物体上。

### 7.5 同时改善显存、速度与记忆一致性

WorldKV 不是单一优化：

- 降低存储；
- 降低注意力窗口长度；
- 保持实时 FPS；
- 提升重访一致性；
- 在部分情况下超过 Full KV。

### 7.6 实验覆盖多个维度

论文实验不仅有主结果，还包括：

- 压缩比例消融；
- 检索策略消融；
- 检索 chunk 数量分析；
- 多模型泛化；
- 定性可视化；
- 与 memory-trained 方法比较。

整体实验设计较完整。

---

## 8. 不足与局限

### 8.1 仍受限于基础世界模型质量

WorldKV 只管理 KV-cache，不提升基础生成模型本身的能力。如果底层模型生成质量较差，或长时间 rollout 中错误逐渐积累，WorldKV 无法从根本上修复这些视觉伪影。

### 8.2 超长时间生成仍可能存在误差累积

论文主要展示几十秒到一分钟级别的长时域生成。对于多分钟甚至更长时间的开放式探索，模型仍可能出现：

- 视觉漂移；
- 场景结构错误；
- 动态物体状态不一致；
- 自回归误差累积。

### 8.3 CPU offloading 尚不能实时

作者指出，将所有 KV-cache 存在 CPU 中、只把当前需要的 chunk 加载到 GPU，可以控制 VRAM 使用。但当前 CPU-GPU 传输延迟仍然阻碍实时生成。

因此，WorldKV 在极长序列下的可扩展性还依赖后续系统优化。

### 8.4 benchmark 规模和标准化程度有限

虽然作者构建了 60 个 scene-trajectory pairs，但这仍是自建 benchmark，可能存在：

- 场景分布有限；
- 轨迹设计偏向方法优势；
- 与真实复杂交互环境存在差距；
- 缺少社区标准化对比。

### 8.5 与记忆训练基线的比较不完全公平

WorldPlay、Yume-1.5、LingBot-World-Fast 和 Matrix-Game-2.0 在以下方面可能不同：

- 模型规模；
- 训练数据；
- 控制方式；
- 输入条件；
- 推理设置；
- 长视频训练程度。

因此，与 memory-trained baseline 的直接数值比较有参考意义，但不应被理解为完全公平的 apples-to-apples 对比。

### 8.6 主要评估视觉相似性，缺少更高层任务指标

论文主要使用 LPIPS、PSNR、SSIM、FID 衡量重访帧一致性，但没有充分评估：

- 几何一致性；
- 三维可导航性；
- 物体持久身份；
- 物理一致性；
- 任务完成率；
- agent 在该世界中的长期规划表现。

对于真实世界模拟和机器人训练，这些指标也很重要。

### 8.7 检索依赖相机 / 动作轨迹质量

World Retrieval 的主方法依赖相机或动作状态。如果：

- 相机姿态不可得；
- 动作与真实位姿不匹配；
- 累积伪位姿漂移；
- 存在大幅动态场景变化；

检索准确性可能下降。

虽然论文展示了 query-based retrieval 的替代可能，但主结果仍主要依赖 camera/action-based retrieval。

---

## 总体评价

WorldKV 提出了一种简单但有效的推理时世界记忆机制：通过检索历史 KV-cache 中与当前视角相关的 chunk，并使用 Key-Key 相似度压缩冗余 token，在不训练、不改模型的前提下，使自回归视频世界模型获得接近甚至优于 Full KV 的长期重访一致性，同时保持接近滑动窗口的实时吞吐。

这篇论文的核心价值在于，它证明了 KV-cache 不只是加速缓存，也可以被视为一种原生世界记忆；而合理的选择性检索和压缩，可能比盲目保留完整历史更高效、更稳定。其方法尤其适合已有交互式世界模型的低成本增强，但在超长时域、复杂动态环境、标准化评估和系统级 offloading 延迟方面仍有进一步研究空间。
