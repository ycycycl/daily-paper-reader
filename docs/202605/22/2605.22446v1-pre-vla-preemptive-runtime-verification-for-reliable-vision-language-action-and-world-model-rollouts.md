## 摘要
尽管大型视觉-语言-动作（VLA）模型和生成式世界模型（WM）推动了长时程具身智能的发展，但其实际部署仍受到基于学习的动作生成中不确定性的挑战。低质量动作可能在执行过程中导致物理失败，或造成具有误导性的世界模型 rollout，并带来冗余的渲染开销。为解决这一问题，我们提出 Pre-VLA，一种统一的运行时验证架构，可在物理执行或世界模型想象之前对动作有效性进行抢占式评估。Pre-VLA 利用高效的多模态骨干网络、模态感知池化以及轻量级双分支头，为候选动作片段同时预测安全置信度和由 critic 导出的优势分数。为应对严重的类别不平衡和不稳定的边界决策，我们采用多任务目标训练 Pre-VLA，该目标结合了 Focal 分类、优势回归和软阈值校准。在部署阶段，双模式抢占式重采样调度器会过滤低质量动作，并在有限计算预算下触发自适应重采样。在 LIBERO 基准上的实验表明，相较于 RynnVLA-002，Pre-VLA 将四个套件上的平均闭环成功率从 30.79% 提升至 37.62%，减少了任务执行步数，实现了每个动作片段平均 183.9 ms 的前向验证时间，并缓解了世界模型 rollout 中的误差累积。

## Abstract
While large vision-language-action (VLA) models and generative world models (WM) have advanced long-horizon embodied intelligence, their practical deployment remains challenged by uncertainty in learning-based action generation. Low-quality actions may cause physical failures during execution or lead to misleading world-model rollouts with redundant rendering costs. To address this issue, we propose Pre-VLA, a unified runtime verification architecture that performs preemptive action validity assessment before physical execution or world-model imagination. Pre-VLA leverages an efficient multimodal backbone with modality-aware pooling and a lightweight dual-branch head to predict both safety confidence and critic-derived advantage scores for candidate action chunks. To handle severe class imbalance and unstable boundary decisions, we train Pre-VLA with a multi-task objective combining Focal classification, advantage regression, and soft-threshold calibration. During deployment, a dual-mode preemptive resampling scheduler filters low-quality actions and triggers adaptive resampling under a limited computation budget. Experiments on the LIBERO benchmark show that Pre-VLA improves the average closed-loop success rate across four suites from 30.79\% to 37.62\% over RynnVLA-002, reduces task execution steps, achieves 183.9 ms average forward verification time per action chunk, and mitigates error accumulation in world-model rollouts.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 核心思想

Pre-VLA 的基本思想是：

> 在每个规划周期中，VLA 先生成候选动作块；Pre-VLA 对该动作块进行有效性验证；只有通过验证的动作才会被送往物理执行环境或世界模型；若验证失败，则触发有限次数的重采样；若多次重采样仍失败，则使用回归分支选择相对最优的 fallback 动作。

它同时服务于两个下游场景：

1. **真实 / 仿真执行场景**
   - 防止危险动作直接进入机器人控制器；
   - 减少碰撞、抓取失败和后续错误累积。

2. **世界模型 imagination 场景**
   - 防止无效动作进入世界模型；
   - 减少错误视觉预测与无意义 GPU 推演成本。

## 2.2 安全监督信号构造

论文没有依赖人工标注来定义动作安全性，而是从 PPO 强化学习 rollout 中提取弱监督信号。

### 动作块优势估计

对于当前时刻 `t`，系统包含：

- 语言指令 `l`
- 图像观测 `o_t`
- 机器人本体状态 `q_t`
- 候选动作块 `C_t = {a_t^1, ..., a_t^K}`

构成多模态上下文：

```text
x_t = {l; o_t; q_t; C_t}
```

作者使用预训练 PPO Critic 计算 K 步动作块优势：

```text
A_t^K = Σ γ^i r_{t+i} + γ^K H_{t,K} Vψ(x_{t+K}) - Vψ(x_t)
```

其中：

- `γ` 是折扣因子；
- `r_{t+i}` 是环境反馈奖励；
- `Vψ` 是 PPO Critic 价值函数；
- `H_{t,K}` 是安全掩码。

### 碰撞 / 失败掩码

如果动作块执行过程中发生碰撞、物体掉落等不可逆失败，则：

```text
H_{t,K} = 0
```

从而截断后续价值估计，避免失败后的价值信号被错误传播。

### 延迟失败惩罚

论文进一步考虑到，错误动作不一定立即导致失败，可能在若干步之后才引发碰撞或任务失败。因此引入了 failure-aware penalty：

```text
F_{t,K} = I[episode fails] · I[0 ≤ T_fail - (t+K) ≤ W] · exp(-(T_fail - (t+K))/κ)
```

含义是：

- 如果 episode 最终失败；
- 且当前动作块位于失败前的回溯窗口 `W` 内；
- 则对该动作块施加指数衰减惩罚。

修正后的安全优势为：

```text
A_t^K ← A_t^K - λ_fail F_{t,K}
```

### 标准化与标签生成

由于不同任务的奖励尺度不同，作者对每个任务 buffer 内的优势值进行标准化：

```text
Ã_t^K = (A_t^K - μ_task) / (σ_task + ε)
```

然后使用阈值 `τ_A` 生成二分类标签：

```text
y_binary = I(Ã_t^K ≥ τ_A)
y_cont = Ã_t^K
```

其中：

- `y_binary` 用于安全 / 不安全分类；
- `y_cont` 作为连续优势回归目标。

## 2.3 模型架构：轻量双分支运行时监测器

Pre-VLA 使用改进版 WorldVLA 作为统一多模态骨干，其底层基于 Chameleon 7B。

### 输入模态

模型同时处理四类输入：

- 文本任务指令；
- 图像观测；
- 机器人本体状态；
- 候选动作块。

论文通过专门的：

- text tokenizer；
- image tokenizer；
- state tokenizer；
- action tokenizer；

将不同模态统一为 token 序列。

### 冻结骨干网络

训练 Pre-VLA 时，作者冻结多模态 backbone 参数，仅训练验证头。这样做的目的包括：

- 保留原 VLA / WorldVLA 的生成能力；
- 降低训练成本；
- 使 verifier 成为一个轻量外挂模块。

### 模态感知池化

由于 backbone 输出的 hidden states 中不同模态信息高度混合，论文设计了 modality-aware pooling。

对于每个模态 `m ∈ {text, image, state, action}`，使用预定义 mask `M_m` 提取对应 token，并做 masked mean pooling：

```text
z_m = Σ_j M_m^j H_t^j / (Σ_j M_m^j + ε)
```

随后拼接四个模态向量：

```text
z_t = [z_text; z_image; z_state; z_action]
```

如果某个模态缺失，则用零向量补齐。

### 双分支预测头

拼接后的全局特征 `z_t` 输入轻量 FFN，产生两个输出：

#### 分类分支

输出动作块通过验证的概率：

```text
p_t = σ(FFN_cls(z_t))
```

用于判断候选动作是否安全。

#### 回归分支

预测连续优势值：

```text
Â_t^K = hϕ(z_t)
```

用于在多个候选动作都未通过分类阈值时，选择相对最优 fallback 动作。

## 2.4 多任务训练目标

Pre-VLA 训练目标由三部分组成。

### Focal Loss 分类损失

由于安全样本远多于失败样本，数据存在严重类别不平衡。论文用 Focal Loss 替代普通交叉熵：

- 降低大量简单安全样本的权重；
- 强化对稀有高风险动作的学习。

### 优势回归损失

用 MSE 拟合标准化后的连续优势值：

```text
L_reg = 1/2 ||Â_t^K - Ã_t^K||²
```

其作用是为模型提供连续的动力学质量监督，而不仅是二分类信号。

### 软阈值校准损失

由于硬标签在安全阈值附近变化突兀，作者构造软标签：

```text
s_t = σ((Ã_t^K - τ_A) / τ_temp)
```

然后使用 soft binary cross-entropy 训练分类概率。

这一项用于缓解边界附近的决策振荡，提高置信度校准能力。

### 总损失

最终训练目标为：

```text
L = L_cls + λ_soft L_soft + λ_reg L_reg
```

## 2.5 双模式抢占式重采样调度器

部署时，Pre-VLA 被插入到 VLA 之后、物理执行或世界模型之前。

### Warm-up 机制

在前 `T_w` 步中不启用 verifier，直接执行动作：

```text
use_verifier = I[t ≥ T_w]
```

这样可以避免早期探索或准备动作被过早约束。

### 物理执行模式

流程如下：

1. VLA 生成候选动作块；
2. Pre-VLA 分类分支输出通过概率 `p_t`;
3. 若 `p_t ≥ τ_p`，动作通过验证并执行；
4. 若 `p_t < τ_p`，动作被拒绝并触发重采样；
5. 重采样最多进行 `N` 次；
6. 若所有候选动作都未通过，则调用回归分支，选择预测优势最高的动作执行：

```text
a_t^{K,*} = argmax_i Â_{t,i}^K
```

### 世界模型模式

流程类似，但目标是节省推演成本：

1. 候选动作先经过 Pre-VLA；
2. 若被判定为低质量，则不送入世界模型；
3. 触发 VLA 重采样；
4. 达到最大尝试次数后，选择回归分数最高的动作进入 WM；
5. 若该动作仍被判断为高风险，则可截断当前 imagination rollout。

- VLA 动作生成分支；

因此适合评估 Pre-VLA 在两类场景中的作用：

- 闭环机器人执行；

## 3.3 训练数据

Pre-VLA 的训练数据来自 LIBERO 中的 PPO rollout 轨迹。

数据规模：

- 总计 1,284,485 个动作块样本；
- 初始正负样本比例约为 95:5；
- 训练 / 测试划分比例为 92:8；
- 分类阈值设为 `-0.21`；
- 训练 batch 中动态维持 30% 负样本比例，以缓解类别不平衡。

## 3.4 对比方法

闭环执行中主要比较三类方法：

1. **RynnVLA-002**
   - 原始 VLA，直接执行第一个生成动作块。

2. **Random Resampling**
   - 从候选动作中随机选择动作；
   - 用于验证性能提升是否仅来自额外采样。

3. **Pre-VLA**
   - 使用动作有效性验证器过滤候选动作；
   - 失败时触发重采样或 fallback 选择。

   - 包括 LIBERO-10、Goal、Object、Spatial；
   - 每个 suite 执行 10,000 episodes。

4. **运行时效率分析**
   - 测量每个 suite 的单动作块验证时间；
   - 测量平均重采样次数。

5. **世界模型 rollout 可视化分析**
   - 比较使用与不使用 Pre-VLA 时的未来图像预测质量。

6. **消融实验**
   - 比较 Raw Score Thresholding；
   - Imbalance-aware Loss Scoring；
   - 完整 Pre-VLA。

7. **高基线任务 case study**
   - 选择 6 个 RynnVLA-002 原本表现较好的任务；
   - 每个任务 250 episodes；
   - 分析在强基线上的增益。

## 5.2 实验是否充分

总体来看，实验覆盖较全面：

- 有离线分类能力验证；
- 有闭环执行指标；
- 有世界模型 rollout 分析；
- 有消融实验；
- 有任务级 case study；
- 有运行时开销统计；
- 闭环实验 episode 数量较大，四个 suite 各 10,000 episodes，规模较充分。

## 5.3 公平性与客观性

论文声称所有方法在相同条件下比较，包括：

- 相同 LIBERO 仿真环境；
- 相同 RynnVLA-002 checkpoint；
- 相同图像预处理；
- 相同动作块长度；
- 相同最大 episode steps；
- 相同随机种子协议；
- 相同候选动作采样预算。

这增强了实验公平性。

不过也存在一些需要注意的地方：

- 世界模型部分主要是定性可视化，缺少更系统的量化指标；
- 真实机器人实验缺失，所有主要结果基于 LIBERO 仿真；
- 监督标签依赖 PPO Critic，可能继承 critic 偏差；
- 论文中部分图注和正文出现 “ARGUS” 命名，与 Pre-VLA 存在不一致，可能是框架早期名称或文本遗留，写作上略显混乱。

## 6. 主要结论与发现

论文的主要发现包括：

- Pre-VLA 能够较可靠地区分有效和无效动作：
  - F1 = 0.8303；
  - Accuracy = 0.9542；
  - invalid recall = 0.9800；
  - false pass rate = 0.0200。

- 在成功轨迹中，Pre-VLA 判断为有效的动作比例明显更高：
  - 成功轨迹平均 pass rate = 0.8424；
  - 失败轨迹平均 pass rate = 0.5167；
  - 差距约 32.6%。

- 在四个 LIBERO suite 上，Pre-VLA 提升闭环成功率：
  - RynnVLA-002 平均成功率：30.79%；
  - Pre-VLA 平均成功率：37.62%；
  - 绝对提升 6.83 个百分点。

- Pre-VLA 同时减少任务执行步数，说明其并不是通过延长探索时间提升成功率，而是帮助策略更快进入有效轨迹。

- 随机重采样不能稳定提升性能，说明收益主要来自 Pre-VLA 对动作质量的判别，而不是单纯增加采样次数。

- 在世界模型 rollout 中，Pre-VLA 能减少错误动作导致的视觉漂移、模糊、ghosting 和虚假完成现象。

- 消融结果表明：
  - 类别不平衡损失有助于改善无效动作识别；
  - 概率阈值校准显著降低 false pass 和 false reject；
  - 多任务训练和校准组合效果最佳。

## 7. 优点与亮点

## 7.1 方法设计亮点

- **抢占式验证思想清晰**
  - 在动作执行前进行安全过滤，而不是事后修正，适合长时程具身任务。

- **同时适用于执行与世界模型**
  - 一个 verifier 同时服务于物理执行和 WM imagination，具有统一性。

- **动作块级验证更贴近 VLA 输出形式**
  - 不是单步动作判断，而是对 K-step action chunk 进行质量评估，更符合现代 VLA 的动作生成方式。

- **充分利用 PPO Critic 信号**
  - 将强化学习中的 advantage 转换为 runtime safety supervision，减少人工标注成本。

- **考虑延迟失败**
  - failure-aware penalty 使模型能够惩罚那些短期看似正常、但长期导致失败的动作。

- **双分支设计合理**
  - 分类分支用于快速拦截；
  - 回归分支用于 fallback 选择；
  - 避免系统在所有候选动作都被拒绝时陷入死循环。

- **多任务损失针对实际问题**
  - Focal Loss 应对类别不平衡；
  - 回归损失提供连续质量排序；
  - soft-threshold loss 缓解阈值边界振荡。

## 7.2 实验设计亮点

- 实验维度较丰富，不只看成功率；
- 包含运行时延迟和重采样次数，关注实际部署成本；
- 使用 Random Resampling 排除了“只是多采样带来收益”的可能；
- 在强基线任务上额外做 case study，说明方法在高成功率场景下仍有增益；
- 报告了较详细的算力配置和训练时间，便于评估复现成本。

## 8. 不足与局限

## 8.1 监督信号依赖 PPO Critic

Pre-VLA 的标签和连续优势值来自 PPO Critic，因此存在潜在偏差：

- 如果 Critic 对某些状态估值不准，verifier 会学习到错误边界；
- 当 base policy 遇到严重分布外状态时，Critic-derived signal 可能不可靠；
- 不同任务和不同机器人 embodiment 下，critic 信号可能需要重新训练或校准。

## 8.2 缺少真实机器人验证

实验主要基于 LIBERO 仿真环境，没有真实机器人部署结果。

这限制了结论的外推性：

- 真实传感噪声；
- 执行器延迟；
- 接触动力学误差；
- 物体材质变化；
- 摄像头视角变化；

这些因素都可能影响 Pre-VLA 的实际可靠性。

## 8.3 世界模型实验偏定性

WM rollout 部分主要通过可视化展示效果，缺少系统量化指标，例如：

- 视频预测误差；
- 物体位置误差；
- rollout consistency；
- false success rate；
- 节省的 GPU 推理量；
- 长时程误差累积曲线。

因此，Pre-VLA 对世界模型的改善虽然直观，但量化证据不够充分。

## 8.4 推理成本仍然增加

虽然单次 verifier 前向约 183.9 ms，但完整闭环系统由于重采样导致推理时间增加明显：

- RynnVLA-002：约 613.77 ms / step；
- Pre-VLA：约 1097.73 ms / step。

在更高频控制或资源受限机器人平台上，这可能成为部署瓶颈。

## 8.5 极端 OOD 状态下可能反复重采样

当系统进入高度异常状态时：

- 多个候选动作都可能被判定为低质量；
- 重采样会增加延迟；
- fallback 动作也未必真正安全；
- 论文虽设置最大尝试次数 `N`，但这只是工程折中，并不能完全保证安全。

## 8.6 泛化范围仍有限

实验集中在：

- LIBERO benchmark；
- RynnVLA-002；
- 特定动作 chunk 设置；
- 特定机器人操作任务。

尚未验证其在以下场景中的泛化能力：

- 其他 VLA 模型，如 OpenVLA、π0 / π0.5、GR00T；
- 移动机器人或双臂机器人；
- 更开放的真实世界任务；
- 多机器人协作任务；
- 复杂动态环境。

## 8.7 论文表述存在小问题

文中图注和部分内容中出现 “ARGUS” 名称，而标题和主体方法称为 Pre-VLA。这可能是命名遗留或框架别名，但论文中未充分解释，可能影响阅读一致性。

---

# 总体评价

Pre-VLA 是一篇针对 VLA 与世界模型运行时可靠性问题的实用型工作。它的关键贡献不在于提出新的基础 VLA 模型，而是在现有 VLA / WM 推理链路中加入一个轻量、可训练、可重采样的动作验证层。论文方法针对类别不平衡、动作质量连续性、边界校准和推理预算等实际问题做了较完整设计，并在 LIBERO 上给出了较大规模闭环实验验证。

从结果看，Pre-VLA 能够显著降低低质量动作进入执行或世界模型的概率，并提升任务成功率与 rollout 稳定性。不过，其可靠性仍依赖 PPO Critic 监督质量，真实机器人和更开放任务上的泛化能力仍需进一步验证。
