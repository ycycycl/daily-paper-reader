## 摘要
自主泊车需要高效的路径规划，以确保在受限环境中的运动学可行性和碰撞规避。Hybrid A* 被广泛使用，但计算代价较高；而强化学习（RL）方法缺乏可靠性，并且往往难以处理长时域几何约束，从而导致次优轨迹。我们提出 N3P，一种用于自动泊车的快速、基于学习的三阶段框架。通过引入一个中间准备位姿并使用学习模块对其进行预测，N3P 将泊车机动分解为更简单的子问题，从而降低计算复杂度并加速路径生成。我们通过将该框架与 Hybrid A* 算法集成来验证其有效性。在垂直泊车和平行泊车场景中的实验表明，N3P 增强的 Hybrid A* 可将规划速度提升 80% 以上。与强化学习基线相比，它在成功率和轨迹质量方面也表现更优，能够生成更短且换挡次数更少的轨迹，同时在大多数情况下实现相当或更低的规划时间。

## Abstract
Autonomous parking requires efficient path planning that ensures kinematic feasibility and collision avoidance in constrained environments. Hybrid A* is widely used but computationally expensive, while reinforcement learning (RL) methods lack reliability and often struggle with long-horizon geometric constraints, leading to suboptimal trajectories. We present N3P, a fast learning-based three-stage framework for automated parking. By introducing an intermediate preparatory pose and using a learning module to predict it, N3P decomposes the maneuver into simpler subproblems, thereby reducing computational complexity and accelerating path generation. We validate the framework by integrating it with Hybrid A* algorithms. Experiments in perpendicular and parallel parking scenarios show that N3P-enhanced Hybrid A* speeds up planning by more than 80%. It also outperforms RL baselines in success rate and trajectory quality, producing shorter trajectories with fewer gear changes, while achieving comparable or lower planning time in most cases.

## 2. 方法论：核心思想、关键技术与算法流程

### 2.1 三阶段自然式泊车框架

N3P 将自动泊车过程划分为三个阶段：

1. **Preparation Stage：准备阶段**
   - 根据停车环境几何信息，预测一个中间准备位姿 \( g^{\hat{p}}_{pre} \)。
   - 该位姿应满足：
     - 起点到该位姿较容易规划；
     - 该位姿到最终泊车位可以通过简单、平滑、可行的轨迹连接。

2. **Approaching Stage：接近阶段**
   - 使用传统规划器，例如 Hybrid A\*，从车辆初始状态 \( x^{\hat{p}}_0 \) 规划到预测的准备位姿 \( g^{\hat{p}}_{pre} \)。

3. **Parking Stage：泊入阶段**
   - 从准备位姿到最终停车位 \( g^{\hat{p}} \)。
   - 主要使用 Reeds–Shepp 路径或带 RS 扩展的 Hybrid A\* 完成最终连接。

---

### 2.2 坐标系与车辆模型

- 论文将问题统一表示在以停车位为中心的局部坐标系 \( \hat{p} \) 中：
  - 停车位中心为原点；
  - \(x\) 轴沿车道方向；
  - \(y\) 轴从停车位指向车道。

- 车辆采用低速泊车常用的**运动学自行车模型**：

\[
\dot{x} = v \cos \theta
\]

\[
\dot{y} = v \sin \theta
\]

\[
\dot{\theta} = \frac{v}{l} \tan \delta
\]

其中：

- \(x, y\)：车辆位置；
- \(\theta\)：航向角；
- \(v\)：纵向速度；
- \(\delta\)：前轮转角；
- \(l\)：轴距。

车辆控制输入满足速度和转角边界约束。

### 2.3 环境抽象：将复杂障碍布局压缩为低维参数

论文认为，准备位姿的选择并不需要完整建模所有障碍点云，而主要受停车位附近可用自由空间影响。

因此，作者将停车环境抽象为四个参数：

\[
E = \{ W_{lane}, W_{spot}, D_{deadend}, \gamma \}
\]

其中：

- \(W_{lane}\)：车道宽度；
- \(W_{spot}\)：停车位宽度；
- \(D_{deadend}\)：死胡同深度或局部边界距离；
- \(\gamma\)：泊车类型，包括：
  - forward parking，前向泊车；
  - reverse parking，倒车泊车；
  - parallel parking，平行泊车。

这种抽象的意义是：

- 将复杂、高维、变化多样的障碍点云转化为可离散采样的低维空间；
- 便于离线生成覆盖不同几何条件的数据；
- 提高学习模型的泛化能力；
- 减少在线推理和规划负担。

---

### 2.4 离线数据生成：利用 Hybrid A\* + Reeds–Shepp 提取示范准备位姿

N3P 的训练数据不是人工标注，而是通过离线规划自动生成。

流程如下：

1. 对 \(W_{lane}\)、\(W_{spot}\)、\(D_{deadend}\)、\(\gamma\) 进行离散采样，构造一批抽象泊车环境。
2. 在每个环境中随机采样车辆初始状态 \( \tilde{x}^{\hat{p}}_0 \)。
3. 使用带 Reeds–Shepp 扩展的 Hybrid A\* 离线求解从初始状态到目标泊车位的路径。
4. 从求得的轨迹中提取 Reeds–Shepp 段的起点 \( \tilde{x}^{\hat{p}}_{rs} \)，作为理想准备位姿。
5. 存储：
   - 输入：初始状态 + 环境参数；
   - 输出：对应的 RS 起始位姿，即准备位姿标签。

这种设计的好处是：

- RS 起点已经位于一条可行 Hybrid A\* 路径上；
- 它天然保证从初始状态到该点可行；
- 同时从该点到目标位姿存在一条 Reeds–Shepp 连接路径。

---

### 2.5 学习式准备位姿选择器

学习模型的输入为：

\[
x = [x^{\hat{p}}_0, y^{\hat{p}}_0, \theta^{\hat{p}}_0, W_{lane}, W_{spot}, D_{deadend}]^\top
\]

输出为准备位姿：

\[
g^{\hat{p}}_{pre} = f_\gamma(x; w)
\]

其中模型按泊车类型 \(\gamma\) 分别训练。

论文采用了两种轻量模型：

#### KNN 模型

- 使用 1-nearest neighbor 进行预测。
- 优点：
  - 输出一定来自离线数据集中真实可行的 RS 起点；
  - 因此理论上更容易保证准备位姿属于可行集合。
- 缺点：
  - 存储开销较大；
  - 查询依赖数据规模。

#### MLP 模型

- 使用四层隐藏层的前馈神经网络：
  - 256–256–256–128；
  - ReLU 激活。
- 优点：
  - 表达非线性关系能力较强；
  - 存储占用更小。
- 缺点：
  - 输出不一定严格落在可行准备位姿集合中；
  - 可能导致碰撞或连接失败；
  - 因此需要额外用 Hybrid A\* 或 HARS 验证和修正。

### 2.6 在线执行流程

在线算法大致如下：

1. 将车辆初始位姿、目标位姿和障碍点云转换到停车位坐标系 \( \hat{p} \)。
2. 从障碍点云中提取环境参数 \(E\)。
3. 将连续环境参数匹配到离线训练集中最近的离散环境配置。
4. 构造模型输入：
   \[
   [x^{\hat{p}}_0, y^{\hat{p}}_0, \theta^{\hat{p}}_0, W_{lane}, W_{spot}, D_{deadend}]
   \]
5. 使用 KNN 或 MLP 预测准备位姿。
6. 使用 Hybrid A\* 等规划器规划：
   - 从初始位姿到准备位姿；
   - 再从准备位姿到最终泊车位。
7. 拼接两段路径作为最终泊车路径。

  - 成功率高；
  - 运行时间低；
  - 节点扩展数极少；
  - 在无噪声仿真中可实现 100% 可行性。
- **MLP-N3P** 存储更小，但偶尔预测不可行准备位姿，因此存在非零失败率。
- 与 RL 方法相比：
  - N3P 成功率更高；
  - 轨迹更短；
  - 换挡次数显著更少；
  - 转向变化更平滑；
  - 对训练 / 测试分布变化更鲁棒。
- RL-Simple 在测试环境中失败率较高，说明强化学习方法存在明显分布偏移问题。
- RL-Eval 在部分场景中成功率改善，但轨迹质量仍然较差，常出现大量方向切换和转向抖动。
- N3P 的关键收益来自任务分解：通过预测准备位姿，将长时域、高复杂度的直接泊车规划问题拆成更易求解的短规划问题。

---

## 7. 优点与亮点

### 7.1 方法设计亮点

- **人类驾驶启发明确**
  - 模仿人类司机先调整到有利位姿、再完成泊车的自然策略。
  - 相比端到端学习，更具可解释性。

- **学习与规划结合合理**
  - 学习模块只负责预测中间准备位姿；
  - 最终路径仍由结构化规划器生成；
  - 保留了传统方法的可行性和安全性优势。

- **环境抽象低维高效**
  - 将复杂障碍布局转化为少量几何参数；
  - 降低训练数据需求；
  - 提高在线效率。

- **离线数据生成方式巧妙**
  - 使用 Hybrid A\* + Reeds–Shepp 自动生成准备位姿标签；
  - 避免人工标注；
  - 标签天然具有可行性意义。

- **KNN 方案具备可行性优势**
  - 1-NN 输出来自离线可行样本；
  - 因此比 MLP 更容易保证连接到最终泊车位。

- **可插拔性强**
  - N3P 不局限于某个具体规划器；
  - 可以与 Hybrid A\* 或优化式规划器结合；
  - 也可作为优化方法的初始化前端。

---

### 7.2 实验设计亮点

- 对比方法较全面，涵盖：
  - 标准 Hybrid A\*；
  - RS 增强 Hybrid A\*；
  - 多启发式 Hybrid A\*；
  - 场景化 Hybrid A\*；
  - 强化学习泊车方法 HOPE。
- 评价指标较丰富，不只看速度，也看：
  - 成功率；
  - 轨迹长度；
  - 转向变化；
  - 换挡次数；
  - 节点扩展数。
- 测试环境比训练环境更复杂，能够体现一定的泛化能力。

---

## 8. 不足与局限

### 8.1 环境抽象可能过于简化

- N3P 只使用：
  - 车道宽度；
  - 停车位宽度；
  - 死胡同深度；
  - 泊车类型。
- 这种抽象忽略了更复杂的障碍形状和布局，例如：
  - 倾斜死胡同；
  - 不规则障碍；
  - 非矩形车道；
  - 动态障碍；
  - 弯曲车道；
  - 停车场中复杂拓扑结构。
- 论文也承认，在死胡同朝目标停车位倾斜等情况下，抽象可能低估可行区域，导致次优解。

---

### 8.2 对特定车辆依赖明显

- 离线数据与车辆尺寸、轴距、最小转弯半径有关。
- 当车辆尺寸变化时，需要重新生成数据并重新训练模型。
- 跨车型泛化尚未解决。

---

### 8.3 对噪声和真实感知误差验证不足

- 实验主要是仿真环境。
- KNN 在无噪声条件下具备高可行性，但真实环境中：
  - 障碍点云可能有噪声；
  - 车位检测可能不准；
  - 车辆定位存在误差；
  - 环境参数估计可能偏差。
- 论文提到可通过安全裕度处理测量噪声，但未进行系统实验验证。

---

### 8.4 缺乏真实车或真实停车场实验

- 文中实验均为模拟随机场景。
- 没有：
  - 实车测试；
  - 真实传感器输入；
  - 多楼层停车场；
  - 人车混行；
  - 动态障碍；
  - 控制闭环执行误差验证。
- 因此距离实际量产部署仍需进一步验证。

---

### 8.5 MLP 版本缺乏可行性保证

- MLP 输出连续预测结果，不一定对应真实可行的 Reeds–Shepp 起点。
- 因此可能出现：
  - 准备位姿碰撞；
  - 无法连接最终目标；
  - 需要额外节点扩展；
  - 少量失败。
- 相比之下，KNN 更稳定，但内存占用更高。

---

### 8.6 极端平行泊车仍有少数失败或加速不足

- 论文指出，在 Extreme 难度平行泊车中，约 1–2% 场景下运行时间降低不明显。
- 这些情况通常发生在：
  - 目标车位靠近死胡同墙；
  - 对向车道空间受限；
  - 可行动作空间极窄。
- 说明当前准备位姿表示能力仍不足。

---

### 8.7 消融分析不够充分

论文没有系统展示以下消融：

- 没有准备位姿预测时的三阶段规划效果；
- 不同环境参数分辨率的影响；
- 不同训练数据规模的影响；
- KNN 中不同邻居数量的影响；
- MLP 层数与隐藏单元变化；
- HARS 与纯 RS 连接的差异；
- 安全裕度大小对成功率和保守性的影响。

---

## 总体评价

N3P 是一篇思路清晰、工程意义较强的自动泊车路径规划论文。它没有追求端到端学习，而是将学习用于选择高价值的中间准备位姿，再交由传统结构化规划器完成可行路径生成。这种“学习辅助搜索”的方式在速度、可靠性和轨迹质量之间取得了较好平衡。实验表明，N3P 尤其是 KNN-N3P 能显著减少 Hybrid A\* 的节点扩展和规划时间，同时相比强化学习方法具有更高成功率和更自然的轨迹。

其主要局限在于环境抽象较简化、依赖车辆特定离线数据、缺少真实世界验证，以及对极端复杂几何场景的表达能力仍有限。未来若能结合更丰富的环境表示、跨车型泛化和真实车辆实验，N3P 有潜力成为自动泊车系统中实用的高效规划前端。
