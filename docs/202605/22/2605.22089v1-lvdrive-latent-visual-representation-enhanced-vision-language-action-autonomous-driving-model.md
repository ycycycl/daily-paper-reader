## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景的框架。然而，现有 VLA 通常依赖稀疏的动作监督，未能充分利用其强大的场景理解与推理能力。近期通过世界模型引入密集视觉监督的尝试，往往过度强调像素级图像重建，而忽视了具有语义意义的场景表征学习。在本文中，我们提出 LVDrive，一种用于自动驾驶的潜在视觉表征增强 VLA 框架。LVDrive 将未来场景预测任务引入 VLA 范式，在预训练视觉骨干网络的辅助监督下，完全在高级潜在空间中学习未来表征。不同于低效的自回归生成，我们在统一的嵌入空间中联合建模未来场景与运动预测，并通过单次前向传播进行处理，以执行具备未来感知能力的推理。我们进一步设计了一种两阶段轨迹解码策略，显式利用学习到的潜在未来表征来优化轨迹生成。在具有挑战性的 Bench2Drive 基准上的大量实验表明，LVDrive 在闭环驾驶性能方面取得了显著提升，优于仅使用动作监督的方法以及基于图像重建的世界模型方法。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising framework for end-to-end autonomous driving. However, existing VLAs typically rely on sparse action supervision, which underutilizes their powerful scene understanding and reasoning capabilities. Recent attempts to incorporate dense visual supervision via world modeling often overemphasize pixel-level image reconstruction, neglecting semantically meaningful scene representation learning. In this work, we propose LVDrive, a Latent Visual representation enhanced VLA framework for autonomous driving. LVDrive introduces a future scene prediction task into the VLA paradigm, where future representations are learned entirely in a high-level latent space under auxiliary supervision from a pretrained vision backbone. Departing from inefficient autoregressive generation, we jointly model future scene and motion prediction within a unified embedding space, processed in a single forward pass to conduct the future-aware reasoning. We further design a two-stage trajectory decoding strategy that explicitly leverages the learned latent future representations to refine trajectory generation. Extensive experiments on the challenging Bench2Drive benchmark demonstrate that LVDrive achieves significant improvements in closed-loop driving performance, outperforming both action supervised methods and image-reconstruction-based world model approaches.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 整体框架

LVDrive 是一个潜在视觉表征增强的 VLA 自动驾驶框架，其输入包括：

- 当前与历史的多视角图像特征；
- 语言导航或驾驶指令；
- 结构化场景特征；
- 历史上下文信息。

模型输出包括：

- 未来若干帧的潜在视觉表征；
- 规划 token；
- 多模态未来轨迹。

整体流程可以概括为：

1. 使用视觉编码器提取多视角图像特征；
2. 将视觉特征和文本指令投影到统一 LLM token 空间；
3. 在 LLM 中插入未来视觉占位 token 和规划 token；
4. 单次前向传播同时预测：
   - 未来潜在视觉特征；
   - 动作 / 规划嵌入；
5. 使用两阶段轨迹解码：
   - 第一阶段生成粗略轨迹；
   - 第二阶段显式利用预测的未来视觉表征 refine 轨迹。

---

## 2.2 潜在未来视觉表征学习

- LVDrive 不直接预测未来图像，而是预测未来图像经过预训练视觉骨干提取后的**潜在视觉特征**。
- 论文设计了一组特殊占位符 token：
  - `<img_start>`
  - `<img_i>`
  - `<img_end>`
- 每一帧未来图像由多个 `<img_i>` token 表示。
- 对每个未来时间步，LLM 最后一层中这些视觉占位符对应的 hidden states 被取出，记为未来帧隐状态。

论文形式化表示为：

\[
V_{t+1:t+F}, s \sim p(V_{t+1:t+F}, s | x_s, x_h, x_q)
\]

其中：

- \(x_s\)：当前结构化多视角视觉特征；
- \(x_h\)：历史视觉特征；
- \(x_q\)：语言指令；
- \(V_{t+1:t+F}\)：未来潜在视觉特征；
- \(s\)：规划 token。

随后通过轻量视觉解码器得到潜在未来视觉表征：

\[
V_{t+j} = VIS_\theta(H_{t+j})
\]

其中：

- \(H_{t+j}\)：LLM 中未来视觉占位符对应的 hidden states；
- \(VIS_\theta\)：轻量视觉解码器；
- \(V_{t+j}\)：预测得到的未来潜在视觉特征。

## 2.3 预训练视觉骨干提供辅助监督

- 论文使用冻结的预训练视觉模型生成未来图像的 ground-truth latent feature。
- 主实验中采用 **VQGAN-ImageNet** 作为未来视觉监督来源。
- 具体实现中：
  - 输入未来前视图图像；
  - 图像尺寸为 \(256 \times 256\)；
  - VQGAN 输出 \(16 \times 16 \times 256\) 的潜在视觉特征；
  - 即每帧未来图像对应 256 个视觉 token，每个 token 维度为 256。
- 论文强调：使用前视图未来预测是因为人类驾驶时主要关注前方视角，前视图未来信息对规划最关键。

## 2.4 单次前向而非自回归生成

- 传统基于世界模型的 VLA 常使用自回归方式逐 token 生成未来图像或动作。
- LVDrive 则预先填入固定长度的视觉占位符和规划占位符。
- 模型在一次 forward 中并行得到所有未来视觉表征和动作表征。
- 这样避免了长序列自回归生成带来的高推理开销。

## 2.5 两阶段轨迹解码策略

论文认为，仅将未来视觉预测作为辅助任务还不够，必须让未来视觉信息显式参与轨迹生成。因此设计了两阶段解码。

### 第一阶段：粗略轨迹生成

- 使用规划 token 的 hidden state \(H_p\) 作为规划嵌入。
- 通过 VAE-style generative planner 生成粗略轨迹。
- 规划嵌入被映射到高斯分布：

\[
z \sim p(\mu, \sigma^2), \quad (\mu, \sigma^2) = DIS_\theta(H_p)
\]

- 采样得到 latent variable \(z\)。
- 再结合规划嵌入 \(H_p\)，通过 GRU-based state decoder 生成未来 ego motion states：

\[
s_{ego} = STATE_\theta(z, H_p)
\]

- 最后通过 MLP 得到多模态粗略轨迹 proposal。

### 第二阶段：显式未来视觉条件轨迹优化

- 第一阶段得到的 motion states 被投影成 ego motion queries。
- 未来视觉 hidden states 被投影为 cross-attention 中的 key 和 value。
- 通过 transformer-based trajectory refiner，让轨迹特征显式 attend 到未来视觉表征：

\[
Q_{ego} = PROJ_\theta(s_{ego})
\]

\[
K_{fut}, V_{fut} = PROJ_\theta(H_{t+1:t+F})
\]

\[
s^*_{ego} = TRAJ\_REFINER_\theta(Q_{ego}, K_{fut}, V_{fut})
\]

- 最终通过两个 MLP 分支分别生成：
  - base trajectory；
  - offset trajectory。
- 两者相加得到最终细粒度轨迹。

## 2.6 训练目标

LVDrive 采用端到端训练，不需要复杂多阶段训练流程。总损失包括：

\[
L = L_{vis} + L_{plan} + L_{plan\_r} + L_{qt} + L_{ce}
\]

其中：

- \(L_{vis}\)：未来潜在视觉表征预测损失；
  - 包括 cosine similarity loss；
  - 以及 L1 regression loss。
- \(L_{plan}\)：粗略轨迹规划损失；
- \(L_{plan\_r}\)：最终 refine 轨迹损失；
- \(L_{qt}\)：结构化多视角视觉特征提取模块的辅助损失；
- \(L_{ce}\)：LLM 生成固定占位 token 序列的交叉熵损失。

轨迹损失包括：

- MSE regression loss；
- boundary loss；
- collision loss。

论文还提到，虽然使用 VAE-style planner，但经验上省略了 KL-divergence loss，因为 KL 约束可能会过度限制多模态 latent motion space 的表达能力。

  - 官方评估协议包含 220 条短路线；
  - 每种场景有 5 个不同路线实例；
  - 覆盖不同天气和城镇组合。

此外，论文还使用官方提供的 **Dev10** 子集进行消融实验：

- Dev10 包含 10 条代表性路线；
- 用于快速验证设计选择；
- 避免在完整 220 条测试路线中过度调参。

    - Emergency Brake；

    - Traffic Sign。

开放环评估中使用：

- **Average L2 trajectory error**
  - 预测轨迹与专家轨迹之间的平均 L2 误差。

## 4. 资源与算力

## 6. 主要结论与发现

论文的核心结论包括：

- **潜在视觉表征预测比像素级未来图像重建更适合规划任务。**
  - LVDrive 不追求未来图像高保真重建，而是学习语义潜在表征；
  - 这更符合自动驾驶规划需要。

- **未来场景预测可以提升 VLA 的场景理解和闭环驾驶能力。**
  - 相比仅动作监督的 VLA，LVDrive 取得更高 DS 和 SR。

- **单次前向的潜在空间预测比自回归世界模型更高效。**
  - LVDrive 推理时间为 2.03s；
  - 自回归式等长生成 baseline 为 36.62s。

- **两阶段轨迹解码非常关键。**
  - 仅加入 latent visual prediction 不一定显著提升 SR；
  - 一阶段直接融合甚至会降低性能；
  - 先生成稳定粗轨迹，再用未来视觉语义 refine，效果最好。

- **视觉监督来源会显著影响性能。**
  - VQGAN-ImageNet 在实验中效果最好；
  - DINOv3 有提升但可能信息过丰富，引入冗余；
  - MoVQGAN 由于特征维度较低，表现较差。

- **在 Bench2Drive 上取得领先闭环结果。**
  - LVDrive 获得：
    - DS：80.71；
    - SR：58.26%。
  - 超过 ORION、DriveMoE、MindDrive、UniDrive-WM 等方法。

## 7. 优点与亮点

- **问题定位清晰**
  - 准确指出当前 VLA 自动驾驶的两个核心矛盾：
    - 动作监督太稀疏；
    - 世界模型重建太像素化。

- **方法设计有针对性**
  - 不预测像素，而预测潜在视觉语义；
  - 更贴合规划目标。

- **效率优势明显**
  - 通过预填占位 token 和单次前向传播避免自回归生成；
  - 推理速度远快于自回归世界模型。

- **显式利用未来视觉表征**
  - 不是简单把未来预测作为辅助 loss；
  - 而是通过 trajectory refiner 让未来视觉特征直接参与轨迹优化。

- **两阶段轨迹解码合理**
  - 第一阶段保证运动特征稳定；
  - 第二阶段引入未来场景语义进行细化；
  - 消融实验显示该设计是性能提升关键。

- **实验覆盖较完整**
  - 包括主 benchmark、分项能力、消融、效率、定性案例；
  - 对比方法数量较多，涵盖传统 E2E 和 VLA 方法。

- **取得较强闭环性能**
  - 在 Bench2Drive 上达到当前非常强的结果；
  - 特别是在 Traffic Sign 和 Merging 场景上表现突出。

---

## 8. 不足与局限

- **主要依赖仿真环境**
  - 实验集中在 CARLA / Bench2Drive；
  - 缺少真实世界驾驶数据和真实车辆闭环测试；
  - 仿真到现实的泛化能力仍未验证。

- **前视图未来预测带来视角偏差**
  - 论文主要预测未来前视图潜在特征；
  - 对后方车辆、侧后方交互、紧急车辆让行等场景关注不足；
  - Give Way 任务表现较差正体现了这一问题。

- **消融实验主要在 Dev10 小规模子集上进行**
  - Dev10 只有 10 条路线；
  - 虽然便于开发调试，但统计稳定性有限；
  - 若完整 benchmark 上也提供所有消融结果，说服力会更强。

- **训练资源需求较高**
  - 使用 32 张 NVIDIA H20 96GB GPU；
  - 对普通研究团队复现存在较高门槛。

- **视觉骨干选择仍有经验性**
  - VQGAN-ImageNet 在实验中表现最好；
  - 但不同视觉 backbone 为什么更适合规划仍缺少更深入理论解释；
  - 论文也承认，使用自动驾驶领域预训练视觉模型可能进一步提升性能。

- **语言监督作用没有被充分探索**
  - 方法主要利用 LLM 的推理空间和语言指令；
  - 但没有深入研究语言监督如何进一步增强视觉-动作表征对齐。

- **实际部署仍有挑战**
  - 虽然比自回归快很多，但单次推理约 2.03s，距离高频实时自动驾驶控制仍可能偏慢；
  - 还需要进一步模型压缩、加速或工程优化。

- **安全性验证有限**
  - Bench2Drive 包含违规惩罚和碰撞指标，但还不足以覆盖真实长尾场景；
  - 对极端天气、传感器故障、罕见交通行为等情况的鲁棒性未充分验证。
