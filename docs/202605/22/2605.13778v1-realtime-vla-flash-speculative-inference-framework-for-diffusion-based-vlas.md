## 摘要
基于扩散的视觉-语言-动作模型（dVLA）在具身智能中前景广阔，但由于完整推理的高延迟，其在实时部署中受到根本限制。我们提出 Realtime-VLA FLASH，这是一种推测式推理框架，通过引入轻量级草稿模型，并结合主模型 Action Expert 的并行验证以及在需要时回退到完整推理流水线的阶段感知回退机制，在重规划过程中消除了大多数完整推理调用。该设计在不牺牲可靠性的前提下，实现了低延迟、高频率的重规划。实验表明，在 LIBERO 上，FLASH 通过用最快 7.8 ms 的推测式轮次替代大量 58.0 ms 的完整推理轮次，在很大程度上保持了任务性能，并将任务级平均推理延迟降低至 19.1 ms（加速 3.04 倍）。我们还在真实世界的传送带分拣任务中展示了其有效性，凸显了其对延迟敏感型具身任务的实际影响。

## Abstract
Diffusion-based vision-language-action models (dVLAs) are promising for embodied intelligence but are fundamentally limited in real-time deployment by the high latency of full inference. We propose Realtime-VLA FLASH, a speculative inference framework that eliminates most full inference calls during replanning by introducing a lightweight draft model with parallel verification via the main model's Action Expert and a phase-aware fallback mechanism that reverts to the full inference pipeline when needed. This design enables low-latency, high-frequency replanning without sacrificing reliability. Experiments show that on LIBERO, FLASH largely preserves task performance by replacing many 58.0 ms full-inference rounds with speculative rounds as fast as 7.8 ms, lowering task-level average inference latency to 19.1 ms (3.04x speedup). We additionally demonstrate effectiveness on real-world conveyor-belt sorting, highlighting its practical impact for latency-critical embodied tasks.

## 2. 方法论：核心思想、关键技术与算法流程

### 2.1 核心思想

- FLASH 借鉴大语言模型中的 **speculative inference / speculative decoding** 思路：
  - 用一个轻量级草稿模型快速生成候选输出；
  - 再由主模型并行验证候选是否可信；
  - 若可信，则执行候选动作；
  - 若不可信，则回退到完整主模型推理。

- 但 dVLA 与自回归语言模型不同：
  - 语言模型输出离散 token，可用 token 概率做验证；
  - dVLA 输出连续动作 chunk；
  - π0 类模型通过 Flow Matching / ODE 多步生成动作，没有显式 token 概率；
  - 因此最大难点是：**如何低成本验证一个连续动作 chunk 是否与主模型一致？**

- 本文的关键观察是：
  - Flow Matching 在训练时学习的是从噪声到目标动作之间的连续插值路径上的速度场；
  - 如果草稿动作是合理的，那么将草稿动作与高斯噪声插值得到中间状态后，主模型 Action Expert 应能从这些中间状态重构出接近草稿动作的端点；
  - 因此可以利用主模型 Action Expert 对若干中间时间步进行并行验证，而不必完整执行多步顺序去噪。

---

### 2.2 双路径推理框架

FLASH 包含两条推理路径：

#### Full Path：完整路径

- 与原始 π0 推理一致：
  - Image Encoder
  - VLM prefill
  - Action Denoise / Action Expert 多步去噪
- 用于：
  - 初始化或刷新上下文；
  - 推测失败后的纠正；
  - 精细调整阶段的高质量动作生成。

#### Flash Path：推测路径

- 仍然对当前图像进行编码；
- 跳过昂贵的 VLM prefill；
- 使用轻量草稿模型生成候选动作 chunk；
- 复用最近一次 full path 得到的视觉-语言 KV Cache；
- 调用主模型 Action Expert 对草稿动作进行并行一致性验证；
- 若验证通过，执行最长可信前缀；
- 若验证失败，则回退到 full path。

---

### 2.3 草稿模型设计

- 草稿模型用于快速生成候选动作 chunk。
- 架构特点：
  - 使用一个与主 VLM 结构相近的 **单层 Gemma block**；
  - 参数量约 **110M**，远小于约 **2.7B** 参数的主 VLM；
  - 输入包括：
    - 当前图像编码特征；
    - 语言指令；
    - 机器人状态；
    - 一组可学习的 action queries。
  - 每个 action query 对应未来 action chunk 中的一个时间步。
  - 所有 action queries 可并行生成完整动作 chunk。
  - 最后通过线性 action head 输出连续动作。

- 训练目标：
  - 使用监督回归训练草稿模型预测目标动作 chunk。
  - 损失形式为加权动作回归：
    \[
    L_{draft} = \sum_{h=1}^{H} w_h \ell(\hat{a}^{d}_{t+h-1}, a_{t+h-1})
    \]
  - 其中：
    - \(H\) 是 action chunk 长度；
    - \(w_h\) 是不同未来步的权重；
    - \(\ell\) 是 Smooth L1 / Huber 等动作回归损失。
  - 文中强调对前缀动作赋予更高权重，因为实际执行时通常只执行 chunk 的前一部分。

---

### 2.4 基于 Flow Matching 的并行验证

- 原始 π0 类 dVLA 通过多步顺序去噪求解 ODE：

\[
\frac{dA_t^\tau}{d\tau} = v_\theta(A_t^\tau, \tau, o_t), \quad A_t^0 \sim \mathcal{N}(0, I)
\]

- FLASH 不完整运行多步去噪，而是对草稿动作端点 \(\hat{A}^{(d)}_t\) 和高斯噪声 \(\epsilon\) 构造中间状态：

\[
\tilde{A}_{\tau} = \tau \hat{A}^{(d)}_t + (1-\tau)\epsilon
\]

- 然后调用主模型 Action Expert 预测该状态下的局部速度：

\[
v_\theta(\tilde{A}_{\tau}, \tau \mid c_t, s_t)
\]

- 再用单步近似重构动作端点：

\[
\hat{A}_t(\tau) = \tilde{A}_{\tau} + (1-\tau)v_\theta(\tilde{A}_{\tau}, \tau \mid c_t, s_t)
\]

- 如果草稿动作与主模型在当前上下文下一致，那么不同验证时间步重构出的端点应接近草稿动作。

---

### 2.5 多时间步并行验证与最长前缀接受

- FLASH 选择一组验证时间步：

\[
T = \{\tau_1, \tau_2, ..., \tau_K\}
\]

- 对所有 \(\tau_k\) 并行执行 Action Expert 验证。
- 对每个未来动作步 \(h\)，计算草稿动作与重构动作之间的连续通道距离：
  - 主要检查位置和旋转等连续控制量；
  - gripper 通道单独处理，因为其语义更接近离散开闭状态。
- 若某个动作步在所有验证分支上距离均低于阈值 \(\delta\)，则认为该步通过验证。
- 系统接受从 chunk 开头开始的最长连续可信前缀。
- 如果接受长度 \(L = 0\)，则说明草稿完全不可信，立即回退 full path。

- 该策略比二元接受 / 拒绝更细粒度：
  - 可以只执行可信前缀；
  - 剩余部分等待下一轮重规划或 full path 修正；
  - 有助于平衡速度与可靠性。

---

### 2.6 阶段感知回退机制

- 仅靠局部一致性验证仍可能在精细操作阶段失败。
- 论文观察到：
  - 在大范围平滑运动阶段，草稿动作的小误差通常可被容忍；
  - 但在抓取、释放、末端对齐等精细调整阶段，小误差会快速累积并导致失败。

- 因此 FLASH 使用 **gripper switch** 作为精细阶段的启发式信号：
  - 在 LIBERO 中，gripper 开闭通常编码为 -1 和 1；
  - 标准化后可通过零阈值检测开闭切换；
  - 若候选 chunk 中出现 gripper 状态切换，则认为即将进入抓取或释放等关键阶段；
  - 系统主动回退到 full path，生成更高质量动作。

- 此外，文中还在部分实验中加入周期性 full path 刷新：
  - 例如每隔若干个 flash-path round 强制执行一次 full path；
  - 用于缓解长时间推测执行带来的轨迹漂移。

  - 每个 draft model 约 **6 小时**

  - Optimizer：AdamW；

## 5. 实验数量与充分性

### 5.1 实验覆盖范围

论文实验包含以下几类：

- **主仿真实验**
  - 4 个 LIBERO suite；
  - 40 个任务；
  - 每任务 50 次；
  - 比较 4 种方法；
  - 评估成功率与延迟。

- **Flash-path 统计分析**
  - 统计 accepted prefix length；
  - 统计 flash path rate；
  - 分析不同 suite 中推测路径覆盖比例与接受长度。

- **消融实验**
  - 在 LIBERO-10 上分析：
    - flash-path only；
    - periodic full-path refresh；
    - phase-aware fallback；
    - fallback 与 periodic refresh 的组合。
  - 验证 phase-aware fallback 和周期性 full path 对可靠性的作用。

- **验证器超参数敏感性实验**
  - 改变验证时间步数量 \(K\)；
  - 改变距离阈值 \(\delta\)；
  - 分析成功率、延迟、接受率、flash path 使用率之间的 trade-off。

- **跨 suite 阈值敏感性分析**
  - 固定 \(K=2\)，在所有 LIBERO suite 上扫不同 \(\delta\)。

- **真实世界实验**
  - 2 类物体；
  - 3 种传送带速度；
  - 3 种方法；
  - 每条件 10 次 trial。

### 5.2 是否充分

- 从仿真实验角度看，实验较为充分：
  - 覆盖 LIBERO 多个标准 suite；
  - trial 数量较多；
  - 同时报告成功率和延迟；
  - 包含系统优化 baseline 和原始 baseline；
  - 消融实验比较完整。

- 从真实世界角度看，实验能支持论文关于“低延迟提升高速动态任务成功率”的主要论点，但覆盖仍有限：
  - 只有一个真实任务类型：传送带分拣；
  - 只有两个物体类别；
  - 每个条件只有 10 次 trial，统计置信度有限；
  - 未覆盖更多机器人平台、更多动态任务或复杂长程任务。

### 5.3 公平性与客观性

- 公平性较好之处：
  - 在线推理均在单张 RTX 4090D 上测试；
  - 真实世界对比中使用相同同步推理循环；
  - 明确不使用异步推理、RTC 等额外延迟隐藏策略，以隔离推理延迟影响；
  - 与 Triton-π0 对比，有助于说明 FLASH 与底层 kernel 优化是互补的。

- 可能的公平性风险：
  - FLASH 额外训练了草稿模型，而 baseline 没有类似额外模块；
  - 每个 LIBERO suite 单独训练 draft model，可能带来 suite-specific 调优优势；
  - phase-aware fallback 使用 gripper switch 启发式，适合含明显抓取 / 释放阶段的任务，但不一定适合所有机器人任务。

## 6. 主要结论与发现

- FLASH 能显著减少完整推理调用，降低平均重规划延迟。
- 在 LIBERO 上：
  - Torch-π0 平均延迟：58.0 ms；
  - Triton-π0 平均延迟：39.7 ms；
  - FLASH-π0 平均延迟：34.9 ms；
  - FLASH+Triton-π0 平均延迟：19.1 ms。
- FLASH+Triton-π0 相比 Torch-π0：
  - 实现 **3.04×** 任务级平均延迟加速；
  - 每动作延迟从 5.0 ms 降至 1.9 ms；
  - 平均成功率仅从 94.1% 降至 93.8%，下降 0.3 个百分点。

- Flash path 的单轮成本最低可达：
  - FLASH-π0：17.9 ms；
  - FLASH+Triton-π0：7.8 ms。

- 在 LIBERO 上，FLASH+Triton-π0：
  - 平均约 **66.8%** 的重规划轮次使用 flash path；
  - flash path 中平均接受前缀覆盖约 **69.7%** 的 replan window。

- 在真实传送带分拣中：
  - 随着传送带速度升高，原始 JAX-π0 和 Triton-π0 成功率快速下降；
  - 在 15 m/min 的 extra high 速度下，JAX-π0 和 Triton-π0 均为 0 成功；
  - FLASH+Triton-π0 仍能在 toy dog 和 hairbrush 上取得非零成功率；
  - 说明降低同步推理延迟能直接扩大 dVLA 可处理的动态任务速度范围。

## 7. 优点与亮点

### 7.1 方法创新性

- 将推测式推理从离散 token 的语言模型 / 自回归 VLA 扩展到连续动作的 Flow Matching dVLA。
- 针对 dVLA 缺少 token 概率的问题，提出基于 Flow Matching 插值路径的动作端点一致性验证。
- 避免完整顺序去噪，通过多时间步并行验证实现低成本可信度判断。

### 7.2 系统设计实用

- 双路径架构清晰：
  - full path 保证高质量；
  - flash path 提供低延迟；
  - fallback 机制平衡速度与可靠性。
- 能与 Triton 等底层推理优化结合，说明方法具有互补性。
- 使用最近一次 full path 的 KV Cache，减少重复 VLM prefill，针对 profiling 发现的瓶颈设计合理。

### 7.3 细粒度接受机制

- 不是简单接受或拒绝整个 action chunk；
- 而是接受最长可信动作前缀；
- 更适合机器人控制中的连续重规划场景；
- 可以避免因 chunk 后部不可信而浪费前部可用动作。

### 7.4 阶段感知回退有效

- 作者观察到精细调整阶段容易因推测误差失败；
- 使用 gripper switch 作为简单有效的阶段切换信号；
- 在 LIBERO bowl-to-plate 等任务中展示了该机制能避免最终放置漂移。

### 7.5 实验较全面

- 包含仿真和真实机器人实验；
- 同时关注成功率与延迟；
- 有消融、阈值敏感性分析和成本分解；
- 对方法加速来源有较清楚解释。

## 8. 不足与局限

### 8.1 验证机制仍是启发式

- 论文明确指出，端点一致性验证不是形式化保证。
- 通过验证只能说明草稿动作在局部与主模型速度场兼容，并不能保证与 full path 生成的完整轨迹等价。
- 误差来源包括：
  - Action Expert 局部速度场近似误差；
  - 复用旧 KV Cache 带来的条件上下文不匹配；
  - 草稿诱导插值路径与训练时目标路径之间的不一致。

### 8.2 超参数敏感

- 验证阈值 \(\delta\) 和验证时间步数量 \(K\) 对性能影响明显。
- 阈值过松会导致错误草稿被接受，成功率下降；
- 阈值过严会频繁回退 full path，降低加速效果。
- 当前阈值和验证时间步主要依靠人工选择，缺乏自适应机制。

### 8.3 Phase-aware fallback 依赖任务结构

- 使用 gripper switch 作为精细阶段信号适合抓取 / 释放类任务；
- 对没有明显 gripper 切换的任务、灵巧手操作、移动机器人导航、多阶段接触任务，可能不适用；
- 该启发式可能漏检某些精细调整阶段，也可能误触发 full path，影响加速。

### 8.4 需要额外训练草稿模型

- FLASH 不是完全 training-free 的推理加速方法。
- 每个 LIBERO suite 单独训练 draft model；
- 真实世界任务也需要训练对应草稿模型；
- 这增加了部署成本，并可能限制泛化到新任务、新机器人或新环境的便利性。

### 8.5 真实世界实验规模有限

- 真实实验只覆盖传送带分拣任务；
- 物体类别只有 toy dog 和 hairbrush；
- 每个条件 10 次 trial，统计样本较少；
- 尚未展示在更复杂真实场景中的效果，例如：
  - 多物体遮挡；
  - 长程任务；
  - 非刚性物体；
  - 双臂操作；
  - 灵巧手操作；
  - 移动平台操作。

### 8.6 与其他实时控制技术尚未结合

- 论文刻意不使用 asynchronous inference、real-time chunking 等延迟隐藏方法，以便隔离 FLASH 的贡献。
- 但实际部署中，这些技术常会同时使用。
- FLASH 在与 RTC、异步控制、轨迹平滑优化结合时的行为仍需进一步研究。
- 因为 FLASH 的接受前缀长度是动态变化的，会引入可变延迟和可变重规划周期，对控制稳定性提出新挑战。

### 8.7 安全性与失败处理仍较基础

- 文中提到真实机器人有底层 joint limits 和 command constraints，但没有系统性安全分析。
- 如果草稿动作通过验证但仍不安全，当前机制主要依赖低层控制约束。
- 对人机共处、高速机械臂、接触丰富任务，还需要更严格的安全验证。

---

## 总体评价

- Realtime-VLA FLASH 是一篇针对 dVLA 实时部署瓶颈的系统性工作。
- 论文最重要的贡献在于提出了一种适用于连续动作 Flow Matching 模型的推测式推理框架，并用并行端点重构解决了连续动作验证难题。
- 实验结果显示，该方法能在基本保持 LIBERO 成功率的同时显著降低平均推理延迟，并在真实高速传送带场景中带来实际收益。
- 不过，该方法仍依赖草稿模型训练、固定阈值、gripper-switch 启发式和任务特定调优；未来若能发展自适应验证、跨任务通用草稿模型以及更系统的安全机制，其应用价值会更强。
