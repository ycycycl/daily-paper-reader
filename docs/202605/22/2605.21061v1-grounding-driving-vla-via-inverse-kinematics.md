## 摘要
现有的驾驶 VLA 在预测轨迹时很大程度上忽略其视觉 token——我们将这一现象归因于任务表述在结构上的不适定，而非训练不足。我们表明，从逆运动学的视角来看，轨迹恢复需要当前视觉状态和未来视觉状态共同作为边界条件；而现有 VLA 仅提供前者，这促使模型仅通过自车状态和文本指令走捷径。为解决这一问题，我们以逆运动学求解器的形式重新设计驾驶 VLA。首先，引入下一视觉状态预测目标，要求 LLM 预测未来视觉场景，从而提供密集的视觉监督并抑制捷径路径。其次，设计一个独立的逆运动学网络（基于交叉注意力的条件扩散模型），其仅以当前和未来视觉状态为输入，以在轨迹解码过程中抑制对自车状态和文本捷径的依赖。仅凭这一简单方案，我们的 0.5B 规模模型便恢复了视觉 grounding，并在闭环 NAVSIM-v2 和 nuScenes 基准上达到与规模大一个数量级以上的 7B–8B VLA 相当的轨迹规划性能。广泛分析进一步表明，这一改进源于模型重新获得了利用视觉特征的能力，且在转弯等动态驾驶场景中效果最为显著。

## Abstract
Existing Driving VLAs predict trajectories while largely ignoring their visual tokens -- a phenomenon we trace not to insufficient training but to a structurally ill-posed task formulation. We show that trajectory recovery, when viewed through the lens of inverse kinematics, requires both a current and a future visual state as boundary conditions; existing VLAs supply only the former, which encourages the model to shortcut through ego status and text commands alone. To address this, we re-design Driving VLA in the style of an inverse kinematics solver. First, a next visual state prediction objective that requires the LLM to predict the future visual scene provides dense visual supervision and suppresses shortcut paths. Second, a separate Inverse Kinematics Network (a cross-attention-based conditional diffusion model) that takes only the current and future visual states as input is designed to suppress reliance on ego status and textual shortcuts during trajectory decoding. With this simple prescription alone, our 0.5B-scale model recovers visual grounding and reaches trajectory planning performance comparable to 7B--8B VLAs more than an order of magnitude larger, on both the closed-loop NAVSIM-v2 and the nuScenes benchmarks. Extensive analysis further shows that this improvement stems from a recovered ability to exploit visual features, with the effect being most pronounced in dynamic driving situations such as turning.

### 研究背景

本文关注自动驾驶中的 **Vision-Language-Action（VLA）模型**。这类模型通常输入：

- 多相机图像 / 视觉 token；
- 自车状态，如速度、加速度、航向角速度等；
- 文本指令，如“直行”“左转”“右转”；
- 输出未来若干秒的车辆轨迹。

Driving VLA 的目标是用一个统一的大模型替代传统自动驾驶中的感知、预测、规划多模块流水线，从而实现端到端驾驶决策。

### 核心问题

论文指出：**现有 Driving VLA 虽然结构上接收视觉输入，但在轨迹预测时往往并没有真正使用视觉信息**。

作者将这一现象称为 **blind planning（盲规划）**，即模型主要依赖：

- 自车状态；
- 历史轨迹；
- 文本导航命令；

而不是依赖当前图像中的车辆、行人、红绿灯、障碍物等关键视觉线索。

论文通过反事实障碍物拼接实验发现：在 nuScenes 验证集上，如果在自车前方合成一个近距离车辆障碍物，许多现有 VLA 的预测轨迹几乎不变，仍然保持直行；而本文方法会明显减速或避让。

### 作者的诊断

作者认为，这不是简单的训练不足，而是 **任务定义本身存在结构性缺陷**。

现有 Driving VLA 通常直接学习：

\[
\hat{T}_t = f_\theta(V_t, E_t, C_t)
\]

其中：

- \(V_t\)：当前视觉状态；
- \(E_t\)：自车状态；
- \(C_t\)：文本命令；
- \(T_t\)：未来轨迹。

问题在于，模型只给定了 **当前视觉状态**，却要求直接输出未来轨迹。作者从“逆运动学”的角度解释：若要恢复一段轨迹，通常需要起点和终点两个边界条件；但现有方法只有当前状态，没有未来视觉状态作为目标锚点，因此模型容易绕过视觉输入，用自车状态和文本命令拟合平均轨迹。

### 整体意义

本文的核心贡献在于指出：

> Driving VLA 的视觉失效不是偶然现象，而是由“只监督轨迹、不监督未来视觉状态”的任务设计导致的捷径学习。

因此，论文尝试通过重新设计任务边界条件和轨迹解码结构，让视觉信息成为轨迹生成中不可绕过的必要条件。

## 2. 方法论：用“下一视觉状态 + 逆运动学网络”恢复视觉 grounding

## 2.1 核心思想

论文将驾驶轨迹生成重新表述为一种“逆运动学”问题：

- 当前视觉状态 \(V_t\) 类似起始状态；
- 未来视觉状态 \(\hat{V}_{t+\Delta}\) 类似目标状态；
- 轨迹 \(T_t\) 则由这两个视觉状态之间的变化推导出来。

因此，模型不再直接从当前图像、自车状态和指令输出轨迹，而是分两步：

1. **LLM 预测未来视觉状态**；
2. **IK Network 仅根据当前视觉状态和预测的未来视觉状态生成轨迹**。

整体形式为：

\[
\hat{V}_{t+\Delta} = w_\phi(V_t, E_t, C_t)
\]

\[
\hat{T}_t = h_\psi(V_t, \hat{V}_{t+\Delta})
\]

其中 \(h_\psi\) 是本文提出的 Inverse Kinematics Network。

关键点是：**轨迹解码器不直接接收自车状态和文本命令**，从而减少模型走捷径的可能。

---

## 2.2 理论分析：为什么直接轨迹回归会忽视视觉

作者将未来轨迹分解为：

\[
T_t = T_0(E_t, C_t) + I_t \cdot \Delta(V_t, E_t, C_t) + \xi_t
\]

含义如下：

- \(T_0(E_t, C_t)\)：仅由自车状态和命令决定的主导行为，例如直行、保持车道、巡航；
- \(\Delta(V_t, E_t, C_t)\)：由视觉信息引起的修正，例如避障、让行、转弯；
- \(I_t\)：当前场景是否是视觉关键场景；
- \(\xi_t\)：数据噪声。

由于大多数驾驶数据是直行、跟车、巡航等非视觉关键场景，所以 \(I_t = 1\) 的比例 \(\epsilon\) 较小。

作者通过 bias-variance 分解说明，在非视觉关键场景中，如果模型输出随视觉输入变化，反而会被轨迹损失惩罚。因此在标准 MSE 或 token loss 下，最优捷径就是让模型的视觉残差趋近于 0，即忽略视觉。

论文进一步在 nuScenes 上估计：

- 若以 ego-status gap 划分视觉关键场景，视觉关键样本比例约为 10%–30%；
- 其余 70%–90% 样本都容易被自车状态解释。

这支持了作者的判断：**直接轨迹监督天然偏向 ego-status shortcut**。

---

## 2.3 下一视觉状态预测目标

为避免视觉 token 被忽略，作者引入 **Next Visual State Prediction（NSP）**。

具体做法：

1. 使用 DINOv3 ViT-S/16 从前视、左前、右前三个相机图像中提取视觉 token；
2. 经过 MLP projector 后输入 Qwen2.5-0.5B-Instruct；
3. LLM 不直接输出轨迹，而是在特殊 token 区间内输出未来视觉状态；
4. 未来视觉状态的监督目标由未来帧图像经过同一个视觉编码器和 projector 得到；
5. 使用 MSE 监督：

\[
L_{\text{visual}} = \| \text{LLM}(V_t) - V_{t+\Delta} \|^2
\]

这一目标的作用是：

- 让 LLM 必须预测未来视觉场景；
- 为视觉 token 提供密集监督；
- 即使在非视觉关键场景中，视觉输入也会持续收到梯度信号。

## 2.4 Inverse Kinematics Network

轨迹生成由独立的 **IK Network** 完成。

### 输入限制

IK Network 只接收：

- 当前视觉状态 \(Z_t\)；
- LLM 预测的未来视觉状态 \(\hat{Z}_{t+\Delta}\)。

它不接收：

- 自车状态 \(E_t\)；
- 文本命令 \(C_t\)；
- LLM 的完整隐藏上下文。

这一点是本文方法区别于普通 world model + policy 方法的关键。作者强调，未来状态不是辅助 loss，而是轨迹解码的必要条件。

### 网络结构

IK Network 是一个 **基于交叉注意力的条件扩散模型**。

流程大致如下：

1. 从随机噪声轨迹 \(x_T \sim \mathcal{N}(0, I)\) 开始；
2. 当前 noisy trajectory 作为 query；
3. 当前视觉 token \(Z_t\) 和预测未来视觉 token \(\hat{Z}_{t+\Delta}\) 作为 key/value；
4. 经过两层 cross-attention；
5. 使用 MLP 预测噪声；
6. 通过 DDPM 反向采样生成最终轨迹。

其形式可概括为：

\[
\epsilon_\theta(x_t, t, Z_t, \hat{Z}_{t+\Delta})
=
\text{MLP}(
\text{CrossAttn}(
\text{CrossAttn}(x_t + TE(t), Z_t),
\hat{Z}_{t+\Delta}
))
\]

训练目标是标准 DDPM 噪声预测损失：

\[
L_{\text{IK}}
=
\mathbb{E}
\left[
\|
\epsilon -
\epsilon_\theta(
\sqrt{\bar{\alpha}_t}x_0 +
\sqrt{1-\bar{\alpha}_t}\epsilon,
t,
Z_t,
\hat{Z}_{t+\Delta}
)
\|^2
\right]
\]

推理时使用 50 步 DDPM 采样。

## 6. 主要结论与发现

## 6.1 性能结果

### NAVSIM-v1

本文 0.5B 模型达到：

- PDMS：92.2。

相比 OpenDriveVLA\*：

- OpenDriveVLA\*：73.2；
- 提升 +19.0。

并超过或接近多种 2B–8B VLA，例如 DriveFine、DriveVLA-W0、ReCogDrive 等。

### NAVSIM-v2

本文模型达到：

- EPDMS：90.6。

相比 OpenDriveVLA\*：

- OpenDriveVLA\*：70.2；
- 提升 +20.4。

同时在 NC、TTC、LK 等安全指标上表现优良。

### nuScenes

本文 0.5B 模型在 open-loop planning 上取得：

- ST-P3 metrics 下平均 L2 / ADE：0.06 m；
- Collision Rate：0.09%。

相比 OpenDriveVLA-7B，其轨迹误差更低，碰撞率相近或更优。

---

## 6.2 视觉 grounding 发现

### 障碍物拼接实验

当在自车前方拼接车辆时：

- Impromptu-VLA 3B：平均端点变化约 0；
- OpenDriveVLA 0.5B：平均端点缩短 -0.06 m；
- Alpamayo-R1 10B：-0.12 m；
- 本文模型：-1.04 m。

说明本文模型对视觉障碍物有明显反应。

### GradCAM 分析

本文模型在 object mask 上的 saliency overlap 更高：

- Pointing Mass：0.268；
- Average Precision：0.281；
- IoU@top20：0.145；
- F1@top20：0.236。

均高于 OpenDriveVLA、Alpamayo-R1 和 Impromptu-VLA。

这说明模型关注区域更集中在车辆、行人等规划相关对象上。

---

## 6.3 消融结论

消融结果显示：

- 完整模型 ADE：0.06；
- 去掉下一状态预测、改为当前状态重建：0.07；
- 去掉 IK Network、直接 MLP 预测轨迹：0.31；
- 去掉 diffusion IK、改为 cross-attention pooling + MLP：0.16。

说明：

1. IK Network 的视觉瓶颈是性能提升的核心；
2. 下一状态预测在平均指标上提升不大，但在转弯、动态场景中更重要；
3. diffusion 形式的 IK 解码比确定性 MLP 更适合轨迹生成。

---

## 6.4 额外发现

论文还发现：

- 下一状态预测质量与轨迹误差的逐样本相关性较弱，说明模型不需要重建完整未来视觉状态，而是学习了轨迹相关的视觉子空间；
- 在 ego-status 难以解释的 VisionCritical 子集上，本文模型相比 baseline 提升更显著；
- 在 nu-Caption 和 NuX 上，本文方法也提升了高层场景理解能力；
- 在 nuScenes-QA 上略低于 OpenDriveVLA，尤其在 Counting、Object、Status 等实例级问题上较弱。

## 7. 优点与亮点

## 7.1 问题诊断有深度

论文没有仅仅追求指标提升，而是系统分析了现有 Driving VLA 为什么会忽视视觉输入，并用 bias-variance 分解解释了视觉 shortcut 形成机制。

这一点使论文具有较强的方法论意义。

## 7.2 方法设计简洁有效

核心改动只有两个：

- 下一视觉状态预测；
- 只依赖当前 / 未来视觉状态的 IK Network。

但这两个改动直接针对捷径学习的根因：

- 前者提供视觉监督；
- 后者阻断 ego-status / text command 直接进入轨迹解码器。

## 7.3 小模型达到大模型水平

本文使用 0.5B LLM，却在多个规划 benchmark 上接近甚至超过 7B–8B VLA。

这说明在自动驾驶规划中，结构设计可能比单纯扩大 LLM 参数更关键。

## 7.4 实验验证多维度

论文不仅报告 benchmark 分数，还通过以下方式验证视觉 grounding：

- counterfactual stitching；
- GradCAM-object overlap；
- 位置与尺寸控制实验；
- ego-status difficulty 分层评估；
- 消融实验；
- QA / caption 任务辅助评估。

这些实验增强了结论可信度。

## 7.5 对部署友好

推理延迟约 344 ms，虽然仍不算极低，但相比许多 Driving VLA 的 1–2 秒级延迟更具实用性。

## 8. 不足与局限

## 8.1 训练算力信息不完整

论文没有明确说明训练所用 GPU 数量、型号、训练时长和总计算成本。这会影响复现和成本评估。

## 8.2 长期闭环验证不足

实验主要集中在：

- NAVSIM；
- nuScenes open-loop；
- 短期闭环或近闭环指标。

论文没有在更复杂的长期闭环环境如 Bench2Drive 上充分验证。因此模型在长时序、多交互、复杂交通博弈中的稳定性仍不清楚。

## 8.3 对实例级感知能力仍有限

由于本文使用 DINOv3 perspective-view 特征，而非 UniAD 这类 BEV 预训练实例感知编码器，因此在 nuScenes-QA 中：

- Counting；
- Object；
- Status；

等需要细粒度实例识别的类别上略弱于 OpenDriveVLA。

这说明该方法更偏向规划相关视觉 grounding，而不是完整精细感知。

## 8.4 反事实拼接仍有偏差风险

虽然作者做了 road vs sky、size vs position 的控制实验，但 stitched sprite 仍然可能引入分布外视觉伪影。

论文也承认：

- 反应强度仍受 sprite size 影响；
- 部分天空区域拼接仍可能引起非零响应；
- 视觉 grounding 的位置精度还有提升空间。

## 8.5 “逆运动学”更多是类比而非严格车辆动力学

作者明确说明本文的 inverse kinematics 是风格化表述，并非严格车辆动力学建模。

因此，该方法本质上仍是学习式轨迹生成，而不是物理约束完整可解释的运动学求解器。

## 8.6 未来视觉状态监督依赖未来帧

训练时需要未来图像经过视觉编码器得到 \(V_{t+\Delta}\) 作为监督。对于已有数据集这不是问题，但在某些在线或弱监督数据场景中，未来状态标注 / 对齐可能增加数据处理复杂度。

---

# 总体评价

本文是一篇针对 Driving VLA 视觉 grounding 问题的高质量工作。它的核心价值不只是提出一个新模型，而是指出现有 VLA 轨迹预测任务存在结构性捷径：当模型只被要求输出轨迹而不被要求预测未来视觉状态时，视觉 token 很容易被损失函数边缘化。

通过引入下一视觉状态预测和视觉-only 的 IK diffusion decoder，论文有效恢复了模型对视觉信息的使用，使 0.5B 小模型在 NAVSIM 和 nuScenes 上达到接近甚至超过大规模 VLA 的规划性能。实验分析较全面，尤其是反事实拼接和 GradCAM 证明了性能提升确实与视觉 grounding 有关。

其主要不足在于训练算力信息不透明、长期闭环验证有限、实例级感知能力略弱，以及方法中的“逆运动学”仍主要是结构性类比而非严格物理建模。总体来看，这篇论文对自动驾驶 VLA 的任务设计和结构约束具有较强启发意义。
