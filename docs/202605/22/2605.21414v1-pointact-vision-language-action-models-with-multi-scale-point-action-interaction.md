## Abstract
Vision-Language-Action (VLA) models have shown strong potential for general-purpose robotic manipulation by leveraging large pretrained vision-language backbones. However, most existing VLAs rely primarily on 2D visual representations, which limit their ability to reason about fine-grained geometry and spatial grounding - capabilities that are essential for precise and robust manipulation in 3D environments. In this paper, we propose PointACT, a dual-system 3D-aware VLA policy that integrates hierarchical 3D point cloud representations directly into the action decoding process. PointACT employs a multi-scale point-action interaction mechanism with efficient bottleneck window self-attention, enabling evolving action tokens to densely attend to both local geometric detail and global scene structure. We evaluate PointACT on the LIBERO and RLBench benchmarks and systematically compare it against monolithic and dual-system VLA baselines, including variants augmented with point cloud inputs. PointACT achieves consistent improvements across both benchmarks, increasing success rates by 10% on the challenging RLBench-10Tasks suite over state-of-the-art pretrained VLAs, with even larger gains when the vision-language backbone is frozen and the action expert is trained from scratch. Extensive ablation studies demonstrate that tightly coupling hierarchical 3D geometry with pretrained 2D semantic representations is critical for robust and spatially grounded robot control. Our results also highlight the promise of pretrained 3D representations for 3D-aware VLA policies.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 整体框架

- 本文提出 **PointACT**，一种双系统 3D-aware VLA 架构。
- 它包含两个主要部分：
  1. **冻结的预训练 VLM backbone**  
     - 负责处理语言指令和 RGB 图像；
     - 提供高层语义、视觉和任务上下文表示。
  2. **点云动作专家 Point-Action Expert**  
     - 负责处理 3D 点云、机器人状态和动作 token；
     - 在动作解码过程中引入多尺度点云几何信息。

- 与直接把点云 token 塞进 VLM 的单体式方法不同，PointACT 将点云主要放入 **动作专家** 中处理，从而避免破坏 VLM 的预训练语义表示。

## 2.2 问题形式化

在每个时间步 \(t\)，机器人接收多模态观测：

\[
O_t = \{I_t, P_t, s_t, L\}
\]

其中：

- \(I_t\)：多视角 RGB 图像；
- \(P_t \in \mathbb{R}^{N_P \times 6}\)：彩色 3D 点云，每个点包含 3D 坐标与 RGB 信息；
- \(s_t\)：机器人本体状态，例如关节状态、末端执行器状态等；
- \(L\)：自然语言指令。

目标是学习策略：

\[
A_t = \pi_\theta(I_t, P_t, s_t, L)
\]

其中 \(A_t = (a_t, ..., a_{t+H-1})\) 是未来 \(H\) 步动作序列。

## 2.3 点云编码器：Point Transformer v3

- PointACT 使用 **Point Transformer v3, PTv3** 作为点云编码器。
- PTv3 的特点：
  - 直接处理无序点云；
  - 使用序列化分组策略构建局部邻域；
  - 在局部窗口内进行 self-attention；
  - 具有层级结构，可输出多尺度点云特征。
- 点云编码器会在不同层产生多尺度点特征：

\[
Z_p^l \in \mathbb{R}^{N_p^l \times D_l}
\]

其中：
- \(l\) 表示 PTv3 的不同层级；
- 早期层点数多，保留局部细节；
- 深层点数少，包含更全局的场景结构。

- 作者使用在大规模 3D 场景上自监督预训练的 PTv3 初始化点云编码器，尽管预训练数据与机器人操作场景存在域差异，但仍能提供有用的几何先验。

## 2.4 多尺度点—动作交互机制

PointACT 的核心技术是 **Multi-Scale Point-Action Interaction**。

### 基本思想

- 动作 token 在生成过程中不仅要关注 VLM 输出的语言与图像语义，还要直接关注 3D 点云中的局部几何与全局结构。
- 因此，PointACT 让动作 token 与 PTv3 不同层级的点云 token 反复交互。
- 这种设计使动作 token 在每一层都能被 3D 几何信息更新。

---

## 2.5 Bottleneck Window Self-Attention

由于点云 token 数量较大，直接让所有点之间进行 dense self-attention 计算代价很高。为此，本文设计了 **瓶颈窗口自注意力机制**。

### 具体流程

1. 将点云划分为 \(K\) 个空间窗口：

\[
(W_1, W_2, ..., W_K)
\]

2. 对于每个窗口 \(W_k\)，将该窗口的点云 token 与动作 token 拼接：

\[
X_k^l = [Z_{p}^{l,W_k}; Z_a^l]
\]

其中：
- \(Z_{p}^{l,W_k}\)：第 \(l\) 层第 \(k\) 个窗口内的点特征；
- \(Z_a^l\)：当前层动作 token 表示。

3. 在每个窗口内应用共享 self-attention：

\[
\hat{X}_k^l = \text{Self-Attn}(X_k^l)
\]

4. 从每个窗口中取出更新后的动作 token，并在所有窗口上平均池化：

\[
\hat{Z}_a^l = \frac{1}{K} \sum_{k=1}^{K} \hat{X}_{k,a}^l
\]

5. 更新后的动作 token 再与 VLM 输出表示进行 cross-attention：

\[
\bar{Z}_a^l = \text{Cross-Attn}(\hat{Z}_a^l, Z_{vlm})
\]

### 机制优势

- 动作 token 作为 bottleneck，汇聚多个局部窗口的几何信息。
- 避免点云 token 之间全局 dense attention 的高计算成本。
- 同时建模：
  - 局部几何细节；
  - 全局空间结构；
  - 语言和图像语义上下文。
- 通过多层级重复该过程，实现多尺度几何融合。

## 2.6 动作预测头

PointACT 根据动作空间采用两类动作头。

### 回归头

用于连续控制和短时动作 chunk，例如 LIBERO 中的 delta end-effector pose。

- 使用 MLP 将动作 token 映射为连续动作序列。
- 使用 L2 损失：

\[
L_{reg} = \frac{1}{H}\sum_{i=t}^{t+H-1} ||a_i - a_i^*||_2^2
\]

### 分类头

用于关键点动作预测，例如 RLBench 中的长时程 keypoint pose。

- 将目标位置相对于最近点云点进行离散化；
- 旋转也离散为多个 bin；
- 将最终点云特征与动作 token 拼接后预测；
- 使用交叉熵损失：

\[
L_{cls} = - \sum_{k \in \{pos, rot, open\}} \sum_{b=1}^{B_k} y_{k,b}\log(\hat{y}_{k,b})
\]

该设计将点云作为空间锚点，有利于关键点定位。

## 6. 主要结论与发现

- **显式 3D 几何对 VLA 有明显帮助**：
  - PointACT 在 LIBERO 和 RLBench 上均带来稳定提升。
  - 在 RLBench-10Tasks 上，PointACT 达到 **82.3%** 平均成功率，高于 EO1 的 **73.2%** 和 HybridVLA 的 **74%**。

- **将点云直接注入 VLM backbone 并不理想**：
  - EO1 + Point 在 LIBERO-Spatial 上略有提升；
  - 但在 RLBench 上表现大幅下降到 **18.6%**；
  - 说明直接加入点云 token 可能扰乱 VLM 的预训练表示。

- **将点云放入动作专家更有效**：
  - GR00T(arch) 从 50.8% 提升到 69.7%；
  - PointACT 进一步提升到 82.3%。

- **细粒度多尺度点—动作交互优于粗粒度点云特征融合**：
  - 简单使用最终层或多尺度拼接点特征效果不如 PointACT；
  - 说明关键不只是“有没有点云”，而是“点云如何参与动作 token 的生成”。

- **2D 图像语义仍然重要**：
  - 去掉图像条件后，性能下降：
    - LIBERO-Spatial：97.4% → 94.2%；
    - RLBench：82.3% → 79.8%。
  - 表明 3D 几何和 2D 语义是互补关系。

- **预训练 3D 表征有帮助，但受模型规模和域差异影响**：
  - PTv3 预训练对大模型更有帮助；
  - 小模型中收益不明显；
  - 说明未来需要更贴近机器人操作场景的 3D 预训练数据。

- **真实机器人实验显示 PointACT 具备一定实际可用性**：
  - 在 SO-100 上整体优于 π0 和 GR00T-N1.5；
  - 在 UR5 上优于 2D VLA，在部分任务中接近或超过 3DLotus。

## 7. 优点与亮点

- **问题切入清晰**：
  - 直接针对当前 VLA 的 2D-centric 局限；
  - 强调机器人操作中的 3D 几何需求，动机充分。

- **架构设计合理**：
  - 冻结 VLM，保留预训练语义能力；
  - 将点云处理放入动作专家，降低破坏 VLM 表示的风险；
  - 体现了 dual-system VLA 的优势。

- **多尺度点—动作交互是核心创新**：
  - 不是简单拼接点云特征；
  - 而是在动作解码过程中持续融合局部与全局几何信息；
  - 对空间定位和精细操作更有针对性。

- **Bottleneck Window Self-Attention 提高效率**：
  - 动作 token 作为瓶颈汇聚局部窗口几何；
  - 避免点云全局 attention 的高成本；
  - 在性能和计算之间取得较好平衡。

- **实验较系统**：
  - 包括仿真、真实机器人、主实验和大量消融；
  - 对比了 monolithic、dual-system 和 PointACT；
  - 能较好支撑方法设计选择。

- **兼容不同动作形式**：
  - 可用于 LIBERO 的连续 delta action；
  - 也可用于 RLBench 的 keypoint action；
  - 方法适用性较广。

- **真实机器人验证增强可信度**：
  - 在 SO-100 和 UR5 两个平台上验证；
  - 覆盖微步控制和关键点控制两类设置。

---

## 8. 不足与局限

- **仍依赖较高质量点云输入**：
  - 在真实机器人中，透明物体、反光物体、遮挡等会导致点云噪声或缺失；
  - 例如 UR5 的 Close Drawer 中，透明抽屉导致点云质量下降，影响 3D 方法表现。

- **单视角设置存在遮挡问题**：
  - 仿真和真实实验多使用前视 RGB-D；
  - Close Fridge 等任务中失败常由部分可见或遮挡导致；
  - 未来可能需要多视角融合或主动感知。

- **缺乏强闭环恢复能力**：
  - 论文失败案例显示，一旦初始动作导致碰撞、物体偏移或抓取失败，模型往往不能有效恢复；
  - 说明 PointACT 更偏向精确初始预测，而非长期闭环纠错。

- **工具使用和间接操作仍困难**：
  - Sweep to Dustpan 和 Water Plants 中，模型能预测抓取点，但难以准确处理工具与目标之间的二级空间约束；
  - 说明方法对复杂物理交互和工具动力学建模仍不足。

- **跨论文比较存在不完全公平性**：
  - 不同 VLA baseline 的预训练数据、相机数量、训练任务数、评估 episode 数不同；
  - 部分结果直接引用其他论文，难以完全控制变量。

- **真实机器人实验规模较小**：
  - 每个真实任务只测试 10 次；
  - 任务数量有限；
  - 缺少更大规模、多环境、多物体、多机器人验证。

- **3D 预训练仍存在域差异**：
  - PTv3 预训练来自建筑级 3D 场景，与桌面操作差异较大；
  - 虽然有收益，但不是最理想的机器人操作几何预训练。

- **模型仍较大**：
  - 整体 VLA 使用 3B 级 VLM；
  - PointACT 动作专家约 300M 参数；
  - 对低算力机器人边缘部署仍有挑战。

# 总体评价

PointACT 是一篇围绕 **3D 感知 VLA** 的系统性工作。它的核心贡献不只是“给 VLA 加点云”，而是明确指出点云应当在动作解码阶段与动作 token 进行细粒度、多尺度交互。实验结果表明，这种设计相比直接注入 VLM 或使用粗粒度点云特征更有效，尤其在需要空间定位和几何精度的 RLBench 与真实机器人任务中表现突出。论文实验较完整，消融充分，结论有较强说服力。不过，方法仍受点云质量、单视角遮挡、闭环恢复不足和真实实验规模有限等因素限制。总体而言，该工作为构建更具空间理解能力的机器人 VLA 策略提供了有价值的架构方向。
