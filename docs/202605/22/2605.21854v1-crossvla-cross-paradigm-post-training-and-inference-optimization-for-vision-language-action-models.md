## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已迅速收敛到少数几类架构模式：离散 token 自回归（如 OpenVLA）和连续动作流匹配（如 pi-0.5）。然而，通过直接偏好优化（Direct Preference Optimisation, DPO）进行偏好对齐——语言模型中事实上的后训练步骤——几乎只在自回归 VLA 上得到研究。我们提出 CrossVLA，这是一项关于跨范式 VLA 后训练的实证研究。本文有三项贡献：（i）提出一种替代性的流匹配对数概率估计器，使 DPO 能够在连续动作骨干网络上运行，而无需进行概率流 ODE 积分；（ii）对 LoRA 与 DoRA 作为 VLA DPO 的参数高效层进行正面对比，发现 DoRA 相较于 OpenVLA SFT 在 LIBERO 4-suite 上平均提升 +10.4 个百分点（600 次试验，3 个随机种子）——各套件分别为 Object +20.0、Long-horizon +11.0、Goal +8.0、Spatial +2.7——并且在 Object 上种子方差为零（3 个种子均为 38/50）；（iii）进行推理时剖析，表明去噪循环占 sample_actions 延迟的 78.6%，而类似 VLA-Cache 的 prefix-K/V 缓存加速上限为 21%——在我们的基准中，无论是块级还是 token 级缓存策略都会使成功率下降至 0–80%。此外，我们在 6000 帧 LIBERO 数据上预训练了一个多视角 + 时间投影头，在同任务检索中实现了 99.5% 的 k-NN recall@1（为随机水平的 36 倍），可作为下游初始化使用。所有代码、检查点、训练日志和复现实脚本均已开源于 https://github.com/lz-googlefycy/vla-lab。

## Abstract
Vision-Language-Action (VLA) models have rapidly converged on a small set of architectural patterns: discrete-token autoregression (e.g. OpenVLA) and continuous-action flow-matching (e.g. pi-0.5). Yet preference alignment via Direct Preference Optimisation (DPO) -- the de-facto post-training step in language models -- has been studied almost exclusively on autoregressive VLAs. We present CrossVLA, an empirical study of cross-paradigm VLA post-training. Three contributions: (i) a surrogate flow-matching log-probability estimator that lets DPO operate on continuous-action backbones without probability-flow ODE integration; (ii) a head-to-head comparison of LoRA and DoRA as the parameter-efficient layer for VLA DPO, finding DoRA improves over OpenVLA SFT by a mean +10.4 pp across LIBERO 4-suite (600 trials, 3 seeds) -- per-suite +20.0 Object, +11.0 Long-horizon, +8.0 Goal, +2.7 Spatial -- with zero seed variance on Object (38/50 on each of 3 seeds); (iii) an inference-time anatomy showing the denoise loop dominates 78.6% of sample_actions latency and prefix-K/V caching a la VLA-Cache caps at a 21% acceleration ceiling -- both chunk-level and token-level cache strategies degrade success rate to 0-80% in our benchmarks. We further pretrain a multi-view + temporal projection head on 6000 LIBERO frames, achieving 99.5% k-NN recall@1 for same-task retrieval (36x over random), available as a downstream initialisation. All code, ckpts, training logs, and reproduction scripts are open at https://github.com/lz-googlefycy/vla-lab.

## 2. 方法论：核心思想与关键技术细节

### 2.1 跨范式 VLA 接口

作者首先抽象出一个统一接口，使不同架构的 VLA 都能接入同一套偏好优化流程。

接口包含以下能力：

- `policy_logp(batch, chunk)`：计算当前策略对动作块的 log-probability。
- `policy_logp_with_ref(batch, chunk)`：同时计算当前策略和 reference 策略的 log-probability。
- `policy_sample(batch, K)`：采样多个动作候选。
- `encode_obs(obs)`：编码视觉、语言、状态等观测输入。
- `sample_actions(obs, num_steps)`：环境交互时生成动作。

对应到不同模型：

- **OpenVLA**：
  - 动作被离散化为 token。
  - log-probability 可以通过 teacher forcing 下 token log-probability 求和获得。

- **π0.5**：
  - 动作为连续动作块。
  - 无法直接得到闭式 log-probability。
  - 因此需要本文提出的代理 log-probability。

### 2.2 面向流匹配 VLA 的代理 log-probability

DPO 的核心需要比较：

- 当前策略对 chosen action 的 log-probability；
- 当前策略对 rejected action 的 log-probability；
- reference 策略对二者的 log-probability。

但对于 flow-matching VLA，如果严格计算动作块 likelihood，需要进行 probability-flow ODE 积分，并涉及昂贵的 Jacobian determinant，计算代价过高。

因此作者提出一种 **基于 flow-matching MSE 的代理 log-probability**。

#### 核心构造

给定：

- 目标动作块：\(x_1\)
- 噪声起点：\(x_0\)
- 中间状态：
  \[
  x_t = (1-t)x_0 + tx_1
  \]
- 目标速度：
  \[
  v_{\text{target}} = x_1 - x_0
  \]
- 模型预测速度：
  \[
  v_\theta(x_t, t, obs)
  \]

作者定义代理 log-probability 为负的速度预测均方误差：

\[
\widetilde{\log p_\theta}(x_1|obs)
=
-\frac{1}{|T_{\text{eval}}|}
\sum_{t \in T_{\text{eval}}}
\|v_\theta(x_t,t,obs)-v_{\text{target}}\|^2
\]

其中：

- \(T_{\text{eval}}\) 取 4 个时间点：
  - 0.125
  - 0.375
  - 0.625
  - 0.875
- 每个时间步加入一次随机扰动。
- 噪声种子固定，以保证 reference forward 可复现。

#### 直观解释

- 如果模型能准确预测从噪声到真实动作的速度场，说明该动作块在当前策略下更“合理”。
- 因此用 **负 MSE** 近似 log-probability。
- 理论上，扩散模型中的 MSE 与 likelihood lower bound 有联系；作者将其中的比例因子吸收到 DPO 温度参数 \(\beta\) 中。

---

### 2.3 DPO 损失

作者使用标准 DPO 目标：

\[
L_{\text{DPO}}(\theta)
=
-\mathbb{E}_{(obs,c^+,c^-)}
\log \sigma(\beta \cdot \Delta_\theta(c^+,c^-))
\]

其中：

\[
\Delta_\theta(c^+, c^-)
=
(\widetilde{\log p_\theta}(c^+)-\widetilde{\log p_{\text{ref}}}(c^+))
-
(\widetilde{\log p_\theta}(c^-)-\widetilde{\log p_{\text{ref}}}(c^-))
\]

含义是：

- \(c^+\)：偏好动作块 / chosen chunk；
- \(c^-\)：非偏好动作块 / rejected chunk；
- reference model 是冻结的 SFT 基座模型；
- 当前模型应相对于 reference 更偏好 chosen 而非 rejected。

主要超参数：

- \(\beta = 0.1\)
- 学习率：\(5 \times 10^{-5}\)
- batch size：1
- 最大训练步数：500
- warmup：100 步

---

### 2.4 LoRA 与 DoRA 的参数高效微调比较

论文比较了两种 PEFT 方法：

#### LoRA

LoRA 将权重更新写成低秩形式：

\[
\Delta W = \frac{\alpha}{r}BA
\]

其中：

- \(r\) 是 rank；
- \(A, B\) 是低秩可训练矩阵；
- 原始权重冻结。

#### DoRA

DoRA 在 LoRA 的基础上进一步将权重分解为：

- magnitude，幅值；
- direction，方向。

其核心思想是：

- LoRA 同时改变权重的方向和幅值；
- DoRA 显式解耦二者，使模型可以更稳定地调整幅值，同时尽量保持预训练模型已有的视觉-语言 grounding 方向。

论文认为这对 VLA 尤其合适，因为：

- LIBERO 这类任务往往是相对窄分布的下游适配；
- 不一定需要大幅改变语义方向；
- 更需要对已有能力做精细重标定。

### 2.5 多视角 + 时间对比预训练

作者还探索了一个辅助方向：在冻结 SigLIP 视觉编码器之上训练一个小型投影头。

架构为：

- 输入图像：224×224×3
- 冻结的 SigLIP-so400m 编码器
- 提取 1152 维 CLS feature
- 投影头：
  - Linear(1152, 512)
  - GELU
  - Linear(512, 128)
  - L2 normalize

训练目标是双流 InfoNCE：

\[
L = w_{\text{mva}}L_{\text{mva}} + w_{\text{tc}}L_{\text{tc}}
\]

其中：

- \(L_{\text{mva}}\)：多视角对比，即同一时间步的 agent-view 与 wrist-view 应接近；
- \(L_{\text{tc}}\)：时间对比，即 agent-view 在 \(t\) 与 \(t+\Delta\) 的表示应接近，\(\Delta=5\)；
- \(w_{\text{mva}} = w_{\text{tc}} = 0.5\)
- 温度 \(\tau=0.07\)
- batch size = 128

该模块仅更新投影头，SigLIP 保持冻结。

    - 尚未验证其对下游 VLA 控制成功率的提升。
  - 所有实验都在模拟器 LIBERO 中完成：
    - 没有真实机器人实验。

### 5.3 公平性与客观性

- 论文较客观地报告了负结果：
  - prefix-K/V cache 在流匹配 VLA 上失败；
  - chunk-level cache 反而变慢且降低成功率。
- 作者也明确列出了实验不完整和非对称之处：
  - LoRA multiseed 不完整；
  - π0.5 Goal / Long-horizon 未完成；
  - 无真实机器人验证。
- 但 OpenVLA SFT baseline 低于原论文部分数字，作者认为与 decoding setting 有关，这可能影响绝对提升幅度的解释。

## 6. 主要结论与发现

### 6.1 DPO 可以跨范式迁移，但需要合适的 log-probability 代理

- 对自回归 VLA，DPO 可直接使用 token log-probability。
- 对 flow-matching VLA，本文提出的负 MSE 代理 log-probability 能让 DPO 训练稳定运行。
- 在 π0.5 上，DPO loss 能下降，chosen/rejected margin 能变正，没有出现退化。
- 不过由于 π0.5 在 Spatial / Object 上 SFT 已几乎满分，性能提升空间有限。

### 6.2 DoRA + DPO 显著提升 OpenVLA

相较 OpenVLA SFT，DoRA + DPO 在 LIBERO 4-suite 上平均提升：

- **+10.4 percentage points**

分 suite 结果为：

| Suite | OpenVLA SFT | DoRA + DPO | 提升 |
|---|---:|---:|---:|
| Spatial | 72% | 74.7% | +2.7 pp |
| Object | 56% | 76.0% | +20.0 pp |
| Goal | 70% | 78.0% | +8.0 pp |
| Long-horizon | 53% | 64.0% | +11.0 pp |
| Mean | 62.75% | 73.2% | +10.4 pp |

尤其是 Object suite：

- 三个 seed 均为 38/50；
- 总计 114/150；
- seed 方差为零。

作者认为这说明 DoRA 对窄分布适配更稳定。

### 6.3 prefix-K/V cache 不适合流匹配 VLA

π0.5 的 sample_actions 延迟分解为：

| 阶段 | 时间 | 占比 |
|---|---:|---:|
| 图像预处理 + tokenize | 约 5 ms | 1.8% |
| prefix forward | 约 60 ms | 21.4% |
| 10-step denoise loop | 约 220 ms | 78.6% |
| 总计 | 约 280 ms | 100% |

结论：

- VLA-Cache 主要优化 prefix forward；
- 但在 π0.5 中主要瓶颈是 denoise loop；
- 因此 prefix cache 的理论加速上限约为 21%；
- 实际上还可能损害成功率。

缓存实验结果：

- chunk-level cache：
  - reuse rate 约 82%；
  - 但 wall time 从 1258s 增至 1796s；
  - 成功率从 100% 降至 80%。

- token-level prefix cache：
  - cache hit 能发生；
  - 但 stale prefix K/V 会破坏后续 attention；
  - 在测试设置中任务成功率可降到 0%。

### 6.4 多视角 + 时间对比预训练得到强检索表示

训练结果：

- 总 InfoNCE loss 从 2.418 降到 0.366；
- 相对于随机 baseline \(\log B = 4.852\)，恢复约 92.5%。

k-NN retrieval：

- same-task recall@1 达 99.5%；
- 是随机水平的 36 倍；
- Object suite recall@1 达 100%。

但该 projection head 尚未用于实际 VLA 下游控制微调验证。

## 7. 优点与亮点

### 7.1 问题选择具有现实意义

- 论文关注 VLA 后训练和推理优化，而不仅是模型架构本身。
- 这些问题对机器人部署非常关键：
  - 如何用偏好数据提升策略；
  - 如何降低推理延迟；
  - 如何稳定适配下游任务。

### 7.2 跨范式视角清晰

- 同时研究自回归 VLA 和 flow-matching VLA。
- 明确区分了哪些方法可迁移，哪些方法不可迁移：
  - DPO 可迁移，但需要 log-probability 代理；
  - prefix KV cache 不适合流匹配 VLA。

### 7.3 代理 log-probability 简洁实用

- 使用 flow-matching 训练损失的负 MSE 作为 surrogate logp。
- 避免了昂贵的 probability-flow ODE likelihood 计算。
- 可以直接接入标准 DPO loss。

### 7.4 DoRA 实验结果有较强实证价值

- DoRA 在 VLA 场景中此前研究较少。
- 本文显示 DoRA 在 OpenVLA DPO 中提升明显，且在 Object suite 上 seed 稳定性非常好。
- 这对 VLA 参数高效微调方法选择有参考价值。

### 7.5 负结果有价值

- 论文没有只报告正向结果，而是系统分析了 prefix cache 在 flow-matching VLA 上失败的原因。
- 延迟剖析指出真正瓶颈是 denoise loop，这为后续工作提供明确方向：
  - consistency distillation；
  - one-step / few-step action generation；
  - denoise loop 压缩。

### 7.6 开源与可复现性较好

- 作者声称开源：
  - 代码；
  - checkpoint；
  - 训练日志；
  - 复现实脚本。
- 对实证论文而言，这是重要优点。

## 8. 不足与局限

### 8.1 π0.5 上的 DPO 验证不充分

- π0.5 只完成 Spatial 和 Object 两个 suite。
- 这两个 suite 的 SFT baseline 已分别达到 100% 和 98%，几乎没有提升空间。
- 更能说明代理 logp 有效性的 Goal / Long-horizon 没有完成。
- 因此“DPO 可用于 flow-matching VLA”的结论目前更多是训练稳定性验证，而非充分性能提升验证。

### 8.2 LoRA 与 DoRA 对比存在不对称

- DoRA 是 3-seed pool；
- LoRA multiseed 只覆盖 Object、Goal、Long-horizon；
- Spatial 只有单 seed。
- 因此 DoRA 对 LoRA 的全面优势还不能完全确认。
- 论文主结论中“DoRA 优于 SFT”较可靠，但“DoRA 明显优于 LoRA”证据相对较弱。

### 8.3 OpenVLA SFT baseline 偏低

- 作者复现的 OpenVLA SFT 在 Spatial / Object / Goal 上低于 OpenVLA 原论文结果。
- 作者认为原因是 decoding setting 不同。
- 这会影响 DoRA 相对 SFT 提升幅度的解释：
  - 如果 baseline 调优更好，提升是否仍然这么大，需要进一步验证。

### 8.4 缺少真实机器人验证

- 所有结果均来自 LIBERO 模拟环境。
- VLA 的真实部署会面临：
  - 视觉域偏移；
  - 动作噪声；
  - 物理接触不确定性；
  - 传感器延迟；
  - 安全约束。
- 因此仿真成功率不能直接等价于真实机器人性能。

### 8.5 多视角预训练只做了 intrinsic evaluation

- k-NN recall 很高，但这只能说明表示具有任务区分能力。
- 尚未证明该 projection head 能提升：
  - DPO 微调；
  - SFT 微调；
  - 控制成功率；
  - 跨任务泛化。
- 因此该部分目前更像是一个有潜力的附加模块，而非完整验证的下游方法。

### 8.6 推理优化只验证了缓存失败，没有实现主要推荐方案

- 论文指出 denoise-loop-targeting acceleration 是正确方向。
- 但本文自身没有实现 consistency distillation 或 one-step flow matching。
- 该建议主要由结构分析和并行工作 SnapFlow 支持。

### 8.7 数据规模和偏好数据构造较小

- 每个 suite 约 200 对 chosen/rejected chunk。
- 虽然实验显示有效，但偏好数据规模较小。
- 偏好对是否足够多样、是否存在 rollout 采样偏差，仍值得进一步分析。
