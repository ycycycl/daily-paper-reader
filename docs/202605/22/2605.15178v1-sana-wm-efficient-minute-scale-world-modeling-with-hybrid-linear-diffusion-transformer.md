## 摘要
我们提出 SANA-WM，这是一个高效的 26 亿参数开源世界模型，原生针对一分钟生成进行训练，能够合成高保真、720p、分钟级视频，并实现精确的相机控制。SANA-WM 的视觉质量可与 LingBot-World 和 HY-WorldPlay 等大规模工业基线相媲美，同时显著提升了效率。我们的架构由四项核心设计驱动：（1）混合线性注意力将逐帧 Gated DeltaNet（GDN）与 softmax 注意力相结合，以实现内存高效的长上下文建模。（2）双分支相机控制确保精确遵循 6-DoF 轨迹。（3）两阶段生成流水线对第一阶段输出应用长视频精修器，从而提升序列整体的质量与一致性。（4）鲁棒标注流水线从公开视频中提取准确的公制尺度 6-DoF 相机位姿，以获得高质量、时空一致的动作标签。在这些设计的推动下，SANA-WM 在数据、训练计算和推理硬件方面展现出显著效率：它仅使用约 21.3 万个带有公制尺度位姿监督的公开视频片段，在 64 块 H100 上 15 天即可完成训练，并可在单张 GPU 上生成每个 60 秒片段；其蒸馏版本通过 NVFP4 量化可部署在单张 RTX 5090 上，并在 34 秒内完成一个 60 秒 720p 片段的去噪。在我们的一分钟世界模型基准上，SANA-WM 相比既有开源基线表现出更强的动作跟随准确性，并在吞吐量提高 36 倍的情况下实现了相当的视觉质量，从而支持可扩展的世界建模。

## Abstract
We introduce SANA-WM, an efficient 2.6B-parameter open-source world model natively trained for one-minute generation, synthesizing high-fidelity, 720p, minute-scale videos with precise camera control. SANA-WM achieves visual quality comparable to large-scale industrial baselines such as LingBot-World and HY-WorldPlay, while significantly improving efficiency. Four core designs drive our architecture: (1) Hybrid Linear Attention combines frame-wise Gated DeltaNet (GDN) with softmax attention for memory-efficient long-context modeling. (2) Dual-Branch Camera Control ensures precise 6-DoF trajectory adherence. (3) Two-Stage Generation Pipeline applies a long-video refiner to stage-1 outputs, improving quality and consistency across sequences. (4) Robust Annotation Pipeline extracts accurate metric-scale 6-DoF camera poses from public videos to yield high-quality, spatiotemporally consistent action labels. Driven by these designs, SANA-WMdemonstrates remarkable efficiency across data, training compute, and inference hardware: it uses only $\sim$213K public video clips with metric-scale pose supervision, completes training in 15 days on 64 H100s, and generates each 60s clip on a single GPU; its distilled variant can be deployed on a single RTX 5090 with NVFP4 quantization to denoise a 60s 720p clip in 34s. On our one-minute world-model benchmark, SANA-WM demonstrates stronger action-following accuracy than prior open-source baselines and achieves comparable visual quality at $36\times$ higher throughput for scalable world modeling.

## 2. 方法论：核心思想、关键技术细节与流程

SANA-WM 基于 SANA-Video 发展而来，核心目标是在保持效率的同时实现：

- 分钟级长上下文建模；
- 精确 6-DoF 相机控制；
- 高质量 720p 视频生成；
- 单 GPU 推理。

整体方法包括四个关键部分：

1. 高压缩 LTX2 VAE；
2. 混合 GDN / Softmax 注意力骨干；
3. 双分支相机控制；
4. 两阶段生成与长视频 refiner。

---

### 2.1 渐进式训练策略

论文采用从短视频到长视频的 **progressive training**，分为四个阶段。

#### Stage 1：高效 VAE 适配

- 将原 SANA-Video 的视频 VAE 替换为 **LTX2-VAE**。
- LTX2-VAE 具有更高的时空压缩率：
  - 相比 ST-DC-AE 表示小约 2 倍；
  - 相比 Wan2.1-VAE 小约 8 倍。
- 由于 latent channel 变化，重新初始化 patchify 层和最终输出层。
- 全模型微调约 50K step，使模型适配新的 latent 空间。

#### Stage 2：混合架构适配

- 将原有累计线性注意力结构替换为 **Hybrid GDN-Softmax** 架构。
- 先在短视频上训练，降低调试和不稳定风险。

#### Stage 3：扩展到一分钟并加入相机控制

- 将训练序列扩展到一分钟，即 961 帧输入条件。
- 加入双分支 6-DoF 相机控制。

#### Stage 4：Chunk-Causal 微调与少步蒸馏

- 从一分钟双向模型出发，微调得到 chunk-causal 自回归版本。
- 使用 self-forcing distillation 将采样步数减少到 4 步。
- 推理时加入 attention sink tokens 和局部时间窗口，使 softmax attention 的显存随 rollout 长度保持近似常数。

---

### 2.2 混合 GDN / Softmax 长上下文建模

#### 背景：累计线性注意力的问题

SANA-Video 使用 ReLU-based cumulative linear attention。它通过累计 key-value 外积形成 recurrent state，避免 softmax attention 的长上下文二次复杂度。

但问题是：

- 状态没有显式遗忘机制；
- 旧信息和新信息权重类似；
- 分钟级序列下，历史信息不断堆积，容易造成 drift 和训练不稳定。

#### Gated DeltaNet：带遗忘与修正的递归状态

GDN 引入两个关键机制：

- **decay gate γ**：控制旧状态遗忘；
- **update gate β**：控制新信息写入；
- **delta-rule correction**：只更新当前状态预测与目标 value 之间的残差。

简化理解：

> GDN 不只是累加历史信息，而是根据门控决定哪些信息要保留、忘记或修正。

#### 从 token-wise GDN 到 frame-wise GDN

原始 GDN 是按 token 扫描，而视频中每一帧有大量空间 token。SANA-WM 改为：

- 每一个 latent frame 作为一个 recurrent step；
- 一次处理该帧内所有空间 token；
- recurrent state 仍保持为固定大小的矩阵。

这样可以在长视频中保持显存可控。

#### 关键稳定化：GDN key scaling

论文指出，直接将所有空间 token 聚合到同一个状态中会导致矩阵膨胀，训练不稳定。

为此提出 key scaling：

- 对 key 做 RMSNorm 和 ReLU；
- 再乘以 `1 / sqrt(D × S)`；
- 其中 D 是 head dimension，S 是空间 token 数。

其作用是约束状态转移矩阵的谱范数，使其非扩张，从而避免状态爆炸。

实验表明：

- 不缩放或仅按 `1/sqrt(D)` 缩放会很快出现 NaN；
- `1/sqrt(D×S)` 是唯一稳定收敛的方案。

#### Hybrid GDN / Softmax Attention

最终模型不是纯 GDN，而是混合架构：

- 总共 20 个 Transformer block；
- 其中 15 个是 frame-wise GDN block；
- 每第 4 个左右插入一个 softmax attention block；
- softmax 层位于第 `{3, 7, 11, 15, 19}` 层。

设计动机是：

- GDN 提供高效长上下文递归；
- softmax attention 提供精确的长程信息召回；
- 二者结合，在效率和质量之间取得平衡。

### 2.3 双分支相机控制

SANA-WM 的相机控制目标是精确跟随连续 6-DoF 轨迹。论文提出 **dual-rate geometric conditioning**，包括粗粒度和细粒度两个分支。

---

#### 粗分支：Ray-local UCPE

UCPE 即 Unified Camera Positional Encoding。

对于每个 latent token：

1. 根据相机内参和外参，将像素反投影成世界空间射线；
2. 得到相机中心和单位方向；
3. 构建 ray-local coordinate frame；
4. 将相机几何信息编码进 attention 的 query / key / value 中。

其作用是：

- 在 latent-frame 级别建模全局相机轨迹；
- 让 attention 感知真实 3D 几何关系；
- 提升相机控制精度。

---

#### 细分支：Raw-frame Plücker Mixing

LTX2-VAE 一个 latent frame 会压缩多个原始帧，论文中为 8 个 raw frames。仅在 latent frame 级别建模相机位姿，会丢失 stride 内的细粒度运动。

因此引入 Plücker raymap：

- 每个 raw frame、每个 pixel 计算 Plücker ray：
  - 方向 `d`；
  - moment `o × d`；
- 每 8 个 raw frames 打包成 48 通道 tensor；
- 经零初始化 3D patch embedder 注入模型；
- 每个 block 的 self-attention 后加入 Plücker embedding。

作用是：

- 补偿 VAE 时间压缩带来的相机运动损失；
- 提升短时间尺度内的相机跟随精度。

---

### 2.4 两阶段生成与长视频 Refiner

SANA-WM 采用两阶段生成：

1. **Stage-1**：主世界模型生成一分钟视频 latent；
2. **Stage-2**：长视频 refiner 提升视觉质量和时间一致性。

Refiner 基于 LTX-2 17B 模型初始化，并通过 LoRA 适配长视频。

#### Refiner 训练目标

使用 truncated-σ flow matching。

核心思想：

- 输入是 stage-1 latent 或 degraded latent；
- 加入较大噪声；
- 模型学习将其映射到高质量 target latent；
- 重点是“修复与增强”，不是完全重建。

训练时还加入 reference image tokens：

- 将干净 target latent 的一部分作为 reference；
- 这些 reference token 提供外观和身份锚点；
- 不参与 loss。

#### Refiner 的作用

实验显示，长视频专用 refiner 可以：

- 提升 VBench Overall；
- 提升末尾 10 秒的 imaging quality；
- 降低长时视觉退化 ΔIQ；
- 同时改善相机控制指标。

相比直接使用原始 LTX-2.3 refiner，论文训练的 long-video refiner 在一分钟任务上明显更有效。

---

### 2.5 数据标注流水线

为了获得带相机位姿的视频训练数据，论文构建了公开视频 pose annotation pipeline。

核心组件包括：

- VIPE：视频 pose engine；
- Pi3X：提供长序列一致的 3D 结构与深度；
- MoGe-2：提供准确的单帧公制尺度；
- 对部分数据集使用 GT pose 或 GT depth；
- 对 DL3DV 静态 3D 场景使用 3DGS 渲染一分钟轨迹。

处理流程：

1. 收集公开视频和 3D 数据；
2. 估计或读取相机位姿；
3. 通过 Pi3X / MoGe-2 恢复公制尺度；
4. 对 DL3DV 做 3DGS 增强；
5. 进行视频质量、运动、相机有效性过滤；
6. 使用 captioning 生成场景静态文本描述，避免文本泄露相机动作；
7. 得到约 213K 个带 metric-scale 6-DoF pose 的训练片段。

### 3.2 一分钟世界模型 Benchmark

论文构建了一个新的 **60 秒 world-model benchmark**。

#### 初始场景

- 共 80 张首帧条件图像；
- 分辨率 1280×720；
- 使用 Nano Banana Pro 生成；
- 四类场景，每类 20 个：
  - game-style；
  - indoor；
  - outdoor-city；
  - outdoor-nature。

#### 轨迹设置

每张图像对应两类轨迹：

1. **Simple split**
   - 平滑路径；
   - 平面运动；
   - 弧线、S 曲线、回环、figure-eight 等。

2. **Hard split**
   - 大幅 yaw / pitch 变化；
   - 垂直运动；
   - whip-pan；
   - double loop；
   - crane-like motion；
   - 更强调重访和长时一致性。

每个 split 有 80 个场景，视频为 60 秒、16 fps，对应约 960 帧。

### 3.3 评价指标

论文从四个方面评价。

#### 视觉质量

使用 VBench，指标包括：

- Subject Consistency；
- Background Consistency；
- Temporal Flickering；
- Motion Smoothness；
- Aesthetic Quality；
- Imaging Quality；
- Dynamic Degree；
- Overall Consistency；
- VBench Overall。

#### 相机控制精度

使用 Pi3X 从生成视频中恢复相机轨迹，再通过 Umeyama Sim(3) 与 ground-truth 对齐。

指标包括：

- RotErr：旋转误差，单位为度；
- TransErr：平移误差；
- CamMC：camera-motion consistency。

#### 重访记忆

在相机返回相近视角时比较两帧：

- PSNR；
- SSIM；
- LPIPS。

用于衡量模型是否记住场景布局。

#### 时间退化

将 60 秒视频分为多个 10 秒窗口，计算：

- IQ 0–10；
- IQ 50–60；
- ΔIQ = 首窗口 IQ - 末窗口 IQ。

ΔIQ 越低，说明长时画质退化越小。

- **SANA-WM + refiner**

部分对比方法为 480p，SANA-WM 和 Matrix-Game 3.0 为 720p。论文同时报告：

- 参数量；

## 4. 资源与算力

论文较明确地报告了训练和推理资源。

### 训练资源

SANA-WM 主模型训练使用：

- **64 张 NVIDIA H100 GPU**
- 总训练时间约 **15 天**
- BF16 混合精度
- AdamW 优化器
- 梯度裁剪 0.5

此外，LTX2 VAE 适配阶段：

- 约 50K steps；
- 使用 64 张 H100；
- 约 3.5 天。

主 DiT / backbone 训练阶段约 15 天，分为：

| 阶段 | 目标 | 时长 | 步数 | 计算预算 |
|---|---|---:|---:|---:|
| Stage 1 | Frame-wise GDN | 5s clips | 30K | 约 2.75 天 |
| Stage 2 | Hybrid Attention | 5s clips | 30K | 约 2 天 |
| Stage 3 | Minute-scale + Camera Control | 1min clips | 31K | 约 8 天 |
| Stage 4 | SFT / chunk-causal | 1min clips | 10K | 约 2.5 天 |

### 模型规模

- 主模型参数量：**2.6B**
- Refiner 初始化自：**17B LTX-2**
- Refiner 使用 LoRA 适配，而非全量微调。

### 推理资源

论文强调单 GPU 推理能力：

- Bidirectional 和 chunk-causal 版本可在单张 H100 上运行；
- Stage-1 SANA-WM 显存约 51.1 GB；
- SANA-WM + refiner 显存约 74.7 GB，仍可在 80GB H100 上运行；
- 蒸馏 + attention sink + NVFP4 量化后，可在单张 RTX 5090 上运行；
- 60 秒 720p clip 去噪时间约 **34 秒**。

### 吞吐量

在 8 H100 评估设置下：

- SANA-WM：约 24.1 videos/hour；
- SANA-WM + refiner：约 22.0 videos/hour；
- 相比某些基线可达到最高约 36× 吞吐优势。

## 5. 实验数量与充分性

### 主要实验类型

论文实验较丰富，大致包括以下几类：

1. **主结果对比**
   - 在 Simple 和 Hard 两个 60 秒 benchmark split 上比较多个世界模型。
   - 同时评价视觉质量、相机控制、显存和吞吐量。

2. **重访记忆与时间稳定性实验**
   - 使用 same-pose revisit pairs；
   - 评估 PSNR / SSIM / LPIPS；
   - 评估 ΔIQ 长时退化。

3. **渐进式训练消融**
   - 原 SANA-Video；
   - + LTX2 VAE；
   - + Hybrid Attention；
   - 比较质量、显存、延迟、吞吐。

4. **GDN key scaling 消融**
   - 无缩放；
   - 仅 `1/sqrt(D)`；
   - `1/sqrt(D×S)`；
   - 验证训练稳定性。

5. **相机控制消融**
   - No control；
   - Plücker only；
   - PRoPE；
   - UCPE only；
   - UCPE + Plücker。

6. **Refiner 消融**
   - 原始 LTX-2.3 refiner；
   - 论文提出的 long-video refiner；
   - 比较 VBench、pose accuracy、末尾 IQ、ΔIQ。

7. **推理效率消融**
   - 60-step H100；
   - 4-step distillation；
   - attention sink；
   - RTX 5090；
   - NVFP4 quantization。

8. **Bidirectional vs Autoregressive 对比**
   - 比较双向生成与 AR rollout 的质量、速度和稳定性。

9. **定性实验**
   - 多组 Hard-Trajectory 60 秒视频对比；
   - Pi3X 3D 重建可视化，展示生成视频中的几何一致性。

### 实验是否充分

整体来看，实验设计较充分，理由包括：

- 不只比较画质，也比较相机跟随、长时退化、重访记忆和效率；
- 既有主实验，也有多个关键模块消融；
- benchmark 覆盖四类场景和两类轨迹难度；
- 报告了显存和吞吐，体现实用性；
- 对相机控制使用恢复轨迹与 GT 轨迹对齐，较直接衡量动作跟随。

### 可能的不公平或局限

不过实验仍有一些需要注意的地方：

- 不同基线分辨率不一致，部分为 480p，SANA-WM 为 720p；
- 不同方法的原生控制接口不同，虽然论文尝试转换为统一 trajectory，但仍可能存在适配偏差；
- benchmark 首帧由 Nano Banana Pro 生成，不完全代表真实世界图像分布；
- Pi3X 既用于标注 / 数据处理，也用于评估相机轨迹，可能引入工具链相关偏差；
- 80 个场景规模对于一分钟世界模型评估有价值，但仍不算非常大；
- 与闭源工业模型的比较受限，部分高性能系统可能无法在同等硬件预算下完整评估。

## 6. 主要结论与发现

论文的主要发现包括：

1. **SANA-WM 可以用相对可控的资源训练一分钟 720p 世界模型**
   - 约 213K 公开视频；
   - 64 张 H100 训练约 15 天；
   - 单 GPU 推理。

2. **混合 GDN / Softmax 架构有效平衡效率和质量**
   - GDN 提供长时高效递归；
   - softmax block 增强长程精确召回；
   - 相比原始线性注意力，质量更高且显存仍可控。

3. **GDN key scaling 是训练稳定性的关键**
   - `1/sqrt(D×S)` 缩放显著避免状态爆炸；
   - 对分钟级视频建模尤为重要。

4. **双分支相机控制显著提升 6-DoF 动作跟随**
   - UCPE 捕获全局轨迹；
   - Plücker mixing 补偿 VAE 时间压缩中的细粒度运动；
   - 二者结合获得最低的相机误差。

5. **长视频 refiner 对视觉质量和长时稳定性有明显帮助**
   - 提升 VBench Overall；
   - 降低 ΔIQ；
   - 改善末尾窗口画质；
   - 比直接使用短视频 refiner 更适合一分钟任务。

6. **SANA-WM 在 benchmark 上取得强相机控制和高吞吐**
   - 在 Simple 和 Hard split 上，SANA-WM + refiner 的相机误差优于对比方法；
   - VBench Overall 接近 LingBot-World；
   - 推理吞吐显著更高。

## 7. 优点与亮点

### 方法设计亮点

- **面向效率的系统设计**
  - 从 tokenizer、attention、训练并行、推理蒸馏到量化部署都围绕效率优化。

- **Hybrid GDN / Softmax 架构合理**
  - 避免纯 softmax 的长上下文显存爆炸；
  - 避免纯线性注意力的长期记忆漂移；
  - 兼顾可扩展性和表现。

- **Frame-wise GDN 适合视频建模**
  - 以帧为 recurrent step，比 token-wise 更适合高分辨率视频；
  - 固定大小状态有利于分钟级 rollout。

- **相机控制模块具有几何解释性**
  - UCPE 和 Plücker raymap 都是几何感知表示；
  - 相比仅依赖文本或隐式 action，更适合可控世界建模。

- **公开视频 pose annotation pipeline 实用**
  - 结合 VIPE、Pi3X、MoGe-2 和 3DGS 增强；
  - 为缺少动作标签的公开视频生成 metric-scale pose。

- **评估维度全面**
  - 不只看视觉质量，也看相机跟随、重访一致性、时间退化和推理效率。

- **部署导向明确**
  - 支持单 H100 生成一分钟视频；
  - 蒸馏量化后支持 RTX 5090；
  - 对实际应用和研究复现更友好。

---

## 8. 不足与局限

### 模型能力局限

- **缺少显式 3D 场景记忆**
  - 虽然生成结果具有一定几何一致性，但模型本身仍是视频生成模型；
  - 没有显式维护可查询、可更新的 3D memory 或场景图。

- **长时间 rollout 仍可能漂移**
  - 一分钟内表现较好，但更长时间、更多 loop closure 或复杂动态场景下仍可能退化。

- **动态场景建模有限**
  - 对行人、车辆、动物等动态对象的长期一致性可能不足；
  - 训练数据过滤中也对实体数量有所限制，可能降低动态复杂度覆盖。

- **少见视角和极端运动仍有风险**
  - Hard split 包含一定极端轨迹，但真实交互场景可能更加复杂；
  - 大 pitch、快速旋转、遮挡穿越等情况仍可能造成结构崩坏。

### 数据与评估局限

- **数据来源存在偏差**
  - 使用公开视频、合成数据、游戏数据和生成图像；
  - 数据分布可能偏向高质量、静态或可重建场景。

- **公开视频授权与使用限制复杂**
  - 不同数据集和工具有不同 license 或 project terms；
  - 商业或大规模部署需要额外审查。

- **评估 benchmark 规模有限**
  - 80 个首帧场景和两个轨迹 split 对研究验证足够，但不足以覆盖所有真实应用场景。

- **评估工具可能带来系统性偏差**
  - Pi3X 同时参与数据标注、尺度恢复和评估；
  - 如果生成视频更符合 Pi3X 的偏好，可能影响 pose accuracy 的客观性。

- **不同基线接口和分辨率不同**
  - 一些基线为 480p，一些为 720p；
  - 有的需要 8 GPU，有的只用 1 GPU；
  - 虽然论文报告了这些差异，但横向比较仍不完全理想。

### 应用风险

- **生成视频可能被误认为真实视频**
  - 高质量长视频生成带来溯源和误用风险。

- **不能直接作为安全关键预测系统**
  - 在机器人、自动驾驶或物理规划中，生成视频不能等同真实物理预测；
  - 需要真实数据验证和安全约束。

- **可能继承训练数据中的偏见**
  - 场景、文化、地理环境、光照条件和物体类型覆盖不均；
  - 对罕见环境或敏感人类场景的泛化能力不确定。

# 总体评价

SANA-WM 是一篇系统性较强的长视频世界模型论文。它的贡献不只是提出一个 2.6B 模型，而是围绕 **分钟级、720p、6-DoF 可控、低成本训练与单 GPU 推理** 这一目标，完整设计了模型架构、训练流程、相机控制、数据标注、benchmark 和部署加速方案。

其最重要的价值在于证明：在不依赖超大规模私有数据和极端算力的情况下，通过结构设计和数据标注工程的结合，开源世界模型也可以接近工业级视觉质量，并在动作跟随和推理效率上取得显著优势。

不过，该方法仍未解决显式 3D 场景记忆、复杂动态对象、极长时交互和真实物理可靠性等问题。因此，它更适合作为高效长视频世界建模的重要基线和平台，而不是一个已经完全可靠的通用物理世界模拟器。
