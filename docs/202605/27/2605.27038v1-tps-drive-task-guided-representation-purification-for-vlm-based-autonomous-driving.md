## 摘要
视觉-语言模型（VLM）为自动驾驶规划提供了有前景的基础，然而，弥合语义推理与精确三维空间预测之间的差距仍是一项关键挑战。现有的表示策略通常遵循两条路径：文本对齐方法将连续空间状态展平为符号，这会损害几何结构并引发“空间幻觉”；密集视觉方法保留空间拓扑，但会使标准分词器被冗余的背景纹理所淹没，从而导致“表示干扰”。为解决这些局限，我们提出 TPS-Drive，一个以任务引导表示净化为核心的新框架，使 VLM 能够在净化空间中思考。其核心是一个以智能体为中心的分词器，利用由冻结的三维检测头监督的任务引导向量量化机制，将有限的码本容量从普遍存在的静态背景显式重新分配给关键动态智能体，并有效隔离空间冗余。借助这一净化后的空间词汇，TPS-Drive 采用解耦的推理流水线，依次执行场景理解、未来预测和动作生成。该框架通过渐进式三阶段训练范式进行优化，并最终通过奖励驱动的精炼超越纯模仿学习。大量实验验证了我们的方法：TPS-Drive 在开环 nuScenes 评估中实现了准确的智能体空间状态预测并降低了碰撞率，同时在严格的闭环 NAVSIMv1 和 NAVSIMv2 基准上创下了新的安全性纪录。

## Abstract
Vision-Language Models (VLMs) provide a promising foundation for autonomous driving planning, yet bridging semantic reasoning and precise 3D spatial forecasting remains a critical challenge. Existing representation strategies generally follow two paths: text-aligned methods flatten continuous spatial states into symbols, which compromises geometric structure and induces "spatial hallucinations"; dense visual methods preserve spatial topology but overwhelm standard tokenizers with redundant background textures, leading to "representation interference". To address these limitations, we introduce TPS-Drive, a novel framework centered on Task-Guided Representation Purification that empowers VLMs to Think in Purified Space. At its core, an Agent-Centric Tokenizer utilizes a task-guided vector quantization mechanism supervised by a frozen 3D detection head, which explicitly reallocates limited codebook capacity from pervasive static backgrounds to critical dynamic agents and effectively isolates spatial redundancy. Leveraging this purified spatial vocabulary, TPS-Drive employs a decoupled reasoning pipeline that sequentially performs scene understanding, future forecasting, and action generation. The framework is optimized via a progressive three-stage training paradigm, culminating in reward-driven refinement that surpasses pure imitation learning. Extensive experiments validate our approach: TPS-Drive achieves accurate agent spatial state forecasting and reduces collision rates in open-loop nuScenes evaluations, while establishing new safety records on the rigorous closed-loop NAVSIMv1 and NAVSIMv2 benchmarks.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体框架：TPS-Drive

TPS-Drive 将自动驾驶规划拆解为三个显式阶段：

1. **场景理解（Scene Understanding）**
   - 从当前多视角图像、BEV 特征、导航命令、车辆状态等输入中提取结构化场景语义。
   - 输出结构化表示 `s_t`，包括：
     - 道路类型；
     - 拓扑结构；
     - 光照条件；
     - 物理约束；
     - 制动能力等。

2. **未来预测（Future Forecasting）**
   - 使用净化后的 BEV token 预测未来时刻的动态智能体空间状态。
   - 目标不是重建完整场景，而是重点预测任务关键的动态目标。

3. **动作生成（Action Generation）**
   - 基于场景理解结果和预测的未来空间 token，使用条件扩散模型生成连续 ego 轨迹。

整体输入包括：

- 多视角相机图像 `I_t`；
- 当前 BEV 特征 `B_t`；
- 高层导航命令 `c`，如左转、直行；
- 任务 prompt `p`；
- 当前车辆运动状态 `k_t = [v_t, a_t]`；
- 输出未来规划轨迹 `τ_{t+1:t+T_plan}`。

## 2.2 Agent-Centric Tokenizer：以智能体为中心的空间 tokenizer

这是论文最核心的技术模块。

### 核心目标

- 将 BEV 特征离散化为 token；
- 但不是普通 VQ-VAE 式地重建所有区域；
- 而是在冻结 3D 检测头监督下，让 token 更关注动态智能体。

### 基本流程

1. 输入 BEV 特征图：

   - `B ∈ R^{D×H×W}`

2. 编码器 `E_ϕ` 将 BEV 特征压缩为低分辨率 latent grid：

   - `e ∈ R^{D'×H'×W'}`

3. 对每个空间位置的 latent feature 进行最近邻码本查找：

   - 每个位置 `e_{i,j}` 被映射到主码本 `V_1` 中最近的 codebook entry；
   - 得到离散 token map `z`；
   - 得到量化表示 `q`。

4. 解码器 `D_ψ` 重建净化后的 BEV 特征。

### 任务引导的主码本训练

论文没有只使用传统 VQ-VAE 的重建损失，而是加入冻结的 3D 检测头进行监督。

训练目标包括：

- **重建损失**
  - 保证净化特征不要偏离原始 BEV 空间分布太远；
- **检测任务损失**
  - 包括 heatmap loss 和 3D bounding box loss；
  - 使 tokenizer 更关注动态目标；
- **VQ 损失和 commitment loss**
  - 保持码本学习稳定。

可以理解为：

> tokenizer 不再只是为了“重建图像 / BEV”，而是为了“保留对自动驾驶检测和预测有用的空间信息”。

## 2.3 残差细化层：Residual Refinement Layers

单层 VQ 量化容易损失精细几何信息，因此论文引入多级残差码本。

### 设计方式

- 主码本 `V_1` 捕获主要语义和结构；
- 后续 `L-1` 个残差码本 `V_2, ..., V_L` 捕获前一层量化后的残差信息；
- 本文实现中使用：
  - 主码本大小：8192；
  - 总残差层数：`L = 4`；
  - BEV 特征从 `200×200` 压缩到 `25×25`。

### 推理效率设计

- 只有主 token `z^(1)` 输入 VLM 进行自回归预测；
- 残差 token 由轻量分类头并行预测；
- 这样既减少自回归序列长度，又保留高精度空间细节。

论文采用渐进式训练：

### Stage 1：Tokenizer 预训练

- 用 advantage 加权更新世界模型分支；
- 用 reward 加权 denoising loss 更新扩散规划器。

该阶段的作用是：

> 让模型超越单纯模仿学习，更关注安全、平滑和法规约束。

整体来看，实验较为充分，原因包括：

- 覆盖了开环和闭环两类评测；
- 同时评估预测能力和规划能力；
- 使用了 nuScenes、NAVSIMv1、NAVSIMv2 三个 benchmark；
- 对比对象包含传统端到端方法、VLM 方法和世界模型方法；
- 有组件级消融实验验证核心设计有效性；
- 有定性案例展示安全性与解释性。

实验设计总体较客观，但仍有一些需要注意的公平性问题：

## 6. 主要结论与发现

论文的主要结论包括：

1. **任务引导的表示净化能够显著提升空间预测能力**
   - 标准重建式 VQ tokenizer 的 NDS 仅为 17.31%；
   - 加入 task-guided tokenizer 后提升到 30.03%；
   - 说明过滤背景冗余、聚焦动态智能体非常关键。

2. **残差码本能补充精细几何信息**
   - 加入 `L=4` 残差层后，NDS 进一步提升到 33.78%；
   - L2 和碰撞率也有所改善。

3. **奖励驱动精炼能提升安全性与闭环表现**
   - EPDMS 从 84.1 提升到 86.7；
   - open-loop CR 降至 0.19%；
   - 表明纯监督模仿学习不足以完全优化安全驾驶行为。

4. **TPS-Drive 在多个 benchmark 上达到较强表现**
   - nuScenes 上，TPS-Drive 在 VLM 类方法中实现较低碰撞率；
   - NAVSIMv1 上 PDMS 达到 89.7；
   - NAVSIMv2 上 EPDMS 达到 86.7；
   - 在 agent spatial forecasting 上达到 34.60% NDS 和 24.03% mAP。

5. **显式预测动态智能体的三维状态有助于提升可解释性**
   - 定性结果显示，TPS-Drive 能更明确识别行人和拥挤环境中的关键障碍；
   - 相比隐式视觉 CoT 方法更容易产生安全动作。

## 7. 优点与亮点

### 7.1 方法设计亮点

- **提出“Think in Purified Space”的清晰理念**
  - 将 VLM 的推理空间从原始密集视觉空间转向净化后的 agent-centric 空间。

- **任务引导 VQ 设计合理**
  - 使用冻结 3D 检测头监督 tokenizer；
  - 避免 codebook 被背景纹理占据；
  - 让离散 token 更服务于驾驶任务。

- **兼顾紧凑性与精细度**
  - 主码本用于 VLM 自回归预测，保持序列长度可控；
  - 残差码本并行预测，补充几何细节。

- **推理流程显式解耦**
  - 场景理解、未来预测、动作生成各自承担明确职责；
  - 相比黑盒端到端模型更具可解释性。

- **引入扩散规划器**
  - 能建模人类驾驶行为的多模态分布；
  - 对复杂场景中的轨迹生成更灵活。

- **加入 reward-driven refinement**
  - 不仅模仿专家轨迹，还通过奖励对齐安全和平滑性目标。

- 同时覆盖：
  - 开环 nuScenes；
  - 闭环 NAVSIMv1；
  - 更严格的 NAVSIMv2；
  - 世界模型预测指标。
- 对比方法较多，包含最新 VLM / VLA / world model 方法。
- 消融实验结构清晰，能逐步证明各模块贡献。
- 定性分析与核心论点一致，展示了表示净化对安全场景的帮助。

## 8. 不足与局限

### 8.1 方法层面的局限

- **依赖冻结 3D 检测头**
  - tokenizer 的表示能力受限于检测头质量；
  - 如果检测头漏检或偏置严重，净化表示也会受影响。

- **解耦多阶段架构可能影响实时性**
  - 包含 tokenizer、VLM、残差分类头、扩散规划器；
  - 推理链路较长；
  - 论文也承认当前架构限制实时推理速度。

- **动态目标之外的信息可能被过度压缩**
  - 任务引导净化强调动态智能体；
  - 但某些静态元素也很重要，例如路沿、施工区域、临时障碍、交通标志；
  - 如果净化过强，可能损失部分安全相关静态信息。

- **reward 设计可能引入偏差**
  - 离线 reward 基于几何误差和平滑性；
  - 是否充分覆盖真实世界安全、舒适性、法规和社交交互仍不明确。

---

### 8.2 实验层面的不足

- **缺少算力与训练成本报告**
  - 未报告 GPU 型号、数量、训练时间、推理延迟；
  - 不利于判断方法实际部署成本。

- **缺少实时性评估**
  - 自动驾驶系统对延迟非常敏感；
  - 论文没有给出 FPS、单帧推理耗时或端到端 latency。

- **复现细节仍不够完整**
  - reward-driven refinement 的具体 reward 设计、权重和 rollout 设置描述有限；
  - Qwen3.5-27B 自动标注流程依赖人工检查，复现成本较高。

- **benchmark 覆盖仍偏离真实道路部署**
  - NAVSIM 是伪仿真 / benchmark 环境；
  - 不能完全代表真实交通中的长尾风险；
  - 缺少真实车端部署或高保真仿真测试。

- **传感器设置对比可能不完全公平**
  - 不同 baseline 可能使用不同数量相机或 LiDAR；
  - 部分方法额外使用 ego status；
  - 虽然论文有标注，但横向比较仍需谨慎。

---

### 8.3 应用风险

- 在复杂长尾场景下，如极端天气、遮挡、异常交通参与者、施工改道等，task-guided tokenizer 是否仍能正确保留关键信息尚未充分验证。
- 如果检测头无法识别未知类别或罕见目标，TPS-Drive 可能会将其视为背景冗余而弱化。
- 扩散规划器虽然能生成多模态轨迹，但其采样稳定性、最坏情况安全性仍需要进一步证明。

---

## 总体评价

TPS-Drive 是一篇围绕 VLM 自动驾驶空间表示问题展开的较完整工作。它针对“文本符号化导致空间幻觉”和“密集表示导致表示干扰”两个痛点，提出任务引导的 agent-centric tokenizer，并将其与解耦式 VLM 推理和扩散规划结合。实验覆盖 nuScenes、NAVSIMv1 和 NAVSIMv2，结果显示其在空间预测和安全规划上均有明显优势。

不过，论文在算力成本、实时推理、reward 细节和真实部署验证方面仍有不足。总体而言，该工作的重要价值在于强调：对于自动驾驶 VLM，关键不只是扩大模型或输入更多视觉 token，而是要构建面向驾驶任务的高质量空间表示。
