## 摘要
具身导航要求智能体将语言和视觉观测映射为一系列空间动作，以驱动真实机器人穿越其从未见过的环境。当前主流方法是在规模不断扩大的机器人轨迹集合上扩展视觉-语言-动作（VLA）基础模型。本文认为，专门针对导航而言，通用性可以通过结构获得，而不仅仅依赖数据规模。导航的底层决策结构可归约为单一的语言-视觉-机器人动作翻译。语言动作生成语义层面的方向指令，视觉动作生成像素层面的视觉目标。二者的输出均位于预训练多模态大语言模型（MLLM）的自然输出流形之内，因此该任务可以由智能体进行推理，而无需从机器人数据中学习。为此，我们提出 Uni-LaViRA，这是一种统一的智能体架构，以零样本方式将同一洞见扩展到四类任务（VLN-CE、ObjectNav、EQA 和 Aerial-VLN）以及四种异构真实机器人（轮式机器人、四足机器人、人形机器人和自研无人机）。两种智能体循环机制使这种统一变得可行。TODO List Memory（TDM）在每一步重写待完成子目标的结构化检查清单，并将未完成项重新送入智能体最近的注意力窗口。Second Chance Backtrack（SCB）将机器人回退到错误发生前的状态，并基于失败的子轨迹来约束智能体的下一步规划，从而将单遍导航转变为自我纠错过程。在无需任何训练的情况下，Uni-LaViRA 在 VLN-CE R2R 上达到 60.7% 的成功率（SR），在 VLN-CE RxR 上达到 51.3%，在 HM3D-v2 上达到 77.7%，在 HM3D-OVON 上达到 60.0%，在 MP3D-EQA 上达到 54.7%，在 OpenUAV 上达到 40.0%，达到甚至超过了近期需要消耗数百万样本和数千 GPU 小时进行训练的导航基础模型。

## Abstract
Embodied navigation requires an agent to map language and visual observations to a stream of spatial actions that drive a real robot through environments it has never seen. The dominant approach has been to scale vision-language-action (VLA) foundation models on ever-larger collections of robot trajectories. This paper argues that, for navigation specifically, generality can be obtained structurally, not only through data scale. The underlying decision structure of navigation reduces to a single Language-Vision-Robot Actions Translation. The language action emits semantic-level directional command and the vision action emits a pixel-level visual target. Both outputs lie inside the natural output manifold of pretrained multimodal large language models (MLLMs), so the task can be reasoned about by an agent rather than learned from robot data. Therefore, we present Uni-LaViRA, a unified agentic architecture that extends the same insight to four task families (VLN-CE, ObjectNav, EQA, and Aerial-VLN) and to four heterogeneous real robots (Wheeled, Quadruped, Humanoid robot, and a self-built UAV) in a zero-shot manner. Two agent-loop mechanisms make this unification practical. TODO List Memory (TDM) rewrites a structured checklist of pending sub-goals at every step, reciting the unfinished items back into the agent's most recent attention window. Second Chance Backtrack (SCB) rolls the robot back to the pre-error state and conditions the agent's next plan on the failed sub-trajectory, turning single-pass navigation into a self-correcting process. With zero training effort, Uni-LaViRA reaches 60.7% SR on VLN-CE R2R, 51.3% on VLN-CE RxR, 77.7% on HM3D-v2, 60.0% on HM3D-OVON, 54.7% on MP3D-EQA, and 40.0% on OpenUAV, matching or even surpassing recent training navigation foundation models that consume millions of samples and thousands of GPU-hours.

## 2. 方法论：核心思想、关键技术与算法流程

### 2.1 核心思想：Language-Vision-Robot Actions Translation

Uni-LaViRA 将导航策略分解为三个层次：

1. **Language Action：语言动作**
   - 输入任务描述、当前多视角观测、历史轨迹和 TODO 列表；
   - 输出语义层面的高层导航动作，例如：
     - `turn(front / left / right / back)`
     - `backtrack(waypoint)`
     - `go_stair(up / down)`
     - `double_check(stop)`
   - 实验中使用 **Gemini-3.1-Pro** 作为 Language Action Model。

2. **Vision Action：视觉动作**
   - 根据语言动作选择的方向，在对应第一人称视角中定位视觉目标；
   - 输出：
     - 目标 bounding box 或点坐标；
     - 目标文本描述，例如“通向卧室的门口”；
   - 实验中使用 **Qwen3.5-27B** 作为 Vision Action Model。

3. **Robot Action：机器人动作**
   - 将视觉模型输出的像素目标结合深度图和相机内参反投影为 3D 目标点；
   - 再由确定性的低层控制器规划和执行动作；
   - 地面机器人使用 2D 占据栅格和 Fast-Marching 规划；
   - UAV 使用 3D voxel map 和可视图搜索。

### 2.2 形式化表达

论文将整体策略写作：

\[
A_t = \pi_{robot}(\phi_{vis}(\phi_{lang}(T, O_t, H_t)), D_t, K, pose_t)
\]

其中：

- \(T\)：任务语言描述；
- \(O_t\)：当前视觉观测；
- \(H_t\)：历史轨迹和决策；
- \(\phi_{lang}\)：语言动作模型；
- \(\phi_{vis}\)：视觉动作模型；
- \(D_t\)：深度图；
- \(K\)：相机内参；
- \(pose_t\)：机器人位姿；
- \(\pi_{robot}\)：确定性的机器人控制器。

该公式体现了本文的核心思想：**不直接学习低层机器人控制策略，而是把导航拆解成 MLLM 擅长的语言决策、视觉 grounding 和几何控制。**

### 2.3 TODO List Memory（TDM）

TDM 用于解决长指令、多阶段任务中的注意力漂移问题。

- 系统维护一个有序 TODO 列表：

\[
L_t = [\ell_t^{(1)}, ..., \ell_t^{(n)}]
\]

每个子任务包含：

\[
\ell_t^{(i)} = (content^{(i)}, status^{(i)}, result^{(i)})
\]

其中：

- `content`：子目标内容；
- `status`：`pending` 或 `completed`；
- `result`：完成该子目标的视觉或轨迹证据。

- 每一步，语言模型可以执行：
  - `update`：标记子任务完成；
  - `rewrite`：改写子任务描述；
  - `add`：添加新子任务；
  - `remove`：删除无关子任务。

**作用**：

- 将长指令拆解成显式检查清单；
- 每一步都把尚未完成的子目标重新放入模型注意力窗口；
- 减少在 RxR、OpenUAV 等长指令任务中的遗忘和提前停止问题。

### 2.4 Second Chance Backtrack（SCB）

SCB 用于错误恢复。

- 系统在每个决策点保存 waypoint；
- 若语言模型判断当前路径错误，可调用：

```text
backtrack(wp_k)
```

- 机器人回退到指定历史 waypoint；
- 与普通 backtracking 不同，SCB 不丢弃失败路径，而是将以下信息重新提供给语言模型：
  - 原始任务；
  - 回退点处的观察；
  - 上一次失败的方向；
  - 从该 waypoint 出发到失败位置之间的图像序列。

**作用**：

- 不是简单“撤销上一步”，而是让模型基于失败经验重新规划；
- 将错误轨迹作为上下文，避免重复犯同样错误；
- 与 TDM 配合，使 TODO 列表在回退后仍保持一致。

### 2.5 整体算法流程

每个 episode 的执行过程大致为：

1. 输入任务描述和初始观测；
2. 初始化 TODO List；
3. 在每个时间步：
   - Language Action Model 更新 TODO List 并选择高层动作；
   - 若选择停止，则进行 double-check；
   - 若选择 backtrack，则触发 SCB 回退并重新规划；
   - Vision Action Model 在指定视角中定位目标；
   - Robot Action Controller 将像素目标转换为机器人动作；
   - 执行动作，更新观测、历史和 waypoint buffer；
4. 直到停止或达到最大步数。

- Agilex Cobot Magic 双臂轮式平台；
- Unitree G1 人形机器人；
- Unitree Go1 四足机器人；
- 自研四旋翼 UAV。

真实环境包括：

- 室内办公室；
- 室内大厅；
- 户外走廊；
- 户外野外场景。

真实机器人实验主要用于展示跨 embodiment 可迁移性，而非严格定量 benchmark。

## 4. 资源与算力

### 4.1 训练算力

- Uni-LaViRA **不进行机器人轨迹训练**；
- 不微调 Language Action Model 或 Vision Action Model；
- 因此论文强调其训练成本为 **0 training samples / 0 GPU-hours**。

### 4.2 推理算力与硬件

论文报告了仿真实验的推理硬件：

- 仿真 rollout 使用 **8 张 NVIDIA RTX 4090 GPU** 并行运行；
- Language Action Model 使用 Gemini-3.1-Pro，通过 API 调用；
- Vision Action Model 使用 Qwen3.5-27B；
- 每个完整六任务 600 episode 评估大约耗时 **5–7 小时**，使用 **40 个并行 API worker**；
- 单次 Language Action 调用：
  - API 正常时约 **30–60 秒**；
  - API 拥塞时约 **80–150 秒**；
- 本地规划器和控制器运行时间小于 **10 ms**。

### 4.3 推理 FLOPs

论文估算：

- Uni-LaViRA-LA 每次调用：
  - 输入约 22.3k tokens；
  - 输出约 1.6k tokens；
  - 约 **9.6 PFLOPs**；
- Uni-LaViRA-VA 每次调用：
  - 输入约 0.99k tokens；
  - 输出约 0.11k tokens；
  - 约 **59 TFLOPs**；
- 一次完整六任务 Full evaluation 约消耗 **10 EFLOPs**。

### 4.4 与训练型方法的算力对比

论文列举部分训练型方法的训练开销：

- NaVid：约 0.95M 训练样本，320 GPU-hours；
- Uni-NaVid：约 3.6M 样本，1,400 GPU-hours；
- NavFoM：约 8.0M 样本，4,032 GPU-hours；
- OmniNav：约 14.0M 样本，14,592 GPU-hours；
- ABot-N0：约 16.9M 样本，6,144 GPU-hours。

论文认为 Uni-LaViRA 将固定前期训练成本转化为按次推理成本，更适合资源有限的研究机构。

### 4.5 需要注意的点

- 虽然训练成本为 0，但推理成本并不低；
- 使用 Gemini-3.1-Pro 这类闭源 API，真实成本、延迟和可复现性受服务商影响；
- 文中对 FLOPs 的估算依赖模型和 API 假设，实际成本可能随服务实现而变化。

## 5. 实验数量与充分性

### 5.1 实验规模

论文实验包括：

- **6 个仿真 benchmark**：
  - R2R；
  - RxR；
  - HM3D-v2；
  - HM3D-OVON；
  - MP3D-EQA；
  - OpenUAV；
- 每个 benchmark 100 episode；
- 每个配置 3 个随机种子；
- 主配置共约：
  - 6 × 100 × 3 = **1800 次试验轨迹**；
- 还进行了：
  - TDM / SCB 消融；
  - 100-episode 子集有效性验证；
  - 推理成本分析；
  - 失败模式统计；
  - 真实机器人部署。

### 5.2 消融实验

论文比较了四种配置：

1. Full：TDM + SCB；
2. w/o TDM：只保留 SCB；
3. w/o SCB：只保留 TDM；
4. w/o both：二者均关闭。

结果显示：

- Full 在六个 benchmark 上均最优；
- 平均 SR / ACC：
  - Full：57.4；
  - w/o TDM：51.3；
  - w/o SCB：51.2；
  - w/o both：48.6。

说明 TDM 与 SCB 都有贡献，并且二者存在互补关系。

### 5.3 失败模式分析

论文对 1800 个 trial 进行了失败类型归类，主要失败模式包括：

- 错误目标处提前停止；
- 到达目标附近但没有 stop；
- EQA 回答错误；
- 接近但未到达；
- 迷失和长距离 wandering。

前三类占大部分失败，说明问题更多来自高层语义判断和停止判定，而非低层几何控制。

### 5.4 实验是否充分与公平

总体上实验覆盖面较广：

- 跨任务；
- 跨数据集；
- 跨仿真环境；
- 跨真实机器人；
- 包含消融和失败分析。

但仍有若干公平性和客观性问题：

- 主要结果基于每个 benchmark 的 **100 episode 子集**，而非完整 val-unseen / test split；
- 尽管论文验证子集与完整 split 的基线表现接近，但 OpenUAV 上部分指标差异较大；
- 与训练型方法比较时，很多对比结果来自原论文报告，不一定完全同环境、同协议；
- Uni-LaViRA 使用非常强的闭源 MLLM，和开源或训练型方法的比较存在资源形态差异；
- 真实机器人实验主要是定性展示，没有系统性成功率统计。

因此，实验整体较充分，但在严格 benchmark 公平性和可复现性方面仍有改进空间。

## 6. 主要结论与发现

- **结构化分解可以显著减少对机器人轨迹训练的依赖**：
  - Uni-LaViRA 无需训练即可在多个导航任务上达到强性能。
- **Language Action + Vision Action + Robot Action 的分层设计有效**：
  - 高层语义推理、中层视觉 grounding 和低层几何控制分别交给最适合的模块处理。
- **TDM 对长指令和多阶段任务有明显帮助**：
  - 尤其在 R2R、RxR、EQA、OpenUAV 中提升明显。
- **SCB 对错误恢复和开放探索有帮助**：
  - 在 HM3D-v2、HM3D-OVON 等目标搜索任务中贡献较大。
- **Uni-LaViRA 在多个任务上超过部分训练型方法**：
  - HM3D-v2：77.7% SR；
  - HM3D-OVON：60.0% SR；
  - MP3D-EQA：54.7% ACC；
  - OpenUAV：40.0% SR。
- **在 VLN-CE 上仍落后最强训练型 VLA**：
  - R2R 上 Uni-LaViRA 为 60.7% SR，低于 OmniNav、ABot-N0、SPAN-Nav 等；
  - RxR 上 51.3% SR，仍低于最强训练型方法。
- **跨机器人部署具有可行性**：
  - 同一套语言和视觉动作模型可迁移到轮式、人形、四足和 UAV，只需替换低层控制器。

## 7. 优点与亮点

### 7.1 方法设计清晰

- 将导航分解为语言动作、视觉动作、机器人动作三个层次；
- 每个层次对应不同抽象粒度，符合导航任务本身结构；
- 避免让 MLLM 直接输出连续控制量。

### 7.2 零训练、跨任务能力强

- 不需要机器人轨迹数据；
- 同一框架覆盖 VLN、ObjectNav、EQA、Aerial-VLN；
- 对开放词汇目标导航友好。

### 7.3 TDM 与 SCB 具有实用价值

- TDM 解决长程任务中的记忆和进度跟踪问题；
- SCB 将失败轨迹转化为重新规划依据；
- 二者均为 prompt-level 机制，不引入额外训练。

### 7.4 实验覆盖广

- 六个仿真 benchmark；
- 四类任务；
- 四种真实机器人；
- 包含主实验、消融、失败分析、算力分析。

### 7.5 对研究范式有启发

- 论文明确提出导航任务可能位于 MLLM 可处理的自然输出流形内；
- 对比“继续扩大 VLA 训练规模”的路线，提供了结构化 agentic system 的替代思路。

## 8. 不足与局限

### 8.1 依赖闭源大模型

- Language Action Model 使用 Gemini-3.1-Pro；
- 闭源 API 带来：
  - 可复现性问题；
  - 成本不可控；
  - 延迟不可控；
  - 模型版本变化风险；
  - 隐私和部署限制。

### 8.2 推理成本与延迟较高

- 单次 Language Action 调用可能需要 30–150 秒；
- 对实时机器人导航来说，这一延迟较大；
- UAV 实验中已经出现因 API 延迟导致 hover-and-redecide 的问题。

### 8.3 评估使用 100-episode 子集

- 虽然论文验证了子集代表性，但仍不是完整 benchmark；
- 对低成功率或高方差任务，如 OpenUAV，100 episode 可能不足以稳定反映性能；
- 与完整 split 的训练型方法直接比较时存在一定偏差风险。

### 8.4 真实机器人实验缺少定量指标

- 真实部署主要是 qualitative validation；
- 没有系统报告每个平台的任务数量、成功率、平均耗时、失败率等；
- 因此真实世界泛化能力仍需更严格验证。

### 8.5 停止判定仍是主要失败来源

- 失败分析显示，错误目标处提前停止和到达目标附近但未停止占很大比例；
- 说明系统对“是否已经完成任务”的判断仍不稳定。

### 8.6 视觉 grounding 仍有限

- 对大区域目标，如“走廊”“大厅”，bounding box 定位不稳定；
- 颜色、细粒度视觉属性识别也会影响 EQA 表现；
- 论文也承认需要引入 SAM、Grounding DINO 等工具辅助。

### 8.7 对动态环境和社交导航支持不足

- 动态障碍、人类行为、社交避让主要由低层控制器被动处理；
- 高层语言模型尚未显式建模人类意图和动态交互。

### 8.8 与训练型方法的比较存在范式差异

- Uni-LaViRA 借助强大通用 MLLM 的预训练知识；
- 训练型 VLA 依赖机器人数据训练；
- 两者资源来源不同，简单比较“训练样本数”并不能完全反映总成本。
