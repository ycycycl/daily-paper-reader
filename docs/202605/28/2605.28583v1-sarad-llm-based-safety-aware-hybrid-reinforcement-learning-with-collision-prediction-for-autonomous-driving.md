## 摘要
确保自动驾驶系统决策的安全性与效率仍是一项根本性挑战。传统深度强化学习（DRL）存在不安全的随机探索和收敛缓慢问题，而大语言模型（LLM）在实时推理操作中表现出固有的延迟。为应对这些局限，本文提出了 SARAD，一种将 LLM 与 DRL 协同结合的安全感知混合框架，用于自动驾驶。SARAD 以检索增强生成（RAG）强化的、由 LLM 指导的决策替代 DRL 的随机探索，这些决策来源于动态专家知识库。本文提出了一种注意力判别器，用于将 LLM 的先验知识融入 DRL 策略优化。此外，还设计了一个利用历史碰撞数据微调的碰撞预测模块，以进一步提升车辆安全性。大量实验表明，SARAD 在 Highway-Env 仿真器中实现了显著的性能提升，验证了所提出模型在自动驾驶中的有效性。

## Abstract
Ensuring both safety and efficiency in decision-making for autonomous driving systems remains a fundamental challenge. Traditional Deep Reinforcement Learning (DRL) suffers from unsafe random exploration and slow convergence, while Large Language Models (LLMs) demonstrate inherent latency in real-time inference operations. To address these limitations, this paper proposes SARAD, a novel safety-aware hybrid framework that synergizes LLMs and DRL for autonomous driving. SARAD substitutes the random exploration of DRL with Retrieval-Augmented Generation (RAG)-enhanced, LLM-guided decisions sourced from a dynamic expert knowledge repository. An attention discriminator is proposed to integrate the prior knowledge of LLMs into DRL policy optimization. A collision predictor module, fine-tuned with historical collision data, is further designed to improve vehicle safety. Extensive experiments show that SARAD achieves significant performance improvements in the Highway-Env simulator, validating the effectiveness of the proposed model in autonomous driving.

\[
MDP = (S, A, P, R, \gamma)

其中：

- \(S\)：状态空间；
- \(A\)：动作空间；
- \(P\)：状态转移概率；
- \(R\)：奖励函数；

状态包含自车与周围车辆的位置和速度信息：

### 2.2 SARAD 总体框架

SARAD 主要由三个模块组成：

1. **LLM 增强的 DRL 框架与注意力判别器**
2. **基于专家知识库的 RAG 与反思模块**
3. **碰撞预测器与规则安全策略**

整体流程可以概括为：

- 当前车辆状态被转换为文本场景描述；
- RAG 从专家知识库中检索相似驾驶经验；
- LLM 基于当前状态与检索经验生成驾驶动作建议；
- 碰撞预测器评估该动作风险；
- 若风险较低，则动作可用于指导 DRL 探索；
- 若风险较高，则触发规则安全策略；
- 若发生碰撞，则 LLM 对失败案例进行反思，并将反思结果写入知识库；
- DRL 通过注意力判别器吸收 LLM 的先验行为知识，逐步学习自主策略。

---

### 2.3 用 LLM 指导探索替代随机探索

传统 DQN 使用 ε-greedy 策略：

- 以较大概率选择当前最优动作；
- 以 ε 概率随机探索。

SARAD 将随机探索替换为 **LLM 专家探索**：

- 在早期训练阶段，较多使用 LLM 生成动作；
- 随着训练推进，逐渐降低 LLM 调用频率；
- 最终让 DRL 策略独立执行。

论文称这一机制为 **ε-expert exploration**。

其意义在于：

- 避免随机动作导致危险驾驶行为；
- 利用 LLM 的语义推理和驾驶常识指导初期探索；
- 加速 DRL 收敛；
- 减少训练初期策略不稳定性。

### 2.4 注意力判别器：融合 LLM 先验与 DRL 策略

论文设计了一个带注意力机制的判别器 \(D(s_t, a_t)\)，用于衡量智能体动作与 LLM 专家动作之间的语义一致性。

该判别器不是简单做行为克隆，而是作为一种 **奖励塑形机制**：

- 不强迫 DRL 完全复制 LLM；
- 而是给与 LLM 行为一致的动作额外奖励；
- 允许 DRL 在环境奖励更优时偏离 LLM 建议。

技术细节包括：

- 状态输入先经过卷积层；
- 再经过 Squeeze-and-Excitation，SE，注意力模块；
- SE 模块输出通道注意力权重：

\[
s = \sigma(W_2 \delta(W_1 z))
\]

其中：

- \(z\)：压缩后的通道描述；
- \(W_1, W_2\)：可学习参数；
- \(\delta\)：ReLU 激活；
- \(\sigma\)：Sigmoid 激活。

随后：

- 状态特征被展平；
- 与 one-hot 编码的动作拼接；
- 输入全连接层；
- 输出判别概率 \(D(s_t, a_t)\)。

模仿奖励定义为：

\[
R_{imit} = -\log(1 - D(s_t, a_t) + \eta)
\]

原始奖励为：

\[
R_{origin} = a \frac{v - v_{min}}{v_{max} - v_{min}} - b \cdot I_{col}
\]

总奖励为：

\[
R_{total} = R_{origin} + \beta R_{imit}
\]

其中：

- \(v\)：自车速度；
- \(I_{col}\)：碰撞指示变量；
- \(a,b,\beta,\eta\)：超参数。

判别器损失为：

\[
L_D = -\frac{1}{B} \sum_{i=1}^{B}
[\log D(s_i^{ex}, a_i^{ex}) + \log(1 - D(s_i^{ag}, a_i^{ag}))]
\]

DQN 策略网络使用 Huber loss 优化：

\[
L_{policy} = \frac{1}{B} \sum_{i=1}^{B}
L(Q(s_t^i, a_t^i), R_{total}^i + \gamma \max_a Q_{target}(s_{t+1}^i, a))
\]

该机制与 GAIL 有一定相似性，但论文强调二者不同：

- GAIL 主要追求专家与智能体分布匹配；
- SARAD 的判别器更多用于衡量动作与 LLM 逻辑先验的语义一致性，并作为软正则项辅助策略优化。

### 2.5 RAG 专家知识库与反思模块

由于预训练 LLM 不一定理解具体的 highway-env 驾驶环境，直接使用可能产生不可靠决策。为此，论文构建了动态专家知识库，并引入 RAG。

#### RAG 机制流程

- 在无碰撞情况下，系统以 20% 概率保存驾驶经验；
- 保存内容包括：
  - 当前状态；
  - 环境观测；
  - 决策动作；
  - 环境反馈；
- 这些数据被编码为低维稠密向量；
- 存入向量数据库；
- 每次决策前，系统将当前状态编码为 embedding；
- 从向量数据库中检索 top-k 个最相似历史经验；
- 将检索结果转写为结构化自然语言；
- 与当前场景描述拼接为 RAG prompt；
- 输入 LLM 生成动作建议。

论文使用：

- Chroma 向量数据库；
- Text-embedding-v2；
- HNSW 检索；
- MMR 重排序或多样性增强。

#### 反思模块

当发生碰撞时：

- 系统回溯导致碰撞的历史动作；
- 构造反思 prompt；
- 要求 LLM 分析碰撞原因；
- 生成替代避险策略；
- 将反思结果写入专家知识库。

该机制旨在提升系统对长尾危险场景的适应能力。

### 2.6 碰撞预测器与规则安全层

SARAD 还设计了一个独立碰撞预测器，用作最终安全防线。

其功能是：

- 对候选动作进行碰撞风险评估；
- 输出碰撞概率；
- 若风险超过阈值，则覆盖原决策；
- 触发基于规则的安全策略。

安全策略包括：

- 紧急制动；
- 保持车道；
- 保持最小安全距离。

碰撞预测器基于预训练 0.6B 参数 LLM 微调得到：

- 使用 Qwen3-0.6B；
- 任务为二分类碰撞风险预测；
- 输入为状态—动作表示；
- 输出为碰撞概率 \([0,1]\)。

训练数据来自仿真环境：

- 负样本：碰撞前三帧的状态—动作对；
- 正样本：无碰撞风险时刻的三帧状态—动作对；
- 正负样本各 20,000 条；
- 总计 40,000 条样本；
- 样本打乱后用于微调。

微调方法：

- LoRA；
- rank = 8；
- 学习率 \(5 \times 10^{-5}\)；
- cosine scheduler；
- 训练 5 个 epoch；
- FP16 精度。

此外，论文还使用 GBDT + Logistic Regression 作为碰撞预测器的机器学习对照基线。

| SARAD-DLH | 22.7478 | 0.9132 | 21.4075 |

后 80k steps 结果如下：

| 模型 | Avg Length | Avg Reward | Total Reward |
|---|---:|---:|---:|
| Vanilla DQN | 23.2547 | 0.9417 | 22.5967 |
| SARAD-G | 24.3784 | 0.9501 | 23.8556 |
| SARAD-DG | 24.4080 | 0.9475 | 23.8098 |
| SARAD-DGH | 24.3857 | 0.9485 | 23.8439 |
| SARAD-DLH | 24.8768 | 0.9479 | 24.1935 |

可以看到：

- 所有 SARAD 变体在 Avg Length 和 Total Reward 上总体优于 Vanilla DQN；
- SARAD-DGH 在全训练过程中奖励指标较高；
- SARAD-DLH 在后期训练阶段综合表现最好，尤其 Avg Length 和 Total Reward 最高；
- LLM 碰撞预测器有助于提升后期安全性和稳定性。

---

### 5.3 实验充分性与公平性

实验具有一定说服力，但仍存在不足。

#### 较充分的方面

- 包含了 Vanilla DQN 基线；
- 做了多个模块组合的消融；
- 区分了全训练过程与后期训练表现；
- 对碰撞预测器设置了传统机器学习对照；
- 使用了标准 highway-env 仿真环境。

#### 不够充分的方面

- 只在一个仿真环境中验证，没有跨数据集或跨模拟器测试；
- 只考虑高速公路场景，没有城市道路、交叉口、环岛、行人、自行车等复杂场景；
- 训练曲线来自单次代表性运行，而不是多随机种子平均；
- 没有报告标准差、置信区间或显著性检验；
- 没有直接报告碰撞率、安全违规次数、TTC 等更直接的安全指标；
- 没有系统评估 LLM 调用成本和延迟；
- 对比方法较少，未与 Double DQN、PPO、SAC、GAIL、LLM-only、其他 LLM+RL 方法等进行系统比较。

因此，实验能初步证明 SARAD 的有效性，但客观性和充分性仍有限。

---

## 6. 主要结论与发现

论文得出的主要结论包括：

- SARAD 能够有效缓解传统 DRL 早期随机探索带来的不安全性和低效率问题。
- 使用 LLM 指导探索可以加快策略学习，并提升训练后期性能。
- 注意力判别器可以将 LLM 的先验行为知识以软奖励形式融入 DRL，而非简单行为克隆。
- RAG 专家知识库有助于提升 LLM 在特定驾驶环境中的决策质量。
- 反思模块可以将碰撞失败案例转化为新的经验，从而增强长尾场景下的鲁棒性。
- 碰撞预测器作为独立安全层，可以在高风险动作执行前触发规则保护策略。
- 在 highway-env 中，SARAD 系列方法整体优于 Vanilla DQN。
- SARAD-DLH 在后期训练阶段表现最好，说明 LLM-based 碰撞预测器对稳定性和安全性有明显帮助。

---

## 7. 优点与亮点

### 7.1 方法设计亮点

- **LLM 与 DRL 分工合理**
  - LLM 不直接承担实时控制，而是在训练阶段辅助探索；
  - DRL 最终负责快速推理，规避 LLM 实时延迟问题。

- **用专家探索替代随机探索**
  - 针对自动驾驶中随机探索危险的问题提出了较直接的解决方案；
  - 有助于提升训练初期安全性。

- **注意力判别器设计较有新意**
  - 不是硬模仿 LLM；
  - 而是通过 imitation reward 软约束策略；
  - 能减少行为克隆的泛化问题。

- **RAG 动态知识库增强环境适配性**
  - 通过历史经验检索弥补 LLM 对具体驾驶环境理解不足；
  - 知识库在训练中持续更新。

- **反思机制提升长尾场景学习能力**
  - 碰撞后自动生成失败原因分析与替代策略；
  - 将失败经验纳入知识库。

- **碰撞预测器提供额外安全防线**
  - 将决策模块与安全评估模块解耦；
  - 对自动驾驶安全系统设计具有现实意义。

---

### 7.2 实验设计亮点

- 采用标准 highway-env 环境，便于复现和对比；
- 进行了多模块消融实验；
- 对传统机器学习碰撞预测器与 LLM 碰撞预测器进行了间接比较；
- 同时关注训练全过程和训练后期性能；
- 指标包括运行时长、平均奖励和总奖励，能够从一定程度反映安全性与效率。

---

## 8. 不足与局限

### 8.1 实验场景覆盖有限

- 仅在 highway-env 中验证；
- 场景较简单，主要是高速公路车道变换与速度控制；
- 未覆盖城市道路、复杂交互、无保护左转、行人穿行、交通灯、施工区域等真实自动驾驶难点。

---

### 8.2 安全性评估指标不够直接

论文强调安全，但实验指标主要是：

- episode 长度；
- 平均奖励；
- 总奖励。

这些指标可以间接反映安全性，但不足以完整证明安全能力。缺少：

- 碰撞率；
- 近碰撞次数；
- 最小 TTC；
- 急刹次数；
- 违规变道次数；
- 舒适性指标；
- 安全规则触发频率；
- 碰撞预测器准确率、召回率、AUC 等。

---

### 8.3 统计可靠性不足

论文说明训练曲线是单次代表性运行，而不是多随机种子平均。

这会带来：

- 结果可能受随机初始化影响；
- 无法判断性能提升是否统计显著；
- 不利于客观比较不同模型。

---

### 8.4 对比基线不够丰富

主要对比 Vanilla DQN，略显不足。

缺少与以下方法的比较：

- Double DQN；
- Dueling DQN；
- PPO；
- SAC；
- GAIL；
- IRL；
- 纯 LLM 决策方法；
- 其他 LLM-guided RL 方法；
- Rule-based driving policy；
- Model Predictive Control，MPC。

---

### 8.5 LLM 延迟与部署成本未充分量化

论文提出通过 ε-expert exploration 将 LLM 主要用于早期训练，从而减轻推理延迟问题。

但论文没有具体报告：

- 每次 LLM 决策平均延迟；
- RAG 检索时间；
- 碰撞预测器推理时间；
- SARAD 训练总耗时；
- 与 DQN 相比的额外成本；
- 在真实车载平台上的可部署性。

因此，实时性结论仍需更多实证支持。

---

### 8.6 碰撞预测数据构造可能存在偏差

碰撞预测器训练数据来自仿真环境：

- 负样本选自碰撞前三帧；
- 正样本选自无风险时刻。

潜在问题包括：

- 样本定义依赖具体仿真环境；
- 三帧窗口是否足够捕捉风险不确定；
- 正负样本可能过于理想化；
- 对真实道路分布泛化能力未知；
- 若预测器误判，可能导致过度保守或安全失效。

---

### 8.7 LLM 决策可靠性仍存在不确定性

虽然 RAG 与反思模块可以增强 LLM，但 LLM 仍可能存在：

- 幻觉；
- 对数值状态理解不稳定；
- 对边界场景推理错误；
- prompt 敏感性；
- 动作输出格式错误；
- 对高速连续动态场景响应不足。

论文未系统分析这些问题。

---

### 8.8 真实自动驾驶应用仍有距离

SARAD 在仿真环境中表现较好，但真实部署还面临：

- 感知噪声；
- 不完全观测；
- 多智能体意图预测；
- 车辆动力学约束；
- 乘坐舒适性；
- 法规约束；
- 传感器延迟；
- sim-to-real gap；
- 安全认证与可验证性。

因此，当前结果更适合作为方法验证，而不是直接证明可用于真实自动驾驶系统。

---

## 总体评价

SARAD 提出了一个较完整的 LLM + DRL 自动驾驶决策框架，核心贡献在于将 LLM 从实时决策主控角色转变为 DRL 训练过程中的专家指导者，并通过 RAG、注意力判别器、反思机制和碰撞预测器构建安全增强闭环。实验结果表明，该方法在 highway-env 中相比 Vanilla DQN 有更好的训练效率和后期性能。

不过，论文实验仍主要停留在单一仿真场景和有限基线上，缺少多随机种子、显著性检验、直接安全指标和实际推理开销分析。整体而言，SARAD 是一个有启发性的安全感知混合强化学习框架，但其真实自动驾驶适用性仍需要更复杂场景、更严格安全评估和更充分算力延迟分析来验证。

（完）
