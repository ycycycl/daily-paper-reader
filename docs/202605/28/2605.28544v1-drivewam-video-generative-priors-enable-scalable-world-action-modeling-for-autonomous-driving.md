## 摘要
预训练基础模型已成为端到端自动驾驶的重要基础。与主要在静态图像-文本对上预训练的视觉-语言模型不同，视频生成模型能够捕捉时间动态和运动先验，因而天然适用于驾驶场景。我们提出 DriveWAM，这是一种驾驶世界-动作模型，可将预训练的视频扩散 Transformer 适配为自回归视频-动作策略。DriveWAM 将视频流和动作流组织为统一的时间 token 序列，并在联合流匹配目标下进行训练，在保留预训练视频生成架构的同时，将其大规模视频先验适配到动作生成中。为融入高层次场景理解，我们引入了场景演化驾驶引导，其中冻结的视觉语言模型生成面向片段的语义意图，以指导视频-动作生成。为使长时域 rollout 保持有界，我们进一步引入选择性 KV 记忆，通过在推理时基于相关性-冗余度的缓存选择，维护有界且具备模态感知的视频与动作记忆池。在 NAVSIM 和 PhysicalAI-Autonomous-Vehicles 基准上的实验表明，DriveWAM 实现了强大的规划性能；从 4k 到 100k 驾驶片段的数据扩展研究进一步证实了世界-动作建模在端到端自动驾驶中的扩展潜力。

## Abstract
Pretrained foundation models have become an important basis for end-to-end autonomous driving. In contrast to vision-language models pretrained primarily on static image-text pairs, video generative models capture temporal dynamics and motion priors that are naturally suited for driving. We present DriveWAM, a driving world-action model that adapts a pretrained video diffusion transformer into an autoregressive video-action policy. DriveWAM organizes video and action streams into a unified temporal token sequence and trains them under a joint flow-matching objective, preserving the pretrained video-generation architecture while adapting its large-scale video priors to action generation. To incorporate high-level scene understanding, we introduce scene-evolving driving guidance, where a frozen VLM produces chunk-specific semantic intent to guide video-action generation. To keep long-horizon rollout bounded, we further introduce selective KV memory, which maintains bounded modality-aware video and action memory pools through relevance-redundancy cache selection at inference time. Experiments on NAVSIM and the PhysicalAI-Autonomous-Vehicles benchmark show that DriveWAM achieves strong planning performance, and a data-scaling study from 4k to 100k driving clips further confirms the scaling potential of world-action modeling for end-to-end autonomous driving.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 总体框架

DriveWAM 的核心是将一个预训练视频扩散 Transformer 改造成统一的视频-动作策略模型。整体包含三个关键模块：

1. **自回归视频-动作生成**
2. **场景演化驾驶语义引导**
3. **选择性 KV 记忆机制**

模型在每个决策步根据历史视频、历史动作、当前自车状态和文本语义引导，预测下一段未来视频 latent 和对应动作。

---

### 2.2 视频与动作统一 token 化

- 输入驾驶片段被划分为连续的 chunk。
- 对于第 \(k\) 个 chunk：
  - 视频帧 \(x_k\) 通过预训练视频 VAE 编码为视频 latent：
    \[
    z_k = VAE(x_k)
    \]
  - 自车动作 \(a_k\) 通过 MLP 动作编码器编码为动作 token：
    \[
    u_k = E_a(a_k)
    \]
  - 历史上下文表示为：
    \[
    H_k = \{(z_i, u_i)\}_{i \leq k}
    \]

- 动作以归一化的自车坐标系平移量和 yaw 变化表示。
- 视频 token 与动作 token 被组织成统一的时序 token 序列，送入同一个 Transformer。

---

### 2.3 自回归世界-动作生成

DriveWAM 将驾驶预测拆成两个阶段：

#### 1）未来世界建模

- 模型先预测下一 chunk 的未来视频 latent。
- 使用 rectified-flow / flow matching 训练目标。
- 给定噪声化的视频 latent \(z_{k+1,\tau}\)，模型预测其速度场：
  \[
  \hat{v}_{z_{k+1,\tau}} = T_\omega(z_{k+1,\tau}; H_k, e_k, g_k, \tau)
  \]
- 其中：
  - \(H_k\)：历史视频-动作上下文；
  - \(e_k\)：当前 ego state，如速度、加速度、曲率；
  - \(g_k\)：VLM 生成的语义引导；
  - \(\tau\)：flow timestep。

#### 2）逆动力学动作生成

- 在生成未来视频 latent 后，再生成动作。
- 动作同样通过 flow matching 建模。
- 模型预测动作速度场：
  \[
  \hat{v}_{a_{k+1,\tau}} = D_a(T_\omega(u_{k+1,\tau}; \tilde{z}_{k+1}, H_k, e_k, g_k, \tau))
  \]
- 其中：
  - \(D_a\) 是 MLP 动作解码器；
  - \(\tilde{z}_{k+1}\) 在训练时使用真实未来视频 latent，推理时使用模型生成的未来 latent。

- 这种设计使动作生成不是独立轨迹头，而是类似于对预测未来世界的**逆动力学读出**。

---

### 2.4 联合流匹配训练目标

DriveWAM 同时优化视频生成和动作生成：

\[
L = E_{k,\tau} \left[
\|\hat{v}_{z_{k+1,\tau}} - v_{z_{k+1,\tau}}\|_2^2
+ \beta_a \|\hat{v}_{a_{k+1,\tau}} - v_{a_{k+1,\tau}}\|_2^2
\right]
\]

- 第一项保持视频生成能力和时空先验；
- 第二项学习将未来世界预测转化为动作；
- \(\beta_a\) 控制动作损失权重，实验中设为 1.0。

---

### 2.5 场景演化驾驶语义引导

视频生成模型虽然擅长短期动态预测，但缺少高层语义规划能力。DriveWAM 因此引入冻结 VLM 作为语义指导器。

- 使用冻结的 **Qwen3-VL-8B**。
- 每个 chunk 查询一次 VLM。
- VLM 输入包括：
  - 当前可见前视图像；
  - 最近 ego 轨迹；
  - 即将到来的 route command，如 straight / left / right。
- 输出简短文本引导：
  \[
  g_k = \Phi_{VLM}(x_k, a_k, c_k)
  \]

该文本描述当前场景和下一时段驾驶意图，例如：
- 保持直行；
- 等待红灯；
- 给行人让行；
- 慢速左转；
- 与前车保持安全距离。

#### 防止未来信息泄漏

- 每个 chunk 的视频-动作 token 只能 attend 到对应的 chunk guidance。
- 作者使用 block-diagonal text mask，实现**局部时间语义注入**。
- 这样避免模型在训练时看到未来 chunk 的文本引导，保证因果一致性。

---

### 2.6 选择性 KV 记忆机制

长时域自回归 rollout 需要保存历史上下文，但完整 KV cache 会随时间线性增长。DriveWAM 提出训练无关的选择性 KV memory。

#### 模态感知记忆池

分成两个缓存池：

- 视频记忆池 \(H_k^v\)，预算 \(B_v\)
- 动作记忆池 \(H_k^a\)，预算 \(B_a\)

这样可以避免视频 token 数量过多导致动作历史被淹没。

#### 相关性-冗余度打分

对于每个缓存 token，计算：

1. **相关性**：当前 query 对该历史 token 的平均注意力权重；
2. **冗余度**：该 token 与其他缓存 key 的平均余弦相似度。

最终保留分数为：

\[
s_j^m = \lambda \rho_j^m - (1-\lambda)\eta_j^m
\]

- \(\rho\)：相关性；
- \(\eta\)：冗余度；
- \(\lambda\)：平衡系数，实验中为 0.07。

高相关、低冗余的 token 被保留；低相关、高冗余的 token 被丢弃。

  - DriveWAM 使用 5B 视频模型加 8B VLM，参数规模和算力需求较高；
  - NAVSIM 中部分方法使用传感器配置不同，例如多视角相机、LiDAR 或单视角相机，公平性需要具体区分。

---

## 6. 主要结论与发现

- DriveWAM 表明，预训练视频生成模型可以被有效改造成端到端自动驾驶世界-动作策略。
- 相比以 VLM 为核心的驾驶策略，视频生成模型能更好地提供时序动态和未来场景演化先验。
- 联合视频-动作 flow matching 对性能非常关键：
  - 仅有预训练初始化但无视频监督会显著退化；
  - 从零训练也不如使用预训练视频先验。
- 场景演化 VLM guidance 能稳定提升性能，并且在不同数据规模下都有效。
- Selective KV memory 可以在接近 full cache 性能的同时显著降低长时域推理内存和计算开销。
- 数据规模从 4k 增至 100k clips 时，ADE/FDE 持续改善，说明该方法尚未达到饱和，具有扩展潜力。

---

## 7. 优点与亮点

- **方法范式新颖**：
  - 不是将视频生成作为辅助任务，而是直接将视频扩散模型作为驾驶策略 backbone。

- **统一视频-动作建模**：
  - 视频与动作共享 Transformer；
  - 通过联合 flow matching 同时学习未来世界和自车动作。

- **动作生成设计合理**：
  - 先预测未来世界，再生成动作；
  - 将动作视为基于未来 latent 的逆动力学输出，而不是独立轨迹回归。

- **语义与动态建模互补**：
  - 视频模型负责低层时空动态；
  - 冻结 VLM 负责高层场景理解和驾驶意图。

- **因果性处理较严谨**：
  - VLM 只使用当前和历史信息；
  - 通过局部文本 attention mask 避免未来 guidance 泄漏。

- **长时域推理机制实用**：
  - Selective KV memory 在保持性能的同时显著降低内存与计算开销；
  - 比 FIFO 更符合驾驶场景需求，因为旧信息可能仍然重要。

- **实验覆盖较全面**：
  - 有主实验、消融、数据扩展、效率分析和定性结果。

---

## 8. 不足与局限

- **计算成本较高**：
  - 训练使用 48 张 H20 GPU；
  - 模型包含 5B 视频 backbone 和 8B VLM；
  - 对实际车端部署存在明显资源压力。

- **推理复杂度较高**：
  - 默认需要 VLM guidance、视频生成和动作 denoising 三个阶段；
  - 虽然 5-step action denoising 能降低延迟，但整体系统仍比普通轨迹预测器复杂。

- **PhysicalAI 测试集存在潜在选择偏差**：
  - 作者构建了自定义 1,000-clip 测试子集；
  - 筛选过程依赖 VLM 标签和 interest score；
  - 可能导致评测分布与完整测试集不同。

- **对 VLM guidance 质量敏感**：
  - 如果 VLM 对交通灯、行人、车道、施工等理解错误，可能引导错误决策；
  - 冻结 VLM 本身可能存在幻觉或不稳定描述。

- **闭环评估不足**：
  - NAVSIM 是非反应式 / 数据驱动评估；
  - PhysicalAI 主要报告轨迹 ADE/FDE；
  - 论文没有充分展示复杂闭环交互、反事实测试或真实车部署结果。

- **传感器配置有限**：
  - DriveWAM 主要使用单前视相机；
  - 虽然这是优势之一，但对复杂路口、侧向车辆、盲区和遮挡场景可能不足；
  - 未系统评估多相机、LiDAR 或 BEV 融合设置。

- **安全性与可解释性仍有限**：
  - 虽然生成未来视频提供一定可视化解释，但模型决策失败时如何诊断仍不清楚；
  - 没有深入讨论不确定性估计、风险约束或安全保证。

- **数据规模主张仍需更大范围验证**：
  - 论文展示了 4k 到 100k clips 的 scaling；
  - 但没有更大规模、更多城市、更多天气和跨域泛化结果；
  - 是否能像 VLA 或大型世界模型一样继续稳定扩展，还需后续验证。
