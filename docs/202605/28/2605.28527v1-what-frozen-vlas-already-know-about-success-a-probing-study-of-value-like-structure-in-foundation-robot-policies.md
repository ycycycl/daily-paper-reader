## 摘要
视觉—语言—动作（VLA）策略被训练用于模仿动作；其损失函数从不要求它们估计奖励、进展或未来成功。然而，它们的冻结表示仍然携带此类信息，并且可以在不重新训练策略的情况下将其读出并用于指导动作选择。基于 LIBERO-Goal 上成功与失败混合的操作轨迹，我们使用冻结特征上的轻量级线性探针恢复蒙特卡洛结果目标。结果目标可稳定地由 OpenVLA、Pi0.5、DINOv2 和 CLIP 特征预测，而由基于进展、剩余时间、任务身份或本体感知构建的基线进行预测时，效果显著较差。为排除任务和时间捷径，我们在相同任务、相同时间步匹配的比较下评估这些探针：Pi0.5 探针的成对排序准确率仍达到约 92%，而标签打乱的对照保持在随机水平。将同一探针用作对采样的 Pi0.5 动作前缀的测试时选择器，这一离线发现被转化为行为表现：在 push-plate 任务上，成功率从贪婪解码下的 26.7% 提升至 44.3%，并且在 wine-rack 任务上也出现了第二个正向案例。这些提升并非普遍适用，并且需要额外的推理计算，但其基本发现是清晰的：冻结的 VLA 已经编码了关于成功的信息，而其模仿目标从未显式要求这些信息。

## Abstract
Vision--language--action (VLA) policies are trained to imitate actions; their loss never asks them to estimate reward, progress, or future success. Their frozen representations nevertheless carry such information, and it can be read out and used to guide action choice without retraining the policy. From mixed successful and failed manipulation trajectories on LIBERO-Goal, we recover Monte-Carlo outcome targets using lightweight linear probes on frozen features. The targets are consistently predictable from OpenVLA, Pi0.5, DINOv2, and CLIP features, and substantially less so from baselines built on progress, time-to-go, task identity, or proprioception. To rule out task and temporal shortcuts, we evaluate the probes under same-task, same-timestep matched comparisons: Pi0.5 probes still reach roughly 92% pairwise ordering accuracy, while label-shuffled controls stay at chance. Used as a test-time selector over sampled Pi0.5 action prefixes, the same probe turns this offline finding into behavior: on push-plate, success rises from 26.7% under greedy decoding to 44.3%, with a second positive case on wine-rack. The gains are not universal and require additional inference compute, but the underlying finding is clean: frozen VLAs already encode information about success that their imitation objective never explicitly demands.

当前主流机器人基础策略，如 RT-1、RT-2、OpenVLA、π0、π0.5、SmolVLA 等，通常被训练为条件动作生成器：输入视觉观测、语言指令和机器人状态，输出动作或动作序列。它们的训练目标主要是**模仿示范动作**，并不显式学习奖励函数、成功概率、进度估计或价值函数。

- 冻结的 VLA / VLM / 视觉编码器表征中，是否已经编码了与最终成功相关的信息？
- 如果有，这些信息是否只是任务 ID、时间步、进度等数据集捷径？
- 更进一步，这种被读出的信号能否在测试时真正改变动作选择、提升任务成功率？

论文的整体含义在于：  
如果冻结 VLA 表征中确实存在可线性读出的成功相关信号，那么 VLA 不只是一个“黑盒动作生成器”，其内部还可能包含一种未被显式监督训练出来的、与结果排序相关的结构。这为**不更新策略参数、仅通过测试时动作选择改进机器人行为**提供了一种新思路。

## 2. 方法论：核心思想、关键技术细节与流程

论文提出了一个从“离线探测”到“测试时选择”的三阶段协议，核心目标不是训练一个新的策略，而是分析并利用冻结表征中的类价值信号。

### 2.1 构造类价值目标

论文使用成功 / 失败轨迹构造 Monte-Carlo 风格的 outcome target。

对于一条成功轨迹，长度为 \(T\)，在时间步 \(t\) 的目标值定义为：

\[
v_t = \gamma^{T-1-t}, \quad \gamma = 0.99
\]

含义是：

- 越接近成功终点的状态，目标值越大；
- 远离成功终点的状态，目标值较小；
- 折扣因子 \(\gamma = 0.99\)，对应约 100 步的有效时间尺度。

对于失败轨迹：

\[
v_t = 0
\]

作者强调，这不是严格的强化学习 Bellman value function，而是从完整轨迹结果构造出的 **value-like target**，因此称为“类价值信号”。

### 2.2 冻结特征提取与线性探针

给定冻结模型或编码器 \(m\)，从观测和语言指令中提取特征：

\[
\phi_m(o_t, g) \in \mathbb{R}^d
\]

然后训练一个线性 ridge regression 探针：

\[
\hat{v}_t = w^\top \tilde{\phi}_m(o_t, g) + b
\]

其中：

- \(\tilde{\phi}\) 表示使用训练集统计量标准化后的特征；
- \(w,b\) 是线性探针参数；
- ridge 正则强度通过交叉验证选择；
- 冻结 VLA / VLM / 视觉模型本身不更新参数。

评价指标包括：

- \(R^2\)
- Spearman 相关系数
- RMSE / MAE
- per-task \(R^2\)

论文使用两类数据划分：

- **Demo split**：同一任务内不同轨迹划分训练 / 测试，考察已见任务上的泛化。
- **Task split**：按任务划分训练 / 测试，测试集来自未见任务，更严格。

### 2.3 去混淆：同任务、同时间步匹配控制

仅看离线 \(R^2\) 可能被捷径污染，例如：

- 某些任务本身成功率更高，探针可能只记住任务 ID；
- 时间步和进度相关，探针可能只读取 elapsed time；
- 成功和失败轨迹可能有粗糙视觉差异。

为排除这些问题，论文设计了 **same-task, same-timestep matched-pair control**。

具体做法：

- 将样本按“任务名 + 时间步”分组；
- 在每个组内构造高 value 和低 value 样本对；
- 要求两者 value-like label 至少相差 0.20；
- 探针若给高标签样本更高分，则判为正确；
- 若打平，则计 0.5 分。

成对准确率为：

\[
\frac{1}{|P|}\sum_{(i,j)\in P}
\left[
\mathbf{1}\{\hat{v}_j > \hat{v}_i\}
+
\frac{1}{2}\mathbf{1}\{\hat{v}_j = \hat{v}_i\}
\right]
\]

其中 \(v_j > v_i\)，且 \(i,j\) 来自同一任务、同一时间步。

此外，作者还做了 **label-shuffle negative control**，随机打乱 pair 中高低标签身份。如果探针只是利用任务或时间捷径，打乱后仍可能有高准确率；如果真实信号有效，打乱后应回到 50% 左右。

### 2.4 测试时动作前缀选择

论文进一步将离线探针用于在线动作选择。

在每个 replanning step：

1. 使用冻结 Pi0.5 策略采样 \(K=16\) 个动作 chunk；
2. 对每个 chunk 只取前 \(h=5\) 个动作作为候选前缀；
3. 将模拟器恢复到当前状态；
4. 执行候选动作前缀；
5. 记录：
   - 是否短期 rollout 已成功；
   - 模拟器 reward；
   - 执行前缀后的观测；
   - 通过冻结特征和线性探针得到的 score；
6. 若有候选前缀在短 rollout 中成功，则选择探针分数最高的成功前缀；
7. 若无候选直接成功，则选择归一化 reward 与归一化 probe score 之和最大的前缀：

\[
k^* = \arg\max_k [z(r_k) + z(s_k)]
\]

最终，模拟器恢复到原始状态，只执行被选中的动作前缀。

需要注意的是，这个过程依赖模拟器来评估候选动作未来结果，因此更像是一种**诊断性干预工具**，而不是可直接部署到真实机器人上的高效控制器。

- 探针目标不是严格 value function，不能证明模型内部有 Bellman 一致价值估计。

总体而言，论文实验在“诊断性研究”层面较充分、客观；但若作为实用控制方法，其验证范围仍有限。

---

## 6. 主要结论与发现

论文得出以下主要结论。

### 6.1 冻结表征中可读出类价值信号

在 LIBERO-Goal 上，多种冻结表示都能较好预测 outcome-derived value target。

代表性结果：

- Pi0.5 task split \(R^2 \approx 0.5510\)
- OpenVLA-OFT task split \(R^2 \approx 0.5505\)
- OpenVLA task split \(R^2 \approx 0.5493\)
- DINOv2 task split \(R^2 \approx 0.5104\)
- CLIP task split \(R^2 \approx 0.5095\)

相比之下：

- progress / time-to-go 只有约 \(R^2 \approx 0.03\)
- task one-hot 接近 0
- proprioception 明显较弱

说明成功相关信号不是简单由时间、进度或任务 ID 解释。

### 6.2 信号不是任务或时间捷径

在 same-task, same-timestep matched-pair control 中：

- primary Pi0.5 probe pairwise accuracy：94.22%
- label-shuffle control：50.05%
- 多 seed / layer 平均：92.16%
- shuffle 平均：49.67%

这表明即使固定任务和时间步，探针仍能区分更接近成功的状态，说明其不是简单记住任务或时间。

### 6.3 离线信号可转化为在线行为改进

在 Pi0.5 测试时动作选择中：

#### Push plate

- greedy：26.7%
- random：33.7%
- value-guided：44.3%
- 相比 random：+10.67 pp
- 相比 greedy：+17.67 pp
- McNemar \(p=0.00294\)

#### Wine rack

- greedy：35.7%
- random：36.0%
- value-guided：44.0%
- 相比 random：+8.00 pp
- 相比 greedy：+8.33 pp
- McNemar \(p=0.0264\)

#### Hard-3 aggregate

- greedy：31.1%
- random：36.9%
- value-guided：42.4%
- 相比 random：+5.56 pp
- 但 \(p=0.0614\)，作者将其视为 borderline。

### 6.4 提升不是普遍的

open middle drawer 上 value-guided 几乎没有明显收益：

- greedy：38.67%
- random：37.33%
- value-guided：39.33%

这说明：

- 探针并非在所有任务上都有效；
- 成功依赖候选动作集合中是否包含有意义的更优前缀；
- 任务结构会影响 value-like signal 是否能转化为控制收益。

## 7. 优点与亮点

### 7.1 问题设定有价值

论文提出了一个重要视角：  
VLA 虽然只通过模仿学习训练，但可能在表征中自然形成 outcome-relevant structure。这对理解机器人基础模型的内部能力非常有意义。

### 7.2 方法简单但有效

作者只使用线性 ridge probe，而不是复杂非线性模型。这样有两个好处：

- 可解释性更强；
- 如果线性探针都能读出信号，说明该信息在表征中较容易访问。

### 7.3 去混淆设计严谨

same-task, same-timestep matched-pair control 是论文最强的实验设计之一。它直接排除了两个最常见捷径：

- 任务 ID；
- 时间步 / 进度。

再加上 label-shuffle control，使结论更可信。

### 7.4 从离线探测走向在线行为验证

很多 probing 研究只证明信息“可被读出”。这篇论文进一步将探针用于动作选择，证明该信号可以影响机器人行为。这使结果从表征分析推进到了控制层面。

### 7.5 不夸大结果

论文明确报告：

- hard-3 aggregate 对 random 的提升只是 borderline；
- open middle drawer 是无收益边界案例；
- ceiling tasks 不纳入主要 claims；
- online 方法有很大计算成本；
- 探针不是 Bellman value function。

这种表述较客观。

---

## 8. 不足与局限

### 8.1 不是严格意义上的价值函数

论文构造的是 Monte-Carlo outcome target，而不是通过 Bellman equation 学到的 value function。因此不能说 VLA 内部已经学习了真正的强化学习价值函数。

更准确地说，它读出的是：

- 与最终成功相关；
- 与状态进展相关；
- 可用于排序的 outcome-like signal。

### 8.2 在线方法依赖模拟器

value-guided selection 需要：

- 保存 / 恢复模拟器状态；
- 对每个候选动作前缀 rollout；
- 提取新观测特征并评分。

这在真实机器人部署中很难直接实现。论文也承认它更像是诊断工具，而不是实用控制器。

### 8.3 计算成本较高

value-guided selection：

- 约为 random 的 2 倍 wall time；
- 比 greedy 慢 20 倍以上；
- 每多获得一个成功可能需要几十分钟额外计算。

因此它目前更适合分析和验证，不适合高效在线控制。

### 8.4 收益具有任务依赖性

在 push plate 和 wine rack 上有效，但在 open middle drawer 上几乎无效。说明该方法依赖：

- 候选动作集合质量；
- 任务是否有可区分的短期状态变化；
- 探针分数是否能反映该任务中的关键成功因素。

### 8.5 跨 benchmark 泛化有限

CALVIN-D 和 RobotWin 的结果显示：

- demo split 中能读出一定信号；
- 但 task split 明显变弱，RobotWin 上甚至出现负 \(R^2\)。

这表明类价值信号的跨任务、跨场景泛化仍不稳定。

### 8.6 缺少真实机器人验证

论文实验主要基于模拟环境和模拟器 rollout。尚未验证：

- 真实机器人传感噪声下是否稳定；
- 状态恢复不可用时如何评估候选；
- sim-to-real gap 是否影响探针排序。

### 8.7 缺少更强的 compute-matched 对照

value-guided selection 消耗大量额外计算。更公平的实用比较可能需要：

- 与同等计算预算下的随机搜索比较；
- 与外部 reward model 比较；
- 与 learned world model / visual MPC 比较；
- 与重新训练 value head 或 policy fine-tuning 比较。

---

# 总体评价

这篇论文的核心贡献不是提出一个立即可部署的机器人控制算法，而是提供了一个清晰的诊断证据：**冻结的 VLA / VLM / 视觉表征中已经包含可线性读出的成功相关结构，并且该结构在部分任务中能够转化为测试时动作选择收益。**

其最强之处在于：

- 多模型、多特征家族一致验证；
- 严格的任务和时间步匹配控制；
- 将 probing 结果接入在线选择；
- 对失败案例和计算成本保持诚实。

其主要限制在于：

- 依赖模拟器；
- 计算昂贵；
- 任务收益不普遍；
- 跨 benchmark 泛化有限；
- 并不构成严格的价值函数学习证明。

总体而言，这是一篇对理解 VLA 内部表征和测试时动作选择很有启发的论文，尤其适合作为后续研究“如何从冻结机器人基础模型中提取可用决策信号”的起点。
