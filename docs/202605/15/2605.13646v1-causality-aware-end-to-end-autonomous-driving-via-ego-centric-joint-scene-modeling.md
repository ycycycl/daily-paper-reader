## 摘要
端到端自动驾驶通过直接从传感器输入预测未来轨迹，绕过传统的模块化流程，近年来取得了显著进展。然而，现有方法往往忽视自车规划中的因果相互依赖关系，未能考虑自车与周围智能体之间的相互作用关系。这种因果层面的忽视会导致轨迹预测不一致且不可靠，尤其是在交互关键场景中，自车决策与邻近智能体行为必须进行联合推理。为解决这一局限，我们提出了 CaAD，一种因果感知的端到端自动驾驶框架，用于在共享的潜在场景表示中捕捉这些依赖关系。首先，我们提出了一个以自车为中心的联合因果建模模块，该模块建立在边缘预测分支之上，学习自车与交互相关智能体之间的因果依赖关系。其次，我们采用通过联合模态嵌入实现的因果感知策略对齐阶段，将随机自车策略与基于周围交通和地图上下文计算得到的面向规划的闭环反馈进行对齐。在 Bench2Drive 和 NAVSIM 基准测试中，CaAD 展现出强大的闭环规划性能，在 Bench2Drive 上取得了 87.53 的驾驶评分和 71.81 的成功率，并在 NAVSIM 上取得了 91.1 的 PDMS。

## Abstract
End-to-end autonomous driving, which bypasses traditional modular pipelines by directly predicting future trajectories from sensor inputs, has recently achieved substantial progress. However, existing methods often overlook the causal inter-dependencies in ego-vehicle planning, ignoring the reciprocal relations between the ego vehicle and surrounding agents. This causal oversight leads to inconsistent and unreliable trajectory predictions, especially in interaction-critical scenarios where ego decisions and neighboring agent behaviors must be reasoned about jointly. To address this limitation, we propose CaAD, a Causality-aware end-to-end Autonomous Driving framework that captures these dependencies within a shared latent scene representation. First, we propose a ego-centric joint-causal modeling module that builds on the marginal prediction branch, and learns causal dependencies between the ego vehicle and interaction-relevant agents. Second, we employ a causality-aware policy alignment stage implemented with joint-mode embeddings to align the stochastic ego policy with planning-oriented closed-loop feedback computed from surrounding traffic and map context. On the Bench2Drive and NAVSIM benchmarks, CaAD demonstrates strong closed-loop planning performance, achieving a Driving Score of 87.53 and Success Rate of 71.81 on Bench2Drive, and a PDMS of 91.1 on NAVSIM.

## 2. 方法论：核心思想、关键技术细节与算法流程

论文提出 **CaAD（Causality-aware End-to-end Autonomous Driving）**，即因果感知的端到端自动驾驶框架。它主要包含两个核心模块：

1. **自车中心联合因果建模（Ego-centric Joint-Causal Modeling）**
2. **因果感知策略对齐（Causality-Aware Policy Alignment）**

---

### 2.1 整体框架

- CaAD 基于一个 query-based 端到端自动驾驶骨干网络，具体参考 HiP-AD 类似的统一解码器架构。
- 该骨干网络统一处理：
  - 目标检测；
  - 地图建模；
  - 周围智能体运动预测；
  - 自车规划。
- 原始骨干网络会输出自车和周围 agent 的边缘预测嵌入。
- CaAD 在此基础上增加一个**训练时使用的自车中心联合预测分支**，用于学习自车与交互相关 agent 的联合未来。

---

### 2.2 自车中心联合因果建模

#### 2.2.1 边缘嵌入与联合模式嵌入

- 对于自车，已有解码器输出自车规划嵌入：
  - \(x_e \in \mathbb{R}^C\)
- 对于第 \(i\) 个周围智能体，已有边缘预测嵌入：
  - \(x^a_i \in \mathbb{R}^C\)

CaAD 引入 \(M\) 个联合场景模式，每个模式代表一个以自车为中心的未来场景假设。

- 自车第 \(m\) 个联合模式嵌入：
  - \(z^e_m\)
- agent \(i\) 在第 \(m\) 个联合模式下的嵌入：
  - \(z^a_{i,m}\)

然后将边缘嵌入与联合模式嵌入组成序列：

- 自车模式序列：
  - \([x_e; z^e_1; \dots; z^e_M]\)
- agent 模式序列：
  - \([x^a_i; z^a_{i,1}; \dots; z^a_{i,M}]\)

这些序列经过 **Agent-Mode Attention**，使边缘嵌入与联合模式嵌入进行交互。

---

#### 2.2.2 Agent-Mode Attention

论文中的 Agent-Mode Attention 本质上是在模式维度上做注意力交互。

其作用是：

- 让每个 joint-mode embedding 读取边缘预测嵌入中的实体级信息；
- 将自车、agent、地图和上下文信息注入联合模式；
- 使同一个模式编号 \(m\) 下的自车未来和相关 agent 未来形成一个共同的场景假设。

经过注意力更新后，每个模式 \(m\) 预测一个联合场景：

\[
S_m = (\hat{\tau}^e_m, \hat{\tau}^a_{1,m}, \dots, \hat{\tau}^a_{N,m})
\]

其中：

- \(\hat{\tau}^e_m\)：第 \(m\) 个模式下的自车轨迹；
- \(\hat{\tau}^a_{i,m}\)：第 \(m\) 个模式下第 \(i\) 个 agent 的轨迹。

---

### 2.3 交互相关 agent 选择

论文认为，并不是所有周围 agent 都应该参与联合监督。

原因是：

- 很多远处车辆与自车未来规划无关；
- 强行将所有 agent 放入同一联合模式，会引入噪声；
- 真实交通交互是稀疏的，通常只有少数 agent 与自车存在关键交互。

因此，CaAD 选择一个交互相关集合 \(A_{int}\)。

选择依据是：

- 使用自车的空间轨迹 \(\tau^e_{sp}\)；
- 使用周围 agent 的边缘预测轨迹；
- 如果某个 agent 的某个候选轨迹与自车空间轨迹存在碰撞风险，则该 agent 被认为是交互相关 agent。

直观来说：

> 如果某个 agent 的未来可能与自车计划路径发生冲突，它就被纳入联合因果建模。

论文特别强调使用**空间轨迹**而不是时间采样轨迹进行选择，因为：

- 空间轨迹更直接反映自车要走的几何路径；
- 时间轨迹容易混入速度、时序因素；
- 空间轨迹更适合判断潜在交互关系。

---

### 2.4 自车中心模式分配

传统联合预测方法通常使用所有 agent 的轨迹误差总和来选择最佳联合模式，即选择一个整体 L2 误差最小的模式。

CaAD 不这样做，而是采用**自车中心 winner-takes-all 模式选择**：

\[
m^\star = \arg\min_m d(\hat{\tau}^e_m, \tau^e_{tp})
\]

即：

- 只根据自车预测轨迹与自车真实轨迹之间的距离选择最佳联合模式；
- 对于交互相关 agent，使用同一个自车选中的模式 \(m^\star\) 进行监督；
- 对于非交互相关 agent，仍然使用各自的边缘预测最佳模式进行监督。

这种设计的含义是：

> 联合场景模式首先围绕自车意图组织，然后让关键周围 agent 在该自车意图下学习合理响应。

这样可以避免把无关 agent 强行绑定到同一个场景模式中。

---

### 2.5 联合预测损失

联合预测损失包括：

- 联合分类损失；
- 联合轨迹回归损失。

分类损失中：

- 自车使用选中的 \(m^\star\)；
- 交互相关 agent 也使用 \(m^\star\)；
- 非交互相关 agent 使用自己的边缘最佳模式。

回归损失被建模为高斯负对数似然：

\[
L_{reg} =
\frac{1}{2}
\left(
\frac{(\mu_{\tilde{m}} - \tau_{gt})^2}{\sigma^2_{\tilde{m}}}
+ 2 \log \sigma_{\tilde{m}}
+ \log 2\pi
\right)
\]

这表示模型不仅预测轨迹均值 \(\mu\)，还预测标准差 \(\sigma\)，从而将原本确定性的轨迹回归头转化为随机策略头。

---

### 2.6 因果感知策略对齐

在监督学习阶段之后，CaAD 使用强化学习进行策略对齐，目标是提升闭环驾驶质量。

#### 核心思想

- 不是在边缘表示上优化自车策略；
- 而是在前面学习到的**联合模式嵌入**上优化自车策略；
- 每个采样的自车 rollout 都与同一联合场景模式下的周围 agent 未来一起评估。

这使得策略优化不再只考虑“自车轨迹本身是否合理”，而是考虑：

- 是否会碰撞；
- 是否在可行驶区域；
- 是否符合交通方向；
- 是否有足够进展；
- 是否舒适；
- 是否与周围交通一致。

---

### 2.7 GRPO 策略优化

CaAD 使用 GRPO（Group Relative Policy Optimization）进行后训练策略优化。

流程大致如下：

1. 对每个联合模式 \(m\)，采样 \(G\) 条自车 rollout；
2. 对每条 rollout 计算规划奖励 \(r_{m,g}\)；
3. 在同一模式组内部对奖励进行归一化，得到相对优势：
   \[
   A_{m,g} = \frac{r_{m,g} - mean(r_m)}{std(r_m)}
   \]
4. 对发生碰撞的轨迹，将优势截断为 -1；
5. 对未碰撞轨迹，仅保留非负优势：
   \[
   \tilde{A}_{m,g} =
   \begin{cases}
   -1, & \text{if collision occurs} \\
   \max(0, A_{m,g}), & \text{otherwise}
   \end{cases}
   \]
6. 使用 clipped GRPO objective 更新自车策略。

重要的是：

- 强化学习只优化自车策略；
- 周围 agent 的预测仍保持监督学习；
- 这样可以避免给每个 agent 分配复杂奖励导致训练不稳定。

消融实验中使用较短训练设置：

- 第二阶段 6 epochs；
- 第三阶段 3 epochs。

### 4.2 NAVSIM 训练资源

论文说明：

- 使用 ResNet-34 作为图像编码器；
- 使用 rasterized BEV LiDAR 作为额外输入；
- 三个前视摄像头图像拼接为 \(1024 \times 256\) 输入；
- 从头在 navtrain split 上训练 80 epochs；
- 再使用 PDMS-based 因果感知策略对齐训练 20 epochs；
- 优化器为 AdamW；
- 学习率为 \(6 \times 10^{-4}\)。

### 4.3 未明确说明的信息

- 总训练时长；

- 推理速度；
- 参数量；
- FLOPs；
- 显存占用。

因此，虽然论文给出了一定的硬件配置和训练 epoch 信息，但资源成本报告并不完整。

## 5. 实验数量与充分性

### 5.1 实验数量概览

论文实验较丰富，大致包括以下几类：

1. **Bench2Drive 主结果实验**
   - 与十余种方法比较；
   - 包含开环 L2 和闭环指标。

2. **Bench2Drive ability metrics 实验**
   - 对 merging、overtaking、emergency brake、give way、traffic sign 等交互能力做细分评估。

3. **NAVSIM 主结果实验**
   - 与多种非 RL 和 RL-based 方法比较；
   - 报告 PDMS 及组件指标。

4. **Bench2Drive-mini 消融实验**
   - 分析 agent selection cue；
   - 分析 joint prediction；
   - 分析 ego-centric assignment；
   - 分析 marginal policy alignment 与 joint policy alignment。

5. **Bench2Drive full routes 额外消融**
   - 比较 all-agent policy alignment 与 ego-centric policy alignment。

6. **nuScenes 开环补充实验**
   - 分析闭环与开环指标之间可能存在的不一致。

7. **定性实验**
   - 展示交互场景；
   - 展示模块消融下的行为差异；
   - 展示 NAVSIM 中避免碰撞的可视化案例。

---

### 5.2 实验是否充分

整体来看，实验较为充分，原因包括：

- 覆盖两个主流规划 benchmark：
  - Bench2Drive；
  - NAVSIM。
- 同时报告闭环指标和细粒度能力指标；
- 与纯 imitation learning 方法、强化学习方法、world model 方法、diffusion/flow 方法都有比较；
- 消融实验能够较好验证论文提出的关键组件：
  - 空间轨迹选择 agent 的有效性；
  - 自车中心模式分配的重要性；
  - 联合场景表示优于边缘表示上的策略优化；
  - 只优化自车策略比 all-agent policy alignment 更稳定。

---

### 5.3 公平性与客观性

论文在公平性方面有一些优点：

- Bench2Drive 消融中使用相同训练设置；
- 对 HiP-AD baseline 做了 reproduced 结果；
- 区分了 RL-based 和非 RL-based 方法；
- 在附录中报告了完整 route subset；
- 使用官方 benchmark 指标。

但也存在一些可能影响客观性的因素：

- Bench2Drive-mini 只包含完整 routes 的约 25%，虽然覆盖场景类型，但仍可能存在采样偏差；
- 部分对比方法的训练设置、输入传感器、模型容量可能不完全一致；
- NAVSIM 中使用了 PDMS 风格奖励，而最终指标也是 PDMS，可能存在 reward-metric alignment 带来的优势；
- 论文主要基于仿真或数据驱动闭环 benchmark，尚未验证真实道路部署效果。

## 6. 主要结论与发现

### 6.1 Bench2Drive 结果

CaAD 在 Bench2Drive 上取得：

- Driving Score：87.53；
- Success Rate：71.81%。

相较 HiP-AD：

- CaAD 的 Driving Score 更高；
- Success Rate 更高；
- 在交互能力上提升明显。

尤其在以下能力上表现突出：

- Merging：60.00；
- Overtaking：86.67；
- Emergency Brake：83.33；
- Give Way：50.00；
- Ability Mean：70.27。

论文认为，这说明 CaAD 对交互关键场景的建模更有效。

---

### 6.2 NAVSIM 结果

CaAD 在 NAVSIM 上取得：

- PDMS：91.1；
- NC：99.4；
- DAC：98.3；
- EP：84.9；
- TTC：95.9。

相较其他 RL-based 方法如 DIVER、DriveDPO、ReCogDrive，CaAD 在总 PDMS 和安全相关指标上均有较强表现。

---

### 6.3 消融实验发现

主要结论包括：

- 仅在 marginal embedding 上做 policy alignment 有提升，但有限；
- 加入 joint-causal scene modeling 后，即使不做 RL，也能显著提高闭环表现；
- 在 joint scene representation 上做 causality-aware policy alignment 效果最好；
- spatial ego cue 比 temporal ego cue 更适合选择交互相关 agent；
- ego-centric mode assignment 是关键组件，移除后性能明显下降；
- 对所有 agent 都做 policy alignment 不如只优化自车策略稳定。

---

### 6.4 开环与闭环差异

在 nuScenes 开环实验中，CaAD 相比 reproduced HiP-AD 降低了 L2 误差，但 collision rate 略高。

论文据此指出：

- 开环轨迹匹配并不能完全反映闭环驾驶能力；
- CaAD 的主要优势在于闭环交互决策，而不一定在所有开环指标上都占优。

---

## 7. 优点与亮点

### 7.1 方法层面

- **显式建模自车与周围 agent 的联合未来**
  - 相比传统边缘预测，CaAD 更关注场景一致性。

- **自车中心联合模式设计合理**
  - 联合模式围绕自车意图组织，更符合自动驾驶规划任务的本质。

- **只选择交互相关 agent 参与联合监督**
  - 避免所有 agent 强行耦合；
  - 降低噪声；
  - 符合真实交通交互稀疏性的特点。

- **空间轨迹用于 agent selection**
  - 比时间轨迹更稳定；
  - 更直接反映自车运动意图。

- **策略对齐建立在联合场景表示上**
  - 相比只在自车或 marginal 表示上做 RL，能够更好利用交互信息。

- **只优化自车策略，保持周围 agent 监督预测**
  - 避免多 agent 强化学习中的信用分配噪声；
  - 提高训练稳定性。

---

### 7.2 实验层面

- 覆盖 Bench2Drive 和 NAVSIM 两个 benchmark；
- 同时报告总指标和细粒度交互能力指标；
- 与 RL-based 和非 RL-based 方法均有比较；
- 消融设计较完整，能支撑核心论点；
- 提供了定性可视化，帮助理解模型在交互场景中的行为差异。

---

### 7.3 结果层面

- 在 Bench2Drive 上接近专家策略 Think2Drive 的 Driving Score；
- 在 NAVSIM 上取得最高 PDMS；
- 在 merging、overtaking 等交互场景中提升明显；
- 说明显式联合场景建模对闭环规划确实有价值。

---

## 8. 不足与局限

### 8.1 方法局限

- **“因果”概念更多体现为交互一致性建模**
  - 论文使用“causality-aware”描述自车与 agent 的相互依赖，但并未进行严格的因果推断、反事实验证或结构因果模型建模。
  - 因此这里的“因果”更接近规划条件下的交互依赖建模。

- **交互相关 agent 的选择依赖碰撞检测启发式**
  - 如果边缘预测质量较差，可能漏选关键 agent；
  - 如果自车空间轨迹本身不准确，也会影响 agent selection。

- **联合分支主要是训练时监督**
  - 论文中提到联合预测和 SRL 主要作为训练时监督使用，推理阶段如何充分利用联合表示的细节仍可进一步讨论。

- **奖励函数与目标 benchmark 存在耦合**
  - 策略对齐使用 PDMS 风格奖励；
  - 在 NAVSIM 上最终也用 PDMS 评价；
  - 这可能带来一定 metric-specific optimization 风险。

---

### 8.2 实验局限

- **真实世界验证不足**
  - 实验主要在 Bench2Drive、NAVSIM、nuScenes 等 benchmark 上完成；
  - 缺乏真实车辆或大规模真实闭环道路测试。

- **长尾社会行为仍未解决**
  - 论文明确承认，CaAD 对罕见社会行为仍存在困难；
  - 例如为接近的紧急车辆主动让出空间，模型没有表现出可靠成功案例。

- **传感器和场景覆盖有限**
  - Bench2Drive 中主要是相机仿真闭环设置；
  - NAVSIM 是数据驱动非反应式仿真；
  - 对更复杂天气、真实交通规则差异、城市长尾场景的泛化仍需验证。

- **计算成本报告不完整**
  - 虽然 Bench2Drive 给出了 8 张 A6000 和训练 epoch；
  - 但缺少总训练时长、推理速度、参数量、显存占用等信息；
  - 不利于评估实际部署成本。

- **部分对比可能受实现差异影响**
  - 不同方法可能使用不同输入、训练数据增强、模型容量或训练策略；
  - 尽管采用官方 benchmark，但完全公平比较仍有难度。

---

## 总结

CaAD 的核心贡献是将端到端自动驾驶中的自车规划从边缘预测范式推进到**自车中心的联合场景建模范式**。它通过选择交互相关 agent、使用自车中心模式分配、在联合模式嵌入上进行策略对齐，使模型能够更好地处理并线、超车、让行、紧急制动等需要交互推理的场景。实验显示，CaAD 在 Bench2Drive 和 NAVSIM 上取得了很强的闭环性能，尤其在交互能力指标上提升明显。

不过，论文中的“因果”更多是交互依赖建模，而非严格因果推断；同时方法仍主要验证于仿真和 benchmark，真实道路泛化、长尾行为处理和部署成本仍需进一步研究。
