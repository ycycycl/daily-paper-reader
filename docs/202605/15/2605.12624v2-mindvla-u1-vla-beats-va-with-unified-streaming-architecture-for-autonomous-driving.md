## 摘要
自动驾驶已从模块化流水线逐步迈向端到端统一，而视觉-语言-动作（Vision-Language-Action, VLA）模型是这一进程在视觉到动作（Vision-to-Action, VA）之外的自然延伸。在实践中，驾驶 VLA 在规划质量上往往落后于 VA，这表明困难并不只是模型规模，而在于将语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的是 VLA 的构建方式——作为彼此孤立的子任务改进，未能组合成连贯的驾驶能力——而非 VLA 本身的问题。我们提出 MindVLA-U1，这是首个用于自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干网络在一个共享表示上通过单次前向传播生成自回归语言 token（可选）以及基于流匹配的连续动作轨迹，同时保留各模态的自然输出形式。完整的流式设计以逐帧方式处理驾驶视频，而不是在代价高昂的时间 VLM 建模下处理固定的视频-动作块。规划轨迹在帧间平滑演化，同时一个学习得到的流式记忆通道承载并更新时间上下文。该统一架构通过灵活的自注意力上下文管理，在密集与稀疏 MoT 骨干网络上支持快/慢系统，并为动作暴露了一条可度量的语言-控制路径：由语言预测的驾驶意图通过无分类器引导（classifier-free guidance, CFG）驱动动作扩散，将语言侧意图转化为连续动作规划的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越有经验的人类驾驶员（8.20 RFS 对比 8.13 GT RFS），仅需 2 个扩散步骤；在规划 ADE 上以大幅优势超过既有 VA/VLA 方法，达到最先进水平；并在保持用于人车交互的自然语言接口的同时，实现与 VA 相当的延迟（1B 规模下 16 FPS，对比 RAP 的 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces AR language tokens (optional) and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A full streaming design processes the driving video framewise rather than as fixed video-action chunks under costly temporal VLM modeling. Planned trajectories evolve smoothly across frames while a learned streaming memory channel carries temporal context and updates. The unified architecture enables fast/slow systems on dense & sparse MoT backbones via flexible self-attention context management, and exposes a measurable language-control path for action: language-predicted driving intents steers the action diffusion via classifier-free guidance (CFG), turning language-side intent into control signals for continuous action planning. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA by large margins, and matches VA latency (16 FPS vs. RAP's 18 FPS at 1B scale) while preserving natural language interfaces for human-vehicle interaction.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 总体架构思想

MindVLA-U1 的核心思想是：

> 用一个共享 VLM 骨干网络，在同一次前向传播中同时生成语言 token 和连续动作轨迹，并通过流式记忆维持长期时间上下文，再用语言预测的驾驶意图通过 CFG 引导动作扩散。

该方法主要包含四个关键部分：

1. **统一共享 VLM 骨干**
2. **连续动作的 Flow Matching 生成**
3. **逐帧流式记忆机制**
4. **语言到动作的 Intent-CFG 控制路径**
5. **快 / 慢推理模式与 MoT 稀疏架构扩展**

---

## 2.2 统一共享骨干：语言与动作共用表示

- 输入包括：
  - 多视角图像 token；
  - ego-state 历史状态；
  - 语言问题 / 指令；
  - 语言答案 token；
  - 流式 memory token；
  - 加噪动作 token。

- 所有 token 都通过同一个 VLM Transformer 骨干网络：
  - 语言 token 通过 LM head 进行自回归预测；
  - 动作 token 通过轻量 MLP 输出连续轨迹的 flow velocity；
  - 动作 token 不经过独立动作专家，而是进入 VLM 的 self-attention，与视觉和语言 token 共同建模。

- 这与以往方法不同：
  - 不是把轨迹坐标离散成语言 token；
  - 不是让 VLM 只作为视觉特征编码器；
  - 不是用独立 action expert 与 VLM cross-attention；
  - 而是让动作生成也发生在共享 self-attention 表示内部。

## 2.3 语言自回归损失与 Flow Matching 动作损失

论文联合优化两个目标。

### 语言自回归目标

对于语言答案 token 序列：

\[
a = (a_1, a_2, ..., a_L)
\]

语言损失为标准自回归交叉熵：

\[
L_{AR} = - \sum_{l=1}^{L} \log p_{\theta}(a_l | I, e, q, a_{<l})
\]

其中：

- \(I\)：图像 token；
- \(e\)：ego-state 历史；
- \(q\)：语言问题；
- \(a_{<l}\)：此前生成的答案 token。

### Flow Matching 连续动作目标

- 真实轨迹为 \(x_0 = w\)；
- 高斯噪声为 \(\epsilon \sim N(0,I)\)；
- 在时间 \(t\) 上构造加噪动作：

\[
x_t = t\epsilon + (1-t)x_0
\]

模型预测 velocity field：

\[
v_{\theta}(x_t,t;I,e,q,a)
\]

Flow Matching 损失为：

\[
L_{FM} = \|v_{\theta}(x_t,t;I,e,q,a) - (\epsilon - x_0)\|^2
\]

总损失为：

\[
L = L_{AR} + L_{FM}
\]

推理时通过 ODE / Euler 积分从噪声逐步生成连续轨迹。

## 2.4 语言到动作桥梁：Intent-CFG

论文强调，VLA 中语言必须有一条**可度量的路径**影响动作，而不是仅仅共享权重。

MindVLA-U1 的做法是：

1. VLM 语言头预测当前驾驶意图 token，例如：
   - left；
   - right；
   - straight；
   - 或 MindLabel 中扩展的 20 类意图。

2. 将预测意图 token 的 embedding 加入动作扩散 MLP 的时间 embedding。

3. 训练时进行 CFG dropout：
   - 有时使用真实 / 预测意图；
   - 有时替换为 unconditional token \(\emptyset\)。

4. 推理时运行条件和无条件两次 forward，得到两个 velocity field：

\[
v_{\theta}(x_t,t;z)
\]

\[
v_{\theta}(x_t,t;\emptyset)
\]

然后用 Classifier-Free Guidance 混合：

\[
v_{cfg}(x_t,t)=v_{\theta}(x_t,t;\emptyset)+s(v_{\theta}(x_t,t;z)-v_{\theta}(x_t,t;\emptyset))
\]

其中 \(s\) 是 guidance scale，论文中设置为 1.5。

- 该设计使语言侧的意图预测真正成为连续轨迹生成的控制信号。
- 实验显示，NTP-predicted Intent-CFG 将 RFS 从 7.83 提升到 7.92。

## 2.5 流式记忆机制

传统 VLA 常将多个历史帧一起输入 VLM，代价高且存在冗余。MindVLA-U1 改为：

- 每次只处理当前多视角帧；
- 历史信息通过一个 FIFO memory channel 传递；
- memory 中存储的是压缩后的记忆 token，而不是历史原始视觉 token。

流程如下：

1. 当前帧 \(i\) 读取 FIFO memory 中过去若干帧的 memory token；
2. 根据 ego pose 进行 SE(2) motion alignment，使历史 memory 对齐到当前车辆坐标系；
3. 当前帧图像、ego-state、语言、动作噪声和 memory token 一起输入共享 VLM；
4. 前向传播后，使用 Q-Former 风格 propagation transformer 从 backbone 输出中压缩出新的 memory token；
5. 新 memory 写入 FIFO，最旧 memory 被移除；
6. 训练和推理都使用相同的 read-forward-write 流式过程。

该机制带来的好处：

- 避免多帧 VLM attention 的高成本；
- 避免固定 action chunk 的边界不连续；
- 支持长序列逐帧平滑规划；
- memory 通过任务损失端到端监督，而不是简单缓存。

## 2.6 快 / 慢系统与 MoT

MindVLA-U1 支持多种推理模式：

- **vqa_first**：先生成语言解释 / 推理，再生成动作；
- **action_first**：先生成动作，再生成语言解释；
- **action_only**：只生成动作，作为快速路径；
- **vqa_only**：只进行语言问答。

这些模式主要通过 attention mask 和 token 顺序管理实现，不需要额外模型。

此外，论文还提出稀疏 **Mixture-of-Transformers, MoT** 变体：

- 将 token 分为：
  - context group：视觉、语言；
  - action group：memory、ego-state、action；
- 两组通过共享 self-attention K/V 池交互；
- FFN 使用不同专家；
- 目标是在保留统一表示的同时提高动作推理效率，并为未来加入 reasoning、safety、world model 等专家预留结构。

  - 包含 better / alternative / conservative / worse 等质量层次；
  - 生成轨迹评价 QA。
- 20 类驾驶 intent 标签；
- chain-of-thought rationale。

规模：

- 约 18.8K annotated clips；
- 约 3.8M VQA pairs；
- 约 250K dreamed trajectories；
- 标注模型包括 Qwen3-VL 和 Qwen3.5-Plus。

不过论文主实验只使用了：

- basic scene-grounded VQA；
- WOD-E2E 官方 3 类 GT intent；
- 官方轨迹监督。

更丰富的 MindLabel 内容被留作未来工作。

## 6. 主要结论与发现

- **VLA 不必牺牲规划精度**
  - MindVLA-U1 表明，只要接口设计合理，VLA 可以达到甚至超过 VA 的规划质量。

- **统一共享骨干有效**
  - 将语言和连续动作放在同一个 VLM 表示中联合建模，可以同时保留语言能力和高精度轨迹规划。

- **连续动作表示优于离散动作 token**
  - Flow Matching 轨迹生成避免了语言 token 离散化带来的精度瓶颈。

- **语言可以真实影响动作**
  - Intent-CFG 实验证明语言预测的驾驶意图可以通过 CFG 改变连续轨迹生成，而不是仅作为附属输出。

- **流式记忆比多帧 VLM 输入更有效**
  - 直接堆叠 4 帧图像 token 反而降低性能；
  - streaming + memory 明显提升 RFS 和长期 ADE。

- **快 / 慢系统可以共享同一权重**
  - 慢路径可用于语言推理；
  - 快路径可用于实时控制；
  - 不需要维护两套模型。

- **RL 后训练显著提升 RFS**
  - SFT 的 RFS 为 7.83；
  - 加 GRPO 后验证集 RFS 达到 8.20；
  - 超过 human logged trajectory 的 8.13 RFS。

- **语言能力和动作质量存在结构性解耦**
  - instruction-tuned VLM 相比 base VLM 语言指标更好；
  - 但动作规划 RFS 差异很小；
  - 说明动作控制主要依赖共享场景表示和 intent embedding，而不完全依赖语言表面生成质量。

---

## 7. 优点与亮点

- **架构统一性强**
  - 语言和动作共享同一个 VLM backbone；
  - 避免了 VLM + 独立 planner 的松耦合问题。

- **保持动作连续性**
  - 使用 Flow Matching 生成连续轨迹，适合自动驾驶高精度控制需求。

- **明确语言到动作路径**
  - Intent-CFG 提供了可解释、可控、可消融的语言控制机制。

- **流式时间建模符合驾驶本质**
  - 逐帧处理而非固定 chunk；
  - memory 机制支持平滑、长时序规划。

- **推理模式灵活**
  - 同一 checkpoint 支持 action-only 快速控制和 VQA-first 慢速推理。

- **实验设计较完整**
  - 包含主指标、消融、速度测试、scale 分析、分布偏移诊断和定性可视化。

- **MindLabel 数据建设有潜在价值**
  - 不仅服务当前模型，也为未来驾驶 VLA 的语言、偏好、轨迹评估和 CoT 训练提供数据基础。

- **性能结果突出**
  - 在 WOD-E2E 上取得领先 RFS 和 ADE；
  - 验证集 RL 后首次超过 human GT RFS；
  - 在约 1B 规模下接近 VA 推理速度。

---

## 8. 不足与局限

- **仅为 open-loop 评估**
  - WOD-E2E 是日志回放式 benchmark；
  - 不能证明模型在真实闭环驾驶中安全、鲁棒或优于人类。

- **核心实验集中在单一 benchmark**
  - 缺少 nuScenes、NAVSIM、CARLA、真实车队或闭环仿真验证；
  - 泛化能力仍需进一步证明。

- **RFS 超越 human GT 需谨慎解释**
  - RFS 是 rater-based open-loop 分数；
  - logged GT 只是某一次人类驾驶轨迹，不一定是评分者最偏好的轨迹；
  - 超过 GT RFS 不等价于真实驾驶水平超过人类。

- **MindLabel 自动标注存在偏差风险**
  - 标注依赖 Qwen3-VL 和 Qwen3.5-Plus；
  - 可能继承这些模型的视觉理解错误、语言偏见和场景误判。

- **Intent 监督较粗**
  - 主实验主要使用 WOD-E2E 3 类 intent：left / right / straight；
  - 对等待、让行、急停、避障等复杂行为监督不足；
  - 论文也指出测试集 waiting 等意图比例更高，导致 val/test 分布差异。

- **慢路径延迟过高**
  - 完整 VQA-first decoding 延迟约 2.6 秒，不适合实时控制；
  - 实时部署主要依赖 action-only 或 template QA 快路径。

- **Backbone scaling 结论有限**
  - 规模增大并未单调提升 RFS；
  - 作者认为受训练 schedule、数据规模和 action head 容量影响；
  - 尚不能形成明确 VLA scaling law。

- **RL 可能带来指标适配**
  - GRPO 直接优化 RFS；
  - 可能提升 benchmark 分数，但对真实驾驶偏好和安全性的迁移仍未验证。

- **部分丰富数据未被利用**
  - MindLabel 中的 20 类 intent、dreamed trajectories、trajectory QA、CoT rationale 等在主结果中基本未使用；
  - 当前模型尚未充分发挥数据系统潜力。
