## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景的范式。然而，现有推理机制仍难以提供面向规划的中间表示：文本形式的思维链（Chain-of-Thought, CoT）无法保留连续的时空结构，而潜在世界推理仍难以作为动作生成的直接条件。在本文中，我们提出 CoWorld-VLA，一个面向自动驾驶的多专家世界推理框架，其中世界表示作为显式条件来指导动作规划。CoWorld-VLA 通过多源监督提取互补的世界信息，并将其编码为 VLA 中的专家 token，从而提供规划器可访问的条件信号。具体而言，我们构建了四类 token：语义交互、几何结构、动态演化和自车轨迹 token，分别用于建模交互意图、空间结构、未来时间动态以及行为目标。在动作生成过程中，CoWorld-VLA 采用一种基于扩散的分层多专家融合规划器，并在联合去噪过程中始终与场景上下文耦合，以生成连续的自车轨迹。实验表明，CoWorld-VLA 在 NAVSIM v1 基准上的未来场景生成和规划任务中均取得了具有竞争力的结果，在避碰能力和轨迹精度方面表现突出。消融研究进一步验证了专家 token 的互补性，以及它们作为动作生成规划条件的有效性。代码将发布于 https://github.com/AFARI-Research/CoWorld-VLA。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for end-to-end autonomous driving. However, existing reasoning mechanisms still struggle to provide planning-oriented intermediate representations: textual Chain-of-Thought (CoT) fails to preserve continuous spatiotemporal structure, while latent world reasoning remains difficult to use as a direct condition for action generation. In this paper, we propose CoWorld-VLA, a multi-expert world reasoning framework for autonomous driving, where world representations serve as explicit conditions to guide action planning. CoWorld-VLA extracts complementary world information through multi-source supervision and encodes it into expert tokens within the VLA, thereby providing planner-accessible conditioning signals. Specifically, we construct four types of tokens: semantic interaction, geometric structure, dynamic evolution, and ego trajectory tokens, which respectively model interaction intent, spatial structure, future temporal dynamics, and behavioral goals. During action generation, CoWorld-VLA employs a diffusion-based hierarchical multi-expert fusion planner, which is coupled with scene context throughout the joint denoising process to generate continuous ego trajectories. Experiments show that CoWorld-VLA achieves competitive results in both future scene generation and planning on the NAVSIM v1 benchmark, demonstrating strong performance in collision avoidance and trajectory accuracy. Ablation studies further validate the complementarity of expert tokens and their effectiveness as planning conditions for action generation. Code will be available at https://github.com/AFARI-Research/CoWorld-VLA.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体框架

CoWorld-VLA 采用三阶段训练流程：

1. **Stage 1：动作条件预测世界模型预训练**
   - 基于 Wan 视频生成模型，在 NuPlan 视频上学习未来场景演化。
   - 输入历史帧和文本化的自车意图 / 运动条件，预测未来视频潜变量。

2. **Stage 2：多专家世界表征学习**
   - 使用 Qwen3-VL 作为 VLM 主干。
   - 在 VLM 输入序列中插入四类专家 token：
     - 语义交互 token；
     - 几何结构 token；
     - 动态演化 token；
     - 自车轨迹 token。
   - 通过多源监督将这些 token 对齐到不同专家模型或任务目标。

3. **Stage 3：分层多专家融合扩散规划**
   - 冻结 Stage 2 的 VLM。
   - 使用 Hierarchical Multi-Expert Fusion（HMEF）扩散规划器，将专家 token 和场景上下文作为条件，通过迭代去噪生成连续自车轨迹。

## 2.2 问题建模

- 传统 VLA 直接建模为：

  \[
  p_\theta(A_{t+1:t+T} \mid o_t, c_t)
  \]

  其中：
  - \(o_t\)：当前图像观测；
  - \(c_t\)：导航指令、历史 ego pose、速度、加速度等条件；
  - \(A_{t+1:t+T}\)：未来自车轨迹。

- CoWorld-VLA 引入多专家世界表示：

  \[
  Z = \{z_{sem}, z_{geo}, z_{dyn}, z_{traj}\}
  \]

  并将轨迹生成改写为：

  \[
  p_\theta(A_{t+1:t+T} \mid o_t, c_t, Z)
  \]

- 这里的 \(Z\) 对应 VLM 内部的四类专家隐藏状态：
  - \(H_{sem}\)：语义交互；
  - \(H_{geo}\)：几何结构；
  - \(H_{dyn}\)：动态演化；
  - \(H_{traj}\)：自车轨迹先验。

## 2.3 Stage 1：动作条件预测世界模型

- 使用冻结的 Wan VAE 将视频序列编码为潜变量：

  \[
  z = E_{vae}(x) = [z_h, z_f]
  \]

  其中：
  - \(z_h\)：历史帧潜变量；
  - \(z_f\)：未来帧潜变量。

- 只对未来片段施加噪声，并使用 flow matching 训练未来演化：

  \[
  \tilde{z}_{f,\sigma} = (1-\sigma)z_f + \sigma \epsilon
  \]

  \[
  v_{target} = \epsilon - z_f
  \]

- 条件 prompt 包括：
  - 场景描述；
  - 速度描述；
  - 导航命令；
  - 未来轨迹文本化表达。

- 训练目标是让世界模型根据历史帧和自车意图预测未来潜在场景演化。

## 2.4 Stage 2：多专家世界表征学习

### 四类专家 token

- **语义交互 token**
  - 由冻结 V-JEPA / JEPA 类模型提供监督。
  - 目标是捕获高层语义、对象关系、交互意图和预测性表征。

- **几何结构 token**
  - 由冻结 VGGT 3D 基础模型提供监督。
  - 目标是捕获道路布局、空间结构、3D 几何约束等。

- **动态演化 token**
  - 用作 Wan 世界模型的条件输入。
  - 通过未来视频生成的 flow matching 损失监督，使其学习未来场景动态和时间一致性。

- **自车轨迹 token**
  - 通过轻量 MLP 头回归未来轨迹。
  - 目标是让该 token 学到与自车行为目标直接相关的表示。

### 多专家监督目标

- JEPA 语义对齐损失：

  \[
  L_{sem} = \lambda_{l1} SmoothL1(\hat{Z}_{sem}, Z_{sem}) + \lambda_{cos}(1 - cos(\hat{Z}_{sem}, Z_{sem}))
  \]

- VGGT 几何对齐损失：

  \[
  L_{geo} = MSE(\hat{Z}_{geo}, Z_{geo})
  \]

- 动态演化监督：

  \[
  L_{dyn} = L_{flow}(\hat{o}_{fut}, o_{fut}; o_t, H_{dyn})
  \]

- 轨迹回归监督：

  \[
  L_{traj} = MSE(\hat{A}_{t+1:t+T}, A_{t+1:t+T})
  \]

- 总损失：

  \[
  L_{total} = w_{dyn}L_{dyn} + w_{sem}L_{sem} + w_{geo}L_{geo} + w_{traj}L_{traj}
  \]

- 文中采用的权重为：
  - \(w_{dyn}=1.0\)
  - \(w_{sem}=0.1\)
  - \(w_{geo}=0.1\)
  - \(w_{traj}=1.0\)

## 2.5 Stage 3：HMEF 分层多专家融合扩散规划器

- HMEF 的输入包括：
  - 场景上下文 token；
  - 当前帧 JEPA / VGGT 上下文 token；
  - VLM 生成的四类专家 action token。

- 为降低计算量，长序列场景 token 通过 Perceiver-style compressor 压缩为固定数量的上下文 token。

- 每个专家分支使用独立的双向 Transformer 编码，并对齐到规划时间步。

- 轨迹在归一化空间中进行扩散去噪训练：

  \[
  A_\tau = (1-\tau)\epsilon + \tau A_{norm}
  \]

- 每个专家预测一条轨迹，最终通过可学习融合权重进行加权：

  \[
  \bar{A} = \sum_e \alpha_e \hat{A}_e
  \]

- 训练损失包括：
  - 每个专家的去噪轨迹损失；
  - 融合后轨迹与真实轨迹之间的损失。

- 推理时从高斯噪声开始，经过若干步去噪，生成最终可执行 ego trajectory。

  - 论文未充分报告模型推理延迟、显存占用和部署效率，这对自动驾驶应用非常关键。
  - NAVSIM 是 open-loop / non-reactive 评估，不能完全代表真实闭环驾驶能力。

---

## 6. 主要结论与发现

- CoWorld-VLA 在 NAVSIM v1 上取得了较强规划性能：
  - PDMS 达到 **89.8**；
  - NC 达到 **99.2**；
  - EP 达到 **83.6**。
  - 在表中优于多种端到端、世界模型和 VLA 方法。

- 在未来视频生成任务中，CoWorld-VLA 的 FVD 为 **32.7**，优于表中对比方法，说明多专家 Latent CoT 能提升未来场景生成质量。

- 多专家 token 的逐步加入显著提升规划性能：
  - 仅 Ego Trajectory token：PDMS 83.7；
  - 加入几何 token：PDMS 85.1；
  - 加入语义 token：PDMS 87.7；
  - 加入动态 token：PDMS 88.7。

- HMEF 进一步提升最终规划结果：
  - Stage 2 VLM：PDMS 88.7；
  - 加入 ReCogDrive-style action expert：PDMS 89.1；
  - 加入 HMEF：PDMS 89.8。

- 扩散规划中 10 步去噪取得最佳平衡：
  - 5 步：PDMS 89.5；
  - 10 步：PDMS 89.8；
  - 20 步：PDMS 89.7。

- 学习到的专家融合权重显示：
  - 动态演化专家权重约 0.35；
  - 轨迹专家权重约 0.31；
  - 语义专家权重约 0.19；
  - 几何专家权重约 0.15。  
  这表明模型在规划时更依赖动态演化和轨迹相关信息。

## 7. 优点与亮点

- **将世界模型表征显式用于规划**
  - 不只是把未来预测作为辅助任务，而是在推理阶段把世界 token 作为规划条件使用。

- **多专家 Latent CoT 设计较清晰**
  - 将自动驾驶规划所需的关键信息拆解为语义、几何、动态和轨迹四类专家表示。
  - 相比单一 latent world representation，信息来源更丰富。

- **充分利用现有基础模型**
  - 使用 V-JEPA 提供语义和预测性监督。
  - 使用 VGGT 提供几何结构监督。
  - 使用 Wan 视频生成模型提供动态演化监督。
  - 使用 Qwen3-VL 作为多模态 VLM 主干。

- **HMEF 规划器设计合理**
  - 通过扩散式去噪生成连续轨迹。
  - 通过多专家分支和可学习权重实现专家融合。
  - 同时利用高层 latent reasoning 和低层 ego-state 信息。

- **消融实验支持核心假设**
  - 多专家 token 的逐步加入带来稳定提升。
  - HMEF 相比直接回归或普通 action expert 有进一步收益。

- **单帧前视相机设置下表现较强**
  - 在不使用 LiDAR、不使用多帧输入的条件下取得较高 PDMS，说明表征学习和世界先验较有效。

## 8. 不足与局限

- **计算成本高**
  - 三阶段训练总共需要大量 A800 GPU 资源。
  - Stage 1 使用 64 张 A800 训练约 74 小时，Stage 2 使用 32 张 A800 训练约 70 小时，整体门槛较高。

- **输入模态有限**
  - 当前模型主要使用单帧前视图像。
  - 未扩展到多摄像头、多帧历史、多模态传感器融合。
  - 对复杂遮挡、侧后方交互、长时序行为预测可能有限。

- **评估场景覆盖仍有限**
  - 规划主要在 NAVSIM v1 上验证。
  - 缺少真实闭环部署、CARLA 交互式闭环仿真或更多城市 / 天气 / 长尾场景验证。

- **open-loop 指标不能完全代表真实驾驶能力**
  - NAVSIM 的 non-reactive open-loop 评估无法完整反映与其他交通参与者交互时的鲁棒性。
  - 真实驾驶中对安全、舒适、法规遵守和异常处理的要求更高。

- **对基础模型依赖较强**
  - 框架依赖 Wan、Qwen3-VL、V-JEPA、VGGT 等多个大型预训练模型。
  - 不同基础模型质量可能显著影响最终效果。
  - 系统复杂度较高，端到端可控性和调试难度较大。

- **推理效率报告不足**
  - 虽然给出了采样步数，但未系统报告推理延迟、帧率、显存占用等。
  - 扩散规划虽然只用 10 步，但在车载实时部署中仍需进一步验证。

- **视频生成指标不够驾驶任务导向**
  - FVD 衡量视觉质量，但不一定等价于驾驶规划所需的物理一致性和交互合理性。
  - 需要更多面向自动驾驶的世界模型评价指标。

- **公平性存在一定复杂性**
  - 对比方法的传感器设置、帧数、是否使用强化学习、是否使用 LiDAR 等并不完全一致。
  - 论文做了标注，但严格控制变量的公平对比仍有改进空间。
