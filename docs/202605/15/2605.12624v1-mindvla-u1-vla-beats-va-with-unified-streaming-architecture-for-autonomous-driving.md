## 摘要
自动驾驶已从模块化流水线迈向端到端统一，而视觉-语言-动作（Vision-Language-Action, VLA）模型是这一进程在视觉到动作（Vision-to-Action, VA）之外的自然延伸。在实践中，驾驶 VLA 在规划质量上往往落后于 VA，这表明困难并不单纯在于模型规模，而在于语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的是 VLA 以孤立子任务改进的方式被构建，未能组合成连贯的驾驶能力，而非 VLA 本身的局限。我们提出 MindVLA-U1，这是首个面向自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干网络在一次前向传播中基于同一共享表示生成自回归语言 token 和基于流匹配的连续动作轨迹，同时保留各模态的自然输出形式。流式设计按帧处理驾驶视频，而非采用固定的视频-动作块；同时，学习得到的记忆通道在帧间传递时间上下文，使规划轨迹能够平滑演化，而无需冗余的多帧 VLM 建模。该统一架构通过灵活的自注意力上下文管理，支持在稠密/稀疏 Mixture-of-Transformers（MoT）骨干网络上进行快/慢执行，并呈现出一条可度量的语言到动作路径：由语言预测的驾驶意图通过无分类器引导（classifier-free guidance, CFG）来引导动作扩散，将语言侧意图转化为连续轨迹生成的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越有经验的人类驾驶员（RFS 为 8.20，相比 GT RFS 的 8.13），仅需 2 个扩散步；相较以往 VA/VLA 方法，在规划 ADE 上以大幅优势达到最先进水平；并在保留自然语言接口的同时，达到 VA 级吞吐率（16 FPS，相比 RAP-DINO 的 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose into coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces autoregressive language tokens and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A streaming design processes the driving video framewise rather than as fixed video-action chunks, while a learned memory channel carries temporal context across frames so planned trajectories evolve smoothly without redundant multi-frame VLM modeling. The unified architecture admits fast/slow execution on dense/sparse Mixture-of-Transformers (MoT) backbones via flexible self-attention context management, and exposes a measurable language-to-action route: a language-predicted driving intent steers action diffusion through classifier-free guidance (CFG), turning language-side intent into a control signal for continuous trajectory generation. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA methods by large margins, and matches VA-class throughput (16 FPS vs. RAP-DINO's 18 FPS) while preserving natural-language interfaces.

  - 缺少显式语言接口；
  - 难以进行开放词汇、组合式语义理解；

  - 将连续轨迹离散化为语言 token，损失控制精度；
  - 使用独立 action expert，导致语言表征和动作生成耦合不足；
  - 用固定视频块建模时序，计算冗余且轨迹不连续；
  - 语言能力只是“附带存在”，缺少可测量的语言到动作控制路径；

  **VLA 与 VA 的差距主要是接口问题，而不是语义推理与连续控制天然冲突。**

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体框架：统一流式 VLA 架构

- 论文提出 **MindVLA-U1**，声称是首个用于自动驾驶的**统一流式 VLA 架构**。
- 核心思想是：
  - 用一个共享 VLM backbone 同时处理：
    - 多视角图像；
    - ego-state 历史；
    - 文本问题 / 回答；
    - 流式记忆 token；
    - 噪声动作 token；
  - 在同一个前向传播中输出：
    - 自回归语言 token；
    - 连续动作轨迹的 flow-matching 速度场。
- 这样既保留语言的自然输出形式，又保留动作的连续控制形式。

---

## 2.2 共享骨干上的语言与连续动作联合建模

- 对于每个帧级输入，模型包含：
  - 多视角视觉 token：\(I\)
  - ego-state 历史：\(e\)
  - 语言 query：\(q\)
  - 回答 token：\(a=(a_1,\dots,a_L)\)
  - 噪声动作 token：\(x_t\)

- 语言部分采用自回归交叉熵损失：

\[
L_{AR}=-\sum_{l=1}^{L}\log p_\theta(a_l|I,e,q,a_{<l})
\]

- 动作部分采用 flow matching / diffusion-style 连续轨迹生成。
- 给定真实轨迹 \(x_0=w\)，噪声 \(\epsilon\sim \mathcal{N}(0,I)\)，构造：

\[
x_t=t\epsilon+(1-t)x_0
\]

- 动作头预测速度场：

\[
L_{FM}=\|v_\theta(x_t,t;I,e,q,a)-(\epsilon-x_0)\|^2
\]

- 总损失为：

\[
L=L_{AR}+L_{FM}
\]

- 推理时通过 ODE / Euler 积分从噪声恢复连续轨迹。主实验中动作预测 5 秒未来轨迹，频率为 4Hz，共 20 个 waypoint，仅用 2 个 Euler / diffusion steps。

---

## 2.3 Intent-CFG：可测量的语言到动作路径

- 论文特别强调，已有 VLA 常常只是“有语言头”，但无法证明语言真的影响动作。
- MindVLA-U1 设计了 **Intent-CFG** 机制：
  - 模型先通过语言头预测驾驶意图 token，例如 left / right / straight；
  - 将意图 embedding 加入动作 MLP 的时间 embedding；
  - 使用 classifier-free guidance 方式控制动作扩散。

- 训练时，以一定概率将意图替换为无条件 token \(\emptyset\)，使模型同时学习有条件和无条件速度场。
- 推理时分别计算：
  - 条件速度场 \(v_\theta(x_t,t;z)\)
  - 无条件速度场 \(v_\theta(x_t,t;\emptyset)\)

- 然后用 CFG 混合：

\[
v_{cfg}(x_t,t)=v_\theta(x_t,t;\emptyset)+s\left(v_\theta(x_t,t;z)-v_\theta(x_t,t;\emptyset)\right)
\]

- 其中 \(s\) 是 guidance scale，论文实现中使用 \(s=1.5\)。
- 这个机制使“语言侧意图”可以显式影响连续轨迹生成，而不是仅仅共享 backbone。

---

## 2.4 流式时序建模：Streaming Memory

- MindVLA-U1 不使用固定视频片段或多帧视觉 token 堆叠，而是按帧处理驾驶流。
- 每一帧输入：
  - 当前多视角图像；
  - 当前 ego-state；
  - 来自历史帧的压缩 memory tokens。
- 记忆通道采用 FIFO 队列：
  - 每帧写入一组压缩 memory tokens；
  - 超出容量后移除最早的记忆；
  - 文中默认每帧写入 \(N_m=128\) 个 memory tokens；
  - FIFO 保留 \(N_g=2\) 个历史帧条目，总容量 256 tokens。
- 记忆读取时会根据 ego pose 做 SE(2) 运动对齐，保证历史记忆和当前坐标系一致。
- 记忆更新通过 Q-Former-style propagation transformer 完成：
  - learnable queries cross-attend 到 backbone 输出；
  - 生成新的 memory tokens；
  - 写入 FIFO。
- 训练和推理都使用同样的 read-forward-write 流程，使部署时行为与训练一致。

## 2.5 Fast / Slow 推理与 MoT 架构

- 论文认为自动驾驶需要“快慢系统”：
  - 常规场景需要快速反射式控制；
  - 复杂长尾场景需要语言推理和解释。
- MindVLA-U1 通过 attention mask 和上下文管理支持多种推理模式：
  - `vqa_first`：先语言理解 / 问答，再动作生成；
  - `action_first`：先预测动作，再生成语言解释；
  - `action_only`：仅动作，最快；
  - `vqa_only`：仅语言。
- 同一个 checkpoint 可在不同模式下运行。
- 论文还提出稀疏 **Mixture-of-Transformers（MoT）** 版本：
  - context group：视觉、语言 token；
  - action group：memory、ego-state、action token；
  - 两组共享 self-attention 的 K/V 池，但使用不同专家和投影。
- MoT 的目标是减少 fast path 的推理成本，同时保留 slow path 的语义能力。

- 推理评估也报告了硬件：
  - 1×H200；
  - batch size = 1。
- 典型推理速度：
  - Qwen3-VL-2B slow path：约 2594 ms / frame，0.39 FPS；
  - Qwen3-VL-2B action_only fast path：约 103 ms，9.70 FPS；
  - Qwen3-VL-2B template QA fast path 1 step：约 63 ms，15.92 FPS；
  - InternVL-2 1B fast 配置：约 64 ms，15.55 FPS；
  - VA 参考 RAP-DINO：约 57 ms，17.68 FPS。
- 论文也报告了 VRAM：
  - Qwen3-VL-2B fast / slow 大约 5.4GB；
  - InternVL-2 1B 约 2.48GB。
- 资源信息相对完整，尤其是训练 GPU、训练时长、推理延迟都有披露。

## 5. 实验数量与充分性

## 5.1 实验类型较丰富

论文实验覆盖面较广，大致包括以下几类：

- 主结果实验：
  - WOD-E2E validation split；
  - WOD-E2E official test split。
- 与 VA / VLA SOTA 对比。
- Intent-CFG 消融：
  - no intent；
  - trajectory-derived intent；
  - GT-supplied intent；
  - NTP-predicted intent。
- MoT 结构消融：
  - dense backbone；
  - 不同 expert grouping。
- 推理吞吐与延迟分析：
  - slow path；
  - action_only fast path；
  - template QA fast path；
  - 1-step / 2-step action diffusion；
  - 与 RAP-DINO 对比。
- Streaming memory 消融：
  - chunk-wise single frame；
  - chunk-wise + image sequence；
  - streaming no memory；
  - streaming + memory。
- RL post-training 实验：
  - SFT 到 GRPO；
  - RFS-only reward。
- Val / test 分布偏移分析：
  - 使用 MindLabel intent 统计验证集和测试集意图差异。
- Backbone scaling：
  - Qwen3.5-VL 0.8B / 2B / 4B / 9B；
  - base vs instruction-tuned；
  - 9B extended training。
- 长序列可视化与流式一致性分析。

## 5.2 充分性评价

- 从单篇论文角度看，实验数量较多，消融比较完整，尤其围绕作者提出的几个核心模块：
  - unified backbone；
  - Intent-CFG；
  - streaming memory；
  - MoT fast / slow；
  - RL post-training；
  均有对应实验支持。
- 但实验也有明显边界：
  - 所有核心结果集中在 WOD-E2E 一个 benchmark；
  - 没有 closed-loop on-vehicle 或仿真闭环测试；
  - MindLabel 的大部分丰富标注没有被用于主模型训练；
  - 许多对比方法的 AD pretraining 状态不完全一致，表格中也有 “?”；
  - 部分对比方法是 leaderboard 或公开结果，可能存在训练数据、预训练策略和工程优化不完全可比的问题。
- 因此，实验对证明“在 WOD-E2E 开环长尾规划上有效”较充分，但对证明“泛化到真实闭环自动驾驶部署”仍不足。

## 6. 主要结论与发现

- MindVLA-U1 在 WOD-E2E 上显著提升了驾驶 VLA 的规划质量。
- 在 validation split：
  - MindVLA-U1 baseline RFS 为 7.83；
  - 加 Intent-CFG 后 RFS 为 7.92；
  - 加 MoT 后 RFS 为 7.92；
  - RL 后 RFS 达到 8.20；
  - 超过 human driver logged trajectory 的 8.13。
- 在 official test split：
  - MindVLA-U1 RFS 为 7.77；
  - MindVLA-U1 + RL RFS 为 7.87；
  - RFS-GT ADE 达到 1.09 / 2.66m，优于已有 VA / VLA 方法。
- Intent-CFG 证明了语言侧意图能够实际影响连续轨迹生成：
  - NTP-predicted Intent-CFG 将 RFS 从 7.83 提升至 7.92。
- Streaming memory 有稳定收益：
  - streaming no memory RFS 为 7.73；
  - streaming + memory RFS 为 7.83。
- 直接堆叠 4 帧图像到 VLM 并没有带来收益，反而降到 RFS 7.61。
- Fast path 可以接近 VA 级别吞吐：
  - 1B 级模型约 15.55 FPS；
  - 与 RAP-DINO 的 17.68 FPS 接近。
- Backbone scaling 并非简单单调：
  - Qwen3.5-VL 2B 表现最好；
  - 9B 默认训练反而略低，但延长训练后有所恢复。
- 语言质量和动作质量在一定程度上解耦：
  - base VLM 的 BLEU / ROUGE 明显下降，但 RFS 接近 instruction-tuned 版本。
- 作者由此认为：
  - VLA 的规划落后问题主要源于接口设计，而非 VLA 范式本身；
  - 合理设计后，VLA 可以在保留语言接口的同时达到甚至超过 VA 的规划质量。

## 7. 优点与亮点

- **统一性强**：
  - 语言和动作共享同一 VLM backbone；
  - 不再是 VLM + 独立 planner 的松耦合结构。
- **动作表示合理**：
  - 保持连续轨迹输出；
  - 避免将轨迹离散化为语言 token 带来的精度损失。
- **语言到动作路径可测量**：
  - Intent-CFG 提供了明确的语言意图控制动作生成机制；
  - 不是仅凭“模型有语言头”宣称语言有用。
- **流式建模符合驾驶任务本质**：
  - 按帧处理；
  - 使用紧凑记忆通道；
  - 避免固定 chunk 带来的边界不连续和多帧视觉 token 冗余。
- **快慢系统设计具有部署意义**：
  - 同一 checkpoint 支持 slow reasoning 和 fast action；
  - 通过 attention mask / MoT 实现不同推理路径。
- **实验覆盖模块较全面**：
  - 主结果、消融、延迟、backbone scaling、RL、val/test shift 均有分析。
- **MindLabel 数据管线有潜在价值**：
  - 提供 VQA、意图、轨迹评价、替代轨迹和 CoT；
  - 对后续驾驶 VLA 数据构建具有参考意义。
- **对 benchmark 分布偏移进行了诊断**：
  - 分析了 WOD-E2E validation/test 的 intent 分布差异；
  - 不只是报告数字，也解释了 val-test gap 的可能来源。

---

## 8. 不足与局限

- **仅开环评估**：
  - WOD-E2E 是 logged open-loop benchmark；
  - RFS 和 ADE 不能完全代表闭环驾驶安全性；
  - 超过 human GT RFS 不等于真实车辆闭环超过人类驾驶员。
- **单一核心 benchmark**：
  - 主结果几乎都在 WOD-E2E；
  - 没有在 nuScenes、NAVSIM、CARLA 或真实车端闭环验证。
- **对比公平性仍有限**：
  - 不同方法是否使用外部 AD pretraining 并不完全一致；
  - 一些方法的预训练状态未知；
  - 工程优化、backbone、数据使用量可能不同。
- **MindLabel 可能引入自动标注偏差**：
  - 数据由 Qwen3-VL / Qwen3.5-Plus 自动生成；
  - 可能继承标注模型的偏见、错误和幻觉；
  - 论文主实验只使用其中一部分，尚未证明完整 MindLabel 对训练的实际收益。
- **Intent 监督较粗**：
  - 主模型使用 WOD-E2E 的 3 类 intent：left / right / straight；
  - 对 urgent stopping、yielding、waiting 等复杂场景表达不足；
  - 作者也指出 test 集中 waiting 等意图比例更高，导致性能下降。
- **VLM scaling 结论尚不充分**：
  - 参数规模增大并未稳定提升；
  - 但训练 schedule、数据规模、动作头容量等变量未完全控制；
  - 不能得出 VLA 不可 scale 的结论。
- **慢路径延迟很高**：
  - vqa_first_decoding 约 2.6 秒 / 帧；
  - 主要瓶颈是自回归 VQA 解码；
  - 对实时安全关键场景，slow path 直接部署仍困难。
- **RL 优化可能偏向 rater score**：
  - RL 后 RFS 提升明显；
  - 但论文也指出 GT-matched ADE 有轻微恶化；
  - 说明模型可能向评分者偏好靠拢，而非严格复现 logged GT。
- **真实安全验证不足**：
  - 没有碰撞率、舒适性、规则遵守、闭环干预率等真实部署指标；
  - 无法直接判断其在 L4 自动驾驶系统中的安全性。
