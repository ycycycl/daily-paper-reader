## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型在通用机器人操作方面展现出强大潜力，但在泛化到未见任务时仍面临困难，因为这些任务需要在对象、场景和动作模式之间迁移相关经验。本文提出 VLA-Pro，一个即插即用框架，旨在通过在训练阶段存储任务相关的程序性记忆，并在推理过程中迁移这些记忆来增强跨任务泛化能力。具体而言，VLA-Pro 在训练期间将任务特定的 LoRA 适配器存储为参数化的程序性记忆。在推理阶段，VLA-Pro 基于当前多模态上下文检索相关程序性记忆，并动态融合这些记忆以生成当前动作片段。在 RoboTwin、RLBench 以及真实世界操作任务上的实验表明，VLA-Pro 在多种骨干模型上均能持续提升跨任务泛化能力，在仿真中最高实现 207% 的相对提升，并将真实世界成功率从 5.8% 提高到 65.0%。这些结果表明，程序性记忆的检索与适配为将操作经验迁移到新任务提供了一种有效机制，同时保持了模块化性和执行稳定性。

## Abstract
Vision-Language-Action~(VLA) models have shown strong potential for general-purpose robotic manipulation, yet they still struggle to generalize to unseen tasks that necessitate transferring relevant experience across objects, scenes, and action patterns. This paper proposes VLA-Pro, a plug-and-play framework designed to enhance cross-task generalization by storing task-relevant procedural memories at training time and transferring these memories during inference. Specifically, VLA-Pro stores task-specific LoRA adapters as parameterized procedural memories during training. At inference time, VLA-Pro retrieves relevant procedural memories based on the current multi-modal context and dynamically fuses these memories for generating the current action chunk. Experiments on RoboTwin, RLBench, and real-world manipulation tasks show that VLA-Pro consistently improves cross-task generalization across multiple backbones, achieving up to a 207% relative improvement in simulation and increasing real-world success rate from 5.8% to 65.0%. These results suggest that procedural memory retrieval and adaptation provide an effective mechanism for transferring manipulation experience to novel tasks while preserving modularity and execution stability.

## 2. 方法论：核心思想、关键技术与流程

### 2.1 核心思想

VLA-Pro 是一个即插即用框架，其基本思想是：

- **训练阶段**：
  - 在已见任务上训练基础 LoRA；
  - 再为每个已见任务训练一个任务特定 LoRA；
  - 将每个任务特定 LoRA 作为一种“程序性记忆”存入记忆库。

- **推理阶段**：
  - 根据当前图像、语言指令和历史交互，提取当前“程序状态”；
  - 用该状态去检索记忆库中最相关的已见任务；
  - 将检索到的任务 LoRA 按相似度加权融合；
  - 临时加载到 VLA 骨干模型中生成当前动作片段；
  - 执行完当前动作片段后卸载，再进入下一阶段重新检索。

---

### 2.2 程序性记忆的表示

论文将每个已见任务的程序性记忆定义为：

- 一个任务记忆包含两部分：

\[
M_i = (D_i, \Delta \theta_i)
\]

其中：

- \(D_i\)：任务 \(T_i\) 的程序状态序列；
- \(\Delta \theta_i\)：该任务对应的 LoRA 参数，即任务特定的参数化经验。

换言之：

- \(D_i\) 负责描述“什么时候这段经验相关”；
- \(\Delta \theta_i\) 负责存储“具体如何执行这类经验”。

---

### 2.3 程序状态设计

VLA-Pro 不直接使用原始图像或自由文本进行检索，而是将当前执行阶段抽象为结构化程序状态：

\[
z = \{a, o, e, p\}
\]

其中：

- \(a\)：当前动作类型，例如 pick、place、press、push、drag；
- \(o\)：物体几何属性，例如 cuboid、spherical、lying-cylindrical、upright-cylindrical 等；
- \(e\)：末端执行器姿态，例如 vertical、horizontal；
- \(p\)：目标交互点，例如 front、back、left、right、center、top、rim 等。

这种结构化表示的优点是：

- 更易于索引；
- 更可解释；
- 能减少自由文本表述带来的语义噪声；
- 有助于细粒度区分相似任务。

---

### 2.4 当前程序状态提取

推理时，模型根据当前观测和历史信息提取状态：

\[
\hat{z}_t = E(O_t, l, H_{<t})
\]

其中：

- \(O_t\)：当前视觉观测；
- \(l\)：完整语言指令；
- \(H_{<t}\)：之前的交互历史；
- \(E(\cdot)\)：程序状态提取器。

论文中使用 **Gemini-3-Flash** 作为程序状态提取器，并通过 prompt 约束其输出结构化 JSON。

---

### 2.5 动作感知的程序匹配机制

对于当前程序状态 \(\hat{z}_t\)，VLA-Pro 会和每个已见任务的程序状态序列 \(D_i\) 进行比较。

任务级相似度定义为该任务序列中与当前状态最相似的状态：

\[
s_i = \max_m Sim(\hat{z}_t, z_i^{(m)})
\]

相似度不是简单地对所有字段平均，而是使用动作感知权重：

\[
Sim(\hat{z}_t, z_i^{(m)}) =
\sum_{f \in \{a,o,e,p\}} w_f(\hat{a}_t) Sim_f(\hat{z}_{t,f}, z_{i,f}^{(m)})
\]

其中：

- \(f\) 表示不同字段；
- \(Sim_f\) 是字段文本嵌入的余弦相似度；
- \(w_f(\hat{a}_t)\) 是由当前动作决定的字段权重。

例如：

- pick 动作更关注物体几何属性；
- place 动作更关注目标交互点；
- push 动作更关注末端执行器方向。

论文使用 **OpenAI text-embedding-3-large** 对程序状态字段进行文本嵌入。

---

### 2.6 LoRA 记忆融合

检索到 top-k 个最相关任务后，VLA-Pro 用 softmax 将相似度转成融合权重：

\[
\alpha_j = Softmax(s_j)
\]

然后对对应的任务 LoRA 做参数级加权求和：

\[
\Delta \theta_t^{fused}
= \sum_{j \in K_t} \alpha_j \Delta \theta_j
\]

其中：

- \(K_t\)：当前阶段检索到的 top-k 任务集合；
- \(\Delta \theta_j\)：第 \(j\) 个任务的 LoRA 参数；
- \(\Delta \theta_t^{fused}\)：当前阶段融合后的临时 LoRA。

融合后的 LoRA 被临时加载到预训练 VLA 骨干模型上，用于生成当前动作片段。执行完后卸载，下一阶段重新检索和融合。

---

### 2.7 算法流程概括

VLA-Pro 的整体流程可以概括为：

1. 在所有已见任务上训练一个 Base LoRA；
2. 以 Base LoRA 为起点，为每个已见任务继续训练任务 LoRA；
3. 提取每个已见任务的程序状态序列；
4. 构建记忆库 \((D_i, \Delta \theta_i)\)；
5. 推理时对每个动作片段：
   - 提取当前程序状态；
   - 检索 top-k 个相关任务记忆；
   - 根据相似度计算融合权重；
   - 融合 LoRA；
   - 临时加载 LoRA；
   - 生成并执行动作；
   - 卸载 LoRA，进入下一阶段。

## 5. 实验数量与充分性

### 5.1 实验类型概览

论文实验较丰富，主要包括：

1. **RoboTwin 主实验**
   - 3 个骨干模型；
   - 9 个未见任务；
   - 每任务 30 次测试；
   - 对比 Base LoRA 与 VLA-Pro。

2. **RLBench 主实验**
   - 多种方法对比；
   - 包括 RDT、π0.5、AtomicVLA、VLA-Pro；
   - 报告 9 个 zero-shot 测试任务；
   - 每任务 25 次测试。

3. **真实机器人实验**
   - 6 个训练任务；
   - 6 个未见测试任务；
   - 每个测试任务 20 次；
   - 比较 π0.5 Base LoRA 与 VLA-Pro。

4. **参数组件消融**
   - 分析 VLM-backbone LoRA 与 action-expert LoRA 的作用；
   - 比较只用 VLM、只用 action-expert、两者都用。

5. **检索器分析**
   - 比较 Gemini-3-Flash 与 Qwen2.5-VL-7B；
   - 使用 MRR 衡量程序状态提取和检索质量。

6. **任务相似度分析**
   - 分析 unseen-seen 相似度与性能提升的关系。

7. **top-k 消融实验**
   - 比较 baseline、k=1、k=2、k=3；
   - 发现 RLBench 上 k=2 效果最好。

8. **运行时分析**
   - 报告 Gemini 与 embedding API 的调用延迟；
   - 分析缓存和 LoRA 复用带来的效率优势。

### 5.2 实验是否充分

总体来看，实验比较充分：

- 覆盖了：
  - 仿真环境；
  - 标准 benchmark；
  - 真实机器人；
  - 多种骨干模型；
  - 多种任务类型；
  - 主结果与消融分析。

- 公平性较好：
  - Base LoRA 与 VLA-Pro 使用相同骨干模型；
  - VLA-Pro 在 Base LoRA 基础上继续训练任务特定 LoRA；
  - 测试任务与训练任务严格区分；
  - RoboTwin、RLBench 和真实世界任务均采用 held-out unseen tasks。

但也存在一定限制：

- RoboTwin 和真实世界任务数量仍然偏少；
- 真实实验每个任务 20 次，虽有参考价值，但统计规模有限；
- RLBench 结果只重点展示 9 个至少有非零成功率的任务，完整 23 个 held-out 任务的整体分布不够突出；
- 程序状态提取依赖外部强 VLM 和 API，可能引入额外变量。

## 6. 主要结论与发现

### 6.1 RoboTwin 结果

在 RoboTwin 上，VLA-Pro 对三个骨干模型均有提升：

- X-VLA：
  - Base：17.0%
  - VLA-Pro：30.0%
  - 相对提升约 76%。

- RDT：
  - Base：11.1%
  - VLA-Pro：34.1%
  - 相对提升约 207%。

- π0.5：
  - Base：40.4%
  - VLA-Pro：59.3%
  - 相对提升约 47%。

这说明 VLA-Pro 不是依赖单一模型结构，而是可以作为通用的插件式机制提升不同 VLA 骨干的跨任务泛化能力。

---

### 6.2 RLBench 结果

在 RLBench 上，VLA-Pro 取得最高平均成功率：

- RDT：10.2%
- π0.5：13.8%
- AtomicVLA：14.7%
- VLA-Pro + π0.5：20.9%

相比 π0.5 baseline，VLA-Pro 提升 7.1 个百分点，相对提升约 51%。

---

### 6.3 真实世界结果

真实机器人实验中，VLA-Pro 显著提升未见任务表现：

- π0.5 Base LoRA：
  - 平均成功率 5.8%。
- VLA-Pro：
  - 平均成功率 65.0%。

提升幅度约为 59.2 个百分点。

这表明程序性记忆迁移不仅在仿真中有效，也能迁移到真实机器人操作场景。

---

### 6.4 消融实验发现

- **VLM-backbone 与 action-expert 的作用不同**：
  - 在 seen tasks 上，同时适配两者效果最好；
  - 在一些 unseen tasks 上，仅适配 VLM-backbone 反而更有利于泛化；
  - action-expert 更偏向精细动作执行，可能在新任务中带来过拟合风险。

- **检索质量影响最终成功率**：
  - Gemini-3-Flash 的程序状态提取 MRR 高于 Qwen2.5-VL-7B；
  - 更高的检索准确性对应更高的任务成功率。

- **任务相似度越高，迁移收益越大**：
  - 未见任务与已见任务的程序相似度越高，VLA-Pro 的提升越明显。

- **top-k 并非越大越好**：
  - RLBench 中 k=2 最优；
  - k=1 可能信息不足；
  - k=3 可能引入无关记忆干扰。

## 7. 优点与亮点

### 7.1 方法设计亮点

- **将程序性记忆参数化**
  - 不只是把历史示例放入 prompt，而是把任务经验存储为 LoRA 参数。
  - 这种方式更接近“可执行记忆”。

- **即插即用**
  - VLA-Pro 不要求重新设计 VLA 主体架构；
  - 可以应用于 X-VLA、RDT、π0.5 等不同骨干。

- **结构化程序状态**
  - 使用动作、物体几何、末端姿态、目标点等字段进行检索；
  - 比直接用图像或自由文本检索更可解释。

- **动作感知检索**
  - 不同动作关注不同属性；
  - 例如 pick 重视物体几何，place 重视目标点；
  - 这种设计符合机器人操作的实际规律。

- **动态 LoRA 融合**
  - 每个执行阶段都可以重新检索和融合记忆；
  - 适合多阶段任务和长程任务。

- **模块化且低存储开销**
  - LoRA 参数相对轻量；
  - 记忆库可以通过增加 LoRA adapter 扩展。

---

### 7.2 实验设计亮点

- 同时在：
  - RoboTwin；
  - RLBench；
  - 真实机器人；
  上验证，实验覆盖面较广。

- 使用多个骨干模型，证明方法具有一定通用性。

- 不仅给出主结果，还分析了：
  - 参数组件；
  - 检索器；
  - 任务相似度；
  - top-k；
  - 运行时延迟。

- 真实机器人实验提升明显，增强了方法的实际说服力。

## 8. 不足与局限

### 8.1 对程序状态提取器依赖较强

- VLA-Pro 的性能高度依赖程序状态提取质量。
- 如果当前图像模糊、遮挡严重、历史信息不完整，状态提取可能错误。
- 一旦提取错误，就会检索到不相关 LoRA，导致动作执行失败。

---

### 8.2 依赖外部大模型与 API

- 推理中使用 Gemini-3-Flash 提取程序状态；
- 使用 OpenAI text-embedding-3-large 进行文本嵌入。
- 这带来几个问题：
  - 系统复杂度增加；
  - 存在网络延迟；
  - 成本和可部署性受限；
  - 对闭源模型能力有依赖；
  - 复现实验可能受到 API 版本变化影响。

---

### 8.3 记忆库规模有限

- RoboTwin 只有 8 个训练任务；
- 真实世界只有 6 个训练任务；
- RLBench 只选取 8 个任务作为 source memories。
- 当前实验主要验证“相关任务之间”的迁移，对大规模、多样化、开放世界记忆库的表现尚不清楚。

---

### 8.4 对高度新颖任务能力有限

论文自身也承认：

- VLA-Pro 主要通过相似度检索并融合已有任务 LoRA；
- 它更擅长处理与已见任务相关的未见任务；
- 对于需要深度组合推理、创造性重组经验或全新物体交互方式的任务，能力仍然有限。

---

### 8.5 可能存在记忆干扰

- top-k 融合可能引入无关任务记忆。
- RLBench 中 k=3 不如 k=2，说明更多记忆并不一定更好。
- 如何判断何时使用记忆、何时退回 base policy，仍是开放问题。

---

### 8.6 真实世界实验规模仍有限

- 真实任务只有 6 个训练任务和 6 个测试任务；
- 每个测试任务 20 次；
- 虽然结果提升显著，但仍不足以全面证明其在复杂真实开放环境中的鲁棒性。
- 场景主要是桌面操作，尚未覆盖：
  - 动态物体；
  - 多机器人协作；
  - 更长时序任务；
  - 强遮挡；
  - 高接触力任务；
  - 多视角与多模态传感。

---

### 8.7 统计显著性与失败分析不足

- 论文主要报告成功率均值；
- 没有充分报告置信区间、方差或统计显著性检验；
- 对失败案例的系统分类也较少。
- 因此难以判断部分任务提升是否具有稳定统计意义。

---

## 总体评价

VLA-Pro 提出了一种较有启发性的 VLA 跨任务泛化框架：将任务经验以 LoRA 形式存储为“程序性记忆”，并在推理阶段根据结构化程序状态检索、融合和临时注入模型。该方法在 RoboTwin、RLBench 和真实机器人实验中均表现出明显提升，尤其真实场景中从 5.8% 提升到 65.0%，说明其对缓解高频动作偏置和提升相关任务迁移有实际价值。

不过，该方法仍依赖高质量程序状态提取、外部大模型 API 和相对有限的记忆库。它目前更适合“与训练任务存在明显程序相似性”的未见任务，而对于高度开放、组合性强、需要创造性推理的新任务，仍需要更灵活的层次化记忆、组合式技能表达和不确定性感知检索机制。
