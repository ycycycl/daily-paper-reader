## 摘要
具身智能通常通过面向单一任务（如操作或导航）的专门模型进行研究，这导致能力碎片化，并限制了其在任务、环境和机器人本体之间的泛化能力。在本文中，我们研究异构的具身决策问题能否在单一的视觉-语言-动作模型中得到统一。我们提出 Qwen-VLA，这是一种统一的具身基础模型，通过基于 DiT 的动作解码器，将 Qwen 的视觉-语言建模体系从感知、理解与推理扩展到连续动作与轨迹生成。Qwen-VLA 采用大规模联合预训练方案，在多样化数据源上进行训练，包括机器人操作轨迹、人类第一视角示范、合成仿真数据、视觉-语言导航数据、以轨迹为中心的监督数据以及辅助视觉-语言数据。为支持多种机器人平台，我们引入本体感知的提示条件机制，其中机器人特定的文本描述用于指定当前本体及其控制约定。我们进一步将操作、导航和轨迹预测表述为统一的动作与轨迹预测框架，从而实现跨机器人形态、任务类别和环境的可迁移视觉定位、空间推理与连续动作生成。在操作、导航和以轨迹为中心的基准测试上的实验表明，该模型在场景布局、背景、光照、物体配置和机器人本体变化下表现出稳定的多任务性能和分布外泛化能力。Qwen-VLA-Instruct 在 LIBERO 上达到 97.9%，在 Simpler-WidowX 上达到 73.7%，在 RoboTwin-Easy/Hard 上分别达到 86.1%/87.2%，在 R2R 上达到 69.0% OSR，在 RxR 上达到 59.6% SR，在真实世界 ALOHA 实验中达到 76.9% 的平均 OOD 成功率，并在 DOMINO 动态操作任务上实现 26.6% 的零样本成功率。

## Abstract
Embodied intelligence is often studied through specialized models for individual tasks such as manipulation or navigation, resulting in fragmented capabilities and limited generalization across tasks, environments, and robot embodiments. In this work, we study whether heterogeneous embodied decision-making problems can be unified within a single vision-language-action model. We present Qwen-VLA, a unified embodied foundation model that extends Qwen's vision-language modeling stack from perception, understanding, and reasoning to continuous action and trajectory generation through a DiT-based action decoder. Qwen-VLA is trained with a large-scale joint pretraining recipe over diverse data sources, including robotics manipulation trajectories, human egocentric demonstrations, synthetic simulation data, vision-and-language navigation data, trajectory-centric supervision, and auxiliary vision-language data. To support multiple robot platforms, we introduce embodiment-aware prompt conditioning, where robot-specific textual descriptions specify the current embodiment and control convention. We further cast manipulation, navigation, and trajectory prediction into a unified action-and-trajectory prediction framework, enabling transferable visual grounding, spatial reasoning, and continuous action generation across robot morphologies, task families, and environments. Experiments on manipulation, navigation, and trajectory-centric benchmarks show consistent multi-task performance and out-of-distribution generalization under variations in scene layout, background, lighting, object configuration, and robot embodiment. Qwen-VLA-Instruct achieves 97.9% on LIBERO, 73.7% on Simpler-WidowX, 86.1%/87.2% on RoboTwin-Easy/Hard, 69.0% OSR on R2R, 59.6% SR on RxR, 76.9% average OOD success in real-world ALOHA experiments, and 26.6% zero-shot success on DOMINO dynamic manipulation.

## 2. 方法论：核心思想、关键技术与训练流程

## 2.1 统一问题建模

论文将多种具身任务统一为条件预测问题：

\[
p_\theta(y_{t:t+H-1} \mid o_t, x, e, z)
\]

其中：

- \(o_t\)：视觉上下文，可以是单帧图像、多帧图像、视频或历史观测；
- \(x\)：自然语言任务指令；
- \(e\)：机器人本体描述，包括机器人类型、控制频率、动作空间、预测步长等；
- \(z\)：可选任务标识；
- \(y_{t:t+H-1}\)：未来 \(H\) 步动作或轨迹。

不同任务对应的输出包括：

- 机器人操作：末端执行器位姿、关节角、夹爪状态、灵巧手关节等；
- 导航：二维平面中的相对位移和朝向变化 \((\Delta x, \Delta y, \Delta \theta)\)；
- 人类第一视角动作：双手腕部运动、手部姿态、MANO 或骨架轨迹；
- 轨迹预测：未来空间轨迹。

核心思想是：**不强行把不同机器人动作转换成同一种物理语义，而是在统一张量接口下保留各自控制约定，并通过文本提示告知模型当前语义。**

## 2.2 模型架构

Qwen-VLA 由两部分组成：

### 2.2.1 视觉语言骨干：Qwen3.5-4B

- 使用 Qwen3.5 多模态模型作为基础骨干。
- 支持图像、视频和文本的早期融合。
- 视觉 token 与文本 token 被放入同一 Transformer 序列中处理。
- 具备较强的：
  - 细粒度视觉感知；
  - 指代表达理解；
  - 空间关系推理；
  - 多语言指令跟随；
  - 结构化推理能力。

### 2.2.2 动作专家：DiT + Flow Matching

- 在 VLM 骨干之上添加一个 **DiT 风格的动作解码器**。
- 动作解码器约 **1.15B 参数**：
  - 16 个 DiT block，占约 1.13B 参数；
  - 其余为动作投影 MLP、VLM hidden state 映射层、时间步嵌入和 AdaLN 调制层。
- 输入包括：
  - VLM 的隐藏状态；
  - 加噪后的动作 chunk；
  - diffusion / flow matching 时间步。
- 输出为连续动作序列。
- 推理时通过少量 Euler 积分步骤从噪声生成干净动作，因此适合实时控制。

## 2.3 本体感知提示 Embodiment-aware Prompt Conditioning

为了支持多种机器人平台，论文不为每种机器人设计独立 head，而是使用文本提示描述当前机器人本体。

模板大致为：

> The robot is {robot_tag} with {single arm / dual arms}[, waist][, and mobile base].  
> The control frequency is {FPS} Hz.  
> Please predict the next {chunk_size} control actions to execute the following task: {ori_instruction}.

提示中包含：

- 机器人名称；
- 单臂 / 双臂；
- 是否有腰部或移动底盘；
- 控制频率；
- 动作 chunk 长度；
- 原始任务指令；
- 当前动作空间或控制约定。

这一设计的关键作用是：

- 把机器人本体差异显式文本化；
- 允许同一个模型适配不同动作维度和控制方式；
- 避免为不同机器人设计单独策略网络；
- 提高跨本体泛化能力。

## 2.4 统一动作与轨迹表示

- 每个训练样本的目标动作表示为：

\[
Y \in \mathbb{R}^{H \times K}
\]

其中：

- \(H\)：统一预测 horizon；
- \(K\)：统一通道维度；
- 实际任务只使用前 \(c \leq K\) 个通道；
- 未使用通道用 0 padding；
- 使用 mask \(M \in \{0,1\}^{H \times K}\) 屏蔽无效维度和无效时间步。

这种设计允许：

- 同一个 DiT 解码器处理不同机器人动作空间；
- 不同任务保留原始动作语义；
- 通过 mask 避免 padding 部分影响梯度；
- 不需要每种本体单独输出头。

## 2.5 训练目标

### 2.5.1 Flow Matching 动作损失

对于连续动作数据，采用 conditional flow matching。

给定干净动作 \(Y_0\) 和噪声 \(Y_1 \sim \mathcal{N}(0,I)\)，构造插值：

\[
Y_\tau = (1-\tau)Y_0 + \tau Y_1
\]

模型学习预测速度场：

\[
Y_1 - Y_0
\]

损失使用 mask 后的逐通道均方误差，并先在时间维度平均，再在有效动作通道上平均，以避免动作维度多的本体主导训练。

### 2.5.2 视觉语言损失

为了避免模型在大规模动作训练中遗忘原有视觉语言能力，保留 next-token prediction 损失：

\[
L_{vl} = - \sum_i \log p_\theta(w_i \mid w_{<i}, o_{1:t})
\]

用于：

- 图像 / 视频问答；
- 空间 grounding；
- 自动驾驶 VQA；
- 细粒度动作 caption；
- 通用视觉语言数据。

### 2.5.3 总损失

\[
L = \lambda_{act} L_{act} + \lambda_{vl} L_{vl}
\]

通过权重平衡动作生成和视觉语言理解。

## 2.6 四阶段训练流程

论文提出渐进式训练流程：

### Stage I：Text-to-Action DiT Pretraining, T2A

- 冻结 VLM，仅训练 DiT 动作解码器。
- 不输入图像，只使用语言指令和本体提示预测动作。
- 目标是让 DiT 学会从压缩的文本描述“解压”为高维动作轨迹。
- 这一阶段相当于建立语言条件下的动作先验。

### Stage II：Continued Pretraining, CPT

- 解冻 VLM 和 DiT。
- 使用多源异构数据联合训练。
- 目标是把 T2A 学到的动作先验与视觉观测对齐。
- 产物是 **Qwen-VLA-Base**。

### Stage III：Supervised Fine-tuning, SFT

分为两条路线：

- 多任务 SFT：
  - 包括 VQA、空间 grounding、操作、导航等；
  - 用于得到更强的通用任务执行能力。
- 真实机器人 SFT：
  - 使用 ALOHA 等真实遥操作数据；
  - 用于验证真实世界迁移。

### Stage IV：Reinforcement Learning, RL

- 从 SFT checkpoint 初始化。
- 使用 PPO + GAE。
- 奖励为仿真器给出的稀疏二值成功信号。
- RL 仅在 SimplerEnv 中进行。
- 最终得到 **Qwen-VLA-Instruct**。

## 6. 主要结论与发现

- **统一建模是可行的**：
  - 操作、导航、人类第一视角动作和轨迹预测可以被统一为动作 / 轨迹预测问题。
  - 单一模型可以同时处理多个任务族和多个机器人本体。

- **通用模型可以达到甚至超过专用模型性能**：
  - Qwen-VLA-Instruct 在 LIBERO 达到 97.9%；
  - 在 Simpler-WidowX 达到 73.7%；
  - 在 RoboTwin-Easy / Hard 达到 86.1% / 87.2%；
  - 多项结果超过或接近专用策略模型。

- **预训练对真实机器人迁移非常关键**：
  - ALOHA 上，从头训练平均 in-domain 成功率为 48.5%；
  - 使用 Qwen-VLA-Base 预训练后提升到 83.6%；
  - OOD 平均成功率从 36.2% 提升到 76.9%。

- **多源联合预训练增强 OOD 泛化**：
  - 在静态 OOD SimplerEnv-OOD 中，Qwen-VLA-Instruct 达到 32.0%，高于 \(\pi_{0.5}\) 的 12.6%。
  - 在 DOMINO 动态操作中，Qwen-VLA-Instruct 零样本达到 26.6% SR 和 39.5 MS，超过多个经过动态数据微调的方法。

- **T2A 是有效的动作先验学习阶段**：
  - 文本到动作预训练能让 DiT 学到语言条件下的动作结构。
  - 约 20% 合成 + 80% 真实数据的 T2A 组合最佳。
  - T2A 阶段不输入图像更好。
  - T2A 使用 Sigmoid-Normal 时间步分布、SFT 使用 Beta 分布效果最好。

- **VL 数据不会显著干扰动作学习，反而有帮助**：
  - 在复杂场景如 RoboCasa 和 RoboTwin 上，加入 VL 数据能提升细粒度识别和组合指令理解。

- **RL 后训练提供额外增益且不会明显遗忘**：
  - RL 在 SimplerEnv 中训练，但在其他 benchmark 上性能基本保持或略有提升。
  - 说明成功率优化可以改善策略执行稳定性和闭环表现。

## 7. 优点与亮点

## 7.1 方法设计亮点

- **统一任务表述清晰**：
  - 将操作、导航、轨迹预测、人类动作建模统一为动作 / 轨迹预测，具有较强扩展性。

- **本体感知提示设计简单有效**：
  - 用自然语言描述机器人平台和控制约定，避免复杂的多 head 或专门架构。
  - 适合多机器人本体共训。

- **动作空间处理务实**：
  - 不强行统一不同机器人的物理动作语义；
  - 保留原始控制约定；
  - 通过 padding + mask 实现统一张量接口。

- **DiT + Flow Matching 适合连续动作生成**：
  - 相比离散 action token，连续动作建模更自然；
  - 能生成平滑动作 chunk；
  - 推理时少量 Euler 步骤支持低延迟控制。

- **T2A 阶段具有启发性**：
  - 把语言到动作看作“压缩语义到高维动作的解压过程”；
  - 先训练动作解码器，再进行视觉 grounding，缓解随机初始化 DiT 对 VLM 的干扰。

- **训练流程完整**：
  - T2A → CPT → SFT → RL；
  - 分别对应动作先验、视觉 grounding、任务对齐、闭环优化。

---

## 7.2 实验设计亮点

- **覆盖范围广**：
  - 操作、导航、真实机器人、静态 OOD、动态 OOD 均有评估。

- **同时比较 Base 和 Instruct**：
  - 可以观察预训练、SFT 和 RL 的阶段性贡献。

- **真实世界验证较充分**：
  - ALOHA 上包含短程、长程、变形物体、精细操作等任务。
  - OOD 设置覆盖颜色、物体、位置、背景、指令变化。

- **消融实验丰富**：
  - 对关键设计选择进行了系统分析，而不仅仅报告最终结果。

- **动态操作零样本结果有价值**：
  - DOMINO 上未使用动态数据微调却超过多种动态操作方法，显示了统一预训练的潜力。

## 8. 不足与局限

## 8.1 数据与复现限制

- 训练依赖大规模公开、私有和合成数据。
- 私有数据和内部仿真 pipeline 的细节有限。
- 算力、训练时长、GPU 配置未公开。
- 完整复现难度很高。

## 8.2 公平比较风险

- Qwen-VLA 的训练数据规模和多样性可能显著超过部分基线。
- 不同模型的训练数据、任务适配方式、微调策略并不完全一致。
- “通用模型超过专用模型”的结论有说服力，但仍需在严格等数据、等算力条件下进一步验证。

## 8.3 长时程能力仍有限

- 虽然论文包含长程导航和一些长程操作任务，但总体评估仍以 benchmark 为主。
- 对真实世界长时间自主运行、失败恢复、任务重规划、持续记忆的验证不足。

## 8.4 物理交互能力仍不完整

- 当前主要依赖视觉和语言。
- 默认不使用 proprioception，因为消融中收益较小。
- 对接触丰富任务、力控、触觉反馈、柔性物体复杂操作等场景仍可能不足。

## 8.5 动态环境能力仍需进一步验证

- DOMINO 零样本结果很好，但成功率仍只有 26.6%，说明动态操作仍远未解决。
- 当前模型主要使用当前帧观测，没有显式建模运动历史或预测未来物体状态。
- 对高速动态物体、遮挡、多人多物体交互等更复杂场景仍存在不确定性。

## 8.6 优化权衡问题

- 论文自己也指出，联合训练 VL、VLN 和 VLA 可能带来目标冲突。
- 动作能力增强可能导致某些纯视觉语言或导航能力轻微退化。
- 如何平衡多任务损失、数据采样和模块专门化仍是开放问题。

## 8.7 安全与部署问题

- 论文主要报告成功率，但对失败模式、安全约束、异常检测、碰撞风险讨论有限。
- 真实机器人部署仍需要额外安全机制。
- 在工业、家庭、医疗等开放环境中直接使用仍有风险。

# 总体评价

Qwen-VLA 是一篇系统性很强的具身基础模型论文。它的主要贡献不是某一个单点算法，而是提出并验证了一套大规模统一 VLA 建模范式：用 Qwen3.5 视觉语言骨干提供感知和推理能力，用 DiT flow-matching 解码器生成连续动作，用本体感知文本提示统一多机器人平台，并通过 T2A、CPT、SFT、RL 的渐进训练流程实现跨任务、跨环境和跨本体泛化。

论文实验覆盖非常广，在仿真、真实机器人、导航、静态 OOD 和动态 OOD 上都展示了较强结果，说明统一 VLA 方向具有实际潜力。不过，论文也存在大模型论文常见问题：训练数据和算力不够透明，复现门槛高，对比公平性受数据规模影响，长时程真实部署和物理接触能力仍有待进一步验证。
