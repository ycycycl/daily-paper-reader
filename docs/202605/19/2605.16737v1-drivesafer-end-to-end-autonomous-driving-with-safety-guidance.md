## 摘要
近年来，端到端（End-to-End, E2E）自动驾驶模型展现出日益增强的能力，并在难度不断提升的基准测试中取得了性能改进。然而，现代生成式端到端规划器在安全关键场景中仍会出现大量灾难性失败。我们发现，许多此类失败源于对物理约束和安全要求的违反，从而导致不安全行为。受此发现启发，本文聚焦于通过有针对性地减少灾难性规划失败来改善生成式端到端驾驶的安全结果，而非提升平均规划质量。为此，我们提出了 DriveSafer，一个面向端到端规划器的失败感知安全框架。DriveSafer 利用训练阶段的安全约束和推理阶段的安全引导，显式地将生成式规划器引向安全行为。与最先进的 DiffusionDrive 模型相比，在 NAVSIM 基准测试上，DriveSafer 将灾难性失败（PDMS=0）的数量减少了 48%，并将可行驶区域合规性失败减少了超过 65%。

## Abstract
End-to-End (E2E) autonomous driving models have shown growing capability in recent years, with performance improving on increasingly challenging benchmarks. However, modern generative E2E planners still suffer from a substantial number of catastrophic failures in safety-critical scenarios. We find that many such failures arise from violations of physical constraints and safety requirements, leading to unsafe behavior. Motivated by this finding, in this paper, we focus on improving safety outcomes in generative end-to-end driving with a targeted reduction of catastrophic planning failures, instead of enhancing average planning quality. Towards this end, we propose DriveSafer, a failure-aware safety framework for end-to-end planners. DriveSafer explicitly steers generative planners towards safe behaviors leveraging both training-time safety constraints and inference-time safety guidance. Compared to the state-of-the-art DiffusionDrive model, on the NAVSIM benchmark, DriveSafer reduces the number of catastrophic failures (PDMS=0) by 48%, with over 65% reduction in drivable-area compliance failures.

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 基础模型：基于 DiffusionDrive

- DriveSafer 以 **DiffusionDrive** 为基础模型。
- DiffusionDrive 是一种截断扩散模型，用于端到端自动驾驶轨迹规划。
- 其输入包括：
  - 摄像头数据；
  - 激光雷达数据；
  - 中间感知表示，如目标检测、跟踪、地图分割等。
- DiffusionDrive 不从纯噪声开始生成轨迹，而是从若干典型轨迹模式附近初始化，再通过少量扩散去噪步骤生成最终轨迹。
- 该模型推理速度约为 **45 FPS**，具备一定实时部署潜力。

### 2.2 DriveSafer 的核心思想

DriveSafer 通过两个阶段增强安全性：

1. **训练阶段安全约束**  
   在原始规划损失之外加入安全相关损失，使模型在训练时学习避免不安全轨迹。

2. **推理阶段安全引导**  
   在模型生成候选轨迹之后，基于地图和周围交通参与者预测，对轨迹进行安全评分与筛选，选择更安全的轨迹输出。

其目标不是简单优化平均轨迹误差，而是直接降低以下灾难性失败：
- 碰撞；
- 驶出可行驶区域；
- 违反驾驶方向；
- 生成物理上不舒适或不合理的轨迹。

---

### 2.3 训练阶段：安全感知损失设计

DriveSafer 的总损失为：

\[
L_{total} = L_{base} + \lambda_{DAC} L_{DAC} + \lambda_{Col} L_{Col} + \lambda_{Comf} L_{Comf}
\]

其中：

- \(L_{base}\)：原始 DiffusionDrive 训练损失；
- \(L_{DAC}\)：可行驶区域合规损失；
- \(L_{Col}\)：碰撞避免损失；
- \(L_{Comf}\)：舒适性损失；
- \(\lambda_{DAC}, \lambda_{Col}, \lambda_{Comf}\)：不同安全损失项的权重。

#### 1）可行驶区域合规损失 \(L_{DAC}\)

- 作用：惩罚预测轨迹驶出可行驶区域。
- 方法：
  - 使用地图提供的二值可行驶区域 mask；
  - 如果预测的 ego 车辆位置不在可行驶区域内，则产生惩罚。
- 该损失直接针对 NAVSIM 中常见的 drivable area compliance failure。

#### 2）碰撞损失 \(L_{Col}\)

- 作用：惩罚 ego 车辆轨迹与其他交通参与者过近。
- 计算方式：
  - 计算 ego 预测位置与周围 agent 的距离；
  - 如果距离小于碰撞阈值 \(d_{col}\)，则产生惩罚。
- 训练阶段中，周围交通参与者的位置使用 ground truth。

#### 3）舒适性损失 \(L_{Comf}\)

- 作用：减少轨迹中的突然运动变化，提升乘坐舒适性和物理合理性。
- 方法：
  - 根据连续轨迹点计算速度；
  - 再根据速度变化计算 jerk；
  - 如果 jerk 超过阈值 \(j_{th}\)，则进行惩罚。
- 该项约束模型生成更平滑的轨迹。

---

### 2.4 轻量级预测模块：联合预测与规划

- 作者指出，DiffusionDrive 原本主要预测当前时刻 agent 位置，并没有专门优化未来 agent 运动预测。
- 但推理阶段若要进行碰撞规避，必须知道周围车辆、行人未来可能位置。
- 因此，DriveSafer 增加了一个 **轻量级 adapter 层**，用于预测其他交通参与者未来时刻的位置。
- 这样可以实现：
  - ego 轨迹规划；
  - 周围 agent 未来运动预测；
  - 基于未来交互关系进行安全评估。

---

### 2.5 推理阶段：安全引导与候选轨迹选择

推理期安全引导是 DriveSafer 的关键组成部分。

#### 1）生成多个候选轨迹假设

- DriveSafer 不直接接受扩散模型输出的最高分轨迹，而是围绕其生成替代方案。
- 候选轨迹来源包括：
  - 对最高排名轨迹进行扰动；
  - 使用扩散模型排名第二、第三的轨迹模式；
  - 对 heading、speed 进行约 5% 的变化；
  - 对轨迹进行亚米级横向偏移。

#### 2）安全评分

- 对所有候选轨迹，基于以下信息计算安全指标：
  - HD map 中的可行驶区域；
  - 预测的其他 agent 未来位置；
  - 碰撞风险；
  - 可行驶区域合规性；
  - 方向和运动合理性等。

#### 3）选择最安全轨迹

- DriveSafer 最终选择安全评分最高的候选轨迹作为输出。
- 该过程类似于一个轻量级安全过滤器或安全重排序机制。
- 作者强调，该方法依赖于较准确的 ego localization，假设自动驾驶车辆可以达到亚车道级定位精度。

- 实验主要集中于 NAVSIM navtest。
- 缺少在其他 benchmark 上的验证，例如：
  - NuPlan；
  - Bench2Drive；
  - CARLA 闭环场景；
  - Waymo Open Dataset E2E；
  - 真实道路数据。
- 因此，方法泛化能力仍需要进一步验证。

### 8.3 不是闭环真实驾驶评估

- NAVSIM 虽然比单纯 open-loop L2 误差更安全导向，但仍不是完整真实闭环驾驶。
- 论文没有验证：
  - 车辆执行 DriveSafer 轨迹后其他交通参与者如何反应；
  - 长时间闭环累积误差；
  - 复杂交互场景中的策略稳定性。

### 8.4 对地图和定位依赖较强

- 推理期安全评分依赖 HD map 和 ego localization。
- 论文假设车辆具备亚车道级定位精度。
- 如果出现：
  - 地图过期；
  - 定位漂移；
  - GNSS 失效且视觉定位不稳定；
  - 施工或临时道路变化；
  
  DriveSafer 的安全引导可能受到影响。

### 8.5 对其他 agent 预测误差敏感

- 碰撞规避依赖未来 agent 位置预测。
- 如果周围车辆或行人行为突变，轻量 adapter 预测不准，安全评分可能失效。
- 论文没有系统测试 agent 预测噪声对最终规划安全性的影响。

### 8.6 仍未消除全部灾难性失败

- DriveSafer 将 PDMS=0 案例从 653 减少到 340，但并未归零。
- 对真实自动驾驶而言，剩余失败仍然可能不可接受。
- 论文也承认未来目标是进一步将灾难性失败降至零。

### 8.7 推理开销未量化

- 候选轨迹生成、安全评分、agent 预测都会带来额外计算成本。
- 论文没有报告 DriveSafer 相比 DiffusionDrive 的实际 FPS、延迟和内存开销。
- 因此其实时部署能力仍需更多证据支持。
