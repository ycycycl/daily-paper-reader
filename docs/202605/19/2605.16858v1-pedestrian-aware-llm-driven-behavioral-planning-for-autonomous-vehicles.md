## 摘要
自动驾驶车辆（Autonomous Vehicles, AVs）必须在密集城市环境中做出可靠决策，而在此类环境中，行人行为具有可变性，有时表现异常，并且往往在训练过程中未曾出现。基于强化学习（reinforcement learning, RL）的自动驾驶控制系统在结构化交通场景中表现良好，但难以泛化到不可预测的行人交互和分布外场景。其对手工设计奖励的依赖以及决策过程的不透明性，进一步限制了其在安全关键且行人密集环境中的适用性。为解决这些局限，我们提出了一种基于大型语言模型（Large Language Model, LLM）的决策框架，用于面向行人感知的行为规划。该系统将结构化场景观测转换为自然语言推理提示，使 LLM 能够推断行人意图、预判风险，并生成谨慎的战术驾驶决策。这些决策由运动规划器执行，以确保平滑且满足运动学可行性的控制。我们在 SUMO 中针对多种行人交互场景评估了该框架，包括意外乱穿马路、折返横穿、犹豫横穿以及双向横穿。在零样本评估中，基于 LLM 的智能体实现了 68% 的无碰撞成功率，显著优于深度 RL 基线（17.7%）。在单行人场景中引入少样本情节记忆后，性能提升至 96.0%，超过了定制的 DQN 控制器（82.0%）。跨行为评估进一步表明，源自折返交互的记忆能够迁移到未见过的犹豫横穿和双向横穿场景，成功率分别达到 82.0% 和 90.0%。该系统始终能够更早地发起响应，保持更宽的安全缓冲区，并生成可解释且符合人类行为预期的决策。

## Abstract
Autonomous Vehicles (AVs) must make reliable decisions in dense urban environments where pedestrian behavior is variable, sometimes abnormal, and often unseen during training. Reinforcement learning (RL)-based AV control systems perform well in structured traffic but struggle to generalize to unpredictable pedestrian interactions and out-of-distribution scenarios. Their reliance on handcrafted rewards and opaque decisions further limits their suitability for safety-critical, pedestrian-rich environments. To address these limitations, we introduce a Large Language Model (LLM)-based decision-making framework for pedestrian-aware behavioral planning. The system converts structured scene observations into natural-language reasoning prompts, enabling the LLM to infer pedestrian intent, anticipate risk, and generate cautious tactical driving decisions. These decisions are executed by a motion planner that ensures smooth, kinematically feasible control. We evaluate the framework in SUMO across multiple pedestrian-interaction scenarios, including unexpected jaywalking, turn-back crossing, hesitation, and bidirectional crossing. In zero-shot evaluation, the LLM-based agent achieves a 68% collision-free success rate, substantially outperforming deep RL baselines (17.7%). With few-shot episodic memory in a single-pedestrian scenario, performance increases to 96.0%, exceeding a custom DQN controller (82.0%). Cross-behavior evaluation further shows that memory derived from turn-back interactions transfers to unseen hesitation and bidirectional crossing scenarios, achieving 82.0% and 90.0% success, respectively. The system consistently initiates earlier responses, maintains wider safety buffers, and produces interpretable, human-aligned decisions.

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 总体框架

论文提出一个 **LLM 驱动的行人感知行为规划框架**，系统主要由四个部分组成：

1. **Scenario Generator：场景生成器**
2. **Decision-Making Module：LLM 决策模块**
3. **Action Parser and Motion Planner：动作解析器与运动规划器**
4. **Memory Module：情节记忆模块**

系统工作在闭环、滚动时域控制框架下：每个时间步读取当前环境状态，生成自然语言提示，调用 LLM 进行推理，输出高层动作，再由运动规划器转化为可执行控制。

---

### 2.2 场景表示与提示构造

- 系统从 SUMO 仿真环境中获取结构化观测，包括：
  - 自车位置、速度、航向角、车道编号；
  - 周围行人和车辆的位置、速度、方向；
  - 道路几何、车道信息、交通规则和速度限制等。
- 自车状态表示为：

\[
s^{ego}_t = (x_t, y_t, v_t, \theta_t, lane_t)
\]

其中：

- \(x_t, y_t\)：自车位置；
- \(v_t\)：自车速度；
- \(\theta_t\)：航向角；
- \(lane_t\)：所在车道。

- 这些结构化信息被转换成自然语言 prompt，输入给 LLM，使其能够基于空间关系、速度、行人位置和潜在风险做推理。

---

### 2.3 LLM 决策模块

- LLM 接收自然语言场景描述后，生成高层战术决策，例如：
  - “由于前方有行人，减速”；
  - “保持当前车道”；
  - “向左变道避让”；
  - “加速通过”；
  - “停车等待行人通过”。
- 论文强调该模块的优势在于：
  - 可解释；
  - 可利用常识推理；
  - 可在未见行人行为中进行零样本或少样本泛化；
  - 不依赖特定场景的大规模训练或奖励函数设计。

---

### 2.4 动作解析器

LLM 的自然语言输出会被解析成离散动作空间：

\[
A = \{0,1,2,3,4\}
\]

对应含义为：

| 动作编号 | 动作含义 |
|---|---|
| 0 | Stay Idle，保持/空闲 |
| 1 | Turn Left，向左转/向左变道 |
| 2 | Turn Right，向右转/向右变道 |
| 3 | Accelerate，加速 |
| 4 | Decelerate，减速 |

动作解析器本质上是一个确定性映射：

\[
M: T \rightarrow A
\]

其中 \(T\) 是 LLM 生成的文本，\(A\) 是离散动作集合。

---

### 2.5 运动规划器

- LLM 不直接输出连续控制量，而是输出高层动作。
- 低层运动规划器负责将这些高层动作转换为满足车辆运动学约束的轨迹和速度控制。
- 车辆采用简化的非完整约束运动学模型：

\[
x_{t+1} = x_t + v_t \cos \theta_t \Delta t
\]

\[
y_{t+1} = y_t + v_t \sin \theta_t \Delta t
\]

\[
\theta_{t+1} = \theta_t + k_\theta(\theta_{target} - \theta_t)
\]

\[
v_{t+1} = v_t + a\Delta t
\]

其中：

- \(\theta_{target}\)：目标航向角；
- \(v_{target}\)：目标速度；
- \(a\)：加速度；
- \(k_\theta\)：航向调整系数；
- \(\Delta t\)：时间步长。

加速度被限制在物理可行范围内：

\[
a = \min(a_{max}, \max(-a_{max}, \frac{v_{target} - v_t}{\Delta t}))
\]

这保证车辆不会出现不现实的瞬时加速、急停或横向跳变。

---

### 2.6 情节记忆模块

论文还设计了一个 few-shot episodic memory，用于提升 LLM 在相似行人交互中的稳定性。

- 每条记忆并不保存完整文本，而是保存结构化特征向量：

\[
x = [d_x, d_y, v_{ego}, v_{ped}, TTC]
\]

其中：

- \(d_x\)：纵向相对距离；
- \(d_y\)：横向偏移；
- \(v_{ego}\)：自车速度；
- \(v_{ped}\)：行人速度；
- \(TTC\)：time-to-collision，预计碰撞时间。

- 系统使用 **kNN 检索** 在归一化特征空间中找到最相似的历史状态。
- 检索出的记忆会被反序列化为自然语言模板，包括：
  - 场景描述；
  - 推理过程；
  - 曾经选择的安全动作。
- 这些示例会作为 few-shot prompt 提供给 LLM，帮助其在当前场景中做出更稳定的决策。
- 论文特别强调：使用数值向量检索而非文本检索，可以减少延迟，并使检索更依赖几何和动态相似性，而不是文字相似性。

## 6. 主要结论与发现

论文得到的主要结论如下：

- **LLM 在零样本行人场景中优于部分 RL 基线**：
  - Gemma3-27B 无记忆时达到 **68.0%** 无碰撞成功率；
  - Dueling DQN 为 **17.7%**；
  - RPF 为 **42.8%**。

- **情节记忆显著提升部分 LLM 的安全性和成功率**：
  - Gemma3-1B 在单行人已见场景中：
    - 无记忆成功率：**72.0%**
    - 有记忆成功率：**96.0%**
  - 同时最小行人距离从 **2.63 m** 提升到 **4.34 m**；
  - 最小横向距离从 **0.96 m** 提升到 **3.17 m**。

- **LLM 倾向于保持更大安全缓冲**：
  - DQN 在某些场景中成功率较高，但横向安全距离非常小。
  - 例如三行人场景中，DQN 成功率达到 **92.0%**，但最小横向距离仅约 **0.034 m**。
  - Gemma3-1B with memory 成功率为 **88.0%**，但横向距离达到 **1.462 m**，安全裕度明显更大。

- **记忆可迁移到未见行人行为**：
  - 使用 turn-back 场景构建的记忆，在未见的 hesitation 和 bidirectional crossing 中仍表现较好：
    - turn-back：**94.0%**
    - hesitation：**82.0%**
    - bidirectional crossing：**90.0%**

- **LLM 决策更可解释**：
  - LLM 输出自然语言推理和决策，便于后验分析。
  - 相比 RL 黑箱策略，LLM 更容易判断其为何减速、避让或保持车道。

- **计算延迟是代价**：
  - DQN 每步决策约 1.3 ms；
  - Gemma3-1B 平均约 0.118 s，最大超过 1.26 s；
  - LLM 较慢，但对于低频战术决策仍可能可用。

---

## 7. 优点与亮点

### 方法层面

- **将 LLM 用于行人感知行为规划**：
  - 论文明确聚焦行人安全，而不是常见的车车交互或高速道路行为规划。
- **结构化状态到自然语言推理的桥接设计清晰**：
  - 既避免端到端感知控制的不可控性，也保留 LLM 的语义推理优势。
- **高层决策与低层运动规划解耦**：
  - LLM 负责战术选择；
  - 运动规划器保证物理可行性。
- **引入数值型情节记忆检索**：
  - 使用 \(d_x, d_y, v_{ego}, v_{ped}, TTC\) 等几何动力学特征进行 kNN 检索；
  - 比纯文本检索更适合驾驶交互场景。
- **具有可解释性**：
  - LLM 输出自然语言原因，使行为规划更便于审查和调试。

### 实验层面

- **覆盖零样本、少样本、多行人和跨行为迁移**：
  - 实验不是单一场景验证，而是逐步测试泛化能力。
- **与多个 RL 基线对比**：
  - 包括 DQN、Dueling DQN、RPF。
- **多指标评价**：
  - 不只看成功率，还看最小距离、横向安全距离和 TTC。
- **报告推理延迟**：
  - 讨论了实际部署中的计算效率问题。

---

## 8. 不足与局限

### 8.1 仿真与场景限制

- 实验仅在 **SUMO** 中进行。
- SUMO 是微观交通仿真器，适合测试交通交互和行为规划，但缺少：
  - 摄像头图像；
  - LiDAR 点云；
  - 遮挡；
  - 感知误差；
  - 复杂道路拓扑；
  - 真实交通参与者外观和行为细节。
- 因此结果不能直接说明该方法可在真实自动驾驶系统中稳定运行。

---

### 8.2 输入假设较强

- 系统使用结构化状态作为输入，默认已知：
  - 行人位置；
  - 行人速度；
  - 车辆位置；
  - 车道信息；
  - TTC 等。
- 现实中这些信息需要感知、跟踪和预测模块提供，且可能存在误差。
- 论文没有评估在不完美感知下的鲁棒性。

---

### 8.3 LLM 决策可靠性仍需验证

- LLM 可能出现：
  - 输出格式不一致；
  - 推理幻觉；
  - 对危险程度判断错误；
  - prompt 敏感性；
  - 在复杂多主体环境下遗漏关键行人。
- 论文使用动作解析器缓解输出问题，但没有系统分析 LLM 输出失败模式。

---

### 8.4 公平性问题

- RL baseline 与 LLM 的先验知识来源不同：
  - RL 依赖特定环境训练；
  - LLM 拥有大规模语言预训练中的常识知识。
- Exp1 中 RL 没有接触过行人，而 LLM 虽然没有任务微调，但可能在预训练中拥有大量交通和行人常识。
- 因此实验更能说明 LLM 的零样本常识泛化优势，但不能完全说明 LLM 在所有条件下优于 RL。

---

### 8.5 规模与泛化仍有限

- 行人数量最多测试到三人，尚未覆盖真正高密度人群。
- 未测试：
  - 夜间；
  - 雨雪；
  - 遮挡后突然出现行人；
  - 复杂交叉口；
  - 非机动车混行；
  - 多车多行人共同博弈。
- memory bank 的规模较小：
  - 单行人记忆为 82 个状态向量；
  - turn-back 记忆为 114 个状态向量。
- 尚不清楚在更大、更复杂记忆库下是否会出现检索噪声、冲突示例或延迟增加。

---

### 8.6 部署效率问题

- LLM 推理延迟明显高于 DQN。
- 虽然论文通过并行 LLM 调用和滚动时域执行缓解延迟，但在真实高速驾驶或紧急制动场景中，百毫秒级甚至秒级延迟仍可能带来风险。
- 论文未充分讨论安全冗余机制，例如：
  - emergency brake；
  - rule-based safety shield；
  - formal verification；
  - control barrier function。

---

## 总体评价

这篇论文提出了一个较有启发性的 LLM 驱动自动驾驶行为规划框架，重点解决传统 RL 在未见行人行为中的泛化和可解释性问题。方法上，它将结构化交通状态转为自然语言推理任务，并结合情节记忆和运动学规划器，使 LLM 能够输出较安全、可解释且符合人类直觉的战术驾驶决策。实验表明，该方法在 SUMO 的多种行人交互场景中可以提升无碰撞成功率和横向安全裕度，尤其在 few-shot memory 设置下表现突出。

不过，该研究目前仍处于受控仿真验证阶段，强依赖准确结构化状态，尚未解决真实自动驾驶中的感知噪声、遮挡、复杂交通和实时安全认证问题。因此，它更适合作为 LLM 辅助行为规划的概念验证，而不是可以直接部署的完整自动驾驶方案。
