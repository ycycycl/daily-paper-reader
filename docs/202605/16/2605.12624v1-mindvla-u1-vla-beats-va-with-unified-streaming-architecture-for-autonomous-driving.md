## 摘要
自动驾驶已从模块化流水线发展到端到端统一，而视觉-语言-动作（Vision-Language-Action，VLA）模型是这一进程超越视觉到动作（Vision-to-Action，VA）的自然延伸。在实践中，驾驶 VLA 在规划质量上往往落后于 VA，这表明难点并不只是模型规模，而在于语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的是 VLA 的构建方式——作为彼此孤立的子任务改进，未能组合成连贯的驾驶能力——而非 VLA 本身的局限。我们提出 MindVLA-U1，这是首个面向自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干网络在对一个共享表示进行单次前向传递时，同时生成自回归语言 token 和基于流匹配的连续动作轨迹，保留每种模态的自然输出形式。流式设计以逐帧方式处理驾驶视频，而非固定的视频-动作块，同时通过学习到的记忆通道在帧间传递时间上下文，使规划轨迹能够平滑演化，而无需冗余的多帧 VLM 建模。该统一架构通过灵活的自注意力上下文管理，支持在密集/稀疏 Mixture-of-Transformers（MoT）骨干网络上进行快/慢速执行，并呈现出一条可度量的语言到动作路径：由语言预测的驾驶意图通过无分类器引导（classifier-free guidance，CFG）引导动作扩散，将语言侧意图转化为连续轨迹生成的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越有经验的人类驾驶员（8.20 RFS vs. 8.13 GT RFS），仅需 2 个扩散步骤；相较既有 VA/VLA 方法，在规划 ADE 上以较大优势达到最先进水平；并在保留自然语言接口的同时，实现了与 VA 相当的吞吐率（16 FPS vs. RAP-DINO 的 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose into coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces autoregressive language tokens and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A streaming design processes the driving video framewise rather than as fixed video-action chunks, while a learned memory channel carries temporal context across frames so planned trajectories evolve smoothly without redundant multi-frame VLM modeling. The unified architecture admits fast/slow execution on dense/sparse Mixture-of-Transformers (MoT) backbones via flexible self-attention context management, and exposes a measurable language-to-action route: a language-predicted driving intent steers action diffusion through classifier-free guidance (CFG), turning language-side intent into a control signal for continuous trajectory generation. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA methods by large margins, and matches VA-class throughput (16 FPS vs. RAP-DINO's 18 FPS) while preserving natural-language interfaces.

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体架构思想

MindVLA-U1 的核心思想是：

> 使用一个统一共享的 VLM backbone，在同一次前向传播中同时处理视觉、语言、车辆状态、记忆和动作 token，并分别输出语言 token 与连续动作轨迹。

它不是把语言模型和动作规划器简单串联，也不是让 VLM 输出离散轨迹 token，而是：

- **语言保持自回归生成**；
- **动作保持连续轨迹形式**；
- **语言与动作共享同一个 Transformer 表征空间**；
- **动作 token 进入 backbone 的自注意力和 FFN，而不是由外部专家单独处理**。

这样既保留 VLA 的语言理解能力，又避免牺牲 VA 的连续控制精度。

---

## 2.2 统一共享 Backbone：AR 语言 + Flow Matching 动作

在每个时间步，模型输入包括：

- 多视角图像 token：`I`
- ego-state 历史：`e`
- 语言 query：`q`
- 语言 answer token：`a`
- noisy action token：`x_t`
- streaming memory token：`m`

模型通过同一个 VLM backbone 处理这些 token。

### 语言分支

语言采用标准自回归建模，损失为：

\[
L_{AR} = -\sum_l \log p_\theta(a_l | I, e, q, a_{<l})
\]

即根据当前视觉、车辆状态和问题，生成场景问答、驾驶意图等语言输出。

### 动作分支

动作采用 **flow matching / diffusion-style continuous action generation**。

- 真实未来轨迹为 \(x_0=w\)；
- 随机噪声为 \(\epsilon\)；
- noisy action token 为：

\[
x_t = t\epsilon + (1-t)x_0
\]

- 模型预测从噪声到真实轨迹的速度场：

\[
L_{FM} = \|v_\theta(x_t,t;I,e,q,a)-(\epsilon-x_0)\|^2
\]

推理时，通过少量 ODE/Euler 积分步骤，从噪声恢复连续轨迹。论文主实验中只使用 **2 个 diffusion / Euler steps**。

### 总损失

\[
L = L_{AR} + L_{FM}
\]

语言和动作虽然共用 backbone，但各自使用独立输出头：

- LM head 负责语言 token；
- 轻量 MLP action head 负责连续动作速度场。

作者认为二者目标不同但互补，不会在实践中产生严重冲突。

## 2.3 语言到动作桥梁：Intent-CFG

论文的重要创新之一是提出 **Intent-CFG**，即用语言侧预测的驾驶意图控制动作扩散。

流程如下：

1. VLM 通过语言头自回归预测当前驾驶意图 token，例如：
   - left
   - right
   - straight
   - 或 MindLabel 扩展的 20 类驾驶意图。

2. 将预测意图 token 嵌入为向量，加入 action MLP 的时间嵌入中。

3. 训练时使用 classifier-free guidance dropout：
   - 部分样本用真实意图；
   - 部分样本替换为 unconditional token \(\emptyset\)。

4. 推理时分别计算：
   - 无条件速度场 \(v_\theta(x_t,t;\emptyset)\)
   - 条件速度场 \(v_\theta(x_t,t;z)\)

5. 通过 CFG 混合：

\[
v_{cfg}(x_t,t)=v_\theta(x_t,t;\emptyset)+s(v_\theta(x_t,t;z)-v_\theta(x_t,t;\emptyset))
\]

其中 guidance scale \(s=1.5\)。

这一机制使得语言预测出的“驾驶意图”能够直接影响连续轨迹生成，而不是停留在解释层面。

## 2.4 帧级 Streaming Memory

传统 VLA 往往输入多帧视频，让 VLM 自己建模时间关系，但这会带来大量冗余 token 和高计算成本。

MindVLA-U1 改为 **逐帧流式处理**：

- 每次只输入当前多视角帧；
- 历史信息通过一个压缩的 FIFO memory channel 传递；
- memory token 不是原始图像 token，而是过去帧 backbone 状态的压缩摘要。

### 记忆更新流程

每一帧执行：

1. 从 FIFO memory 中读取过去若干帧的 memory token；
2. 根据车辆位姿变化进行 SE(2) motion alignment；
3. 将 memory token 与当前图像、ego-state、语言和动作 token 一起输入 backbone；
4. 前向完成后，用 Q-Former-style propagation transformer 从 backbone 输出中压缩得到新的 memory token；
5. 新 memory 写入 FIFO，最旧 memory 被移除。

论文默认配置：

- 每帧写入 \(N_m=128\) 个 memory token；
- FIFO 保留 \(N_g=2\) 个历史帧；
- 总 memory capacity 为 256 tokens。

这一机制让模型在长序列中保持时序连续性，同时避免固定 chunk 带来的边界不连续。

## 2.5 快/慢系统与 MoT 设计

MindVLA-U1 支持多种推理模式：

- `vqa_first`：先生成语言/推理，再规划动作；
- `action_first`：先规划动作，再生成语言解释；
- `action_only`：只做动作规划，跳过语言生成；
- `vqa_only`：只做问答或解释。

这些模式通过 attention mask 和 token context 管理实现，不需要额外模型。

### Sparse Mixture-of-Transformers

论文还探索了 MoT 版本：

- context group：视觉、语言 token；
- action group：memory、ego-state、action token；
- 两组通过共享 self-attention K/V 池交互；
- FFN 专家按功能分组。

这样可以在慢速模式中保留语言推理，在快速模式中移除语言生成 token，实现更高推理效率。

## 3.3 对比方法

论文与多类方法比较。

### VA 方法

包括：

- VAD；
- UniAD；
- RAP-DINO；
- DiffusionDrive / UniPlan；
- Swin-Trajectory；
- ViT-Adapter-GRU 等。

其中 RAP-DINO 是验证集上非常强的 VA baseline。

### VLA 方法

包括：

## 4. 资源与算力

论文明确给出了部分训练和推理资源。

### 训练资源

主模型训练配置：

- GPU：**8 × NVIDIA H200**
- 精度：BF16
- 分布式训练：DeepSpeed ZeRO-2
- 训练时长：约 **7 小时**
- 训练轮数：50 epochs
- backbone：主要使用 Qwen3-VL-2B
- 优化器：AdamW
- 学习率：\(10^{-4}\)
- weight decay：0.1

### 推理资源

推理吞吐评估：

- 单卡：**1 × H200**
- batch size = 1

部分推理速度：

- Qwen3-VL-2B dense slow path：
  - `vqa_first_decoding` 约 2594 ms；
  - 约 0.39 FPS。
- Qwen3-VL-2B dense fast path：
  - `action_only` 约 103 ms；
  - 约 9.70 FPS。
- `vqa_first_fast` template QA，2 steps：
  - 约 108 ms；
  - 约 9.26 FPS。
- 1-step fast：
  - 约 63 ms；
  - 约 15.92 FPS。
- InternVL-2 1B fast configuration：
  - 约 64 ms；
  - 约 15.55 FPS。
- RAP-DINO 作为 VA 参考：
  - 约 57 ms；
  - 约 17.68 FPS。

作者指出慢速路径主要瓶颈是自回归 VQA decoding，约占 slow path 延迟的 94%。

## 5. 实验数量与充分性

## 5.1 实验覆盖范围

论文实验较多，主要包括：

1. **WOD-E2E validation 主结果对比**
   - 与 VA、VLA、人类驾驶轨迹对比。

2. **WOD-E2E official test 主结果对比**
   - 与官方 leaderboard 或已有方法对比。

3. **Intent-CFG 消融**
   - no-intent baseline；
   - trajectory-derived intent；
   - GT-supplied intent；
   - NTP-predicted intent。

4. **Intent-CFG 多模态可控性可视化**
   - 展示同一场景下不同 intent 产生不同轨迹。

5. **MoT 专家分组消融**
   - dense；
   - `{V,L,M,S}+{A}`；
   - `{V,L}+{M,S,A}`。

6. **推理吞吐实验**
   - slow / fast path；
   - 1-step / 2-step diffusion；
   - 与 RAP-DINO 对比。

7. **Streaming memory 消融**
   - chunk-wise single frame；
   - chunk-wise + 4-frame image sequence；
   - streaming without memory；
   - streaming + memory。

8. **RL post-training 实验**
   - SFT checkpoint；
   - GRPO using RFS reward。

9. **Val/Test 分布偏移诊断**
   - 使用 MindLabel intent 分析验证集和测试集意图分布差异。

10. **Backbone scaling 实验**
    - Qwen3.5-VL 0.8B、2B、4B、9B；

11. **长序列 streaming qualitative examples**
    - 17 秒多 clip 拼接；
    - per-frame streaming trajectory smoothness。

整体来看，实验数量较多，覆盖了主性能、架构消融、效率、时序、语言控制、模型规模和数据分布分析。

---

## 5.2 充分性与公平性评价

### 充分之处

- 同时报告 validation 和 official test，避免只在验证集调参后报最优。
- 与强 VA 方法 RAP-DINO 比较，能说明其确实接近或超过 VA 规划质量。
- 消融较细，分别验证：
  - 语言到动作路径；
  - streaming memory；
  - MoT；
  - fast/slow inference；
  - RL 后训练；
  - backbone scaling。
- 论文没有只报告最终大模型结果，也报告了无 RL、无外部 AD pretraining 版本，较有助于判断架构本身贡献。
- 对 val/test 分布差异做了诊断，说明作者意识到 benchmark 评测存在分布偏移。

### 可能不足

- 所有核心结果集中在 **WOD-E2E 一个 benchmark** 上，缺乏 nuScenes、NAVSIM、闭环仿真或真实车部署验证。
- 部分对比方法是否在完全相同训练数据、外部预训练、推理模式下比较，并不总是完全可控。
- MindLabel 自动标注依赖强 VLM，可能引入标注偏差或 teacher model bias。
- 人类 RFS 被超越是在 open-loop 评价下，不代表真实闭环驾驶一定优于人类。
- RL 使用 RFS reward 后提升明显，但可能更适配该 benchmark 的评分机制，而非泛化驾驶安全性。

## 6. 主要结论与发现

- **MindVLA-U1 在 WOD-E2E 上达到 SOTA。**

### Validation split

- MindVLA-U1 dense：
  - RFS 7.83。
- MindVLA-U1 + Intent-CFG：
  - RFS 7.92。
- MindVLA-U1 + MoT：
  - RFS 7.92。
- MindVLA-U1 + RL：
  - RFS 8.20。
- 人类 logged trajectory：
  - RFS 8.13。

因此，作者声称 MindVLA-U1 + RL 首次在 WOD-E2E validation 的 RFS 指标上超过经验人类驾驶员。

### Official test split

- MindVLA-U1：
  - RFS 7.77；
  - RFS-GT ADE 1.16 / 2.67。
- MindVLA-U1 + RL：
  - RFS 7.87；
  - RFS-GT ADE 1.09 / 2.66。

优于列出的 VA 和 VLA 方法。

---

## 6.1 关于语言到动作的发现

- 普通 intent injection 不一定提升性能：
  - trajectory-derived intent：RFS 7.81；
  - GT-supplied intent：RFS 7.83；
  - no-intent baseline：RFS 7.83。
- 但 **NTP-predicted Intent-CFG** 达到：
  - RFS 7.92；
  - 说明语言侧预测意图经过合适 embedding 后，可以有效引导连续动作扩散。

论文认为这证明语言不是装饰性模块，而是具有可测量的动作控制作用。

---

## 6.2 关于 Streaming Memory 的发现

Streaming memory 带来明显提升：

- chunk-wise single frame：RFS 7.69；
- streaming no memory：RFS 7.73；
- streaming + memory：RFS 7.83。

直接把 4 帧图像输入 VLM 的 DeepStack 方案反而下降到 RFS 7.61，说明：

> 对驾驶 VLA 来说，压缩记忆通道比冗余多帧 VLM token 更有效。

---

## 6.3 关于模型规模的发现

Qwen3.5-VL backbone scaling 非单调：

- 0.8B：RFS 7.81；
- 2B：RFS 7.94；
- 4B：RFS 7.86；
- 9B：RFS 7.84；
- 9B 训练 200 epochs 后：RFS 7.91。

说明在当前数据和训练预算下，单纯增大 VLM backbone 不必然提升规划性能。

同时，base vs instruction-tuned 结果显示：

- 语言 BLEU/ROUGE 大幅下降；
- 但 RFS 基本保持。

作者据此认为：

> 动作规划更依赖共享 scene representation，而不是语言生成表面质量。

---

## 7. 优点与亮点

## 7.1 架构设计亮点

- **统一共享 backbone**  
  语言和动作真正共享 Transformer 表征，而不是简单 pipeline 或外挂 action expert。

- **保留连续动作形式**  
  避免把轨迹离散化成 token，减少精度损失。

- **语言到动作路径明确可测**  
  Intent-CFG 提供了可实验验证的语言控制机制。

- **流式时序建模合理**  
  逐帧处理 + FIFO memory 更符合真实驾驶输入流，而不是固定 chunk。

- **快/慢模式统一在一个 checkpoint 内**  
  可根据场景需要选择语言推理或快速动作规划。

- **MoT 设计有扩展潜力**  
  将 perception/context 与 motor/action 分组，为未来加入 safety、world model、reasoning expert 提供接口。

---

## 7.2 实验设计亮点

- 对比覆盖 VA、VLA、人类驾驶轨迹；
- 同时报告精度、RFS、语言质量和吞吐率；
- 做了比较完整的组件消融；
- 不只报告 SOTA，还分析：
  - 推理瓶颈；
  - 分布偏移；
  - backbone scaling；
  - RL reward 与 ADE 的 trade-off。
- MindLabel 数据生成管线为驾驶 VLA 语言监督和偏好监督提供了可复用资源。

---

## 8. 不足与局限

## 8.1 Open-loop 评价限制

- WOD-E2E 是 logged dataset，模型预测不会影响环境。
- RFS 和 ADE 衡量的是 open-loop 轨迹质量。
- 超过 human RFS 并不等价于闭环真实驾驶超过人类。
- 未验证模型在自身动作分布下长期运行的安全性与稳定性。

---

## 8.2 Benchmark 覆盖有限

- 所有核心结果集中在 WOD-E2E。
- 未在 nuScenes、NAVSIM、CARLA closed-loop 或真实车上验证。
- 因此泛化能力仍不明确。

---

## 8.3 数据与标注偏差

- MindLabel 依赖 Qwen3-VL / Qwen3.5-Plus 自动标注。
- 自动生成的 QA、intent、dreamed trajectory 可能继承 teacher model 偏差。
- 尽管主结果只使用基础 VQA 和 3 类 intent，但语言监督仍可能存在系统性偏差。

---

## 8.4 Val/Test 分布偏移明显

论文自己指出：

- validation 中 accelerating、starting 比例高；
- test 中 waiting、following 等减速/等待类场景显著增多。

这会影响 RFS，并说明当前 benchmark 不同 split 的意图分布并不一致。

---

## 8.5 Intent 表达仍较粗

- 主实验主要使用 WOD-E2E 的 3 类 intent：
  - left；
  - right；
  - straight。
- 对复杂驾驶行为如：
  - yielding；
  - emergency stop；
  - waiting；
  - avoiding obstacle；
  - parking；
  - U-turn  
  表达能力不足。

作者也承认更丰富的 20 类 MindLabel intent 尚未充分用于主模型。

---

## 8.6 推理速度仍有部署挑战

- 虽然 fast path 接近 VA 速度，但 Qwen3-VL-2B dense fast path 仍约 9–10 FPS。
- slow path 因自回归语言解码只有约 0.39 FPS，不适合高频控制。
- 真正车载部署还需要：
  - KV cache 复用；
  - 1-step flow distillation；
  - slow language path 低频运行；
  - 硬件优化。

---

## 8.7 Scaling 结论尚不充分

- backbone scaling 非单调，但不能说明 VLA 不具备 scaling law。
- 论文承认需要同时控制：
  - 训练 compute；
  - 数据规模；
  - action head 容量；
  - 训练 schedule；
  - open-loop / closed-loop 评价。  
- 当前结果更多说明“当前实验预算下规模不是唯一瓶颈”。

---

## 总体评价

MindVLA-U1 是一篇目标明确、系统性较强的自动驾驶 VLA 论文。它的核心贡献不是单个模块，而是重新设计了驾驶 VLA 的接口：用共享 VLM 表征连接语言与连续动作，用流式记忆处理时间上下文，并用 Intent-CFG 给语言到动作建立可度量通路。实验显示该设计在 WOD-E2E 上显著提升规划质量，并在一定配置下达到接近 VA 的吞吐率。

不过，其“超过人类驾驶”和“VLA beats VA”的结论应限定在 WOD-E2E open-loop RFS 指标下理解。是否能迁移到闭环真实驾驶、多 benchmark 和车载部署，还需要进一步验证。

（完）
