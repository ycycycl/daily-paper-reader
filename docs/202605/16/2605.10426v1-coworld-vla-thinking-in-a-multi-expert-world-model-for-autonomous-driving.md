## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景范式。然而，现有推理机制仍难以提供面向规划的中间表征：文本式思维链（Chain-of-Thought, CoT）无法保留连续的时空结构，而潜在世界推理仍难以作为动作生成的直接条件。在本文中，我们提出 CoWorld-VLA，这是一种用于自动驾驶的多专家世界推理框架，其中世界表征作为显式条件来指导动作规划。CoWorld-VLA 通过多源监督提取互补的世界信息，并将其编码为 VLA 内的专家令牌，从而提供规划器可访问的条件信号。具体而言，我们构建了四类令牌：语义交互、几何结构、动态演化和自车轨迹令牌，分别建模交互意图、空间结构、未来时间动态和行为目标。在动作生成过程中，CoWorld-VLA 采用基于扩散的分层多专家融合规划器，并在联合去噪过程中与场景上下文耦合，以生成连续的自车轨迹。实验表明，CoWorld-VLA 在 NAVSIM v1 基准上的未来场景生成和规划任务中均取得了具有竞争力的结果，在碰撞避免和轨迹精度方面表现出强劲性能。消融研究进一步验证了专家令牌的互补性及其作为动作生成规划条件的有效性。代码将发布于 https://github.com/potatochip1211/CoWorld-VLA。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for end-to-end autonomous driving. However, existing reasoning mechanisms still struggle to provide planning-oriented intermediate representations: textual Chain-of-Thought (CoT) fails to preserve continuous spatiotemporal structure, while latent world reasoning remains difficult to use as a direct condition for action generation. In this paper, we propose CoWorld-VLA, a multi-expert world reasoning framework for autonomous driving, where world representations serve as explicit conditions to guide action planning. CoWorld-VLA extracts complementary world information through multi-source supervision and encodes it into expert tokens within the VLA, thereby providing planner-accessible conditioning signals. Specifically, we construct four types of tokens: semantic interaction, geometric structure, dynamic evolution, and ego trajectory tokens, which respectively model interaction intent, spatial structure, future temporal dynamics, and behavioral goals. During action generation, CoWorld-VLA employs a diffusion-based hierarchical multi-expert fusion planner, which is coupled with scene context throughout the joint denoising process to generate continuous ego trajectories. Experiments show that CoWorld-VLA achieves competitive results in both future scene generation and planning on the NAVSIM v1 benchmark, demonstrating strong performance in collision avoidance and trajectory accuracy. Ablation studies further validate the complementarity of expert tokens and their effectiveness as planning conditions for action generation. Code will be available at https://github.com/potatochip1211/CoWorld-VLA.

## 2. 方法论：核心思想、关键技术细节与流程

## 2.1 核心思想

- 论文提出 **CoWorld-VLA**，一种用于自动驾驶的多专家世界推理框架。
- 它将不同来源的世界知识编码为 VLM 隐空间中的四类专家 token：
  - **Semantic Interaction Token**：语义交互 token，用于建模交通参与者、场景语义和交互意图。
  - **Geometric Structure Token**：几何结构 token，用于建模道路布局、空间结构和 3D 几何约束。
  - **Dynamic Evolution Token**：动态演化 token，用于建模未来场景变化和时间动态。
  - **Ego Trajectory Token**：自车轨迹 token，用于建模行为目标和未来自车运动。

- 这些 token 共同构成一种 **多专家 Latent CoT**，作为轨迹规划器的显式条件。

---

## 2.2 整体框架：三阶段训练流程

CoWorld-VLA 采用三阶段训练：

### Stage 1：动作条件化预测世界模型

- 使用 Wan2.2-5B 视频生成模型作为基础世界模型。
- 输入历史视频帧和描述自车意图的文本条件，预测未来视频潜变量。
- 文本条件由四部分组成：
  - 场景描述 `[Scene]`
  - 速度描述 `[Speed]`
  - 导航命令 `[Navigation]`
  - 未来轨迹描述 `[Trajectory]`

- 训练方式：
  - 使用冻结的 Wan VAE 将视频编码为 latent。
  - 历史 latent 保持无噪声，未来 latent 加噪。
  - 采用 flow matching 目标学习未来 latent 的去噪速度场。
  - 损失只作用在未来 latent 部分，使模型专注于未来场景演化。

---

### Stage 2：多专家世界表征学习

- 使用 **Qwen3-VL-2B** 作为 VLM 主干。
- 将当前图像、文本驾驶任务提示和四组专家 token 输入 VLM。
- VLM 输出：
  - 场景上下文 hidden states：`Hctx`
  - 语义专家 hidden states：`Hsem`
  - 几何专家 hidden states：`Hgeo`
  - 动态专家 hidden states：`Hdyn`
  - 轨迹专家 hidden states：`Htraj`

- 四类监督来源：
  - **JEPA / V-JEPA 分支**  
    使用冻结的 V-JEPA 编码未来观测，提供高层语义和预测性表征监督。
  - **VGGT 分支**  
    使用冻结的 3D 几何基础模型 VGGT 提供几何结构监督。
  - **Wan 世界模型分支**  
    使用 Stage 1 训练好的 Wan 世界模型，让 `Hdyn` 作为条件信号进行未来场景生成，从而监督动态演化 token。
  - **轨迹回归分支**  
    使用轻量 MLP 从 `Htraj` 回归未来自车轨迹，提供行为目标监督。

- 总损失为多分支加权和：
  - 动态演化损失 `Ldyn`
  - 语义对齐损失 `Lsem`
  - 几何对齐损失 `Lgeo`
  - 轨迹回归损失 `Ltraj`

- 文中采用的权重为：
  - `wdyn = 1.0`
  - `wsem = 0.1`
  - `wgeo = 0.1`
  - `wtraj = 1.0`

---

### Stage 3：分层多专家融合扩散规划器 HMEF

- 论文提出 **Hierarchical Multi-Expert Fusion, HMEF**，用于将多专家 token 转换为最终可执行轨迹。
- 输入包括：
  - VLM 场景上下文 token
  - 当前帧 JEPA / VGGT 上下文 token
  - 四类专家动作 token：`Hsem, Hgeo, Hdyn, Htraj`
  - 历史自车状态、当前速度、加速度等低层车辆信息

- 关键设计：
  - **Perceiver 压缩场景 token**  
    由于场景 token 序列较长，使用 Perceiver query 将其压缩为固定数量的上下文 token，降低扩散去噪计算成本。
  - **专家特定 Transformer 编码器**  
    每类专家 token 经过各自的双向 Transformer 编码，并对齐到未来规划时间步。
  - **扩散式轨迹生成**  
    将目标轨迹归一化到动作空间，对轨迹加高斯噪声，然后通过条件去噪逐步预测干净轨迹。
  - **双流 denoiser 架构**  
    一路处理干净的场景上下文 token，另一路处理带噪声的动作 token，并通过联合 self-attention 交互。
  - **专家轨迹融合**  
    每个专家分支预测一条轨迹，模型学习全局融合权重 `α`，通过加权平均得到最终轨迹。

- 推理时：
  - 从高斯噪声轨迹开始。
  - 经过若干步扩散去噪。
  - 将不同专家预测轨迹按学习到的权重融合。
  - 反归一化得到最终自车轨迹。

  - 加入 HMEF：PDMS 89.8。

- 去噪步数方面：
  - 10 步扩散去噪在性能和效率之间表现最好。

## 7. 优点与亮点

- **提出了规划导向的 Latent CoT**
  - 不依赖冗长文本推理，而是在 VLM 隐空间中构建连续潜在推理 token，更适合自动驾驶轨迹规划。

- **多专家设计具有明确物理含义**
  - 语义、几何、动态、轨迹四类 token 分别对应自动驾驶中的关键规划因素，结构清晰。

- **将世界模型知识显式用于动作生成**
  - 不是只把未来预测作为辅助训练，而是在推理时通过专家 token 和扩散规划器直接条件化轨迹生成。

- **融合多种基础模型能力**
  - 使用 V-JEPA 提供语义和预测表征。
  - 使用 VGGT 提供几何结构先验。
  - 使用 Wan 视频生成模型提供动态世界建模能力。
  - 使用 Qwen3-VL 作为多模态 VLM 主干。

- **扩散式 HMEF 规划器设计较完整**
  - 支持不同专家轨迹分支。
  - 使用 Perceiver 降低长 token 序列计算成本。
  - 通过可学习权重自适应融合不同专家。
  - 能够生成连续轨迹，而不是离散动作或文本指令。

- **实验透明度较高**
  - 给出了训练步数、GPU 数量、训练时长和主要超参数。
  - 消融实验能够较直接支撑论文核心主张。

## 8. 不足与局限

- **计算成本高**
  - 三阶段训练总共需要大量 A800 GPU 资源。
  - Stage 1 使用 64 张 A800 训练约 74 小时，Stage 2 使用 32 张 A800 训练约 70 小时，训练门槛较高。

- **输入模态受限**
  - 当前模型主要基于单帧前视相机输入。
  - 未充分利用多帧历史、多相机环视或 LiDAR 信息。
  - 对遮挡、侧后方车辆、复杂交叉口等场景的感知可能受限。

- **泛化验证不足**
  - 规划实验主要集中在 NAVSIM v1。
  - 视频生成评估主要基于 NuPlan / NAVSIM。
  - 缺少跨城市、跨数据集、跨传感器配置的系统泛化实验。

- **公平性比较存在一定复杂性**
  - 对比方法传感器设置不同，有些使用 Camera + LiDAR，有些使用多帧输入，而 CoWorld-VLA 使用单帧 Camera。
  - 虽然 CoWorld-VLA 在弱传感器条件下表现突出，但严格公平比较仍需要统一输入设置。

- **真实部署指标不足**
  - 未详细报告推理延迟、显存占用、模型参数量和车端实时运行可行性。
  - 扩散式规划虽然只用 10 步，但在实际自动驾驶系统中的时延仍需进一步验证。

- **安全性与鲁棒性分析不足**
  - 没有充分分析极端天气、夜间、施工区、罕见交通参与者等长尾场景。
  - 缺少对模型失败案例、错误世界预测如何影响规划的深入讨论。

- **多专家监督依赖外部基础模型**
  - 方法性能依赖 V-JEPA、VGGT、Wan、Qwen3-VL 等大型预训练模型。
  - 如果外部专家模型存在偏差或错误，可能会传递到最终规划结果中。
