## 摘要
自动驾驶已从模块化流水线逐步发展到端到端统一，而视觉-语言-动作（Vision-Language-Action, VLA）模型是这一进程在视觉到动作（Vision-to-Action, VA）之外的自然延伸。在实践中，驾驶 VLA 在规划质量上往往落后于 VA，这表明困难并不单纯在于模型规模，而在于将语义推理、时间上下文与连续控制相结合的接口。我们认为，这一差距反映的是 VLA 迄今的构建方式——作为彼此孤立的子任务改进，未能组合成连贯的驾驶能力——而非 VLA 本身的局限。我们提出 MindVLA-U1，这是首个用于自动驾驶的统一流式 VLA 架构。统一的 VLM 骨干网络在一次前向传播中基于同一个共享表示生成 AR 语言 token（可选）和基于流匹配的连续动作轨迹，从而保留各模态的自然输出形式。完整的流式设计以逐帧方式处理驾驶视频，而不是在代价高昂的时间 VLM 建模下使用固定的视频-动作块。规划轨迹在帧间平滑演化，同时学习到的流式记忆通道承载并更新时间上下文。该统一架构通过灵活的自注意力上下文管理，在稠密与稀疏 MoT 骨干网络上支持快/慢系统，并为动作提供一条可度量的语言-控制路径：由语言预测的驾驶意图通过无分类器引导（classifier-free guidance, CFG）来驱动动作扩散，将语言侧意图转化为连续动作规划的控制信号。在长尾 WOD-E2E 基准上，MindVLA-U1 首次超越有经验的人类驾驶员（8.20 RFS 对比 8.13 GT RFS），且仅需 2 个扩散步；相较先前的 VA/VLA 方法，其规划 ADE 达到最先进水平并具有大幅优势；同时在保持用于人车交互的自然语言接口的情况下，达到与 VA 相当的延迟（1B 规模下为 16 FPS，而 RAP 为 18 FPS）。

## Abstract
Autonomous driving has progressed from modular pipelines toward end-to-end unification, and Vision-Language-Action (VLA) models are a natural extension of this journey beyond Vision-to-Action (VA). In practice, driving VLAs have often trailed VA on planning quality, suggesting that the difficulty is not simply model scale but the interface through which semantic reasoning, temporal context, and continuous control are combined. We argue that this gap reflects how VLA has been built -- as isolated subtask improvements that fail to compose coherent driving capabilities -- rather than what VLA is. We present MindVLA-U1, the first unified streaming VLA architecture for autonomous driving. A unified VLM backbone produces AR language tokens (optional) and flow-matching continuous action trajectories in a single forward pass over one shared representation, preserving the natural output form of each modality. A full streaming design processes the driving video framewise rather than as fixed video-action chunks under costly temporal VLM modeling. Planned trajectories evolve smoothly across frames while a learned streaming memory channel carries temporal context and updates. The unified architecture enables fast/slow systems on dense & sparse MoT backbones via flexible self-attention context management, and exposes a measurable language-control path for action: language-predicted driving intents steers the action diffusion via classifier-free guidance (CFG), turning language-side intent into control signals for continuous action planning. On the long-tail WOD-E2E benchmark, MindVLA-U1 surpasses experienced human drivers for the first time (8.20 RFS vs. 8.13 GT RFS) with 2 diffusion steps, achieves state-of-the-art planning ADEs over prior VA/VLA by large margins, and matches VA latency (16 FPS vs. RAP's 18 FPS at 1B scale) while preserving natural language interfaces for human-vehicle interaction.

  - 开放词汇的场景理解；

  - 对长尾复杂场景的语义泛化能力。

## 2. 方法论：核心思想、关键技术与流程

## 2.1 总体框架

MindVLA-U1 提出一种 **统一流式 VLA 架构**，核心目标是在一个共享 VLM 骨干中同时完成：

- 视觉场景理解；
- 可选语言问答 / 推理；
- 连续动作轨迹生成；
- 流式时间记忆传播；
- 快慢系统切换；
- 语言意图到动作控制的显式桥接。

其整体思想可以概括为：

> 用一个共享 VLM 表示同时承载语言 token 和连续动作 token，使语言保持自回归形式，动作保持连续扩散 / 流匹配形式，并通过流式记忆在逐帧驾驶流中传播时间上下文。

## 2.2 统一共享骨干：AR 语言 + Flow Matching 连续动作

MindVLA-U1 的核心是一个统一 VLM backbone。输入包括：

- 多视角视觉 token；
- ego-state 历史状态；
- 文本 query；
- 语言 answer token；
- 流式 memory token；
- noisy action token。

这些 token 共同进入同一个 Transformer backbone，共享 self-attention 和 FFN 参数。

### 语言生成

语言分支采用标准自回归建模，训练目标为交叉熵损失：

- 给定视觉、状态和问题，模型预测回答 token；
- 用于场景问答、意图预测等语言任务。

论文中语言损失可表示为：

\[
L_{AR} = - \sum_{l=1}^{L} \log p_\theta(a_l | I, e, q, a_{<l})
\]

其中：

- \(I\)：视觉 token；
- \(e\)：ego-state；
- \(q\)：语言 query；
- \(a_l\)：第 \(l\) 个回答 token。

### 连续动作生成

动作分支不将轨迹离散化为文本 token，而是使用 **flow matching / diffusion-style continuous action generation**。

- 地面真实轨迹为 \(x_0 = w\)；
- 噪声为 \(\epsilon \sim \mathcal{N}(0, I)\)；
- 在时间 \(t\) 上构造 noisy action：

\[
x_t = t\epsilon + (1 - t)x_0
\]

模型预测从噪声到数据的速度场：

\[
L_{FM} = \| v_\theta(x_t, t; I, e, q, a) - (\epsilon - x_0) \|^2
\]

推理时通过 ODE / Euler integration 从噪声逐步得到未来轨迹。

### 联合训练目标

总损失为：

\[
L = L_{AR} + L_{FM}
\]

该设计的关键点是：

- 语言和动作共享 backbone；
- 语言输出仍是自然语言 token；
- 动作输出仍是连续轨迹；
- 两者使用不同 readout head，因此不会强行把动作塞进语言 token 空间。

## 2.3 Intent-CFG：语言到动作的显式控制路径

论文提出 **Intent-CFG**，用于证明语言侧状态可以真实影响连续动作生成。

流程如下：

1. VLM 语言头预测当前场景的驾驶意图 token，例如：
   - left；
   - right；
   - go straight；
   - waiting；
   - yielding 等。

2. 将预测意图 token 的 embedding 加入动作扩散模型的时间 embedding。

3. 训练时使用 classifier-free guidance dropout：
   - 部分样本将意图替换为 unconditional token \(\emptyset\)；
   - 模型同时学习有条件和无条件速度场。

4. 推理时分别计算：
   - 无条件速度场 \(v_\theta(x_t, t; \emptyset)\)；
   - 意图条件速度场 \(v_\theta(x_t, t; z)\)。

5. 使用 CFG 混合：

\[
v_{cfg}(x_t, t) = v_\theta(x_t, t; \emptyset) + s \left( v_\theta(x_t, t; z) - v_\theta(x_t, t; \emptyset) \right)
\]

其中 \(s\) 是 guidance scale。

该设计使语言意图不只是附加解释，而是直接成为连续轨迹生成的控制信号。

## 2.4 流式记忆机制

MindVLA-U1 不采用固定多帧视频块输入，而是逐帧处理驾驶视频流。

每一帧输入包括：

- 当前多视角图像；
- 当前 ego-state；
- 历史 memory token。

### FIFO 记忆通道

模型维护一个有限长度 FIFO memory bank：

- 每帧处理后，将当前 backbone 输出压缩为 memory token；
- 加入 memory bank；
- 超出容量后移除最旧 memory；
- 下一帧读取这些 memory token 作为时间上下文。

### 运动对齐

由于每一帧车辆位姿不同，历史 memory 在读取时需要根据 ego pose 做 SE(2) 对齐。

论文使用相对位姿变换：

\[
T_{j \rightarrow i} = P_i \cdot P_j^{-1}
\]

将历史帧 \(j\) 的 memory 对齐到当前帧 \(i\) 的 ego 坐标系。

### Q-Former 风格的 memory 更新

- 使用 learnable query token 对 backbone 输出进行 cross-attention；
- 将当前帧信息压缩为固定数量 memory token；
- 该 memory token 参与后续帧的预测；
- 梯度可以跨帧传播，使 memory 成为端到端学习的状态，而不是简单缓存。

## 2.5 快慢系统与 MoT 架构

MindVLA-U1 支持多种推理模式：

- **vqa_first**：先生成语言回答 / 推理，再用语言结果辅助动作生成；
- **action_first**：先生成动作，再生成动作解释；
- **action_only**：跳过语言生成，仅做快速控制；
- **vqa_only**：仅做语言理解。

这使得同一个模型可以在：

- 普通场景中走快速 action-only 路径；
- 复杂或安全关键场景中走慢速语言推理路径。

论文还提出稀疏 **Mixture-of-Transformers（MoT）** 版本：

- context group：处理视觉和语言 token；
- action group：处理 memory、ego-state、action token；
- 两组通过共享 self-attention K/V pool 交互；
- FFN 专家按功能分组。

这样既保留统一表示，又提高 action-only 推理效率。

## 3.2 MindLabel 自动标注数据

规模约为：

- 约 3.8M VQA pairs；

## 6. 主要结论与发现

- MindVLA-U1 在 WOD-E2E 上显著提升了驾驶 VLA 的规划质量。

- 在 validation split 上：
  - MindVLA-U1 baseline RFS 为 7.83；
  - Intent-CFG 提升到 7.92；
  - MoT 版本可达到 7.92 或 8.01；
  - RL 后训练达到 8.20；
  - 超过 human driver 的 8.13 RFS。

- 在 official test split 上：
  - MindVLA-U1 + RL 达到 RFS 7.87；
  - RFS-GT ADE 为 1.09 / 2.66；
  - 超过已有 VA / VLA 方法。

- Intent-CFG 证明语言意图可以通过 CFG 直接影响连续动作扩散，不只是“语言头存在”。

- 流式 memory 比简单堆叠多帧视觉 token 更有效：
  - chunk-wise single frame RFS 7.69；
  - 4-frame DeepStack 反而降至 7.61；
  - streaming + memory 达到 7.83。

- 快慢系统显示 VLA 可以接近 VA 推理速度：
  - 1B 规模下约 15.55 FPS；
  - RAP-DINO 为 17.68 FPS；
  - 表明统一架构不必然牺牲实时性。

- Backbone scaling 结果显示：
  - 在当前数据和训练预算下，规划质量并不随 VLM 参数量单调提升；
  - 2B 模型表现较好；
  - 语言质量下降不一定导致动作质量下降，说明动作规划依赖的是共享场景表示，而非表层语言生成能力。

---

## 7. 优点与亮点

- **统一性强**  
  同一个 VLM backbone 同时处理视觉、语言、状态、memory 和动作 token，避免了 VLM 与 action expert 脱耦。

- **保持动作连续性**  
  使用 flow matching 生成连续轨迹，避免将轨迹离散化为语言 token 带来的精度损失。

- **语言到动作路径明确可测**  
  Intent-CFG 将语言预测的驾驶意图显式注入动作扩散，是论文最重要的机制之一。

- **流式设计符合驾驶场景**  
  自动驾驶输入天然是连续视频流，MindVLA-U1 逐帧处理并传播 memory，比固定 action chunk 更自然。

- **推理模式灵活**  
  同一模型支持慢速语义推理和快速 action-only 控制，部署上更有弹性。

- **实验覆盖较广**  
  包括主指标、消融、速度、RL、scaling、分布偏移、定性可视化，分析较系统。

- **使用真实世界长尾 benchmark**  
  WOD-E2E 比纯模拟器更贴近真实自动驾驶挑战。

- **MindLabel 数据管线有扩展价值**  
  自动生成 VQA、意图、dreamed trajectories 和 trajectory evaluation QA，为后续驾驶 VLA 训练提供了丰富监督。

---

## 8. 不足与局限

- **仅为 open-loop 评估**  
  WOD-E2E 是日志回放式评估，不能证明模型在闭环驾驶中同样安全可靠。论文也承认 surpass human RFS 只是 open-loop 层面的结果。

- **主要集中在单一 benchmark**  
  论文几乎所有核心结果都来自 WOD-E2E，尚未证明在 nuScenes、NAVSIM 或真实车端部署上的泛化能力。

- **测试集和验证集存在分布偏移**  
  论文发现 test split 中 waiting、following、yielding 等意图比例更高，而 validation 中 accelerating、starting 更多。这会影响结果解释。

- **RFS 指标本身可能存在偏好偏差**  
  RL 后训练直接优化 RFS，可能使轨迹更接近 rater panel 偏好，但不一定更接近真实 logged GT 或闭环安全最优策略。

- **MindLabel 自动标注可能引入模型偏差**  
  VQA 和意图由 Qwen 系列模型自动生成，可能继承其视觉理解和语言偏差；如果标注错误，会影响语言监督和 Intent-CFG。

- **对 MindLabel 的利用仍不充分**  
  主实验只使用基础 VQA 和 3 类 intent，未充分利用 20 类 intent、dreamed trajectories、CoT rationale 和 trajectory evaluation QA。

- **大模型 scaling 结论尚不充分**  
  参数量增大并未带来单调提升，但论文也指出这可能受训练预算、数据规模、action head 容量和评估方式影响，不能得出 VLA 不可扩展的结论。

- **推理速度仍有优化空间**  
  部署级 Qwen3-VL-2B fast path 约 9.7 FPS，慢速语言解码约 0.39 FPS；虽然 1B 版本接近 VA，但完整 VLA 慢路径仍不适合高频实时控制。

- **缺少真实车闭环验证**  
  对自动驾驶而言，真实道路中的安全性、鲁棒性、故障恢复、交互行为等仍未验证。

---

# 总体评价

MindVLA-U1 是一篇针对自动驾驶 VLA 架构接口问题的系统性工作。它的贡献不只是提出某个单独模块，而是将连续动作生成、语言推理、流式时序记忆、快慢系统和语言到动作控制统一到一个共享 VLM 框架中。实验结果表明，在 WOD-E2E 这一真实长尾 open-loop benchmark 上，合理设计接口后的 VLA 可以达到甚至超过强 VA 方法和人类参考 RFS。

不过，该结论仍主要局限于单一真实世界 open-loop benchmark。要证明其真正具备自动驾驶部署价值，还需要进一步的跨数据集验证、闭环仿真、真实车测试，以及更充分利用 MindLabel 多模态监督的后续研究。
