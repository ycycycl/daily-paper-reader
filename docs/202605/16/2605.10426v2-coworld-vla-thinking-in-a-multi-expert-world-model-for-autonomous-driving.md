## 摘要
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为端到端自动驾驶的一种有前景的范式。然而，现有推理机制仍难以提供面向规划的中间表示：文本形式的思维链（Chain-of-Thought, CoT）无法保留连续的时空结构，而潜在世界推理又难以作为动作生成的直接条件。本文提出 CoWorld-VLA，一种用于自动驾驶的多专家世界推理框架，其中世界表示作为显式条件来指导动作规划。CoWorld-VLA 通过多源监督提取互补的世界信息，并将其编码为 VLA 中的专家 token，从而提供规划器可访问的条件信号。具体而言，我们构建了四类 token：语义交互、几何结构、动态演化和自车轨迹 token，分别用于建模交互意图、空间结构、未来时间动态和行为目标。在动作生成过程中，CoWorld-VLA 采用基于扩散的分层多专家融合规划器，并在联合去噪过程中始终与场景上下文耦合，以生成连续的自车轨迹。实验表明，CoWorld-VLA 在 NAVSIM v1 基准上的未来场景生成和规划任务中均取得了具有竞争力的结果，在避碰和轨迹精度方面表现出色。消融研究进一步验证了专家 token 的互补性及其作为动作生成规划条件的有效性。代码将发布于 https://github.com/AFARI-Research/CoWorld-VLA。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for end-to-end autonomous driving. However, existing reasoning mechanisms still struggle to provide planning-oriented intermediate representations: textual Chain-of-Thought (CoT) fails to preserve continuous spatiotemporal structure, while latent world reasoning remains difficult to use as a direct condition for action generation. In this paper, we propose CoWorld-VLA, a multi-expert world reasoning framework for autonomous driving, where world representations serve as explicit conditions to guide action planning. CoWorld-VLA extracts complementary world information through multi-source supervision and encodes it into expert tokens within the VLA, thereby providing planner-accessible conditioning signals. Specifically, we construct four types of tokens: semantic interaction, geometric structure, dynamic evolution, and ego trajectory tokens, which respectively model interaction intent, spatial structure, future temporal dynamics, and behavioral goals. During action generation, CoWorld-VLA employs a diffusion-based hierarchical multi-expert fusion planner, which is coupled with scene context throughout the joint denoising process to generate continuous ego trajectories. Experiments show that CoWorld-VLA achieves competitive results in both future scene generation and planning on the NAVSIM v1 benchmark, demonstrating strong performance in collision avoidance and trajectory accuracy. Ablation studies further validate the complementarity of expert tokens and their effectiveness as planning conditions for action generation. Code will be available at https://github.com/AFARI-Research/CoWorld-VLA.

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 总体框架

CoWorld-VLA 采用三阶段训练流程：

1. **Stage 1：动作条件未来世界模型预训练**
   - 使用 Wan 视频生成模型的 latent space 学习未来场景演化。
   - 输入历史图像和文本化的自车意图 / 轨迹条件，预测未来视频 latent。

2. **Stage 2：多专家世界表示学习**
   - 以 Qwen3-VL-2B 作为 VLM backbone。
   - 在 VLM 输入中插入四类专家 token：
     - 语义交互 token；
     - 几何结构 token；
     - 动态演化 token；
     - 自车轨迹 token。
   - 通过不同外部专家模型或任务监督，让这些 token 学习互补的世界知识。

3. **Stage 3：分层多专家融合扩散规划**
   - 冻结 Stage 2 中训练好的 VLM。
   - 使用 Hierarchical Multi-Expert Fusion，简称 HMEF，作为扩散式轨迹规划器。
   - 在联合去噪过程中融合场景上下文和多专家 token，生成连续自车轨迹。

### 2.2 问题建模

论文将自动驾驶轨迹规划建模为条件动作生成问题：

- 输入包括：
  - 当前前视图像；
  - 导航指令；
  - 历史自车位姿；
  - 当前速度与加速度。

传统 VLA 形式为：

\[
p_\theta(A_{t+1:t+T} | o_t, c_t)
\]

即直接根据观测和条件生成未来轨迹。

CoWorld-VLA 引入多专家 latent 世界表示：

\[
Z = \{z_{sem}, z_{geo}, z_{dyn}, z_{traj}\}
\]

并将轨迹生成改写为：

\[
p_\theta(A_{t+1:t+T} | o_t, c_t, Z)
\]

其中四类表示分别对应：

- \(z_{sem}\)：语义交互；
- \(z_{geo}\)：几何结构；
- \(z_{dyn}\)：动态演化；
- \(z_{traj}\)：自车轨迹目标。

---

### 2.3 Stage 1：动作条件预测世界模型

- 使用冻结的 Wan VAE 将视频序列编码为 latent：
  - 历史 latent：\(z_h\)
  - 未来 latent：\(z_f\)

- 只对未来部分加噪并使用 flow matching 学习去噪速度场。

- 条件 prompt 由四部分组成：
  - Scene：场景描述；
  - Speed：自车速度，使用粗粒度自然语言短语；
  - Navigation：导航指令，如左转、直行、右转、未知；
  - Trajectory：未来轨迹点序列，用 ego-centric 坐标表示。

- 目标是学习：

\[
p_\theta(z_f | z_h, c)
\]

其中 \(c\) 是由冻结 Wan 文本编码器得到的条件表示。

---

### 2.4 Stage 2：多专家表示学习

Stage 2 的目标是将外部世界知识压缩进 VLM 内部的专家 token。

#### 四类专家 token

- **语义交互 token \(H_{sem}\)**
  - 由冻结 V-JEPA 提供监督。
  - 学习高层语义、交互意图、场景理解等信息。
  - 使用 SmoothL1 与 cosine loss 对齐 JEPA 特征。

- **几何结构 token \(H_{geo}\)**
  - 由冻结 VGGT 几何 foundation model 提供监督。
  - 学习道路结构、空间布局、3D 几何约束。
  - 使用 MSE loss 对齐 VGGT 特征。

- **动态演化 token \(H_{dyn}\)**
  - 用作 Wan 世界模型的条件输入。
  - 不让 VLM 直接生成图像，而是通过 Wan 生成未来场景，并用 flow matching loss 反向监督动态 token。
  - 学习未来视觉动态和时间一致性。

- **自车轨迹 token \(H_{traj}\)**
  - 通过轻量 MLP 预测未来轨迹点。
  - 使用 MSE loss 进行轨迹监督。
  - 该头主要用于塑造 latent 表示，并不是最终规划器。

#### 总损失

整体训练目标为多任务加权和：

\[
L_{total} = w_{dyn}L_{dyn} + w_{sem}L_{sem} + w_{geo}L_{geo} + w_{traj}L_{traj}
\]

论文中设置：

- \(w_{dyn}=1.0\)
- \(w_{sem}=0.1\)
- \(w_{geo}=0.1\)
- \(w_{traj}=1.0\)

---

### 2.5 Stage 3：HMEF 分层多专家融合规划器

HMEF 是最终用于轨迹生成的扩散式规划模块。

#### 输入

- 场景 token：
  - VLM 上下文 token；
  - 当前帧 JEPA token；
  - 当前帧 VGGT token。

- 动作专家 token：
  - \(H_{sem}\)
  - \(H_{geo}\)
  - \(H_{dyn}\)
  - \(H_{traj}\)

#### 核心流程

- 使用 Perceiver-style compressor 压缩长场景 token，降低计算成本。
- 每个专家分支使用独立双向 Transformer 编码。
- 将专家 token 对齐到未来规划时刻，形成每个时间步的专家特征。
- 在归一化轨迹空间中进行扩散去噪：
  - 从高斯噪声轨迹开始；
  - 逐步预测干净轨迹；
  - 每个专家分支都输出一个轨迹预测；
  - 通过可学习权重融合多个专家轨迹。

#### 推理过程

- 从随机高斯噪声开始；
- 经过若干步扩散去噪；
- 得到每个专家的轨迹预测；
- 使用 softmax 学到的专家权重融合；
- 反归一化得到可执行自车轨迹。

论文中推理阶段：
- 视频生成使用 20 个采样步；
- 轨迹规划使用 10 个去噪步。

## 6. 主要结论与发现

- CoWorld-VLA 在 NAVSIM v1 上取得了较强的规划性能：
  - PDMS 达到 89.8；
  - NC 达到 99.2；
  - EP 达到 83.6；
  - 在单帧、前视摄像头输入条件下超过多种 VLA 和世界模型方法。

- 多专家 latent token 对规划有明显帮助：
  - 仅使用 Ego Trajectory token 时 PDMS 为 83.7；
  - 加入 Geometry 后提升到 85.1；
  - 再加入 Semantic 后提升到 87.7；
  - 加入 Dynamic 后提升到 88.7。

- HMEF 进一步提升轨迹质量：
  - Stage 2 VLM 直接输出 PDMS 为 88.7；
  - 加入 ReCogDrive action expert 后为 89.1；
  - 使用 HMEF 后达到 89.8。

- 未来视频生成质量也得到提升：
  - CoWorld-VLA 的 FVD 为 32.7；
  - 优于表中 SVD、GenAD、DrivingGPT、Epona、DriveLaW 等方法。

- 定性结果表明：
  - Stage 2 相比 Stage 1 更能保持正确驾驶方向和车道级演化；
  - Stage 3 相比 Stage 2 在车道保持、转弯和绕行场景中轨迹更接近真值。

## 7. 优点与亮点

- **将世界模型表示显式用于规划**
  - 不只是把未来场景预测作为辅助任务，而是将世界表示转化为规划器可访问的条件信号。

- **多专家 Latent CoT 设计清晰**
  - 语义、几何、动态、轨迹四类 token 分工明确，能够覆盖自动驾驶规划所需的不同知识类型。

- **充分利用已有 foundation models**
  - 使用 V-JEPA 提供语义预测监督；
  - 使用 VGGT 提供几何结构监督；
  - 使用 Wan 提供未来视频动态监督；
  - 使用 Qwen3-VL 作为 VLM backbone。

- **HMEF 规划器设计合理**
  - 通过扩散去噪生成连续轨迹；
  - 通过可学习权重融合多个专家；
  - 同时利用场景上下文和专家动作 token。

- **实验覆盖主任务和辅助任务**
  - 同时评估未来视频生成与轨迹规划；
  - 消融实验能较直接证明各模块贡献。

- **资源信息透明**
  - 论文详细披露了训练所需 GPU 数量、型号、训练时长和训练步数，便于评估可复现成本。

## 8. 不足与局限

- **计算成本高**
  - 三阶段训练需要大量 A800 GPU；
  - Stage 1 使用 64 张 A800 约 74 小时；
  - Stage 2 使用 32 张 A800 约 70 小时；
  - 对普通研究团队复现不友好。

- **输入模态有限**
  - 当前主要使用单帧前视图像；
  - 未扩展到多摄像头、多帧历史、多传感器融合设置；
  - 对复杂城市驾驶中的盲区、侧后方交互建模可能不足。

- **Benchmark 覆盖有限**
  - 主规划实验集中在 NAVSIM v1；
  - 视频生成主要在 NuPlan / NAVSIM 相关设置中进行；
  - 缺少跨城市、跨天气、跨数据集泛化测试。

- **对比公平性存在一定不确定性**
  - 表中部分方法使用 LiDAR、多帧或不同训练策略；
  - CoWorld-VLA 虽然在单帧 camera-only 设置下表现突出，但不同方法输入条件并不完全一致。

- **真实部署能力未验证**
  - 论文没有展示真实车辆部署结果；
  - 没有充分讨论推理延迟、内存占用、在线闭环稳定性；
  - 对安全关键极端场景的覆盖仍有限。

- **依赖多个外部大模型**
  - 方法依赖 Wan、Qwen3-VL、V-JEPA、VGGT 等多个基础模型；
  - 系统复杂度较高；
  - 误差来源和模块间耦合关系较难完全解释。

- **世界模型与规划之间仍可能存在分布偏差**
  - 动态 token 通过视频生成任务学习未来演化，但生成质量和规划最优性并不必然完全一致；
  - 在罕见交互、异常驾驶行为或突发事件中，模型可能仍受训练数据分布限制。
