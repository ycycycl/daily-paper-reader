## 摘要
在嵌入式系统上应用模型预测控制仍然具有挑战性，因为求解最优控制问题的计算成本很高。为解决这一局限，可以采用对隐式模型预测控制律进行计算高效的高斯过程近似。然而，对于轨迹跟踪应用而言，要在不同参考轨迹之间实现成功泛化，需要大量训练数据，这构成了显著挑战。为提高数据效率，我们提出将模型变换到围绕参考轨迹的曲线坐标系中。其次，我们使用名义前馈分量，使高斯过程仅学习残差控制输入，从而使轨迹跟踪控制器的近似成为可行。为强调该方法的适用性，我们将控制器部署在小型车辆的 Raspberry Pi 上，并进行了实验验证。与采用实时迭代的模型预测控制实现相比，基于高斯过程的近似方法计算控制输入的速度约快五倍，同时实现了相近的闭环跟踪性能。

## Abstract
Applying model predictive control on embedded systems remains challenging due to the high computational cost of solving optimal control problems. To address this limitation, computationally efficient Gaussian process approximations of the implicit model predictive control law can be employed. However, for trajectory-tracking applications, the large amount of training data required for successful generalization across distinct reference trajectories poses a significant challenge. To improve data efficiency, we propose to transform the model into curvilinear coordinates around the reference trajectory. Secondly, we use a nominal feedforward component, allowing the Gaussian process to learn only the residual control input, making the approximation of a trajectory-tracking controller feasible. To underline the applicability of the approach, we deploy the controller on a Raspberry Pi in a small-scale vehicle and validate it experimentally. Compared to a model predictive control implementation using real-time iterations, the Gaussian process based approximation computes control inputs about five times faster while achieving similar closed-loop tracking performance.

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 MPC 轨迹跟踪问题建模

- 系统被描述为离散非线性动力学：

  \[
  x_{k+1}=f(x_k,u_k,\eta_k)
  \]

  其中：
  - \(x_k\)：系统状态；
  - \(u_k\)：控制输入；
  - \(\eta_k\)：可能随时间变化的模型参数。

- MPC 在每个时间步求解一个有限时域最优控制问题：

  \[
  \min_{\{\hat u_i\}_{i=0}^{N-1}}
  \sum_{i=0}^{N-1}
  \ell(\hat x_i,\hat u_i,x^{ref}_{k+i},u^{ref}_{k+i})
  + E(\hat x_N,x^{ref}_{k+N})
  \]

  约束包括：
  - 系统动力学；
  - 输入/状态约束；
  - 终端约束；
  - 初始状态等于当前状态。

- MPC 控制律可理解为一个隐式映射：

  \[
  \mu(x,\Theta)=\hat u_0^\*
  \]

  其中 \(\Theta\) 包含未来参考状态、参考输入和模型参数等。

- 直接近似该映射的困难在于：
  - 对于轨迹跟踪，\(\Theta\) 包含整段未来参考轨迹；
  - 若预测时域为 \(N\)，状态维度为 \(n_x\)，输入维度为 \(n_u\)，参数维度为 \(n_\eta\)，则参数维度大致为：

    \[
    n_\Theta=(N+1)n_x+N(n_u+n_\eta)
    \]

  - 这会导致高维输入空间，使 GP 或神经网络都需要大量训练数据。

---

### 2.2 高斯过程近似 MPC 控制律

- 本文使用高斯过程回归近似 MPC 控制律。
- 对于数据集：

  \[
  D=\{(\xi_i,y_i)\mid i=1,\dots,n_D\}
  \]

  其中：
  - \(\xi_i\)：输入特征；
  - \(y_i\)：标签，即 MPC 求得的控制输入或残差输入。

- GP 先验形式为：

  \[
  y_i=g(\xi_i)+\epsilon,\quad \epsilon\sim \mathcal N(0,\sigma^2)
  \]

  \[
  g(\xi)\sim GP(m(\xi),k(\xi,\xi'))
  \]

- 论文使用：
  - 零均值先验 \(m(\xi)=0\)；
  - 神经网络核函数；
  - 通过最大化对数边际似然确定超参数。

- GP 后验均值为：

  \[
  m^+(\xi^\*)=k_\*^\top(K+\sigma^2I)^{-1}Y
  \]

- 为降低在线计算复杂度，论文离线预计算：

  \[
  \alpha=(K+\sigma^2I)^{-1}Y
  \]

  因此在线预测变为：

  \[
  m^+(\xi^\*)=k_\*^\top\alpha
  \]

- 在线复杂度与训练点数量线性相关：

  \[
  O(n_D)
  \]

  因此选择合适数量和位置的数据点对于嵌入式实时运行非常关键。

### 2.3 曲线坐标系下的车辆模型重构

- 原始车辆模型采用运动学自行车模型，状态包括：
  - 平面位置 \(p_x,p_y\)；
  - 航向角 \(\psi\)；
  - 速度 \(v\)。

- 控制输入包括：
  - 速度命令 \(v_u\)；
  - 转向角 \(\delta\)。

- 原始模型为：

  \[
  \dot p_x=v\cos\psi
  \]

  \[
  \dot p_y=v\sin\psi
  \]

  \[
  \dot\psi=\frac{v\tan\delta}{L}
  \]

  \[
  \dot v=\frac{1}{T}(v_u-v)
  \]

- 直接在全局坐标系中描述参考轨迹，需要未来多个时刻的位置、航向和转向角信息，参数维度高。

- 为解决这一问题，论文将模型转换到参考轨迹周围的曲线坐标系。新状态为：

  \[
  x=[s,n,\alpha,v]^\top
  \]

  其中：
  - \(s\)：沿参考路径的弧长坐标；
  - \(n\)：车辆到参考路径的横向距离；
  - \(\alpha\)：车辆航向与参考路径切线方向的夹角；
  - \(v\)：车辆速度。

- 曲线坐标系动力学为：

  \[
  \dot{s}= \frac{v\cos\alpha}{1-n\kappa_c(s)}
  \]

  \[
  \dot n=v\sin\alpha
  \]

  \[
  \dot\alpha=\frac{v}{L}\tan\delta-\kappa_c(s)\dot s
  \]

  \[
  \dot v=\frac{1}{T}(v_u-v)
  \]

  其中 \(\kappa_c(s)\) 是参考路径在弧长 \(s\) 处的曲率。

- 该变换的关键意义：
  - 参考轨迹跟踪目标可以简化为：

    \[
    n^{ref}=0,\quad \alpha^{ref}=0,\quad v^{ref}=r_v
    \]

  - 原本需要完整未来参考轨迹描述的问题，被压缩为局部或短期曲率信息；
  - 论文中进一步将曲率信息截断为仅使用当前局部曲率，即 \(N_\kappa=0\)，最终 GP 输入特征为：

    \[
    \xi=[n,\alpha,v,\kappa_c]^\top
    \]

  - 这显著降低了近似控制律的输入维度。

### 2.4 前馈控制与残差学习

- 论文没有让 GP 直接学习完整控制输入，而是引入一个解析的名义前馈控制项：

  \[
  u_{ff}(\kappa_c)=
  \begin{bmatrix}
  r_v\\
  \arctan(L\kappa_c)
  \end{bmatrix}
  \]

- 该前馈项表示车辆在给定路径曲率下保持在参考路径上所需的理想速度和转向角。

- GP 学习的是 MPC 最优控制输入与前馈输入之间的残差：

  \[
  \Delta u=\hat u_0^\*(x,\kappa_c)-u_{ff}(\kappa_c)
  \]

- 在线控制输入重构为：

  \[
  \hat u=u_{ff}(\kappa_c)+\bar m^+(\xi)
  \]

  其中 \(\bar m^+(\xi)\) 是 GP 预测的残差控制输入。

- 由于控制输入有两个维度，论文为每个控制输入维度分别训练一个 GP。

### 2.5 训练数据选择策略

- GP 在线计算复杂度与数据点数量 \(n_D\) 成线性关系，因此需要控制训练集规模。
- 论文采用一种贪婪采样策略：
  1. 首先定义可能的初始条件集合 \(\Xi\)；
  2. 使用采样方法近似闭环可达集；
  3. 在可达区域内寻找当前 GP 误差较大的点；
  4. 将这些点加入训练集。

- 贪婪选择的目标近似为：

  \[
  \xi^\*=\arg\max_{\xi\in\Xi} y(\xi)-\bar m^+(\xi)
  \]

- 由于 \(y(\xi)\) 需要通过求解 MPC 得到，精确求解较难，因此论文通过采样近似求解该选择问题。

    - MPC 控制器：

      \[
      J_{cl}=0.0081
      \]

  - 说明 GP 近似控制器在明显降低计算量的同时，仅带来较小性能损失。

## 7. 优点与亮点

- **问题切入实际**：
  - 面向嵌入式自动驾驶车辆控制，具有明确工程应用价值。

- **曲线坐标建模有效降低维度**：
  - 这是本文最关键的设计之一；
  - 它将复杂的轨迹跟踪参考信息压缩为曲率信息，缓解了近似 MPC 在轨迹跟踪任务中的高维泛化问题。

- **前馈 + 残差学习设计合理**：
  - 前馈项保留了路径曲率与理想转向之间的解析结构；
  - GP 只学习补偿项，而不是从零学习完整控制律；
  - 有助于提升样本效率和泛化能力。

- **GP 的可解释与扩展潜力**：
  - 虽然本文主要使用 GP 后验均值，但 GP 还提供后验方差；
  - 未来可用于不确定性评估、安全约束、在线更新或分层规划。

- **真实硬件验证有说服力**：
  - 控制器部署到 Raspberry Pi；
  - 在小型车辆上完成闭环实验；
  - 证明方法不只是仿真可行，而具备嵌入式落地潜力。

- **benchmark 选择较强**：
  - ACADOS RTI 是高性能 MPC 实现工具；
  - 能相对 ACADOS 获得约 5 倍加速，说明方法在计算效率上有实际优势。

## 8. 不足与局限

- **实验场景较窄**：
  - 仅展示了单一小型轨道上的实验；
  - 尚不能充分证明方法在复杂城市道路、高速驾驶、急转弯、多障碍环境中的泛化能力。

- **参考速度设定较简单**：
  - 实验中参考速度固定为 \(0.5\,m/s\)；
  - 未验证变速轨迹跟踪、加减速工况或更高速度下的性能。

- **曲率信息使用过于简化**：
  - 最终 GP 输入只使用当前局部曲率，即 \(N_\kappa=0\)；
  - 虽然降低了维度，但在曲率快速变化或需要提前预判的路径中，可能不足以近似真正 MPC 的前瞻行为。

- **安全与约束保证不足**：
  - 近似控制器不再显式在线求解约束优化问题；
  - 论文没有给出 GP 近似误差下的递归可行性、稳定性或输入约束严格满足保证；
  - 对安全关键自动驾驶系统而言，这仍是重要缺口。

- **训练数据生成依赖 MPC 求解**：
  - 离线训练需要反复求解 MPC，生成标签数据；
  - 若车辆模型、代价函数、约束或任务发生变化，可能需要重新生成数据或重新训练。

- **算力与训练成本报告不完整**：
  - 未说明离线训练耗时、数据生成耗时、GP 超参数优化耗时；
  - 不利于全面评估方法部署成本。

- **对比范围有限**：
  - 没有与神经网络近似 MPC、稀疏 GP、显式 MPC、其他数据驱动控制器对比；
  - 因此难以判断 GP 相对其他近似器的综合优势。

- **真实实验统计不足**：
  - 缺少多次重复实验结果；
  - 未报告方差、置信区间或失败率；
  - 对鲁棒性和长期稳定性的评估有限。
